## Summary
Unit tests validating that steps dictionary format is correctly converted to scenarios list in evidence_utils.py without duplication or data loss. Tests cover error detail preservation, invalid data type handling, and scenarios list passthrough.

## Key Claims
- **create_evidence_bundle**: Converts steps dict to scenarios list without creating duplicate entries
- **Error Detail Preservation**: Error messages from failed steps are captured in evidence markdown
- **Invalid Data Type Handling**: Gracefully handles invalid step data types (e.g., strings instead of dicts)
- **Scenarios List Passthrough**: Direct scenarios list format passes through unchanged
- **No Duplication**: 3-step dict produces exactly 3 scenario entries (not 6 duplicates)

## Key Test Cases
- `test_steps_to_scenarios_no_duplicates`: Verifies 3 steps produce exactly 3 scenarios
- `test_steps_to_scenarios_with_error_details`: Verifies error messages are preserved
- `test_steps_invalid_data_type`: Verifies invalid data types handled gracefully
- `test_scenarios_list_format_unchanged`: Verifies list format passes through
