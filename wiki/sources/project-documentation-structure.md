## Summary
Comprehensive documentation guide covering the docs/ directory structure, including core documentation, Architectural Decision Records (ADR), feature development evidence, process documentation, and technical specifications. Defines documentation standards, common patterns, integration with development workflow, and maintenance practices.

## Key Claims

- **Documentation Structure**: Organized into core docs, ADR, feature evidence, process docs, and technical specs
- **Documentation Standards**: Clarity, evidence-based content, actionable steps, version control, cross-references
- **Feature Documentation Pattern**: requirements.md, implementation.md, testing_evidence/, performance_metrics.md, lessons_learned.md
- **Documentation Lifecycle**: Creation → Updates → Review → Archival
- **Quality Assurance**: Link validation, screenshot updates, code example testing, coverage tracking

## Key Quotes

> "Write for developers who are new to the project"

> "Evidence-Based - Include screenshots, logs, and concrete examples"

> "All new features require user-facing documentation"
