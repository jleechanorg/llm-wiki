## Summary

Analysis of four open PRs (#6262, #6263, #6264, #6268) attempting to fix level-up atomicity and rewards_box normalization, plus the v3 architecture design that supersedes all four. Key finding: all four PRs share the same root bugs (off-by-one in stuck-completion, wrong dict keys, dead ensure_level_up_planning_block) and duplicate each other (#6263 and #6264 add identical functions). PR #6265 (streaming passthrough normalization) was merged separately and is preserved.

## Key Claims

- All 4 PRs have identical off-by-one bug: `target_level = current_level + 1` when `level_up_complete=True` — wrong because level is already incremented
- All 4 PRs use wrong dict keys: `"target_level"` / `"level"` instead of `"resolved_target_level"` / `"stored_level"`
- `ensure_level_up_planning_block` (~110 lines) is dead code — never called from any production path
- PR #6264 introduces production TypeError: `inject_persisted_living_world_fallback` signature change breaks caller in main.py
- #6263 and #6264 add identical `ensure_level_up_rewards_box` + `ensure_level_up_planning_block` functions independently
- #6268 patches code that #6264 deletes — merge conflict guaranteed

## Architecture Decision

**Design v3** (branch `feat/rewards-engine-single-responsibility`) supersedes all 4 PRs:

**Files and single responsibilities:**
- `llm_parser.py` (renamed from streaming_orchestrator.py) — handles BOTH streaming AND non-streaming, single entry point
- `game_state.py` — runs FIRST: computes LevelProgressionState, pure D&D 5e math + Firestore I/O
- `rewards_engine.py` — runs SECOND: builds canonical rewards_box + planning_block, normalizes types, decides should_show(). IDEMPOTENT for DeferredRewardsProtocol
- `world_logic.py` — THIN modal wrapper ONLY: receives pre-computed payload, does NOT call back into game_state or rewards_engine

**Flow (no file twice):**
```
llm_parser → game_state → rewards_engine → world_logic → persist → frontend
```

**Key constraints from wiki research:**
- DeferredRewardsProtocol calls rewards_engine every 10 turns — must be idempotent
- DNC (DefensiveNumericConversion) pattern for malformed LLM output
- ASI class-specific levels: Fighter@6, Rogue@10, not just 4/8/12/14/16/19
- Multi-level-up iteration: one level at a time, frontend handles sequencing
- Architectural tests enforce world_logic layer constraint
- Property-based tests for off-by-one + XP ranges 0-355000
