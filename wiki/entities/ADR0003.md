## Summary
Architecture Decision Record specifying JSON Schema validation for GameState. This ADR establishes the canonical schema and validation infrastructure for runtime game state validation.

## Context
The project needed a standardized way to validate game state at runtime against a canonical schema, ensuring data integrity and enabling backward compatibility handling.

## Decision
Implement JSON Schema validation infrastructure with:
- Schema loading and caching
- Custom format checking (RFC 3339 datetime)
- Legacy field aliases for backward compatibility
- Validation message enrichment for user-actionable errors

## Status
Phase 2 implementation complete with the game_state_schema.py module.
