## Overview

Stream Governance (Confluent) has three pillars: Stream Lineage (graphical UI for event stream mapping and impact analysis), Stream Catalog (RBAC-tagged schema contexts), and Stream Quality (data rules controlling what gets in).

## Key Properties

- **Stream Lineage**: Graphical UI for event stream mapping and impact analysis
- **Stream Catalog**: Schema contexts grouped per environment with RBAC tagging
- **Stream Quality**: Data rules and integrity controls — "what data gets in and what does not"
- **Quote**: "Stream governance enables data in motion, giving real-time data governance tools built to help foster secure, compliant, and scalable collaboration"

## Connection to Governance

Stream governance is directly relevant to AO's governance layer — it shows how to provide observability (lineage), organization (catalog), and enforcement (quality) for data flows. Apply the same three-pillar model to PR governance.
