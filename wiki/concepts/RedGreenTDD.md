## Summary
Test-driven development methodology where tests are written before implementation. "Red" phase: write failing tests. "Green" phase: implement code to pass tests. "Refactor" phase: improve code while keeping tests passing. Used in the sophisticated wizard test to validate campaign reset workflow.

## Key Principles
- Tests first, implementation second
- Fail fast, iterate quickly
- Continuous validation of behavior
- Regression prevention
