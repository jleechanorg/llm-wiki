UI Verification Testing is an E2E testing approach that uses browser automation to verify UI elements display correctly. The wiki contains several tests using this methodology:

- **Playwright-based**: [[Simple UI Check Test]], [[UI Verification Screenshots Test]]
- **Screenshot verification**: Captures visual state at key interaction points
- **Element-level checks**: Validates specific DOM elements (.session-header, .dice-rolls, etc.)

This testing approach complements unit tests by verifying the full user-facing behavior across the campaign flow.
