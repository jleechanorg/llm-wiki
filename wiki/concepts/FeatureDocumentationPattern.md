## Definition
A standardized structure for documenting features that ensures comprehensive coverage of requirements, implementation, testing, and lessons learned.

## Standard Structure
```
feature_name_evidence/
├── requirements.md         # Original requirements
├── implementation.md       # Technical implementation details
├── testing_evidence/      # Screenshots and test results
├── performance_metrics.md  # Performance analysis
└── lessons_learned.md     # Development insights
```

## Benefits
- Consistent documentation across features
- Easy to locate specific information
- Supports onboarding and debugging
- Enables feature comparison and analysis
