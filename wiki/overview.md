**JSON Schema Non-Compliance** — LLM failing to produce valid JSON at concerning rates: 50% raw failure in `schema_validation_real_api` (5/10 scenarios) and 100% in `schema_validation_extended` (5/5 scenarios). Root cause: `[Mode: STORY MODE]` prefix causing parser to see array instead of object. The JSON recovery logic incorrectly treated text prefix as JSON array bracket. Fix applied: regex pattern `r"^\[Mode:\s*[^\]]+\]\s*"` now strips mode prefixes before bracket detection. Post-processing achieves 100% pass rate but masks significant raw LLM failures — suggests workarounds rather than addressing root cause in LLM prompting.
