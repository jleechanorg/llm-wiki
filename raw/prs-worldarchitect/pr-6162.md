## Background
Global `time_events` and `faction_updates` were being incorrectly filtered out during scene changes because the logic treated them as scene-scoped. Additionally, sparse LLM payloads for `world_events` would clobber existing global markers during the backfill process.

## Goals
- Ensure global time-sensitive events (e.g., plagues, long-term countdowns) and faction updates persist across all scenes.
- Harden the `world_events` backfill mechanism to robustly handle partially populated payloads from the LLM.
- Prevent regressions in Living World trigger cadence on turn 0.

## High-level Description
- **world_logic.py**:
  - Modified `_filter_story_entry_world_events_to_scene` to exclude `time_events` and `faction_updates` from scene-based filtering.
  - Hardened `_try_backfill_story_entry_world_events` to merge missing global keys into existing `world_events` objects instead of skipping backfill when a partial object is present.
- **game_state.py**:
  - Updated `last_living_world_turn` reset logic to allow `player_turn >= 0`, preventing incorrect resets on the first turn.
- **E2E Testing**:
  - Introduced `test_time_events_scene_filter_e2e.py` which verifies that a plague event injected in Scene 1 remains present in Scene 3 after a region change, while local rumors are correctly filtered.

## Testing
- Automated E2E test: `cd testing_mcp && ../vpython test_time_events_scene_filter_e2e.py` (PASS)
- Local manual verification of state-updates showing backfilled events for sparse LLM turns.
