## Background
PR #6170 incorrectly documented that `gemini-3.1-flash-lite-preview` could not combine code execution with structured JSON output. That exclusion caused this model to miss the same `MODELS_WITH_CODE_EXECUTION` path used by other Gemini 3 models.

## Goals
- Add `gemini-3.1-flash-lite-preview` to `MODELS_WITH_CODE_EXECUTION` when official Google documentation supports the capability.
- Remove the stale comment claiming the model does not support the combination.
- Keep the change scoped to model capability configuration.

## Tenets
- Prefer official provider documentation for capability gating.
- Avoid runtime logic changes for a constants-only correction.
- Keep the PR small enough for fast review and low regression risk.

## High-level description of changes
This PR updates `mvp_site/constants.py` so `gemini-3.1-flash-lite-preview` is treated like the other Gemini 3 models that support code execution with structured output. It also removes the obsolete exclusion note and adds a short inline comment documenting the April 2026 Google AI docs confirmation.
