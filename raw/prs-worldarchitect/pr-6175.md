## Raw Body
### Background
The repo-local `.claude/` tree had drifted from the active user-scope `~/.claude/` tree across hooks, command docs, settings, and skills. That drift mattered because the recent context-bloat fix for `PostToolUse` edit/write payloads landed in user scope first, while the repo copy still carried the older hook behavior. The goal of this PR is to bring the repo `.claude/` tree back in line with the current user-scope versions for files that exist in both places.

### Goals
1. Sync repo-local `.claude/` files with the current user-scope versions where both copies exist.
2. Bring the `command_output_trimmer.py` hook compaction fix and its regression tests into the repo.
3. Keep user scope and repo scope aligned after the sync.
4. Preserve a reviewable, repo-tracked snapshot of the currently active Claude command/hook/skill configuration.
5. Add the PR design artifact required by the repo's skeptic/design-governance rules.

### Tenets
- Use the user-scope `~/.claude/` files as the source of truth for shared files.
- Avoid partial syncs for files that intentionally exist in both places.
- Verify both behavioral tests and byte-level parity instead of assuming a copy succeeded.
- Keep repo history clean by landing the sync from a branch based on `origin/main` only.
- Make governance artifacts explicit when the repo expects them rather than leaving them implicit.

### High-level description of changes
This PR syncs the repo `.claude/` tree to the current user-scope versions for every overlapping file that differed at the time of the comparison. The most important behavior change is in the hook layer: `command_output_trimmer.py` now recognizes `PostToolUse` payloads for `Edit`, `Write`, and `MultiEdit` and replaces oversized raw `tool_response` blobs with compact structured summaries.
