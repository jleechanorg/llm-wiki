> **Overview**
> Skeptic automation is refactored into a two-phase flow: GitHub Actions now **only** pre-checks 6-green eligibility and posts a trigger comment, while the AO worker runs `ao skeptic verify` and posts the `VERDICT` back to the PR.
> 
> `s.keptic-cron.yml` now filters open PRs (including a docs-only skip) through gates 1–5 before posting `SKEPTIC_CRON_TRIGGER`, then merges only when a matching, SHA-marked `VERDICT: PASS` comment exists; merging is hardened with a head-SHA recheck and `gh pr merge --admin --match-head-commit`.
> 
> `s.keptic-gate.yml` adds `workflow_dispatch`, disables cancel-in-progress, adds required read permissions, performs the same gates 1–5 precheck, posts a SHA-marked trigger with a server `updated_at` cutoff, and polls comments with pagination-safe jq and exit-code-based error handling.
