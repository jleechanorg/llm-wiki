## Summary
- Skeptic gate was reporting false PASS for PRs where Codex quota was exhausted
- Root cause: Codex session log echoes the full prompt, which contains `VERDICT: PASS — [one sentence...]` as a template example. The `grep -qi "VERDICT: PASS"` on the full comment body matched this template text

## Root cause chain
1. Codex fails (quota/error) → `execFileSync` throws with stdout embedded in `err.message`
2. Codex stdout = session header + full prompt echo (contains `VERDICT: PASS` template text)
3. Verdict comment body includes this full error text
4. `grep -qi "VERDICT: PASS"` on full body → false PASS → Skeptic Gate reports SUCCESS

## Companion fix
`jleechanorg/agent-orchestrator` llm-eval.ts also patched to truncate Codex error to first line only (belt-and-suspenders: prevents the template text from reaching the comment body at all).

> **Overview**
> Prevents `Skeptic Gate` from reporting false PASS when the verdict comment body contains prompt/session logs that include the template text `VERDICT: PASS`.
> 
> The workflow now extracts only the header portion of the verdict comment (up to `--- Full skeptic output ---`, capped to the first 10 lines) and searches that header for `VERDICT: PASS/FAIL/SKIPPED` instead of grepping the entire comment body.
