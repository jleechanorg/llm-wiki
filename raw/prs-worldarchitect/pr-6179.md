## Summary

- inject_persisted_living_world_fallback was gated by debug_mode=True, preventing non-debug users from seeing living world updates after a page reload
- The frontend comment explicitly says living world is always visible when data is present (not debug-gated) — this was a backend bug
- Remove the debug_mode parameter entirely (was unused after fix) and update all callers

## Production Code Changes

Before: if not debug_mode or not story_entries ... — skipped fallback for all non-debug users

Now: if not story_entries ... — fallback runs for all users

Why: On page reload, inject_persisted_living_world_fallback backfills living world data from game_state.world_events into story entries that lack per-turn world events. Gating on debug_mode meant non-debug users always saw empty living world panels after reload.

## Test Changes

- Removed debug_mode kwarg from all 20 test call sites
