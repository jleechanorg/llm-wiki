## Background

On current main, the game UI does not render **dice roll entries** or **debug messages** (debug_info) even when the user has `debug_mode=true` in their settings. On the mvp-stable production deployment (~Mar 21 2026, commit `02a6e5fb3`), both features work correctly. This is a regression window of approximately 3 weeks.

Critically, **the user reports these symptoms predate PR #6161** (merged Apr 11 as commit `d868fec0c`). That means PR #6161 is NOT the root cause for dice/debug — it IS the root cause for the separately-reported rewards_box regression, which is addressed in the parallel PR #6193.

## Goals

- [x] Rule out frontend as the cause (verified: render code identical stable ↔ main)
- [x] Rule out PR #6161 as the cause (scope limited to rewards_box)
- [ ] Identify which backend commit(s) in the window stopped emitting `dice_rolls` / `debug_info` inside `structured_response`
- [ ] Perform runtime reproduction to confirm the suspected commit
- [ ] Open a follow-up fix PR

## Tenets

- This is an **investigation-only** PR. It contains no production code changes and is not intended to merge until root cause is identified.
- Frontend was ruled out before backend archaeology to avoid wasted speculative fixes.
- Dice/debug are tracked as a separate regression from rewards_box — coupling them risks conflating two root causes.
