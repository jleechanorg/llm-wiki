## Background

While fixing PR #6161 (rewards_box atomicity), the browser GIF in the PR description was broken — it linked to a release asset on the **private** \`worldarchitect.ai\` repo, which returns 404 for anonymous viewers and does not render as an inline image in GitHub PR descriptions. This exposed three harness gaps:

1. No rule that GIF/MP4 hosting must be on a **public** repo
2. No explicit "self-contained" requirement (a stranger on a clean computer should be able to reproduce)
3. The \`/er\` slash command carried enforcement rules directly, instead of treating the slash command as a thin alias pointing to a canonical skill

## Goals

1. Restore the alias pattern: \`/er\` and \`/es\` are thin wrappers; the real rules live in \`evidence-review\` and \`evidence-standards\` skills.
2. Codify public-URL hosting as a mandatory evidence check.
3. Codify the "clean computer" rule: a reviewer with no prior context can follow the gist URL and reproduce from scratch.

## Tenets

- Slash commands point to skills. Skills own the rules. Don't duplicate enforcement content across both.
- Evidence that's only readable by repo members isn't evidence — it's an in-group signal.
- Video hosting failure mode (private repo release → 404 → broken inline image) is a recurring class; encode the fix in the standards skill, not a one-off workaround.

- \`.claude/skills/evidence-review.md\` — new canonical skill for the review side: PASS/PARTIAL/FAIL rubric, six mandatory pre-PASS checks, claim-to-artifact mapping procedure, verdict table template, anti-pattern list
