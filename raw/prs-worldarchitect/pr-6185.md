## Raw Body
### Background

PR #6185 fixes a skeptic-cron Gate-5 blocker found while trying to land the heavily reviewed PR #6161. Gate 5 checks unresolved inline review threads before skeptic verification and merge consideration, but the workflow only queried the first 100 review threads. PRs with more than 100 threads could remain permanently blocked even when every actionable thread was resolved.

Related context:
- PR #6161: https://github.com/jleechanorg/worldarchitect.ai/pull/6161

### Goals

- Evaluate all GitHub review threads for Gate 5 instead of only the first page.
- Preserve fail-closed behavior when GitHub GraphQL responses are empty, malformed, partial, or contain top-level errors.
- Avoid counting PR-author comments as unresolved reviewer blockers.
- Keep evidence for the workflow change durable and reviewer-accessible.

### Tenets

- LLM decides, server executes: keep the workflow as deterministic gate execution, not heuristic intent logic.
- Prefer fail-closed merge gates over silent pass-through on API uncertainty.
- Keep the fix scoped to skeptic-cron Gate 5 and its evidence trail.
- Do not hide review evidence behind local /tmp paths.

### High-level description of changes

This PR updates .github/workflows/skeptic-cron.yml so both Gate-5 call sites paginate pullRequest.reviewThreads with cursors, accumulate unresolved non-author, non-nit inline comments across all pages, and continue until GitHub reports no next page.

The workflow now fails Gate 5 when GraphQL returns an empty response, a top-level errors field, missing reviewThreads.pageInfo, or thread-level comment pagination that would make the inspected comment set incomplete. The PR-author filter was also hardened so the trigger path uses the already-fetched PR payload and fails closed if the author is missing, while the merge path reuses the preserved PR_AUTHOR value instead of clobbering it
