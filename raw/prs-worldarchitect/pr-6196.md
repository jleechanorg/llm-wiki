## Background

User insisted "rewards_box was broken **before** PR #6161" and pointed at PR #6137 as the suspect. Deep subagent investigation confirmed: PR #6137 (`abd799c2e`, merged 2026-04-09 — two days before #6161) introduced the Dragon Knight template bypass in `mvp_site/world_logic.py:5012-5038` but silently omits `FIELD_REWARDS_BOX` from the extraction dict.

## Goals

- Restore `rewards_box` emission on the opening turn of Dragon Knight campaigns.
- Match the canonical response shape that non-template (real LLM) first-turn responses produce.

## Tenets

- Minimal change: one line added.
- Do not touch any other extraction logic or the normalizer (separate concern; already handled in PR #6193).
- No behavior change for non-Dragon-Knight campaigns.

## High-level description of changes

`mvp_site/world_logic.py:5012-5038` — Add `constants.FIELD_REWARDS_BOX: template_structured.get(constants.FIELD_REWARDS_BOX, {})` to the `fields` dict used when the Dragon Knight pre-generated template bypasses the LLM call. The template itself (`constants.py:1326`) already defines `"rewards_box": {}` — the extraction was just silently dropping it.
