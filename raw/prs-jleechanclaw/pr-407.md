## Summary

Add `skeptic-cron.yml` workflow to jleechanclaw — runs every 30 minutes
to evaluate open PRs and auto-merge when **7-green criteria** are met.

## 7-Green Criteria

| # | Gate | Check |
|---|------|-------|
| 1 | CI green | GH Actions all return `success` |
| 2 | No conflicts | `mergeable=true` |
| 3 | CR APPROVED | `coderabbitai[bot]` review is `APPROVED` |
| 4 | Bugbot clean | `cursor[bot]` zero error-severity comments |
| 5 | Comments resolved | Zero unresolved non-nit inline comments |
| 6 | Evidence pass | `evidence-review-bot` has `APPROVED` |
| 7 | Skeptic PASS | `VERDICT: PASS` from `github-actions[bot]` |

This matches the gate in `agent-orchestrator`'s skeptic-cron.yml.

> [!NOTE]
> **High Risk**
> Automatically merges PRs on a schedule using repo write permissions, so any logic/false-positive in the gating checks could result in unintended merges or branch deletions. Relies on `gh`/API signal heuristics (comments/reviews/mergeable) that may be flaky or delayed, increasing operational risk.
> 
> **Overview**
> Adds a new GitHub Actions workflow `skeptic-cron.yml` that runs every 30 minutes (or manually) to scan all open non-draft PRs and **auto-merge** those that meet a "7-green" gate.
> 
> The job queries GitHub via `gh`/`jq` to require: successful check-runs, `mergeable=true`, `coderabbitai[bot]` approval, zero `cursor[bot]` "error" comments, zero non-nit top-level review comments, `evidence-review-bot` approval, and a latest `github-actions[bot]` comment containing `VERDICT: PASS`; qualifying PRs are merged via `gh pr merge --squash --auto --delet
