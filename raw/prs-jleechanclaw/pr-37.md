## What This Does

The `openclaw-mcp-adapter` plugin lets the OpenClaw agent use any MCP server as a tool provider. It:
- Reads server definitions from `openclaw.plugin.json` (stdio or HTTP transports)
- Registers each server's tools into OpenClaw under namespaced IDs (`<server>_<tool>`)
- Proxies `callTool` calls to the correct downstream server
- Hot-reloads config on gateway restart via `parseConfig()` in `start()`

Slack MCP is the first server wired in: the `slack-mcp` binary connects via stdio with an XOXB token, giving the agent read/write access to Slack.
