## Background

AO currently has 7+ separate launchd plists (lifecycle-agent-orchestrator, orchestrators, dashboard, notifier, etc.). Individual services silently die and there's no single command to manage the full stack.

## Goals

1. Single launchd plist (`ai.agento.manager`) manages all AO components
2. Manager reads project list dynamically from `agent-orchestrator.yaml` — no hardcoded project list
3. Monitor loop restarts crashed components automatically
4. One command to start/stop/restart the entire AO stack

## Tenets

- Prefer dynamic discovery over hardcoded values (session prefixes derived from YAML, tmux sessions discovered at runtime)
- Keep the launchd plist dumb (just runs the manager script); all logic in the script
- Do not delete existing plists yet — new manager runs alongside them

## Changes

### `scripts/ao-manager.sh`
Core manager. Reads all projects from `agent-orchestrator.yaml`, starts:
- lifecycle-workers (one per project via `ao lifecycle-worker`)
- orchestrator tmux sessions (one per project via `ao start`)
- notifier (via `ao-manager-notifier.sh`)
- dashboard (started by first project via `ao start`)
- monitor loop: checks all components every 60s, restarts any that died

Key behaviors:
- `--once`: start everything once without the monitor loop (for testing)
- `--status`: print health of all components
- Dynamic session discovery via `tmux list-sessions` — no hardcoded session names

### `scripts/ao-manager-notifier.sh`
Wrapper script. Sources env vars then execs `agento-notifier.py`. Called by `ao-manager.sh`.
