## Background
`~/.openclaw/scripts/dropped-thread-followup.sh` produces spurious stderr under launchd:
- `mkdir: command not found` on line 32
- `echo: write error: Broken pipe` on line 128 (SIGPIPE from python3 heredoc piping)
- `ne: command not found` on line 369 (corrupt cached version symptom)

Additionally, the launchd plist had `StartInterval: 300` (5 min) instead of the 4-hour cadence documented in the script header.

## Goals
- Eliminate stderr errors from launchd runs
- Correct run interval to 4 hours

## Changes
- **scripts/dropped-thread-followup.sh**: Add `export PATH=...` at top of script (before `set -euo pipefail`) to guarantee basic commands in restricted launchd environments; suppress SIGPIPE stderr on hours_old heredoc
- **launchd/ai.openclaw.schedule.dropped-thread-followup.plist**: Update `StartInterval` from `1800` → `14400` (4 hours, matching documented intent); live plist also updated via `plutil`

## Testing
- Script tested manually with `bash -x` (clean exit, no stderr)
- Verified launchd plist updated: `plutil` confirms `StartInterval = 14400`
- Live plist reloaded via launchctl bootstrap

## Notes
- The `ne: command not found` and `mkdir: command not found` errors appear to come from a cached/old version of the script held by launchd (the current file on disk is correct per MD5 = HEAD). PATH export guards against recurrence.
