## Summary
- Fix `install-launchagents.sh` to add `OPENCLAW_GATEWAY_TOKEN` to launchd EnvironmentVariables after `openclaw gateway install`

## Root Cause
`openclaw gateway install --token` stores the token in config but doesn't add it to launchd's `EnvironmentVariables` dict. Launchd services need env vars explicitly defined in the plist to survive restarts.

## Fix
After running `openclaw gateway install`, explicitly add the token via `plutil -insert` and restart the service.

## Testing
- Gateway now healthy at http://127.0.0.1:18789/health
- Token persists across launchd restarts
