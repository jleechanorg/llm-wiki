## What this does

Establishes a shared memory layer so openclaw, Claude Code, and Codex all read/write the same qdrant collection (`openclaw_mem0`), scoped by `agent_id`. Fixes the 24-hour amnesia problem where each agent starts fresh and cannot benefit from what the others have learned.

This PR is **design + scaffolding**. Implementation beads are tracked and ready to execute.

## Problem

- Three agents (openclaw, Claude Code, Codex) have isolated memories — no cross-agent recall
- openclaw session history contains ORCH->branch mappings that Claude and Codex cannot access
- No sleep-time compute: facts stay buried in raw session JSONLs, never extracted into searchable form
- mem0 default filter silently excludes untagged points (P0 bug: must_not+True not must+False)

## Changes

### Design (roadmap/SHARED_MEM0_ARCHITECTURE.md)
- Single qdrant collection shared by all agents, scoped via user_id + agent_id
- Two-layer architecture: L1 = orch-branch-index.md (fast lookups), L2 = qdrant (semantic search)
- Hourly launchd job extracts facts from session JSONLs -> LLM extraction via claude-haiku-4-5 (~$0.10/day) -> atomic facts in qdrant
- Session-end trigger in supervisor.py: fire-and-forget subprocess on every session close
- P0 filter fix documented: use must_not:[{is_legacy:true}] not must:[{is_legacy:false}]
- batch_id (ISO timestamp) on every m.add() for rollback capability
- Double-extraction guard via extraction-state.json keyed by (path, mtime)
- Embedding model locked: text-embedding-3-small (changing it breaks existing vectors)
- Extraction LLM: claude-haiku-4-5-20251001
- Goals-first memory schema: goal -> objective -> decision policies -> evidence
- Superpowers roadmap: TTL/confidence decay, contradiction detection, cross-agent memory voting, proactive injection
