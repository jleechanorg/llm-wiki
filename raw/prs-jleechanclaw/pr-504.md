## Background

Steve Yegge's Zero-Framework Cognition (ZFC) principle: applications should be thin deterministic shells that delegate ALL judgment, classification, routing, and ranking to AI model calls — never implement heuristics, keyword matching, semantic scoring, or routing logic in application code.

A search of all local branches, PRs, CLAUDE.md, AGENTS.md, and openclaw config found ZERO enforcement of ZFC existed anywhere. This PR adds it to the AO worker agentRules in agent-orchestrator.yaml.

## Goals

- Enforce ZFC at the AO worker prompt level via agentRules
- Ensure all workers spawned by the Agent Orchestrator are instructed never to write heuristic routing/classification code
- Provide a "ZFC violation!" shortcut phrase for workers to self-enforce

## Tenets

- Applications are thin shells; intelligence lives in model calls
- Forbidden: keyword matching for intent detection, regex-based routing, heuristic scoring
- Correct: pass context to model, use model response as decision
- Exemptions: pure syntax parsing, deterministic transforms, test fixtures

## High-level description of changes

Added a ZFC rule to the `agentRules` section of `agent-orchestrator.yaml`:

```
**Zero-Framework Cognition (ZFC):** Never implement routing, classification, scoring, or judgment logic in application code. Delegate all such decisions to model API calls. Forbidden: keyword matching for intent detection, regex-based routing, heuristic scoring. Correct: pass context to model, use model response as decision. Say "ZFC violation!" when catching yourself writing one. Exemptions: pure syntax parsing, deterministic transformation, test fixtures.
```

This rule is appended to the existing agentRules string, making it part of every AO worker's system prompt.
