> [!NOTE]
> **Medium Risk**
> Touches scheduling/automation and installer scripts that can affect backups/check-ins and launchd configuration on developer machines. Changes are mostly additive but could disrupt job execution if migration assumptions (IDs/timezone/tools) are wrong.
> 
> **Overview**
> Adds a new `./doctor.sh` / `scripts/doctor.sh` *end-to-end health audit* that validates repo↔`~/.openclaw` config sync, required binaries, launchd registration, gateway token/port consistency, and runtime `/health` + `openclaw gateway` probes, failing CI/local runs on detected drift.
> 
> Migrates *repo-managed recurring jobs* from OpenClaw in-app cron to macOS `launchd` by introducing `openclaw-config/ai.openclaw.schedule.*.plist` templates, a `run-scheduled-job.sh` runner, and an `install-openclaw-scheduled-jobs.sh` installer that loads plists and disables the corresponding IDs in `~/.openclaw/cron/jobs.json` (tracked via `migratedLaunchdJobIds`). Documentation and the cron guardrail are updated to require launchd for repo-m
