## Background

OpenClaw repeatedly made commitments ("I'll always do X going forward") that were forgotten next session. Root cause: there was a write-path for commitments (Promise Gate) but no read-path — nothing forced the agent to reload commitments at session start. Tracked in orch-7df.

## Goals

1. Agent enumerates and applies active commitments at the start of every session
2. Commitments must be trigger-based rules (not vague intentions) to be valid
3. Heartbeat audits that commitments were actually followed
4. Scannable `## COMMIT:` format makes commitments discoverable programmatically

## Tenets

- Rules without enforcement are theater — every commitment needs a read-path
- Trigger-based rules ("when X → do Y") are verifiable; intentions ("I'll try") are not
- Minimize LLM discretion on "did I follow my commitments?" — make it grep-able

## High-level description of changes

**`workspace/SOUL.md`** — three changes:
- Adds `## Session Initialization` protocol at top: agent silently scans for `## COMMIT:` sections and applies each one before handling any user task
- Tightens Promise Gate: step 2 now requires trigger-based format, step 3 requires `## COMMIT:` heading; vague intentions explicitly banned
- Migrates existing MCP Mail Ack Policy to `## COMMIT: mcp-mail-ack` with Trigger/Action/Added fields

**`workspace/HEARTBEAT.md`** (symlinked as `HEARTBEAT.md` at repo root):
- Adds `## Active Commitment Audit`: each heartbeat scans `workspace/SOUL.md` for `## COMMIT:` sections, confirms each was applied, reports explicitly if any was missed

**`skills/promise-gate.md`**:
- Documents the `## COMMIT:` format (Trigger/Action/Added)
- Updates how-to-use to require trigger-based rule and correct heading format

## Testing

Verification: start a new openclaw session after merge, ask "what are your active commitments?" — should enumerate `## COMMIT: mcp-mail-ack` without prompting. If it does, the session-init read-path is working.
