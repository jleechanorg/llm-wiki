## Background

Incident 1774380244.271829: a Slack thread task for `cmux list-surfaces --workspace 23 --json` produced a large stderr help dump (invalid `--json` flag) with no terminal in-thread status message. The thread went silent with no `done/blocked/needs-input` finalizer.

## Goals

- Every Slack thread task always emits a terminal status message (`done`/`blocked`/`needs-input`), even on tool errors.
- Cmux commands are validated before execution, with helpful suggestions for invalid subcommands/flags.
- Large stderr/help dumps are truncated for Slack display while still progressing to terminal status.
- Active sessions that go silent for >5 minutes receive an automated blocker/progress nudge.
- Slack token hydration falls back to `openclaw.json` when env vars are absent.

## Tenets

- Prefer real execution over mocks — cmux validator is unit-tested with 27 passing tests.
- No breaking changes to existing `human_channel_bridge.py` API surface.
- Terminal guarantee is fail-closed: always post exit status, never silently drop.

## Changes

### `src/orchestration/cmux_validator.py` (new)
Cmux CLI preflight validator. Parses commands via `shlex`, checks against allowlist of valid subcommands and flags, returns structured `CmuxValidationResult` with error + suggestion.

Key validations:
- `--json` only valid BEFORE the action (`cmux --json list-surfaces`), not after
- `list-surface` → suggests `list-surfaces`
- `tree` → suggests `cmux list-surfaces`
- `--workspace`/`-w` and `--surface`/`-s` are recognized as targeting args (not flagged as unknown)
- `truncate_output()` helper limits output to 20 lines / 2000 bytes for Slack
