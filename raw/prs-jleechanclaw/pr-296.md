## Background
PR review findings identified 4 portability blockers: install.sh doesn't install local code, launchd templates have hardcoded nvm/python paths, monitor defaults target specific Slack channels, and MCP mail invocation is inconsistent across modules.

## Goals
- Make `git clone && bash install.sh` work on a fresh machine without path assumptions
- Unify MCP mail CLI usage across all Python modules
- Rewrite README to be useful (quick start first, no operational noise)

## Tenets
- Portable by default — detect paths at install time, don't hardcode
- Single code path — one `send_mail()` function, not two implementations
- README serves the "clone on a new machine" use case first

## High-level description of changes
1. **install.sh** gains `pip install -e .` step so the local Python package is importable
2. Launchd templates use `@NODE_BIN_DIR@`, `@NODE_PATH@`, `@PYTHON3_PATH@` placeholders resolved at install time
3. `mcp_mail.py` gets a fallback chain (mcporter preferred, openclaw fallback); `action_executor.py` delegates to it
4. README rewritten from 464 to 161 lines
