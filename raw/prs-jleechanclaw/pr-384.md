## Background
The thread-reply-nudge LaunchAgent (every 2 min) is the fallback for unanswered thread messages when the live WebSocket misses them. It was hardcoded to a single channel (C0AKYEY48GM). Messages in C0AJ3SD5C79 and all other channels were silently skipped — root cause of the 2026-03-24 miss where Jeffrey message at 08:30 UTC went 1.5h unanswered.

## Goals
- Nudge checks all channels from openclaw.json, not just one hardcoded channel
- Env var override and legacy fallback preserved
- TDD: tests written red-first, implementation written to make them green

## Tenets
- Config-first: read channel list from existing openclaw.json
- Minimal change: only touch the nudge script and add tests

## Changes
resolve_nudge_channels() in thread-reply-nudge.sh now reads all explicit channel IDs from openclaw.json (excluding wildcard), with env var override and legacy fallback.

IS_SOURCED=1 guard for testability. DRY_RUN=1 / NUDGE_LOCK_DIR / NUDGE_LOG_DIR for isolated testing.

## Testing
6 TDD tests in tests/test_thread_reply_nudge.sh — all green:
1. Env var override (comma-separated)
2. All openclaw.json channels included
3. Wildcard excluded
4. Fallback when no json/env
5. DRY_RUN=1 prompt contains all channels
6. C0AJ3SD5C79 (previously missed) now covered

## Low-level details
- scripts/thread-reply-nudge.sh: extracted resolve_nudge_channels(), IS_SOURCED guard, DRY_RUN support, overrideable lock/log dirs
