## Raw Body
## Problem

OpenClaw was silently dropping all Slack ACK reactions (👀 `eyes` emoji) on inbound messages. Investigation found:

1. **`invalid_auth`** — Slack bot token expired, causing all API calls (including `reactions.add`) to fail
2. **Config misplacement** — `ackReaction`/`ackReactionScope` were placed inside `channels.slack` (wrong location), causing: `config reload skipped (invalid config): channels.slack: Unrecognized keys: "ackReaction", "ackReactionScope"`
3. **Gateway startup failure** — Missing `OPENCLAW_GATEWAY_TOKEN` env var preventing gateway from starting

## Fix

Adds 4 regression tests to `tests/test_openclaw_configs.py` to prevent the config-related regressions from recurrence:

### ORCH-ack1: ACK Reaction Config Guard
- `test_ack_reaction_is_set` — `messages.ackReaction` must be present and non-empty
- `test_ack_reaction_scope_is_set` — must be `"all"` or `"group-mentions"`
- `test_ack_keys_not_in_channels_slack` — **directly guards against the exact regression** that caused the outage

### ORCH-gw1: Gateway Token Config Guard
- `test_gateway_token_uses_env_var` — `gateway.auth.token` must use `${...}` env var placeholder, never hardcoded
