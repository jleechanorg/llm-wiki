## Summary

Implements the **CI gate** (Stage 3) of the 3-stage staging pipeline design from Slack #C0AJ3SD5C79 (orch-1ps epic).

- Adds `.github/workflows/staging-canary-gate.yml` — runs on every PR open/sync/reopen
- Runs **portable canary checks** (schema validation, crash-keys, SDK version) directly in GitHub Actions
- Posts a **PR comment table** with PASS/FAIL per check
- **Blocks merge** on any FAIL (exit 1)
- Documents the **manual gate requirement** for gateway-dependent checks (health, Slack connectivity, native module ABI, heartbeat latency) via `scripts/staging-canary.sh --port 18790`

## Background

The 3-stage staging pipeline was designed in Slack #C0AJ3SD5C79. Stage 3 (CI gate) was the highest-value remaining piece — preventing broken openclaw configs (crash keys, missing critical keys, incompatible SDK versions) from ever reaching production.

## What this PR changes

- **New file**: `.github/workflows/staging-canary-gate.yml`
  - Trigger: `pull_request` (open, synchronize, edited, reopened, ready_for_review)
  - Sparse-checkout: only `openclaw.json`, `.openclaw/openclaw.json`, `config/openclaw.json`, `package.json`, `package-lock.json` — minimal clone, no secrets
  - Checks run: config schema validation (crash-keys + critical keys) + SDK protocol version
  - Posts PR comment with markdown table of results (uses `always()` so comment appears even on FAIL)
  - Job exits non-zero on FAIL → blocks merge in `skeptic-cron` merge gate
