## Background

jleechanclaw was missing `skeptic-gate.yml` — the per-PR workflow that posts a VERDICT (PASS/FAIL) as `github-actions[bot]`. Without it, Gate 7 in `skeptic-cron.yml` could never transition to PASS for new PRs (only the cron itself could post PASS). Additionally, Gate 5 (unresolved comments) was too strict and Gate 6 (evidence-review-bot) was fail-closed on missing runs.

## Goals

1. Add `skeptic-gate.yml` so Gate 7 has a FAIL path for PRs that fail skeptic review
2. Bypass Gate 5 when CR has approved (CR supersedes stale comment threads)
3. Make Gate 6 evidence-review-bot optional (N/A pattern) so PRs without evidence bundles can still merge

## Tenets

- **Fail-closed on real evidence**: DISMISSED/CHANGES_REQUESTED evidence reviews still block merge
- **No breaking changes**: skeptic-cron merge logic is only relaxed, never tightened

## Changes

### `.github/workflows/skeptic-gate.yml` (new file)
- Trigger: `pull_request` (opened, synchronize, edited, reopened)
- Installs Python 3.12 + Claude CLI (`pip install claude-code`)
- Fetches PR diff via `gh pr diff`, pipes to `claude -p` with skeptic system prompt
- Posts findings as a PR comment regardless of outcome (`if: always()`)
- Job exit 1 on VERDICT: FAIL (blocks merge); exit 0 on VERDICT: PASS
- SKIPPED verdict when `ANTHROPIC_API_KEY` is not configured (no false failures)

### `.github/workflows/skeptic-cron.yml`

**Gate 5 (Comments resolved)**: Now bypasses when `CR_PASS == "true"`. A CodeRabbit APPROVED review supersedes unresolved nit comments — these represent stale threads where CR has since given overall approval.

**Gate 6 (Evidence review)**: Now uses N/A pattern matching AO's approach. The only blocking states are explicit failures (DISMISSED, CHANGES_REQUESTED). Missing, APPROVED, COMMENTED, PENDING all pass as N/A.
