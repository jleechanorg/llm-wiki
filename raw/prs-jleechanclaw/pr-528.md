## Background
A recent merge of PR #520 introduced a regression (flagged by Cursor Bugbot) where unexpanded environment variables in `openclaw.json` (e.g., `"apiKey": "$OPENAI_API_KEY"`) were treated as literal strings when the variable was unset. This caused `mem0_hooks_enabled()` to return `True`, leading to failed API calls rather than failing open.

## Goals
- Correctly handle unexpanded environment variables by treating them as empty strings.
- Prevent hooks from attempting API calls with invalid literal key strings.
- Support both uppercase and lowercase environment variable names.

## High-Level Description
- Added a `_expand_env_vars()` helper in `.claude/hooks/mem0_config.py` that uses a regex to detect unexpanded $\{\} or $ variables.
- Replaced raw `os.path.expandvars` calls with the new helper.
- Updated the regex to support mixed-case environment variable names.

## Testing
- Verified with local pytest: `PYTHONPATH=src python -m pytest src/tests/test_ollama_embedder.py src/tests/test_mem0_shared_client.py -q`
- Manually verified that `_expand_env_vars("$MISSING_VAR")` and `_expand_env_vars("$missing_var")` return `""`.
