## Background
The OpenClaw staging→production deployment pipeline is spread across multiple scripts (deploy.sh, staging-canary.sh, gateway-preflight.sh, install-launchagents.sh), launchd plist templates, and CLAUDE.md sections. A new machine setting up from scratch had no single place to reference — the knowledge was implicit in the code and incident history.

## Goals
Create a self-contained, portable skill that documents the complete staging→prod pipeline so any agent or human can install, operate, and troubleshoot it without reading source code.

## Tenets
- Portable — one skill file, no external dependencies beyond the repo itself
- Complete — every path, command, and exit code documented
- Honest about gaps — TODOs flagged for incomplete areas (Linux staging, etc.)
- Discovery-first — structured so someone landing on a fresh machine can follow steps 1-6 in order

## High-level description of changes
Created `.claude/skills/staging-prod-pipeline/SKILL.md` containing:
- ASCII architecture diagram (staging dir → deploy.sh → prod dir, ports, labels)
- Prerequisites table
- 6-step fresh-machine install procedure
- Full deploy.sh flow description with --skip-push and --prod-only flags
- 7-check canary verification with exit code meanings
- Rollback procedure (backup restore, consensus fix)
- Complete file path reference table
- Troubleshooting section (session locks, AJV overflow, native module ABI, plist encoding, throttle, duplicate plists)
- Maintenance section (upgrading through the staging gate)
- Launchd labels reference table
- TODOs for gaps found during analysis

- No mocks — all paths, ports, labels, exit codes verified against actual source
