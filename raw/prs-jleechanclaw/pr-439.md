## Summary
- Removes the bundled `extensions/openclaw-mem0/` from the jleechanclaw repo — the globally installed npm package (`@mem0/openclaw-mem0`) is already configured in `openclaw.json` and takes precedence, making the bundled copy redundant and causing duplicate plugin warnings
- Patches `mem0ai` to pass `checkCompatibility=false` to the Qdrant client, eliminating the persistent version warning on every mem0 command
- Updates `mem0-memory-operations.md` skill with full install/runbook, verification steps, and ops notes

## Changes

### `extensions/openclaw-mem0/` (deleted)
Bundled copy removed — globally npm-installed plugin at `~/.openclaw/extensions/openclaw-mem0` (same version 0.3.0) is configured in `openclaw.json` and overrides the bundled one anyway. Keeping both caused duplicate plugin ID warnings.

### `~/.openclaw/extensions/openclaw-mem0/node_modules/mem0ai/dist/oss/index.js` (patched)
One-line patch: add `checkCompatibility: false` to `QdrantClient` constructor params. Safe to re-patch after `npm update` if mem0ai is updated upstream.

### `.claude/skills/mem0-memory-operations.md`
- Install/runbook: Qdrant Docker, Ollama, `npm rebuild better-sqlite3`, mem0ai patch
- End-to-end verification steps
- Rules: rebuild native modules after Node upgrade, use npm package not bundled

- Backends: Qdrant 1.17.0, Ollama (nomic-embed-text + llama3.2:3b)
