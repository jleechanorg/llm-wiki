## Background

There are 8 false mem0/Qdrant memories referencing a non-existent person "Benjamin" — created by errant LLM sessions that invented a fictional human collaborator. These pollute semantic search (score ~0.7+) and need to be deleted. The memory_delete tool is not available in the current runtime, so a careful manual script is required.

## Goals
- Add `memory_store` and `memory_list` to the OpenClaw tool allowlist (already present — no config change needed)
- Provide a safe, auditable one-off deletion script for the 8 confirmed Benjamin IDs
- Document the runbook and safety model so future deletions follow the same discipline
- Track the one-off execution in beads (orch-rjn)

## Tenets
- Dry-run is the default — no deletion without `--confirm`
- All deletions are ID-allowlist-only (no glob, no pattern, no all())
- SHA256 confirmation hash required for live runs
- Post-run verification is mandatory

## Changes

### `scripts/mem0-purge.sh`
One-off memory deletion script with:
- **Dry-run default**: previews all IDs + memory text before any action
- **ID allowlist only**: file (`--ids-file`) or inline (`--ids-inline`)
- **Confirmation hash**: SHA256 of sorted UUIDs via `--confirm-hash`
- **Count guard**: `--confirm-count N` must match
- **5-second abort window** before live deletion
- **Post-run Qdrant verification**: queries point store directly to confirm IDs are gone
- `--verify-only` mode for store health checks

### `docs/mem0-purge-runbook.md`
Full runbook covering: safety model, pre-flight checklist, dry-run procedure, live-run procedure, rollback/mitigation, targeted IDs table.

### `tests/test_mem0_purge_smoke.py`
17 deterministic smoke tests: shellcheck clean, UUID validation, guard behavior, hash determinism, confirmation hash verification.
