## Summary
- Remove all keyword-based CLI routing from TaskPoller (`detect_cli()`, `DEFAULT_CLI_KEYWORDS`, prompt library, `build_claudem_dispatch()`, multi-CLI subprocess dispatch)
- TaskPoller now delegates entirely to `ai_orch run` which handles CLI selection, model choice, and claudem/minimax internally
- Removes the banned anti-pattern (keyword-based intent detection to bypass LLM judgment) per CLAUDE.md
- Net deletion: ~760 lines of routing/dispatch logic

## Changes
| Before | After | Why |
|--------|-------|-----|
| `detect_cli()` matched task titles to CLIs by keyword (e.g., "fix" → cursor) | No CLI routing — `ai_orch run` decides | Banned anti-pattern per CLAUDE.md |
| `_build_clean_env()` stripped `CLAUDECODE` vars | Removed — ai_orch CLI profiles already handle `env_unset: ["CLAUDECODE"]` | Redundant |
| `_dispatch_via_subprocess()` with per-CLI command patterns (claude/codex/gemini/cursor) | `_dispatch_via_ai_orch()` — just `ai_orch run "task summary"` | ai_orch handles all CLI dispatch |
| Prompt library (load/save/resolve/update) learned CLI preferences from history | Deleted entirely | Keyword routing anti-pattern |
| `build_claudem_dispatch()` injected minimax env vars | Deleted — ai_orch has its own minimax profile | MC shouldn't manage model config |
