## Raw Body
## Problem
Skeptic Gate / skeptic-cron must report reliably for the 7-green automated merge path. Gate 1 CI filtering was easy to misread (jq precedence), and piping `gh api | jq` made `gh` failures look like “no verdict” (MISSING) instead of a real API error.

## Changes
- **skeptic-gate.yml**: `workflow_dispatch` with `pr_number` for manual dry-runs; job-level `PR_NUM`; guard empty PR_NUM; PASS/FAIL detection uses `\b` word boundaries; **GitHub Actions job summary** (`GITHUB_STEP_SUMMARY`) with truncated output; comment posting uses `github.repository_owner` + `github.event.repository.name` so manual runs work; `PR_HEAD_SHA` falls back to `github.sha` on dispatch.
- **skeptic-cron.yml**: explicit `select(.name != "Skeptic Gate" and .name != "Staging Canary Gate")` for $external_runs; **split gh vs jq** for issue comments so `gh` failure sets `SKEPTIC_ERROR` (fail-closed) instead of `SKEPTIC_NO_VERDICT`.
- **scripts/validate-skeptic-gate.sh**: local jq + node checks (no API).

## Evidence gate
Gate 6 remains **evidence-review-bot** review state in skeptic-cron (no separate workflow in this repo). This PR does not add a new evidence workflow; it fixes skeptic plumbing only.
