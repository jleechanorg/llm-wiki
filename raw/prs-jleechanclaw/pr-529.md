> [!NOTE]
> **Medium Risk**
> Changes GitHub Actions gating behavior and PR comment semantics, which can affect merge readiness and automation triggers if the new markers/conditions are wrong. Also introduces a new webhook worker dispatch path that shells out to `ao skeptic verify`, adding operational risk if later wired incorrectly.
> 
> **Overview**
> **Skeptic Gate now avoids spurious failures and dispatch bypasses.** The workflow is gated to run only for `pull_request`/`workflow_dispatch` events (and explicitly skips `push`), preventing empty PR context from causing false FAILs.
> 
> When the 6-green gate passes, it now posts `@github-actions /skeptic-verify` as a *separate* PR comment from the gate results to prevent automation from misreading the gate’s own `VERDICT: PASS` as an AO skeptic verdict. Gate result comments also switch to a new `<!-- HEAD-SHA: ... -->` marker and delete prior comments based on that marker.
> 
> Separately, `scripts/bootstrap.sh` loads gog auth from the canonical `${HOME}/.openclaw/openclaw.json` path, and `webhook.py` adds a `RemediationWorker._dispatch_skeptic()` helper to run `ao skeptic verify` for a PR (with optional trigger SHA).
