## Summary

Add `defaults.notifiers: [desktop, openclaw]` to `agent-orchestrator.yaml` so AO workers route escalation/urgent events to the openclaw notifier plugin in addition to desktop notifications.

## Background

Bead [orch-6ae](https://github.com/jleechanorg/jleechanclaw/issues?q=is%3Aissue+orch-6ae): Configure agent-orchestrator OpenClaw notifier plugin. The `notifiers.openclaw` block was already configured in the file; this change wires it into `defaults.notifiers` so AO workers actually use it.

## Goals

- AO workers post escalation events to openclaw gateway hook endpoint (`http://127.0.0.1:18789/hooks/agent`) during lifecycle transitions (session dead, zero-output exit, PR stalled, etc.)

## Tenets

- Minimal change — only add the `notifiers` field to defaults, rest of config unchanged
- OpenClaw gateway hooks already configured with `allowRequestSessionKey: true` and `allowedSessionKeyPrefixes: ["hook:", "hook:ao:", "hook:mctrl:"]`

## Changes

- **agent-orchestrator.yaml**: Add `notifiers: [desktop, openclaw]` under `defaults:` — activates the `notifiers.openclaw` webhook already defined in the file
