## Summary

This PR adds functionality to automatically resolve GitHub review threads whose file locations were modified in the latest push. This helps unblock PRs with `mergeStateStatus=UNSTABLE` by resolving outdated review comments.

## Changes

- **src/orchestration/auto_resolve_threads.py**: New Python module with:
  - `get_review_threads()`: Gets unresolved, non-bot review threads for a PR
  - `get_files_changed_in_push()`: Gets files changed in latest push
  - `_get_pr_files_gh()`: Helper to get PR files via gh CLI
  - `resolve_reviewThread()`: Resolves a thread via GraphQL mutation
  - `auto_resolve_threads_for_pr()`: Main entry point that ties it all together

- **scripts/ao-resolve-threads.sh**: Shell script for CLI usage
  - Can be wired into ao-backfill.sh for automated execution
  - Supports `--dry-run` for preview mode

- **tests/test_auto_resolve_threads.py**: TDD tests covering:
  - Getting and filtering review threads
  - Filtering bot-only threads
  - Matching threads to changed files
  - Resolving threads via GraphQL mutation
  - Handling edge cases (no threads, empty push)

## Usage

```bash
# Auto-resolve threads for a PR
./scripts/ao-resolve-threads.sh jleechanorg jleechanclaw 123

# Dry-run mode (preview what would be resolved)
./scripts/ao-resolve-threads.sh jleechanorg jleechanclaw 123 --dry-run

# Or use the Python module directly
python -m orchestration.auto_resolve_threads jleechanorg jleechanclaw 123
```
