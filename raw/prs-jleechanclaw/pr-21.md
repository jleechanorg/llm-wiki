## Summary
- Add hard repo-targeting guardrails to OpenClaw guidance so PR actions always resolve and pin the intended repository.

## Changes
- Updated `openclaw-config/SOUL.md`
  - Added **Repo Targeting Protocol (Hard Guardrail)** with mandatory target resolution and mismatch handling.
  - Added **Repo Ownership Map (Default Targets)** to make cross-repo routing deterministic.
  - Clarified Slack permalink thread retrieval wording to refer to configured OpenClaw Slack read-history capability.
- Updated `openclaw-config/TOOLS.md`
  - Standardized placeholder format to `<owner>/<repo>`.
  - Replaced `git remote get-url origin` with `git remote -v` in safety snippets to avoid assuming `origin`.

## Why
- We saw repeated wrong-repo PR placement across related repos (including cross-system design docs).
- These config-level rules make repo ownership checks deterministic before any GitHub mutation.

## Testing
