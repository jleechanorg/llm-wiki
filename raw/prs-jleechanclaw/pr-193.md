> [!NOTE]
> **Medium Risk**
> Adds new persistent outcome logging and wires it into the parallel-retry execution path; main risk is unintended filesystem writes/format issues or subtle session-id handling impacting orchestration runs.
> 
> **Overview**
> Adds outcome recording for `ParallelRetryAction` so the system can learn which fix strategies win for a given CI failure class.
> 
> `action_executor` now derives an `error_class` from the CI failure and passes it (and the parent `session_id`) into `execute_parallel_retry`. `parallel_retry.execute_parallel_retry` captures the winner/losers after a green result and non-blockingly appends an outcome entry via the new `OutcomeRecorder` (JSONL under `~/.openclaw/state/outcomes.jsonl`), while also fixing variable shadowing by separating spawned session IDs from the parent session ID.
> 
> `outcome_recorder.py` introduces the JSONL-backed `OutcomeRecorder` with `record_outcome`/`query_outcomes`, plus a local `FixStrategy` shape to avoid circular imports.
