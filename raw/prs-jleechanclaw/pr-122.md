## Why this PR is needed

These changes accumulated in the `worktree_debug` workspace during active debugging and hardening sessions. They address three real failure classes observed in production:

1. **Hardcoded REDACTED tokens in openclaw.json** — all tokens stored as literal strings; now all secrets use `${ENV_VAR}` refs so the config is safe to commit and syncs cleanly from the live environment
2. **doctor.sh had no API token validation** — it checked gateway health but couldn't tell you if Slack, MiniMax, xAI, Discord, or OpenAI keys were broken, leading to false "healthy" reports
3. **sync-openclaw-config.sh had no guardrails** — a broad config sync could silently overwrite live openclaw.json with placeholder tokens and break the gateway instantly

---

## Changes

### `openclaw-config/openclaw.json` — token cleanup + env refs
- All secrets moved to `${ENV_VAR}` refs: `MINIMAX_API_KEY`, `XAI_API_KEY`, `OPENCLAW_SLACK_BOT_TOKEN`, `OPENCLAW_SLACK_APP_TOKEN`, `DISCORD_BOT_TOKEN`, `OPENCLAW_GATEWAY_TOKEN`, `OPENAI_API_KEY`
- Removed unused `MISSION_CONTROL_TOKEN`
- MCP Agent Mail auth header removed (server runs no-auth now)
- Slack channel set to `enabled: false` in repo copy (live config enables it via env — prevents repo sync from toggling this accidentally)
- Memory flush prompt unicode escape fixed

### `scripts/doctor.sh` — full API token probe suite (+169 lines)
- Added `resolve_secret_ref()` and `resolve_bearer_token_ref()` to expand `${ENV_VAR}` refs before probing
- New live token probes: Slack bot (`auth.test`), Slack app (`apps.connections.open`), MiniMax (`/anthropic/v1/messages`), OpenAI (`/v1/models`), xAI (`/v1/models`), Discord (`users/@me`), MCP Agent Mail (`tools/list`)
- Without this, doctor.sh would pass gateway health but miss broken API keys — which is exactly what caused today's monitor STATUS=PROBLEM false negatives
