## Background
bug-hunt-daily.sh reported "0 bugs found" as a clean sweep after all 5 bug-hunt agents failed completely (Codex had CA certificate errors, others produced 0-byte output files). Root causes: `|| true` on all agent invocations swallowed failures, empty output files were counted as 0 bugs (not failures), outputs went to ~/.openclaw/bug_reports/ (polluting the live repo), and Codex was invoked with a broken flag.

## Goals
- Agents that fail or produce empty output counted as failures, not 0-bug runs
- "0 bugs" only reported as clean sweep when agents actually ran
- Bug report outputs go to /tmp (not the live repo)
- Codex CA cert connectivity failure handled gracefully

## Tenets
- Fail-closed: empty output = failure, never 0 bugs
- Never use `|| true` on agent invocations
- Output files in /tmp to keep the repo clean

## Changes
- **BUG_REPORTS_DIR**: `/tmp/openclaw/bug_reports` (was ~/.openclaw/bug_reports)
- **Agent invocation**: removed all `|| true` guards; exit code propagates
- **Empty-file detection**: `[ ! -s "$OUTPUT_FILE" ]` check before jq; counts as agent failure
- **Codex**: replaced broken `codex exec` with `claude --dangerously-skip-permissions`
- **Failure tracking**: new `AGENT_FAILURES` counter and `ALL_AGENTS_FAILED` flag
- **Report/Slack**: "Agent failures: N/5" line added; FAILURE_WARNING block with `:warning:` when agents failed
- **MINIMAX_API_KEY**: added `:-` default to prevent unbound variable errors
