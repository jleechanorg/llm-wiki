## Background
OpenClaw needed a way to join Zoom meetings as a visible bot, capture live transcripts, and summarize them. Meeting BaaS provides a managed bot API.

## Goals
- Deploy Openclaw bot to any Zoom meeting via CLI
- Receive real-time transcript chunks via webhook  
- Post [OpenClaw AI] prefixed summaries to Slack on meeting end
- Identity disclosure so participants know they're talking to an AI

## Tenets
- No fake tests — webhook tested with real Meeting BaaS deploy
- Config-first — bot name, Slack channel, webhook URL all via env vars

## High-level description
New meetingbaas.py integrates Meeting BaaS v2 API. Bot deploys via deploy_bot(), receives webhooks at /meetingbaas/webhook port 9101, accumulates transcript chunks, and on call.ended posts summary to Slack. SOUL.md gains zoom-meeting-identity COMMIT rule enforcing [OpenClaw AI] prefix.

> [!NOTE]
> **High Risk**
> Introduces a new externally exposed webhook service plus Discord posting and `subprocess` execution paths, which increases security and operational risk if env secrets or signature verification are misconfigured. Handling live meeting transcripts also raises data-leak risk and requires careful production hardening.
> 
> **Overview**
> Adds a new `src/orchestration/meetingbaas.py` module that can **deploy a Meeting BaaS bot to a Zoom URL** and **run a threaded webhook server** (`/meetingbaas/webhook`) to ingest meeting events.
> 
> Webhooks are **SVIX-signed** (with a `MEETING_BAAS_DEV_MODE` bypass), transcript chunks are accumulated per `bot_id`, and on idle/meeting-end the full transcript is posted
