## Summary
- **Architecture revision**: agent-orchestrator (AO) replaces mctrl as primary orchestration layer
- **Deterministic-first principle**: AO reaction engine handles obvious cases (CI fail, review comments), OpenClaw LLM handles judgment calls only

## Changes

- `docs/ORCHESTRATION_RESEARCH_2026.md` — new research doc with web sources, production data (Spotify Honk, Composio AO), and multi-model second opinions

## Key Design Decisions
| Before | After | Why |
|---|---|---|
| OpenClaw → mctrl → ai_orch | OpenClaw → AO → headless agents | AO has battle-tested lifecycle manager, plugin arch, declarative YAML |
| LLM decides everything | Deterministic first, LLM for judgment | Spotify pattern: "reduced flexibility increases predictability" |
| Nested loops = LLM accumulates context | Nested loops = fresh headless call each time | Avoids context collapse identified by Gemini/Grok reviews |
| ai_orch is core layer | ai_orch is optional convenience | Multi-CLI support only, not architecturally required |
