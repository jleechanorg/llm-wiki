## Summary

Documents comprehensive research into Slack Socket Mode event loss issues affecting OpenClaw.

## Problem

OpenClaw missing Slack thread reply events despite correct config. Three confirmed instances on 2026-03-16.

## Changes

Added Socket Mode reliability investigation to workspace/MEMORY.md:
- Problem statement with timestamps
- Research findings from verified sources
- Root cause: inherent protocol limitation
- Production solutions from battle-tested apps
- Action items for improvements

## Key Findings

1. Socket Mode event loss is inherent (no replay mechanism)
2. SDK v1.3.0+ required (earlier versions crash)
3. Slack recommends HTTP mode for production
4. No polling fallback patterns in production

## Sources

- slackapi/bolt-js issue 1840
- slackapi/node-slack-sdk issue 1243  
- Slack official docs on event delivery

## Recommendations

Long-term: Migrate to HTTP mode (Tailscale ready)
Short-term: SDK upgrade, health checks, verbose logging
