## Background

PR #268 delivered Phase 1 of the two-agent reviewer loop (orch-tlkk.1): `reviewer_agent.py` core with merge gate checking, diff review, and GitHub PR review posting. However, MCP mail communication between coder and reviewer agents was stubbed as a Phase 2 placeholder, and `merge_gate.py` condition 6 still required the old `verdict.json` evidence bundle.

## Goals

1. Wire MCP mail so reviewer can send findings to coder agents
2. Update merge_gate condition 6 to accept reviewer agent's GitHub APPROVED review
3. Add AO reaction config for reviewer dispatch
4. Keep verdict.json path as fallback (Phase 3 will remove it)

## Tenets

- Config-first: AO reaction config over hardcoded dispatch logic
- Backward compatible: verdict.json still works as Tier 2 fallback
- Thin wrappers: mcp_mail.py wraps mcporter CLI, no new dependencies
- Fail-open on mail: MCP mail failures are warnings, not blockers

## High-level description of changes

- **New `mcp_mail.py`**: Thin wrapper around `mcporter` CLI for `send_mail()` and `fetch_inbox()`. Used by `reviewer_agent.py` to send findings to coder agents.
- **merge_gate condition 6**: New Tier 1 check — looks for reviewer agent's APPROVED GitHub review (identified by "Independent PR Review" body marker) before falling through to verdict.json Tier 2.
- **AO config**: New `review-requested` reaction dispatches reviewer agent when coder signals ready.
- **reviewer_agent.py**: Linter wired `coder_agent` param to `run_review()` with MCP mail send.
