## What happened

Commit `7d5b9e1efd` (chore: untrack .openclaw-backups/) ran `git rm -r` from repo root instead of just removing the backups directory from tracking. This deleted everything including `scripts/backup-openclaw-full.sh`, all workflows, `openclaw-config/`, and 31k other files.

## Changes

- Restores all files from parent commit `200ac086d4` (before the deletion)
- Adds to `.gitignore`: `.openclaw-backups/`, `.beads/interactions.jsonl` (the actual intent of the broken commit)
- Excludes live worktree dirs from tracking

## Testing

- `scripts/backup-openclaw-full.sh` present and correct (rsync to `latest/`, no timestamped dirs)
- `scripts/run-openclaw-backup.sh` present (entry point for LaunchAgent)
