## Changes

Before: AO only reacts to webhooks. If no webhook fires, PRs sit stale indefinitely.
After: 5-min polling ensures any non-green PR without an active session gets a respawn.
Why: Closes the stale-PR gap in full autonomy — webhook reactions handle happy path, polling handles fallthrough.

## Testing

To load on macOS: cp launchd/ai.ao-pr-poller.plist ~/Library/LaunchAgents/ && launchctl load ~/Library/LaunchAgents/ai.ao-pr-poller.plist

## Known Limitations

- ao spawn requires agent-orchestrator CLI at ~/projects_reference/agent-orchestrator
- Active-session detection depends on ao status --json output format
