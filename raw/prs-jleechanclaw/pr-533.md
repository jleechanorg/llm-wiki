## thinclaw repo (https://github.com/jleechanorg/thinclaw)

**Stack:** Node.js (ES modules), `@modelcontextprotocol/sdk` (StdioServerTransport), `axios`, `zod`

**Design:** Zero LLM inference — thinclaw is a pure HTTP REST bridge. Claude Cowork/Perplexity provides all reasoning.

**Architecture:**
```
Claude Cowork/Perplexity (brain)
  → thinclaw MCP stdio (bridge, zero inference)
    → OpenClaw Gateway localhost:18789 (body / tool executor)
```

**5 tools implemented:**
1. `openclaw_execute` — POST /tools/invoke (any OpenClaw tool)
2. `send_whatsapp` — POST /skills/whatsapp/send
3. `schedule_cron` — POST /cron/schedule
4. `run_shell` — POST /shell/exec
5. `trigger_cowork_workflow` — writes JSON flag to ~/AI_Bridge/inbox/ (cron→Cowork handoff)

**AI_Bridge** folder created at ~/AI_Bridge/{inbox,outbox,processed} for cron→Cowork cognitive handoff.

**README.md** covers: architecture diagram, token setup, env vars, Claude Desktop/Cowork/Perplexity setup, launchd auto-start, AI_Bridge setup, expansion guide.

**Testing:** Perplexity Computer + Claude Cowork zero-inference verification will be done manually after merge.
