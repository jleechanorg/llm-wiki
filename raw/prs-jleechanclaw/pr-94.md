## Raw Body
Follow-up from PR #89 (merged). Addresses P2 review comment #2911000196.

**Issue:** `StartCalendarInterval` encodes local wall-clock times with no timezone. Original cron schedules were `America/Los_Angeles`. On systems in other timezones, launchd would fire at the wrong absolute time.

**Fix:** Add `TZ=America/Los_Angeles` to `EnvironmentVariables` in all 6 schedule plists so jobs run at the intended Pacific time regardless of system timezone.
