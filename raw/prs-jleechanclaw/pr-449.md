## Summary

Prevention harness for the mem0/better-sqlite3 NODE_MODULE_VERSION mismatch that caused the 2026-03-28 incident (Slack reply drops, ws-stream 500 storm).

## Changes

### 1. `scripts/gateway-preflight.sh` — Enhanced (cb7fb5574c)
New checks appended:
- **SDK version check**: validates `@agentclientprotocol/sdk` version before upgrade; fails if major version jumps (e.g. 0.16 → 0.17) without explicit ack
- **checkCompatibility key validation**: warns if found outside `extensions.*.vectorStore` (setting it at `gateway` or `agents.defaults` level crashes the gateway)
- **NODE_MODULE_VERSION baseline tracking**: records current Node MODULE_VERSION to `~/.openclaw/.gateway-node-version`; warns on mismatch

### 2. `scripts/mem0-native-module-watchdog.sh` — New
- Checks active Node MODULE_VERSION vs stored baseline
- `--fix` flag: auto-rebuilds better-sqlite3 via node-gyp on mismatch
- `--status` flag: prints status and exits 0
- Log rotation at 5MB, log to `~/.openclaw/logs/mem0-watchdog.log`
- Exit codes: 0=OK/baseline set, 1=mismatch (needs rebuild), 2=baseline missing

### 3. `scripts/openclaw-upgrade-safe.sh` — New
Safe wrapper around `npm install -g openclaw@<version>`:
1. Runs `gateway-preflight.sh` first
2. Validates SDK compatibility
3. Shows what will change
4. Requires `--yes` to proceed non-interactively
5. `--ack-minor-jump` to explicitly acknowledge a SDK minor version jump
6. After upgrade: rebuilds native modules, records new baseline

### 4. `~/Library/LaunchAgents/com.openclaw.mem0-watchdog.plist` — New
- Runs `mem0-native-module-watchdog.sh --fix` every 4 hours
- `RunAtLoad`: starts on launchd boot
- Uses full nvm paths for node/npm

### 5. `scripts/install-openclaw-launchd.sh` — Enhanced (f1e6f3d65)
- Records initial NODE_MODULE_VERSION baseline after first install
- Bootstraps the mem0 watchdog on install
