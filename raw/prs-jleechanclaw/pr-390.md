## Background
Claude Code has a file-based auto-memory system at `~/.claude/projects/PROJECT/memory/*.md` that persists learnings across conversations. Our slash commands were not reading from or writing to it — making the memory system silently underutilized. This PR wires 5 commands into that system.

## Goals
- `/history` surfaces relevant memories before searching conversation JSONL
- `/research` checks prior knowledge before multi-engine search
- `/debug` detects previously-seen bug patterns; writes feedback memory on success
- `/learn` writes durable learnings to file-based auto-memory (in addition to Memory MCP)
- `/checkpoint` saves durable session insights to auto-memory in --summary mode

## Tenets
Minimal edits only. Graceful failure. File-based complements Memory MCP.

## High-level description of changes
Each command gets a new labeled phase using glob to find memory files, keyword-filter, and surface or write frontmatter-formatted memory entries.

## Testing
Files reviewed manually. No breaking changes to existing command phases.

## Low-level details
- history.md: Phase 0 memory read before JSONL search
- research.md: Phase 0 prior knowledge check
- debug.md: Phase 0 seen-before check + item 14 write on success
- learn.md: Phase N+1 writes to file-based auto-memory after Memory MCP
- checkpoint.md: Auto-memory write in --summary mode
