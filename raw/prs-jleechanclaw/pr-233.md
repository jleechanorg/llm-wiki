## Summary

Fixes the ao-pr-poller.sh to properly handle existing AO sessions instead of skipping PRs just because a worktree exists.

## Changes

1. **REMOVED** `branch_checked_out()` function that skipped PRs with existing worktrees (worktree existing is NOT a reason to skip)

2. **ADDED** resume logic:
   - `find_session_for_branch()` - Find tmux sessions matching PR branch
   - `is_agent_running_in_session()` - Detect live agent vs bash idle  
   - `check_session_idle()` - Detect bash panes
   - `resume_session()` - Send 'ao message' to idle sessions
   - `handle_session_resume()` - Main logic:
     - If session exists AND pane shows live agent → do nothing
     - If session exists AND pane shows bash (idle) → send resume
     - Only if NO session → spawn new

3. **ADDED** TDD test file for resume logic
