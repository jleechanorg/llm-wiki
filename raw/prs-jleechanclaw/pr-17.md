## Summary
- auto-transition task poller dispatches from `in_progress` to `done`
- keep existing token-usage custom field update on the `in_progress` step
- add regression coverage for failed `done` transition handling

## Changes
- Updated `TaskPoller._dispatch_task` to perform a second status update:
  - `in_progress` update remains first (with optional `custom_field_values.cost`)
  - successful `in_progress` now followed by `done`
  - dispatch is counted only if both updates succeed
- Updated task poller tests to assert the two-step status transition and call counts.
- Added test for `done` update failure returning `0` dispatched count.

## Testing
- `PYTHONPATH=src python -m pytest src/tests/test_task_poller.py -q --tb=short`
- Result: `34 passed`

## Known Limitations
- Auto-complete currently moves directly to `done`; it does not enforce an intermediate `review` stage.
