## Summary
- add concrete GitHub Actions workflow for `@jleechanclaw` PR comment triggers at `.github/workflows/agent-pr-fix-trigger.yml`
- enforce trusted-actor gate (`OWNER`/`MEMBER`/`COLLABORATOR`) and PR-only guard before self-hosted execution
- harden payload creation and API invocation with `jq` JSON construction, no `xargs`, and `curl --fail-with-body`
- codify Batch A policy decisions in `docs/mcp-mail-openclaw-mission-control-design.md` (source-of-truth defaults, CodeRabbit quorum fallback, evidence schema baseline, webhook preference)

## Changes
- Workflow hardening:
  - job-level `if` now requires PR context + trusted actor + `@jleechanclaw` mention
  - shell step uses `set -euo pipefail`
  - comment instruction extraction is handled via `jq -rn` to preserve whitespace/newlines and avoid shell mangling
  - request payload is built entirely with `jq -n`
  - HTTP call uses `curl --silent --show-error --fail-with-body`
- Validation:
  - added static workflow assertions in `src/tests/test_gh_integration.py` to guard trusted actor gate, PR-only filter, jq usage, and fail-fast curl behavior
- Documentation:
  - updated mission control design doc with explicit Batch A decisions and rationale

## Testing
- `pytest -q src/tests/test_gh_integration.py -k "AgentPRFixTriggerWorkflow"`
- `pytest -q src/tests/test_gh_integration.py`

## Known Limitations
- This PR implements the comment-trigger hardening path only; full CI/review webhook migration remains future work.
- Fallback reviewer quorum is documented as policy; enforcement logic for merge gating lives in downstream orchestration/review state handling, not this trigger workflow.
