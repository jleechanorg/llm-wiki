## Changes

### `.github/workflows/skeptic-cron.yml` — Bugbot jq filter

**Before:**
```
'[... | select(... | test("\\\\*\\\\*HIGH SEVERITY\\\\*\\\\*|...")) | select(.original_commit_id == $pr_head or (.original_commit_id | type) == "null") | .id] | length'
```

**After:**
```
'[.[] | select(.user.login == "cursor[bot]" and (.body | test("High Severity|Critical Severity"; "i")) and (.original_commit_id == $pr_head or .original_commit_id == null)) | .id] | length'
```

**Fixes:**
1. `\\*\\*HIGH SEVERITY\\*\\*` — backslash-asterisk regex doesn't match `**HIGH SEVERITY**` (literal asterisks in cursor[bot] text)
2. `(.original_commit_id | type) == "null"` — simplified to direct jq null comparison
3. Consolidated two `select()` pipes into one `and()` for clarity

## Evidence
PR #412 skeptic-cron log at 17:18 UTC showed `[4] Bugbot: -1 errors` — this fix resolves that.
