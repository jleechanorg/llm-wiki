## Summary

This PR trims the mctrl orchestration stack back to its essential responsibilities: spawn an agent, watch it, notify on completion. Everything added "for future use" or that duplicated higher-level system concerns (retry queues, proof payloads, multi-registry fan-out, terminal archival) is removed.

- **−1357 / +132 lines** across 8 files

## Changes (before → after → why)

### `dispatch_task.py`

| Before | After | Why |
|--------|-------|-----|
| Cross-repo task detection (`_is_cross_repo_task`, `_extract_repo_name_hint`, `_inject_cross_repo_context`, `_resolve_target_repo`) | Removed | False-positive prone keyword matching; OpenClaw/task instructions handle repo routing |
| `_parse_ai_orch_output(output, known_worktree)` | `_parse_ai_orch_output(output)` | `known_worktree` was only used by the cross-repo path |
| `notify_openclaw()` called at spawn time | `notify_slack_started()` | Started events are Slack-only; OpenClaw notification belongs at completion |
| `subprocess.run(cmd)` inherits full env | `env={k:v for k,v in os.environ.items() if k != "PYTHONPATH"}` | Prevents `PYTHONPATH=src` leaking into `ai_orch`, which would shadow its own modules |
| No `--slack-trigger-channel` | Added `--slack-trigger-channel` CLI arg + `dispatch()` param | Flows through registry → reconciliation → notifier so thread replies reach the originating channel |

### `openclaw_notifier.py`

| Before | After | Why |
|--------|-------|-----|
| `completion_notification_max_runtime_seconds()`, `openclaw_notification_max_runtime_seconds()` | Removed | Callers can budget their own timeouts |
| `DEFAULT_DEAD_LETTER_PATH`, dead-letter queue, `_normalize_trigger_ts/channel` normalizers | Removed | Dead-letter handling belongs at a higher level; normalizers were hiding bugs |
| Outbox retry metadata (`_retry_count`, `_last_attempt_at`, `_first_queued_at`), `DEFAULT_RETRY_LIMIT` | Removed | Retry logic added
