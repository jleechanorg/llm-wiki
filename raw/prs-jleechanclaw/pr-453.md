## Background

The Meeting BaaS integration routes Zoom meeting chat to Discord so @Consensus can answer live. After `fix(meetingbaas): add webhook signature verification` was merged, ALL Meeting BaaS webhooks were silently rejected, causing the consensus Discord bot to stop receiving Zoom meeting questions.

## Goals

- Restore webhook processing for Meeting BaaS events
- Ensure `_verify_webhook_signature()` correctly accepts valid SVIX signatures
- Add test coverage so this class of breakage can't recur silently

## Tenets

- Fix the actual SVIX spec — don't work around it
- Tests must watch the bug fail before the fix, not be written after

## High-level description of changes

Three bugs in `_verify_webhook_signature()`:

| Bug | Was | Should be |
|-----|-----|-----------|
| Signature format | looked for `v1=<hex>` | SVIX sends `v1,<base64>` (comma, not equals) |
| HMAC encoding | `hexdigest()` (hex string) | `base64.b64encode(digest())` |
| Multiple signatures | not supported | space-separated entries for key rotation |

The function always returned `False` for any real SVIX signature, silently dropping every webhook.

Also added `import base64` (was missing) to support the fix.
