## Summary
- add `scripts/load_test_gateway.py` for repeatable OpenClaw gateway load testing
- support two modes: lightweight `/health` and real `/v1/chat/completions`
- output machine-readable JSON summary with latency percentiles, success/failure counts, and error samples

## Changes
- new script: `scripts/load_test_gateway.py`
- token resolution order: `--token`, `OPENCLAW_GATEWAY_TOKEN`, then `~/.openclaw/openclaw.json`
- configurable request count, concurrency, timeout, base URL, and output path

## Testing
- `python3 -m py_compile scripts/load_test_gateway.py`
- executed load tests in local environment (reports saved under `/tmp/openclaw-gateway-loadtest-*`)

## Known Limitations
- chat mode load quality depends on downstream model/provider health and any live gateway drain/restart events
- this script intentionally avoids external benchmark dependencies (`wrk`, `k6`, `autocannon`) for portability
