## Background

Core markdown policy files (SOUL.md, TOOLS.md, USER.md, IDENTITY.md, HEARTBEAT.md, AGENTS.md, MEMORY.md, BOOTSTRAP.md) were inaccessible in git worktrees because git symlinks in `workspace/` were committed with wrong targets (`workspace/FILENAME` instead of `FILENAME`), creating circular disk references. A spurious `workspace/workspace/` directory was also committed.

## Goals
- [x] Restore all broken workspace/ symlinks as real files
- [x] Remove spurious workspace/workspace/ directory
- [x] Add monitor-agent.sh health check for all core MD files
- [x] Prevent recurrence via monitor-agent enforcement
- [x] Add test coverage (7 test cases)
- [x] Document recovery steps in runbook

## Tenets
- Fail-closed: missing or broken = alert immediately
- Deterministic: no LLM calls in the check logic
- Minimal: reuse existing monitor-agent probe pattern

## Changes

### Root cause
Commits `cf8f009db` and `70b8ec4b2` introduced git symlinks in `workspace/` pointing to `workspace/FILENAME`. On disk, these resolve to `workspace/workspace/FILENAME` (absent), making root-level symlinks appear to work while `workspace/*.md` symlinks were broken.
