## Background

PR #454 added staging-canary.sh with 6 checks but never wired it into CI. The canary existed on disk but prevented nothing because no automatic process called it. Root cause tracked in orch-dv6.

The deeper failure: even without Slack context, an agent should verify 'who calls this?' before declaring an automation task done. No harness rule existed to enforce this.

## Goals

Add an 'Automation completeness' rule requiring any PR adding a script to also add the trigger that calls it automatically.

## High-level description of changes

**CLAUDE.md**: Added automation completeness section with 3-question checklist (who calls it? when? what fails if it doesn't fire?) and red flags list.

**.github/workflows/skeptic-gate.yml** (CI plumbing fix — necessary prerequisite):
The workflow had a chronic YAML-parsing bug where any Python/heredoc content in the run block caused "could not find expected ':'" at the import line. Root cause: YAML literal block scalar with embedded Python starting at column 1 was closing the block scalar prematurely. Fix uses printf/body-file approach from origin/main, avoiding heredoc complexity entirely.

Bundled here because the automation completeness rule will generate similar CI workflow PRs — the workflow needed to be self-correcting first.
