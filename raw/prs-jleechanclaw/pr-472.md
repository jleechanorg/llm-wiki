## Background

The `jleechanclaw` harness (the operator control layer) can be exported as a portable "seed" repo called `smartclaw`. This PR adds the tooling to audit, classify, and sync portable files to `jleechanorg/smartclaw`.

## Goals

- Classify which jleechanclaw files are portable (general-purpose) vs. private/runtime
- Auto-generate and maintain `scripts/smartclaw-export-map.tsv`
- Run sanitization on sync (replace `jleechanorg/jleechanclaw` → `smartclaw`, `jleechan` → `$GITHUB_USER`, etc.)
- Keep smartclaw PRs human-review-only (not auto-merged)

## Tenets

- No secrets, credentials, or personal runtime state in the export
- Launchd templates renamed from `ai.openclaw.*` → `smartclaw.*`
- Sync via `scripts/sync-to-smartclaw.sh` regenerates the map before every sync
