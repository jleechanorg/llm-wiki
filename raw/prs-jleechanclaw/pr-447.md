> **Overview**
> Fixes zombie worktree cleanup by switching registration detection to parse `git worktree list --porcelain` `worktree` lines (instead of grepping a non-existent `path` key) and by calling `git worktree remove --force` with the *checkout directory path* rather than `.git/worktrees/...` metadata paths.
> 
> Adds safety checks to only remove missing-on-disk worktrees that match the AO naming convention and have no live tmux session, plus a multi-repo `cleanup_all_worktrees` helper. Introduces unit tests (mocked subprocess) and integration tests (real temp repos/worktrees) to prevent regressions for both bugs and the safety filters.
