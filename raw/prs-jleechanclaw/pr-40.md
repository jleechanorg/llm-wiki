## Production Code Changes

**Before**: No local Dropbox backup; only GitHub git backup via `backup-openclaw-full.sh`

**Now**: Parallel Dropbox rsync backup running every 4h at :10 offset, excludes workspace, credentials/whatsapp, .git, .DS_Store, jleechanclaw

**Why**: Single-point-of-failure for openclaw config backup; Dropbox provides local/private redundancy with no git commits needed
