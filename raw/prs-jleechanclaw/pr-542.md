## Raw Body
### Background
The PR gate workflow was named "Skeptic Gate" even though this workflow is deterministic CI logic and does not perform LLM-based skeptic review. This PR renames the workflow to "Green Gate" while preserving compatibility with downstream gate parsing.

### Goals
- Rename the deterministic workflow and emitted check/comment text to Green Gate.
- Prevent false CI PASS results by requiring all repo-relevant named check-runs to register and complete.
- Require CodeRabbit approval only when the latest approval is newer than any CodeRabbit blocker.
- Preserve the existing HEAD-SHA marker consumed by skeptic-cron validation.
- Keep private-repo workflow execution on self-hosted runners.

### Tenets
- Deterministic gates should fail closed when required evidence is missing.
- Existing downstream parsers should keep working during the rename.
- Review-state logic should use GitHub's formal review states, not PR comment text.
- Non-decisive review comments should be handled by the unresolved-comments gate, not by the approval gate.
- Workflow harness checks should validate the renamed workflow file directly.

### High-level description of changes
The workflow file was renamed from `skeptic-gate.yml` to `green-gate.yml` and its display name/comment markers were updated to match. Gate 1 now waits for all repo-relevant deterministic check-runs (`Staging Canary Gate`, `find-openclaw-json`, `Full canary (6/6)`) instead of allowing completion after only two checks.

Gate 3 now matches the stronger skeptic-cron approval logic: CodeRabbit passes only when an `APPROVED` review is newer than any CodeRabbit `CHANGES_REQUESTED` or `DISMISSED` blocker. Later `COMMENTED` reviews do not erase a formal approval; unresolved comments are handled separately by Gate 5. The paginated review response is slurped and then piped into `jq`, matching `gh api`'s supported flag combinations.
