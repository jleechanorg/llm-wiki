## Background

The `ao-session-reaper.sh` cron job runs nightly to clean up zombie tmux sessions that are no longer associated with live agent work. However, it only scanned for `jc-*` named tmux sessions, completely ignoring `ao-*` sessions (e.g. `ao-748`, `ao-749`, `ao-750`) that accumulate when AO workers are killed but their tmux sessions persist.

## Goals

- Prevent `ao-*` zombie tmux sessions from accumulating indefinitely
- Apply the same age-based and PR-state-based kill logic to `ao-*` sessions as already exists for `jc-*`

## Tenets

- Minimal, surgical change — only the two session-prefix patterns were updated
- Same kill thresholds (2h orphaned, 4h no-PR, merged/closed PR) apply to both prefixes
- No behavioral change for existing `jc-*` sessions

## Changes

### `scripts/ao-session-reaper.sh`

Two changes:

1. **`parse_session_info` regex** (`scripts/ao-session-reaper.sh:32`):
   - Before: `^jc-[0-9]+$`
   - After: `^((jc)|(ao))-[0-9]+$`

2. **Main loop filter** (`scripts/ao-session-reaper.sh:235`):
   - Before: `grep '^jc-'`
   - After: `grep -E '^(jc|ao)-'`

### `tests/test_session_reaper.sh`

Added 3 new test cases:
- `test_ao_orphaned_old_killed`: verifies `ao-*` sessions older than 2h with no worktree are killed
- `test_ao_orphaned_new_not_killed`: verifies recent `ao-*` orphaned sessions are protected by the 2h grace period
- `test_parse_ao_session`: directly verifies the new regex accepts `ao-*` names
