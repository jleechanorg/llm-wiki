## Background

The Python evidence pipeline (PR #265: evidence_bundle.py + stage2_reviewer.py + verdict.json) works but is brittle — custom file formats, CLI dispatch fragility, no feedback loop. This PR replaces it with a reviewer agent that communicates with the coder via MCP mail and posts real GitHub PR reviews.

## Goals

1. Reviewer agent that checks all 6 merge gate conditions and reviews diffs independently
2. MCP mail communication between coder and reviewer for a fix loop
3. Real GitHub PR reviews instead of verdict.json files
4. Roadmap + beads tracking for the full implementation plan

## Tenets

- Real agent-to-agent communication over file-based evidence
- Leverage existing infrastructure: merge_gate.py, MCP mail, AO dispatch
- Different model family for reviewer enforced by AO config, not runtime checks

## High-level description of changes

New reviewer agent that runs merge_gate checks, analyzes diffs for common issues, builds review bodies, and posts GitHub PR reviews. Includes MCP mail formatting for coder communication. Roadmap doc describes the full two-agent loop architecture.

## Testing

- 17 unit tests covering diff analysis, review body generation, mail formatting, and e2e flow
- Live dry-run against PR #265 — correctly identified gate conditions
- Full test suite: 606 passed, 12 pre-existing failures unrelated to this PR

## Low-level details

- **roadmap/AGENT_REVIEWER_LOOP.md**: Architecture doc — two-agent review loop design, what stays/changes/is new, 3-phase plan
- **src/orchestration/reviewer_agent.py**: Core reviewer — check_gate, review_diff, build_review_body, post_github_review, format_mail_findings, run_review
- **src/tests/test_reviewer_agent.py**: 17 tests — TestReviewDiff 7, TestBuildReviewBody 4, TestFormatMailFindings 3, TestCheckGate 1, TestRunReview 3

## Known Limitations

- Diff review is structural regex-based — the real value comes from running on a different model family
- MCP mail loop not yet wired into AO reaction
