## Background

The openclaw mem0 Qdrant store accumulates session facts from openclaw agent sessions. Claude Code sessions (jleechan terminal sessions) were not connected to this store — every Claude session started with no memory of prior work.

This PR wires Claude Code into the shared mem0 memory store via two hooks registered in `.claude/hooks/`.

## Goals

- **Recall**: Before Claude sees a user prompt, search Qdrant for relevant past facts and inject them as `additionalContext` (silent, not shown as output)
- **Save**: After each Claude response, extract facts via mem0 LLM inference and dual-write to Qdrant + a human-readable markdown log in the project's auto-memory dir
- **Zero-blocking**: Every error path silently exits 0 — hooks never interrupt Claude sessions

## Tenets

- Silent by default — hooks should be invisible unless they find something relevant
- Fail-open — Qdrant down, no OPENAI_API_KEY, mem0 not installed → skip gracefully
- Dual-write for debuggability — markdown log lets humans see what got saved without querying Qdrant
- No new dependencies required at import time — `from mem0 import Memory` is inside the try block

## High-level description of changes

**`.claude/hooks/mem0_recall.py`** (UserPromptSubmit hook)
- Reads prompt from hook input, searches `openclaw_mem0` Qdrant collection
- Filters to `score >= 0.35`, returns top-6 as `additionalContext` JSON
- Guards: prompt < 10 chars, no OPENAI_API_KEY, mem0 import failure → all skip silently

**`.claude/hooks/mem0_save.py`** (Stop hook)
- Reads `last_assistant_message` from hook input
- Guards: `stop_hook_active=True` (loop prevention), response < 100 chars, code-only heuristic
- Calls `m.add(last_message, user_id="jleechan", infer=True)` — mem0 LLM extracts facts
- **Dual-write**: also appends ADD/UPDATE facts to `memory/mem0_extractions.md` in the project's auto-memory dir (derived from `transcript_path`)
