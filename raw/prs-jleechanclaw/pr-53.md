> **Overview**
> Introduces a minimal Mission Control integration to manage task state and prevent double-dispatch by **atomically claiming tasks**.
> 
> Adds `MCClient` with a `TaskStatus` enum (including new `DISPATCHING`) and a `TaskPoller._dispatch_task()` flow that first patches the task to `DISPATCHING` (skipping on `409`/`None`), then dispatches via `ai_orch` (or injected `dispatch_fn`) and updates the task to `DONE` with metadata or `FAILED` on errors. Includes unit tests asserting the claim-before-dispatch ordering and skip behavior when a task is already claimed.
