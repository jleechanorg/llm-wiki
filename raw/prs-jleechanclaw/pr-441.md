> [!NOTE]
> **Medium Risk**
> Introduces new Slack Web API integration that fetches and posts channel digests and persists cursors; main risks are rate-limiting/permission issues and incorrect state leading to missed or duplicated messages.
> 
> **Overview**
> Adds a new `slack_catchup` workflow that scans *allowed* Slack channels from `openclaw.json`, fetches recent messages (with thread expansion), classifies them (threads/questions/updates vs noise), and renders a structured handoff digest.
> 
> Supports incremental runs via a persisted per-channel `latest_ts` cursor (`~/.openclaw/state/slack_catchup_state.json`), plus CLI options for lookback window, dry-run, and posting the digest back to Slack (optionally in a thread).
>
