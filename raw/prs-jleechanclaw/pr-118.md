## Summary
- Dedupe local Symphony runtime ownership so daemon setup becomes a thin launcher wrapper.
- Stop dynamic workflow generation and use a repo-owned workflow contract file.
- Gate `memory_tracker_issues` enqueue to benchmark-only mode by default.
- Add explicit docs for retained local extensions, non-goals, and rollback plan.

## Intended Architecture
- Symphony runtime behavior is owned by upstream Symphony + repo-owned `WORKFLOW.md`.
- Local `jleechanclaw` scripts are thin wrappers for launch/dispatch.
- Local custom logic is limited to input shaping and benchmark catalogs.

## Changes
- `scripts/setup-symphony-daemon.py`
  - Before: generated `WORKFLOW.md` dynamically and materialized plugin/issue payload state at bootstrap.
  - After: references repo-owned `openclaw-config/symphony/WORKFLOW.md`, validates it exists, and writes launcher/metadata only.
- `scripts/enqueue-symphony-tasks.sh`
  - Before: metadata fallback for `task_plugin_input`, always enqueued to `memory_tracker_issues`.
  - After: requires explicit `SYMPHONY_TASK_PLUGIN_INPUT`; default `SYMPHONY_MEMORY_QUEUE_MODE=benchmark-only`; benchmark plugins enqueue, others skip with explicit message.
- `scripts/prepare-symphony-payload.py`
  - Before: embedded workflow policy block in payload output.
  - After: outputs plugin + issues only.
- Added `openclaw-config/symphony/WORKFLOW.md`.
- Added `docs/symphony-runtime-dedupe.md` (retained extensions, non-goals, rollback).
- Updated `tests/test_symphony_daemon.py` for the new contract.
