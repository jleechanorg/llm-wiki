## Changes

### New: `stage2_reviewer.py` — Independent Reviewer Dispatcher
- Tries Codex CLI first (OpenAI family), Gemini second (Google), Claude last (Anthropic)
- Must use different model family than stage 1 (always Anthropic/Claude)
- **Dispatcher** writes independence attestation fields (`independence_verified`, `model_family_differs_from_stage1`) — the reviewer never touches these
- Parses PASS/FAIL verdict + findings from reviewer markdown output
- Writes `independent_review.md` to the evidence bundle

### Updated: `evidence_bundle.py` — Stage 1 Generator
- Added `--stage2` flag for combined stage 1 + stage 2 pipeline
- Added 5MB cap on `pr_diff.patch` to prevent GitHub file-size rejections
- Human-readable claims.md with artifact file references
- New artifacts: `pr_files.json`, `review_threads.json` for reviewer verification
- Scoped pytest: only runs tests for changed source files

### Updated: `merge_gate.py` — Condition 6 (Evidence)
- **Removed** legacy PASS comment fallback
- All code PRs now require `verdict.json` with `overall == "PASS"` + `stage2.status == "PASS"`
- Gate enforces `independence_verified` + `model_family_differs_from_stage1`
- Fail-closed on missing SHA, parse errors, or pending stage 2

### Fixed: `test_escalation_router.py`
- Added `branch` field to `StuckSessionPayload` so CI mocks are actually called
- 6 pre-existing failures fixed (tests expected KillAndRespawnAction but got WaitForCIAction)
