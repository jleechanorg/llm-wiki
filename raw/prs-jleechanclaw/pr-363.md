## Background
`/agentor` and `/agento_report` had all their logic buried in a shell script (`agento-report.sh`). The commands were fat layers that called `bash ~/.claude/scripts/agento-report.sh` with no inline display and Slack posting via curl.

## Goals
- Command files become thin layers (5 lines max)
- Logic lives in a reusable skill (`agento_report`)
- Report displays **inline** in the conversation
- Slack post uses MCP (`mcp__slack__conversations_add_message`), not curl

## Tenets
- Skill = logic; command = thin dispatch
- No new shell scripts; logic expressed as Claude instructions
- MCP-first for Slack (no curl fallbacks in skills)

- **New**: `.claude/skills/agento_report.md` — full execution skill with GitHub API queries, 6-point green check per PR, inline report table, Slack post via MCP, AO session cross-reference

- **Updated**: `.claude/commands/agentor.md` — slimmed from 17 lines to 7; 1-line alias pointing to skill
