## Background
OpenClaw (gateway agent) handles user requests without visibility into Claude Code memory — user preferences, feedback rules, and project state stored in ~/.claude/projects/*/memory/MEMORY.md files. This creates a blind spot: the same constraints or preferences that Claude Code sessions respect are invisible to OpenClaw.

## Goals
- OpenClaw sessions start with awareness of the user's active projects, feedback patterns, and constraints
- Memory sync runs automatically on a schedule, staying fresh without manual intervention

## Changes
- **scripts/sync-claude-memory.sh**: Finds all MEMORY.md files under ~/.claude/projects/, concatenates them with project headers, writes to ~/.openclaw/workspace/claude-memory-context.md
- **workspace/HEARTBEAT.md**: Adds At session start, read ~/.openclaw/workspace/claude-memory-context.md instruction for OpenClaw
- **launchd/claude-memory-sync.plist.template**: launchd plist running the sync script every 900s (15 min), @REPO_ROOT@ tokens for install script substitution

## Testing
- Script smoke-tested: synced 30 memory files from real ~/.claude/projects/ dirs into context file
- Output verified: headers, project names, and memory entries all present

Closes #orch-wnf
