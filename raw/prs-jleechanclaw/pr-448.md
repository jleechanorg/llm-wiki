## Goals

- Fix gateway token leak (token read inside subshell only, never in parent shell)
- Fix response buffering (direct file write, no shell variable)
- Fix predictable /tmp path (mktemp with random suffix — already used but confirmed)
- Fix process substitution race condition (heredoc Python parser, no `<<<` here-string)
- Add HTTP 4xx/5xx non-JSON error detection
- Replace heredoc SSE pipeline with proper heredoc Python parser
- Add token validation guard

## High-level description of changes

Rewrites the Path B (gateway HTTP) dispatch in the `/claw` command to eliminate several security and reliability issues in `.claude/commands/claw.md`:

- **Token never in parent shell**: the token is now read via `python3` *inside* the `nohup bash -c` subshell — not stored in a parent-shell variable. Removed the stale `GATEWAY_TOKEN=$(python3 -c ...)` assignment that left the token in the parent environment between the assignment and the `unset`.

- **Token validation guard**: before making the HTTP call, the token is validated against a regex (`^[a-zA-Z0-9_-]{20,}$`) — rejects obviously empty/malformed tokens before they reach curl.

- **Subshell trap for all temp files**: the subshell sets `trap cleanup EXIT INT TERM HUP` to guarantee `BODY_FILE`, `HTTP_CODE_FILE`, and `PAYLOAD_FILE` are removed on any exit path.

- **Parent-shell trap for PAYLOAD_FILE**: the parent shell sets `trap 'rm -f "$PAYLOAD_FILE"' EXIT INT TERM` to clean up the payload file if the `nohup` subshell fails to start.

- **HTTP 4xx/5xx detection before JSON parse**: status code is checked
