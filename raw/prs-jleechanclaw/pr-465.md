## Raw Body
### Background
A March 31 bug-hunt review identified a few remaining stability issues that were not fully closed by prior patches:
- Meeting BaaS webhook signature verification could fail on non-UTF-8 payloads.
- Transcript session state was not always removed after idle summary, risking unbounded in-memory growth.
- Worktree cleanup reporting could mark a worktree as removed even when prune did not actually unregister it.

### Goals
- Make webhook signature verification byte-safe and robust for binary-like payloads.
- Ensure idle transcript sessions are summarized and removed from active state.
- Make zombie-worktree cleanup reporting accurate by only reporting actual removals.
- Preserve current functionality and existing tests while addressing these issues.

### Tenets
- Prefer robust edge-case handling over assumptions about encoding/content shape.
- Keep cleanups idempotent and verifiable.
- Minimize behavior changes outside the bug-hunt scope.

### High-level description of changes
This change updates `meetingbaas` to compute SVIX HMAC input from raw bytes instead of UTF-8 decoded body text, preventing decoding errors and signature false negatives. It also changes idle session flushing to evict sessions from the global session map once flushed, preventing stale entries from accumulating.

On the worktree side, cleanup now verifies registration state after `git worktree remove` and after `git worktree prune` before appending paths to the removed list, preventing false-positive cleanup reporting.
