## What This PR Does

Establishes the **design foundation** for the Mission Control delivery pipeline: a deterministic, auditable path from task intake through TDD execution, PR creation, CI/review resolution, to human merge judgment. The PR contains:

1. **A design doc** defining the full lifecycle, state machine (16 states), component responsibilities, security controls, and acceptance criteria
2. **A code fix** tightening the merge-readiness gate to block on unresolved review threads
3. **A design justification doc** explaining why each file exists vs existing alternatives
