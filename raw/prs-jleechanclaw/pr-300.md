## Background

The jleechanclaw repo contains 12 Python orchestration modules (`src/orchestration/`) that were originally built as the MVP implementation. These modules have since been superseded by production-quality TypeScript equivalents in [agent-orchestrator](https://github.com/jleechanorg/agent-orchestrator). Maintaining both creates drift risk, duplicated bug fixes, and confusion about which is canonical.

Related: bd-aa3 dedup tracker bead

## Goals

1. **Mark all 12 Python modules as deprecated** with clear pointers to their AO TypeScript replacements
2. **Preserve backward compatibility** — all existing imports continue to work, with deprecation warnings guiding callers to AO equivalents
3. **Retire `session_reaper.py`** entirely (raises `ImportError` with migration guidance) since AO's `reapStaleSessions` is the canonical replacement
4. **Provide real integration test coverage** proving the pipeline (webhook -> AOEvent -> escalation -> merge gate -> outcomes -> patterns) works with real I/O
5. **Create a reusable real E2E test script** (`testing_llm/e2e_pipeline_real_pr.py`) that creates its own PR, spawns a real AO session, and exercises the full pipeline

## Tenets

- **Prefer real tests over mocks** — integration tests hit real GitHub APIs, real `ao` CLI, real file I/O
- **Honest test classification** — files named `_integration_` are integration tests, not E2E. E2E requires creating its own test data and waiting for real pipeline completion
- **Session safety** — tests must never kill unrelated AO sessions; only the exact spawned session ID is cleaned up
- **Deprecation over deletion** — modules remain importable with warnings, easing migration for any downstream callers

## High-level description of changes

Each of the 12 `src/orchestration/*.py` modules receives:
- A `DEPRECATED` header in the module docstring pointing to the AO TypeScript equivalent
- A `warnings.warn(..., DeprecationWarning)` at import time
- All existing functionality preserved
