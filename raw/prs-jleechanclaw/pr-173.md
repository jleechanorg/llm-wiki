## Background

The openclaw harness had accumulated significant structural debt across two dimensions:

1. **LaunchAgent sprawl** — 25+ plist files across `launchd/`, many orphaned (exit 127, stale gateway version, disabled with no re-enable path, double-dispatching webhooks). Each agent had its own restart logic, env block, and scheduling interval copy-pasted verbatim.

2. **Webhook pipeline fragmentation** — the GitHub webhook path was split across 8 files (`webhook_daemon.py`, `webhook_bridge.py`, `webhook_ingress.py`, `webhook_queue.py`, `webhook_worker.py`, `webhook_metrics.py`, `escalation_handler.py`, `escalation_router.py`). Adding a feature required touching 4–6 files; tracing a single event required jumping across all of them.

Both problems had the same root cause: **copy-paste growth with no deduplication pass**. ~30 utility functions (datetime formatting, Slack normalization, path resolution, JSON I/O) existed in 3–5 variants scattered across modules.

## Solution — 4 phases, all landed in this PR

### Phase 1 — Remove orphaned LaunchAgents
Deleted 5 dead plists that were either non-functional or actively harmful:
- `ai.openclaw.consensus.plist` — stale v2026.2.12 gateway, superseded
- `ai.symphony.daemon.plist` — exit 127 on every invocation, never functional
- `ai.worldarchitect.pr-automation.fixpr.plist.disabled` + `fix-comment.plist.disabled` — disabled with no re-enable path; PR automation now runs inside the gateway cron
- `ai.openclaw.github-intake.plist` — caused double-dispatch: events were ingested by both this agent and `webhook_ingress.py`
