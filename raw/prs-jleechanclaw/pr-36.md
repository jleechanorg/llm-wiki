## Summary

- Defines a generalized EvidencePacket schema (src/orchestration/evidence.py) for auditable execution proof at each pipeline stage.
- Adds webhook parsing helpers to gh_integration.py for normalizing GitHub event payloads.
- Extends webhook_bridge.py with receive_github_event() (trusted-actor gate, HMAC validation) and current_github_event_mode() (auto-detects webhook vs polling).
- Adds .github/workflows/agent-pr-trigger.yml: self-contained GitHub Actions webhook-first trigger; no external server required.
- Updates design doc to resolve all Known Open Decisions and document evidence attach points, webhook integration, and resolved policies.

## Changes

- src/orchestration/evidence.py (new): EvidencePacket, ArtifactType, PipelineStage, EvidenceLevel, completeness derivation, as_dict() serialization.
- src/orchestration/gh_integration.py: parse_github_webhook_{pr_number,repo,actor,author_association}() helpers.
- src/orchestration/webhook_bridge.py: receive_github_event(), GitHubEventMode, current_github_event_mode(), TRUSTED_AUTHOR_ASSOCIATIONS.
- .github/workflows/agent-pr-trigger.yml (new): triggers on issue_comment + pull_request_review; trusted-actor gate; graceful fallback when MC not configured.
- src/tests/test_evidence.py (new): 44 tests covering evidence schema, completeness, serialization, webhook parsing, event bridge.
- docs/mcp-mail-openclaw-mission-control-design.md: evidence attach-point table, webhook integration section, resolved-decisions table, updated MVP scope + acceptance criteria.

## Known Limitations

- receive_github_event() HMAC path is unit-tested but not live-tested (no server running).
- agent-pr-trigger.yml requires MISSION_CONTROL_BASE_URL/TOKEN/BOARD_ID vars/secrets; exits cleanly with notice when absent.
- Polling remains default for CI/review monitoring; real-time webhook upgrade
