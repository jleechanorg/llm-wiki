## Summary
Adds **automatic** full **6/6** `staging-canary.sh` runs on PRs that touch `openclaw.json`, using a **self-hosted** GitHub Actions runner (your Mac with the staging gateway). GitHub-hosted runners cannot reach localhost or load local extensions.

## Changes
- **`.github/workflows/staging-canary-full.yml`** — job `staging-canary-full` on labels `self-hosted` + `openclaw-staging`, default port **18810** (override via repo variable `STAGING_CANARY_PORT`).
- **`scripts/staging-canary.sh`** — if port is **18810** and `~/openclaw/openclaw.staging.json` exists, use it for schema checks; optional `OPENCLAW_STAGING_CONFIG` override.
- **`.github/SELFHOSTED_TEST.md`** — runner registration steps.
- **`docs/STAGING_PIPELINE.md`** + **`.html`**, **`README.md`** — Stage 3b documented.

## Not in scope
- **`monitor-agent.sh`** — still LaunchAgent-only (Slack/doctor/thread checks); not suitable as a PR gate.

## After merge (repo admin)
1. **Settings → Actions → Runners → New self-hosted runner** for `jleechanorg/jleechanclaw`.
2. Add label **`openclaw-staging`** (in addition to `self-hosted`).
3. Keep **`ai.openclaw.staging`** running on the runner machine.

**Opt-out:** set repo variable `ENABLE_STAGING_CANARY_SELFHOSTED=false` if the runner is offline.
