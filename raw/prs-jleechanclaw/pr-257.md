## Summary

- Design doc for replacing comment-based evidence gate with git-tracked evidence bundles
- Two-stage pipeline: agent self-review then independent LLM verification
- Standard schema: `docs/evidence/{repo}/PR-{N}/{YYYYMMDD}_{HHMM}_pst/`
- Merge gate reads `verdict.json` (requires independent reviewer sign-off)

## Changes

- `roadmap/EVIDENCE_REVIEW_SCHEMA.md` -- full design with directory schema, verdict.json spec, pipeline diagram, implementation plan
- `.beads/issues.jsonl` -- epic orch-j9e0 with 7 child tasks

## Key Design Decisions

1. **Git-tracked, not comments** -- evidence bundles are committed, immutable, auditable
2. **Two-stage independence** -- coding agent cannot rubber-stamp own work; stage 2 uses different model
3. **Scope-limited** -- only code PRs require evidence (src/, scripts/, lib/, SOUL.md); docs skip
4. **Codex preferred for stage 2** -- different model family from Claude coding agent
