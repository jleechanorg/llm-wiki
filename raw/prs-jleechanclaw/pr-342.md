## Background

The AI reviewer stress test infrastructure (v2 Python modules) was built on `feat/ai-reviewer-stress-test-v2` but the core fix loop (step 5) was stubbed out — it logged "Would run agento" without actually invoking any agent. The test PR → real PR pipeline never executed end-to-end.

## Goals

1. Implement the real agento fix loop: agent invocation → push → re-review → retry
2. Create real PRs in `jleechanorg/jleechanclaw` with AI-reviewer-discovered fixes
3. Fix security vulnerability (`eval()` in review_collector.py)
4. Add launchd plist for 4-hour scheduled runs

## Tenets

- Real execution over stubs — no "simulated" steps
- Security first — no `eval()` on untrusted input
- Fail gracefully — missing CLI tools logged, not crashed

## High-level description of changes

Replaced the stubbed step 5 in `run.py` with a real agent fix loop that:
- Discovers `ao` or `claude` CLI (graceful fallback)
- Formats AI reviewer comments into agent-digestible summaries
- Retries up to 3 times with 15-min re-review waits between attempts
- After success, diffs fixes and creates a real PR in the original repo via `gh api` REST
- Always closes the test PR after validation

Also fixed `eval()` → `json.loads()` security bug and added the launchd plist.

- Verified `eval()` is completely removed from codebase
- Verified agento loop has proper retry/timeout logic
- Verified PR creation uses REST API (avoids GraphQL rate limits)
