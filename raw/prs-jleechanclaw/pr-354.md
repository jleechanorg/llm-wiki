## Summary

## Background
When an agent session ends mid-flight (timeout, crash, or early exit), the PR is left orphaned — no active session, no respawn, no forward progress. The `agent-exited` reaction was `action: notify` only, so the orchestrator just sent a notification and moved on.

This is the primary blocker for full autonomy: PRs with CI failures, CR CHANGES_REQUESTED, or unresolved Bugbot findings would sit forever without a human manually re-triggering an agent.

## Goals
- Orphaned PRs automatically get a replacement agent spawned when the previous session exits
- Replacement agent triages state before acting (exits if already merged/green, resumes from highest-priority blocker otherwise)
- No wasted work: agent reads existing PR context before starting

## Tenets
- Config-first: fix is one reaction change, no new Python code
- Fail-safe: triage gate prevents infinite loops on already-merged PRs

## High-level description of changes
Change `reactions.agent-exited` from `action: notify` to `action: send-to-agent` with a triage message that:
1. Exits immediately if PR is merged/closed
2. Posts green signal and exits if all 6 conditions already met
3. Otherwise resumes from highest-priority remaining blocker

## Testing
Config change — verified diff is correct, no Python tests required.

## Low-level details
- **agent-orchestrator.yaml**: `reactions.agent-exited`: `action: notify` → `action: send-to-agent` with triage message
