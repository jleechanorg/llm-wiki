## Background
Two bugs identified via post-merge review of PRs from the last 3 days:
- **orch-t63 (P0)**: PR #354 introduced `agent-exited` → `send-to-agent` reaction with no `retries`/`escalateAfter` guard, creating unbounded respawn loops
- **orch-pwo (P1)**: PR #378 introduced `jq` parsing of AO spawn output that silently drops bug findings on parse failure

## Goals
1. Cap agent-exited respawns to prevent infinite loops
2. Make bug-hunt-daily.sh fail-closed on invalid JSON output

## Tenets
- Minimal surgical changes only
- Match existing patterns already in the codebase

## High-level description of changes
**agent-orchestrator.yaml**: Added `retries: 3` and `escalateAfter: 2` to `agent-exited` reaction, matching the `ci-failed` pattern already present. Caps respawn attempts and escalates after 2 minutes if agent keeps exiting without posting the green signal.

**scripts/bug-hunt-daily.sh**: Added `jq empty` validity check before parsing. Non-JSON output (API errors, timeouts, plain text) now warns and skips with `continue` instead of counting as zero. `null` return from jq (missing field) also warns and skips rather than silently passing the case guard.

- jq validity pattern (`jq empty`) is standard shell idiom

- **agent-orchestrator.yaml**: 2 lines added to `agent-exited` block
- **scripts/bug-hunt-daily.sh**: `jq empty` pre-check + `null`/unexpected-value guards replace silent `|| echo "0"` fallback
