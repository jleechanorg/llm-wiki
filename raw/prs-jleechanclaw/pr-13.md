## Summary

- Adds context expansion instruction to SOUL.md
- When jleechanclaw sees a message referencing prior conversation ("we discussed", "from earlier", etc.), it now reads the last 20 DM messages and extracts the full spec before creating a Mission Control task
- The coding agent receives an expanded description — never a raw stub like "build the discord bot we discussed"

## Changes

**Before:** `"build the discord bot we discussed"` → MC task with that literal string → minimax agent guesses

**After:** `"build the discord bot we discussed"` → jleechanclaw reads DM history → extracts: eng_qa agent, sandbox non-main, tool denylist, slash commands → MC task with full spec → minimax agent builds correctly

## Why config, not code

openclaw already provides everything needed:
- `dmHistoryLimit: 50` — last 50 DM messages already in context
- `memorySearch.enabled: true` — memory searched on every turn
- `sessionMemory: true` — session context preserved

The fix is a SOUL.md instruction, not new Python. See `roadmap/NATURAL_LANGUAGE_DISPATCH.md`.
