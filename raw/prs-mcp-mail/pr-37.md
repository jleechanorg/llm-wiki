## Summary
- Add first-class build slot coordination tools (acquire/renew/release) with FastMCP exposure plus CLI plumbing.
- Refresh CLI ergonomics: standalone `amctl-env`, updated guard install/status, simplified identity resolution, richer env dumps.
- Document and instrument manual validation via the new roadmap reports and fixed manual scenario tests.
- Strengthen regression coverage (agent-name sanitization, FastMCP resources/macros, CLI/build-slot suites) and move pytest-asyncio into main deps for runtime parity.

## Feature Highlights
### Build Slot Coordination
- Introduce `src/mcp_agent_mail/slots.py` with advisory lease management backed by the archive filesystem and async file locks.
- Surface the new MCP tools (`acquire_build_slot`, `renew_build_slot`, `release_build_slot`) with logging, metadata, and documentation entries so agents can coordinate long builds/tests.
- Extend CLI `am-run` to understand safe filesystem components, cache keys, and slot bookkeeping (still gated by WORKTREES_ENABLED — see review findings below for current gate bug).

### CLI / Guard Updates
- Rename `amctl env` to `amctl-env` and emit plain `KEY=VALUE` lines for scripting; compute slugs/project UID directly from the filesystem.
- Simplify guard install path resolution (no DB lookup required), expose `_env_flag` helper, and update `mail status` / `guard status` diagnostics to show remote normalization, hook locations, and environment flags.
- Update CLI integration tests to cover the renamed commands, guard flows, and slot conflict handling; guard-related tests now run synchronously to avoid nested event loops.
