## Summary

Adds bidirectional Slack integration to MCP Agent Mail, enabling agents to communicate through Slack channels.

## Changes

### New Files
- **`src/mcp_agent_mail/slack_integration.py`** (794 lines) - Core Slack client module
  - `SlackClient` async wrapper for Slack Web API
  - Thread mapping between MCP `thread_id` and Slack `thread_ts`
  - Signature verification for webhook security
  - Message formatting using Slack Block Kit

- **`SLACK_BIDIRECTIONAL_SYNC.md`** (371 lines) - Comprehensive setup guide
  - Architecture diagram and setup instructions
  - Thread mapping explanation
  - Troubleshooting guide and security recommendations

- **`tests/integration/test_slack_mcp_mail_integration.py`** (699 lines) - 12 integration tests

### Modified Files
- **`src/mcp_agent_mail/app.py`** (+478 lines)
  - Server lifespan management for SlackClient
  - Fire-and-forget notification hooks for message creation/acknowledgment
  - 3 new MCP tools: `slack_post_message`, `slack_list_channels`, `slack_get_channel_info`

- **`src/mcp_agent_mail/config.py`** (+65 lines)
  - New `SlackSettings` dataclass with 15+ configuration options
  - Secure-by-default: Slack disabled unless explicitly enabled

- **`src/mcp_agent_mail/http.py`** (+207 lines)
  - `/slack/events` webhook endpoint with signature verification
  - Rate limiting exemptions for Slack webhooks
  - Auto-creates `SlackBridge` system agent for message sourcing

- **`.env.example`** - 18 new Slack environment variables
- **`README.md`** - Documentation updates with setup guide
- **`tests/test_lazy_loading.py`** - Updated tool counts for new Slack tools

## Features

- **Slack → MCP**: Messages in configured Slack channels automatically create MCP messages
- **MCP → Slack**: MCP messages automatically post to Slack via background notifications
- **Thread Mapping**: Slack threads map to MCP `thread_id` for conversation continuity
- **Rich Formatting**: Block Kit support for formatted messages
- **Secure by Default
