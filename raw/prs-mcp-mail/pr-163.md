## Summary
- Fixed SessionStart hook format in `scripts/add_global_auto_register.sh` to use the correct matcher-based structure with `hooks` array
- Fixed `_exampleHooks.SessionStart_example` in `.claude/settings.json` to use the correct format

## Problem
Claude Code hooks require a matcher-based structure where each hook entry must contain a "hooks" array. The previous format caused settings errors:

```
hooks.SessionStart[0].hooks: Expected array, but received undefined
```

## Fix

**Incorrect format (before):**
```json
"SessionStart": [
  {
    "type": "command",
    "command": "..."
  }
]
```

**Correct format (after):**
```json
"SessionStart": [
  {
    "hooks": [
      {
        "type": "command",
        "command": "..."
      }
    ]
  }
]
```
