## Summary
Updated the regex pattern in `_parse_tool_names_from_output` to correctly match tool names that use hyphens as separators in addition to underscores.

## Changes
- Modified the regex pattern from `r"mcp__mcp(?:_)?agent(?:_)?mail__([a-zA-Z_][a-zA-Z0-9_]*)"` to `r"mcp__mcp[-_]?agent[-_]?mail__([a-zA-Z_][a-zA-Z0-9_]*)"`
- Changed optional underscore groups `(?:_)?` to character classes `[-_]?` to match both hyphen and underscore separators
- Added clarifying comments documenting the three supported naming formats:
  - Underscore format: `mcp__mcp_agent_mail__send_message`
  - Hyphen format: `mcp__mcp-agent-mail__send_message`
  - No separator format: `mcp__mcpagentmail__send_message`

## Details
The previous regex pattern only matched optional underscores between `mcp`, `agent`, and `mail` components. This change allows the pattern to match both hyphens and underscores, making the tool name parsing more flexible and robust to different naming conventions.
