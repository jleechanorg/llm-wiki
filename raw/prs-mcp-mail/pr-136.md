## Raw Body
This removes the functionality that copied messages to the local filesystem in the .mcp_mail/projects/ directory structure. All message storage is now handled exclusively via SQLite database.

Changes:
- Replaced storage.py with stub functions for backwards compatibility
- Removed archive integration from app.py (message delivery, agent profiles)
- Updated config.py to mark archive settings as deprecated
- Simplified guard.py and slots.py since they depended on archive
- Deleted archive-related tests (test_default_mcp_mail_storage.py, test_project_key_storage.py, test_gitignore_config.py)
- Simplified test_storage_edges.py

The stub functions in storage.py allow existing code to continue running without errors - they simply do nothing. This maintains backwards compatibility while removing the filesystem-based message archiving.
