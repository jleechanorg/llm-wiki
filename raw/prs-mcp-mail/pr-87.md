## Raw Body
When sending messages to agents that don't exist yet, the system now:
1. Creates a placeholder agent (is_placeholder=True) to receive the messages
2. Stores messages in the placeholder's inbox
3. Allows later registration to "claim" the placeholder and access all pending messages

Key changes:
- Add is_placeholder field to Agent model (default False)
- Add _create_placeholder_agent() function for auto-creating placeholders
- Add _get_placeholder_agent_globally() helper for finding placeholders
- Modify send_message to create placeholders instead of full agents for unknown recipients
- Modify _get_or_create_agent to claim placeholder agents during registration
- Add database migration for is_placeholder column
- Add comprehensive tests for the placeholder flow

This enables the pattern: send messages to an agent that doesn't exist yet, and when they register later, they can read all their pending messages.
