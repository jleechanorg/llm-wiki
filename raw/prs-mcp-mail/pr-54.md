## Problem
The thread resource template was rejecting URIs with query parameters when the project path contained slashes (e.g., project=/Users/jleechan/mcp_mail). This caused "Unknown resource" errors even though the project parameter was explicitly provided in the query string.

## Root Cause
FastMCP's URI template matching uses the pattern `{param}` which translates to regex `[^/]+` - matching any character except `/`. When a URI like `resource://thread/1?project=/Users/test/backend` was provided, the regex would stop matching at the first `/` in the query parameter value, causing the entire URI to fail the template match.

## Solution
Changed the resource template from:
  `resource://thread/{thread_id}`
to:
  `resource://thread/{thread_id*}`

The `*` suffix tells FastMCP to use the greedy pattern `.+` instead of `[^/]+`, allowing it to match the entire URI including query parameters with slashes in their values.
