## Raw Body
This change enables agent names to serve as the primary key for routing instead of project-scoped identifiers. Key changes:

Models (models.py):
- Agent.project_id is now Optional[int] with default None
- Message.project_id is now Optional[int] with default None
- FileReservation.project_id remains required (file reservations are project-scoped)

Database (db.py):
- Added migration _migrate_project_id_nullable() to convert existing tables
- SQLite tables are recreated with nullable project_id columns
- Existing data retains project_id values; new records can have NULL

Application (app.py):
- Added DEFAULT_PROJECT_KEY = "global" for agents without explicit project
- Added _get_default_project() helper for backwards compatibility
- Added _get_project_for_agent() to look up agent's project
- Updated _create_message() to accept Optional[Project]
- Made project_key optional in register_agent tool (defaults to global project)
- Made project_key optional in send_message tool (derives from sender)

Slack Integration (http.py):
- Updated Slack webhook to derive project from thread context first
- Falls back to sync_project_name or default project
- Agents are looked up globally (not filtered by project)

This is backwards compatible:
- Existing APIs continue to work with project_key parameter
- New code can omit project_key to use default project
- Existing data retains project associations
