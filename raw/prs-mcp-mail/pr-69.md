## Raw Body
Handle anyio.ClosedResourceError gracefully in StatelessMCPASGIApp when
clients (like Codex) disconnect during message routing. The crash occurred
because the message router in the MCP library tried to read from streams
that were closed when the client disconnected.

Changes:
- Cancel server_task before exiting context to prevent race conditions
- Catch anyio.ClosedResourceError and suppress gracefully
- Handle ExceptionGroup/BaseExceptionGroup containing ClosedResourceError
- Add comprehensive unit tests for the fix
- Add integration tests that start real server and verify behavior

Verified locally by:
1. Starting real MCP server
2. Making streaming requests and aborting mid-stream
3. Confirming ClosedResourceError is reproduced
4. Confirming fix prevents crash and server continues to work

This fixes the error:
  ERROR:mcp.server.streamable_http:Error in message router
  anyio.ClosedResourceError
  ERROR:    Exception in ASGI application
