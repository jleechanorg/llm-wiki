## Summary

Fixes Python 3.14 incompatibility by implementing defense-in-depth protection against `collections.abc.ByteString` ImportError in the beartype dependency chain.

## Problem

Python 3.14 removed `collections.abc.ByteString` (deprecated in 3.13), causing startup failures:
- **Root cause:** beartype (transitive dependency: fastmcp → docket → key_value → beartype) imports the removed module
- **Impact:** Server crashes before initialization when Python 3.14 is selected
- **Affected paths:** PyPI launchers and environments that prefer latest Python version

## Solution - Three Layers of Protection

### 1. Package-Level Constraint ✅
**File:** `pyproject.toml`
- Restrict `requires-python = ">=3.11,<3.14"`
- PyPI will refuse installation on Python 3.14+

### 2. Launcher-Level Selection ✅
**Files:** `scripts/run_server_pypi.sh`, `scripts/run_server_local_build.sh`
- Remove Python 3.14 from interpreter candidate list
- Prefer order: 3.13 → 3.12 → 3.11
- Scripts will never select Python 3.14, even if available

### 3. Runtime-Level Guard ✅
**File:** `src/mcp_agent_mail/cli.py`
- Added startup version check in `serve_http()`
- Detects Python >=3.14 and exits with clear error message
- Displays supported versions and incompatibility reason

### 4. Documentation Updates ✅
**Files:** `README.md`, `CLAUDE.md`
- Updated badges and installation instructions
- Added dedicated Python Compatibility section
- Corrected all Python 3.14 references to 3.13
- Explained beartype incompatibility and monitoring plan

## Testing

Verified all changes:

- ✅ Launcher scripts: Python 3.14 removed from selection
