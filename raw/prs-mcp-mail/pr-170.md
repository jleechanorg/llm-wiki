## Summary

This PR fixes three critical bugs identified in the MCP Mail codebase:

1. **Pydantic NameError fix (HIGH)**: Remove `from __future__ import annotations` causing type resolution issues
2. **Security update (HIGH)**: Update fastmcp from 2.12.5 to >=2.14.0
3. **Compatibility fix (MEDIUM)**: Update authlib from <1.6 to >=1.6.5

Additionally, fixes a test regression caused by the dependency updates.

## Changes

### 1. Remove `from __future__ import annotations` (app.py:3)

**Problem**: With PEP 563, this import causes type hints to become strings evaluated at runtime. Pydantic/FastMCP cannot resolve types like `Optional[str]`, resulting in:
```
NameError: name 'Optional' is not defined
```

**Solution**: Remove the import. The codebase doesn't need it, and removing it allows Pydantic to properly evaluate type hints at definition time.

**File**: `src/mcp_agent_mail/app.py`

### 2. Update fastmcp version (pyproject.toml:28)

**Problem**: fastmcp 2.12.5 has known security vulnerabilities.

**Solution**: Update to `fastmcp>=2.14.0`

**File**: `pyproject.toml`

### 3. Update authlib version (pyproject.toml:56)

**Problem**: authlib constraint `>=1.5.2,<1.6` conflicts with fastmcp>=2.14.0 which requires authlib>=1.6.5.

**Solution**: Update to `authlib>=1.6.5`

**File**: `pyproject.toml`

### 4. Fix test regression (test_slack_webhook_api.py)

**Problem**: The dependency updates changed when `_slack_client` is initialized during lifespan startup. The test was setting the dummy client BEFORE entering the lifespan context, causing it to be overwritten.

**Solution**: 
- Move `app_module._slack_client = dummy_client` AFTER lifespan starts
- Add explicit `SLACK_SYNC_THREAD_REPLIES=true` environment variable

**File**: `tests/integration/test_slack_webhook_api.py`
