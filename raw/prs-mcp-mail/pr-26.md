## Raw Body
Implemented the pre-commit framework (Python equivalent of Husky) to automate code quality checks on git commit and git push. This catches issues early and ensures consistent code quality across all contributors.

Changes:
- Installed pre-commit framework via uv tool
- Configured pre-commit and pre-push hooks in .git/hooks/
- Updated scripts/setup_git_hooks.sh to use pre-commit framework
- Updated CONTRIBUTING.md with pre-commit documentation
- Added .githooks/README.md explaining legacy custom hooks
- Auto-fixed trailing whitespace and EOF issues across codebase

Pre-commit hooks now run:
• Ruff linter (with auto-fix)
• Trailing whitespace fixer
• End-of-file fixer
• YAML/JSON syntax validation
• Large file detection
• Merge conflict detection
• Type checking with ty (non-blocking)
• Fast unit tests

Pre-push hooks run:
• Integration tests

Configuration is in .pre-commit-config.yaml and can be customized as needed.
