## Raw Body
This commit increases various timeout values across the codebase to prevent timeout errors when fetching inbox messages, particularly in slow or high-load scenarios.

Changes:
- Gunicorn worker timeout: 120s → 300s (5 minutes)
- SQLite connection timeout: 30s → 60s
- SQLite PRAGMA busy_timeout: 30s → 60s (30000ms → 60000ms)
- AsyncFileLock timeout: 60s → 120s
- AsyncFileLock stale timeout: 180s → 300s
- archive_write_lock default timeout: 60s → 120s in storage.py and app.py

These changes provide more breathing room for operations that may involve:
- Large inboxes with many messages
- Slow database queries under concurrent load
- File lock contention during parallel archive operations
- Network delays or resource constraints

The increased timeouts are still reasonable and will eventually fail if there's a genuine deadlock or stuck operation.
