## Summary
- SQLite storage (messages, agents, projects) now ALWAYS works regardless of archive settings
- Archive storage (`.mcp_mail/` git repo with .md files) is now optional and disabled by default
- Core MCP operations (`register_agent`, `ensure_project`) no longer require archive to be enabled

## Problem
Previously, when `STORAGE_LOCAL_ARCHIVE_ENABLED=false`, core operations like `register_agent` would fail with:
```
Error: Local .mcp_mail storage is disabled (STORAGE_LOCAL_ARCHIVE_ENABLED=false) and
project-key storage is not enabled.
```

## Solution
- Added helper function `is_archive_enabled(settings)` in `storage.py`
- Made `ensure_archive` calls conditional in `app.py` for `ensure_project` and `register_agent`
