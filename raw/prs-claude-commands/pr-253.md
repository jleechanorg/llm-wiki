## Summary
Refactored the installation script to properly support Claude Code's standard `~/.claude` directory structure with separate subdirectories for agents, commands, scripts, and skills. The installer now copies plugin files from the repository to the appropriate system-level directories instead of installing locally.

## Key Changes

- **Directory Structure**: Changed from local `.claude/commands` directory to system-level `~/.claude/` with subdirectories:
  - `~/.claude/agents/` for agent definitions
  - `~/.claude/commands/` for command definitions and modules
  - `~/.claude/scripts/` for executable scripts
  - `~/.claude/skills/` for skill definitions

- **Installation Logic**: 
  - Added `install_agents()` function to copy agent markdown files
  - Added `install_skills()` function to copy skill markdown files
  - Enhanced `install_commands()` to handle both top-level files and subdirectories (e.g., `_copilot_modules`, `_shared`)
  - Updated `install_scripts()` to copy and make scripts executable in the target directory
  - Modified `install_infrastructure()` to only make root-level scripts executable without copying

- **Path Management**:
  - Introduced `PLUGIN_SRC_DIR` to track the plugin repository location
  - Added `CLAUDE_HOME` environment variable support (defaults to `$HOME/.claude`)
  - Separated source paths (`SRC_*_DIR`) from target paths (`CLAUDE_*_DIR`)

- **Plugin Configuration**: Updated `plugin.json` to include scripts directory in the plugin manifest

- **Code Quality**: Improved logging clarity, removed emoji decorations for consistency, and added better error handling with warnings for missing source directories

## Implementation Details

- Uses `cp -f` for file copying to ensure overwrites during updates
- Preserves directory structure for command subdirectories with `cp -rf`
- Maintains backward compatibility by detecting legacy directory structures
- Validates installation by checking file counts in target directories
