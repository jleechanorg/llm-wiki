## Summary

This fix adds custom paths to `plugin.json` to point to the `.claude/` subdirectories, enabling the plugin to work correctly when installed via `/plugin install`.

## Problem

The current repo structure uses `.claude/commands/` (for standalone/manual installation) but the Claude Code plugin system expects commands at root level (`commands/`). This causes commands to not appear when installed via `/plugin install`.

## Solution

Add custom paths in `plugin.json` to tell the plugin system where to find components:

```json
{
  "commands": [".claude/commands/"],
  "agents": [".claude/agents/"],
  "skills": [".claude/skills/"]
}
```

This allows:
- ✅ Existing export workflow to remain unchanged (still exports to `.claude/`)
- ✅ Plugin installation to work correctly (paths point to `.claude/`)
- ✅ Both standalone (manual copy) and plugin installation methods to work

## Test plan

1. Install plugin: `/plugin install claude-commands@claude-commands`
2. Verify commands appear: `/help` should show commands
3. Test a command: `/pr` or `/execute` should work
