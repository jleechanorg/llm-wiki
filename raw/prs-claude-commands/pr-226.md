## Summary
This PR removes the custom agent system documentation and all agent implementations from the `.claude/agents/` directory. The agent system, while sophisticated, is being deprecated in favor of a simpler, more maintainable approach.

## Changes
- **Removed** `agents/CLAUDE.md` - Comprehensive documentation of the custom agent architecture, including agent inventory, coordination patterns, and implementation guidelines
- **Removed** 7 specialized agent implementations:
  - `code-review.md` - Multi-language code analysis specialist
  - `codex-consultant.md` - External Codex CLI analysis specialist
  - `cerebras-consultant.md` - Cerebras Model Studio integration specialist
  - `gemini-consultant.md` - External Gemini consultation specialist
  - `copilot-fixpr.md` - PR issue implementation specialist
  - `long-runner.md` - Extended task execution specialist
  - `testexecutor.md` & `testvalidator.md` - Testing specialists
  - `code-centralization-consultant.md` - Code consolidation specialist
- **Removed** supporting agent files and related documentation

## Rationale
The custom agent system provided sophisticated task delegation through the Task tool, but introduced complexity in:
- Maintaining multiple specialized agent definitions
- Coordinating between agents with separate context windows
- Managing timeout enforcement and external tool integration
- Documenting and training on the custom architecture

This removal simplifies the codebase and reduces maintenance burden while allowing for more direct, streamlined approaches to code analysis and automation tasks.

## Impact
- Existing workflows relying on the Task tool with `subagent_type` parameters will need to be refactored
- Direct Claude execution or alternative automation approaches should be used instead
- Documentation burden is reduced for future maintainers
