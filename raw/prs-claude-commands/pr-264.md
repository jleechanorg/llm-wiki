## Summary
This PR consolidates the planning command structure by removing the deprecated `plan.md` command file and enhancing the `planexec.md` workflow with a mandatory code simplification step. The changes streamline the command hierarchy and ensure code quality standards are applied to all plan executions.

## Key Changes
- **Removed** `.claude/commands/plan.md` - Deprecated planning command file that has been superseded by `planexec.md`
- **Added Phase 5: Simplify** to `planexec.md` - A mandatory final step that automatically invokes `/simplify` after execution to review changed code for reuse opportunities, quality, and efficiency
- **Updated command references** in `CLAUDE.md` to reflect that `planexec.md` is now the primary planning command (replacing `plan.md`)
- **Enhanced universal composition** documentation to include `/simplify` as a mandatory step in the planning workflow

## Implementation Details
- The new Phase 5 ensures that every `/plan` and `/planexec` execution produces clean, efficient, deduplicated code
- `/simplify` is invoked automatically and any issues found must be fixed before the plan is considered complete
- Updated the command composition principles to highlight `/simplify` as **MANDATORY** rather than optional
- This change enforces code quality standards as part of the standard execution workflow rather than as an afterthought
