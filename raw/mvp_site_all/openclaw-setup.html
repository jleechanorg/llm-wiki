OpenClaw Setup for WorldArchitect

Use this page to expose a local OpenClaw gateway to a remote WorldArchitect server.

Quick Start

  1. Install and run OpenClaw locally on this machine.
  2. Start your gateway and verify http://127.0.0.1:18789/health.
  3. Run: ./openclaw_gateway_tunnel.sh
  4. Copy the printed public HTTPS URL.
  5. In Settings, set Gateway URL to that URL and set provider to OpenClaw.

Download the tunnel helper

Download openclaw_gateway_tunnel.sh

Notes

  • Local localhost:18789 is not reachable by Cloud Run by itself.
  • The public URL is needed for PR preview / remote environments.
  • If localhost.run fails, use cloudflared or ngrok (script auto-selects available providers).
  • Script order: localhost.run → cloudflared → ngrok.
