## Goal
Ensure the proxy status output reflects the actual upstream endpoint when running in alternate provider modes.

## Modifications
- Persist the upstream base URL used during provider configuration for later status checks.
- Add a helper to read the stored upstream URL with a fallback to the default Codex backend.
- Update startup and status output to display the upstream URL instead of the local usage hint.

## Necessity
Without this change, the status command always shows the localhost usage hint, which is misleading when the proxy is forwarding to an alternate provider such as Cerebras.
