# Bead: Include streaming checks in Real E2E smoke comments

## GOAL
Make `/smoke` real-E2E PR comments explicitly list streaming contract checks that are already executed by the canonical smoke suite.

## PROBLEM
The mock-mode comment renders scenario-level test details, which already includes
streaming chunk timing contracts.
The real-E2E comment uses a static checklist in the formatter and does not mention
streaming contracts, even when they are part of the executed test suite.

## MODIFICATION
Update the default item list used by `formatRealE2EComment` in
`scripts/lib/comment-formatter-core.cjs` to include:
- `Streaming chunk timing contracts (character / think / god)`

## NECESSITY
Improve transparency in PR feedback so reviewers can see that streaming behavior is
covered during `/smoke` checks and avoid false assumptions about test scope.

## INTEGRATION PROOF
- Updated file: `scripts/lib/comment-formatter-core.cjs`
- Real-E2E comments generated by `scripts/mcp_smoke_comment.js` now include the streaming
  check item in their executed-test list.
