## Background

The current `chat_worker` tool uses regex-based voice heuristics and branch entries lack character consistency. The fiction's core claim — distinct AI workers with personalities — is undermined by generic, inconsistent character rendering. This PR implements the Phase 2 Novel Persona Library per the design spec at `docs/superpowers/specs/2026-03-30-novel-persona-library-design.md`.

## Goals

- 6 named worker archetypes with distinct voice, speech patterns, and emotional baselines
- Deterministic persona lookup by workerId — no randomness, no database
- Branch generator and daily summary include persona attribution
- chat_worker improves LLM prompt with persona context
- All changes backward-compatible with existing voice system

## Tenets

- Zero new dependencies — persona data is a TypeScript object
- Deterministic lookup — same workerId always returns same persona
- Graceful degradation — existing behavior preserved when persona not available

## High-level description of changes

### New: `src/novel/personas.ts`
Six archetypes (Optimizer, Debugger, Architect, Tester, Deployer, Documentarian), each with:
- `id`, `name`, `role`
- `voiceSamples[]` — 3 example utterances
- `speechPatterns[]` — behavioral rules
- `emotionalBaseline` — one-sentence characterization
- `systemPromptSnippet` — LLM injection text

`getPersona(workerId)` uses `workerId.replace(/\D/g, '').slice(-2) % 6` for deterministic bucketing.

`getVoiceInstructions(persona)` returns a formatted string for LLM system prompt injection.

### Updated: `src/novel/branch-generator.ts`
- Imports and calls `getPersona(sessionId)` and `getVoiceInstructions(persona)`
- POV line shows persona name: `*POV: ao-826 · 2026-03-29 · The Optimizer*`
- Adds `*Voice: {emotionalBaseline}*` line
- Adds `*Persona: arc-opt — The Optimizer*` to continuity section
- Adds `persona_id`, `persona_name`, `persona_role` to traceability block
- `makeBranchEntryMetadata` includes persona fields
