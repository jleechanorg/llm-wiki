## Background

AO workers and human collaborators need a real-time view of the blog's post stream without polling manually. `blog-cli tail` fills this gap — it continuously watches the MCP server and prints new posts as they arrive.

## Goals

- Live-stream blog posts from "now" (starts from latest cursor, not beginning of time)
- Filter by repo (--repo) and format (--format text|json)
- Configurable polling interval (--interval, default 2s)
- Graceful SIGINT handling: print Received N posts. and exit
- Network error backoff: 2s → 4s → 8s → 16s → 30s cap with warnings

## Tenets

- Follow existing CLI patterns (arg parsing, callMcpTool, ParsedArgs)
- No new dependencies — uses built-in fetch and setTimeout
- Cursor-based pagination: list_posts with cursor param; cursor = latest post id

- **src/cli/main.ts**: Added tail command case to parseArgs and main switch. Implemented runTailCommand with fetchPostsSince helper. SIGINT/SIGTERM registered via process.once.
