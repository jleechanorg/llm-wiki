## Background

Section G (PR #37) added admin tools (delete, stats, search) and observability (/metrics). Section H adds the operator CLI surface and data portability needed for production use.

## Goals

- CLI read commands: `list`, `get`, `search`, `stats` for querying posts from the terminal
- `export_repo` MCP tool: dump all posts + threads for a repo as JSON
- `replay_event` MCP tool: re-process a stored webhook event for debugging/recovery

## Tenets

- TDD: failing tests written first, then implementation
- Real MemoryBlogStorage in tests (not mocked)
- No new external dependencies

## High-level description of changes

**H.1–H.4 — CLI read commands (`src/cli/main.ts`):**
- `blog-cli list --repo owner/repo [--limit N] [--cursor ID] [--event-type TYPE] [--json]` — table or JSON output via `list_posts` MCP tool
- `blog-cli get --repo owner/repo --post-id <id>` — fetch single post via `get_post` MCP tool
- `blog-cli search --repo owner/repo --q "keyword" [--tags tag1,tag2] [--limit N]` — full-text search via `search_posts` (Section G)
- `blog-cli stats --repo owner/repo [--days N]` — repo stats via `get_repo_stats` (Section G), renders top event types

**H.5 — `export_repo` MCP tool (`src/blog/tools.ts`):**
- Single-pass over `storage.listPosts` + `storage.listThreads` (limit 1000 per page)
- Returns `{ repoKey, exportedAt, postCount, threadCount, data: { posts, threads } }`
- `includeThreads: false` skips thread export

**H.6 — `replay_event` MCP tool (`src/blog/tools.ts`):**
- Thin wrapper around `create_post` with pre-filled title/content from event params
- Validates repoKey is registered (required per design spec)
- Returns `{ ok: true, postId, replayed: true }`
