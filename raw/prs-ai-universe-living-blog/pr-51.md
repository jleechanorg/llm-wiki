## Summary

Adds a Grafana dashboard template (`docs/grafana/blog-mcp-dashboard.json`) and import guide (`docs/grafana/README.md`) for the Blog MCP server's `GET /metrics` Prometheus endpoint.

## Background

Issue jleechan-sr64 — bead for Grafana dashboard for the blog MCP server `/metrics` endpoint.

## Goals

- Provide a pre-built Grafana dashboard that graphs all currently emitted Prometheus metrics from `src/blog/server.ts`
- Include scrape configuration instructions (Prometheus `job_name: blog-mcp-server`, target `localhost:8081/metrics`)
- Be honest about what metrics exist vs. what was requested but not yet instrumented

## High-level description of changes

Two new files under `docs/grafana/`:

- **`blog-mcp-dashboard.json`** — Standard Grafana 38-schema JSON export with 5 panels: Posts Created, HTTP Request Rate (by method), Error Rate, Posts Deleted, Requests by Status. Uses `__DS_PROMETHEUS__` as datasource UID placeholder so it is self-contained.
- **`README.md`** — Import instructions, Prometheus scrape config snippet, panel descriptions, available metrics table, and an honest "missing metrics" section documenting that p99 latency and active-sessions gauge are not yet instrumented.

### Metrics available (per `src/blog/server.ts`)

| Metric | Type | Labels |
|--------|------|--------|
| `blog_posts_created_total` | Counter | — |
| `blog_posts_deleted_total` | Counter | — |
| `blog_requests_total` | Counter | `method`, `status` |

### Metrics requested but not yet instrumented

| Panel | Reason |
|-------|--------|
| HTTP Request Latency (p99) | No `http_request_duration_seconds` histogram in the server. Follow-up: add `prom-client` histogram in `src/blog/server.ts`. |
| Active Workers | No session gauge is maintained. Server is stateless per request. |
