## Background
The blog generates novel-quality narrative content but there was no human-readable way to read it without writing raw `curl` JSON-RPC commands. The Phase 2 Web Reader UI adds a browser-accessible interface served directly from the existing blog MCP server.

## Goals
- A human can open `http://localhost:8888` and read all blog posts
- Posts rendered with formatting: title, date, eventType badge (color-coded), content with expand/collapse, tags
- Repo selector dropdown (populated from `list_repos` MCP call)
- Works with zero additional dependencies — single static HTML file, no build step

## Tenets
- No auth/login UI (read-only, mirrors server auth state)
- No write operations from UI (use CLI for writes)
- Vanilla JS — no React, no CSS frameworks
- `express.static` path uses `path.join(process.cwd(), 'public')` per spec risk note

## High-level description of changes

### `public/index.html` (new)
Single-file vanilla JS blog reader with:
- Dark terminal aesthetic (`#0d1117` background, monospace font)
- Repo selector dropdown via `list_repos` MCP call
- Post list with color-coded eventType badges (pr_opened=yellow, pr_merged=green, novel=purple/blue, etc.)
- Expandable post content (truncated at 240px with gradient fade)
- Pagination via `nextCursor` field from `list_posts`
- Fetches from `POST /mcp` JSON-RPC 2.0 endpoint

### `src/blog/server.ts`
- Added `import path from 'path'`
- Added `app.use(express.static(path.join(process.cwd(), 'public')))` before route handlers
- Changed `GET /` from JSON `{tools:[...]}` to `res.sendFile('public/index.html')`

### `testing_mcp/server.test.ts`
- Replaced `GET / lists all 18 tools` test (JSON assertion) with `GET / returns text/html with status 200 (Phase 2 Web Reader UI)` — verifies HTML response with expected content markers
