## Background

The living blog design doc (Section C, Rate limiting) specifies IP-based rate limits for the MCP server:
- Global: 100 req/min/IP
- Write operations: 20 req/min/IP  
- `chat_worker`: 10 req/min/IP

This was not yet implemented in any open PR.

## Goals

- Implement all three rate limit tiers using `express-rate-limit`
- Keep integration tests passing by providing `disableRateLimiting` and `rateLimits` options
- Add 3 tests covering each tier

## Tenets

- Smallest change that adds the feature
- No mock storage — tests use real MemoryBlogStorage
- Configurable thresholds for test ergonomics

## High-level description of changes

Adds `express-rate-limit` as a dependency and wires up three rate limiters in `createBlogApp()`:
1. Global limiter applied as `app.use('/mcp', globalLimiter)` 
2. Write tools limiter applied inline via `applyLimiter()` helper
3. Chat limiter applied inline for `chat_worker`

`createBlogApp()` accepts optional `rateLimits` and `disableRateLimiting` to allow tests to override thresholds without hitting real limits.

## Testing

- `npm test` — 226 tests pass, 8 skipped
- 3 new rate limit tests in `tests/blog/server-http.test.ts`
- Concurrency test updated to use `disableRateLimiting:true`

## Low-level details

**src/blog/server.ts**: Added `rateLimit` import; `createBlogApp()` options extended; three limiters created; `applyLimiter()` helper for per-method inline limiting; global limiter applied as middleware.

**tests/blog/server-http.test.ts**: Added 3 new describe blocks testing each rate limit tier with small custom thresholds (5, 3, 4) for fast execution.

**tests/blog/concurrency.test.ts**: Updated to pass `disableRateLimiting:true` since it sends 21 write requests in sequence.

**package.json**: Added `express-rate-limit@^8.3.1`.
