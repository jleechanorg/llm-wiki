## Background

Sections A–F gave us a working living blog: storage, server, CLI, rate limiting, demo mode, and FIFO chat. Section G adds the MCP tools and observability needed for production operation.

## Goals

1. **`search_posts`** — full-text filter on title/content/tags within a repo
2. **`delete_post`** — hard-delete a post (admin; prunes orphaned thread)
3. **`get_repo_stats`** — aggregate counts: posts, threads, tags, daily breakdown
4. **`/metrics` endpoint** — Prometheus-format counters for posts created/deleted, requests/errors
5. **Firestore emulator in CI** — run the 6 currently-skipped Firestore tests on every PR

## Tenets

- No external prom-client dependency — plain string templating for Prometheus text format
- Single-pass scan over `listPosts` for search/stats — acceptable for demo-scale
- TDD: tests written before implementation for all 5 subsections
- Firestore emulator step added to CI without touching app code

## Changes

### G.1 `search_posts` tool (`src/blog/tools.ts`)
Full-text in-memory scan over `listPosts`. Case-insensitive `q` match against `title + content`. AND-tag filter. `eventType` filter standalone or with `q`. Cursor pagination.

### G.2 `delete_post` + storage interface (`src/blog/storage.ts`, `src/blog/storage-firestore.ts`, `src/blog/storage-file.ts`, `src/shared/types.ts`)
Added `deletePost(id)` and `deleteThread(id)` to `BlogStorage` interface. Implemented in all 3 storage backends. `delete_post` tool calls `deletePost`, then prunes thread if no posts remain.

### G.3 `get_repo_stats` tool (`src/blog/tools.ts`)
Single-pass scan over `listPosts`. Returns `totalPosts`, `totalThreads`, rolling N-day count, top-10 tags, top-10 eventTypes, daily breakdown array.

### G.4 `/metrics` endpoint (`src/blog/server.ts`)
`GET /metrics` returns Prometheus text format. Counters: `blog_posts_created_total`, `blog_posts_deleted_total`, `blog_requests_total{method,status}`. Incremented in tool dispatch middleware.
