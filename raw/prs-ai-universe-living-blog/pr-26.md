## Background

The TDD roadmap (docs/plans/2026-03-27-tdd-roadmap.md) defined 5 phases of test coverage. Phase 1 (GitHubClient unit tests) was already shipped. This PR implements Phases 2-5.

## Goals

- Implement all remaining TDD roadmap phases
- Bring test suite from 186 → 226 tests (40 new tests, 4 new files)
- Cover: CLI arg parsing, GitHubClient integration, CLI-MCP e2e, WorkerChat

## Tenets

- Tests are focused on actual exported functions (no mock overreach)
- Use real MemoryBlogStorage (not mocked) for storage tests
- Mock only external APIs (fetch/Anthropic) and at the module boundary
- Each test verifiable in isolation: `npx vitest run <file>`

## High-level description of changes

**Phase 2** — `tests/cli/parser.test.ts` (15 tests): Tests `parseKvArgs` from `src/novel/cli.ts`. Covers `--key=value` and `--key value` forms, mixed forms, edge cases (no value, value containing `=`, empty args), and all CLI flags used by branch-entry/daily-summary commands.

**Phase 3** — `tests/cli/gh-client-integration.test.ts` (7 tests): Tests `GitHubClient` from `src/shared/github-client.ts` with a stubbed global `fetch`. Covers `getPR`, `getCommits`, `getCheckRuns`, `getReviews`, `listRecentActivity`, and constructor token/no-token behavior.

**Phase 4** — `tests/cli-mcp-e2e.test.ts` (7 tests): End-to-end test using the programmatic API. Creates a real MCP server with MemoryBlogStorage, runs the novel engine `runBranchEntryPipeline` and `runDailySummaryPipeline`, and verifies posts appear in storage and via MCP `list_posts`. Also tests MCP health_check and create_post → list_posts round-trip.

**Phase 5** — `tests/fifo/chat-fifo.test.ts` (11 tests): Tests `extractVoice` tone heuristics (contracted, formal, staccato, introspective, neutral) and `WorkerChat.chat` — Anthropic API call when entry exists, fallback message when no entry, fallback on API error, custom model routing.
