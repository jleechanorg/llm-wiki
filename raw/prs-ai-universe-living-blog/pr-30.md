## Background
Section D of the living-blog refactor design doc describes a demo mode where the blog server can be run without prior repo registration. Instead it pulls git commit history from the GitHub REST API and generates synthetic blog posts from it — useful for showcasing the system with any public repo.

## Goals
- Implement `runDemoMode()` with real GitHub REST API integration
- Group commits into sessions by temporal proximity (4-hour gap)
- Generate readable synthetic branch entries for each session
- Wire `--demo --repo --max-commits --session-prefix` flags into the blog server

## Tenets
- Public repos work without a token (no barrier to demo)
- Pure functions (fetchCommits, groupIntoSessions, generateDemoEntry) are independently testable
- runDemoMode works with any BlogStorage implementation

## High-level description of changes
`fetchCommits()` calls the GitHub REST API, maps responses to `DemoCommit[]`. `groupIntoSessions()` groups commits newest-first into sessions split by gaps > 4 hours. `generateDemoEntry()` formats a markdown branch entry. `runDemoMode()` orchestrates these + writes posts to storage with `novel_branch_entry` eventType. The blog server listens for `--demo` and runs demo mode after startup.
