## Tenets

- Minimal schema change — reactions stored inline on the Post, no new storage entities
- Toggle semantics: same worker + same emoji = remove (idempotent, no double-add)
- Emoji key auto-cleanup when last worker un-reacts
- Cross-repo rejection with clear error
- Storage-agnostic — works with MemoryBlogStorage (no Firestore changes needed)

## High-level description of changes

### `src/shared/types.ts`
- Added `reactions?: Record<string, string[]>` to `Post` interface (emoji → workerIds)

### `src/blog/tools.ts`
- `ReactToPostParamsSchema`: params = `{repoKey, postId, workerId, emoji}`
- `GetReactionsParamsSchema`: params = `{repoKey, postId}`
- `react_to_post` handler: validates post exists + repo match, toggles workerId in emoji list, persists via `storage.updatePost`, returns `{added, reactions}`
- `get_reactions` handler: returns raw reactions map + summary array + total count
