## Background

Skeptic workflows were broken in two ways:
1. **skeptic-cron.yml** referenced `.github/actions/skeptic-setup` (doesn't exist in this repo) and `packages/cli/dist/index.js` (agent-orchestrator CLI, not present here) — causing every run to fail with "Can't find action.yml under .github/actions/skeptic-setup"
2. **skeptic-gate.yml** used `--slurpfile /dev/null` which is an invalid `gh api` flag, causing immediate API failure and rapid exit. Also polled for verdicts from `jleechan2015` but cron posts as `github-actions[bot]`.

## Goals
- skeptic-cron runs successfully without external dependencies
- skeptic-gate polls correctly and finds VERDICT comments
- PRs with green CI + mergeable + no CHANGES_REQUESTED get auto-merged

## Tenets
- No external CLI dependencies (no AO, no node packages/cli)
- Fail-closed: missing verdicts = FAIL, not PASS
- Simple and auditable

## High-level description of changes

**skeptic-cron.yml** — complete rewrite as standalone lightweight workflow:
- Checks (1) CI green via commit status + check-runs, (2) mergeable=true, (3) no CHANGES_REQUESTED reviews
- Posts `VERDICT: PASS` or `VERDICT: FAIL` comment with `<!-- skeptic-gate-trigger-{SHA} -->` marker
- Updates existing verdict comment if present (avoids duplicate comments on re-runs)
- Auto-merges passing PRs via `gh pr merge --squash`

**skeptic-gate.yml** — two targeted fixes:
- Removed `--slurpfile /dev/null` (invalid flag, caused `gh api` to error on every call)
- Changed `SKEPTIC_BOT_AUTHOR` default from `jleechan2015` to `github-actions[bot]` (cron posts as github-actions[bot])

## Testing
- Local review of workflow YAML
- skeptic-gate no longer uses invalid gh flag → API calls will succeed
- skeptic-cron no longer requires local action or CLI
