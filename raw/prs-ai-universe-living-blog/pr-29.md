## Background
Section E of the design doc describes a FIFO-based bidirectional IPC channel so Claude Code operator sessions can have character-consistent chats with AO worker sessions via named pipes (`~/.blog/inbox/{workerId}`). This PR implements that mechanism and integrates it as the Tier 0 inference backend in `chat_worker`.

## Goals
- Add `chatViaFifo()` to `src/novel/chat.ts` — opens named pipe `O_RDWR|O_NONBLOCK`, writes JSON request, polls for JSON response within timeout
- Extract `parseFifoLines()` as a pure exported function for testability
- Wire as Tier 0 in `chat_worker` tool (before OPENCLAW inference)
- Full test coverage: 10 new tests (7 pure unit + 3 FS integration)

## Tenets
- Pure function extraction makes FIFO parsing testable without real FIFOs
- Real FIFO semantics (O_RDWR, poll loop) preserved in production code
- No new dependencies

## High-level description of changes
`parseFifoLines(accumulated)` scans newline-delimited JSON lines in reverse order (newest first), returning the first `from=worker` response found. `chatViaFifo()` uses it after each `readSync` batch. `chat_worker` in `tools.ts` tries FIFO before all other inference tiers.

- 3 chatViaFifo FS tests: null when no file, timeout returns null, FIFO_POLL_INTERVAL_MS constant
