## Background

AO workers need a real-time feed of new blog posts without polling GitHub webhooks or manually running `blog-cli list`. The `watch` command provides a terminal-native live stream of new posts as they arrive.

## Goals

- `blog-cli watch` polls the MCP server for new posts and prints them to stdout
- Polls every N seconds (configurable via `--interval`, default 5)
- Cursor-based deduplication: only emits posts not seen in previous polls
- Optional `--repo` filter to watch a specific repo
- Graceful SIGINT with summary: "N new posts seen."
- Exponential backoff on connection errors (1s → 30s cap)

## Tenets

- No new files — watch command lives in `src/cli/main.ts` alongside existing commands
- Reuse existing `callMcpTool` HTTP helper
- Test at unit level: parseArgs, formatPostLine, cursor-tracking logic
