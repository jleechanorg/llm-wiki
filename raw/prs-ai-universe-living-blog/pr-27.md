## Background

The living blog design doc (`docs/plans/2026-03-27-living-blog-refactor-design.md`) describes a `blog-cli` command router as the primary CLI surface for AO workers. Previously, the only CLI was `src/novel/cli.ts` which calls the novel engine directly. This PR implements `src/cli/main.ts` — a typed, testable entry point that covers all 5 commands from the design doc.

## Goals

- Implement `src/cli/main.ts` with typed argument parsing, validation, and command routing
- `branch-entry` command: fetch GitHub PR data and write structured prompt files for AO workers — no LLM calls
- `chat` command: proxy to MCP server `chat_worker` tool
- `config --show/--init`: effective config inspection and scaffolding
- `register-repo`: proxy to MCP server `register_repo` tool
- 33 unit tests covering all commands, validation paths, env var overrides

## Tenets

- No LLM calls in `branch-entry` — CLI is a prompt provider, not a generator
- All commands are independently testable (no process.chdir, no side effects in tests)
- Reuse `GitHubClient` from `src/shared/github-client.ts` (already tested)

## High-level description of changes

**`src/cli/main.ts`** (new):
- `parseKvArgs()`: low-level `--key=value` / `--key value` / boolean flag parser
- `parseArgs()`: typed command router with validation (throws on missing required args)
- `getEffectiveConfig()`: reads `GITHUB_TOKEN`, `BLOG_SERVER_URL`, etc. from env
- `runBranchEntryCommand()`: fetches PR/commits/checkRuns/reviews in parallel via `GitHubClient`, writes `~/.blog/prompts/{sessionId}.md`
- `runChatCommand()`: HTTP POST to MCP server `chat_worker`
- `runConfigCommand()`: `--show` prints effective config, `--init` creates `novel.config.json`
- `runRegisterRepoCommand()`: HTTP POST to MCP server `register_repo`
- `main()`: entry point wired to `process.argv`
