## Summary
Fixes backend crash when listing conversations for users with no conversations.

## Problem
The backend returned this error when trying to list conversations for `jleechantest@gmail.com`:
```
Error: Cannot read properties of undefined (reading 'map')
```

## Root Cause
- Location: `backend/src/tools/ConversationMCPTool.ts:495`
- The `toConversationListResponse` method called `.map()` on `response.conversations` without checking for undefined
- The remote conversation MCP server can return `undefined` instead of an empty array when a user has no conversations

## Solution
Added null coalescing operator to safely default to empty array:
```typescript
const conversations = response.conversations ?? [];
```

Now the backend properly handles:
- Users with no conversations
- Undefined database query results
- New users accessing the conversation list for the first time
