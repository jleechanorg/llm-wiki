## Summary

Adds a test to verify the pagination sorting fix in `getConversationContext` that was implemented in PR #634.

## Test Coverage

This test verifies that when `getConversationContext` collects messages across multiple paginated pages, they are sorted chronologically **globally**, not just per-page.

### Test Scenario

Reproduces the bug that was fixed:
- **Page 1** (newest messages): `[msg6, msg5, msg4]` → sorted per-page → `[msg4, msg5, msg6]`
- **Page 2** (older messages): `[msg3, msg2, msg1]` → sorted per-page → `[msg1, msg2, msg3]`
- **Without global sort**: `[msg4, msg5, msg6, msg1, msg2, msg3]` ❌ Wrong order!
- **With global sort**: `[msg1, msg2, msg3, msg4, msg5, msg6]` ✅ Correct order!

### What the Test Verifies

1. ✅ `slice(-limit)` returns the newest N messages correctly
2. ✅ All messages are in strict chronological order (oldest to newest)
3. ✅ Pagination across multiple pages maintains correct order
