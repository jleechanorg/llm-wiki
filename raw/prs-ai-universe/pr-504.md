## Problem
Multi-model synthesis metadata was not being persisted to Firestore, causing synthesis cards to disappear after page reload or conversation refetch.

**Evidence:** Console logs showed synthesis flows correctly through frontend (TRACE-1 through TRACE-5 all confirm "synthesis present? true"), but when conversation history is reloaded from Firestore, `metadata.secondOpinion` is missing.

## Root Cause
`SecondOpinionAgent.addMessage()` call did not include metadata parameter, so synthesis data existed only in React state but never saved to database.

## Solution
- Attach `secondOpinion` metadata (synthesis, secondaryOpinions, primary) to assistant message before persisting to Firestore
- Pass `metadata` parameter to `conversationTool.addMessage()`
- More efficient than PR #503 (no JSON re-parsing, cleaner data structure)
