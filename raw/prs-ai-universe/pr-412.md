## Rationale
The previous distributed detection logic was causing issues:
- Auto-detecting as distributed when it shouldn't (Render single-instance)
- Requiring environment variable overrides for normal operation
- Complex logic with many edge cases

The new approach:
- Default to non-distributed (normal rate limits)
- Only apply strict limits when explicitly requested via `RATE_LIMIT_FORCE_DISTRIBUTED=true`
- Simpler code that's easier to understand and maintain
