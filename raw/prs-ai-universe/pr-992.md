## Summary
Enhance deployment notification emails by including pull request information (URL, title, and description) when available. This provides better deployment context and traceability.

## Changes
- ✅ Added `pr_url`, `pr_title`, `pr_description` inputs to `send-deploy-notification` action
- ✅ Updated email body template to display PR information when provided
- ✅ Added PR fetching step to both `deploy-dev.yml` and `deploy-production.yml` workflows
- ✅ Updated README documentation with PR integration details and examples

## How it Works
1. Deployment workflows automatically search for PRs associated with the commit being deployed
2. If a PR is found, its URL, title, and description (first 500 chars) are extracted
3. This information is passed to the email notification action
4. The email template conditionally includes PR details when available

## Benefits
- 📧 Better deployment email context with direct PR links
- 🔍 Improved deployment audit trail and traceability
- 📝 Easy reference to the changes being deployed
- 🔄 Backward compatible - works with or without PR information
