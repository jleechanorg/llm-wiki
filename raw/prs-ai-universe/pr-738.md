## ⚠️ Breaking Changes

This PR includes **security-motivated breaking changes**:

1. **userId without authentication** now creates an **anonymous user** (previously trusted client-provided ID)
   - **Rationale**: Prevents userId spoofing attacks
   - **Migration**: Use `Authorization` header → server verifies → injects `_authenticatedUserId`
   - **Evidence**: 4 test failures document this security improvement working correctly

2. **Client library updated to v2.0.0** with new authentication flow
   - Requires server middleware for authentication
   - Legacy `idToken` parameter deprecated

3. **Frontend migration required** - See `docs/FRONTEND_AUTH_MIGRATION.md`
