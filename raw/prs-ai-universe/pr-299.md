## Summary

This PR adds a detailed implementation roadmap for replacing the FastMCP dependency with a direct MCP protocol implementation. This document serves as a complete specification for another coding agent to execute the replacement.

## Context

**Problem:** FastMCP with `stateless: true` buffers all responses, preventing true real-time streaming. Switching to `stateless: false` requires complex session management that FastMCP doesn't handle well.

**Discovery:** Analysis revealed FastMCP provides minimal value (~250 LoC of wrappers) for significant complexity cost. Direct implementation enables:
- ✅ True real-time SSE streaming
- ✅ Full control over streaming behavior  
- ✅ Simpler codebase without framework abstraction
- ✅ Better error handling and debugging

**Key Insight:** origin/main has NO SSE support. The previous PR #178 added buffered SSE (new feature), not fixes. This roadmap enables proper streaming from the start.

## What This PR Adds

## Architecture Overview

### Core Components

1. **ToolRegistry** (~120 lines) - Tool registration and management
2. **ProtocolHandler** (~200 lines) - JSON-RPC 2.0 request/response handling
3. **SessionManager** (~100 lines) - Session lifecycle management
4. **EventBus** (~80 lines) - Real-time event streaming
5. **HttpServer** (~200 lines) - Dual-endpoint architecture

### Dual-Endpoint Design

- `/mcp` - Stateless JSON responses (no sessions needed for `initialize`, `tools/list`)
- `/mcp/stream/:sessionId` - Real-time SSE streaming with EventBus
