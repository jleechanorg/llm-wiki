## Raw Body
This commit implements a comprehensive rate limiting enhancement for the second opinion library:

## Key Features

1. **VIP Allowlist (GCP Secret Manager)**
   - New `vip-allowlist` secret for privileged users
   - Cached for 5 minutes for performance
   - Supports JSON array or comma-separated format

2. **Tiered Rate Limits**
   - Admin (via ADMIN_EMAILS): 1000/hour
   - VIP (via vip-allowlist secret): 50/hour
   - Authenticated (non-VIP): 30/day + 15/hour (dual window)
   - Anonymous: 100/hour

3. **Special Rate Limit Response**
   - Non-streaming: Returns structured MCP response with: - rateLimitExceeded: true - message: User-friendly text with contact info - contactEmail: jleechan@gmail.com - limit, resetTime: Current limit details
   - Streaming: Error event with metadata containing same info

4. **Implementation Details**
   - Custom RateLimitExceededError class for proper handling
   - Dual-window checking for authenticated non-VIP users
   - Updated StreamEvent type to support error metadata
   - Updated tests to reflect new rate limits

## Usage

Create VIP allowlist secret:
```bash
echo '["user1@example.com", "user2@example.com"]' | \
  gcloud secrets create vip-allowlist \
    --data-file=- \
    --project=ai-universe-2025
```

## Client Integration

Clients can now detect rate limit responses:
```javascript
const response = await secondOpinionClient.query(question);
if (response.rateLimitExceeded) {
  // Show contact email and reset time to user
  console.log(response.message);
  console.log(`Contact: ${response.contactEmail}`);
  console.log(`Resets at: ${response.resetTime}`);
}
```
