## Raw Body
## Summary
- Added MCP session initialization in the local smoke runner to satisfy newer JSON-RPC handshake expectations.
- Added auth deprecation warning suppression for repeated unauthenticated userId usage to reduce log noise.
- **Default OpenAI model is `gpt-5-mini`** — intentional choice for cost efficiency ($0.25/$2.00 per 1M input/output tokens vs $1.25/$10.00 for gpt-5). All documentation, display names, and cost mappings updated to reflect this.
- Hardened monitoring metric flushing to avoid concurrent duplicate flush issues and stabilize dedup handling.
- Added a focused gpt-5-mini real-mode latency addendum in docs, including evidence paths and stage-level breakdown.
- Consolidated CI to Node 22 only (Node 20 EOL April 2026).

## Review Comments Addressed

### Fixed Issues
| Issue | Resolution |
|-------|------------|
| **Recursive `flushMetrics` infinite loop** | Removed recursive call in finally block; periodic `setInterval` handles residual metrics |
| **Redundant double normalization of labels** | Fixed — labels normalized once on push, `getLabelEntries` reads already-normalized labels |
| **Suppressed warning logs per-request** | Added sampling mechanism (`USER_ID_WARNING_LOG_INTERVAL`) to log every Nth occurrence |
| **`OPENAI_MODEL` env var concern** | Removed env var; model now hardcoded to `gpt-5-mini` with documented rationale |

### Known Documentation Notes
| Item | Note |
|------|------|
| **Stage-level vs client timings** | Stage breakdown shows cumulative parallel spans; client timings show wall-clock end-to-end. Stage totals may exceed client durations due to parallel execution overlap. |
| **Aggregate vs per-request max** | Aggregate max values (e.g., `parallel_secondary_responses: 240,658ms`) represent different time windows than individual request peaks (327,919ms). Aggregate tracks across all requests in the sample period. |
