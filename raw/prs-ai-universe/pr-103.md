## Summary
Fixes #77 - Conversation MCP tools not being exposed by backend server

## Problem
The conversation tool files existed (`ConversationMCPTool.ts`, `ConversationStorage.ts`) from PR #77, but the tools weren't registered with the MCP server. This caused:
- ❌ `tools/list` only returned 5 tools (missing all conversation tools)
- ❌ Frontend showing "No conversations yet" with infinite loading
- ❌ Frontend unable to call conversation endpoints

## Solution
1. Created `ConversationAgent` class to wrap `ConversationMCPTool`
2. Registered all 5 conversation tools with the MCP server:
   - `conversation.start`
   - `conversation.add-message`
   - `conversation.get-history`
   - `conversation.list`
   - `conversation.health`
3. Added `ConversationAgent` to the agents array in `createFastMCPServer.ts`

## Changes
- ✅ New file: `backend/src/agents/ConversationAgent.ts`
- ✅ Modified: `backend/src/createFastMCPServer.ts`

curl -s -X POST http://localhost:2000/mcp \

  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' \
