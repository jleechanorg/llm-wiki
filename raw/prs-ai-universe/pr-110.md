## Summary

Fixes critical backend crashes on Render caused by unhandled promise rejections from GCP Secret Manager credential fetching attempts.

## Root Cause

Node.js v22.16.0 terminates the process on unhandled promise rejections. The GCP Secret Manager client attempts async credential fetching, creating unhandled rejections that crashed the server immediately after startup. Winston's rejectionHandlers caught these first but did not prevent the default process termination behavior.

## Evidence

- Server logs showed 12 consecutive crashes over 1 hour
- Pattern: Server starts → GCP credentials error → unhandled rejection → exit code 1 → Render restarts
- Cycle repeated every 2-5 minutes
- Health checks continuously failed, causing 502 errors

## Solution

Two-part fix:

1. Added global unhandled rejection handler in backend/src/server.ts - Catches GCP credential errors non-fatally
2. Removed Winston's rejectionHandlers from backend/src/utils/logger.ts - Winston was catching rejections before server.ts handler

## Impact

✅ Backend now stays running on Render
✅ Health checks succeed (returns 200 OK)
✅ 502 errors resolved
✅ GCP credential errors logged as non-fatal warnings
✅ Server falls back to environment variables gracefully

## Deployment Evidence

- Deployment status: live

- Logs show: [warn]: GCP credentials unavailable (non-fatal - using environment variables)
