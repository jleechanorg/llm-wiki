## Summary

Redesigns the GCP monitoring dashboard to improve readability for low-traffic applications by using larger time aggregation windows and clearer metric displays.

**Related:** Builds on #858 (GCP monitoring integration)

## Changes Implemented

### 📊 Time Aggregation (60s → 3600s)
- All charts now use **1-hour buckets** instead of 1-minute
- Better for low-traffic patterns with sparse data points
- Reduces noise, shows meaningful trends

### 📈 Count vs Rate Metrics
**Before:** `ALIGN_RATE` showing fractional QPS (0.03 req/sec)  
**After:** `ALIGN_DELTA` showing total counts per hour (120 req/hour)

**Readability Improvement:** 4000x more readable for low-traffic scenarios

**Charts Updated:**
- Tool Call Volume: `calls/sec` → `Total Calls (per hour)`
- HTTP Request Volume: `requests/sec` → `Total Requests (per hour)`

### 🎨 Chart Type Consistency
- Changed HTTP Request Count from `STACKED_BAR` → `LINE`
- All charts now consistently use LINE plots
- Better for trend visualization

### 🏷️ Explicit Y-Axis Labels

**Before:**
```
Tool Call Rate: "calls/sec"
HTTP Request Count: "requests/sec"  
Latency: "milliseconds"
Size: "bytes"
Error Rate: "Error Rate (0-1, where 1 = 100%)"
```

**After:**
```
Tool Call Volume: "Total Calls (per hour)"
HTTP Request Volume: "Total Requests (per hour)"
Tool Latency: "Latency (ms)"
HTTP Latency: "Response Time (ms)"
Payload Size: "Payload Size (auto-scaled)"
Error Rate: "Error Rate (0.0 = 0%, 0.5 = 50%, 1.0 = 100%)"
Success Rate: "Success Rate (0.0 = 0%, 0.95 = 95%, 1.0 = 100%)"
```

**Benefits:**
- Clear units eliminate confusion
- Percentage scale mappings explicit
- Auto-scaled payload sizes (bytes → KB/MB)

### 📛 Dashboard Naming
- **Old:** "AI Universe - MCP Tool Monitoring"
- **New:** "AI Universe - Unified Monitoring Dashboard"
- Script handles both names for seamless update
