## Critical CI Fix: MCP Smoke Tests Now Actually Work

### What Was Wrong
The MCP smoke tests were running in **mock mode** (MCP_TEST_MODE=mock), which meant:
- No real HTTP calls to the deployed preview server
- No real authentication - just skipped
- Mocked responses generated locally by scripts/mcp-smoke-test-mocks.mjs
- Zero validation of actual server behavior

The tests were essentially testing the test harness itself, not the deployed code.

### What Is Fixed Now
- Real mode enabled (MCP_TEST_MODE=real) - tests hit the actual deployed preview server
- Firebase authentication required - uses FIREBASE_REFRESH_TOKEN org secret
- New token generator (scripts/generate-ci-auth-token.cjs) exchanges refresh token for ID token
- Web search diagnostics validated - confirms webSearchDiagnostics.enabled: true in responses

### New Org Secrets Created
| Secret | Purpose |
|--------|---------|
| FIREBASE_REFRESH_TOKEN | Long-lived token (~60 days) for AI Universe auth |
| WORLD_AI_FIREBASE_AUTH_PROJECT_ID | WorldArchitecture-AI project ID |
| WORLD_AI_FIREBASE_SA_KEY | WorldArchitecture-AI service account |
