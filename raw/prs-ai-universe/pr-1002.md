## Raw Body
### Motivation
- Prevent false-positive MCP initialization when a JSON‑RPC notification returns an `error` payload, and preserve compatibility with older MCP servers that don't implement `initialize`.
- Provide an auditable, consolidated mapping of reviewer comments to FIXED / CONSIDERED states for automation and follow-ups.

### Description
- Make `sendMcpNotification` fail when the JSON‑RPC response contains an `error` field so a HTTP 200 with an RPC error is not treated as success by the smoke runner. (file: `scripts/mcp-ci-tests.mjs`)
- Make `initializeMcpSession` tolerant of legacy servers by treating RPC method-not-found (code `-32601` / message `method not found`) as non‑fatal, logging a warning, and continuing without sending `notifications/initialized`. (file: `scripts/mcp-ci-tests.mjs`)
- Add `responses.json` to record consolidated reviewer responses with `FIXED` vs `CONSIDERED_ACKNOWLEDGED` entries and `html_url` links for traceability. (new file: `responses.json`)
