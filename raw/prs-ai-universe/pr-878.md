## Summary

- Update `ci.yml` shared-libs install to use `npm ci` instead of `npm install`
- Sync `package-lock.json` files for auth-context, second-opinion-client, and typescript-server-launcher

## Why This Change

| Mode | Behavior | Used By |
|------|----------|---------|
| `npm install` | **Tolerant** - regenerates lock file if mismatched | CI (before this PR) |
| `npm ci` | **Strict** - fails if lock file doesn't match exactly | deploy-preview, smoke-tests |

The inconsistency meant sync issues slipped through CI but failed in smoke tests.

### Benefits of `npm ci`
- âś… **Deterministic builds** - exact same versions every time
- âś… **Faster** - doesn't resolve dependency tree, just reads lock file
- âś… **Security** - ensures pinned versions, no surprise upgrades
- âś… **Catches issues early** - fails fast with clear errors
