Add comprehensive test coverage using Test-Driven Development (TDD) to catch schema-related bugs before they reach production. These tests would have caught the frontend failures reported for conversation.list metadata validation and ESM module import issues.

## Changes

### New Tests (TDD Red-Green-Refactor)
- **schemas.test.ts**: Validates JSON schemas enforce required fields
  - Tests that synthesis.response is required when synthesis present
  - Tests conversation.list metadata validation
  - Catches backend responses missing expected frontend fields
  - Uses AJV for strict JSON Schema Draft 07 validation

- **esm-import.test.ts**: Validates ES module compatibility
  - Tests dynamic imports work in strict ESM environment
  - Tests JSON module imports (resolveJsonModule)
  - Would have caught require.resolve() in ES module bug
  - Ensures schemas load without CommonJS patterns

These tests directly address the gap identified in the frontend issue:
