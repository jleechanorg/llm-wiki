## Summary
- Simplifies CORS configuration to allow any `ai-universe-frontend*` deployment
- Removes production-only CORS_ALLOWED_ORIGINS requirement
- Removes unnecessary PR preview pattern (already covered by wildcard)
- Fixes CORS blocking for frontends deployed to non-us-central1 regions
- Adds missing `async-mutex` dependency

## Problem
The backend CORS pattern `https://ai-universe-frontend*.us-central1.run.app` was too specific and blocked frontends deployed to other regions like `.uc.a.run.app`.

Frontend URL: `https://ai-universe-frontend-elhm2qjlta-uc.a.run.app`
❌ Blocked by: `https://ai-universe-frontend*.us-central1.run.app`
✅ Allowed by: `https://ai-universe-frontend*`

## Changes
1. **backend/src/server.ts**:
   - Removed production-only CORS_ALLOWED_ORIGINS enforcement
   - Removed unnecessary PR preview patterns
   - Changed `allowNullOrigin` from `let` to `const` (ESLint fix)

2. **deploy.sh**:
   - Changed prod CORS pattern from `.us-central1.run.app` to `*`
   
3. **backend/package.json**:
   - Added `async-mutex` dependency (was missing)
