## Summary

Dev environment now uses admin rate limits by default (1000 requests/minute for all users).

## Changes Made
- **backend/src/tools/RateLimitTool.ts**: Use admin limits by default when NODE_ENV != 'production'
- **scripts/deploy.sh**: Set correct NODE_ENV for different environments (development for dev/staging, production for production)

## Behavior
- **Dev/Staging**: Admin rate limits (1000/min) for all users automatically
- **Production**: Normal rate limits for security
- **No flags needed**: Simplified deployment workflow

## Usage
```bash
# Dev automatically gets admin rate limits
./scripts/deploy.sh dev

# Production uses normal rate limits
./scripts/deploy.sh production
```

This simplifies the development experience by removing the need for special flags while maintaining security in production.
