## ✨ New Features (PR-Specific)

### Local Auth Bypass Feature

**Purpose**: Enable local development without requiring Firebase authentication tokens.

**How It Works**:
1. **Environment Variable**: `RUN_LOCAL_SERVER_DISABLE_AUTH=true`
2. **Safety Guard**: Only works in **local environments** (disabled in Cloud Run)
3. **Bypass Logic**: When enabled, skips Firebase token verification and trusts client-supplied auth fields

**Implementation Details**:

#### Flag Reading (`backend/src/server.ts:55-56`)
```typescript
const authBypassRequested = process.env.RUN_LOCAL_SERVER_DISABLE_AUTH === 'true';
const isLocalAuthBypassEnabled = authBypassRequested && !isCloudRun;
```

#### Auth Enforcement Bypass (`backend/src/server.ts:221-228`)
When `isLocalAuthBypassEnabled` is true:
- **Skips** Firebase token verification (`sharedAuthTool.verifyIdToken()` never called)
- **Skips** Authorization header extraction and validation
- **Trusts** client-supplied `_authenticatedUserId`, `_authenticatedUserUid`, `_authenticatedUserEmail`, `_auth
