## Summary

This PR delivers two major improvements:

1. **Performance Optimization (~10-11s savings)**: Eliminates separate synthesis prompt generation LLM call by requesting both PRIMARY_ANSWER and SECOND_OPINION_PROMPT in a single prompt to the primary model
2. **Unified Human-Readable Logging**: Adds comprehensive logging system that writes to `/tmp/backend/<branch>/` with three file types:
   - `.log` - Human-readable execution trace with Unicode box drawing
   - `.summary.txt` - Quick overview with bottleneck analysis
   - `.metrics.json` - Structured data for programmatic analysis

## Flow Changes

**Before** (3 LLM calls):
1. Primary response (21.7s): Get answer from primary model
2. Synthesis prompt generation (10.7s): Create specialized prompt ← **ELIMINATED**
3. Final synthesis: Combine responses

**After** (2 LLM calls):
1. Primary response (21.7s): Get answer AND second opinion prompt in ONE call
2. Final synthesis: Combine responses

## Technical Details

- Modified `applySourceRequirements()` to request both outputs in context='primary'
- Added `parseCombinedResponse()` method to extract PRIMARY_ANSWER and SECOND_OPINION_PROMPT markers
- Synthesis flow now uses extracted prompt (or falls back to original question)
- Real-world verification: synthesis_prompt_generation now 0ms (down from 10,695ms)

### HumanLogger Implementation
- Git branch auto-detection with sanitization
- Visual formatting with Unicode box drawing (━, ✓, ✗, ⊘)
- Performance bottleneck identification (slowest stage highlighted)
- Three file types for different use cases (human vs programmatic)
- Timestamp format: MM:SS.mmm for easy reading
