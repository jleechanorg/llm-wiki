## Problem 1: Production Crash (Critical)

**Error**: `Cannot read properties of undefined (reading 'map')` in conversation.get-history

**Observed**: Production logs 2025-11-02 at 11:11-11:12 showed repeated failures every 2 seconds (retry loop)

**Root Cause**: When conversation service returns `undefined` for messages array (empty conversations or API errors), code crashed on line 592:
```typescript
messages: response.messages.map((message) => ({  // ❌ Crash if undefined
```

**Solution**: Added defensive null-coalescing (follows same pattern as line 509):
```typescript
const messages = response.messages ?? [];
return {
  history: {
    conversationId,
    messages: messages.map((message) => ({  // ✅ Safe
```

## Problem 2: Slow Development Iteration

**Issue**: Developers had to manually restart server after every code change

**Solution**: Auto-restart now enabled by default via tsx --watch
```bash
./run_local_server.sh              # Auto-restart enabled (DEFAULT)
./run_local_server.sh --production # Disable for production-like testing
```

**Benefits**:
- ⚡ No manual restarts needed during development
- 🔄 TypeScript hot-reloading via tsx --watch
