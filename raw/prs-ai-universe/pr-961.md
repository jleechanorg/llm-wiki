## Summary
Fixed single message mode failing with Cerebras 400 error when conversations exceeded 131K token limit. Implemented context-aware message selection with proper priority ordering.

## Algorithm
Priority: Latest User Message > Baseline (first 3) > Recent Middle Messages

1. **Latest message ALWAYS included** - never drop the current user request
2. **Baseline messages** (first 3) included if budget allows
3. **Recent middle messages** fill remaining token budget
4. **History fetch capped at 500** - prevents expensive pagination on long threads
5. **10K token minimum floor** - handles edge cases where system prompt is large

## Fixes Applied (from review feedback)
- HIGH: Latest user message can no longer be dropped - it is reserved first
- HIGH: History fetch capped at 500 messages (was unlimited)
- MEDIUM: Added 10K token minimum floor to prevent negative budget edge cases
