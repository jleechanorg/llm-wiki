### Auto-Resolve Workflow Improvements (Added in this PR)

**Problem Fixed:** Auto-resolve commits using `GITHUB_TOKEN` weren't triggering CI workflows (only CodeRabbit ran), and auto-resolve was merging even when no conflicts existed.

**Solution Implemented:**
1. **Enable CI triggers after auto-resolve commits**
   - Use `PAT_TOKEN` (with `GITHUB_TOKEN` fallback) to trigger workflows
   - Prevent auto-resolve from triggering on its own merge commits
   - Check commit message prefixes: `[MERGE RESOLVED]` and `[MERGE MANUAL:]`

2. **Only auto-merge when conflicts exist**
   - Fast conflict detection using `git merge-tree` (non-invasive dry-run)
   - Skip merge entirely if branch is behind but has no conflicts
   - Reduces CI resource usage on clean merges (10s vs 30s+)

3. **Documentation**
   - Added "🔐 CI Workflow Triggers & Auto-Resolve" section to CLAUDE.md
   - Explains PAT_TOKEN setup and behavior

**Files Modified:**
- `.github/workflows/resolve-conflicts.yml` - Early conflict detection + PAT_TOKEN integration
- `scripts/auto_resolve_conflicts.sh` - Simplified (conflict check moved to workflow)
- `CLAUDE.md` - Documentation

**Behavior:**
- ✅ Auto-resolve only runs when Git cannot auto-merge
- ✅ All CI workflows trigger on auto-resolve commits (with PAT_TOKEN)
- ✅ No infinite loops (skips its own merge commits)
- ✅ Fast exit when no conflicts (~10s instead of ~30s)
