## Raw Body
## Summary
- Introduces `CerebrasMode` enum to explicitly control Cerebras endpoint routing
- **FAST**: Direct Cerebras API (80k context, faster, no web search)
- **SMART**: OpenRouter endpoint (32k context, web search capable)
- Updates `ToolRegistry.getLLMCaller` to accept optional `cerebrasMode` parameter
- Refactors `SecondOpinionAgent` to use explicit modes instead of bypassing `getLLMCaller`
- Removes duplicated endpoint selection logic
- Runtime behavior (validated by REAL smoke tests on PR preview 2025-12-01 12:02Z):
  - Normal `conversation.send-message`: uses OpenRouter Cerebras with web search enabled.
  - SecondOpinion primary & synthesis: use direct Cerebras (no web search) while secondary opinions keep provider-specific search.

## Problem Solved
Previously, Cerebras routing was implicit and duplicated:
- `ToolRegistry` silently hijacked 'cerebras' to use OpenRouter when a key existed
- `SecondOpinionAgent` duplicated the same key-checking logic
- `SecondOpinionAgent` bypassed `getLLMCaller` entirely for some calls

## Benefits
- **Clear intent**: Code explicitly states "I want FAST mode" vs relying on API key detection
- **Centralized logic**: All endpoint selection lives in `ToolRegistry.getLLMCaller`
- **No magic behavior**: No silent hijacking based on API key presence
- **Easier maintenance**: Single source of truth for Cerebras routing
