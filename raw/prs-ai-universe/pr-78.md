> [!NOTE]
> Installs shared libs during Docker build using a new helper script with /shared-libs support, hardens production npm installs, and minorly refactors config-utils model options.
> 
> - **Backend Docker**:
>   - **Shared libs resolution**: Copy `shared-libs` to `/shared-libs`; add and run `install_shared_libs.sh` in both builder and production stages.
>   - **Production install hardening**: Use `npm ci --only=production --ignore-scripts`; symlink `/shared-libs/node_modules` to `/app/node_modules` and install shared libs with `--only=production --ignore-scripts`.
> - **Scripts**:
>   - **New** `docker/install_shared_libs.sh`: Locates shared libs in `shared-libs` or `/shared-libs` and installs `config-utils`, `firestore-utils`, `auth-utils`; supports `--npm-args`.
> - **Shared libs (config-utils)**:
>   - Use `DEFAULT_PRIMARY_MODEL` in `PRIMARY_MODEL_OPTIONS` instead of a string literal.
