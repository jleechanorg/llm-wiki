## Raw Body
## Summary
- Pin Node.js version to 22.x to fix Render build failure
- Render was using Node.js 24.x (npm 11.x) which is incompatible with lock files generated by npm 8.x/10.x

## Root Cause
The `package.json` engine constraint `>=20.0.0 <25.0.0` allowed Render to pick Node.js 24.11.1, which bundles npm 11.x. The `package-lock.json` was generated with an older npm version and `npm ci` fails with:

```
npm error Missing: fsevents@2.3.3 from lock file
npm error Missing: @esbuild/aix-ppc64@0.25.11 from lock file
...
```

## Changes
- Constrain engines to `>=20.0.0 <23.0.0` in root and backend package.json
- Add `.node-version` file with `22` for explicit version pinning

## Why This Fixes It
- Render respects both `engines` field and `.node-version` file
- This aligns with the Dockerfile which uses `node:22-alpine`
- Ensures consistent Node.js versions across GCP Cloud Run and Render
