Implements a centralized fallback primary model constant using TDD methodology. This provides a single source of truth for the fallback model (Grok) to be used when the primary model (Cerebras) fails or is unavailable.

**Changes:**

Shared-libs (config-utils):
- Add FALLBACK_PRIMARY_MODEL = 'grok' constant in SecondOpinionConfig.ts
- Export FALLBACK_PRIMARY_MODEL from config-utils/index.ts
- Add comprehensive test suite (20 tests) in SecondOpinionConfig.test.ts
  - Tests constant existence, type safety, and centralization
  - Validates integration with PRIMARY_MODEL_OPTIONS
  - Ensures fallback model is available for error recovery

Backend:
- Re-export FALLBACK_PRIMARY_MODEL from backend/src/config/primaryModel.ts
- Add FallbackPrimaryModel.test.ts with 16 tests covering:
  - Constant exports and type safety
  - Integration with existing configuration
  - Real-world usage scenarios and fallback logic

**Architecture:**
- Single source of truth: shared-libs/packages/config-utils/src/SecondOpinionConfig.ts
- Backend re-exports for compatibility
- No hardcoded 'grok' strings for fallback scenarios needed
- Type-safe with const literal types ('grok')

**Testing:**
- TDD approach: Tests written first (red), then implementation (green)
- 36 new tests total (20 in shared-libs, 16 in backend)
- All tests pass: 112 tests in config-utils, 300 tests in backend
- Zero test failures or regressions

**Usage Example:**
```typescript
import { DEFAULT_PRIMARY_MODEL, FALLBACK_PRIMARY_MODEL } from '@ai-universe/config-utils';

const modelToUse = primaryModelFailed ? FALLBACK_PRIMARY_MODEL : DEFAULT_PRIMARY_MODEL;
// Returns 'grok' when primary fails, 'cerebras' otherwise
```

Addresses centralization requirement per CLAUDE.md model configuration architecture.
