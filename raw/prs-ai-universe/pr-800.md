## Summary

This PR improves conversation validation and test error handling across the backend and integration test suite:

**Backend Changes:**
- **Conversation Validation**: Added fail-fast validation in `SecondOpinionAgent.ensureConversationContext` to reject invalid conversationIds before processing
- **Error Re-throwing**: Updated error handling to re-throw validation errors (conversationId, missing params, invalid input) so FastMCP converts them to proper JSON-RPC error format
- **HTTP Logging**: Registered `requestLoggingMiddleware` in server.ts for debugging (enabled via `ENABLE_REQUEST_LOGGING=true`)

**Test Infrastructure:**
- **Auto-Detection**: New `testing_integration/conftest.py` auto-detects test output directory based on git repo and branch name (`/tmp/<repo>/<branch>/integration_tests/`)
- **Simplified Runner**: Renamed `run_integration_tests.sh` → `run_tests.sh` with zero-parameter execution and 5-worker parallelization
- **HTTP Logging**: Integration tests now enable HTTP request/response logging for debugging

**Test Fixes:**
- **metadata.secondOpinion**: Fixed 3 tests that incorrectly expected this field on all messages (it's only required for `agent.second_opinion` responses per JSON schema)
- **Error Format**: Updated 2 tests to accept both JSON-RPC error format (`{error: {...}}`) and FastMCP error format (`{result: {isError: true}}`)
- **Rate Limiting Validation**: Enhanced test to validate EXACT rate limit error message structure (fields, content, timestamp format)
- **Mock History**: Fixed `createMockConversationTool` to return non-empty conversation history by default

**Test Results:**
- Before: 4 passed, 20 failed (17% pass rate)
- After: 23 passed, 1 failed (96% pass rate)
- 1 remaining failure is rate limiting test (correctly fails when rate limiting is disabled via `RUN_LOCAL_SERVER_DISABLE_RATE_LIMIT=true`)
