## Summary

Address Cursor bot review comment from PR #539 about silent spawn failures in buildSharedLibs function.

## Problem

The buildSharedLibs function only checked result.status for npm command failures, missing result.error. This could silently ignore failures where the npm command itself couldn't spawn (e.g., npm not found).

## Solution

Added explicit error checks for spawn failures before status checks:
- Check installResult.error before installResult.status
- Check buildResult.error before buildResult.status
- Throw explicit error messages for spawn failures
