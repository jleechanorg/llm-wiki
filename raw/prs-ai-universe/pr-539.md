## Summary

Fix the DEFAULT_MAX_INPUT_TOKENS export error by ensuring shared-libs packages are built before backend npm install. This mirrors the deploy.sh logic for consistency across development and deployment workflows.

## Problem

After pulling changes, running the local server would fail with:
```
SyntaxError: The requested module @ai-universe/config-utils does not provide an export named DEFAULT_MAX_INPUT_TOKENS
```

This happened because:
1. Shared-libs packages weren't built after git pull
2. Backend's npm install created symlinks to unbuilt shared-libs
3. ESM modules couldn't find exports in missing dist/ directories

## Solution

Added buildSharedLibs() function to run-local-server.ts that:
- Builds each shared-libs package before backend dependencies
- Installs package dependencies if node_modules missing
- Runs build script if available
- Mirrors deploy.sh logic for consistency
