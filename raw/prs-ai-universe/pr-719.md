## Problem

PR #711 introduced a YAML syntax error on line 139 of `.github/workflows/deploy-dev.yml` that breaks all auto-deploy workflow runs.

**Error:**
```
error parsing called workflow "./.github/workflows/deploy-dev.yml" : You have an error in your yaml syntax on line 139
```

**Root Cause:**
GitHub Actions expressions `${{ }}` cannot be used inside heredoc literals. The workflow attempted to use `${{ steps.commit_meta.outputs.message }}` inside a `cat <<'MSG'` block, which the YAML parser treats as invalid syntax.

## Solution

- ✅ Replace heredoc with simple env variable pass-through
- ✅ Add `COMMIT_MESSAGE_CONTENT` to the env block
- ✅ Use `echo "$COMMIT_MESSAGE_CONTENT"` instead of heredoc

**Before:**
\`\`\`yaml
COMMIT_MESSAGE_FILE=\$(mktemp)
cat <<'MSG' > "\$COMMIT_MESSAGE_FILE"
\${{ steps.commit_meta.outputs.message }}
MSG
\`\`\`

**After:**
\`\`\`yaml
COMMIT_MESSAGE_FILE=\$(mktemp)
echo "\$COMMIT_MESSAGE_CONTENT" > "\$COMMIT_MESSAGE_FILE"

# ... in env block:
COMMIT_MESSAGE_CONTENT: \${{ steps.commit_meta.outputs.message }}
\`\`\`
