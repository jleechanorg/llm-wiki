> [!NOTE]
> Conversation tools now consistently resolve auth via AuthContextResolver (with idToken support and anonymous fallback) and tests cover auth-only scenarios and all tool endpoints.
> 
> - **Backend**:
>   - **Auth centralization**: `conversation.send-message` now always resolves auth via `AuthContextResolver`, independent of rate limiting.
>   - **Anonymous fallback**: Generates anonymous users when no verified auth is present; uses this for rate limiting context if needed.
>   - **Rate limiting**: Builds `rateLimitUser` from resolved auth or anonymous fallback; propagates full context (`ip`, `fingerprint`, `userAgent`, `sessionId`).
>   - **Schema updates**: Add `idToken` to `listConversationsSchema`, `historySchema`, and `conversation.delete` params.
>   - **Tool wiring**: `conversation.delete`, `conversation.list`, and `conversation.get-history` now use `AuthContextResolver` to derive `effectiveUserId`.
> - **Tests**:
>   - Extend `ConversationAgent.adminTokenVerification.test.ts` to mock additional conversation methods and cover:
>     - Auth without rate limiting.
>     - Centralized auth usage for delete/list/get-history.
>     - Anonymous handling for legacy `userId` without token.
>     - Rate limit context propagation.
