## Raw Body
Install @google-cloud/monitoring and add monitoring for:
- All HTTP/network call latency and error rates
- All LLM tool calls (Cerebras, Anthropic, Gemini, Perplexity, OpenAI, Grok)
- Token usage tracking across all models
- Streaming chunk metrics
- Server health checks

Features:
- Automatic batching (10s intervals, 100 metric batches)
- Graceful degradation (no crashes if GCP credentials missing)
- Comprehensive metric labels for filtering and grouping
- Helper functions for easy integration
- Full test coverage

Metrics tracked:
- tool_call_latency, tool_call_count, tool_call_errors
- http_request_latency, http_request_count, http_request_errors
- token_usage, streaming_chunks
- active_requests (gauge)
- auth_attempts, rate_limit_hits

Integration points:
- HttpClient.fetch() - automatic HTTP monitoring
- All LLM tools - automatic call/stream monitoring
- Health check endpoint - includes monitoring stats
- Easy-to-use wrappers for custom monitoring

Documentation:
- Comprehensive MONITORING.md with setup, usage, and examples
- Test suite for monitoring service
- Type-safe API with TypeScript

Setup:
- Requires GOOGLE_APPLICATION_CREDENTIALS env var
- Auto-enabled on Cloud Run via Workload Identity
- Configurable via optional monitoring.enabled config

Performance:
- Minimal overhead (~0.1ms per metric)
- Non-blocking batch writes
- Bounded memory usage
