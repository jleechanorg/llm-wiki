## Summary

Fixes HTTP 400 errors in OpenAI Responses API calls by correcting the `tool_choice` parameter format.

## Problem

OpenAI LLMTool was failing with HTTP 400 errors:
```
Error: Unknown parameter: 'tool_choice.name'
```

**Root Cause**: Invalid fields in `tool_choice` object introduced in commit 3964522e (PR #355, Oct 15 2025)

## Solution

Changed `tool_choice` format to comply with OpenAI Responses API specification:

### Before (WRONG)
```typescript
toolChoice: {
  type: 'tool',
  name: 'web_search',      // Invalid - causes HTTP 400
  tool: 'web_search',      // Invalid - causes HTTP 400
  tool_name: 'web_search'  // Invalid - causes HTTP 400
}
```

### After (CORRECT)
```typescript
toolChoice: {
  type: 'web_search'  // Valid - API accepts this
}
```
