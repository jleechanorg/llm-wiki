### 2. Dead Code Removal
After PR #590 moved rate limiting to Secret Manager, the Firestore/RuntimeConfigService rate limit code became non-functional dead code.

**Removed:**
- `rateLimit` field from `RuntimeConfig` interface
- Firestore rate limit defaults and environment variables
- `set-rate-limit` CLI command from `manage-config.ts`

**Why This Matters:**
- Operators could set rate limits via Firestore/env that would be **silently ignored**
- Creates confusion about how rate limits actually work
- Documentation misleads users about runtime configuration
