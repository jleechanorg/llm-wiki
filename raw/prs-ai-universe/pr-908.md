## Root Cause Analysis
**Problem:** Dev server synthesis consistently failing with:
```
Cerebras API error: 503 - {"message":"We're experiencing high traffic right now!",
"type":"too_many_requests_error","code":"queue_exceeded"}
```

**Investigation:**
1. GCP logs showed dev server has 4.5x more traffic than prod (178 vs 40 requests/day)
2. Both servers use identical code and direct Cerebras API
3. Cerebras rate limiting returns HTTP 503 with `queue_exceeded`, not standard 429
4. Existing retry logic only handled 429

**Fix:** Extended retry logic to handle 503 errors that contain rate-limiting indicators:
- `queue_exceeded`
- `too_many_requests`
- `high traffic`
