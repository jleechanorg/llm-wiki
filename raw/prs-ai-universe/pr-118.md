## Summary

Fixes authentication error: **"Firebase ID token has incorrect aud claim"**

The backend was using the wrong Firebase project ID for authentication validation, causing all user authentication attempts to fail.

## Changes

- ✅ Update `FirebaseAuthTool` to use `FIREBASE_AUTH_PROJECT_ID` (ai-universe-b3551)
- ✅ Add `FIREBASE_AUTH_PROJECT_ID` to deployment Secret Manager configuration
- ✅ Update `.env.example` with proper documentation for both project IDs
- ✅ Maintain backward compatibility with `FIREBASE_PROJECT_ID` fallback

## Root Cause

The backend was using the GCP project ID (`ai-universe-2025`) for Firebase Auth validation, but auth tokens are issued by the Firebase project (`ai-universe-b3551`). 

This is a valid architectural pattern where:
- **GCP Project** (`ai-universe-2025`): Infrastructure hosting (Cloud Run, Storage, etc.)
- **Firebase Project** (`ai-universe-b3551`): Authentication services
