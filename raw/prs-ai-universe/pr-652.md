## Raw Body
Security Fix:
- Backend now extracts userId from verified Firebase auth token
- Prevents userId spoofing attacks where malicious clients could impersonate other users
- Added injectAuthContext callback to HTTP proxy for auth context injection

Changes:
1. shared-libs/mcp-server-utils:
   - Added injectAuthContext callback to startFastMcpHttpProxy
   - Callback extracts userId from Authorization header before forwarding to FastMCP
   - Returns 401 if token verification fails

2. backend/server.ts:
   - Implemented injectAuthContext to verify Firebase token and inject _authenticatedUserId
   - Authenticated userId takes precedence over payload userId

3. backend/SecondOpinionAgent.ts:
   - Updated validateAndPrepareInput to use _authenticatedUserId
   - Logs security warning when payload userId doesn't match authenticated userId
   - Marks user as authenticated only when userId comes from verified token

4. Test:
   - Added userid-authorization.test.ts demonstrating the security vulnerability
   - Tests verify backend rejects userId spoofing attempts

Fixes the authorization error where conversation ownership checks failed because backend trusted payload userId instead of verified token userId.
