## Problem

Render frontends were failing with **"Rate limit exceeded"** errors when sending messages.

### Root Cause

The default rate limits were too restrictive for production usage:
- **Anonymous users**: 5 requests/hour
- **Authenticated users**: 50 requests/hour

Multi-model requests (like `/secondo`) make **3-5+ backend calls per user action**, causing users to hit the 5 req/hour limit after just 1-2 requests.

## Solution

Increased default rate limits in `RuntimeConfigService.ts`:
- **Anonymous**: 5 → **100 requests/hour** (20x increase)
- **Authenticated**: 50 → **500 requests/hour** (10x increase)

## Impact

- ✅ Fixes "Rate limit exceeded" errors on all Render deployments
- ✅ Allows ~20 multi-model requests per hour for anonymous users
- ✅ Allows ~100 multi-model requests per hour for authenticated users
- ✅ No breaking changes - only increases limits
