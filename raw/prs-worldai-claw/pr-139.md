## Background

Phase 4 of the Living Companions feature requires players to see companions from other sessions at the same location. This endpoint enables cross-session companion presence without a full world-state broadcast. Related: companions-phase4.

## Goals

- Add a GET /api/companions/nearby?sessionId=X endpoint returning companions from other sessions co-located with the requesting session's companions.
- Keep implementation minimal and consistent with existing repository patterns.

## Tenets

- Prefer real database queries over in-memory aggregation.
- No breaking API changes; new endpoint only.
- No schema migration needed (location already stored in companions.current_location).

## High-level description of changes

1. New repository method getCompanionsByLocation(location, excludeSessionId?) on SQLiteCompanionRepository querying companions by current_location, optionally filtering out the requester's own session.
2. New route GET /api/companions/nearby?sessionId=X fetches the session's companions, collects unique locations, queries others at those locations, deduplicates by companion ID.
3. Unit tests in packages/backend/src/storage/companions_nearby.test.ts: basic lookup, exclusion by sessionId, empty location, and multi-location deduplication.
