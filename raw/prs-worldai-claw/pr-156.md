## Summary

listActiveCampaigns() in packages/backend/src/world/sqlite_world_store.ts was changed in PR #63 to return at most one campaign (the most recently updated within 2 hours). All other active campaigns were silently excluded from the scheduler loop and would never tick.

## Root cause

The SQL query was modified to add LIMIT 1, collapsing results to a single entry. Callers in world_scheduler.ts and faction_store.ts iterate over the returned list expecting all due campaigns.

## Impact

Any deployment with more than one active campaign would see all but the most recently updated campaign stall indefinitely.

## Fix

Removed LIMIT 1 from the store query and return all active campaigns via .all() + map().

Closes #77
