## Background

worldai_claw was not fully wired into the agent orchestrator (`ao`) lifecycle. The `agent-orchestrator.yaml` had an entry for `worldai-claw` but with incorrect paths (`~/projects/worldai_claw`, `~/.worktrees/worldai-claw-main`) that don't exist. The repo itself was missing the supporting files (ralph/, launchd/, setup docs) that make autonomous agent sessions reliable and self-documenting.

## Goals

- Wire worldai_claw into the agent orchestrator with correct worktree paths
- Add ralph/ PRD-driven autonomous workflow toolkit
- Add launchd lifecycle worker plist template
- Add onboarding docs (SETUP.md, BOOTSTRAP.md)
- Add orchestrator session lifecycle rules to AGENTS.md

## Tenets

- Mirror jleechanclaw harness conventions for consistency
- Make setup self-service via scripts and docs, not tribal knowledge
- Keep paths pointing to actual filesystem locations (`~/.workai-claw/`, not hypothetical ones)

## High-level description of changes

- **`agent-orchestrator.yaml`** — Fixed `worldai-claw` project entry: `path` → `~/.worktrees/worldai-claw/wc-1`, `worktreeDir` → `~/.worktrees/worldai-claw`, added `backfillAllPRs: true`, enriched `agentRules` with worldai_claw-specific quality gates (real LLMs in test suites, no mock flags, no hardcoded fantasy keywords, no template narration fallbacks).
- **`ralph/`** — Full ralph PRD-driven autonomous toolkit copied from jleechanclaw reference: `ralph.sh`, `ralph-pair.sh`, `CLAUDE.md`, `prd.json`, all `lib/` helpers (workspace, tools, status, metrics, evidence, terminal_recorder, dashboard), evidence_recorder scripts.
- **`launchd/`** — `com.worldai-claw.lifecycle.plist.template`: launchd plist for `ao lifecycle-worker worldai-claw` daemon.
- **`scripts/install-worldai-claw-lifecycle.sh`** — One-shot installer that validates prerequisites, generates plist from template, loads via launchctl.
- **`SETUP.md`** — Complete setup guide covering ao CLI verification, project registration, lifecycle worker install, ralph
