## Background

This PR extends the 8-bit RPG engine (`rpg-engine/`) and the shared `packages/game-engine` library with actor inventory management, proper module encapsulation, and updated documentation. It addresses all outstanding CodeRabbit review comments from the previous iteration.

## Goals

- **Actor inventory system**: Add an `Item` interface and `inventory` member to `Actor`, with `addItem()` and `getItems()` methods.
- **Data integrity**: `getItems()` returns a deep-cloned array to prevent external callers from mutating the actor's internal state.
- **Module encapsulation**: Export `Stats` and `Actor` directly from the `Actor` module (not re-declared in `index.ts`). Export `Item` from the new `inventory/Item` module.
- **Standardised library surface**: `packages/game-engine/src/index.ts` now re-exports `entities/Actor`, `inventory/Item`, and all existing modules so consumers import from a single entry point.
- **Documentation hardening**: `rpg-engine/README.md` updated with MCP server setup guide, REST API reference (`GET /api/game`, `POST /api/game/action`), and an expanded architecture note listing all major subsystems (`TileMap`, `Actor`, `DialogueSystem`, `InventorySystem`, `CombatSystem`).

## Changes

### `packages/game-engine/src/inventory/Item.ts` *(new)*
Defines the `Item` interface with `id`, `name`, `description`, `type`, and `value` fields.

### `packages/game-engine/src/entities/Actor.ts`
- Added `import { Item }` from the new inventory module.
- Added public `inventory: Item[] = []` field.
- Added `addItem(item: Item)` mutator.
- Added `getItems(): Item[]` returning a deep-cloned copy of the inventory.
- `Stats` and `Actor` are both `export`ed directly from this file.

### `packages/game-engine/src/index.ts`
- Added `export * from './entities/Actor'` and `export * from './inventory/Item'` so both types are reachable from the package root.
