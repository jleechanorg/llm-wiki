## Background
IpfsBridge in `packages/backend/src/storage/ipfs_bridge.ts` used browser `fetch()` APIs for all IPFS HTTP calls. These do not work reliably in Node.js environments, causing health checks, pin operations, and content fetches to fail.

## Goals
- Replace all browser `fetch()` calls with Node.js native `http`/`https` modules
- Maintain identical public API and behavior
- Prove the migration works with TDD tests using real HTTP servers

## Tenets
- No public API changes — drop-in replacement
- Real HTTP test servers over mocks
- TDD: tests written and verified failing before implementation

## High-level description of changes
Introduced a `nodeRequest()` helper function that wraps Node.js `http.request()`/`https.request()` with timeout support. Replaced all 6 `fetch()` call sites (health checks, pin operations, fetch operations for both local IPFS and Pinata providers). Removed the `createAbortWithTimeout()` method and the `globalThis.fetch` version check that are no longer needed.

## Testing
- **17 new TDD tests** in `ipfs_bridge_node_http.test.ts` using real `http.createServer()` test servers
  - 3 tests verify `global.fetch` is NOT called (the core migration proof)
  - 4 tests verify local IPFS health check behavior
  - 3 tests verify local IPFS pinBlock behavior
  - 3 tests verify local IPFS fetchByCid behavior
  - 2 tests verify Pinata provider without JWT
  - 2 tests verify timeout handling
- **14 existing tests** updated to remove `global.fetch` mocks — all pass
- Total: **31 tests passing**

## Low-level details

- **`packages/backend/src/storage/ipfs_bridge.ts`**: Replaced `fetch()` with `nodeRequest()` using Node.js `http`/`https` modules. Added imports for `node:http`, `node:https`, `node:url`. Removed `createAbortWithTimeout()` and `globalThis.fetch` check. `nodeRequest()` handles timeouts via `req.timeout` + `req.destroy()`.

- **`packages/backend/src/storage/ipfs_bridge.test.ts`**: Removed all `global.fetch` mock setup/assertions. Tests now
