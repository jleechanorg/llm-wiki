> **Overview**
> **Optimizes `entity_chain` startup by avoiding redundant schema work.** `SQLiteEntityChain.initializeTables()` now runs the `CREATE TABLE`/migration logic only once per `better-sqlite3` `Database` instance via a `WeakSet` of initialized DBs, and `clearEntityChainCache()` resets this tracking.
> 
> **Tightens migration error handling.** The `ALTER TABLE ... ADD COLUMN` step now rethrows unexpected errors instead of swallowing all exceptions.
> 
> Adds a regression/perf test (`entity_chain_perf.test.ts`) asserting that creating multiple chains on the same in-memory DB does not re-run schema creation SQL.
