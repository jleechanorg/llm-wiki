## Background
Two P0 determinism bugs in `faction_simulator.ts` caused companion actions to produce different results on repeated calls with the same inputs. This breaks reproducibility for testing, debugging, and simulation replay.

## Goals
- Make `resolveCompanionAction` fully deterministic for any given companion + seed combination
- Eliminate all `Math.random()` and `Date.now()` usage from companion action resolution

## Tenets
- Minimal changes â€” only fix the determinism bugs
- TDD â€” failing tests written before implementation
- Preserve existing behavior when seeds are provided

## High-level description of changes
**WC-a22**: `resolveCompanionAction` previously fell back to `Math.random` when no `tickSeed` was provided. Now it always uses `createSeededRng()` with `tickSeed ?? 0` as the deterministic fallback.

**wc-qh7**: `seedForIds` used `Date.now()` when `tickSeed` was undefined, producing different item IDs each call. Now uses the same deterministic effective seed.

Also hardened `resolvePlayerOfflineAction` default parameter (was `Math.random`, now a seeded RNG). This was dead code since all callers already pass a seeded RNG, but prevents future misuse.
