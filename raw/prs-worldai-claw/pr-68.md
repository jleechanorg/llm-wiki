## Summary

This PR introduces the design specification for an LLM-driven custom campaign wizard that allows players to create campaigns through either a guided step-by-step wizard or by pasting a freeform prompt. The wizard consists of 7 hardcoded planning block rounds (Setting, World/IP, Gender, Name, Background, Class, Faction) with LLM-generated suggestions for each round based on prior selections.

## Key Changes

- **New Backend Endpoint**: `POST /campaign-wizard/suggestions` that generates contextual LLM suggestions for each wizard round, accepting prior selections and returning up to 6 suggestion cards with labels and descriptions
- **Frontend State Machine**: Complete rewrite of `NewCampaignWizard.tsx` with phase-based navigation (choose-type → custom-mode → wizard-round/paste-prompt → expertise → launch)
- **Wizard Round UI**: Planning block interface showing progress dots, 4 LLM-generated suggestion cards, free-text input, and "Let AI pick the rest" escape hatch at every round
- **Campaign Launch Integration**: Wizard selections compiled into a system prompt for world generation, with universe_type derived from the setting selection
- **Comprehensive TDD Plan**: Detailed test specifications across 4 phases covering schema validation, prompt construction, LLM response parsing, state machine transitions, UI interactions, and payload generation

## Notable Design Decisions

- Rounds are hardcoded for structure; choices are LLM-generated for flexibility
- "Let AI pick the rest" available at every round for fast-track campaign creation
- No client-side caching of suggestions—each round fetches fresh contextual suggestions
- Paste prompt path provides power-user alternative with no LLM calls during creation
- Universe-agnostic suggestion prompts that adapt to any setting chosen by the player
- 30-second timeout with no retry for interactive fast-fail behavior
