## Background

`anchorHash` in `packages/backend/src/network/chain_anchor.ts` allows any caller to overwrite any entity anchor because the `anchorOwners` map is never populated. Ownership recording was deferred to after successful RPC tx confirmation, but since the RPC stub never succeeds, no ownership is ever recorded â€” making `isAnchorOwnerAuthorized` always return `true`.

## Goals

- First caller to anchor an entity becomes its owner
- Only the anchor owner or contract owner can update that entity anchor
- Unauthorized callers receive a clear rejection

## Tenets

- Minimal fix â€” activate existing access control, no class restructuring
- TDD â€” tests written first, verified failing, then fix applied
- Stub-safe â€” access control works even before ethers.js RPC integration

## High-level description of changes

Moved ownership recording (`anchorOwners.set()`) from the commented-out post-RPC section to before the RPC attempt. This activates the existing `isAnchorOwnerAuthorized` check that was already in place but had no data to work with.

## Testing

11 new TDD tests covering:
- First-caller ownership recording
- Owner can update (including case-insensitive address matching)
- Unauthorized caller rejection (different address, missing address, empty address)
- Contract owner override (including case-insensitive)
- Independent per-entity ownership
- Access control via `anchorEntityChain` path

All 22 tests pass (11 new + 11 existing entity chain lifecycle tests).

## Low-level details

- **`packages/backend/src/network/chain_anchor.ts`**: Added 4-line ownership recording block after authorization check, before RPC try block. Removed stale comment about deferred ownership recording.
- **`packages/backend/src/__tests__/chain_anchor_access_control.test.ts`**: New test file with 11 tests across 6 describe blocks covering all access control scenarios.
