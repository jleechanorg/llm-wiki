## Raw Body
## Summary

- **P0 bug fix — system instruction stack now preserved**: The session/system prompt stack sent to the LLM now combines canonical `SYSTEM_INSTRUCTION` with stored `session.system_prompt` when present, so custom campaign/session setup is still honored without losing the 9-layer GM guardrails.
- **Budget accounting aligned to actual prompt stack**: `enforceBudget()` now receives the same combined system prompt string used for the LLM call on both POST and GET turn routes.
- **Transport capture backward compatibility**: OpenClaw transport trace capture falls back to `GEMINI_HTTP_REQUEST_RESPONSE_CAPTURE_PATH` when `LLM_HTTP_REQUEST_RESPONSE_CAPTURE_PATH` is not set.
- **Smoke test hardening**: Additional envelope assertions, campaign+character 3-action smoke flow, and response handling checks for status before schema validation.
- **System design doc**: `docs/system_design.md` with Mermaid sequence + component diagrams, storage schema, env var reference.
- **README refresh**: Links to system design, adds storage/inference details, `OPENCLAW_GATEWAY_TOKEN` in Getting Started.

## Changes vs `origin/main`

### Backend turn handling (`packages/backend/src/app.ts`)
- POST `/sessions/:id/turn` and GET `/sessions/:id/turn` now use:
  - system message: `SYSTEM_INSTRUCTION`
  - appended with `session.systemPrompt` when provided
- `enforceBudget()` now receives the same concatenated system prompt that is passed to the LLM.

### OpenClaw client (`packages/backend/src/services/openclawClient.ts`)
- `_appendLlmTransportCapture()` reads env at call time and falls back to legacy `GEMINI_HTTP_REQUEST_RESPONSE_CAPTURE_PATH`.

### Smoke tests (`testing_mcp/`)
- `test_worldai_claw_api.py`: removes `domain_events` from banned legacy key list, validates `choices`/`state_delta`/`mechanic_requests`, and checks action HTTP status before schema validation in `test_smoke_campaign_3_actions`.
- `lib/base_test.py`, `lib/evidence_utils.py`, `lib/worldai_claw_test_utils.py`: trace path
