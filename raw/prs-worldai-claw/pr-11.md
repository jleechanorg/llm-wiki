## Summary
- Unified local runtime: single `run_local_server.sh` process serves both API and frontend on one port, replacing the separate Vite dev server model.
- Removed dead-code backend stubs: deleted 8 files (`openclaw_client.ts`, `routes/turn.ts`, `routes/sessions.ts`, `memory/session_store.ts`, `services/sessionStore.ts`, `storage/firestore.ts`, `storage/index.ts`, `tests/firestoreSync.test.ts`) that were superseded by the real OpenClaw proxy in `server.ts`.
- Added auth token injection to all frontend API calls via an expanded `apiFetch` wrapper (reads from `localStorage.worldclaw_auth_token` or `VITE_WORLDCLAW_TOKEN`; does not overwrite explicit headers).
- Fixed auth/rate-limit bypass for `/health?...` (query-string paths were not matching the literal `request.url === '/health'` check).

## Production Code Changes

### File: `packages/backend/src/app.ts`
**GOAL**: Fix auth bypass check failing for `/health?full=1` (query string appended).
**MODIFICATION**: Changed `request.url === '/health'` to `requestPath === '/health'` in both the rate-limit allowList and the `onRequest` auth guard.
**NECESSITY**: `request.url` includes query string; `_getRequestPath` strips it. Without this fix, health checks with any query param hit the auth wall.
**INTEGRATION PROOF**: `_getRequestPath` was already defined in `app.ts` and used in the auth hook for all other path comparisons.

### File: `packages/backend/src/llm/openclaw_client.ts` *(deleted)*
**GOAL**: Remove obsolete stub client.
**NECESSITY**: Placeholder never wired into production flows; real gateway calls go through the OpenClaw proxy route in `server.ts`.

### File: `packages/backend/src/routes/turn.ts` *(deleted, 157 lines)*
**NECESSITY**: The real `/api/sessions/:id/turn` is handled by the OpenClaw proxy. This was only a stand-in.
