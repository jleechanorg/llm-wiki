## Raw Body
## Background

Explorer and Quester companions could not effectively level up because the XP-to-level math contained division thresholds that produced 0 and double failure penalties. The `discoveryBoost / 2` and `cueBonus / 2` formulas always evaluated to 0 since the `'discovery'` cue tag was never present in any act phase. Additionally, failure XP was double-penalized: `rollCompanionXp` already uses a reduced failure base, but Explorer/Quester applied an additional 0.35x/0.45x multiplier on top.

## Goals

1. Ensure Explorer and Quester mode-specific XP bonuses are non-zero regardless of cue tags
2. Remove double failure penalties so failure XP is at least 25-30% of success XP
3. Scale XP awards with companion level so higher-level progression remains viable against exponentially growing D&D 5e thresholds
4. Add `'discovery'` cue tag to all act phases so the discoveryBoost bonus actually activates

## Tenets

- TDD: all 14 regression tests written before any production code change
- Minimal scope: only touch XP calculation paths for Explorer/Quester
- No breaking changes to other companion modes (Guard, Trader, Aggressive, Balanced)

## High-level description of changes

Three root causes fixed in `faction_simulator.ts`:

1. **Dead cue tag**: `'discovery'` added to all `getActPhase` cue tags (Act 1/2/3), so `discoveryBoost / 2` and `cueBonus / 2` can produce non-zero values.

2. **Double failure penalty removed**: Explorer's `0.35x` and Quester's `0.45x` failure multipliers removed. `rollCompanionXp` already returns reduced XP on failure (explore: 20 vs 55, quest: 35 vs 95), so the additional multiplier was redundant.

3. **Level-scaled XP bonus**: Added `Math.max(companionLevel * 7, discoveryBoost / 2)` for Explorer and `Math.max(companionLevel * 8, cueBonus / 2)` for Quester. This ensures progression scales with level to offset exponentially growing D&D 5e thresholds (300 → 900 → 2700 → 6500 → 14000...).
