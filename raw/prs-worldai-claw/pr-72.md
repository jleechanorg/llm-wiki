## Background

The original `game_state_instruction.md` is 36,443 tokens — far too large for the system prompt. Per the design (`roadmap/worldai_claw_mvp.md`, "Exact 50k Context Business Logic"), game state must be injected into the 40k non-system context budget alongside existing memory blocks.

This PR defines the per-turn context injection format: how the `ContextBuilder` assembles the per-turn context payload.

## Goals

- Define a structured `game_state` block carrying authoritative per-turn game state (player stats, location, party, objectives, compact artifacts)
- Ensure `game_state` is NEVER trimmed so the LLM always has accurate state
- Add structured TypeScript types for all game-state fields
- Add deterministic format with fixed section ordering for stable token counts
- Define the full format in a design doc

## Tenets

- **Never trim game state**: game_state is floored at 0 tokens — it must always be preserved
- **Structured over embedded**: game-state fields are typed and assembled at build time, not parsed from free-form history
- **Deterministic output**: sections appear in fixed order; only non-empty sections are emitted
- **Minimal diff**: changes are additive to existing block infrastructure

## High-level description of changes

1. **`token_budget.ts`**: Add `game_state: 0` to `TARGET_ALLOCATION`; add `game_state` to `ContextBlock` type. Update trim order comment to exclude `game_state`.

2. **`context_builder.ts`**: Extend `StoryState` with `player_character_data`, `campaign_state`, `current_location`, `party`, `current_objectives`, `compact_artifacts`. Add `assembleGameStateBlock()` private method that formats these fields as `[section]\ncontent` blocks in fixed order. Position `game_state` block after `recent_transcript` and before `current_turn`. Remove `game_state` from trim priorities.

3. **`context_builder.test.ts`**: Add 8 new tests covering: game_state inclusion with player_character_data, campaign_state/location/party, current_objectiv
