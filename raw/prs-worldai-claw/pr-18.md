## Raw Body
### Motivation

- Restore mobile SSE compatibility by re-enabling a GET-based streaming path for EventSource clients that broke when `/sessions/:id/turn` was POST-only.  
- Harden shared bgTheme persistence to avoid invalid persisted values and unhandled storage errors.  
- Prevent out-of-range runtime errors in star constellation generation for small `totalCount` values.  
- Align unit tests and provide an audit entry tracking which review comments were fixed/considered.

### Description

- Reintroduced a backward-compatible route for `/sessions/:id/turn` using `app.route({ method: ['GET','POST'], url: '/sessions/:id/turn', handler: ... })` and added GET query parsing/validation (including optional JSON `client_capabilities`) so EventSource GET requests are served and SSE behavior is enforced for GET clients.  
- Made streaming detection include GET semantics (`wantsStreaming` now treats GET as streaming) and preserved the POST non-streaming flow.  
- Hardened `useBgThemeCore` by adding `[storage]` to the restore effect, validating persisted values against `VALID_BG_THEMES`, and catching read/write errors to avoid unhandled rejections.  
- Fixed `initStarData` to compute a `safeCount = Math.min(totalCount, CONSTELLATION_COUNT)` and use it for both constellation marking and edge generation to avoid out-of-bounds access.  
- Updated the shared test `useBgThemeCore.test.ts` to expect fallback-to-default for invalid persisted themes, and added `responses.json` with FIXED/CONSIDERED review URLs and the codex automation marker for traceability.
