## Scope
Production-focused hardening for autonomous living-world generation on `feat/bg3-world-generators`.

This PR now covers two areas:
1. Prompt contract/runtime budget safety and required invariants.
2. Generator execution correctness under concurrent MCP world-action calls.

## Key Bug Fixes
1. Prompt packing contract correctness
- Optional-drop flow now removes optional sections before fallback and preserves valid required contract.
- Hybrid prompt guidance (narrative + mechanics) is preserved under budget pressure.
- Required coupling guard keeps `game_state` + planning protocol markers valid as a pair.

2. MCP generator concurrency safety
- Added campaign shard waiter signaling so competing generators are notified on lease release (not just poll loops).
- Fixed lost-update risk in concurrent `generate_world_group_actions` persistence by rebasing leased-faction updates onto latest world state before save.
- Reapplies generated world events to rebased state before advancing world time and persisting.

3. MCP generator contract cleanup
- Tool schema/handler now enforce identifier consistency (`campaign_id` + `world_id` must match if both provided).
- Imported faction initialization helpers from canonical module path for consistency.

4. World-output quality regression fix
- Restored combat descriptions to player-facing prose (removed debug key/value output signatures).
