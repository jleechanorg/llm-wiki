## Background

LLM-generated `scene_text` in WorldArchitect responses can claim permanent outcomes (combat rewards, NPC deaths, XP gains, level-ups) without corresponding `state_delta` or `mechanic_requests`. This violates the Evidence Fidelity Contract documented in Layer 5 of the system prompt (already present in `system_instruction.md` lines 154-158) but had no programmatic enforcement.

## Goals

- Add a `validateEvidenceFidelity()` function that checks `scene_text` claims are backed by `state_delta` or `mechanic_requests`
- Cover the 3 violation types: `FINALIZED_HIDDEN_OUTCOME`, `PENDING_MECHANIC_UNRESOLVED`, `CLAIM_WITHOUT_STATE_SUPPORT`
- TDD: 10 unit tests driving the implementation

## Tenets

- Prefer real code over mocks; unit tests use the real validation logic
- No universe-specific keyword bias — regexes match structural game mechanics, not fantasy vocabulary
- Minimal, auditable change — one new file + one modified file

## High-level description of changes

Added `validateEvidenceFidelity()` to `packages/backend/src/llm/system_instruction.ts` with 4 enforcement rules:

1. **FINALIZED_HIDDEN_OUTCOME** — if `scene_text` is non-empty but both `state_delta` and `mechanic_requests` are empty, the response is claiming permanent outcomes with no backing
2. **PENDING_MECHANIC_UNRESOLVED** — if `mechanic_requests` is non-empty and `scene_text` claims a reward or NPC outcome, the outcome cannot be finalized before the mechanic is resolved
3. **CLAIM_WITHOUT_STATE_SUPPORT (XP)** — if `scene_text` claims XP without an XP field in `state_delta`, flagged
4. **CLAIM_WITHOUT_STATE_SUPPORT (level)** — if `scene_text` claims a level-up without a level/progression field in `state_delta`, flagged

A new test file `packages/backend/src/llm/system_instruction.test.ts` provides 10 TDD-driven unit tests covering all rules and edge cases.
