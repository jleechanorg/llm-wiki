## Background

Track E (party multiplayer) requires party creation and join/leave. The existing codebase only had `GET /sessions/:id/party`. Tests in `testing_mcp/test_party_multiplayer.py` hit `POST /parties` and got 404.

## Goals

- Add `POST /parties`, `POST /sessions/:id/party/join`, `DELETE /sessions/:id/party/leave`
- Add `GET /sessions` to list all sessions for authenticated user
- All routes SQLite-backed for durability

## Tenets

- Use existing SQLite DB pattern (same as sessions, campaigns)
- Minimal schema: `parties(id, created_at, session_ids)` + `party_id` on sessions
- Fix pre-existing turn route bug (`playerInput = playerInput ?? ...` self-reference)

## Changes

### packages/backend/src/app.ts
- `GET /sessions`: lists sessions from SessionStore + party_id from SQLite
- `POST /parties`: creates party with UUID, returns `{partyId, sessionIds: []}`
- `POST /sessions/:id/party/join`: upserts session to SQLite, adds to party.session_ids, sets sessions.party_id
- `DELETE /sessions/:id/party/leave`: removes session from party.session_ids, clears sessions.party_id
- Bootstrap migration: `ALTER TABLE sessions ADD COLUMN party_id TEXT` with PRAGMA table_info verification (fails fast if schema unexpected)
- Bootstrap parties table: `CREATE TABLE IF NOT EXISTS parties` with PRAGMA backfill for missing columns
- Remove lazy `CREATE TABLE IF NOT EXISTS` from POST /parties handler (schema guaranteed at bootstrap)
- Remove silent catch blocks from joinPartyTx so errors abort the transaction (atomic join)
- Fix turn route self-reference: `const playerInput = (payload.player_input ?? payload.action) as string`

### packages/backend/src/schemas/contracts.ts
- `player_input` now optional in turnRequestSchema; `action` field added as alias
- `party_id` accepted in turn requests

### packages/backend/tests/party_routes.test.ts
- 10 Jest unit tests covering all new routes + auth
- Per-test SQLite isolation via `setDatabasePath` + `resetDatabase` in beforeEach/afterEach
