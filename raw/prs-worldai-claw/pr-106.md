## Raw Body
## Background
Catch-up narration displays "gained 0 levels" when companions have actually leveled up during offline time. The root cause is that `resolveCompanionAction` only stored post-action stats (level, xp) but not the pre-action baseline, making it impossible for the catchup endpoints to compute level gains.

## Goals
- Correctly report companion level gains in catch-up narration
- Provide structured `companionLevelGains` data in the narration API response
- Include level gain context in the LLM narration prompt so the narrator can accurately describe progression

## Tenets
- Minimal, targeted fix — only touch the level tracking gap
- Backwards compatible — actions without `levelBefore` gracefully default to 0 gains
- TDD — tests written first, then implementation

## High-level description of changes
1. **`resolveCompanionAction`** now stores `levelBefore` and `xpBefore` in the action delta before XP progression is applied. This gives downstream consumers the pre-action baseline needed to compute level gains.

2. **`GET /sessions/:id/companions/catchup`** now computes `levelsGained` per companion by comparing the first action's `levelBefore` with the last action's `stats.level`.

3. **`GET /api/catch-up-narration/:campaignId`** now:
   - Computes per-companion level gains from action rewards
   - Includes level gain summaries (e.g. "Kira gained 2 levels during this time") in the event data sent to the LLM narrator
   - Returns a `companionLevelGains` array in the response
