## Background

The `entity_chain.ts` SQLite database has `PRAGMA foreign_keys = ON` in `initializeTables()` and a `FOREIGN KEY(entity_id) REFERENCES entity_chains(entity_id)` constraint on the `entity_blocks` table. However, there were no tests verifying that FK enforcement actually works — orphan inserts could silently succeed if someone removed the PRAGMA.

## Goals

- Add regression tests proving FK constraints are enforced on the `entity_blocks` table
- Ensure the PRAGMA is ON after construction
- Catch any future removal of the `foreign_keys = ON` pragma

## Tenets

- TDD: tests written to cover existing untested behavior
- Prefer real SQLite over mocks
- Minimal scope — test file only, no implementation changes needed

## High-level description of changes

Added 5 tests in `entity_chain_foreign_keys.test.ts` that verify:
1. PRAGMA `foreign_keys` is `1` after `SQLiteEntityChain` construction
2. Direct INSERT into `entity_blocks` with a non-existent `entity_id` throws `FOREIGN KEY constraint failed`
3. Direct INSERT succeeds when the parent `entity_chains` row exists
4. FK enforcement works across multiple chain instances sharing one DB connection
5. Orphan block rejection works functionally (end-to-end FK guard)
