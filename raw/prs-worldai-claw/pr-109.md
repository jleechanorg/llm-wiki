## Background
`coerceAutonomyMode()` silently accepted any string input and defaulted to `BALANCED`, meaning typos, garbage data, or invalid mode strings propagated through the companion system without any error signal.

## Goals
- Reject invalid autonomy mode strings at runtime with a descriptive error
- Catch invalid modes early at the `resolveCompanionAction` entry point
- Preserve existing `coerceAutonomyMode` for backward-compatible coercion

## Tenets
- Fail fast on invalid input at system boundaries
- TDD: tests written and verified failing before implementation
- Minimal scope

## High-level description of changes
Added `validateAutonomyMode()` in `faction_types.ts` that checks against enum values and throws with a descriptive error. Wired into `resolveCompanionAction()` as the entry-point guard.

## Testing
- 8 new TDD tests in `faction_types.test.ts` (valid, invalid, empty, undefined, null, numeric, typos, case)
- 3 new TDD tests in `faction_simulator.test.ts` (valid passthrough, invalid/empty throw)
- All 40 world tests pass with no regressions

## Low-level details
- **packages/backend/src/world/faction_types.ts**: Added `VALID_AUTONOMY_MODES` set and `validateAutonomyMode()` export
- **packages/backend/src/world/faction_types.test.ts**: Added 8 tests for `validateAutonomyMode`
- **packages/backend/src/world/faction_simulator.ts**: Imported `validateAutonomyMode`, added call at top of `resolveCompanionAction`
- **packages/backend/src/world/faction_simulator.test.ts**: Added 3 tests for validation at the boundary
