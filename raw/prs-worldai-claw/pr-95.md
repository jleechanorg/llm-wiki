## Background
When a player reconnects after being offline, they have no context about what happened in the game world while they were away. The existing `/sessions/:id/companions/catchup` endpoint returns raw action data but no narrative summary.

## Goals
- Provide an LLM-generated narrative summary of missed events when a player reconnects
- Query both companion and player offline actions since a given timestamp
- Return structured JSON with narration text, event count, and time span

## Tenets
- Real LLM narration only — no template fallbacks (per CLAUDE.md)
- Minimal changes — reuse existing repository patterns and narrator infrastructure
- Campaign-scoped — works across all sessions in a campaign

## High-level description of changes
Added `GET /api/catch-up-narration/:campaignId?lastSeen=<ISO timestamp>` endpoint that:
1. Validates the `lastSeen` ISO timestamp
2. Verifies the campaign exists
3. Queries `companion_actions` and `player_offline_actions` since lastSeen (joining through sessions by campaign_id)
4. Resolves companion names for readable event summaries
5. Calls `SLMNarrator.narrate()` with `scene_type: 'companion_catchup'` to generate prose
6. Returns `{ narration, eventCount, timeSpan: { from, to } }`

Returns `{ narration: null, eventCount: 0 }` when no events occurred since lastSeen.
