## Background

CompanionPanel was receiving companions data from its parent but had no dedicated API client layer. The companion fetch logic in App.tsx was inline.

## Goals

- Add a typed API client for companion endpoints
- Replace inline fetch in App.tsx with the new client
- Add unit tests for the API client and CompanionPanel

## Tenets

- Companions are optional data — API errors are non-fatal, default to empty array
- TDD: tests written first, implementation follows

## Changes

### packages/web/src/api/apiCompanions.ts (new)
- `fetchCompanions(sessionId)` — GET list, returns `Companion[]` (empty on any error)
- `fetchCompanion(sessionId, companionId)` — GET single, returns `Companion | null` (null on 404)

### packages/web/src/App.tsx
Replaced inline `apiFetch(.../companions...)` block with `fetchCompanions()` call.

### packages/web/src/components/CompanionPanel.tsx
Fixed: default `companions = []` prop to prevent crash on undefined.

### packages/web/tests/companionPanel.test.tsx (new)
18 tests: rendering, status badges, HP bar, bond level, collapse toggle, empty state.

### packages/web/tests/apiCompanions.test.ts (new)
11 tests: URL construction, response parsing, 404 → null, non-fatal error → `[]`.
