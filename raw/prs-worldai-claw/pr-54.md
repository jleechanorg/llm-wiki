> [!NOTE]
> **Medium Risk**
> Touches core simulation/narration paths (LLM call semantics, tick scheduling, companion progression, and DB write behavior), so regressions could affect runtime stability and data consistency despite changes being mostly additive and well-scoped.
> 
> **Overview**
> This PR **removes template-based narration as a runtime fallback** by refactoring `SLMNarrator` to call a single OpenClaw-backed model (`WORLDCLAW_DEFAULT_MODEL`, default `minimax/m2.5`) and updating tests to assert failures throw (with `fallbackNarration()` reduced to a sentinel string only). It also extends narrator prompting with campaign context/constraints and adds a new structured `gm_narrate()` path for full-scene JSON outputs.
> 
> It **changes living-world tick behavior and persistence**: adds `/world/tick` (all due campaigns) and `/world/tick/:campaignId` endpoints plus `/debug/world`; updates tick scheduling to include campaigns that have never been ticked; and significantly expands companion simulation to D&D 5e-style XP/level cap (1–20), loot drops with inventory caps, richer interaction/memory data, act-phase/dial progression, and lore-aware context injection.
> 
> It **normalizes companion autonomy and stats handling across APIs and storage**: defaults DB autonomy to `balanced`, migrates legacy `passive` rows to `balanced`, coerces/validates autonomy mode consistently, merges partial stat patches, caps level edits to 20, and hardens `companion_actions` persistence by handling ID collisions deterministically instead of `INSERT OR IGNORE`.
> 
> Docs and artifacts are updated to codify the “no fake narration” invariant and include a 50-tick campaign transcript; `package-lock.json` adjusts dependency classification (notably moving `@testing-library/dom` usage to dev contexts).
>
