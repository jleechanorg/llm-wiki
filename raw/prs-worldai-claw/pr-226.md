## Summary

- Wrap `factionBootstrapSchema.parse` in try-catch in `generateStructuredWorldBootstrap`
- On parse failure, fall back to `generateFactionsFromUniverseSeed` using the same bootstrap config
- Resolves 4 MCP + 12 HTTP test failures caused by Zod crash when LLM returns malformed JSON missing `factions` field

## Root cause

`faction_simulator.ts:224` called `factionBootstrapSchema.parse(result.json)` without error handling.
When the LLM returned a response missing the `factions` key, Zod threw `ZodError: expected 'array', received 'undefined', path: ['factions']`.
This crashed the world tick, which crashed `runSimulation`, which caused the HTTP streaming requests and MCP tool calls to timeout.
