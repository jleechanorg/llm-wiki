## Background

P2PSync module (`packages/backend/src/network/p2p_sync.ts`) and its app.ts wiring already existed as stubs from prior work, but lacked TDD test coverage for the integration layer. The mDNS module-not-found detection also had a bug where Jest's `ModuleNotFoundError` was not recognized, and a pre-existing test had an incorrect assertion about singleton behavior during shutdown.

## Goals

- Add comprehensive TDD tests for P2PSync integration with app.ts (ENABLE_P2P_SYNC env var, lifecycle hooks, error handling)
- Add unit tests for P2PSync class behavior (mDNS, CID dedup, block sync, peer management, shutdown)
- Fix mDNS MODULE_NOT_FOUND detection to handle all runtime environments
- Fix pre-existing broken test in p2pSync.test.ts

## Tenets

- TDD first — tests written before fixes
- Prefer real behavior verification over mocking
- Non-breaking — all existing functionality preserved

## High-level description of changes

Added 35 tests across 3 test files covering the P2P sync subsystem end-to-end. Fixed a bug in the mDNS optional dependency detection where the `MODULE_NOT_FOUND` check was too strict (only checked `err.code`, not `err.message`), causing the error to fall through to a generic error log instead of the graceful degradation path. Also fixed a pre-existing test that asserted `getP2PSync()` returns the old instance during shutdown — the actual (correct) behavior creates a fresh instance immediately so callers never see a draining singleton.

## Testing

- 35 tests pass across 3 suites:
  - `p2p_sync_app_integration.test.ts` (9 tests) — env var gating, lifecycle logging, graceful shutdown, error handling
  - `p2p_sync_class.test.ts` (24 tests) — init, mDNS, peers, chains, block sync, CID dedup, shutdown, singleton
  - `p2pSync.test.ts` (2 tests) — singleton reset behavior (fixed assertion)
- All run via `npx jest --config packages/backend/jest.config.cjs`
