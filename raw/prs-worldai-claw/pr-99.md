## Background

`chain_anchor.ts` `anchor()` previously had a bug where it could report success even when no RPC URL or contract address was configured (WC-vsr). The fix was applied in commit `0906f14`, but no tests existed to guard against regression.

## Goals

- Add comprehensive TDD regression tests verifying `anchor()` returns `success: false` with `reason: 'not_configured'` when env vars are missing
- Cover all partial configuration combinations (no vars, one var, two vars)
- Test delegation through `anchorHash()`, `verifyOnChain()`, `isHealthy()`, and `getConfigStatus()`

## Tenets

- Prefer real tests over mocks — all tests use the real `ChainAnchor` class
- Minimal scope — only adds the test file, no production code changes needed

## High-level description of changes

Added 13 tests in `chain_anchor.test.ts` covering:
1. **Missing config (5 tests)**: Every combination of missing BASE_RPC_URL, ANCHOR_CONTRACT_ADDRESS, ANCHOR_PRIVATE_KEY returns `success: false` with `reason: 'not_configured'`
2. **Full config (2 tests)**: Fully configured but no ethers.js RPC client returns `success: false`; missing caller address returns `success: false`
3. **Delegation (1 test)**: `anchorHash()` delegates to `anchor()` and propagates failure
4. **Health checks (2 tests)**: `isHealthy()` returns false/true based on config
5. **Config status (2 tests)**: `getConfigStatus()` reports partial config correctly
6. **Verify (1 test)**: `verifyOnChain()` returns `hasAnchor: false` when not configured

## Testing

- All 13 tests pass locally via `npx jest src/network/chain_anchor.test.ts`
- No production code changes — regression guard only

## Low-level details (1 file)

- **packages/backend/src/network/chain_anchor.test.ts**: New test file. Tests the `ChainAnchor` class directly with explicit constructor args (null/null/null for unconfigured, real strings for configured) to avoid env var pollution. Uses `resetChainAnchor()` in afterEach for singleton cleanup.
