## Background
worldai_claw-d25: Add level-up planning_block mandate to system prompt for companion progression.

When a companion gains XP that would trigger a level-up, the LLM must emit a `planning_block` before executing the level-up. This ensures the level-up is deliberate, narrative-justified, and covers ability/stat changes plus personality shifts.

## Goals
- Require `planning_block` emission before any companion level-up event
- Planning block must cover: abilities/stats changes, narrative justification, personality/mood shifts
- Choices: `companion_level_up_now` and `companion_defer_level_up`
- Stat changes applied via `state_delta`, never `scene_text`
- No world event or combat advancement while level-up modal is active
