## Background
The worldai_claw skeptic-gate.yml used `-d "{\"body\": $BODY}"` to post comments via `gh api`, where `$BODY` is a multi-line printf string. This produces invalid JSON, causing the 'Post skeptic trigger comment' step to fail with exit code 1 for all PRs (#170, #171, #172).

## Goals
- Fix Skeptic Gate CI check for wc#170, wc#171, wc#172
- Remove dependency on lifecycle-worker for skeptic evaluation

## High-level description
Replaces the trigger-poll approach with the deterministic 6-green check from agent-orchestrator. The new check:
- Runs gates 1-5 inline (CI, conflicts, CR, Cursor, inline comments)
- Posts VERDICT: PASS/FAIL comment directly
- Uses `--field` instead of `-d` for correct JSON handling
- Runs on ubuntu-latest

> [!NOTE]
> **Medium Risk**
> Changes the PR gating workflow and its pass/fail criteria, so misconfiguration could block merges or allow regressions. Scope is limited to a single GitHub Actions workflow with no product/runtime code changes.
> 
> **Overview**
> **Skeptic Gate is rewritten from a trigger+poll (AO worker/VERDICT comment) flow into a deterministic, single-job gate runner.** The workflow now runs on `pull_request` (plus `workflow_dispatch` inputs), uses `ubuntu-latest`, and avoids concurrency cancellation.
> 
> The job evaluates gates inline (CI check-runs green, mergeable/no conflicts, CodeRabbit approval, Cursor Bugbot check-run, and unresolved review threads via GraphQL), then deletes prior `skeptic-gate-result` comments and posts a fresh markdown table with `VERDICT: PASS/FAIL` using `gh api --field` (fixing prior JSON/body formatting issues). The workflow exits with the computed result to set
