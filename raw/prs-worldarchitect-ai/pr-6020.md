## Summary

Fixes error message exposure for production launch in mvp_site/.

### Changes Made

1. **main.py**:
   - Line 1894: Changed  to generic message "Invalid request parameters"
   - Line 3372: Changed  to generic message "Internal error processing request"

2. **mcp_client.py**:
   - Added  helper function that:
     - Filters sensitive patterns (file paths, credentials, database names, etc.)
     - Removes file paths and URLs from error messages
     - Truncates long messages
   - Updated  to sanitize all error messages returned to users

### Security Improvement

- Detailed errors (stack traces, file paths, database details) are logged server-side for debugging
- Users receive generic, user-friendly error messages only
- Prevents internal information leakage in production

### Testing

- Verify error responses don't expose internal details
- Check MCP endpoint returns sanitized JSON-RPC errors
- Verify get_story_paginated returns generic validation errors
