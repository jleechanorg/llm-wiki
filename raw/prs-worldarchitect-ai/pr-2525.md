## Summary
- Adds `arc_milestones` field to `custom_campaign_state` for structured tracking of narrative arc completion
- Implements immutable completion states - once an arc is "completed", it cannot regress to "in_progress"
- Provides `get_completed_arcs_summary()` for including deterministic state info in LLM context

## Problem Statement
LLM compressed memories capture narrative events as prose text, but there is no structured state tracking for arc completion. This causes the LLM to "forget" that major arcs (like "Wedding Tour") are complete, leading to timeline confusion where it tries to revisit completed events as if they were still in progress.

### Evidence from Campaign Analysis
From the "alexiel swtor" campaign (ID: tAE30bFvyfO0rUd9cgyv):
- Memory 418: Wedding ceremony completed
- Memory 419: 3-month time skip recorded  
- Memory 429: LLM confused, tried to "move party back to Corellia"
- Memory 430: User had to correct again

Root cause: No completed_events or arc_milestones tracking existed.

## Solution
New methods on GameState:
- mark_arc_completed(arc_name, phase, metadata) - Sets status with timestamp, immutable once set
- update_arc_progress(arc_name, phase, progress) - Tracks in-progress arcs
- is_arc_completed(arc_name) - Query completion status
- get_arc_phase(arc_name) - Get current phase
- get_completed_arcs_summary() - Formatted string for LLM context
