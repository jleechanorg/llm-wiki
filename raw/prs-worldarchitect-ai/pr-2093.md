## Summary
- Fixes Render Docker build failure caused by incorrect build context
- Adds render.yaml with explicit dockerContext: . (project root)
- **NEW: Fixes OOM kill by detecting RENDER=true env var for 1-worker default**

## Root Causes Fixed

### 1. Docker Build Context (render.yaml)
Commit 2d81897 (PR #2059) refactored infrastructure code from mvp_site/ to infrastructure/ package. The Dockerfile was updated to copy both directories, but Render was using mvp_site/ as the Docker build context instead of the project root.

Build errors:
- /mvp_site: not found
- /infrastructure: not found

### 2. OOM Kill (worker_config.py)
After Docker build succeeded, container was OOM killed during startup:
- Render sets RENDER=true in environment
- worker_config.py only checked for ENVIRONMENT=preview (GCP)
- On Render, 17 workers spawned (2x8+1) exceeding 512MB limit

Logs showed:
```
Workers: 17
==> Out of memory (used over 512Mi)
```

## Fixes Applied

1. **render.yaml**: `dockerContext: .` with `dockerfilePath: ./mvp_site/Dockerfile`
2. **worker_config.py**: Added RENDER=true detection to return 1 worker
3. **test_worker_config.py**: Added 2 tests for Render environment
