> Low risk: removes local developer Git hook scripts only, with no runtime/production code impact. Main risk is losing the automated pre-commit/push checks those hooks previously enforced for contributors.
> 
> **Overview**
> Removes the Husky-managed Git hook shims under `.husky/_/` (`post-checkout`, `post-merge`, `pre-commit`, `pre-push`, `prepare-commit-msg`) that delegated to `bd hooks run ...`.
> 
> This prevents Git operations from failing in environments where the installed `bd` binary does not support the expected `hooks` subcommand, at the cost of no longer running those hook-time automations.
