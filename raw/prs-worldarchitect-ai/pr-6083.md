## Summary
- **Root cause**: `WAHttpTest` extended `HttpTestBase` (generic base) which does NOT manage server lifecycle. All HTTP tests hardcoded `BASE_URL = "http://localhost:8086"` with no server starting, causing "connection refused" on every test run.
- **Fix**: `WAHttpTest.setup()` now starts a local gunicorn server before tests run; `teardown()` stops it after. `BASE_URL` resolved via `_resolve_http_base_url()`. External `TEST_SERVER_URL` env var respected for CI preview server.

## Summary

- **Root cause**: `WAHttpTest` extended `HttpTestBase` (generic base) which does NOT manage server lifecycle. All HTTP tests hardcoded `BASE_URL = "http://localhost:8086"` with no server starting, causing "connection refused" on every test run.
- **Fix**: `WAHttpTest.setup()` now starts a local gunicorn server before tests run; `teardown()` stops it after. `BASE_URL` resolved via `_resolve_http_base_url()`. External `TEST_SERVER_URL` env var respected for CI preview server.

## Changes (2 files)

- **`testing_http/lib/config.py`**:
  - `BASE_URL = _resolve_http_base_url()` — resolves from `TEST_PORT_HTTP` or `TEST_SERVER_URL`
  - `_WSServerState.process` singleton holds the gunicorn subprocess (atexit cleanup included)
  - `WAHttpTest.setup()` — starts gunicorn with `TESTING_AUTH_BYPASS=true` env, waits for health
  - `WAHttpTest.teardown()` — stops shared gunicorn after test class completes
  - `_WSServerState.reset()` (atexit) — calls `terminate()` then `kill()` before clearing state, preventing orphan processes on abnormal exit
- **`specs/exit-criteria.md`**: Documents exit criteria for this PR

Uses gunicorn (same as `MCPTestBase`) rather than Flask dev server to avoid Firebase import hangs.
