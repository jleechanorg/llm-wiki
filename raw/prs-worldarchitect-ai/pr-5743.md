## Raw Body
### Motivation
- Address outstanding reviewer comments about self-hosted CI reliability and runner bootstrap scripts that caused flaky venv creation, hardcoded cache keys, and unsafe runner installation behavior. 
- Reduce risk of opaque failures on self-hosted macOS/Linux runners by adding explicit interpreter guards and clearer service error surfacing. 

### Description
- Hardened interpreter discovery across workflows by resolving `python3.11 || python3 || python` with a fail‑fast guard and non‑1 exit when no interpreter is found, and added `--clear` when creating persistent venvs for Ruff and mypy in `.github/workflows/presubmit.yml`.
- Made test workflow cache keys and venv behavior version-aware by adding a `Resolve Python version for cache keys` step and using the runtime `major.minor` in cache keys, plus an interpreter guard before creating `venv` in `.github/workflows/test.yml`.
- Improved `scripts/setup-all-self-hosted-runners.sh` robustness by exporting/warning on `GITHUB_TOKEN`, replacing `exit 1` with `exit 2` for early usage errors, surfacing `systemctl enable` failures, and making `launchctl bootout` explicitly detect benign “not loaded” cases vs real errors.
- Fixed security/control-flow issues in `scripts/rename_runners.sh` by removing the committed registration token and requiring `RUNNER_REGISTRATION_TOKEN` from the environment, using non‑1 early exits, and skipping missing runner directories instead of aborting the whole run.
- Posted one consolidated PR response summarizing fixes and produced an audit artifact with comment URLs for traceability (local `/tmp/responses.json`).
