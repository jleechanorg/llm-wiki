## Problem
PRs like #2588 and #2590 weren't being re-processed because:
1. GitHub API returns some bots without `[bot]` suffix
2. Detection order was wrong - codex patterns excluded bots before `KNOWN_GITHUB_BOTS` check

## Solution
1. Added `KNOWN_GITHUB_BOTS` frozenset with known review bots
2. Reordered detection to check known bots FIRST
3. Added bot comment check in `_is_pr_actionable()` before returning False
