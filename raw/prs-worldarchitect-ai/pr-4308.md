## Raw Body
### Motivation
- Move inline `code_execution` prompt guidance into versioned prompt variants and ensure the loader extracts only the intended dice section. 
- Correct extraction bugs and unsafe message references that could cause silent misbehavior or runtime errors. 
- Improve test coverage to assert the loader expectations and to ensure default prompts do not contain code-execution snippets.

### Description
- Fixed the extraction regex in `mvp_site/llm_providers/gemini_provider.py` to use `\s*` so the `BEGIN_CODE_EXECUTION_DICE_SECTION`/`END_CODE_EXECUTION_DICE_SECTION` markers are correctly matched and the dice section is extracted. 
- Initialized `variant_path` and made the missing-file message safe in `mvp_site/prompt_loader.py` to avoid referencing an undefined variable when falling back to the default prompt. 
- Strengthened `mvp_site/tests/test_prompt_variants.py` by removing an unused import, adding `encoding="utf-8"` to file opens, asserting the default prompt excludes `random.seed(time.time_ns())`, and adding `test_code_execution_variant_has_dice_section_markers` to verify the presence of the marker comments. 
- Kept the existing silent-but-warn fallback when markers are absent and added explicit marker presence tests to catch that case during CI.

### Testing
- Ran the focused test file with `python -m pytest mvp_site/tests/test_prompt_variants.py`, which resulted in `15 passed, 1 skipped`. 
- Pre-commit format/lint checks ran as part of the local commit and passed (`ruff` fixes applied and files reformatted as needed). 
- No additional tests were added beyond the prompt-variant suite; a dedicated `FileNotFoundError` path unit test was considered but left as a follow-up to avoid scope creep.
