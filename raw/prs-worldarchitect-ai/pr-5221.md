## Raw Body
Problem
- Cloud Run dev returned 500 for GET /api/campaigns/<id>/stats when Firestore stored skills as a nested schema (e.g. {\"skills\": {\"proficiencies\": [...]}}).
- Backend crashed in stats_display.build_stats_summary with TypeError: unhashable type: \"list\".

Change
- Make skills parsing in mvp_site/stats_display.py flexible:
  - supports skills.proficiencies (list)
  - supports skills.expertise (list or dict)
  - preserves legacy {skill: true/\"expertise\"/2} behavior
  - ignores non-scalar dict values to avoid crashing

Tests
- python -m unittest -q mvp_site.tests.test_stats_display
