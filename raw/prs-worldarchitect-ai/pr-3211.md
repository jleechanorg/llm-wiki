Problem:
- Campaign creation initialized GameState with empty world_data={}
- Opening story's world_time was not applied to game state
- Without an in-world anchor, subsequent LLM calls could emit real-world ISO dates (e.g., 2026) instead of lore-consistent dates (1492 DR)

Solution (following LLM Architecture Principles - "LLM decides, server executes"):
1. Added prompt instruction in build_background_summary_instruction() telling the LLM to establish world_time with lore-consistent calendar on first turn
2. Server now merges LLM's state_updates.world_data into initial_game_state without modification - the LLM decides the timeline, we persist it

Key changes:
- agent_prompts.py: Added explicit instruction for LLM to include world_time using setting's calendar (e.g., 1492 DR for Forgotten Realms)
- world_logic.py: Simplified to merge LLM's world_data directly
- Tests updated to verify passthrough behavior

This ensures the first campaign entry has a lore-consistent world_time, preventing real-world ISO date drift in subsequent game sessions.
