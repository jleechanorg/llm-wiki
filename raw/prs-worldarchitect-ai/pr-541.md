## Summary
Adds a new `/header` command to improve compliance with the mandatory branch header protocol.

## Problem Solved
- **Header forgetting**: I frequently forget the 🚨 CRITICAL requirement to include branch headers
- **High friction**: Manually running 3 git commands creates cognitive load
- **Inconsistent formatting**: Manual header generation can have formatting errors

## Solution
**New `/header` Command:**
- Automatically runs all 3 required git commands
- Formats output in the correct header format
- Reduces friction from 3 commands to 1
- Creates a natural reminder checkpoint

## Changes
- **Added**: `.claude/commands/header.md` - Complete command specification  
- **Updated**: `CLAUDE.md` - Lists `/header` as PREFERRED method
- **Benefits**: Makes compliance easier, reduces forgetting, ensures consistency

## Usage
```
/header
```
Outputs: `[Local: branch  < /dev/null |  Remote: upstream | PR: #num url]`

## Why This Will Work
- **Reduces cognitive load**: 1 command vs 3
- **Creates habit**: Natural checkpoint before responding  
- **Removes friction**: Easy to execute = more likely to use
- **Consistent format**: Automated generation prevents errors

This addresses the #1 most violated rule in CLAUDE.md by making compliance easier.
