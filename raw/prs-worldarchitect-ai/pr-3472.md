## Summary
Adds detailed logging to capture malformed JSON content when parsing fails, helping debug 'Unterminated string' and other JSON parsing errors.

## Changes
- **narrative_response_schema.py**: Logs JSON context around error position (500 chars before/after) and full content when available
- **provider_utils.py**: Logs Phase 1 response context when JSON parsing fails

## Why
Currently when JSON parsing fails, we only see the error message but not the actual malformed JSON content. This makes it impossible to debug what the LLM is actually returning.

## Testing
- Logs will show actual JSON content on next parsing error
- Helps identify patterns in malformed responses
