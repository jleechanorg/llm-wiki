## Learning Summary

**Triggered by**: Git pull failure confusion - user asked "why didn't it work the first time and why did it ignore my comment?"

**Root Cause**: When `git pull origin main` failed due to diverging branches, I silently resolved it with `git merge --no-ff` without explanation, leading to user perception that their instruction was ignored.

## Solution

Added **Command Failure Transparency Protocol** to CLAUDE.md Git Workflow section:

### 🚨 **COMMAND FAILURE TRANSPARENCY**: ⚠️ MANDATORY
- ✅ Immediately explain what failed and why
- ✅ Show system messages/errors received  
- ✅ Explain resolution approach being taken
- ✅ Ask preference for alternatives (merge vs rebase, etc.)
- ❌ NEVER silently fix without explanation
- **Pattern**: Command fails → Explain → Show options → Get preference → Execute
