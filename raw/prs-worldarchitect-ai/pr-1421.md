## Summary
Implements content-aware CLAUDE.md generation that inspects actual directory contents instead of using generic templates. This addresses the critical user feedback: *"files still look duplicated. They should be generated by inspecting the actual file content of every directory"*

## Problem Solved
**Previous Approach Issues**:
- Template-based generation using generic module types
- Multiple directories with same type got identical protocols  
- No reference to actual files like CampaignCreation.tsx, Dashboard.tsx
- Functional duplication across directories

## Content-Aware Solution
**New Methodology**:
- **Directory Content Analysis**: Inspects actual files in each directory
- **Specific File References**: Lists actual components, tests, documents
- **Usage Pattern Documentation**: Real commands and workflows
- **Architecture Awareness**: Directory-specific development guidelines

## Proof of Concept (5 Files)
### 1. **Components Directory** (`mvp_site/frontend_v2/src/components/`)
- **Before**: Generic "Frontend Development" protocols
- **After**: Lists 18 actual React components (CampaignCreation.tsx, Dashboard.tsx, etc.)
- **Improvement**: Component-specific architecture guidelines, sub-module analysis

### 2. **Tests Directory** (`mvp_site/tests/`)  
- **Before**: Generic "Testing & QA" protocols
- **After**: Categorizes 120+ actual test files by functionality
- **Improvement**: Test infrastructure documentation, execution patterns

### 3. **PR Guidelines Directory** (`docs/pr-guidelines/`)
- **Before**: Generic "Documentation & Evidence" protocols
- **After**: References actual base-guidelines.md + 12 numbered PR directories
- **Improvement**: Process integration workflows, evidence-based guidelines

### 4. **Commands Directory** (`.claude/commands/`)
- **Before**: Generic "System Configuration" protocols  
- **After**: Analyzes 80+ command files with categorization
- **Improvement**: Command architecture explanation, usage patterns
