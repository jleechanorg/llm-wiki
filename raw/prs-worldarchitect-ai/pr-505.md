## Summary

Enhances command-line workflow with custom branch naming for `/integrate` and intelligent PR auto-detection for `/review` commands.

### Changes Made

#### 1. Custom Branch Name Support (/integrate)
- **integrate.sh**: Updated argument parsing to accept optional custom branch name
  - Added support for `/integrate [branch-name]` syntax
  - Maintains backward compatibility with existing `/integrate` usage
  - Supports combined usage: `/integrate newb --force`
  
- **.claude/commands/integrate.md**: Updated documentation with new usage examples

#### 2. Enhanced Review Command (/review)
- **.claude/commands/review.md**: Multiple enhancements for seamless workflow
  - **Auto-Detection**: Automatically detects and reviews PR for current branch
  - **Ultrathink Mode**: Uses `/think` mode by default for thorough analysis
  - **Branch Integration**: No need to remember or look up PR numbers
  - **Fallback Support**: Manual PR specification when needed

### Usage Examples

#### /integrate Command
- `/integrate` - Creates dev{timestamp} branch (unchanged behavior)
- `/integrate newb` - Creates "newb" branch
- `/integrate feature/my-feature` - Creates "feature/my-feature" branch  
- `/integrate fix/bug-123 --force` - Creates "fix/bug-123" branch with force mode

#### /review Command (New Auto-Detection)
- `/review` - **NEW**: Automatically reviews PR associated with current branch
- `/review [PR#]` - Reviews specific PR number (existing behavior)
- Automatically enables ultrathink mode for enhanced analysis

### Implementation Details

#### Auto-Detection Protocol
- Uses `git branch --show-current` to get current branch
- Uses `gh pr list --head $(current-branch)` to find associated PR
- Handles cases with no PR or multiple PRs gracefully
- Seamlessly integrates with branch-based development workflow
