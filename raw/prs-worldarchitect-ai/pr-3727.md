## Summary
- Fixed critical bug where god_mode_response content was being replaced by "You pause to consider your options..." placeholder
- Root cause: `create_from_structured_response` was not receiving the combined narrative text from `parse_structured_response`

## Problem Description
Scene #187 in campaign `qHCtkGaQdhoAeelmAP0f` showed the issue:
- User asked: "what does the game state say for my spell DC? Show me the exact fields/values"
- LLM calculated correctly: `{"calculation": {"base": 8, "proficiency": 4, "charisma_mod": 6, "equipment": 4, "total": 22}}`
- User saw: "You pause to consider your options..." (37 chars)
- Expected: The full explanation with spell DC = 22 (1005 chars)

## Root Cause Analysis
1. LLM returns `god_mode_response` (with actual answer) + empty `narrative` + `planning_block`
2. `_apply_planning_fallback()` replaces empty narrative with placeholder when planning_block exists
3. `parse_structured_response()` returns combined response (1005 chars) as first tuple element
4. `create_from_structured_response()` was NOT receiving this value - fell back to `structured_response.narrative` (the placeholder)
5. Actual god_mode_response content was discarded

## Fix
Pass `narrative_text` as `combined_narrative_text` parameter to all 4 call sites of `create_from_structured_response`:
- `_process_initial_story()` (2 locations - mock mode)  
- `_process_initial_story()` (1 location - real mode)
- `continue_story()` (1 location - main code path)
