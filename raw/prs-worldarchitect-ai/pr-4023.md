## Problem 1: macOS Bash Permission Dialogs

Even after PR #3987 removed explicit bash from tmux commands and changed bash -c to sh -c in timeout wrappers, macOS was STILL showing bash permission dialogs when creating new agent sessions.

Root cause analysis revealed:
- Dynamically generated scripts in /tmp/ had NO SHEBANG (first line was blank)
- When tmux executes a script without a shebang, macOS uses the default shell
- The default shell interpreter triggers bash permission dialogs
- This happened despite us not explicitly invoking bash anywhere

**Solution**: Added `#!/bin/sh` shebang as the first line of all dynamically generated agent run scripts. Now when tmux executes these scripts, the OS knows to use /bin/sh directly (no bash involved).

## Problem 2: Cursor Quota Exhaustion Misreported as Auth Error (CRITICAL)

Preflight validation in `orchestration/cli_validation.py` was passing when Cursor CLI quota was exhausted, allowing agents to fail with 275+ "Authentication required" errors that were actually quota exhaustion.

Root cause analysis revealed:
- Cursor API returns "authentication required" when quota is exhausted (NOT for real auth issues)
- Preflight validation's auth error handler (lines 262-276) returns `success=True` for OAuth CLIs
- This masked quota exhaustion errors, causing them to bypass validation
- User confirmed Cursor CLI is authenticated and working (`cursor-agent v2026.01.23-916f423`)

**Solution**: 
- Added Cursor-specific quota patterns to quota detection (lines 240-254)
- Quota detection now checks for "authentication required" and "please run agent login first" for Cursor CLI
- Quota detection runs BEFORE auth detection, preventing masking
- Preflight validation now fails fast when Cursor quota is exhausted
