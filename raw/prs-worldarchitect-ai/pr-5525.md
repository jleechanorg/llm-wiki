## Summary

Fix GitHub Actions heredoc delimiter syntax - remove single quotes around EOF that cause parsing failure.

## Root Cause Analysis

**The Real Issue**: GitHub Actions file command syntax doesn't support quoted delimiters.

**Error Message**:
```##[error]Invalid value. Matching delimiter not found ''EOF''
                                                  ^^^^^
                                      Notice double quotes wrapping 'EOF'
```

**Why PR #5521 Didn't Fix It**:
- PR #5521 added trailing newline to script output ✅ (correct fix for bash heredocs)  - But the workflow uses **GitHub Actions file commands**, not bash heredocs
- Different syntax requirements!

## The Difference

**Bash Heredoc** (what we thought we were fixing):
```bash
cat <<'EOF'    # Single quotes prevent variable expansion
content
EOF
```

**GitHub Actions Output** (what we actually have):
```bash
echo "body<<EOF" >> $GITHUB_OUTPUT   # ✅ Correct
echo "body<<'EOF'" >> $GITHUB_OUTPUT # ❌ Wrong - quotes become part of delimiter
```

## Fix

**File**: `.github/workflows/mcp-smoke-tests.yml:458`

```diff
-            echo "body<<'EOF'"
+            echo "body<<EOF"
```

Remove single quotes from EOF delimiter to match GitHub Actions syntax.

## Why This Was Confusing

1. PR #5520 fixed context API bug ✅ - workflow starts correctly  2. PR #5521 added trailing newline ✅ - correct for bash, but wrong target
3. Actual issue: GitHub Actions delimiter syntax (this PR) ✅

## Evidence

**Before Fix**:
- Workflow uses: `echo "body<<'EOF'"`  
- GitHub Actions looks for delimiter: `'EOF'` (with quotes)
- Script outputs: `EOF` (without quotes)
- Mismatch! "Matching delimiter not found ''EOF''"

**After Fix**:
- Workflow uses: `echo "body<<EOF"`
- GitHub Actions looks for delimiter: `EOF` (no quotes)
- Script outputs: `EOF` (no quotes)  
- Match! Heredoc parses correctly
