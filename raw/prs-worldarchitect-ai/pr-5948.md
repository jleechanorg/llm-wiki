## Production Code Changes
- **Before**: No auto-start; runners required manual `./start-runner.sh` after every Mac restart
- **Now**: `com.worldarchitect.oss-runners` launchd agent starts 2 runner containers at login, polling Docker until ready
- **Why**: Docker Desktop doesn't reliably auto-start containers after reboot; launchd ensures runners come back up without manual intervention

## Test Changes
None.

## Known Limitations
- Requires Docker Desktop to start within 150s of login (configurable in launchd-start.sh)
- Run `self-hosted-oss/install.sh` once on each Mac to activate
