## Summary
- Fixed Gemini CLI agent prompt handling that was causing fixpr automation to fail silently

## Root Cause
The Gemini CLI `-p/--prompt` flag is deprecated and only **appends** to stdin. With `stdin_template: "/dev/null"` (empty stdin), the prompt file path was passed as a literal string instead of file contents.

This caused Gemini agents to:
1. Load project context (CLAUDE.md)  
2. Print "I am ready for your first command"
3. Exit immediately with code 0 without doing any work

## Fix
Pass prompt via stdin instead of the deprecated `-p` flag:

```diff
- "command_template": f"{{binary}} -m {GEMINI_MODEL} --yolo -p {{prompt_file}}"
- "stdin_template": "/dev/null"
+ "command_template": f"{{binary}} -m {GEMINI_MODEL} --yolo"
+ "stdin_template": "{prompt_file}"
```
