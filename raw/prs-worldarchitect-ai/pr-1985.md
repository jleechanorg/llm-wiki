## Raw Body
Update the /redgreen slash command (aliased as /rg) to implement a more systematic test-first approach to debugging:

1. Search for existing tests that should catch the error
2. Run existing tests to confirm they reproduce the error
3. Update or create new tests only if needed (integration/functional/unit)
4. Confirm tests fail with the expected error
5. Implement minimal fix
6. Confirm tests pass as primary validation

Key changes:
- Phase 1 (RED): Added explicit steps to find and run existing tests before creating new ones
- Phase 3 (GREEN): Made test passing the primary validation metric
- Critical Rules: Updated to enforce test-first approach
- Example Workflow: Updated to show the complete test-first flow
- Documentation: Emphasized using existing tests and choosing appropriate test types based on judgment

This makes the debugging workflow more systematic and prevents duplicate test creation while ensuring proper test coverage.
