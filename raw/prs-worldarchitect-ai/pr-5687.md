## Why this PR is needed

This PR fixes reliability and observability regressions in the MCP test harness that currently affect merge safety:

- Real-API MCP tests were still susceptible to credential bootstrap failures when `gcloud` is not discoverable from PATH (common in CI/nohup/cron-like environments).
- Runtime memory enforcement in `run_tests.sh` could miss nested pytest worker processes, leading to inaccurate memory accounting and less deterministic OOM protection.
- Schema MCP evidence and fingerprints were inconsistent across runs, reducing our ability to compare behavior between branches and making regressions harder to diagnose.

## Changes in this PR (vs `origin/main`)

### 1) Harden secret loading / server test bootstrap
- `testing_mcp/lib/server_utils.py`
  - Added robust gcloud discovery via known installation fallback directories (`_GCLOUD_FALLBACK_DIRS`).
  - Use resolved `gcloud` binary path in subprocess calls.
  - Replace silent/implicit failure with explicit `logging.warning(...)` when gcloud is unavailable.

### 2) Make test memory monitoring cross-platform and process-tree aware
- `run_tests.sh`
  - Replaced Linux-only child-process query path with portable descendant tracking:
    - prefer `pgrep -P`
    - fallback to `ps -eo pid=,ppid=` filtering when needed.
  - Keeps tracked test PIDs and descendants in sync for more accurate total-memory enforcement and process cleanup.

### 3) Improve schema validation evidence and request fingerprinting
- `testing_mcp/lib/component_validation_test_base.py`
  - Ensures reusable request fingerprint capture helpers are available to schema enforcement journey tests.
- `testing_mcp/schema/test_schema_enforcement_journey_real_api.py`
  - Integrates fingerprint capture into turn execution and keeps per-turn + scenario evidence metadata.
- `testing_mcp/schema/test_schema_validation_extended.py`, `testing_mcp/schema/test_schema_validation_real_api.py`, `testing_mcp/schema/test_schema_validation_fallback.py`
