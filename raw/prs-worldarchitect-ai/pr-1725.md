## Summary

This PR implements directory-based CI testing that intelligently runs tests only for directories with file changes, significantly optimizing CI execution time and resource utilization.

### Key Features

- 🎯 **Directory-Based Test Filtering**: New `--test-dirs` flag in `run_tests.sh` supports targeted testing by directory
- 🔍 **Intelligent Change Detection**: GitHub Actions workflow analyzes git diff to determine which directories have changes
- ⚡ **Parallel Matrix Execution**: Tests run in parallel for different directory groups (mvp_site, .claude, orchestration, scripts, etc.)
- 🔄 **Backward Compatibility**: Maintains full compatibility with existing CI infrastructure
- 📊 **Performance Optimization**: Only runs relevant tests instead of full test suite

### Implementation Details

#### Enhanced run_tests.sh
- Added `--test-dirs` parameter with comma-separated directory support
- Intelligent test discovery scans specified directories for `test_*.py` files
- Example usage: `./run_tests.sh --test-dirs=mvp_site,.claude,orchestration`

#### GitHub Actions Workflow Restructuring
- **detect-changes job**: Analyzes file changes and maps to relevant directories
- **Matrix strategy**: Parallel execution for affected directory groups
- **Fallback mechanism**: Runs core tests if no specific changes detected
- **Comprehensive caching**: Optimized pip and venv caching per directory group

#### Test Validation Results
- ✅ **mvp_site**: 179 test files discovered and executed correctly
- ✅ **.claude**: 13 test files for command system validation  
- ✅ **orchestration**: 11 test files for orchestration system
- ✅ **Multi-directory**: Successfully handles comma-separated input (.claude,orchestration = 24 total tests)

### Performance Benefits

- **Faster CI execution**: Only runs tests for changed directories instead of full suite
- **Better resource utilization**: Parallel testing across directory groups
- **Scalable architecture**: Performance improves as codebase grow
