## Summary

Implement worktree location enhancement where `/orch` commands create worktrees under `~/projects/orch_{repo_name}/` by default instead of the current directory. This provides centralized worktree organization while maintaining full backward compatibility.

## Key Features

🎯 **Default Location**: `~/projects/orch_{repo_name}/{agent_name}` instead of `./{agent_name}`
🔍 **Repository Detection**: Automatic repo name extraction from SSH/HTTPS git remotes
🏗️ **Directory Management**: Automatic creation of parent directories as needed
🔒 **Security**: All subprocess calls use `shell=False` and `timeout=30`
⬆️ **Backward Compatible**: All existing workspace_config options work unchanged

## Implementation Details

### New TaskDispatcher Methods
- `_extract_repository_name()` - Extract repo name from git remote URLs with fallback
- `_get_worktree_base_path()` - Calculate `~/projects/orch_{repo_name}/` base path  
- `_calculate_agent_directory()` - Calculate final agent directory path
- `_expand_path()` - Expand ~ and resolve paths safely
- `_ensure_directory_exists()` - Create directories with proper error handling
- `_create_worktree_at_location()` - Create git worktree at calculated location

### Repository Name Extraction Examples
```bash
# SSH Format
git@github.com:user/worldarchitect.ai.git → "worldarchitect.ai"

# HTTPS Format  
https://github.com/user/my-project.git → "my-project"

# Local Repository
/path/to/local-repo → "local-repo"
```

### Default Behavior Change
```bash
# Old: Creates worktrees in current directory
./agent_workspace_task-agent-123/

# New: Creates worktrees under centralized location
~/projects/orch_worldarchitect.ai/task-agent-123/
```
