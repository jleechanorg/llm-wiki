## Specification Overview

### Core Concept
**Orchestration Manager** extends the existing `/orchestrate` command to coordinate multiple coding agents across different directories, worktrees, and repositories with a hierarchical manager-worker architecture.

### Key Features
1. **Multi-Directory Support**: Manager coordinates agent pairs (claude + codex) across unlimited directories
2. **Agent Specialization**: Claude agents handle coding/testing, Codex agents handle verification
3. **Slack Communication**: Real-time collaboration via #agent-orchestration channel with manager approval workflow
4. **A2A Protocol**: All agents support Agent-to-Agent protocol for machine communication
5. **MCP Server Wrapping**: Every agent can operate as an MCP server for external integrations
6. **Natural Language Goals**: Automatic directory detection from natural language specifications

### User Stories Covered
- **US1**: Multi-repository feature development (API + SDK + docs)
- **US2**: Git worktree parallel development  
- **US3**: Agent communication with manager approval
- **US4**: A2A protocol + MCP server support
