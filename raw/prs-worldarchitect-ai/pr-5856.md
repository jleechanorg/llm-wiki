## Summary

- **/reviewdeep + /reviewsuper**: Add confidence scoring (0-100 per issue, Haiku agents, filter <80) from the code-review plugin — eliminates false positives before posting GitHub PR comments
- **/copilot-expanded**: Add aspect-flag routing (comments, tests, errors, types, code, simplify, all) from pr-review-toolkit plugin — targeted reviews instead of full 4-phase workflow
- **/push**: Add allowed-tools frontmatter restriction + live context injection from commit-commands plugin — scopes tool access and provides immediate state context
- **New /feature-dev**: 7-phase guided feature development from feature-dev plugin, adapted for WorldArchitect.AI stack (Flask/Firebase/Gemini)
- **New /revise-claude-md**: Session-end CLAUDE.md update command from claude-md-management plugin

## Production Code Changes

- reviewdeep.md: Parallel agents posted all issues → Haiku agents score each 0-100, filter <80 → eliminates false positive noise
- reviewsuper.md: Jumped straight to analysis → Haiku eligibility pre-check + per-issue confidence scoring
- copilot-expanded.md: Full 4-phase bash always ran → routes to targeted aspect agents when flags given
- push.md: No tool restrictions → allowed-tools scopes git/gh + live context injected at invocation

## Test Changes

None — these are slash command prompt files (.claude/commands/*.md), not production code.

## Known Limitations

- revise-claude-md requires manual invocation at session end
- feature-dev architecture agents are generic; may need WorldArchitect.AI-specific tuning
- Confidence scoring adds Haiku agent overhead per review
