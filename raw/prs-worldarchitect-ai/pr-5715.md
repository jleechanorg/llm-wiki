> **Overview**
> Updates `TaskDispatcher._detect_pr_context` to recognize PR numbers in **slash-command style tasks** (e.g. `"/copilot 5691"`, `"/fixpr #123"`, `"/fix-comment 5678"`, including backticked `#1234`), so these are treated as *PR update* requests instead of creating a new PR.
> 
> Adds regression tests covering supported slash command variants, case-insensitivity, and a negative case ensuring inline documentation mentions like `"Document /copilot 123"` do *not* trigger update mode.
