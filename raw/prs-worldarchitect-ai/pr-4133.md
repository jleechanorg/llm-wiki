## Raw Body
### Motivation
- Ensure the Living World dual-trigger (turns OR in‑game hours) is reliable for fantasy calendars and does not silently disable time triggers. 
- Prevent runtime errors when Firestore contains malformed tracking fields and enforce repo import/structure policies.

### Description
- Implemented 30‑day/month calendar arithmetic in `world_time.calculate_hours_elapsed()` to avoid `datetime`/Gregorian validation failures and correctly compute elapsed hours for fantasy dates. 
- Centralized living‑world trigger logic so `PromptBuilder` delegates to `GameState.check_living_world_trigger()` and retains a stateless fallback when `game_state` is not present. 
- Added defensive coercion and warning logs for `player_turn`, `last_living_world_turn`, and `last_living_world_time` in `GameState.__init__` to handle malformed Firestore values without raising `TypeError`. 
- Moved inline imports to module scope and updated prompt files and documentation to reflect the new cadence (`LIVING_WORLD_TURN_INTERVAL = 10` and `LIVING_WORLD_TIME_INTERVAL = 24`), and integrated living‑world tests into existing test modules.

### Testing
- Ran unit tests for the affected areas: `python -m pytest mvp_site/tests/test_world_time.py`, `mvp_site/tests/test_game_state.py`, and `mvp_site/tests/test_prompts.py`, and the full test suite via `./run_tests.sh`; the test run covering the changes completed successfully (all tests passed). 
- Ran linters/pre‑commit checks (`ruff`/`pre-commit`) and addressed issues surfaced during the rollout; style checks were applied before final commits.
