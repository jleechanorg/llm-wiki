## Raw Body
The item validation system was causing false positives when the LLM narrative mentioned common words like "boots" in descriptive context, not when the player was actually trying to USE items they don't have. This created a cascade of problems including narrative overwrites and social_hp_challenge corruption.

Changes:
- Remove item validation TODO and character creation skip logic from llm_service.py
- Remove "Item not owned" instruction from narrative_system_instruction.md

The LLM should be trusted to handle item usage correctly via its system instructions. False positives cause worse UX than the occasional item exploit.
