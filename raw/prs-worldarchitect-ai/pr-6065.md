## Summary
Fixes avatar drag-to-reposition so it works in BOTH axes (horizontal AND vertical) instead of being locked to one axis based on aspect ratio.

## Root Cause
The drag bounds formula used `Math.min(0, ...)` as the upper bound for both axes:
```javascript
// Before (buggy)
offsetX = Math.min(0, Math.max(-(drawW - zoneSize), startOX + dx));
offsetY = Math.min(0, Math.max(-(drawH - zoneSize), startOY + dy));
```
This produced asymmetric constraints:
- **Portrait images** (taller than wide): `drawW == zoneSize` → horizontal range was `[0, 0]` → only vertical dragging worked
- **Landscape images** (wider than tall): `drawH == zoneSize` → vertical range was `[0, 0]` → only horizontal dragging worked

## Fix
`Math.max(0, ...)` as the upper bound and `Math.min(negative_min, ...)` as the lower bound for both axes:
```javascript
// After (fixed)
offsetX = Math.max(0, Math.min(-(drawW - zoneSize), startOX + dx));
offsetY = Math.max(0, Math.min(-(drawH - zoneSize), startOY + dy));
```
