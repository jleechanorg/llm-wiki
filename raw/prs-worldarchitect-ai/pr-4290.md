## Root Cause Analysis

The PyPI workflow was configured for OIDC trusted publishing, which requires manual PyPI web UI configuration. This was never set up, causing 100% failure rate since workflow creation on Jan 17, 2026.

The working `/automation-publish` command uses `twine upload` with PyPI API token. This PR aligns the GitHub workflow to use the same token-based authentication method.
