## 🚀 Action Resolution Consolidation & Guardrails

This PR consolidates the action resolution logic, enforces strict resource guardrails, and fixes critical validation bugs. It integrates the work from PR #3212 and adds comprehensive lifecycle testing.

### 🔑 Key Changes

#### 1. Schema Consolidation (`action_resolution`)
- **Migration:** Replaced the deprecated `outcome_resolution` field with the standardized `action_resolution` schema.
- **Backward Compatibility:** Updated `narrative_response_schema.py` and prompts to support the new format while maintaining legacy support (with warnings) during transition.

#### 2. Resource Guardrails (Merged from #3212)
- **Impossible Actions:** Integrated `test_llm_refuse_impossible_actions.py` to ensure LLMs refuse actions when resources are *already* exhausted (e.g., casting Fireball with 0 slots).
- **Consumption Lifecycle:** Added `test_resource_consumption_lifecycle.py` to validate the full cycle: **Available → Consumed → Depleted → Refused**.

#### 3. Critical Bug Fixes
- **Entity Validator Crash:** Fixed a `re.error` in `mvp_site/entity_validator.py` caused by unescaped parentheses in character names/descriptions.
- **Test Harness:** Fixed evidence logging in lifecycle tests to correctly capture nested resource states.

#### 4. Test Infrastructure
- **Multi-Model Support:** Updated `test_outcome_declaration_guardrails.py` and `test_resource_consumption_lifecycle.py` to support `--models` argument for validating against both Gemini and Qwen.
- **Config:** Added `WORLDAI_DEFAULT_GEMINI_MODEL` environment variable support.
