## Production Code Changes
**Before**: Installation would fail if ~/.token file didn't exist, even if tokens were in bashrc
**Now**: Installation detects environment tokens first, only requires ~/.token as fallback
**Why**: Users often have API tokens configured in bashrc/profile for multiple projects

## Benefits
- Eliminates token file requirement for most users
- Backward compatible with existing ~/.token workflows
- Clearer error messages and guidance
- Graceful degradation with warnings instead of hard failures
