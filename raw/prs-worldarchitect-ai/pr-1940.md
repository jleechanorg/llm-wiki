## Summary

Implements automatic deployment from `main` branch to Cloud Run development environment using GitHub Actions.

Based on proven implementation from [ai_universe backend PR #500](https://github.com/jleechanorg/ai_universe/pull/500).

## Changes

- **`.github/workflows/auto-deploy-dev.yml`** - Auto-deployment workflow
  - Triggers on push to `main` branch
  - Deploys `mvp_site` to Cloud Run (`mvp-site-app-dev`)
  - Includes health check validation (10 retries with 5s intervals)
  - Provides deployment summary with monitoring links
  - Uses SHA-pinned GitHub Actions for security
  
- **`docs/GITHUB_ACTIONS_AUTO_DEPLOY.md`** - Comprehensive documentation
  - Configuration details and prerequisites
  - Service account setup instructions
  - Troubleshooting guide
  - Branch mapping and environment details

## Configuration

- **GCP Project:** `ai-universe-2025`
- **Region:** `us-central1`
- **Service:** `mvp-site-app-dev`
- **Health Endpoint:** `/health`
- **Timeouts:** 45min job / 30min deploy step

## Prerequisites

Before merging, ensure the following are configured:

- [ ] **GCP_SA_KEY secret** configured in repository settings
- [ ] Service account created with required roles:
  - `roles/run.admin`
  - `roles/iam.serviceAccountUser`
  - `roles/storage.admin`
- [ ] Cloud Run, Cloud Build, and Artifact Registry APIs enabled in GCP project
- [ ] `/health` endpoint implemented and returns HTTP 200

## Testing Plan

After merge:
1. Monitor first auto-deployment from `main` branch
2. Verify health check passes
3. Check deployment summary appears in Actions UI
4. Verify service is accessible at Cloud Run URL
