## Update: Null Value Handling (latest)
- **social_hp_challenge validation fix**: When LLM outputs `{"npc_name": null}`, `str(dict.get("key", ""))` returns `"None"` (truthy) because `.get()` returns the explicit null, not the default. Changed to `str(x.get("key") or "")` pattern to properly treat null as missing.

## Update: DC Ordering Checks (post-review)
- **False positive fix**: DC ordering checks no longer flag AC-only attack rolls; enforcement now requires a `dc` field present in code execution output.
- **Per-roll signal**: added `dc_set_before_next_rng` to differentiate "DC after earlier RNG" from "DC after its own roll."
- **Warnings**: user-facing warnings now reflect the two-tier logic (global ordering vs per-roll ordering).
