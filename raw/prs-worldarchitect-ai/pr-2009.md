## Raw Body
Problem:
- Copilot documentation stated "ALL comments" need responses but didn't explicitly clarify this includes bot comments (CodeRabbit, GitHub Copilot, etc.)
- Led to misunderstanding where bot comments were incorrectly assumed to not need responses
- Example: Assistant said "only human comments need responses" and skipped all 5 bot comments on a PR

Solution:
- Added explicit clarification across all copilot command files that "ALL comments" includes BOTH bot and human comments
- Only exception: Comments starting with "[AI responder]" (our own responses)
- Made it crystal clear that bot code review comments are NOT optional
- Required 100% reply rate: every bot comment must be either FIXED or explained with "NOT DONE: [reason]"

Files Updated:
- copilot.md: Added 3 sections clarifying bot comment requirements
- commentfetch.md: Added explicit bot comment inclusion statement
- commentreply.md: Added bot comment requirement to purpose and success criteria
- commentcheck.md: Updated verification mandate and fail conditions for bot comments
- copilot-lite.md: Added bot comment clarification to coverage requirement

Impact:
- Eliminates ambiguity about whether bot comments need responses
- Prevents future incidents of skipping bot code review comments
- Ensures 100% comment coverage means human AND bot comments
- Maintains zero tolerance policy for unresponded comments
