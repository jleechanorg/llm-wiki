## Summary
Rename the `--lite-mode` flag to `--async` throughout the codebase to better reflect the actual behavior of launching agents asynchronously without a monitoring loop. This improves clarity and aligns terminology with common async/background execution patterns.

## Changes
- Renamed `--lite-mode` CLI argument to `--async` in `orchestrate_unified.py`
- Updated argument destination from `lite_mode` to `async_mode` in argparse configuration
- Updated all references in `pair_execute.py` to use `--async` flag when launching agents
- Updated all agent prompt files (claude, codex, cursor, gemini, minimax variants) to use `--async` in example commands
- Updated all log messages and comments to reference "async mode" instead of "lite mode"
- Updated test cases to reflect the new flag name and terminology

## Benefits
- **Clearer semantics**: "Async mode" better describes the actual behavior (asynchronous/background execution) compared to "lite mode" which is vague
- **Better discoverability**: Aligns with common async/background execution terminology that developers expect
- **Improved documentation**: Comments and log messages now accurately reflect what the flag does
- **Consistency**: All references throughout the codebase now use consistent terminology

## Testing

- Updated test expectations in `test_pair_monitor.py` to verify `--async` flag usage
- All existing tests continue to pass with the renamed flag
- No functional changes to the underlying behavior, only terminology updates
