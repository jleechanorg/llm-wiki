## Raw Body
### Motivation

- Prevent false positives where files like `component.deadbeef.js` were treated as already-hashed and skipped, causing stale assets to remain cached.  
- Keep hash-length and hashing behavior aligned with the shared constant to avoid maintenance drift between build-time and runtime logic.

### Description

- Update `scripts/cache_busting.py::is_already_hashed()` to compute the expected hash of the original file and only treat a file as already-hashed if the embedded hash matches that computed value.  
- Centralize and use the shared constant from `mvp_site/constants.py` (`CACHE_BUST_HASH_LENGTH`) and ensure `md5` is used with `usedforsecurity=False` (documented as cache-only).  
- Apply small code-quality fixes in the script (rename unused loop var, add `# noqa` on the large function, remove stray commented lines that triggered lints).  
- Preserve existing behavior of creating hashed copies and updating HTML references while making the skip-detection robust against filenames that incidentally match the hex pattern.
