## Production Code Changes

**orchestration/task_dispatcher.py**
- Before: `"env_unset": ["ANTHROPIC_API_KEY", "ANTHROPIC_AUTH_TOKEN"]`
- After: `"env_unset": ["ANTHROPIC_API_KEY", "ANTHROPIC_AUTH_TOKEN", "CLAUDECODE"]`
- Why: Preflight spawns `claude --model MiniMax-M2.5 -p @prompt`. When run inside a Claude Code session, child inherits CLAUDECODE=1 and exits with "Claude Code cannot be launched inside another Claude Code session." Both jobs --fix-comment and --fixpr with --cli-agent minimax silently skip all PRs as a result.

- test_minimax_binary_is_claude — binary is claude not a separate minimax binary
