## Summary

Enhanced the cerebras-coder agent to always perform a mandatory small test before any actual code generation, ensuring the /cerebras script works properly and preventing generation failures.

### 🎯 Key Changes

- **Mandatory Pre-Test**: Added required small test execution before any code generation
- **Test Output Saving**: All test results saved to `/tmp` with timestamps for debugging
- **Serious Warning System**: Agent exits immediately with serious warnings if test fails
- **Exclusive /cerebras Usage**: Ensures we use the script vs coding ourselves
- **Enhanced Workflow**: Updated 10-step workflow with pre-test as step 2
- **Improved Failure Handling**: Better error messages and validation

### 🔧 Technical Implementation

1. **Pre-Test Logic**: Runs simple "Hello World" test before any generation
2. **Output Persistence**: Test results saved to `/tmp/cerebras_validation_test_TIMESTAMP.log`
3. **Validation Chain**: API Key → Command Exists → Pre-Test → Actual Generation
4. **Success Marker Detection**: Validates `🚀🚀🚀 CEREBRAS GENERATED` appears in output
5. **Zero Tolerance**: Any failure = immediate exit with warning

### 📋 Updated Workflow

```
1. Validation Step: Check API keys
2. Pre-Test Step: MANDATORY small test with /cereب script  ← NEW
3. Exit on Test Failure: Serious warning if test fails     ← NEW  
4. Analyze: Understand request (only after test passes)
5. Prepare: Create detailed prompt
6. Execute: Run /cerebras command
7. Verify: Check success markers
8. Process: Present output with metrics
9. Create: Use Write tool with generated content
10. Document: Log usage
```
