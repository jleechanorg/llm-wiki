## Summary

Input validation utilities for production launch:

### Changes Made

1. **Created `input_validation.py`** - New validation module with:
   - `validate_campaign_id` - validates UUID or alphanumeric IDs (wired into get_campaign endpoint)
   - `validate_user_id` - validates user ID formats (introduced, not yet wired at endpoints)
   - `sanitize_string` / `sanitize_user_input` - remove null bytes, normalize unicode (no HTML escaping — see docstring); introduced for future use
   - `validate_request_size` - enforce configurable limit (default: **10MB**); introduced for future use
   - `validate_array_size` - enforce 1000 item limit; introduced for future use
   - `validate_export_format` - restrict to allowed formats; introduced for future use

2. **Added MAX_CONTENT_LENGTH** - Flask config to limit request body sizes (**10MB default** via `MAX_REQUEST_SIZE_BYTES` env var)

3. **Added campaign_id validation** - to get_campaign endpoint (only `validate_campaign_id` is wired in this PR; remaining validators are ready for future endpoint integration)

4. **Removed unused `base64` import** from `main.py` — confirmed unused after JWT decode fallback removal in PR #6075

### Security Improvements
- Campaign ID SQL injection / path traversal patterns rejected
- Large requests (>10MB) rejected at Flask layer
- Additional validators available for endpoint integration in follow-up PRs

## Background

This PR adds input validation utilities for the WorldArchitect.AI production environment. The changes introduce a new `input_validation.py` module with validators for campaign IDs, user IDs, request sizes, and array sizes. Only `validate_campaign_id` is wired into production endpoints in this PR; other validators are available for future integration.

## Goals

1. Introduce `input_validation.py` module with reusable validators
2. Wire `validate_campaign_id` into the get_campaign endpoint
3. Configure Flask MAX_CONTENT_LENGTH (10MB) to reject oversized requests
