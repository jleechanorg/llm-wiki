> [!NOTE]
> **Medium Risk**
> Changes how automation is scheduled and includes logic to disable existing OpenClaw cron jobs, so misconfiguration could stop jobs or cause duplicate/ missed runs despite the narrow target set and dry-run option.
> 
> **Overview**
> Adds a new `automation/install.sh` entrypoint to install five PR-automation jobs as **native schedulers** (launchd on macOS, systemd timers on Linux), including a bounded random start delay for `pr-monitor` and support for `--dry-run` plus platform override.
> 
> After successful native installation, the script attempts to **disable only the matching five OpenClaw cron jobs by exact name** (leaving Slack check-ins and `ai-orch-session-keepalive` untouched), and README docs are updated to describe the new install/validation flow and updated schedules/CLI args. A new bash regression test (`automation/tests/test_install_native_scheduler.sh`) validates generated launchd/systemd definitions and that exactly the intended OpenClaw jobs are disabled.
