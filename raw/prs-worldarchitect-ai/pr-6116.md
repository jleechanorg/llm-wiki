Fixed the issue in the evidence check gate logic. The jq pipeline now:

1. Filters to evidence check runs matching `^evidence([ -]?gate| review)$`
2. Maps each to a minimal object with `{name, completed_at, id, conclusion}`
3. Groups by `.name` 
4. Selects the run with highest `.id` (most recent) per group
5. Then counts total/latest from this deduplicated list

This ensures Gate 6 reflects only the most recent outcome for each evidence check on the current SHA.
