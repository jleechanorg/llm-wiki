## Production Code Changes

**`mvp_site/frontend_v1/js/campaign-wizard.js:1359-1367`**

- **Before**: `descriptionInput?.value || (isDragonKnight ? DEFAULT : '')` — whitespace is truthy, so `'   '` short-circuited to `'   '`.
- **Now**: trim-test the raw value before falling back; fall back to `DEFAULT_DRAGON_KNIGHT_DESCRIPTION` when the trimmed value is empty.
- **Why**: prod users with a stray space/newline/tab in the textarea were silently falling off the fast-path and paying 32s. PR #6137 plugged the SHA-mismatch hole; this plugs the input-mutation hole.
