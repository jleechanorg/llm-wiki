## Problem
Fixes bug where second god mode command would respond to first command instead of the second one.

## Root Cause
The caching logic cached `story_context` when `campaign_data` was already loaded, causing the second command to use stale context that didn't include the first command's entry.

## Solution
- Remove all caching logic for `campaign_data` and `story_context`
- Always reload from Firestore for each request to ensure fresh data
- Add comprehensive end2end test for sequential god mode commands

## Rationale for Removing Caching
- `MAX_TEMPORAL_CORRECTION_ATTEMPTS = 0` (loop only runs once per request)
- Campaign data is just one document read (cheap operation)
- Story context must be fresh (includes latest entries)
- Caching adds complexity and risk of stale data bugs
- No performance benefit since temporal correction is disabled
