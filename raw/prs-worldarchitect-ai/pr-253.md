## Summary

Adds explicit pre-generation checks to prevent the AI from suggesting banned names during character creation.

## Problem

The AI DM suggested "Lysander" as a character name despite it being on the banned names list. The banned name check was only happening after names were chosen, not before generation.

## Solution

### 1. Added MANDATORY PRE-GENERATION CHECK
- Updated `master_directive.md` (v1.5) with 5-point mandatory check
- Must check CRITICAL NAMING RESTRICTIONS BEFORE suggesting any character name
- Applies to ALL characters throughout the campaign (PCs, NPCs, everyone)

### 2. Clear Section References with Consistent Terminology
- Unified all references to use "CRITICAL NAMING RESTRICTIONS" terminology
- World loader labels section as "CRITICAL NAMING RESTRICTIONS (from banned_names.md)"
- Instructions consistently reference this section name throughout
- Created complete terminology alignment across the entire system

### 3. Clean Structure
- Fixed duplicate headers in banned_names.md (consolidated to single header)
- Removed redundant numbered rules in master_directive.md
- Added clarification that banned names are to AVOID, not use

### 4. Behavior-Based Testing
- Added rule #18 to rules.mdc about Unit Test Content Independence
- Refactored tests to check behavior instead of exact content strings
- Tests now survive content updates and reformatting

## Testing

- Created `test_banned_name_prevention_v2.py` - tests behavior, not exact strings
- Created `test_banned_names_visibility_v2.py` - tests structure, not wording
- All tests pass ✅
