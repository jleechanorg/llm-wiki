## Test PR: Verify Duplicate Comment Fix

This PR verifies that the duplicate comment fix from PR #5535 works correctly.

### What We're Testing

After this PR is deployed, the `workflow_run` event should trigger and post **exactly ONE comment** with test results - not duplicates.

### The Fix Being Tested

PR #5535 changed `.github/workflows/mcp-smoke-tests.yml:196`:
- **Before**: `try-self-hosted` used `always()` → posted comment even on failure
- **After**: `try-self-hosted` uses `success()` → only posts on success
- **Result**: When fallback runs, only the fallback job posts a comment

### Expected Behavior

1. ✅ Deployment completes successfully
2. ✅ `workflow_run` event triggers MCP smoke tests
3. ✅ **Only ONE comment** is posted to this PR
4. ✅ Comment includes test results and SHA

### Success Criteria

- **ONE comment only** (no duplicates) ✓
- Comment posted within 1 minute of deployment ✓
- Comment shows correct tested SHA ✓
- Comment includes per-scenario test details ✓
