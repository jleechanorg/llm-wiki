## Problem

Each `pending_review_monitor` script was created with a unique temp file path using `tempfile.mkstemp()` (e.g., `/tmp/tmpXXXXX_pending_review_monitor_3015.sh`). 

macOS treats each unique script path as a separate "application" for permission purposes. Even though bash already had permission, when bash executed a NEW script file (with a new unique path), macOS saw it as a new "app" and asked for permission again.

With 13+ scripts running simultaneously, each checking every 5 seconds, this caused constant permission prompts.

## Solution

Changed from unique temp files to a **fixed script path per PR**: `/tmp/pending_review_monitor_{pr_number}.sh`

- macOS now recognizes it as the same app and doesn't repeatedly ask for permission
- Script still cleans up after itself, but uses a predictable path
- Removes existing script if it exists from a previous run before creating new one

## Changes

- Modified `_start_pending_review_monitor()` in `orchestrated_pr_runner.py`
- Changed from `tempfile.mkstemp()` to fixed path construction
- Changed from `os.fdopen(script_fd)` to `script_path_obj.open()`
