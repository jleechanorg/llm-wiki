## Summary
Adds a comment tracking table to PR descriptions that categorizes all processed comments (fixed, deferred, ignored, unresolved) with their resolution reasons. This provides visibility into how the copilot handled each piece of feedback.

## Changes
- Added `build_tracking_section()` function to generate a markdown table of all PR comments categorized by response type (FIXED, DEFERRED, ACKNOWLEDGED, SKIPPED, NOT_DONE, ALREADY_IMPLEMENTED)
- Added `update_pr_description_with_tracking()` function to append or idempotently replace the tracking section in the PR description using HTML comment markers
- Added `_get_comment_url()` helper to build GitHub URLs for different comment types (review, inline, copilot, issue)
- Added `_get_comment_author()` helper to extract author from comment objects
- Added `_derive_tracking_reason()` function to generate tracking reasons from response fields when `tracking_reason` is missing
- Added `TRACKING_STATUS_MAP` to map response types to status labels and categories
- Integrated tracking update into main workflow as Step 6b, after posting consolidated summary comment
- Updated `_copilot_reference.md` with `tracking_reason` field documentation and example tracking table
- Updated `copilot-lite.md` and `copilot.md` with `tracking_reason` requirements and Phase 4.5 tracking documentation

## Benefits
- Provides clear accountability for how each comment was handled
- Helps reviewers understand which feedback was addressed, deferred, or intentionally skipped
- Tracks coverage metrics (% of comments processed vs unresolved)
- Idempotent updates allow safe re-runs without duplicating tracking sections
- Mandatory `tracking_reason` field ensures decisions are documented with context

- No automated tests needed; functionality is straightforward data transformation and GitHub API calls
