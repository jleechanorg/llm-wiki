## Summary
- Fixed critical bug in coverage.sh causing 0% coverage reports
- Coverage script was incorrectly using vpython wrapper in coverage run command
- All tests were failing during coverage analysis but passing in normal runs

## Root Cause
The coverage script was running:
```bash
coverage run --source=. "$VPYTHON" "$test_file"
```

This treated the vpython wrapper script as a Python module, causing all tests to fail.

## Solution
Changed to run test files directly:
```bash
coverage run --source=. "$test_file"
```

## Results
- **Before**: 0% coverage (all 94 tests failing in coverage mode)
- **After**: 67% overall coverage, 72% main.py coverage (all 94 tests passing)
