## Problem
The planning block enforcement was limiting context to only the last 500 characters of the narrative, which caused the LLM to lose track of the actual scene and generate irrelevant planning options (e.g., "dragon offer" in an Astarion/Baldur's Gate campaign).

## Solution
1. **Full Context**: Now provides complete narrative context to the LLM when generating planning blocks
2. **Simplified Prompts**: References `think_planning_block` and `standard_choice_block` template IDs from system instructions instead of redefining the entire structure
3. **Maintained Functionality**: All existing tests pass, ensuring backward compatibility

## Changes
- Modified `_validate_and_enforce_planning_block()` in `gemini_service.py`:
  - Changed `{response_text[-500:]}` to `{response_text}` for full context
  - Replaced inline template definitions with references to system instruction templates
- Added comprehensive test suite `test_planning_block_simplified.py`

## Benefits
- Fixes LLM hallucinations caused by insufficient context
- Reduces prompt size and complexity
- Maintains single source of truth for planning block formats
- Improves maintainability
