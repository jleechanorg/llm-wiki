## Summary

This PR implements **adaptive context truncation** for the LLM service, ensuring prompts fit within model context limits while preserving narrative quality. It also fixes critical Gemini API compatibility issues and enforces a strict "no pre-existing issues" testing policy.

## Key Changes

### 1. Adaptive Context Truncation System
- **Percentage-based allocation**: 25% for start context, 70% for end context
- **Middle compaction**: Preserves key events (combat, discoveries, decisions) from dropped turns
- **Budget guarantee**: Truncated content always fits within token limits
- **Robust sentence splitting**: Handles edge cases in narrative text

### 2. Gemini API Fix
- **Removed `code_execution` tool**: Incompatible with `response_mime_type` (JSON mode)
- Gemini API rejects requests combining controlled generation with code execution
- Updated tests to verify JSON mode works without code_execution

### 3. Test Fixes (4 failures → 0)
- Fixed model selection tests: Patch `_select_provider_and_model` directly
- Fixed validation tests: Updated assertions to match new error message format
