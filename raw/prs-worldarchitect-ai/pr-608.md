## Summary

Fixes the malformed planning block error reported by user and adds comprehensive debugging capabilities to prevent future silent failures.

### 🐛 Issues Fixed

1. **AttributeError in validate_checkpoint_consistency**: Fixed `'dict' object has no attribute 'lower'` when location is a dict
2. **Malformed planning block fallback**: Fixed fallback content to use JSON format expected by frontend
3. **Silent planning block regeneration failures**: Added detailed logging to debug why regeneration returned empty content

### 🔍 Root Cause Analysis

Through log analysis discovered:
- Original AI response had malformed JSON that was successfully parsed
- Planning block field was missing from structured response  
- System attempted regeneration but silently failed
- No visibility into what API actually returned during regeneration

**Defensive Code Fixes:**
- Handle dict location objects in game state validation
- Improved character creation detection for truncated responses
- Convert all fallback planning blocks to JSON format

**Comprehensive Debug Logging:**
- `_log_api_response_safely()` - Safe API response content logging
- Raw response logging in planning block generation pipeline
- Detailed validation failure logging with exact content
- Exception handling with full traceback
- Malformed JSON content logging in RobustJSONParser

**Benefits:**
- Next time this happens, we'll know exactly what the API returned
- Clear visibility into where parsing/validation failed
- Data-driven debugging instead of guesswork
- Eliminates silent failures
