## Summary

- **Custom action button**: Always appended at the bottom of every planning block choices list; clicking focuses the text input instead of submitting a preset choice
- **Scroll on choice click**: `scrollToBottom` called (via form submit handler at 30ms) when a planning block choice is selected, so the streaming response is immediately visible
- **New campaign scroll**: On campaign creation, scrolls to the first AI narrative entry (`entries[1]` / Scene #1, skipping the god-mode setup entry) using a container-scoped `getBoundingClientRect`-based offset rather than `scrollIntoView`
- **Streaming scroll guard**: Auto-scroll during streaming only fires when the user is already within 100px of the bottom, so users can scroll up to re-read without being forcibly snapped back
- **`_isNewCampaign` flag safety**: Global flag captured into a local variable and cleared before the `try` block; retries thread the value via parameter so the flag cannot leak across campaign loads
- **`__custom_action__` reserved ID**: Uses double-underscore prefix to avoid collisions with model-provided `CHOICE_SCHEMA` choice IDs
- **`scrollToBottom` uses `instant`**: Bypasses CSS `scroll-behavior: smooth` so rapid streaming chunks don't lag behind the bottom
- **E2E browser test**: `testing_ui/test_scroll_behavior.py` validates resume-scroll, new-campaign-scroll, custom-action button visibility, choice-click scroll, streaming auto-scroll, and form-submit scroll

## Production Code Changes

| Before | Now | Why |
|--------|-----|-----|
| No custom action option | `__custom_action__` button always appended at end of choices | Users want to type their own decision without submitting a preset |
| No scroll on choice click | Form submit handler scrolls to bottom at 30ms | User needs to see streaming response appear |
| `element.scrollTop = element.scrollHeight` | `element.scrollTo({ top: scrollHeight, behavior: 'instant' })` | Bypass smooth-scroll CSS lag during streaming |
