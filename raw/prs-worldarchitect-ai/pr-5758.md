## Production Code Changes

**`orchestration/task_dispatcher.py`**
- Before: All agents launched via tmux only
- Now: `no_tmux` parameter (default `False`) enables `subprocess.Popen` launch path for CI/containers
- Why: Phase 1 of asyncio orchestration migration (REV-xi6). tmux stays default; subprocess is opt-in for environments without tmux

- Verifier generates both left contract (coder spec) and right contract (verification criteria)
- Rich narrative retry feedback — conversational report mapping contract clauses to findings
- LLM-resilient callbacks — every step has the LLM decide, never crash on missing files
