## Relates to PR #1526
This PR complements PR #1526 by fixing the error detection issues in the backup script. Both PRs improve the backup functionality but address different aspects:
- PR #1526: Fixes backup content (includes .claude.json files)  
- This PR: Fixes backup error detection and reporting

## Problem  
- Backup script used `>/dev/null 2>&1` which completely suppressed all rsync errors
- Individual file failures (e.g., files with `com.apple.provenance` extended attributes) were silently ignored
- Script reported SUCCESS even when files failed to backup
- Root cause of missing recent conversation files from 11:30 PM was silent failures

## Solution
- **Removed blanket error suppression** (`>/dev/null 2>&1`)
- **Added proper rsync error logging** with separate stdout/stderr capture  
- **Implemented exit code handling**: 0=success, 23=partial transfer, other=failure
- **Added extended attributes support** (`--extended-attributes`) for macOS files
- **Made compatible with macOS rsync** (removed unsupported `--log-file` option)
- **Added detailed error reporting** and file-specific failure detection
