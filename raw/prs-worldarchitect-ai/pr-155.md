- Resolves timeout issues occurring in production environment

## Problem
The application was experiencing timeouts because:
- Gunicorn web server is configured with 300-second timeout
- Cloud Run was using default 60-second timeout
- Requests were being terminated prematurely after 60 seconds

## Solution
Updated `deploy.sh` to explicitly set Cloud Run timeout to 300 seconds, matching the Gunicorn configuration.

## Testing
To verify this fix:
1. Deploy using `./deploy.sh`
2. Test long-running AI generation requests
3. Confirm requests no longer timeout after 60 seconds
