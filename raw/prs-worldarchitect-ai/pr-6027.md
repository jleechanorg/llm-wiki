## Background
The /claw command dispatches tasks to OpenClaw via nohup openclaw agent. However, nohup subprocesses do not inherit shell env vars defined in ~/.bashrc, causing openclaw plugins (e.g. openclaw-mem0) to fail with errors like GROQ_API_KEY is not set.

Two prior commits on feature branches (b958515255 and dc906cd5a4) attempted fixes but were never merged to main.

## Goals
- Ensure all env vars from ~/.bashrc are available to the openclaw agent subprocess
- Merge the fix that has been sitting on feature branches

## Tenets
- Minimal change to claw.md
- Use bash -ic (interactive shell) so .bashrc guards do not skip exports

## High-level description of changes
Wrap the nohup dispatch in bash -ic instead of calling openclaw directly. Export CLAW_MSG so the task description is available inside the subshell (since ARGUMENTS from the parent context will not be in scope).

## Testing
- Manual verification: bash -ic sources .bashrc fully including API key exports
- Confirmed env inside bash -ic includes GROQ_API_KEY and other user-defined vars

## Low-level details
- .claude/commands/claw.md: Changed nohup openclaw agent to nohup bash -ic openclaw agent. Added export CLAW_MSG before dispatch. Changed -m ARGUMENTS to -m CLAW_MSG.
