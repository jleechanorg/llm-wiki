## Summary
- route native launchd/systemd PR automation jobs through an `mctrl`-native wrapper instead of the old Mission Control naming
- keep wrapper `--exec` mode so scheduled jobs preserve exact `jleechanorg-pr-monitor` commands while emitting catch-up lifecycle metadata
- namespace per-job evidence/health files under `automation/evidence/mctrl` and mark native runs as `trigger_source=catch_up`

## Changes
- `install.sh` now renders native launchd/systemd units that invoke `openclaw_mctrl_entry.sh`
- the wrapper prefers `MCTRL_*` env/config names, keeps legacy `MISSION_CONTROL_*` variables only as compatibility fallbacks, and continues to emit `workflow_lane`, `trigger_source`, and idempotency metadata
- shell tests were updated to cover the renamed wrapper and the `mctrl` env contract

## Testing
- `bash automation/tests/test_install_native_scheduler.sh`
- `bash automation/tests/test_openclaw_mctrl_entry.sh`

## Known Limitations
- this PR updates naming and scheduler handoff to align with the `mctrl` architecture, but it still stops at wrapper execution and metadata emission rather than calling a full downstream `mctrl` dispatch API
- broader historical Mission Control references elsewhere in the repo are out of scope for this PR
