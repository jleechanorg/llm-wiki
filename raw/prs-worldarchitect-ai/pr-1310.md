## 🧠 Intelligent PR Splitting Command

### Summary
Adds comprehensive `/split` command for analyzing PRs and recommending smart splitting strategies based on file dependencies, commit domains, and risk assessment.

### 🎯 Key Features

#### Smart Analysis Framework
- **Pre-Analysis Checks**: Size thresholds, coherence checks, risk assessment
- **Domain Classification**: Security, Infrastructure, Frontend, Backend, Testing, Documentation, Foundation
- **File Dependency Analysis**: Detects overlapping files between commits that create splitting constraints
- **Decision Matrix**: Clear criteria for when to split vs when to keep combined

#### Multiple Splitting Strategies
1. **Foundation-First**: For dependent files with overlapping changes
2. **Domain Separation**: For independent changes across different areas
3. **Risk-Based Separation**: High/Medium/Low risk isolation

#### Built-in Safeguards
- **File Count Verification**: Ensures all split PRs account for every file in original
- **Dependency Order**: Maps merge sequence for dependent PRs
- **Smart Recommendations**: Context-aware advice on when NOT to split

### 🔧 Implementation Workflow

#### Phase 1: Analysis
- Check if current branch has upstream PR
- Analyze commit history and file changes
- Detect file overlaps and dependencies
- Classify commits by domain and risk

#### Phase 2: Planning
- Create scratchpad file in `roadmap/scratchpad_[branch].md`
- Document file count verification (splits must sum to original)
- Provide implementation commands for chosen strategy
- Include dependency order and merge sequence

#### Phase 3: Execution (Optional)
- Prompt user if they want to execute the split
- Create branches according to strategy
- Cherry-pick appropriate commits to each branch
- Push branches and create PRs with proper base branches
