## Raw Body
### Background
Preview environments were leaking storage cost in two places: the rotating preview pool kept stale image digests in `gcr.io`, and legacy one-service-per-PR previews (`mvp-site-app-pr-*`) were left behind outside the pool cleanup path. Separately, MCP smoke traffic was still capable of reaching paid Gemini preflight paths when mock-services mode only short-circuited `llm_service` branches. The earlier cleanup work was tracked on https://github.com/jleechanorg/worldarchitect.ai/pull/6140, but that PR is now closed without merge, so this branch needs a fresh landing path.

### Goals
1. Eliminate residual preview artifact cost from both rotating-pool images and legacy PR preview services.
2. Make mock-services mode fail closed at the provider boundary so Gemini client/token/cache paths stay local.
3. Keep private-repo CI workflows on self-hosted runners and preserve the preview/deploy automation needed to verify the cleanup.

### Tenets
- Prefer fail-closed cost controls over best-effort cleanup.
- Keep `llm_service` orchestration real where practical; mock the provider boundary instead of duplicating the whole orchestration layer.
- Use self-hosted runners by default for private-repo workflows.
- Add automation for recurring cleanup instead of relying on one-off operator sweeps.

### High-level description of changes
This branch finishes the preview artifact cleanup by deleting stale preview digests on PR close, adding a daily preview-image janitor, and adding a separate janitor for legacy `mvp-site-app-pr-*` Cloud Run previews plus their `gcr.io` images. It also hardens mock-services mode by routing Gemini through a provider gateway mock client so token counting and cache activity stay local even if orchestration paths evolve. Along the way, the branch keeps the affected private-repo workflows on self-hosted runners and fixes the preview cleanup scripts so they are portable and label-safe.
