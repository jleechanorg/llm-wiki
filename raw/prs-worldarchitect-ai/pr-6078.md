> [!NOTE]
> **Low Risk**
> Low risk workflow-hardening change that only affects how the cron job fetches open PRs; main risk is altering behavior when the PR list is empty or GitHub API returns unexpected output.
> 
> **Overview**
> Hardens the `skeptic-cron` workflow’s “List open PRs” step against transient `gh api --paginate` failures by preventing the step from failing on non-zero exit codes.
> 
> Adds JSON validation of the `gh` output and a one-time fallback retry without pagination when the paginated response is empty or not a valid non-empty array, so the cron run can continue processing PRs.
