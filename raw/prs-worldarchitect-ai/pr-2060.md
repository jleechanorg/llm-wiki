## Raw Body
Replace per-PR service creation with a rotating pool of 10 pre-allocated Cloud Run services (s1-s10) to enable faster PR previews and better resource management.

Key improvements:
- Fast deployments: Reuse existing services instead of creating new ones
- Predictable URLs: Services s1-s10 always exist with stable URLs
- Automatic cleanup: Server labels removed on PR close, servers reused
- LRU eviction: When pool is full, oldest PR is evicted automatically
- Stateless: Cloud Run labels are the source of truth (no state files)

Components:
- pr-server-pool.sh: Bash script for server assignment/release/status
- pr-preview.yml: Updated workflow using server pool for PR deployments
- pr-cleanup.yml: New workflow to release servers when PRs close
- ROTATING_PREVIEW_SERVERS.md: Comprehensive documentation

This follows the exact methodology from the GCP preview server rotation guide and replaces the old per-PR service creation approach.
