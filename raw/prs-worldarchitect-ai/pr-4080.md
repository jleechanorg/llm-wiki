## Summary
- Fixes campaign list API latency from ~1.2s to ~0.1s by using the correct Firestore count aggregation API
- Removes deprecated AggregationQuery class usage that was causing errors

## Problem
PR #3480 (merged 2026-01-12) introduced a count aggregation pattern that fails with Firestore SDK 2.16.0:

The old broken pattern fails with: `'CollectionReference' object has no attribute '_parent'`

This caused fallback to streaming ALL documents, resulting in ~1.2s latency.

## Fix
Use the modern Firestore count API (SDK 2.11+) which calls `collection_ref.count(alias="total")` directly.

- Fixed 3 count aggregation call sites in firestore_service.py
- Removed unused AggregationQuery import
