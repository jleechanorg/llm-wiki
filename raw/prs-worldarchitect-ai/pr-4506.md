## Root Cause Analysis

### What Made It Worse: PR #4334 Regression

**PR #4334** (merged 2026-02-01, commit ae9bfb93f) was titled "fix(dice): replace fragile DC ordering checks with schema-based validation". This PR added `dice_roll_strategy` parameter propagation to ALL agents for consistency.

**The problem:** The change passed `dice_roll_strategy` to `GodModeAgent.build_system_instructions()` when it should have been `None`.

**Code path that caused the regression:**

1. `llm_service.py` line ~4315 called:
   ```python
   system_instruction_final = agent.build_system_instructions(
       dice_roll_strategy=dice_roll_strategy  # BUG: Should be None for god mode!
   )
   ```

2. `GodModeAgent.build_system_instructions()` inherited from `BaseGameAgent` which calls `build_from_order()`

3. `build_from_order()` checks `if dice_roll_strategy is not None` and loads dice instruction files:
   ```python
   if dice_roll_strategy is not None:
       # Load dice rolling system instruction
       dice_instruction_path = prompts_dir / "dice_rolling_system_instruction.md"
       ...
   ```

4. **Result:** God mode responses included dice rolling instructions, causing the LLM to roll dice and advance narrative when it should be frozen.

### Why It Worked Before

Before PR #4334, god mode either:
- Did not call `build_system_instructions()` with dice_roll_strategy
- Or the parameter was not propagated to god mode agent

The "fix" in PR #4334 inadvertently made all agents consistent, including GodModeAgent which should NEVER have dice instructions.
