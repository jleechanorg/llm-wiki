## Raw Body
## Problem

Even after removing the background monitor script, macOS was still showing permission prompts "once per run" because agents were executing `gh api` commands via bash, which triggers macOS permission dialogs.

## Solution

Replace all bash-based GitHub operations with Python functions that use the GitHub REST API directly via the `requests` library.

**Changes:**
- ✅ Added `post_pr_comment_python()` - Posts PR comments via Python GitHub REST API
- ✅ Added `cleanup_pending_reviews_python()` - Cleans up pending reviews via Python API  
- ✅ Added `get_github_token()` - Gets token from environment or gh CLI config
- ✅ Removed call to `_start_pending_review_monitor()` - No more background bash scripts
- ✅ Updated agent instructions to prefer Python functions over bash commands

**Benefits:**
- 🚫 **No bash scripts** → No macOS permission prompts
- ⚡ **Direct API calls** → Faster and more reliable  
- 🐍 **Python-native** → Easier for agents to use
- ✅ **Same functionality** → Posts comments and cleans up pending reviews

## Agent Usage

Agents now use Python functions instead of bash:

```python
# Post a comment (no bash, no prompts)
from automation.jleechanorg_pr_automation.orchestrated_pr_runner import post_pr_comment_python
post_pr_comment_python('owner/repo', 123, 'Comment text')

# Clean up pending reviews (no bash, no prompts)  
from automation.jleechanorg_pr_automation.orchestrated_pr_runner import cleanup_pending_reviews_python
cleanup_pending_reviews_python('owner/repo', 123, 'username')
```

This completely eliminates macOS permission prompts since no bash scripts are involved.
