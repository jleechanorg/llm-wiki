## Summary

Add **change photo** and **remove avatar** functionality to the game page avatar card, with a full E2E browser test that hits a real local server and captures screenshots/video evidence.

## Changes

### Feature: Change/Remove Avatar from Game Card
- **`index.html`** — Add "Change Photo" and "Remove" buttons to the avatar card overlay, plus hidden file input for photo selection
- **`avatar.css`** — Styled `.avatar-card-actions` buttons (ghost purple/red theme) with mobile responsive breakpoints
- **`app.js`** — `changeAvatarFromCard()` and `removeAvatarFromCard()` handlers; fixed auth bypass using `authTokenManager.getAuthHeaders()` and eliminated race condition by `await`ing upload before game load

### Bug Fixes
- **`app.js`** — Wizard avatar upload used `firebase.auth().currentUser` (null in test/bypass mode), silently skipping upload. Now uses `authTokenManager.getAuthHeaders()` for both real and test auth
- **`app.js`** — Avatar upload was fire-and-forget, racing with `handleRouteChange()`. Now properly `await`ed
- **`campaign-wizard.js`** — `uploadCampaignAvatar()` only accepted a Bearer token string. Now accepts either a headers dict (test bypass) or legacy token string  
- **`main.py`** — Upload and delete avatar routes called `firestore_service.get_firestore_client()` which doesn't exist → fixed to `get_db()` + added traceback logging
- **`firestore_service.py`** — Added `get_firestore_client = get_db` backward-compat alias for any cached references

### Styleguide
- **`arcane-scholar-styleguide.html`** — Added mobile game page mock (390px frame) with responsive avatar layout

### E2E Test
- **`test_avatar_e2e.py`** — Full Playwright browser test covering: campaign creation with avatar wizard upload → gameplay with avatar pip verification → expanded card with Change/Remove buttons → photo change → avatar removal. Runs against real local server with real LLM campaign generation (~23s). Captures 12 screenshots.
