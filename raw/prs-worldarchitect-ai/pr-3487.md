## Raw Body
This commit enables players to choose a parallel execution strategy when presented with multiple sequential tasks in story mode.

Changes:
- Added "Parallel Execution Option" section to planning_protocol.md
  - LLM now required to include parallel/delegation option when 2+ choices exist
  - Defined choice key conventions (parallel_execution, do_all_in_parallel, etc.)
  - Added examples for sequential tasks and combat scenarios
  - Documented mechanics: higher DC (+2 to +4), partial success handling, XP rewards
  - Specified when NOT to include parallel option (mutually exclusive choices, etc.)

- Updated CHOICE_SCHEMA documentation in narrative_response_schema.py
  - Clarified that analysis field can contain parallel execution metadata
  - Examples: delegation_targets, personal_focus, coordination_dc

Implementation follows LLM Architecture Principles:
- LLM decides execution strategy based on character capabilities and resources
- Server executes the resulting actions
- Schema-based approach (not templates) preserves LLM autonomy

Example use case:
Player completes Heapside liquidation and faces 3 objectives:
1. Audit High Hall Transition (political stability)
2. Consolidate Heapside Nodes (garrison setup)
3. Refine Hive Marrow (research advancement)

With this feature, player can now choose "Execute All Simultaneously" option, which coordinates all three operations in parallel using delegation and split focus.
