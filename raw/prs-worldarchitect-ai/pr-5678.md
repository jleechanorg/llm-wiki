> [!NOTE]
> **Medium Risk**
> Touches both frontend settings-save behavior and backend provider inference, which could affect persisted user settings if the new fallback/validation logic has edge cases; changes are localized and covered by new unit + E2E regression tests.
> 
> **Overview**
> Fixes settings persistence so saving no longer unintentionally flips `llm_provider` to the schema default when provider radios are temporarily unchecked/disabled. The frontend now tracks the last loaded/saved provider, resolves `llm_provider` from selected radio or cached server value (validated against schema allowed values), explicitly injects it into the payload, and updates the cache after successful saves.
> 
> On the backend, provider auto-inference from model fields now only runs for true model-only updates, avoiding provider drift when payloads include other settings. Adds regression coverage in `settings_listeners.test.js` for missing/invalid provider selection and an OpenClaw E2E scenario ensuring `llm_provider` stays stable when omitted on mixed updates, plus a small test helper update to support remote auth headers.
