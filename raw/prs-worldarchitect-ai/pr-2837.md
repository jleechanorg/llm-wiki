## Problem
Production Nocturne campaign has `active_missions` but **no `completed_missions` field**, preventing missions from auto-completing.

### Root Cause
- Older campaigns predate the `completed_missions` feature
- Missions can't move from `active_missions` to `completed_missions` when the target field doesn't exist  
- Sunderbrook mission stayed active despite narrative completion
- Required manual god mode intervention

### Evidence
Production Firebase query shows:
```
has_completed_missions field: False  ← BUG
active_missions: 1 (The Gilded Schism)
completed_missions: N/A (field doesn't exist)
```

## Solution

### Auto-Initialization Logic
```python
# firestore_service.py:496-502
if "active_missions" in state_to_update and "completed_missions" not in state_to_update:
    logging_util.info("Auto-initializing completed_missions field for older campaign")
    state_to_update["completed_missions"] = []
```

### Features
1. **Transparent Migration**: Auto-creates `completed_missions[]` on first state update
2. **Smart Conversion**: Supports dict-to-list and append syntax like `active_missions`
3. **Backward Compatible**: Existing campaigns with `completed_missions` unchanged
