## Summary
- Fixed `cerebras_provider.py` to handle responses where content is returned in `reasoning` field instead of `content`
- This fixes production failures on dev environment when using Qwen 3 models

## Root Cause
The Cerebras API returns content in the `reasoning` field for Qwen 3 models in some cases (intermittent behavior). The parser only looked for `message["content"]` which caused `ValueError: Invalid Cerebras response structure`.

## Changes
| File | Change |
|------|--------|
| `mvp_site/llm_providers/cerebras_provider.py` | Check both `content` and `reasoning` fields |
| `mvp_site/tests/test_cerebras_provider.py` | Added 2 tests for reasoning field handling |
