## Summary
Updated the copilot execution script to prefer the PyPI-packaged `orch` CLI when available, with automatic fallback to the local `orchestrate_unified.py` script for development environments. This improves portability and reduces dependency on local file paths.

## Changes
- Added `shutil` import for CLI detection
- Added `ORCH_CLI` constant that detects the `orch` command using `shutil.which()`
- Updated command construction in `main()` to conditionally use either:
  - PyPI `orch run` command (preferred when installed)
  - Local `python3 orchestrate_unified.py` script (fallback for development)
- Added informative logging to indicate which execution path is being used

## Benefits
- **Improved portability**: Packaged CLI doesn't rely on local repository structure
- **Better development experience**: Automatically uses local script when package isn't installed
- **Clearer execution flow**: Explicit logging shows which orchestration method is active
- **Backwards compatible**: Existing development setups continue to work without changes
