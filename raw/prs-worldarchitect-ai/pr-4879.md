## Summary
Adds explicit guardrails across all system instruction documents to prevent dice roll mechanics from appearing in narrative text. Dice rolls must be placed exclusively in the `action_resolution.mechanics.rolls` JSON field, while narrative descriptions should focus on outcomes only.

## Changes
- **combat_system_instruction.md**: Added rule prohibiting dice notation in narrative text with specific examples of forbidden patterns
- **dialog_system_instruction.md**: Added critical section on dice roll separation with examples of correct vs incorrect approaches for persuasion and deception checks
- **master_directive.md**: Updated example flow to show proper separation of mechanics (in JSON) from narrative, demonstrating how to narrate outcomes without showing dice notation
- **narrative_system_instruction.md**: 
  - Added dice-in-narrative guardrail to essentials section
  - Expanded "CRITICAL" section with comprehensive forbidden patterns list
  - Added "CORRECT APPROACH" subsection with clear examples
  - Explained the rationale: dice rolls appear in UI dice tray automatically from JSON, so narrative duplication creates confusion

## Benefits
- **Clearer player experience**: Eliminates confusing duplicate dice information (in both narrative and UI)
- **Consistent enforcement**: All system instructions now align on this critical rule
- **Better examples**: Provides concrete forbidden patterns and correct approaches for developers
- **Improved narrative quality**: Encourages descriptive outcomes over mechanical notation, enhancing immersion
- **Reduced ambiguity**: Explicit rationale helps developers understand the "why" behind the rule
