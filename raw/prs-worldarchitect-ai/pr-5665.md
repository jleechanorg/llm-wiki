## Raw Body
### Motivation

- Provide a single, trackable execution plan by expanding the harness roadmap into status-marked milestones and atomic actionable steps.  
- Harden pair orchestration and benchmarking so `pairv2` can be reliably simulated and compared to legacy `pair_execute`.  
- Address reviewer-identified reliability, correctness, and test flakiness issues across docs, runtime scripts, and tests.

### Description

- Roadmap: expanded `roadmap/2026-02-19-harness-engineering-consolidated-plan.md` to clarify tier rules, fix wording (e.g., "boot the app") and convert the 6–8 week sequence into status-marked milestones with steps and checkboxes.  
- Benchmarking: updated `.claude/scripts/benchmark_pair_executors.py` to decode `TimeoutExpired` stdout/stderr to strings, guard `_duration_summary` against empty inputs, and make `benchmark_parallel` resilient to worker exceptions while preserving per-iteration mapping.  
- Pair orchestration: updated `.claude/scripts/pair_execute.py` to remove an overly-broad fatal marker and pass an explicit `ORCHESTRATION_TMUX_SOCKET` into readiness checks, and updated `.claude/scripts/pair_execute_v2.py` to increment live retry `cycle`, stop retrying on terminal left-contract failures, centralize live-session status polling, generate unique session IDs, and cleanup partially-launched agents on failures.  
- Tests & docs: adjusted tests to use `CODEX_MODEL` instead of hard-coded `gpt-5`, refined time mocking in `test_pair_execute.py`, corrected the `pair` docs to recommend `--simulate` for local scripted runs, and committed a consolidated follow-up commit with comment-tracking metadata (`[codex-automation-commit]`).
