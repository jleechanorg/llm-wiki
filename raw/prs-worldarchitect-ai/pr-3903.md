## Production Code Changes

### 1. `mvp_site/world_logic.py` (+168 lines)
**Server Auto-Set for `rewards_processed` Flag**

Previously, the server detected when the LLM forgot to set `rewards_processed=true` and injected correction messages into the next prompt. This failed because:
- Prose instructions get buried in 16KB prompts
- LLM prioritizes narrative over administrative flags

**New behavior:** Server now owns `rewards_processed` as an administrative flag and auto-sets it when:
- **Check 1:** Combat ended (`combat_phase` in finished phases + `combat_summary` exists)
- **Check 2:** Encounter completed (`encounter_completed=true` + `encounter_summary` exists)
- **Check 3:** XP increased during combat
- **Check 4:** XP increased during encounter
- **Check 5:** Fallback for narrative-only rewards (XP increased outside combat/encounter)

### 2. `mvp_site/llm_providers/gemini_provider.py` (+80 lines)
**Test Stub Client for CI**

Adds a lightweight Gemini stub that activates when BOTH conditions are true:
- `TESTING_AUTH_BYPASS=true`
- API key is missing OR starts with `test-` or `dummy-`

This prevents real Gemini API calls in CI while still allowing local development with real keys.

Functions affected:
- `get_client()` - Returns `_TestClient` in test mode
- `count_tokens()` - Returns approximate token count (len/4)
- `generate_json_mode_content()` - Returns stub JSON response
