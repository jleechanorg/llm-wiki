## Summary

This PR migrates all slash commands from the custom MCP memory server (mcp__memory-server__*) to Claude's native memory commands (memory_search, memory_save, memory_read, memory_forget).

## Changes

### execute.md
- Added native memory reading at the start (using memory_search)
- Added native memory writing after completion (using memory_save)
- Removed bead tracking from the execute command

### Other Commands Updated
- memory.md: Now uses native memory commands instead of MCP
- learn.md: Uses memory_save for persisting learnings
- guidelines.md: Uses memory_search and memory_save
- goal.md: Uses native memory for goal persistence
- archreview.md: Uses native memory for architectural insights
- debug-protocol.md: Uses native memory for debugging patterns
- design.md: Uses native memory for design patterns

## Benefits

- Simplifies memory operations (no MCP overhead)
- Uses Claude's built-in memory capabilities
- Removes dependency on custom MCP memory server
- Consistent interface across all commands

## Breaking Change

Custom MCP memory server is no longer used. All commands now use Claude's native memory for persistence.
