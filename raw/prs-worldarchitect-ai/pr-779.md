## Summary

This PR adds comprehensive documentation for creating **threaded replies to GitHub PR review comments** using the REST API - a capability that works via the web interface but isn't well documented for programmatic access.

### 🔧 Working Solution Discovered

```bash
# CORRECT: Creates threaded reply
gh api repos/owner/repo/pulls/PR/comments \
  -f body="Reply text" \
  -F in_reply_to=COMMENT_ID  # ← -F flag (numeric) is key
```

**Critical insights**:
- Use `-F` flag for `in_reply_to` parameter (treats as number)
- Use `-f` flag for `body` parameter (treats as string)
- Parameter name is `in_reply_to` (not `in_reply_to_id`)
- Endpoint is `/comments` (not `/reviews`)
