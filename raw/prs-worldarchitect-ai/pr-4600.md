## Raw Body
### Motivation
- Prevent duplicate GitHub Actions runs for the same PR by cancelling in-progress workflow runs while preserving normal push behavior.
- Reduce wasted runner time and noise from repeated PR updates by grouping workflow concurrency by PR number or ref.
- Apply a consistent concurrency policy across core test and validation workflows to standardize CI behavior.

### Description
- Added a workflow-level `concurrency` block to multiple workflow files with `group: ${{ github.workflow }}-${{ github.event.pull_request.number || github.ref }}` and `cancel-in-progress: ${{ github.event_name == 'pull_request' }}` to scope and cancel PR runs.
- Updated the following workflows: `.github/workflows/test.yml`, `coverage.yml`, `presubmit.yml`, `pr-preview.yml`, `hook-tests.yml`, `mcp-smoke-tests.yml`, `doc-size-check.yml`, and `test-deployment.yml`.
- Kept existing triggers and permissions unchanged so push (post-merge) runs are not cancelled by this PR-specific behavior.
- Change is configuration-only and does not modify runtime application code.

### Testing
- Pre-commit hooks ran during the commit step and passed successfully via the local commit checks.
- No CI workflow runs were executed as part of this change because it is a config-only update.  
- No functional or unit tests were run locally for this patch since no application code was changed.
