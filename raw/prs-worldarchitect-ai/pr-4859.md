## Production Code Changes

### 1. `mvp_site/dice_integrity.py` (+20/-2 lines)
**Purpose:** Detects fabricated dice values in code_execution strategy

- **Added `tool_requests_executed` parameter to `_is_code_execution_fabrication()`** - Previously returned `True` (fabrication) whenever `code_execution_used=False` -> Now checks if tool_requests were executed via fallback path (gemini_provider.py:1494-1515) with real server-side RNG -> Eliminates false-positive fabrication flags for legitimate dice

### 2. `mvp_site/llm_service.py` (+5/-1 lines)
**Purpose:** Orchestrates LLM calls and post-processing

- **Pass `tool_requests_executed` to `_is_code_execution_fabrication()`** - Previously called with only 2 args -> Now passes `tool_requests_executed=bool(getattr(api_response, "_tool_requests_executed", False))` -> Enables the fallback-aware fabrication check

### 3. `mvp_site/prompts/dice_system_instruction_code_execution.md` (+1/-1 lines)
**Purpose:** Dice instructions for Gemini code_execution mode
