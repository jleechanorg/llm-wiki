## Summary

- **Streaming contracts** — SCENARIO 7 (character / think / god-mode) now pass 9/9 in both mock and real CI mode; removed `response_json_schema` from streaming calls that broke code execution and caused double-streaming
- **Smoke auth token** — `SMOKE_TOKEN` injected into preview server via `pr-preview.yml`; sent as `X-MCP-Smoke-Token` header by CI; server validates it to bypass Firebase auth for preview smoke tests without exposing production credentials
- **CI hardening** — `STREAM_RESPONSE_SIGNING_SECRET` wired into GitHub Actions; stream signature + preview URL checks enforced in `mcp-smoke-tests.yml`; FastEmbed model pre-warmed via cache so classifier loads without downloading on every CI run
- **Deploy fix** — `--remove-secrets=SMOKE_TOKEN` added to preview deploy step to clear stale Cloud Run secret-type reference before setting it as a plain env var (fixes "cannot update env var to string literal" error)
- **REV-fd2 design** — Two-job preview CI architecture designed (roadmap): Job 1 auto-runs against mock-mode GCP preview and verifies HMAC signature as proof of GCP origin; Job 2 (`/smoke`) runs against real-LLM GCP preview
