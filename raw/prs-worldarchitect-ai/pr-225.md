## Summary
Fixes incomplete JSON responses from Gemini API when using JSON response mode for entity tracking. The AI sometimes returns truncated JSON when hitting token limits, causing parsing failures.

## Problem
- When using `response_mime_type: "application/json"`, Gemini sometimes returns incomplete JSON like:
  ```json
  {"narrative": "Story text...
  ```
- This happens even with narratives that aren't particularly long
- Users would see truncated stories or errors

## Solution
1. **Robust JSON Parser** with 5-strategy fallback system:
   - Standard JSON parsing
   - Fix JSON boundaries 
   - Complete incomplete JSON
   - Extract fields via regex
   - Aggressive reconstruction

2. **Reduced JSON mode token limit** from 30k to 20k for better reliability

3. **Graceful recovery** - extracts narrative even from severely truncated responses

## Key Points
- **Users never see raw JSON** - only the narrative text
- System already strips JSON before returning to users
- JSON structure is used internally for entity tracking only
- Enhanced parser handles various truncation scenarios
