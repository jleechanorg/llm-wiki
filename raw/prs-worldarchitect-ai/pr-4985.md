## Production Code Changes

### 1. `mvp_site/narrative_response_schema.py` (+35 lines)
**Purpose:** Parses structured JSON responses from LLMs

- **Added Strategy 3: "unterminated_string_recovery"** - No recovery existed for `Unterminated string` JSONDecodeError → Now truncates at last valid `}` before error position, closes unbalanced braces/brackets, and recovers the narrative → Grok returns truncated JSON mid-string; without this, 3000+ char narratives were replaced with 49-char error message "Invalid JSON response received. Please try again."

### 2. `mvp_site/llm_providers/openai_compatible_provider_core.py` (+22 lines)
**Purpose:** Shared core for OpenAI-compatible chat-completions providers (OpenRouter, Cerebras)

- **Added finish_reason + usage logging** - No visibility into why responses truncated → Now logs `finish_reason`, `prompt_tokens`, `completion_tokens` after every API call → Enables diagnosing whether Grok truncation is `finish_reason=length` (token limit) or `finish_reason=stop` (model choice)
- **Added RESPONSE_TRUNCATED warning** - No alert when `finish_reason=length` → Now logs explicit warning with max_tokens config → Surfaces token limit issues in Cloud Logging
