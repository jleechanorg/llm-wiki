> [!NOTE]
> Consolidates CI to a group-based workflow with deterministic grouping, enhanced outputs, and multi-directory test execution; removes the legacy workflow.
> 
> - **CI Workflows**:
>   - **Consolidation**: Delete legacy `/.github/workflows/test-directory-based.yml`; use updated `/.github/workflows/test.yml`.
>   - **Triggers**: Broaden ignored docs to `**/*.md`.
>   - **Matrix/Outputs**: `detect-changes` now uses `include` format and exports `selected-groups`; matrix keys switched from `test-dir` to `test-group`.
>   - **Job Naming/Artifacts**: Label jobs as `Directory tests (${{ matrix.test-group }})`; artifact names use `test-group`.
>   - **Deps**: Cache key includes `test-group`; conditional extra installs aligned to groups (`claude`, `orchestration`, `automation`, `scripts`, `mcp`).
>   - **Execution**: Run tests for one or multiple dirs via `matrix.test-dirs`; aggregate test counts and fallback to core if none found; print selected test dirs.
>   - **Summary**: Final summary echoes `test-dirs` and `selected-groups`.
> - **Change Detection Script (`scripts/ci-detect-changes.sh`)**:
>   - **Grouping**: Replace per-dir flags with `GROUP_CONFIG` and `DIR_TO_GROUP`; maintain deterministic `ordered_groups`.
>   - **Outputs**: Emit `selected-groups`, consolidated `test-dirs`, and matrix for both `include` and `simple` modes; fallback to `core` when empty.
>   - **Robustness**: Handle trimming, deduping, and multi-dir groups; improved logging.
