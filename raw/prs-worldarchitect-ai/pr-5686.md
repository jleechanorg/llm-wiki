## Production Code Changes

**`integrate.sh`**
- Before: `git log --oneline ... | head -N | sed ...` — SIGPIPE on short output exits the script
- Now: `... | head -N | sed ... || true` — SIGPIPE is tolerated
- Before: `git checkout main` / `git checkout -b $branch` — triggers Husky which calls `bd`, fails if daemon is down
- Now: `git -c core.hooksPath=.git/hooks checkout ...` — uses raw git hooks, skips Husky
- Before: Always ran `detect_squash_merged_commits` regardless of history size
- Now: FORCE_MODE skips detection when commit_count > FORCE_SQUASH_CHECK_MAX (default 1000)

## Root Cause

The beads pre-commit hook (`bd sync --flush-only`) hard-exits when the SQLite DB is missing (daemon was killed). `git stash` runs the pre-commit hook internally, so stash failures cascade into integrate.sh failures. The hook was patched locally to warn-only; this PR documents the pattern and hardens the script.

## Test Changes

None — shell script defensive changes only.

## Known Limitations

- The pre-commit hook warn-only fix is local (`.git/hooks/` is not committed). Documented in the skills file with `bd init` as the recovery path.
