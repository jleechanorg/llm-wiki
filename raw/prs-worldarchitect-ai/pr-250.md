## Summary

Fixed a critical bug where users were seeing raw JSON output instead of formatted narrative text in campaign responses.

## Root Cause

The bug was in `narrative_response_schema.py:142` where a `return` statement was incorrectly indented outside the `except` block. This caused the function to continue executing after exception handling, potentially returning wrong values and displaying raw JSON to users.

## Fix Applied

- Fixed indentation of the `return narrative, fallback_response` statement
- Moved it inside the `except` block where it belongs
- This ensures proper fallback behavior when JSON parsing fails
