## Problem
The `/api/campaigns/<campaign_id>/interaction` endpoint had extremely restrictive rate limits causing 429 errors:
- **Rate limit**: 30 requests per hour (1 request every 2 minutes)
- **Keyed by IP address** (`get_remote_address`), not user ID
- Multiple users from same network/proxy share the limit
- Internal Cloud Run IP (169.254.169.126) suggests proxy compounding the issue

## Solution
Change rate limits to sanity-check thresholds rather than restrictive quotas:
```python
@limiter.limit("30000 per hour, 1000 per minute")  # High limits for normal conversation flow
```

## Impact
- ✅ Users can have normal conversations without hitting rate limits
- ✅ Per-minute limit still prevents abuse (1000 req/min = reasonable burst protection)
- ✅ Per-hour limit acts as sanity check, not artificial restriction
