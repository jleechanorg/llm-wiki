## Summary

This PR fixes dice code execution validation and establishes `action_resolution.mechanics.rolls` as the single source of truth for dice output, with backend extraction to `dice_rolls` and `dice_audit_events` for UI compatibility.

## Key Changes

### Core Fixes
- **FactionManagementAgent**: Add `PROMPT_TYPE_NARRATIVE` to `REQUIRED_PROMPT_ORDER` (was causing missing narrative instruction)
- **Classifier-first routing**: Remove early return that forced FactionManagementAgent selection
- **Code execution evidence**: Save `executed_code` field in `extract_code_execution_evidence()`

### Dice Output Architecture
- **Single source of truth**: `action_resolution.mechanics.rolls` is the canonical location for dice data
- **Backend extraction**: `add_action_resolution_to_response()` auto-populates `dice_rolls` and `dice_audit_events`
- **Structured audit events**: Convert rolls to proper audit event format with source, label, notation, rolls, modifier, total

### Prompt Engineering
- **Damage roll on hit only**: Gemini prompt instructs to skip damage roll when attack misses
- **Centralized dice output**: All dice output consolidated in `action_resolution.mechanics.rolls`
- **Chi-square fallback**: Parse from action_resolution or dice_rolls when stdout unavailable

### Test Infrastructure
- **MCPTestBase**: Refactored all testing_mcp tests to use shared base class
- **Code execution validation**: New `validate_code_execution()` helper with `CodeExecutionEvidence` dataclass
- **Raw response capture**: Save raw LLM responses for debugging
