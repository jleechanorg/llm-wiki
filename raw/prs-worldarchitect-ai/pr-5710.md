## Summary
Implements a complete provably fair dice roll system that allows players to cryptographically verify that dice rolls are fair and not manipulated. The system uses SHA-256 commitments and seed injection into Gemini's code execution to ensure deterministic, auditable randomness.

## Changes
- **New module `dice_provably_fair.py`**: Core cryptographic primitives for provably fair dice
  - `generate_server_seed()`: Creates a 64-char hex seed using `secrets.token_hex()`
  - `compute_commitment()`: Generates SHA-256 hash of seed as pre-roll commitment
  - `verify_commitment()`: Validates revealed seed against stored commitment
  - `inject_seed_into_prompt()`: Replaces `random.seed(time.time_ns())` with server seed in code
  - `extract_seed_from_executed_code()`: Recovers injected seed from executed code via regex
  - `verify_seed_in_executed_code()`: Confirms seed presence in executed code samples

- **Modified `gemini_provider.py`**:
  - Added `server_seed` parameter to `apply_code_execution_system_instruction()`
  - Injects seed into code execution system instruction when provided
  - Updated `generate_content_with_code_execution()` to generate and log provably fair seed commitment

- **Modified `gemini_code_execution.py`**:
  - Enhanced `extract_code_execution_evidence()` to detect and extract injected seeds
  - Added `dice_seed_verified`, `dice_server_seed`, and `dice_seed_commitment` fields to evidence dict
  - Seeds are extracted from executed code and stored in Firestore for player audit

- **Comprehensive test suite `test_dice_provably_fair.py`** (258 lines):
  - 25 pure function tests covering all cryptographic operations
  - Tests for seed generation uniqueness, commitment determinism, verification logic
  - Tests for seed injection and extraction from code samples
  - Integration tests for system instruction modification
  - Integration tests for evidence extraction from Gemini responses
  - Mocks Google SDK to run without external dependencies
