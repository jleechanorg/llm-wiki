## Raw Body
## Summary
- Fix fixpr cron job failing with "Invalid API key" errors
- Created wrapper script that exports MINIMAX_API_KEY before calling jleechanorg-pr-monitor
- Updated all cron jobs to use the wrapper script
- Added wrapper script to repo (`automation/jleechanorg-pr-monitor-wrapper.sh`)
- Updated install_cron_entries.sh to install wrapper to ~/.local/bin
- Fixed CLI validation to not require exit code 0 (Claude CLI exits 1 but produces valid output)

## Root Cause
1. Cron runs with a minimal environment that doesn't include user environment variables like MINIMAX_API_KEY
2. The preflight validation was failing because it couldn't find the API key
3. Additionally, Claude CLI exits with code 1 even when producing valid output, causing false negatives in validation

## Setup Required
Users need to create ~/.automation_env:
```bash
echo "export MINIMAX_API_KEY=\$MINIMAX_API_KEY" > ~/.automation_env
echo "export ANTHROPIC_BASE_URL=https://api.minimax.io/anthropic" >> ~/.automation_env
```
