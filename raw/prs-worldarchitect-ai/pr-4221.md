### Motivation
- Enforce the centralized dice source (`action_resolution.mechanics`) while preventing bypasses from LLM-populated legacy fields. 
- Fix a runtime crash where `_check_dice_integrity` referenced an undefined `dice_rolls` variable when dice came from the primary path. 

### Description
- Treat both `action_resolution.mechanics.rolls` and `action_resolution.mechanics.audit_events` as the primary structured dice sources in `_detect_narrative_dice_fabrication`. 
- Log a `SCHEMA_VIOLATION` when LLMs populate legacy `dice_rolls`/`dice_audit_events` and mark those legacy-populated responses as containing dice for validation to prevent bypass attacks. 
- Prevent `NameError` in `_check_dice_integrity` by tracking `rolls_count`, `audit_events_count`, and `dice_rolls` and build a unified error message showing counts instead of referencing an undefined variable. 
- Clean up nested conditionals and add targeted `# noqa` annotations to satisfy lint rules and reduce Ruff false positives, and update some example string quoting for consistency. 
- Files changed include `mvp_site/dice_integrity.py` and associated unit tests in `mvp_site/tests/test_dice_integrity_helpers.py`.
