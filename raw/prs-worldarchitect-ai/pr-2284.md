## Summary
- Add `OUTPUT_TOKEN_RESERVE_RATIO` (20%) to ensure sufficient output space for quality responses
- Fail fast when input exceeds 80% of context instead of sending doomed requests with `max_output_tokens=1`
- Cerebras provider now detects `finish_reason='length'` with no content and raises clear error
- Clear error messages showing token usage and limits

## Problem
Campaign with 113,037 tokens caused empty API responses because:
1. System set `max_output_tokens=1` when context was nearly full
2. API returned `finish_reason='length'` with no content
3. Provider crashed with unclear "Invalid Cerebras response structure" error

## Solution
| Before | After |
|--------|-------|
| Max input: ~99% of context | Max input: **80%** of context |
| Output reserve: 1K minimum | Output reserve: **20%** of context |
| Error: "Invalid response structure" | Error: "Context too large... reserving 180K tokens (20%) for output" |
