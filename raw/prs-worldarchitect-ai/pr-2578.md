## Summary

This PR adds explicit, mandatory relationship and reputation mechanics to the system prompts. The mechanics were previously scattered and implicit, causing them to not trigger properly during gameplay.

### Problem
- Relationships were mentioned but had no schema or triggers
- Reputation had no public/private distinction
- No explicit instructions on WHEN to check or update these values
- A previous PR (`ec105ed`) compressed prompts by 76%, losing detailed Narrative Ripples content

### Solution

**1. Relationship Mechanics (§Relationship Mechanics)**
- Structured schema: `trust_level`, `disposition`, `history`, `debts`, `grievances`
- Trust scale: -10 (hostile) to +10 (bonded) with 8 disposition levels
- Mandatory CHECK triggers before NPC interactions
- Mandatory UPDATE triggers after significant actions
- Behavior modifiers for each trust tier (prices, DC adjustments, access)

**2. Reputation System (§Reputation System)**
- **Public reputation** (-100 to +100): What everyone knows
  - Components: score, titles, known_deeds, rumors, notoriety_level
  - 8 levels from Infamous to Legendary with mechanical effects
- **Private reputation** (-10 to +10 per faction): What specific groups know
  - Components: score, standing, known_deeds, secret_knowledge, trust_override
  - 8 levels from Enemy to Champion

**3. Priority Hierarchy**
```
1. PRIVATE TRUST OVERRIDE (strongest) → reputation.private[faction].trust_override
2. PRIVATE RELATIONSHIP → npc_data.relationships.player.trust_level
3. PRIVATE REPUTATION → reputation.private[faction].standing
4. PUBLIC REPUTATION → reputation.public.notoriety_level
5. DEFAULT (weakest) → Neutral
```

Key principle: **Direct experience trumps hearsay**

**4. Restored Narrative Ripples**
- Reputation spread mechanics (immediate → hours-days → days-weeks → weeks-months)
- Ripple types: Political, Social, Economic
- Cascade rules for faction interactions
