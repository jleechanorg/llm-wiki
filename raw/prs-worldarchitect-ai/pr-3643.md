## Raw Body
Fix TOML syntax errors in .gemini/commands

This PR fixes syntax errors in several TOML files in `.gemini/commands/` that were preventing the `FileCommandLoader` from parsing them.

## The Issue
TOML multi-line strings (`"""`) only support specific escape sequences (e.g. `\n`, `\t`, `\\`, `\"`).
The files contained invalid escape sequences like `\.`, `\(`, `\)`, `\$`, etc., which are common in regex and shell scripts but invalid in TOML strings unless the backslash is escaped.

## The Fix
All invalid backslashes inside the TOML strings have been escaped (e.g. `\.` -> `\\.`). Valid escapes and line continuations were preserved.

## Verified
Validated using `tomllib` (Python 3.11+) that the files are now parseable.
