## Summary
Implements dual-scope MCP server installation to ensure MCP servers are available both in project `.claude/settings.json` (git-tracked) and system-wide user scope (via Claude CLI database).

## Problem Statement
Previously, `mcp_common.sh` used `MCP_SCOPE=user` by default, which installed MCP servers to a hidden Claude CLI database. This meant:
- MCP servers weren't visible in project `.claude/settings.json`
- Server configurations couldn't be tracked in git
- Project configuration wasn't synchronized to home directory

## Solution
1. **Changed default scope**: `MCP_SCOPE` now defaults to `local` instead of `user`
2. **Added dual-scope flag**: `MCP_INSTALL_DUAL_SCOPE=true` enables automatic dual installation
3. **Modified `add_mcp_server()`**: Automatically installs to both scopes when enabled
   - First installs to `local` scope (project `.claude/settings.json`)
   - Then installs to `user` scope for system-wide availability
   - Handles both grok-mcp special case and standard npx packages
4. **Populated project settings.json**: Added all 15 MCP servers to `.claude/settings.json`
