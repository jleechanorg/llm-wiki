## Production Code Changes

### 1. `orchestration/cli_validation.py` (+15/-20 lines)
**Purpose:** Fix preflight validation to actually catch quota errors

- Increase timeout from 10s to 30s (unified for all CLIs)
- Change timeout behavior: return `success=False` instead of `True` (fail-safe)
- Change exception behavior: return `success=False` instead of `True` (fail-safe)
- Add verification that output contains expected answer "4"
- Add "hit your usage limit", "usage limit" to quota detection phrases
- Remove false-positive help/version fallback detection

### 2. `orchestration/task_dispatcher.py` (+2/-5 lines)
**Purpose:** Use unified timeout and skip MCP servers for faster Gemini validation

- Add `--allowed-mcp-server-names none` to Gemini validation command (2s vs 6s)
- Remove CLI-specific timeout variable imports (now unified)

## Test Changes
- Updated tests to expect `success=False` for timeouts/exceptions (was `True`)
- All 64 tests pass

## Root Cause
The preflight "2+2" validation was timing out before seeing quota errors:
```
Before: ✅ "validation passed" (timed out after 10s, returned True)
After:  ⚠️ "quota/rate limit detected" (caught error in 2s, returned False)
```

This caused automation to post "Fix-comment run queued" comments and start watchers
for PRs even when the CLI agents were quota-exhausted and would immediately fail.
