## Raw Body
### Motivation
- Close multiple security and reliability review findings for the WorldAI Tools MCP proxy (untrusted role headers, Firestore prefix / op checks, gcloud filter injection, stale deploy tokens, unsafe log artifacts, and fragile harness behavior).
- Align runtime defaults with the repo guardrails (600s request-layer timeout) and make the real-harness more portable and robust for integration evidence runs.

### Description
- Added policy & validation defaults and helpers to `mvp_site/worldai_tools_mcp_proxy.py`: request-timeout default from `WORLDARCH_TIMEOUT_SECONDS`, explicit unsafe-auth bypass flag (`WORLDTOOLS_UNSAFE_SKIP_AUTH`), `Content-Length` bounding, deploy-confirm token freshness checks, gcloud service regex validation and sanitized log `contains` input, case-normalized gcloud allowlist checks, segment-aware Firestore prefix matching, and allowed-op validation for collection-group queries.
- Replaced fragile text->JSON parsing with `ast.literal_eval` for dict literals and cached local tool names to avoid hot-path allocations, and created secure temp files for ops launcher logs.
- Strengthened HTTP handler: clearer 400 responses for missing/invalid `Content-Length` and centralized JSON response writer.
- Hardened the real test harness `testing_mcp/test_worldai_tools_mcp_proxy_real.py`: removed trusting of `X-Worldtools-Roles` in the harness, made process termination safe via `communicate()`, made upstream python selection portable (fallback when `vpython` missing), documented the upstream mock-fixture choice, and ensured nonzero exit on top-level failures.
- Tests: adjusted unit tests to expect `ProxyJsonRpcError` and added a test for stale deploy-token rejection in `mvp_site/tests/test_worldai_tools_mcp_proxy.py`.
- Posted a consolidated consolidated response summary to the PR comment thread covering the reviewed comments and generated a responses file for traceability.
