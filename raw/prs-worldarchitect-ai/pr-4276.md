## Summary
Implement local branch name generation (`fixpr_` prefixed) for PR conflict resolution to isolate work from remote branches and prevent concurrent modification issues.

## Changes
- Added `generate_local_branch_name()` function to create sanitized local branch names from remote branch names (e.g., `feature/add-cool-feature` → `fixpr_feature-add-cool-feature`)
- Updated `dispatch_agent_for_pr()` to generate and use local branch names in git instructions
- Modified conflict resolution instructions to:
  - Fetch remote branch explicitly: `git fetch origin {branch}`
  - Create/reset local branch: `git checkout -B {local_branch} origin/{branch}`
  - Push to remote using refspec: `git push origin {local_branch}:{branch}`
- Updated fallback instructions for worktree creation to reference remote branch correctly
- Added comprehensive test coverage for local branch name generation with special character sanitization

## Benefits
- **Isolation**: Local branches prevent conflicts when multiple agents work on the same PR simultaneously
- **Safety**: Explicit fetch and refspec push reduce accidental modifications to remote branches
- **Clarity**: Clear separation between local work (`fixpr_*`) and remote tracking (`origin/*`)
- **Robustness**: Handles branch names with special characters through sanitization
