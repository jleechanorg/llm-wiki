## Summary

Adds new evidence standards for proving runtime behavior (truncation, filtering, deduplication) based on lessons learned from the memory budget testing session.

## Key Additions

### 1. Inferential vs Direct Evidence
- "Action succeeded = worked" is **not proof**
- Must have runtime logs FROM THE CODE showing selection/exclusion

### 2. Code Instrumentation
- If claiming internal behavior, code MUST produce logs proving it
- Example: `[MEMORY_BUDGET] TRUNCATED: 554 selected, 51 excluded`

### 3. Server Log Capture
- Start server with logs to file
- Run tests against that server
- Extract behavior proof from logs

### 4. Full Git Provenance
- Diff stats (`git diff --stat`)
- Commits ahead/behind origin/main
- Modified files list

### 5. Per-File Checksums
- `file.txt.sha256` alongside `file.txt`
- Easier to verify individual artifacts
