## Summary
Improved handling of narrative text display when god mode is active. The backend returns a `god_mode_response` instead of a standard narrative in god mode, so the frontend now detects this and displays an appropriate message instead of treating it as an error.

## Changes
- Added detection for god mode by checking if `god_mode_response` exists and is non-empty
- Updated narrative text fallback logic to distinguish between god mode (expected empty narrative) and actual errors
- Changed error message to `'[God mode - narrative not advancing]'` when in god mode with empty narrative
- Updated console logging to use `console.log()` with success indicator for god mode, and `console.error()` only for actual errors
- Improved defensive checks to handle empty strings properly in both god mode and normal mode

## Benefits
- Eliminates false error messages when god mode is functioning correctly
- Provides clearer feedback to developers about the application state
- Makes the distinction between expected behavior (god mode) and actual errors more explicit
- Improves debugging experience with appropriate log levels
