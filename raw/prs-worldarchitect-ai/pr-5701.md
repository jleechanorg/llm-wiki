## Summary
Remove the `confidence` field from player-facing UI rendering across the codebase. The `confidence` field is now strictly mechanics-only (used for DC modifier calculations) and should never be displayed to players. Instead, confidence is expressed through the language and framing of pros/cons.

## Changes
- **test_planning_block_parsing.js**: Removed confidence from `hasExpandedDetails` condition and button text rendering; updated test cases to reflect that confidence is mechanics-only and not rendered
- **app.js**: Removed confidence from `hasExpandedDetails` condition and button text rendering (mirrors test file changes)
- **planning_protocol.md**: Updated documentation to clarify that `confidence` is mechanics-only, with guidance on expressing confidence through pros/cons language and plan quality stat-gating

## Benefits
- Clearer separation of concerns: mechanics (confidence DC modifier) vs. narrative (pros/cons)
- Improved player experience: confidence is conveyed through natural language rather than explicit labels
- Reduced UI clutter: removes "Assessment:" label from choice buttons
- Better alignment with design intent: pros/cons provide richer, more contextual information than a single confidence label
