## Summary

Adds support for exporting `.github/workflows` directory when running `/exportcommands`. The workflows are exported as examples with clear documentation that they require integration into the target codebase.

## Changes

### export_config.py
- Added `"workflows"` to `EXPORTABLE_COMPONENTS` list
- Added description for workflows component

### exportcommands.py  
- Added `_export_github_workflows()` method to export workflow files
- Added `_apply_workflow_filtering()` to genericize project-specific values (GCP IDs, repo refs, etc.)
- Added `_create_workflows_readme()` to generate integration instructions README
- Added `workflows_count` counter initialization
- Updated `phase1_local_export()` to call workflows export
- Updated `_copy_to_repository()` with workflows directory mapping
- Updated commit message and PR body to include workflows count
- Added git config for user.email/user.name and disabled gpgsign in clones
- Added `~/.local/bin/gh` to gh CLI search paths

## Workflow Export Features

Each exported workflow includes:
- **Example header** at top of file explaining adaptation requirements
- **Placeholders** for project-specific values:
  - `$GCP_PROJECT_ID` for Google Cloud project IDs
  - `$GITHUB_REPOSITORY` for repository references
  - `$GITHUB_OWNER` for GitHub username/org
  - `$USER` for usernames
- **README.md** with detailed integration instructions
