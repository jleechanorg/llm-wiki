## Raw Body
## Problem

workflow_run-triggered mock tests were running **silently** without posting results to PRs. Users couldn't see that mock tests ran after deployment because workflow_run events don't appear in PR checks.

## Solution

Add comment posting to mock test jobs when triggered by workflow_run events.

## Changes

### PR Number Export
- Modified `Get PR ref for workflow_run trigger` step to export PR number
- Uses `core.setOutput('pr_number', ...)` for downstream steps

### Comment Posting Steps
Added to both `try-self-hosted` and `fallback-github-hosted` jobs:
- ✅ Success comment: Posts when tests pass
- ❌ Failure comment: Posts when tests fail

### Comment Format
```markdown
✅ **Mock API Smoke Tests Passed** (workflow_run)

Mock tests ran successfully after deployment completion.

- **Trigger**: Post-deployment validation
- **Mode**: Mock APIs (TESTING_AUTH_BYPASS=true)
- **SHA**: `abc1234`
- **Run**: [View logs](https://...)

🤖 Auto-posted by workflow_run event
```

## Why This Matters

**Before:**
- Mock tests ran silently after deployment
- No visibility on PR that tests executed
- Users confused about where results were

**After:**
- Clear comment on PR showing test results
- Direct link to logs
- Labeled as workflow_run trigger (not a PR check)
