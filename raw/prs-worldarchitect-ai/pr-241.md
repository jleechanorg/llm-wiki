## Summary
This PR improves test organization by separating unit and integration tests into dedicated directories and enhances the `run_tests.sh` script to handle test exclusions properly.

## Changes Made

### 🔧 **Enhanced run_tests.sh Script**
- **Added exclusion for `test_integration` subdirectories**: Prevents integration tests in `./tests/test_integration/` from running during GitHub export mode
- **Enhanced integration test inclusion logic**: Checks both root `./test_integration/` and nested `./tests/test_integration/` directories
- **Updated warning messages**: Clear feedback about what tests are excluded
- **Better test discovery**: More robust find commands with proper exclusion patterns

### 📁 **Test Organization Improvements** 
- **Moved 5 integration test files** from `mvp_site/tests/` to `mvp_site/test_integration/`:
  - `test_banned_names_integration.py`
  - `test_entity_retry_integration.py`
  - `test_mitigation_integration.py`
  - `test_think_block_integration.py`
  - `test_world_loader_integration.py`
- **Moved documentation**: `README_INTEGRATION_TESTS.md` to integration directory
- **Clean separation**: Unit tests (60 files) vs Integration tests (30 files)

## Benefits

### 🚀 **Improved Test Execution**
- **GitHub export mode**: Now runs 60 unit tests only (down from 65)
- **Full mode**: Runs 90 total tests (60 unit + 30 integration)
- **Faster CI**: Cleaner separation allows for better test parallelization
- **Better organization**: Developers can easily distinguish test types

### 📈 **Enhanced Script Robustness**
- **Flexible integration test discovery**: Handles multiple directory structures
- **Clear status messages**: Users know exactly what tests are included/excluded
- **Backward compatibility**: Existing test execution patterns continue to work
- **Future-proof**: Easy to add more exclusion patterns
