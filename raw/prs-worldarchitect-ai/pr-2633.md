## Changes

### New Requirements Added
1. **Command Output with Exit Codes** - Assertions alone are not evidence; capture actual command output and exit codes
2. **Working Directory Anchor** - Include `pwd` before git commands to prove context
3. **CI Checks Tied to HEAD SHA** - Use `gh api .../check-runs?head_sha=...` instead of PR checks
4. **Server-Process Git Linkage** - Tie gunicorn PID to its working directory and verify git HEAD there
5. **Timestamp Synchronization** - Single-pass evidence collection for aligned timestamps
6. **Summary vs Raw Data Consistency** - Validate automated summaries against raw data

### New Anti-Patterns
- Assertions without command output
- Timestamp gaps breaking provenance chains  
- Summary/raw data mismatches

## Context
These lessons emerged from creating a bulletproof evidence package for PR #2588 (arc milestones fix). Multiple iterations (v1→v2→v3) identified gaps that are now codified as requirements.
