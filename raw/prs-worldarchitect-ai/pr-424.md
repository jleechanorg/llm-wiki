## Summary

This PR fixes the JSON display bug where the 'raw_response' field was being shown to users instead of the clean narrative text. The issue was caused by the frontend displaying a field that should have been internal-only.

## Key Changes

### 1. Removed raw_response dependency (MAJOR FIX)
- Removed the problematic 'raw_response' field from gemini_response.py that was causing the JSON display bug
- Changed frontend (static/app.js) to use 'response' field instead of 'raw_response'
- This ensures users see clean narrative text, not JSON structures

### 2. Refactored debug content stripping
- Moved debug content stripping functions from main.py to gemini_response.py where they belong
- Functions moved:
  - `_strip_debug_content` 
  - `_strip_state_updates_only`
  - `_strip_other_debug_content`
- Created new `get_narrative_text(debug_mode)` method in GeminiResponse class
- This centralizes all response text processing in the appropriate module

### 3. Fixed all 119 unit tests  
- Updated imports to use `gemini_response.GeminiResponse` instead of `main.StateHelper`
- Fixed mock objects to include the new `get_narrative_text` method
- Updated test expectations to match actual implementation
- All tests now passing

### 4. Added comprehensive logging
- Added JSON response detection logging in multiple places
- Added critical warnings when raw_response is accessed
- This helps track the data flow and catch any future issues

## Testing

- All 119 unit tests passing ✅
- Manual testing confirmed:
  - Users see clean narrative text
  - No JSON structures displayed
  - Debug mode still works correctly

## Files Changed

- **mvp_site/gemini_response.py**: Removed raw_response, added debug stripping methods
- **mvp_site/main.py**: Removed debug stripping methods (moved to gemini_response.py)

- **8 test files**: Updated imports and mocks to work with refactored code
