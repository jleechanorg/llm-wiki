> **Overview**
> Adds a new PR-only **merge-commit correctness gate** to `.github/workflows/test.yml` that explicitly checks out the default synthetic merge commit (base+head) to catch integration failures that won’t show up when other jobs test only the PR head SHA.
> 
> The gate installs minimal Python deps, validates imports of several core `mvp_site` modules, and runs a small smoke-test subset via `pytest`, failing the workflow with a targeted error if merge-commit validation breaks.
