## Summary
Excludes integration tests from GitHub Actions CI using a clean directory-based approach to prevent CI failures and improve test organization.

## Problem
- Integration tests fail in CI due to missing API credentials
- Tests that call real services (Gemini, Firestore) are non-deterministic
- API calls in CI can hit rate limits and incur costs
- Fork PRs don't have access to repository secrets
- Individual file exclusion lists are hard to maintain

## Solution
- **Directory-based exclusion**: Exclude entire `test_integration/` directory from CI
- Created script for running integration tests locally
- Removed API key secrets from workflow (no longer needed)
- Cleaned up duplicate test files and addressed review feedback
- Simplified maintenance with directory-level exclusion
