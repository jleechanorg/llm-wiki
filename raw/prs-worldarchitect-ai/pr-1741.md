## Summary
✅ **CRITICAL FIX**: Update claude_mcp.sh to install the working Perplexity MCP server package instead of the problematic one causing 401 authentication errors

**Problem Solved**:
- Original `server-perplexity-ask` package causes persistent 401 Unauthorized errors
- Users experience "total failure" with Perplexity MCP integration out of the box  
- Fresh installations via claude_mcp.sh get non-functional Perplexity servers

**Solution Implemented**:
- Replace `server-perplexity-ask` with working `@chatmcp/server-perplexity-ask` package
- Update server configuration for optimal compatibility
- Add comprehensive removal of old/new server names to prevent conflicts

## Technical Changes

### Package & Server Name Updates
```bash
# OLD (problematic):
claude mcp add --scope user "perplexity-search" "npx" "server-perplexity-ask"

# NEW (working):
claude mcp add --scope user "perplexity-ask" "npx" "@chatmcp/server-perplexity-ask"
```

### Enhanced Conflict Prevention
```bash
# Added removal for both old and new server names
claude mcp remove "perplexity-ask" >/dev/null 2>&1 || true
claude mcp remove "perplexity-search" >/dev/null 2>&1 || true
```

### Updated Variable References
- All `INSTALL_RESULTS["perplexity-search"]` → `INSTALL_RESULTS["perplexity-ask"]`
- Error logging updated to reference correct server name
- Maintains consistency throughout installation process

## Impact & Benefits

### For Fresh Installations
- ✅ **Working Perplexity Integration**: No more 401 authentication errors
- ✅ **Real-time Web Search**: AI-powered analysis functions immediately

### For Existing Users

- ✅ **Seamless Transition**: Users can re-run script to get working version

### For Development
