## Summary

Add new `/fake` slash command that uses command composition to detect fake, demo, or simulated code patterns in the codebase.

### Key Features
- **Command Composition**: Combines `/arch /thinku /devilsadvocate /diligent` for comprehensive analysis
- **Branch Comparison**: Analyzes local branch vs main and remote PR context
- **Pattern Detection**: Identifies fake implementations, placeholder comments, demo files, and duplicate protocols
- **Integration Verification**: Checks that code actually works with real systems

### Addresses Critical CLAUDE.md Rules
- **NO FAKE IMPLEMENTATIONS**: Prevents creation of non-functional demonstration code
- **ORCHESTRATION OVER DUPLICATION**: Identifies duplicate implementations of existing functionality
- **NO UNNECESSARY EXTERNAL APIS**: Flags code that simulates rather than implements

### Detection Targets
- Placeholder comments (`# Note: In the real implementation`)
- Demo files that don't actually work
- Python files simulating .md logic
- Duplicate systematic protocols
- Template responses without real analysis
- Mock implementations that fake functionality

### Command Usage
```bash
/fake
```

The command provides detailed reporting with:
- File locations (file:line format)
- Specific evidence of fake patterns
- Impact assessment
- Actionable remediation guidance

### Implementation Details
- **File**: `.claude/commands/fake.md`
- **Architecture**: Uses existing command composition system
- **Scope**: Comprehensive codebase analysis
- **Output**: Structured audit report with remediation steps
