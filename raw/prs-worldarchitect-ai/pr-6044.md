## Background

During the exportcommands export pipeline review (PR [jleechanorg/claude-commands#280](https://github.com/jleechanorg/claude-commands/pull/280)), two genuine bugs in source files were identified by automated reviewers (Greptile, Copilot) and confirmed as real defects worth fixing at the source.

## Goals

- Fix IndexError in `mock_gh_cli.py` when `--json` flag is the last argument
- Fix hardcoded home directory path in `recovery_coordinator.py`
- Pass pre-commit ruff checks (EXE001, RUF002, T201, ARG001, PLR2004)

## Tenets

- Fix root causes in source, not just in the export copy
- Minimal scope: only touch files with confirmed bugs
- No breaking changes to behavior

## High-level description of changes

**`automation/testing_util/mock_gh_cli.py`**
Guards against `IndexError` when `--json` is the last CLI argument (missing value). Previously `args[idx + 1]` would raise `IndexError`; now falls back to empty string. Also: replaced EN DASH characters in docstring with ASCII hyphens, added `# noqa` suppressions for T201/ARG001 (print-based CLI tool by design), defined `MIN_ARGS`/`MIN_SUBCOMMAND_ARGS` constants for PLR2004, made file executable.

**`orchestration/recovery_coordinator.py`**
Replaced hardcoded `~/projects/worldarchitect.ai/worktree_roadmap/agent_workspace_<name>` path with a portable `os.path.expanduser`-based fallback that respects a `PROJECT_BASE_DIR` environment variable. This makes the coordinator usable in any environment without modification.

- No functional tests changed; the mock_gh_cli fix is a defensive guard for malformed invocations
