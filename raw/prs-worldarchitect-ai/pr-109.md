## Problem
Integration tests were failing hard when environment dependencies weren't available:
- `test_live_firestore.py` crashed when Firebase credentials missing
- `test_game_state.py` serialization test failed due to new combat_state field
- Tests provided no graceful degradation for missing dependencies

## Solution
### Firebase Test Improvements
- **Environment Detection**: Check for `TESTING=true` environment variable
- **Credential Validation**: Verify `serviceAccountKey.json` exists before initialization  
- **Graceful Skipping**: Use `@unittest.skipIf` decorators with informative messages
- **Setup Protection**: Skip test setup when Firebase unavailable

### GameState Test Fix
- **Updated Expectations**: Include `combat_state: {"in_combat": false}` in serialization test
- **Maintains Compatibility**: Existing functionality preserved

## Test Results
**Before**: 
- Hard failures when Firebase credentials missing
- GameState serialization test failing
- Tests crash CI/CD environments

**After**:
- 123 tests total
- 3 gracefully skipped (Firebase when unavailable)
- 1 remaining failure (resource management - separate issue) 
- 2 remaining errors (investigation needed)

## Benefits
- 🎯 **CI/CD Friendly**: Tests run cleanly in environments without Firebase credentials
- 🎯 **Developer Experience**: Clear skip messages explain why tests don't run
- 🎯 **Backward Compatible**: When credentials available, tests run normally
- 🎯 **Robust Testing**: No more hard crashes due to missing dependencies
