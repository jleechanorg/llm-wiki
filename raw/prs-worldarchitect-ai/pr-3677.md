## Raw Body
## Problem

The get_github_token() function calls `gh auth token` via subprocess.run(), which triggers macOS permission prompts for "bash" accessing data from other applications.

## Solution

Replace subprocess call with direct file read from gh CLI config file:

- Reads token directly from ~/.config/gh/hosts.yml using PyYAML
- Eliminates all bash/subprocess calls for token retrieval
- Falls back gracefully if PyYAML not available or config file missing
- Still checks GITHUB_TOKEN environment variable first

## Changes

- Updated get_github_token() to read from config file instead of calling gh CLI
- Added optional yaml import (graceful fallback if not available)
- No subprocess calls - completely eliminates macOS permission prompts

## Benefits

- No bash/subprocess calls - No macOS permission prompts
- Faster - Direct file read instead of spawning process
- More reliable - No dependency on gh CLI being available
- Same functionality - Token retrieval works identically

## Testing

- Verified token retrieval works from config file
- Verified no subprocess calls to gh CLI
- Tested fallback behavior when config file missing
- All existing functionality preserved
