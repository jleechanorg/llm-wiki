## Summary
Fixes the equipment visibility bug where the LLM provides generic flavor text instead of listing actual player equipment. Implements the **InfoAgent Hybrid Approach** with deterministic backend extraction + LLM flavor.

## Problem
When players asked "What equipment do I have?", the LLM returned generic responses like "You have various items" instead of listing their actual items with stats.

## Solution: InfoAgent Hybrid Architecture

### 1. Backend Deterministic Extraction (`equipment_display`)
- **New function**: `extract_equipment_display()` in `llm_service.py`
- Extracts equipment from `game_state` with 100% accuracy
- Returns structured data: `[{slot, item, stats}, ...]`
- Passed to frontend via response JSON

### 2. String ID Entity System (`item_registry`)
- **New attribute**: `item_registry` in `GameState` class
- Maps item_id strings → full item definitions
- Equipment slots store string IDs instead of inline descriptions
- Enables consistent item lookup across the codebase

### 3. Equipment Query Detection
- **New function**: `is_equipment_query()` in `llm_service.py`
- Detects equipment-related queries (inventory, gear, weapons, etc.)
- Triggers `equipment_display` extraction when relevant

### 4. Frontend Rendering (NEW)
- **Updated**: `app.js` - `generateStructuredFieldsPreNarrative()`
- Renders `equipment_display` field with light blue background
- Shows slot, item name, and stats for each equipped item
- Uses deterministic backend data - does NOT rely on LLM narrative
