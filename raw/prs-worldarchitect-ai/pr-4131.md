## Summary
Enhanced rate limiting API responses to include reset time information, enabling the frontend to display countdown timers for rate limit recovery. Also added `test@example.com` to the rate limit exemption list for testing purposes.

## Changes
- Modified `handle_interaction()` in `main.py` to conditionally include `reset_time_daily` and `reset_time_hourly` fields in the 429 rate limit response when available
- Refactored rate limit error response construction to build response data separately before returning, improving code readability
- Added `test@example.com` to `RATE_LIMIT_EXEMPT_EMAILS` set in `rate_limiting.py` as a default test account exemption

## Benefits
- Provides frontend with precise reset time information to display accurate countdown timers to rate-limited users
- Improves user experience by showing when rate limits will be lifted
- Maintains backwards compatibility by only including reset time fields when present in the rate limit result
- Enables easier testing with a dedicated test account exempt from rate limiting
