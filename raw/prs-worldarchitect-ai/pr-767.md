## Summary

This PR enhances the `/roadmap` command to support multi-task parallel processing with integrated /plan /handoff /think workflow and subagent coordination.

## Key Features

- **Multi-task input parsing**: Support for comma-separated tasks or multiple arguments
- **Integrated workflow**: Automatically executes /plan /handoff /think sequence for each task
- **Parallel subagent spawning**: Uses /handoff to spawn parallel subagents via /orchestrate
- **Isolated workspaces**: Creates work branches in parallel with isolated worktrees
- **Progress tracking**: Updates roadmap.md with parallel task progress automatically

## Implementation

- Updated `.claude/commands/roadmap.md` documentation
- Created `claude_command_scripts/roadmap.sh` implementation script
- Created `.claude/commands/roadmap_helper.py` Python helper for workflow integration

## Usage Examples

```bash
# Single task (traditional mode)
/roadmap implement authentication

# Multiple tasks (comma-separated)
/roadmap implement auth system, create API docs, add unit tests

# Multiple tasks (separate arguments)
/roadmap "implement auth system" "create API docs" "add unit tests"
```
