## Summary
Updated documentation to emphasize proactive bead creation as the canonical issue tracking mechanism. Agents should now create beads immediately when discovering bugs, regressions, test failures, or architectural issues during their work, rather than only logging or reporting them.

## Changes
- Added "Proactive Bead Creation" section to File Tracking guidelines in `.claude/agents/CLAUDE.md` explaining that beads should be created for any discovered issues worth mentioning
- Added "Bead Lifecycle" section documenting how to update and close beads as work progresses
- Updated "Beads tracking" rule in `CLAUDE.md` to explicitly call out proactive bead creation with `bd create` and `bd update` commands
- Clarified that `.beads/` diffs must always be included in commits

## Benefits
- Establishes beads as the single source of truth for issue tracking rather than scattered logs or comments
- Encourages immediate issue capture during development, reducing context loss
- Provides clear guidance on bead lifecycle management (creation → progress updates → closure)
- Ensures all discovered issues are properly tracked and visible in version control
