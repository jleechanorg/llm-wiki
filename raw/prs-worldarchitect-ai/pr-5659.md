# PR #5659 Description (updated)

## Summary

Set `jleechantest@gmail.com` as the default test user in testing_mcp and testing_ui instead of relying on prod code defaults (e.g. `test@example.com`).

## Production Code Changes

- **rate_limiting.py**: Add `jleechantest@gmail.com` to `RATE_LIMIT_EXEMPT_EMAILS` when `TESTING_AUTH_BYPASS=true` (same guard as `test@example.com`). Ensures MCP/UI tests using this default don't hit rate limits.

## Test Infrastructure Changes

- **testing_mcp/lib/campaign_utils.py**: Added `DEFAULT_TEST_EMAIL` constant and updated `collect_route_stream_events` and `post_streaming_request` to use it when `user_email` is `None`. Always sends `X-Test-User-Email` header.
- **testing_ui/config.py**: Added `DEFAULT_TEST_EMAIL` constant.
- **testing_ui/lib/browser_test_base.py**: Default `test_user_email` now uses `DEFAULT_TEST_EMAIL` (was `test@example.com`). JS fallback updated.
- **testing_ui/lib/byok_browser_base.py**: Same default for `BYOK_TEST_USER_EMAIL` env fallback.
- **testing_ui/browser_test_base.py** (shim): Minor whitespace cleanup.

## Unit Tests Added

- **mvp_site/tests/test_campaign_utils_default_email.py**: Verifies campaign_utils uses default when `user_email=None` and respects explicit email.
- **mvp_site/tests/test_rate_limiting.py**: Added `test_jleechantest_exempt_when_in_set`.
- **mvp_site/tests/test_testing_ui_default_email.py**: Verifies BrowserTestBase and ByokBrowserTestBase use jleechantest default and respect env overrides.

## Known Limitations

- Rate limiting tests (`test_rate_limiting_real_e2e.py`) still use `test@example.com` intentionally to test exempt user behavior.
