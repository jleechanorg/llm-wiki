## Summary

This PR transforms the `/copilotsuper` command to use command composition, eliminating unnecessary code duplication and following established patterns. Additionally includes orchestration system cleanup.

## Changes

### 🔄 Architecture Evolution

1. **Initial Implementation** (first 2 commits):
   - Transformed from sequential to parallel orchestration
   - Added dependency checking and robustness improvements
   - But violated "ORCHESTRATION OVER DUPLICATION" principle

2. **Final Architecture** (latest commits):
   - **Deleted 310-line shell script** that duplicated orchestration logic
   - Transformed to pure .md-only implementation using command composition
   - Added natural language PR filtering support
   - Created `/copilots` alias for easier access
   - Added agent verification step
   - **Cleaned up opus-master placeholder** from orchestration system

### 📝 Key Changes

1. **Removed Duplication**:
   - ❌ Deleted `claude_command_scripts/commands/copilotsuper.sh` 
   - No more reimplementing spawn_agent_for_pr(), monitor_agents(), etc.
   
2. **Command Composition**:
   - The .md file now directly calls `/orch` for each PR
   - Leverages existing orchestration infrastructure
   - Trusts Claude's natural language understanding

3. **Natural Language Support**:
   - `/copilots recent 5` - Last 5 PRs
   - `/copilots all PRs except agent ones in last 2 days`
   - `/copilots open PRs from last 24 hours`

4. **Agent Verification**:
   - Waits for agents to initialize
   - Verifies each agent is running in tmux
   - Reports count of confirmed agents

5. **Orchestration Cleanup**:
   - Removed legacy opus-master placeholder session
   - Updated orchestration detection to use proper system components
   - Cleaner, more robust orchestration system

- **Simplicity**: From 310 lines to ~30 lines of instructions
- **No Duplication**: Uses existing `/orch` command
