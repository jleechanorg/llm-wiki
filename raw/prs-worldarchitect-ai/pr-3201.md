## Raw Body
## Summary
- Remove slim mode from frontend and backend that was stripping user-visible UI fields on mobile
- Mobile users were seeing "Resources: None" and missing session_header, planning_block, dice_rolls, etc.

## Root Cause
PR #3144 introduced "slim mode" for mobile as a payload optimization, but it stripped ALL structured fields except basic text/actor/timestamp. This was never requested and broke the mobile UI.

## Fix
- Remove `slim` query parameter handling from backend (`main.py`)
- Remove slim mode detection from frontend (`app.js`)
- Keep pagination (100-300 entry limit) which is sufficient to manage payload sizes
