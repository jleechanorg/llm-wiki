## Problem
Addresses narrative context lag where LLM might prioritize historical context over current action. This is especially problematic for god mode commands where immediate response is expected.

**Root Cause Identified:** The original `priority_instruction` only told the LLM to ignore `story_history`, but NOT `game_state`. When `game_state` contained content from previous turns (e.g., "Silver Aegis Soul Trophy"), the LLM would elaborate on that instead of answering the user's question.

## Solution
- Implement structured prompt format that explicitly separates `USER_ACTION` from both `STORY_HISTORY` and `GAME_STATE`
- Guide the LLM to focus on the current action first
- Prevents LLM from responding to previous commands or game_state content instead of the current question

## Bug Reproduction Evidence

**Test case:** User asked "can i enthrall raiders or will it give me hp penalty?" but LLM responded about "Silver Aegis Soul Trophy" from previous turn.

| Test | Response |
|------|----------|
| Without fix | "Soul Trophy Audit: Kith'rak Voss... Silver Aegis..." (wrong - focused on game_state) |
| With fix | "Can you enthrall the Raiders? Yes... Will it give you an HP penalty?..." (correct) |
