## Raw Body
Root Cause:
- Dockerfile expects build context to be project root (to access both mvp_site/ and infrastructure/ directories)
- Build was being submitted from mvp_site directory, causing: "COPY mvp_site/requirements.txt" to fail (file not found) "COPY infrastructure/" to fail (directory not found)

Solution:
1. Modified deploy_common.sh:
   - Added dockerfile parameter to submit_build function
   - Use inline cloudbuild.yaml config to specify dockerfile path
   - Build context is now project root with explicit dockerfile location

2. Modified deploy.sh:
   - Set BUILD_CONTEXT to project root (pwd) instead of mvp_site
   - Pass DOCKERFILE_PATH to submit_build
   - Simplified world directory copying logic (no more temp context)

Benefits:
- Dockerfile can access both mvp_site/ and infrastructure/ directories
- No need for temporary build context
- Cleaner, more maintainable build process

Fixes: Cloud Build error "COPY mvp_site/requirements.txt: file does not exist"
