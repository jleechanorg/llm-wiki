## Summary
- Switches beads backend from legacy SQLite (`beads.db`) to Dolt
- Canonicalizes `issues.jsonl` to REV-prefix-only format (268 issues, was 596 mixed-prefix)
- Enables `no-daemon` mode to prevent worktree branch commit conflicts

## Production Code Changes
- **before**: beads used legacy SQLite (`beads.db`, 2.6MB) with mixed-prefix JSONL (BD-, DICE-, WA-, worktree_*, REV-)
- **now**: beads uses Dolt backend (`beads_REV` database) with 268 REV-only issues in JSONL
- **why**: SQLite is not a supported backend in beads v0.55+; Dolt is the canonical store; mixed-prefix JSONL caused import failures

## Test Changes
None — beads configuration only.

## Known Limitations
- `beads.db.bak` left in `.beads/` (gitignored) as a safety backup; can be deleted manually
- Legacy non-REV issues (BD-, DICE-, WA-) are archived in `/tmp/issues_mixed_backup.jsonl` locally
