## Problem

**Observed Issue:** Campaign ARrfJ39LhNEi5rcGq1c7 skipped character creation and started immediately without a character being created.

**Root Cause:** Semantic intent classifier (Priority 5 in routing) can override CharacterCreationAgent state check (Priority 4). When `character_creation_in_progress == true` and user says "I want to talk to someone", the classifier routes to DialogAgent instead of staying in CharacterCreationAgent.

**Current Routing Priority (Buggy):**
```
Priority 1: GodModeAgent
Priority 2: Character Creation Completion
Priority 3: CampaignUpgradeAgent
Priority 4: CharacterCreationAgent ← Can be bypassed
Priority 5: Semantic Classification ← Overrides P4
Priority 6: API Explicit Mode
Priority 7: StoryModeAgent
```

## Solution

**Modal Lock Pattern:** Add Priority 2 check that returns CharacterCreationAgent when `character_creation_in_progress == true`, bypassing ALL other routing including semantic classification.

**New Routing Priority (Fixed):**
```
Priority 1: GodModeAgent
Priority 2: MODAL LOCK (new) ← Blocks all other routing
Priority 3: Character Creation Completion
Priority 4: CampaignUpgradeAgent
Priority 5: CharacterCreationAgent
Priority 6: Semantic Classification (skipped during modal lock)
Priority 7: API Explicit Mode
Priority 8: StoryModeAgent
```
