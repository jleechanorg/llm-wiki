## Raw Body
### Motivation
- Prevent false negatives where dice are recorded in `action_resolution.mechanics.audit_events` but not detected by fabrication checks. 
- Clarify schema-violation messaging to point to the documented `dice_rolls` guidance instead of a brittle line-number reference. 

### Description
- Updated `_detect_narrative_dice_fabrication()` to treat non-empty `action_resolution.mechanics.audit_events` as a valid structured-dice source in addition to `action_resolution.mechanics.rolls`. 
- Kept legacy detection (`dice_rolls` / `dice_audit_events`) as a schema-violation path and updated the warning text to reference the `dice_rolls` documentation section instead of a fixed line number. 
- Added unit tests in `mvp_site/tests/test_dice_integrity_helpers.py` that cover: (1) primary `action_resolution.mechanics.rolls`, (2) `action_resolution.mechanics.audit_events`, (3) legacy-only fields, and (4) mixed primary+legacy cases. 

### Testing
- Ran the new/updated unit tests with `python -m pytest mvp_site/tests/test_dice_integrity_helpers.py`, and all tests passed (`19 passed`). 
- Existing dice-integrity helper tests were exercised to verify no regressions in `dice_integrity` helper behavior.
