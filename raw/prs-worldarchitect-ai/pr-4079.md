## Raw Body
## Summary
- Enables `preload_app=True` in Gunicorn to share memory across workers via copy-on-write
- Reduces memory usage by ~1GB (embedding model loaded once instead of 3x)

## Problem
Workers were each loading the embedding model (`BAAI/bge-small-en-v1.5`, ~300-500MB) independently:
- 3 workers × 300-500MB = ~900-1500MB just for the model
- Plus Flask/Python overhead = ~1.5-2GB total
- Exceeded 2Gi limit, causing OOM errors

## Solution
With `preload_app=True`:
1. Master process loads app (including embedding model) before forking
2. Workers share memory via copy-on-write
3. Model exists in memory only once (~300-500MB instead of ~1.5GB)
