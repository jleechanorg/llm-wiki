## Summary

This PR adds real Gemini API integration to the Milestone 0.4 test framework, enabling validation with actual LLM responses.

### What's New
- Complete test framework for real API testing (Steps 13-17 from roadmap)
- Cost tracking with budget enforcement ($1.00 limit)
- Test runners for all three approaches with real API
- Dry run mode for testing without costs
- Comparison reports for real vs mock results

### Key Components

#### Configuration & Setup
- `config.py` - API settings, model selection, budget limits
- `setup_real_llm.sh` - Dependency installation script
- Budget: $1.00 hard limit with warnings at $0.50

#### API Integration
- `gemini_client.py` - Complete API wrapper with:
  - Cost tracking per call
  - Retry logic with exponential backoff
  - Token counting and usage reporting
  - Structured response parsing (JSON/XML)
  - Mock fallback for environments without SDK

#### Test Framework
- `test_structured_generation_real.py` - Enhanced harness with real API support
- System instructions tailored per approach
- Budget-aware execution (stops at 90% budget)
- User confirmation before API calls

#### Test Runners
- `run_real_baseline_tests.py` - Validation-only approach
- `run_real_pydantic_tests.py` - Structured generation
- `run_real_combined_tests.py` - Combined approach
- `run_real_comparison.py` - All three with comparison
- `run_real_dry_run.py` - Demo without API calls

### Test Configuration
- Campaigns: sariel_v2_001, thornwood_001
- Scenarios: multi_character, combat_injured, npc_heavy
- Total: 18 API calls (~$0.01 estimated cost)

### Usage Instructions

1. Install dependencies:
   ```bash
   cd prototype/tests/milestone_0.4
   ./setup_real_llm.sh
   ```

2. Set API key:
   ```bash
   export GEMINI_API_KEY='your-key-here'
   ```

3. Run tests:
   ```bash
   # Dry run (no API calls)
   python run_real_dry_run.py
   
   # Full comparison
   python run_real_comparison.py
   ```
