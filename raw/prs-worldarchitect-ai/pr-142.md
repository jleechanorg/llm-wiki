## Summary
Simple fix for FileNotFoundError when deploying to Google Cloud Run - just copy the world directory into mvp_site before building.

## Problem
The application was failing in production with:
```
FileNotFoundError: [Errno 2] No such file or directory: '/app/../world/celestial_wars_alexiel_book.md'
```

## Solution
One-line fix in deploy.sh:
```bash
cp -r world "$TARGET_DIR/"
```

This copies the world directory into mvp_site before the Docker build, so the files are available at runtime.

- `deploy.sh` - Added 4 lines to copy world/ into mvp_site/ before build
