## Learning Documentation: Branch vs Website Misconception

### Problem Identified
Common user confusion: "I switched branches and CI is passing, but the website looks exactly the same"

### Root Cause
**Misconception**: Branch switching → Immediate website changes visible
**Reality**: Many branches contain non-UI improvements (tooling, CI, backend)

### Learning Added to CLAUDE.md

**New Section**: "Website Testing & Deployment Expectations" in Development Guidelines

**Key Protocol**: 5-step process for "website looks same" scenarios:
1. Check PR description - tooling vs UI changes?
2. Ask what URL user is viewing (local vs production)
3. Verify change type (user-facing vs developer tooling)
4. For UI: Hard refresh + local development server
5. Explain deployment vs branch switching concepts

### Specific Case
- **PR #583**: "push-pr-coherence" = Developer tooling enhancement
- **Contains**: Push script improvements, PR coherence checking
- **Expected Result**: No website changes (backend workflow improvement)
- **User Confusion**: Expected visual changes after branch switch

### Prevention Value
- **HIGH**: Prevents repeated "website unchanged" confusion
- **Scope**: Applies to CI/tooling/backend changes across project
- **Pattern**: Common misconception for developers new to deployment concepts

### Implementation
- **Critical Rule**: Branch ≠ Website deployment
- **Protocol**: Systematic troubleshooting for website testing
- **Examples**: Common non-visual change types documented

This learning prevents the "website looks the same" pattern from recurring.
