## Summary

Enhanced the existing Claude backup system with parameter support, device-specific naming, and macOS compatibility to prevent backup collisions across multiple devices.

## Key Features

### 🔧 Enhanced Backup Functionality
- **Parameter support**: Accept custom destination paths via command line
- **Device-specific naming**: Backup folders include device suffix (e.g., `claude_backup_jeffreys-mbp`)
- **Collision prevention**: Multiple devices can safely backup to the same location
- **macOS compatibility**: Updated from WSL paths to native macOS CloudStorage paths

### 📁 Backup Locations
- **Default**: `~/Library/CloudStorage/Dropbox/claude_backup_{devicename}`
- **Custom**: Any destination + device suffix automatically appended
- **Smart parameter handling**: Detects flags vs destination paths

### ⏰ Optimized Scheduling  
- **4-hour intervals**: Changed from hourly to every 4 hours to reduce system load
- **Built-in cron management**: `--setup-cron` and `--remove-cron` with destination support
- **Environment integration**: Loads email credentials from .bashrc automatically

### 🎯 Maintained Features
- ✅ Selective sync (settings, projects, local, hooks only)
- ✅ Email failure alerts with comprehensive reporting
- ✅ Detailed logging with timestamps
- ✅ Error handling and validation
- ✅ Safe rsync operations (no --delete flag)
