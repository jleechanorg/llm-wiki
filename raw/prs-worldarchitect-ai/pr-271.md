## Changes
- **New Rule #15**: Pull Request Workflow for All Changes
- Mandates PRs for ALL changes including:
  - Code changes
  - Documentation updates  
  - Configuration modifications
  - **Roadmap changes** (scheduling, task updates, etc.)

## Workflow Process
1. Make changes on feature/dev branch
2. Commit changes with descriptive messages
3. Create pull request using `gh pr create`
4. Review and merge PR (do not merge directly to main)
5. Run `./integrate.sh` to create fresh clean branch

## Benefits
- ✅ Better visibility into all changes
- ✅ Proper review history and change tracking
- ✅ Prevents accidental direct pushes to main
- ✅ Consistent workflow for all modifications
- ✅ No exceptions - applies to everything
