## Summary

Introduces the **Ralph** autonomous workflow toolkit: a PRD-driven development automation suite for running and monitoring iterative tasks (e.g., test implementation, feature work) via `amp` or `claude` with structured progress tracking.

**Source:** Sourced and generalized from [jleechanorg/worldai_claw#25](https://github.com/jleechanorg/worldai_claw/pull/25). Phase breakdown and file-activity logic are now derived from the PRD (no project-specific hardcoding). macOS compatibility fix for `find` (removed GNU `-printf`).

## Changes

- **Dashboard** (`dashboard.html`, `dashboard_server.sh`) – Local web UI with auto-refresh, phase breakdown, recent commits, and `ralph.sh` run-status detection
- **Runner** (`ralph.sh`) – Iterative loop over PRD stories; runs `amp` or `claude` with `ralph/CLAUDE.md`; archives runs when `branchName` changes (via `.last-branch`)
- **CLI status** (`status.sh`) – Terminal view of progress with watch mode for real-time monitoring
- **Test planning** (`prd.json`) – Example PRD with Ralph toolkit stories and Amazon clone benchmark (customize for your project)
- **Docs** (`progress.txt`, `CLAUDE.md`) – Progress tracking and Ralph-specific instructions

## Benefits

- Enables autonomous, PRD-driven development cycles with clear progress visibility
- Reduces manual coordination for multi-step tasks
- Provides dashboard and CLI options for different workflows

## Risk / Scope

**Low risk.** Adds new dev/ops scripts and a local dashboard that read repo state and run subprocesses. Does not modify application or runtime code. Dependencies: `jq`, `python`, `lsof`, `pgrep` assumed available locally.
