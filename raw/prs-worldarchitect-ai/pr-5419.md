## Summary

This PR completes the non-`GodMode` prompt-optimization carry-forward by hardening the prompt-footprint measurement tooling and test harness used to drive future, behavior-changing token-reduction work.

## Goals / Tenets

- Keep this PR scoped to **measurement, observability, and non-behavioral refactoring**.
- Do **not** include God Mode-specific work.
- Keep behavior changes in future PRs so token-saving changes are measurable and reversible.
- Improve reliability/perf of test utilities by reducing heavy imports at package load time.

## What This PR Does

1) Prompt measurement utility hardening (`testing_mcp/lib/prompt_file_utils.py`)
- Keeps deterministic prompt footprint helpers for agent prompt files and totals.
- Converts module-level `mvp_site` imports to lazy resolution to avoid import-time cost.
- Adds optional dice inclusion:
  - `include_dice`
  - `dice_roll_strategy` (`native_two_phase` -> `dice`, `code_execution` -> `dice_code_execution`)
- Preserves `REQUIRED_PROMPT_ORDER`-based resolution behavior while allowing optional runtime-aligned dice prompt accounting.

2) Lazy export surface (`testing_mcp/lib/__init__.py`)
- Removes eager re-exporting of prompt utility helpers that triggered heavy imports.
- Uses PEP 562 `__getattr__` for lazy loading of prompt utility helpers.

3) MCP test harness alignment (`testing_mcp/test_prompt_file_utils.py`)
- Refactors from ad-hoc pytest functions into `MCPTestBase` execution with scenario-based reporting.
- Adds a dice-inclusion scenario and guards around list indexing.
- Sets `HAS_LLM_CALLS = False` and `REQUIRE_FULL_TRACE_LOGS = False` for this non-LLM utility test.

4) Scope documents updated (`roadmap/agent-prompt-optimization-implementation.md`)
- Carries forward the work plan with explicit status markers.
- Excludes prompt-order reductions and God Mode from this branch scope.
