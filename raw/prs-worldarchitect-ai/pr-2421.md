## Raw Body
Root cause: test_concurrent_operations_execute_in_parallel was using patch.object() with MagicMock() inside ExitStack which had issues properly intercepting llm_service.continue_story calls when used with asyncio.to_thread().

Changes:
- Switch both tests to use @patch decorator pattern (proven to work)
- Use mvp_site.world_logic.llm_service path for reliable patching
- Fix mock response objects to include all required attributes (get_state_updates.return_value, get_location_confirmed, dice_rolls)
- Add proper mock for LLMRequestError exception class
- Remove unused ExitStack import

The decorator pattern patches at module level before test runs, ensuring consistent behavior with asyncio.to_thread() callable capture.
