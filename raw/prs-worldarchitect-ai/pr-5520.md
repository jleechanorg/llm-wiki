## Root Cause

**Context object API mismatch** between job-level condition and JavaScript code:

- **Job-level condition** (line 196): Uses `github.event.issue.pull_request` ✅ Correct
- **JavaScript code** (line 232): Uses `context.issue.pull_request` ❌ Bug - always undefined
- **Should use**: `context.payload.issue.pull_request` ✅ Fix

In the `github-script` action:
- `context.issue` is a convenience helper containing only `{owner, repo, number}`
- `context.payload.issue` contains the full webhook payload including `pull_request` property

## Changes

**File: `.github/workflows/mcp-smoke-tests.yml`**

- Line 232: `context.issue?.pull_request` → `context.payload.issue?.pull_request`
- Line 236: `context.issue.number` → `context.payload.issue.number`

This aligns with the existing pattern used for `workflow_run` events (line 238: `context.payload.workflow_run`).

**After:** Workflow correctly identifies PR from issue comment and proceeds with smoke tests.
