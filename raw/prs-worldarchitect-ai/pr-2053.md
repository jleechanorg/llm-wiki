### 🐛 Critical Fix: Dev Deploy Image Tag Mismatch
**Root Cause:**
- `gcloud builds submit` with multiple `--tag` flags only creates the LAST tag
- Build created: `gcr.io/worldarchitecture-ai/mvp-site-app:dev-latest` only
- Deploy expected: `gcr.io/worldarchitecture-ai/mvp-site-app:dev-ce369f2`
- Result: "Image not found" error on every deployment (5+ consecutive failures)

**Solution:**
1. Build with primary commit SHA tag first
2. Wait for build completion
3. Add "latest" tag AFTER using `gcloud container images add-tag`
4. Both tags now exist on same image

**File Changed:**
- `scripts/deploy_common.sh`: Fixed `submit_build()` function
