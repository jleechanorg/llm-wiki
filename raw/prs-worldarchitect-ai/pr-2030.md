## Summary

Add `$HOME/.claude/hooks` as a universal fallback location for all Claude Code hooks, enabling portability across projects without requiring symlinks.

## Changes

**Updated:** `.claude/hooks/hooks.plugin.json`

All 11 hook commands now search in this order:
1. `$PROJECT/.claude/hooks/HOOK_NAME` (project-specific)
2. `$PLUGIN/.claude/hooks/HOOK_NAME` (plugin overrides)
3. **`$HOME/.claude/hooks/HOOK_NAME` ← NEW FALLBACK**

**Hooks Updated:**
- **PreToolUse**: `pre_command_optimize.py`, `context_monitor.py`, `pre_creation_blocker.sh`, `check_root_files.sh`
- **PostToolUse**: `detect_speculation_and_fake_code.sh`, `command_output_trimmer.py`, `auto_fix_placement.sh`, `post_file_creation_validator.sh`, `smart_fake_code_detection.sh`, `python_async_lint.py`
- **UserPromptSubmit**: `compose-commands.sh`
- **Stop**: `slack_notify.sh`

## Benefits

- ✅ **No symlinks required** - Hooks work across all projects automatically
- ✅ **Project override capability** - Projects can still provide local hooks
- ✅ **Clean git history** - No hook-related commits needed in projects
- ✅ **Portability** - Single hook source can serve multiple projects
- ✅ **Maintainability** - Update hooks in one place, benefit everywhere
