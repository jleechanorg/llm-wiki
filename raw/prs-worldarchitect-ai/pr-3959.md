## Summary

Add comprehensive logging for Gemini API usage metrics to verify and monitor implicit caching effectiveness.

## Problem

Gemini 2.5 models have **automatic implicit caching** (75% discount on cached tokens) enabled by default since May 2025, but we currently have:
- ❌ No verification that caching is working
- ❌ No tracking of `cached_content_token_count` in responses
- ❌ No visibility into cost savings
- ❌ Unknown cache hit rates

## Solution

Added usage metadata logging in `mvp_site/llm_service.py` that tracks:
- `prompt_token_count` - Total input tokens
- `cached_content_token_count` - Tokens served from cache (implicit caching indicator)
- `candidates_token_count` - Response/output tokens
- `cache_hit_rate` - Calculated percentage of cached vs total prompt tokens

## Changes

**File**: `mvp_site/llm_service.py` (lines 1217-1234)

```python
# Log usage metadata for implicit caching verification (Gemini only)
if current_provider == constants.LLM_PROVIDER_GEMINI and hasattr(response, 'usage_metadata'):
    usage = response.usage_metadata
    prompt_tokens = getattr(usage, 'prompt_token_count', 0)
    cached_tokens = getattr(usage, 'cached_content_token_count', 0)
    response_tokens = getattr(usage, 'candidates_token_count', 0)

    # Calculate cache hit rate
    cache_hit_rate = (100 * cached_tokens / prompt_tokens) if prompt_tokens > 0 else 0

    logging_util.info(
        f"🔍 GEMINI_USAGE: "
        f"prompt_tokens={prompt_tokens}, "
        f"cached_tokens={cached_tokens}, "
        f"response_tokens={response_tokens}, "
        f"cache_hit_rate={cache_hit_rate:.1f}%, "
        f"model={current_model}"
    )
```
