## Summary
- **Fixed:** `get_agent_for_input()` now honors `mode='god'` parameter from UI, not just "GOD MODE:" text prefix
- **Root Cause:** Agent selection only checked text prefix, ignoring mode parameter passed from UI dropdown
- **Impact:** Users switching to god mode via UI now correctly route to `GodModeAgent`

## Bug Details
When users switched to god mode via the UI dropdown (sending `mode='god'` parameter), their commands were incorrectly routed to `StoryModeAgent` because `get_agent_for_input()` only checked for "GOD MODE:" text prefix.

This caused:
- Story mode prompts loaded instead of `god_mode_instruction.md`
- LLM responses confused (mixed story/god mode behavior)
- God mode directives not being saved (0 in Firestore)
