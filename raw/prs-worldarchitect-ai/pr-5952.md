## Summary
Fixes a critical bug where `.mjs` scripts (auth-cli.mjs, auth-aiuniverse.mjs) were being silently omitted from exports, and corrects the directory paths for secondo command scripts that were incorrectly referenced from the wrong location.

## Changes
- **exportcommands.py**: 
  - Added `*.mjs` glob pattern to `_export_claude_scripts()` to include JavaScript module files in exports
  - Removed `auth-cli.mjs` and `secondo-cli.sh` from `secondo_scripts` list (they live in `.claude/scripts/`, not `scripts/`)
  - Updated comment to clarify that these files are exported by `_export_claude_scripts` instead
  - Kept only `test_secondo_pr.sh` in `secondo_scripts` since it actually exists in `scripts/`

- **test_exportcommands.py**:
  - Added comprehensive `TestExportScriptIntegrity` test class with 5 new tests:
    - Verifies `auth-cli.mjs` exists in `.claude/scripts/`
    - Verifies `secondo-cli.sh` exists in `.claude/scripts/`
    - Validates that `_export_claude_scripts` includes `*.mjs` glob pattern
    - Confirms scripts listed in `secondo_scripts` actually exist in `scripts/`
    - Ensures `auth-cli.mjs` and `secondo-cli.sh` are NOT in the `secondo_scripts` list (regression guard)

## Benefits
- Prevents silent omission of `.mjs` files from exported claude-commands repository
- Catches future bugs where scripts are moved, renamed, or never committed
- Clarifies the correct directory structure for script exports
- Provides regression guards against similar path mismatch issues

## Testing
- Added 5 new unit tests that verify script existence and export configuration
- Tests catch the specific class of bug where export code references scripts in wrong directories
- All new tests pass and provide clear error messages for future regressions
