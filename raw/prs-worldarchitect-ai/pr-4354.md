## Production Code Changes

### 1. `scripts/ci-detect-changes.sh` (+1/-1 lines)
**Purpose:** Add `tests/` to core test group for comprehensive test coverage

- **Changed:** `[core]="mvp_site"` → `[core]="mvp_site,tests"`
- **Why:** Root-level `tests/` directory contains cross-cutting tests for utilities, scripts, and PyPI packages
- **Effect:** Changes to any unmapped directory (defaults to core) now run both mvp_site/ and tests/ suites

## Test Changes

**Before this fix:**
- Changes to `moltbook_poster/` → triggers "core" group → only runs `mvp_site/` tests
- `tests/test_moltbook_poster.py` NEVER ran in CI ❌

**After this fix:**
- Changes to `moltbook_poster/` → triggers "core" group → runs `mvp_site/` AND `tests/` tests
- `tests/test_moltbook_poster.py` runs automatically ✅

**Verification:**
```bash
$ echo "moltbook_poster/moltbook_poster/poster.py" | bash scripts/ci-detect-changes.sh push "" "" "simple" | grep test-dirs
test-dirs=mvp_site,tests
```

**Impact on CI runtime:**
- `tests/` directory has only 5 lightweight unit tests
- Expected increase: <10 seconds per CI run
- Benefit: Comprehensive test coverage for all utility code
