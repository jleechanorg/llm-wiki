## Summary

- **Preserved**: `scripts/install_mcp_servers.sh` (canonical version)

## Rationale

### Architectural Decision
MCP installation scripts belong in `scripts/` directory because they are:
- ✅ General project utilities (not Claude Code-specific)
- ✅ Portable scripts meant to be copied to other projects
- ✅ Used by multiple products (Claude, Codex, Cursor)
- ✅ Co-located with their caller (`install_mcp_servers.sh` sources `mcp_common.sh`)

`.claude/scripts/` should contain only Claude Code framework-specific utilities.

### Export Handling
Both export commands properly handle consolidation:
- **`/exportcommands`** (exportcommands.py): Exports MCP scripts from `scripts/` subdirectory to `.claude/scripts/` in bundle
- **`/localexportcommands`** (localexportcommands.md): Consolidates `scripts/` to `.claude/scripts/` during local export

## Benefits
- ✅ Eliminates duplicate files (2,706 lines of duplicate code removed)
- ✅ Single source of truth in `scripts/` directory
- ✅ Clearer separation: project utilities vs. Claude Code framework
- ✅ Export commands handle consolidation automatically
