## Problem

The MCP smoke test workflow (#2050) has two critical issues preventing proper operation:

### Issue 1: Upload Artifact Permission Error
**Error:** `EACCES: permission denied, scandir '/tmp/snap-private-tmp'`

**Cause:** The workflow used glob pattern `/tmp/**/smoke_tests/*.log` which recursively scans ALL subdirectories under `/tmp`, including the restricted `/tmp/snap-private-tmp` directory on Ubuntu GitHub Actions runners.

**Evidence:** PR #2058 workflow run failed at upload-artifact step with permission error.

### Issue 2: Missing Failure Comments
**Error:** No PR comment posted when workflow fails

**Cause:** Failure comment condition checked `steps.smoke_tests.outcome == 'failure'`, which doesn't trigger when:
- Smoke tests pass but artifact upload fails
- Other workflow steps fail
- Job fails for infrastructure reasons

**Evidence:** PR #2058 smoke tests passed but workflow failed - no comment posted.

## Solution

### Fix 1: Specific Upload Paths
```yaml
# Before (BROKEN):
path: /tmp/**/smoke_tests/*.log

# After (FIXED):
path: /tmp/worldarchitect.ai/*/smoke_tests/*.log
```

**Why this works:**
- Targets only our specific log directory structure
- Avoids scanning restricted snap directories
- Matches actual log path from smoke test script

### Fix 2: Comprehensive Failure Detection
```yaml
# Before (INCOMPLETE):
if: github.event_name == 'issue_comment' && steps.smoke_tests.outcome == 'failure'

# After (COMPREHENSIVE):
if: github.event_name == 'issue_comment' && failure()
```

**Why this works:**
- `failure()` triggers for ANY step failure in the job
- Catches artifact upload failures, smoke test failures, and infrastructure failures
- Ensures users always get feedback when workflow fails
