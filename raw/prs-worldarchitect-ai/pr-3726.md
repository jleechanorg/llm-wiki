## Raw Body
Fix coverage workflow to enable delta coverage reporting

**Problem:**
The coverage workflow was only running on PRs, not on main branch pushes. This prevented the `py-cov-action` from creating/updating the baseline branch (`python-coverage-comment-action-data`), which is required for delta coverage comparisons.

**Solution:**
- Removed the PR-only condition (`if: github.event_name == 'pull_request'`) from the "Coverage comment" step
- Now the action runs on both PRs and main branch pushes
- When it runs on main, it creates/updates the baseline branch with current coverage data
- Future PRs can then compare against this baseline to show delta coverage

**Additional Changes:**
- Improved comment clarity in `mvp_site/testing_framework/integration_utils.py` (line 256)

**Testing:**
- Coverage workflow will run automatically on this PR
- After merge, the next push to main will create the baseline branch
- Subsequent PRs will show delta coverage comparisons
