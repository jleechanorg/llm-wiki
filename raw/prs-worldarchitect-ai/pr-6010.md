## Summary

- Fixes `_classify_raw_narrative` to reject any text starting with `{` or `[`, even when JSON parsing fails (e.g. truncated/syntax-error JSON)
- Previously, malformed JSON that raised a parse exception was classified as narrative, causing raw JSON fragments to be persisted as story text

## Production Code Changes

**Before:** `_classify_raw_narrative` only rejected text that *successfully* parsed to a non-string JSON value. Malformed JSON (truncated responses, syntax errors) fell through and was classified as narrative.

**After:** Any text whose stripped form starts with `{` or `[` is rejected immediately, before attempting `json.loads`. Malformed JSON can no longer be stored as story text.

**Why:** Streaming edge cases (e.g. cache FAILED_PRECONDITION mid-response) can produce truncated JSON fragments. Without this guard, those fragments get written to the story collection as player-visible text.

## Test Changes

Updated `test_classify_raw_narrative.py` to cover:
- Malformed JSON objects/arrays (truncated, syntax errors)
- Confirmed valid narrative strings still pass

## Known Limitations

- Broadened stream exception handling (any exception while cache active triggers uncached retry); non-cache errors like RESOURCE_EXHAUSTED will still fail after retry — documented acceptable latency tradeoff with no functional bug
