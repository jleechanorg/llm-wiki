## Raw Body
Root cause: subshell redirect captured ao spawn + ao send stdout (confirmation lines) instead of the agent's actual work. The agent runs independently in its AO-managed tmux session; the subshell exits immediately after ao send returns.

Fix:
- Remove broken >OUTPUT_FILE redirect from the spawn subshell
- Pass OUTPUT_FILE path to the agent via task prompt; agent writes JSON directly
- Track session names via temp .session files
- After spawn subshells exit, poll 'ao session ls' until session disappears (agent finished) before checking the output file
- Retain all existing JSON validation + fail-closed aggregation logic

Proof: session polling wait loop added to bug-hunt-daily.sh; agent task prompt now instructs writing to OUTPUT_FILE directly
