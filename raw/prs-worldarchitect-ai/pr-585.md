## Purpose
Addresses the issue of mixed PRs with unrelated changes by warning before push.

## Features
- Categorizes changed files by type (Python, JS, CSS, Tests, Docs, etc.)
- Warns if PR has >2 categories or >10 files
- Shows file list and categories for review
- Prompts user to reconsider pushing unfocused PRs
- Can cancel push if user decides to split PR

## How it works
1. Analyzes `git diff --name-only origin/main...HEAD`
2. Categorizes files by extension and path
3. Warns if PR seems unfocused
4. User can choose to continue or cancel
