## Summary
Add automated testing to prevent documentation files from growing too large and causing API timeouts. Also reorganizes archive files into a separate folder.

## Motivation
Following the lessons.mdc reduction project (#257), we need automated monitoring to ensure documentation files don't grow beyond safe limits.

## Changes

### 1. Performance Testing Tools
- **`mvp_site/test_documentation_performance.py`**
  - Comprehensive file size analysis
  - Read performance testing
  - API chunking simulation
  - Detailed recommendations

- **`mvp_site/monitor_doc_sizes.sh`**
  - Quick bash script for CI/CD
  - Simple pass/fail checks
  - Easy to integrate

- **`.github/workflows/doc-size-check.yml`**
  - Automatic PR validation
  - Comments on PR if files too large
  - Runs on documentation changes

### 2. Archive Organization
- **Created `.cursor/rules/archives/` directory**
  - Moved `lessons_archive_2024.mdc`
  - Moved `lessons_backup_20250701.mdc`
  - Moved `lessons_backup_20250104.mdc`
  - Added README explaining purpose

- **Updated references**
  - lessons.mdc now points to `archives/lessons_archive_2024.mdc`
  - Test files exclude archives directory
  - GitHub workflow ignores archives

## Benefits
- ✅ Prevents API timeout issues
- ✅ Maintains documentation readability
- ✅ Automated monitoring in CI/CD
- ✅ Clear thresholds and recommendations
- ✅ Archives preserved but not monitored
- ✅ Clean separation of active/archived docs
