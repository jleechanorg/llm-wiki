## Problem
The backup script was excluding the main `.claude.json` file, which contains:
- Project metadata and configuration  
- Historical usage statistics (tokens, costs, session data)
- Conversation history references

When file corruption occurred (Aug 28th), all historical data was lost because only conversation files (.jsonl) were backed up, not the main configuration file.

## Solution
Modified the rsync include patterns to backup:
- `.claude.json` (main configuration file)
- `.claude.json.backup*` (any backup variants)

This ensures complete restoration capability and prevents data loss from future corruption events.
