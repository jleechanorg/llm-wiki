## Production Code Changes

### 1. `mvp_site/agents.py` (+77 lines)
**Purpose:** Route faction enable requests through classifier and load minigame prompt on first enable

- Added `enable_faction_minigame` to FACTION_ENABLEMENT_PATTERNS for classifier matching
- Added `force_minigame_prompt` parameter to FactionManagementAgent constructor
- When classifier routes to faction intent, agent receives `force_minigame_prompt=True` if minigame not yet enabled
- This ensures faction_minigame_instruction.md prompt is loaded even when enabling for the first time
- Removed redundant state-based dialog routing (simplification)
- Renumbered routing priorities for clarity

### 2. `mvp_site/faction/ai_factions.py` (+18/-18 lines)
**Purpose:** Rescale AI faction power ranges so player characters are competitive

- Old ranges: Easy 50K-500K, Medium 750K-3M, Hard 4M-15M
- New ranges: Easy 5K-50K, Medium 75K-300K, Hard 400K-1.5M
- This makes a Duke with ~33K FP rank around #165 instead of dead last

### 3. `mvp_site/faction/tools.py` (+14 lines)
**Purpose:** Accept alternate parameter names for faction tools

- `faction_calculate_ranking` and `faction_fp_to_next_rank` now accept `faction_power` as alias for `player_faction_power`
- Reduces tool call failures when LLM uses slightly different parameter names
- Preserves backward compatibility with existing callers
