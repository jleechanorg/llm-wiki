## Overview
Adds Think/Plan mode end-to-end and makes the planning block schema a single source of truth. Planning protocol and schema placeholders are now loaded into system prompts, think-mode routing is enforced on the backend with time/state freezing, and the UI exposes a Think/Plan mode with tooltips and return-to-story handling. Includes new tests and removes deprecated prototype tests.

## Changes vs origin/main
### Prompts + schema (single source of truth)
- Added `mvp_site/prompts/planning_protocol.md` (canonical planning block schema + mode/time rules).
- Added `mvp_site/prompts/think_mode_instruction.md` (Think Mode behavior and JSON response format).
- Updated `mvp_site/prompts/game_state_instruction.md` to reference Planning Protocol and inject valid risk levels.
- Added schema injection in `mvp_site/agent_prompts.py` so prompt placeholders are filled from `narrative_response_schema.py`:
  - `PLANNING_BLOCK_SCHEMA`, `CHOICE_SCHEMA`, `VALID_RISK_LEVELS`, `VALID_CONFIDENCE_LEVELS`, `VALID_QUALITY_TIERS`.
- Introduced canonical planning block constants in `mvp_site/narrative_response_schema.py` and expanded choice key validation to allow `think:` prefix.
- Planning fallback narrative now returns a minimal placeholder instead of copying planning_block.thinking (prevents double rendering).
- `mvp_site/llm_providers/provider_utils.py` updated planning_block description to include `think:` keys.

### Agent routing + LLM pipeline
- New `PlanningAgent` in `mvp_site/agents.py` with required prompts (master, think mode, planning protocol, game state, SRD).
- `get_agent_for_input(...)` now accepts `mode` and routes think input before info/combat/rewards/story.
- StoryMode now requires `planning_protocol` in `REQUIRED_PROMPTS`.
- `mvp_site/llm_service.py` uses `mode` in agent selection and simplifies `_validate_and_enforce_planning_block` signature.
- `mvp_site/constants.py` adds think mode constants and `is_think_mode(...)` helper.
