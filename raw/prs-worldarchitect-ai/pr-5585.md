## Summary

Adds comprehensive shell command safety rules for **all** `git` and `gh` commands to prevent terminal hangs and enforce safe execution patterns.

### Problem
Commands like `gh pr comment --body '...'` and `git commit -m '...'` with multi-line strings cause terminal EOF detection failures, hanging commands indefinitely. This has been a recurring issue across multiple agent sessions (Gemini, Claude, and Antigravity).

### Changes

**`GEMINI.md`** — New "Shell Command Safety — Global Policy" section:
- **Default posture**: Never auto-execute git/gh unless in the read-only allowlist
- **Read-only allowlist**: 10 git + 8 gh commands safe to auto-run (e.g., `git status`, `gh pr view`)
- **Mutating command rules**: Require confirmation + warning for commits, pushes, rebase, PR creation, etc.
- **Temp-file convention table**: Standardized patterns for all multi-line content (`--body-file`, `-F`, `--notes-file`)
- Removed contradictory HEREDOC rule (was telling agents to use HEREDOCs AND `--body-file`)

**`.agent/workflows/shell-safety.md`** — Generic Antigravity workflow:
- Covers all git/gh commands, not just PR comments
- Documents read-only vs mutating command patterns
- Includes temp-file examples for all content flags
- `// turbo-all` for auto-execution of safe steps
- Invokable via `/shell-safety` slash command
