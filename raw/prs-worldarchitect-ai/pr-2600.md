## Raw Body
Add critical safeguards to ensure character inventory and key secondary character (companion) equipment are never lost, compressed, or overwritten during state updates.

Changes:
- Add _handle_inventory_safeguard() to protect inventory/equipment/backpack/items fields from complete overwrites - converts list overwrites to safe merges
- Add _handle_companion_equipment_safeguard() to protect equipment of NPCs with companion/party_member/ally/follower relationships or is_important flag
- Integrate both safeguards into update_state_with_changes() flow
- Add PROTECTED_INVENTORY_FIELDS constant for protected field names
- Update docstring to document new safeguard patterns

Test coverage:
- Add TestInventorySafeguard class with 8 test cases covering:
  - Protected fields constant validation
  - Non-protected key handling
  - List merge behavior with simple and complex items
  - Type conversion blocking (dict-to-list, list-to-string)
- Add TestCompanionEquipmentSafeguard class with 8 test cases covering:
  - Non-npc_data key handling
  - Non-companion NPC passthrough
  - Companion/party_member/ally/follower protection
  - is_important flag protection
  - DELETE_TOKEN passthrough
  - Multiple NPC handling
