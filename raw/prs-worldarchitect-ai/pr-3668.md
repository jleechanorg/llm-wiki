## Changes

- Add per-comment cache structure for efficient incremental updates
- Add cache_metadata.json tracking (timestamp, comment count, TTL)
- Add is_cache_fresh() function to check cache age (5 min TTL default)
- Update /copilot to check cache before calling /commentfetch
- Update cache path structure to /tmp/{repo}/{branch}/
- Update all path references in copilot.md, commentfetch.md, commentcheck.md, commentreply.md
- commentfetch.py now saves cache metadata on each fetch
- commentreply.py supports per-comment cache with legacy fallback

## Benefits

- Avoids unnecessary GitHub API calls when comments are fresh
- Reduces latency for repeated /copilot executions
- Maintains cache per repo/branch for isolation
- 5 minute default TTL balances freshness vs performance

## Structure

`/tmp/{repo}/{branch}/`
├── comments.json          # Cached comments (legacy format)
├── cache_metadata.json    # Cache freshness tracking
├── comments_index.json    # Per-comment cache index
├── comments/              # Per-comment files
│   ├── {comment_id}.json
│   └── ...
├── responses.json         # Response data
└── agent_status.json      # Agent coordination
