## Raw Body
### Motivation

- Prevent server errors and test regressions by avoiding passing non-dict campaign/player data into upgrade/tier resolvers. 
- Ensure campaign tier prompts (Divine/Sovereign) are applied consistently to all agents and avoid duplicate ascension choices in the UI.

### Description

- Validate `custom_campaign_state` and `player_character_data` in `_inject_campaign_upgrade_choice_if_needed()` so non-dict values are replaced with safe empty dicts before calling `campaign_divine.get_pending_upgrade_type()`.
- Make `PromptBuilder.finalize_instructions()` dict-safe by resolving `campaign_tier` from a dict-shaped `game_state` or via `get_campaign_tier()` when present, and centrally append Divine/Sovereign system prompts when appropriate.
- Adjust prompt-inclusion branching and small formatting/lint concerns to satisfy pre-commit checks (`noqa` annotations and branch consolidation) while keeping behavior unchanged.
- Ensure planning-block choice normalization maps matching list-derived choices to the canonical `upgrade_campaign` key and avoids duplicate UI entries by normalizing/removing original keys when found.
