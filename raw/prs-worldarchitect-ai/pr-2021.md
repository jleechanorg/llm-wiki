## Raw Body
## Problem
The auto-deploy-dev workflow was failing with log streaming error:
```
This tool can only stream logs if you are Viewer/Owner of the project
```

The GitHub Actions service account doesn't have project Viewer/Owner role required for log streaming.

## Solution
Added `--suppress-logs` flag to `gcloud builds submit` command in `scripts/deploy_common.sh` (line 12).

## Impact
- Build continues to work correctly (verified with build 2fbfcee9-8be6-4b89-a5df-ae811fdfb62b)
- Deployment succeeds and health checks pass
- Workflow no longer fails on log streaming permission error

## Testing
- Manual workflow test confirmed build SUCCESS despite log streaming error
- This fix eliminates the error without requiring overly broad IAM permissions
