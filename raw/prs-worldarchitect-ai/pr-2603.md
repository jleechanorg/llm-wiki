## Summary
Adds the Living World system (deterministic every‑3‑turn cadence) that advances off‑screen world state, integrates it into system instructions, and extends response schema + tests. Also includes supporting evidence tooling updates, conflict documentation, and a batch of orchestration test/fixture cleanups merged from main.

## Detailed Changes (vs origin/main)
### Living World system
- New prompt: `mvp_site/prompts/living_world_instruction.md` with explicit background event schema (3 immediate + 1 long‑term), lifecycle status (`pending|discovered|resolved`), discovery metadata, rumors, faction updates, time events, and scene events.
- Instruction wiring: `mvp_site/agent_prompts.py` adds living world prompt type, cadence helper, and instruction assembly with turn context.
- Cadence config: `mvp_site/constants.py` adds `PROMPT_TYPE_LIVING_WORLD`, `LIVING_WORLD_TURN_INTERVAL = 3`, and path wiring.
- Turn passing: `mvp_site/llm_service.py` computes player turn number (excluding GOD mode) and passes it to the agent so Living World turns trigger consistently.
- Agent interface: `mvp_site/agents.py` accepts `turn_number` for system instruction building; ignored in God/Combat modes.
- Narrative system cross‑reference: `mvp_site/prompts/narrative_system_instruction.md` documents Living World integration and cadence.
- Response schema: `mvp_site/narrative_response_schema.py` expands `state_updates` examples to include world events, faction updates, rumors, time events, and NPC status changes.

### Tests & evidence collection
- New E2E test: `testing_mcp/test_living_world_real_e2e.py` with turn‑by‑turn artifacts, instruction‑file detection, lifecycle status validation, and evidence summaries.
