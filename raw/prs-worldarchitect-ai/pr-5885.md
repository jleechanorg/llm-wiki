## Summary

- Adds `scripts/disk_audit.sh` — opinionated disk audit/cleanup for this macOS dev machine
- Replicates the 2026-03-08 manual investigation that found 224G of orphaned openclaw conversation backups eating a 926G disk down to 9.4G free
- Also added to `user_scope` repo for cross-machine reuse

## Three modes

```bash
./scripts/disk_audit.sh                       # survey only (default)
./scripts/disk_audit.sh --clean --dry-run     # preview what --clean would delete
./scripts/disk_audit.sh --clean               # auto-clean SAFE targets
./scripts/disk_audit.sh --clean-all           # interactive: Docker, Colima, Codex sessions
```

## SAFE targets cleaned automatically by `--clean`

- `~/.openclaw/conversation-backups/` — orphaned V1 backup data (was 224G)
- `/private/tmp/throwaway-dropbox-*`, `ai-orch-worktrees`, `pr-orch-bases`
- npm cache, Xcode DerivedData, puppeteer/uv/gemini caches
- Chrome automation profiles

## Interactive-only (`--clean-all`)

- Docker: `docker system prune -a`
- Colima VM deletion (if redundant with Docker Desktop)
- Codex sessions (`~/.codex/sessions/`)

## Known Limitations

- `~/Library/` traversal requires 120s timeout; may show 0G if system is under heavy I/O
- Docker prune requires Docker Desktop to be running
- Does not auto-clean Codex sessions (too risky without user confirmation)
