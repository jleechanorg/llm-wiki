## Summary

This PR introduces an AI-powered code authenticity detection system that identifies fake, simulated, or demo code patterns in our codebase.

## What's New

### 1. **Code Authenticity Analysis Script** (`analyze_code_authenticity.sh`)
- Uses Claude's natural language understanding to detect fake code patterns
- Identifies:
  - Hardcoded responses that simulate real logic
  - Functions claiming sophistication but using simple keyword matching
  - Mock API responses with placeholder data
  - Demo code that doesn't implement real functionality
  - Generic template responses without actual analysis

### 2. **Integration with Push Workflow**
- Added code authenticity check to `push.sh` after tests pass
- Provides warnings but doesn't block push (developer decision)
- Automatically calls `/learn` command to document detected patterns

## How It Works

1. Analyzes all changed code files (Python, JS, TS, Java, Go, etc.)
2. Creates a markdown analysis file with code content
3. Uses Claude CLI to perform LLM-based analysis
4. Returns warning level (1) if fake code detected, success (0) otherwise
5. Integrates findings with the learning system for future reference

## Example Detection

The script can identify patterns like:
```python
def _create_contextual_response(self, body: str, user: str, file_path: str, line: str) -> str:
    body_lower = body.lower()
    if 'error' in body_lower or 'bug' in body_lower:
        response_parts.append("Thank you for identifying this issue\! ")
    elif 'suggestion' in body_lower:
        response_parts.append("Excellent suggestion\! ")
```
This would be flagged as fake because it uses hardcoded responses based on simple keyword detection.
