## Summary
Extract living world trigger evaluation into a standalone module and expand test coverage for the trigger system.

**Key themes:**
- Living world trigger extraction into pure function module
- Modulo-based scheduling for consistent trigger cadence
- Defense-in-depth with isinstance guards

## Production Code Changes

### 1. `mvp_site/living_world.py` (+48 lines) **[NEW FILE]**
**Purpose:** Pure function module for living world trigger evaluation

- **New `evaluate_living_world_trigger()`** - Extracted trigger logic from `GameState.check_living_world_trigger()` into a standalone pure function → Enables unit testing without GameState instantiation and centralizes trigger logic

### 2. `mvp_site/game_state.py` (+46/-35 lines)
**Purpose:** Campaign state management with living world integration

- **Import `living_world` module** - Had inline trigger logic → Delegates to `living_world.evaluate_living_world_trigger()` → Separation of concerns, testable pure function
- **Updated `check_living_world_trigger` signature** - Accepted only `current_turn` → Now accepts optional `current_time` kwarg → Allows caller (world_logic.py) to pass world_time directly instead of relying on self.world_data lookup
- **Added input validation for `current_time`** - No validation → Validates isinstance(dict) with logging_util.warning on invalid types → Prevents silent failures from non-dict world_time values

### 3. `mvp_site/world_logic.py` (+6/-1 lines)
**Purpose:** World logic processing with living world tracking

- **Added isinstance guard for world_data** - Direct `.get("world_time")` on potentially non-dict world_data → Checks `isinstance(world_data, dict)` first → Prevents AttributeError in production when world_data is None or non-dict
- **Pass `current_time` to `check_trigger()`** - Called `check_trigger(turn_number)` without time context → Passes `current_time=current_time` kwarg → Enables time-based trigger evaluation from caller context
