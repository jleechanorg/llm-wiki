## Problems Found
1. **Missing Planning Blocks**: AI wasn't consistently including planning blocks in story mode
2. **God Mode Bug**: Planning blocks appeared when switching to god/dm mode (shouldn't happen)
3. **Cross-Campaign Contamination**: Planning blocks contained references to other campaigns

## Root Causes
1. **Missing Blocks**: AI not following instructions from `narrative_system_instruction.md`
2. **God Mode**: No detection for mode switching in planning block enforcement
3. **Contamination**: LLM context confusion pulling options from other campaigns

## Solution Implemented

### 1. Planning Block Validation (`_validate_and_enforce_planning_block`)
- Automatically detects missing planning blocks
- Uses LLM to generate contextually appropriate planning blocks
- Skips enforcement during character creation

### 2. Mode Detection Fix
- Skip planning blocks when user says "god mode", "dm mode", etc.
- Skip when AI response contains `[Mode: DM MODE]` or `[Mode: GOD MODE]`
- Prevents inappropriate planning blocks in meta-game modes

### 3. Context Isolation
- Planning prompts now explicitly reference current character name and location
- Added "CRITICAL: Generate options ONLY for {character} in {location}"
- Prevents cross-campaign contamination like Alexiel's options appearing for Sir Braddock
