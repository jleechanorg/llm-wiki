## Summary
Enable Codex automation to work from any directory by using git worktrees in `/tmp`, fixing the "not a git repository" errors that prevented PR creation.

**Key themes:**
- Worktree-based isolation - eliminates directory dependency
- Repository auto-detection - works with any configured repo
- Cleanup automation - prevents /tmp pollution

## Production Code Changes

### 1. `automation/jleechanorg_pr_automation/openai_automation/codex_cli_tasks.py` (+218/-75 lines)
**Purpose:** Codex CLI automation for applying code changes and creating PRs

- **Added worktree management system** - No worktree support → Added `_create_worktree()`, `_cleanup_worktree()` methods → Enables running from any directory
- **Added repo path detection** - Hardcoded directory assumption → Auto-detect from `~/.config/worldarchitect/repo_path`, current dir, or default path → Works with any repository
- **Updated `__init__` with repo_path param** - No repo config → Added optional `repo_path` parameter and detection logic → Configurable per-repository
- **Refactored `create_pr_from_diff` to use worktrees** - Ran git commands in current directory → Creates temp worktree in `/tmp`, runs git commands with `-C` flag → No longer requires being in git repo
- **Refactored `apply_and_push` to use worktrees** - Ran git commands in current directory → Uses temp worktrees for all operations → Same isolation benefits
- **Added branch existence checking** - Always created new branch with `-b` → Check if branch exists first, use appropriate checkout command → Handles leftover branches from previous runs
- **Added branch name sanitization** - Used branch name directly in temp path → Replace `/` with `_` in temp directory names → Prevents invalid directory paths

## Configuration

The automation now looks for the main repository path in this order:
1. `~/.config/worldarchitect/repo_path` file (if exists)
2. Current directory (if it's a git repo)
3. `~/projects/worldarchitect.ai` (default location)
