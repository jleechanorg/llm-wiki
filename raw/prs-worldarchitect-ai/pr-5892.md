> **Medium Risk**
> Touches git fetch/checkout/push and error-handling paths in the Codex automation, which can affect branch updates and PR creation behavior. Changes are localized but could impact reliability across reruns and edge cases (stale diffs, missing branches, older git versions).
> 
> **Overview**
> Improves Codex PR automation robustness by tightening remote-branch detection and branch handling: `ls-remote` failures now raise, remote branches are fetched into explicit `refs/remotes/origin/*`, and worktrees prefer detached checkouts via `git switch --detach` (with `checkout` fallback).
> 
> Hardens diff application by normalizing diffs to end with a newline, adding debug previews on `git apply` failures, and refining fallback behavior around *stale/corrupt patch* errors. Push/commit flow is made safer and more deterministic (better commit failure returns, explicit `HEAD:{branch}` pushes, and `--force-with-lease` where appropriate).
