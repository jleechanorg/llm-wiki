## Summary
- Integrate coverage collection into `test.yml` core group so PRs get coverage reports WITHOUT a separate workflow re-running all ~276 tests
- Remove `pull_request` trigger from `coverage.yml` (now main-only for baseline updates)
- Delete misplaced duplicate `mvp_site/test_documentation_performance.py`
- Update all doc-size-check references to use the correct `tests/` location

**Key themes:**
- CI deduplication: coverage.yml was re-running all ~276 tests already executed by test.yml (~14 min wasted per PR push)
- Coverage preserved: PR coverage comments + threshold checks now happen in test.yml via artifact passing
- File placement: removing duplicate test file from wrong location per project audit

## How Coverage Works Now

```
PR Push:
  test.yml "Directory tests (core)" job
    |-- runs tests with --coverage flag
    |-- uploads .coverage artifact
  test.yml "PR Coverage Report" job (new, ~30s)
    |-- downloads .coverage artifact (no test execution)
    |-- generates report + posts PR comment with delta
    |-- checks coverage threshold (warn <80%, fail <60%)

Main Merge:
  coverage.yml (unchanged behavior)
    |-- runs full test suite with coverage
    |-- updates py-cov-action baseline (used for PR deltas)
    |-- generates badge
```

| Scenario | Before | After |
|----------|--------|-------|
| PR push | coverage.yml re-runs ALL ~276 tests (~14 min) | test.yml collects coverage during existing test run + lightweight report job (~30s) |
| PR coverage comment | Posted by coverage.yml after duplicate test run | Posted by test.yml coverage-report job (same comment, no duplicate tests) |
| Coverage threshold check | Done in coverage.yml after duplicate test run | Done in test.yml coverage-report job |
| Merge to main | coverage.yml updates baseline + badge | Same (unchanged) |
