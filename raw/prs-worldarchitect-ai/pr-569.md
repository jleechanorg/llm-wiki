## Context
This PR creates the foundation for a real-mode testing framework that was designed while fixing the Firestore persistence bug in PR #551. The framework addresses the issue where test mocks don't match real service behavior, which allowed bugs to slip through.

## Changes
- **Scratchpad**: Comprehensive plan for real-mode testing with data capture
- **Slash Commands**:
  - `/teste` - Run end2end tests in mock mode (current behavior)
  - `/tester` - Run end2end tests with real services
  - `/testerc` - Run end2end tests with real services + data capture
- **Scripts**: Bash scripts for each test mode
- **Updated /4layer**: Refocused to prioritize integration testing first

## Benefits
1. **Bug Prevention**: Catches persistence issues like the think command bug
2. **Mock Accuracy**: Ensures test doubles match real service behavior  
3. **Regression Detection**: Identifies when services change behavior
4. **Confidence**: Tests validate actual system behavior, not just contracts
