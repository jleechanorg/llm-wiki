## Raw Body
## Problem

The level-up E2E test (`test_level_up_modal_flow_real.py`) fails on `origin/main` because `planning_block.choices` is returned as a **dict** (`{"key": {...}}`) internally but the frontend expects **array-of-objects** (`[{"id": "key", ...}]`) per ADR-0003 Phase 6.

The injection functions (`_inject_levelup_choices_if_needed`, `_inject_modal_finish_choice_if_needed`) work with dict-format choices internally via `normalize_planning_block_choices` (list→dict), but never convert back to array format before returning.

## Solution

### New: `normalize_choices_to_array()` in `campaign_upgrade.py`
Inverse of `normalize_planning_block_choices` — converts dict-format choices to array-of-objects with the dict key injected as the `id` field. Non-dict values are filtered out.

### Modified: Injection functions in `world_logic.py`
Added `normalize_choices_to_array` as the final step in:
- `_inject_levelup_choices_if_needed` (level-up choice injection)
- `_inject_modal_finish_choice_if_needed` (modal finish choice injection)

## Tests

8 TDD tests in `test_level_up_choices_normalization.py`:
- Dict→array conversion with id injection
- Array passthrough (no-op)
- Edge cases: empty dict, missing key, non-dict values
- Integration: both injection functions produce array output

All 16 level-up tests pass (8 new + 8 existing).
