## Problem
When running tests locally with `TESTING=true`, the clock skew would default to 0 seconds, causing Firebase JWT errors:
```
invalid_grant: Invalid JWT: Token must be a short-lived token (60 minutes)
```

## Solution
Local development now defaults to 600 seconds (10 minutes) of clock skew adjustment, making Firebase auth work out of the box.
