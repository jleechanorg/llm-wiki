## Raw Body
### Motivation

- Prevent accidental reordering of world lore in initial story generation by centralizing how story-mode instructions are assembled.
- Reduce duplicated PromptBuilder/Prompt ordering logic between `get_initial_story` and story continuation flows.
- Simplify agent-based routing in `continue_story` and make instruction assembly polymorphic across agents.
- Make tests more robust by normalizing test `sys.path` setup to reference the project root.

### Description

- Add `StoryModeAgent.build_system_instruction_parts(...)` to produce the ordered list of system-instruction parts (core, character, selected prompts, references, continuation reminders) without appending world content; `build_system_instructions` then finalizes with world content via `PromptBuilder.finalize_instructions`.
- Update `get_initial_story` to use `agent.build_system_instruction_parts(...)`, append initial-story-specific parts (companions/background), and then call `builder.finalize_instructions(...)` so world lore remains last in the instruction hierarchy.
- Move `mvp_site.agents` imports into the top-level import block of `mvp_site/llm_service.py` and simplify `continue_story` to call `agent.build_system_instructions(...)` polymorphically (removes unreachable fallback branch and extra aliasing).
- Adjust test module path setup in `mvp_site/tests/test_agents.py` and `mvp_site/tests/test_agents_integration.py` to insert the project root into `sys.path` explicitly (improves import clarity across environments).
