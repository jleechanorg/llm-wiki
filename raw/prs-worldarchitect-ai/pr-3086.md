## Summary

- Add comprehensive schema documentation for `active_effects` in game_state_instruction.md
- Add persistent buff example to god_mode_instruction.md with clear "MANDATORY" directive language

## Problem
When users tell the LLM "stop forgetting Enhance Ability" or "assume I always have Haste", the LLM:
1. Acknowledges it in `god_mode_response` (text only, not saved)
2. Doesn't return `directives.add` (rule not persisted)
3. Doesn't add to `active_effects` (buff not stored)

Result: LLM forgets the buff on the next turn.

## Solution
1. **Display existing buffs**: `active_effects` from `player_character_data` now shows in the "### Active Effects (ALWAYS APPLY)" section of the character identity block
2. **Teach LLM how to save buffs**: Added clear example in god_mode_instruction.md showing the correct `state_updates` and `directives` format
3. **Strengthen directive enforcement**: Added "MANDATORY" language and failure mode warning
