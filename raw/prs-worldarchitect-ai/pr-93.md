### Summary of Changes

This pull request focuses on improving the flexibility and clarity of the AI's core instructions and enhancing the system's logging for better debugging.

**Key Changes:**

*   **Improved Token Logging (`gemini_service.py`):**
    *   The token counting logic has been refactored to be fully comprehensive. It now accurately calculates and logs the total tokens for every API call, providing a breakdown of tokens used for the user-facing prompt versus the system instructions. This resolves the issue of misleadingly low token counts during campaign creation.

*   **Ruleset-Agnostic Instructions (`mechanics_system_instruction.md`, `narrative_system_instruction.md`):**
    *   The core instruction prompts have been updated to be more flexible and less dependent on D&D-specific terminology.
    *   The system now explicitly prioritizes the "Destiny" ruleset when specified, while still allowing the 5e SRD to be used as a generic fantasy fallback, making the platform more adaptable to different game systems like Pathfinder or custom rulesets.

*   **Enhanced Narrative & Character Generation (`narrative_system_instruction.md`):**
    *   Added an "Opening Scene Character Mandate" to ensure all characters who appear in the first scene are fully defined in the game state, preventing the AI from "hallucinating" undefined characters and creating narrative inconsistencies.
    *   A "Primacy of Narrative Profile" rule has been added to force the AI to prioritize a character's established psychology and motivations over their raw stats when determining NPC actions, leading to deeper and more consistent roleplaying.

*   **Improved User Experience (`calibration_instruction.md`):**
    *   The initial campaign setup prompt has been enhanced with helpful examples for different genres (Fantasy, Sci-Fi, Modern) to make it easier for users to get started with a new campaign.
