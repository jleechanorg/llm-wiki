## Raw Body
## Problem
The server crashes during reprompt loops when required fields are missing. Reprompts cause back-to-back Gemini API calls (~38K tokens each) which exhaust resources and kill the server process.

## Changes
- **Disabled ALL reprompt checks** in `_check_missing_required_fields`:
  - `planning_block`: Disabled - reprompts cause crashes
  - `dice_rolls`: Disabled - reprompts cause crashes  
  - `dice_integrity`: Disabled - reprompts cause crashes
  - `session_header`: Removed - cosmetic only
  - `social_hp_challenge`: Removed - reprompts cause crashes
- Function now always returns empty list to prevent any reprompt attempts
- Updated docstring and comments to reflect all checks are disabled

## Rationale
- All reprompts cause back-to-back heavy API calls (~38K token API calls each)
- Missing fields trigger 3 reprompt attempts (3x ~38K token API calls)
- Back-to-back heavy API calls exhaust resources and crash the server
- Missing fields are either cosmetic (session_header) or can be handled gracefully without reprompts
- The reprompt loop code remains but will never execute since `missing_fields` is always empty
