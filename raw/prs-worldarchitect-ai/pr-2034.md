## Summary

Adds new `/clonefix` slash command that automates the workflow of:
1. Cloning a PR repository
2. Adding comprehensive TDD tests
3. Verifying tests pass
4. Pushing to the **correct** remote branch

## Problem Solved

This addresses a common workflow where we need to:
- Clone a PR locally
- Add tests for new logic
- Push back to the PR branch

Previously, this required manual steps and was error-prone (e.g., pushing to wrong branch).

## Key Features

### Branch Verification Protocol (CRITICAL)
- **MANDATORY**: Verifies actual branch name from GitHub API/CLI
- **NEVER**: Assumes branch name from PR number
- **ALWAYS**: Double-checks before push
- Prevents the mistake of pushing to wrong remote branch

### Complete Workflow
1. Extract PR info and verify branch name
2. Clone repo to /tmp with unique directory
3. Identify new logic requiring tests
4. Execute /tdd workflow for test creation
5. Verify all tests pass locally
6. Save proof in /tmp subdirectory
7. Verify branch name again (double-check)
8. Commit and push to correct branch

### Integration
- Integrates with existing `/tdd` command
- Follows established command structure
- Uses TodoWrite for progress tracking
- Creates proof directory with test results

## Usage

```
/clonefix https://github.com/jleechanorg/ai_universe_frontend/pull/244
/clonefix jleechanorg/ai_universe_frontend#244
```

## Safety Features

- Branch name verification from GitHub (not assumptions)
- Test verification before push
- Proof directory creation
- Error handling for common failure modes
- Clear reporting of branch name before push

This command was created following the exact workflow it automates, ensuring it works correctly.
