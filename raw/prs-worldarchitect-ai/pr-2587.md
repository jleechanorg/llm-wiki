## Summary
- Adds `chi_square_two_phase.py` to validate dice distribution uniformity for Gemini 2.0 Flash (native_two_phase strategy)
- Complements existing `chi_square_gemini_seeded.py` for code_execution strategy

## Why This Matters
Unlike code_execution where the LLM generates RNG code (fixed in PR #2551), native_two_phase uses server-side dice tools with Python's random module (OS-seeded RNG). This test validates that server-side RNG is properly uniform.

## Test Evidence

### code_execution (Gemini 3 Flash)
```
Rolls: [6, 19, 4, 13, 17, 13, 4, 11, 6, 15, 7, 4, 16, 11, 8, 1, 19, 18, 19, 6]
Chi-square: 22.0 < 46.0 threshold
Result: PASS
```

### native_two_phase (Gemini 2.0 Flash)
```
Rolls: [2, 8, 14, 9, 2, 3, 8, 15, 11, 15, 4, 19, 12, 8, 15, 16, 20, 19, 8]
Chi-square: 24.16 < 46.0 threshold  
Result: PASS
```

Both strategies produce statistically uniform dice distributions.
