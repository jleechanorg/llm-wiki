## Summary (BREAKING CHANGE)
- **BREAKING**: Added `wrap_prompt: bool = True` parameter to `analyze_task_and_create_agents()` between `task_description` and `forced_cli` — callers passing `forced_cli` positionally must update to keyword argument
- When `wrap_prompt=False`: skips PR context detection, subprocess calls to `gh`, PR mode instructions, and `pr_context` injection — returns raw `task_description` as prompt
- When `wrap_prompt=True` (default): behavior is identical to previous version
- Beads: REV-udu2h (P1), REV-y4zmu (P2), REV-4bi76 (P3)

### Why a breaking change?
The `wrap_prompt` parameter is intentionally placed before `forced_cli` because it is the more commonly used control parameter. Existing callers passing `forced_cli` positionally (e.g., `analyze_task_and_create_agents(task, "codex")`) must migrate to keyword syntax: `analyze_task_and_create_agents(task, forced_cli="codex")`. This is a deliberate API cleanup.

### Migration
```python
# Before (positional — will break)
dispatcher.analyze_task_and_create_agents(task, "codex")

# After (keyword — required)
dispatcher.analyze_task_and_create_agents(task, forced_cli="codex")
```
