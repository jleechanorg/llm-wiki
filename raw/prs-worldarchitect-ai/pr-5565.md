## What Changed (before → now → why)

### `.claude/scripts/pair_execute.py`
- **Before**: Used global temp directory, generated basic agent names
- **Now**: Uses unique path `/tmp/<branch>/<run_ts>/pair_sessions/<session_id>/`, session-scoped agent names with hash
- **Why**: Prevents collisions between concurrent runs, enables MCP Mail coordination

### `.claude/scripts/pair_monitor.py`
- **Before**: Session directory inferred from branch name
- **Now**: Accepts explicit `--session-dir` argument, log-based health monitoring with auto-restart
- **Why**: Removes ambiguity, improves reliability with health checks

### `orchestration/constants.py`, `orchestration/orchestrate_unified.py`, `orchestration/task_dispatcher.py`
- **Before**: Various orchestration issues (token handling, timeout coupling, prompt wrapping)
- **Now**: Decoupled timeouts, safer GITHUB_TOKEN loading, wrap_prompt default=False, optional model support
- **Why**: Foundation for pair/copilot integration, improved reliability
