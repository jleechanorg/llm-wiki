## Raw Body
Problem:
- Backend was failing with validation error when combined campaign fields exceeded 1M characters
- Frontend allowed up to 100k chars per field but backend combines them with separators and entity tracking
- Total could exceed 1M limit even if individual fields were within limits

Solution:
- Backend: Improved error message to clearly explain the issue and how to fix it
- Frontend API: Added combined length validation before API call (950k safe limit accounting for 50k overhead)
- Frontend Component: Added same validation in form validation step for immediate user feedback

Changes:
1. mvp_site/gemini_request.py: Updated ValidationError message to be user-friendly
2. mvp_site/frontend_v2/src/services/api.service.ts: Added combined length validation with detailed error
3. mvp_site/frontend_v2/src/components/CampaignCreationV2.tsx: Added validation in form validation step

Benefits:
- Users get immediate feedback in UI instead of server error
- No silent truncation that could lose user data
- Clear error messages explain exactly what's wrong and how to fix it
- Defense in depth: validation in both frontend and backend
