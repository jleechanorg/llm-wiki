The rewards_box was showing incorrect next_level_xp values because the LLM was deriving the level from XP (e.g., 400 XP → Level 2 → next threshold 900) instead of using the actual level from player_character_data.level.

For a Level 1 character with 400 XP after a reward:
- WRONG: 400/900 (44%) - derived Level 2, shows Level 3 threshold
- RIGHT: 400/300 (133%) - actual Level 1, shows Level 2 threshold

Changes:
- Add critical instruction to use player_character_data.level
- Add concrete example showing correct vs incorrect calculation
- Update rewards_box example to show correct values
- Add note in essentials section for token-constrained mode
