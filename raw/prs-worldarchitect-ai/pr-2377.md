## Root Cause
Cerebras API with legacy `{"type": "json_object"}` sometimes returns the format config itself (`{"type": "object"}`) instead of actual content. This causes empty narratives and fallback planning blocks.

## Solution
Use `json_schema` with `strict:true` which provides constrained decoding:
```python
"response_format": {
    "type": "json_schema",
    "json_schema": {
        "name": "narrative_response",
        "strict": True,
        "schema": {...}
    }
}
```
