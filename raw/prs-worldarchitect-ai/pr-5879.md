## Background

WorldAI uses OpenClaw as a local AI gateway. To enable remote (GCP-hosted) WorldAI servers to reach a user's local OpenClaw instance, the gateway must be reachable at a stable public URL.

**This PR proves that a Tailscale URL works as the OpenClaw gateway URL with WorldAI.** Tailscale provides a stable, secure, always-on network hostname (`jeffreys-macbook-pro.tail5eb762.ts.net`) that GCP Cloud Run can reach via the tailnet — without a traditional tunnel script, without port forwarding, and without public DNS registration.

The Tailscale URL **is** the public URL for this PR. It resolves only within the tailnet, but GCP Cloud Run is connected to the tailnet and can reach it — which is the requirement for WorldAI OpenClaw routing.

## Goals

1. **Prove Tailscale URL works end-to-end** — GCP WorldAI server calls `https://jeffreys-macbook-pro.tail5eb762.ts.net/v1/chat/completions` and routes inference to the local OpenClaw gateway.
2. **Fix the "Test Connection" 401 bug** — Settings page button was returning 401 because the server used a stripped (masked) token. Fixed by falling back to raw Firestore when token is absent/masked.
3. **Harden the E2E browser test** — Add `OPENCLAW_PRESET_TUNNEL_URL` so the browser E2E test works with Tailscale (always-on, no tunnel script needed).
4. **Personal API keys for MCP access** — Programmatic Bearer token lifecycle API.
5. **Preview/CI/MCP reliability hardening** — Trace logging, evidence packaging, shard stability.

## Tenets

- Prefer real end-to-end tests over mocks.
- "Public URL" for Tailscale means reachable from GCP via tailnet — not publicly DNS-resolvable from arbitrary internet hosts.
- `OPENCLAW_REQUIRE_PUBLIC_URL_DNS=true` is wrong for Tailscale tests; use `OPENCLAW_REQUIRE_PUBLIC_URL_DNS=false` with `OPENCLAW_PRESET_TUNNEL_URL`.
- Minimize scope: fix root causes, not workarounds.
