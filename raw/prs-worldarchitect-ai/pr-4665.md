> [!NOTE]
> **Medium Risk**
> Changes affect automation control flow around when queued comments are posted and when runs are blocked, which could alter behavior under quota/rate-limit conditions. Timeouts and retained artifacts should reduce flakiness, but the removal of per-PR gating may reintroduce misleading queued comments if quotas change after startup.
> 
> **Overview**
> **Preflight behavior is consolidated and simplified.** The per-PR “quick preflight” check before posting `fix-comment`/`fixpr` queued comments is removed, and the run now relies on a single startup CLI chain preflight; this startup preflight is skipped in test/CI modes and can retain and log the validation output file paths.
> 
> **CLI validation is made more reliable and diagnosable.** Validation timeouts are increased (`90s` execution / `10s` help), `ValidationResult` now consistently includes `output_file` on failures when available, and the default assistant mentions list now includes `@copilot`.
> 
> **Tests are updated for the new contract.** Queued-comment validation tests are simplified to assert “dispatch success gates queued comments” (dropping the pre-comment preflight scenarios), and Codex integration/comprehensive tests make `aiohttp` optional via `pytest.importorskip` signature compatibility to avoid hard failures across pytest versions.
