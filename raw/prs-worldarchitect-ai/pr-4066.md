## Raw Body
### Motivation
- Fix dialog-continuity regressions and missed description-only choices by ensuring planning-block choices are fully scanned for dialog intent. 
- Remove runtime keyword-based routing fallbacks so routing consistently relies on the semantic intent classifier and state checks.
- Address test and lint failures introduced by the earlier changes and make classifier errors visible for debugging.

### Description
- Scan all planning_block choices (no 5-choice limit) and classify each choice by combining `text` and `description` so description-only entries are evaluated for dialog intent. Classifier exceptions are logged for visibility. 
- Remove the keyword-based fallback routing for Faction/Rewards (deleted the input-pattern fallback block) so selection now relies on state checks and `intent_classifier.classify_intent`. 
- Update `get_agent_for_input` to return rich `AgentResult` as before while cleaning up logging and routing priority annotations. 
- Test and test-helper fixes: normalize `get_agent_for_input(...).agent` usage in tests, fix many test callsites, and adjust prompt/instruction-building assertions. 
- Code hygiene and lint fixes: added targeted `# noqa` annotations for unavoidable unused args or complexity warnings, improved debug logging messages, and cleaned up formatting to satisfy pre-commit checks.
