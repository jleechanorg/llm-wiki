## Summary

Migrate the 2 most expensive workflows to self-hosted runners to reduce GitHub Actions costs.

- **Immediate savings: $482.25/month**
- **Yearly savings: $5,787** on these two workflows
- **ROI: Immediate** (self-hosted infrastructure is free with your server)

## Production Code Changes

### Before
- `test.yml`: Runs heavy test suite on `ubuntu-latest` → $429/month
- `coverage.yml`: Generates coverage on `ubuntu-latest` → $53/month
- Combined: **71,570 minutes/month** (53.6% of all Actions usage)

### Now
- `test.yml`: Runs on `[self-hosted, claude]` → $0
- `coverage.yml`: Runs on `[self-hosted, claude]` → $0
- Combined: Uses your 5 self-hosted runners (76k minutes fit within available capacity)

### Why
- These 2 workflows consume 76,375 minutes/month at $0.006/minute = $458.25/month
- Your 5 runners (deployed today) have ~240k minutes/month available capacity
- Cost is reduced to $0 (just server electricity/bandwidth)

## Test Changes

Workflow behavior remains identical:
- Same test suite executes
- Same Python 3.11 environment
- Same timeout constraints (30m for test.yml, 20m for coverage.yml)
- **New benefit:** Faster execution (no queue wait for GitHub-hosted runners)

## Known Limitations

1. **Requires runners online:** If all 5 self-hosted runners are down, jobs will queue until one becomes available. Fallback: Re-add ubuntu-latest as secondary job.

2. **GCP credentials needed:** Jobs don't use GCP services, so no setup needed.

3. **Future scaling:** If you need more capacity, add more self-hosted runners (no cost increase per runner).

## Deployment

No manual deployment needed:
- Self-hosted runners already deployed and running
- GitHub Actions automatically routes jobs to `[self-hosted, claude]` labeled runners
- Fallback: Quick to restore `ubuntu-latest` if needed
