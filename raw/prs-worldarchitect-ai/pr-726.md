## Raw Body
## Problem
The `claudepw` alias fails with error: `unknown option '--worker'` because the claude CLI doesn't recognize this flag.

## Root Cause
The alias `claudepw='source ./claude_start.sh --worker'` passes `--worker` to the script, but the script was passing all arguments directly to the claude CLI, including the unrecognized `--worker` flag.

## Solution
Updated `claude_start.sh` to:
1. Parse command-line flags (`-w/--worker`, `-d/--default`, `-s/--supervisor`)
2. Set MODE variable based on the flag
3. Skip interactive menu when mode is specified
4. Launch claude with appropriate model without passing the flag

## Changes
- Added MODE variable and command-line argument parsing
- Added logic to handle direct mode selection
- Maintains backward compatibility with interactive menu

## Testing
After this fix:
- `claudepw` will work correctly, launching worker mode directly
- `./claude_start.sh --worker` will skip the menu and start worker mode
- `./claude_start.sh` without flags will show the interactive menu as before

This fixes the immediate issue and also enables other shortcuts like:
- `alias clauded='source ./claude_start.sh --default'`
- `alias claudes='source ./claude_start.sh --supervisor'`
