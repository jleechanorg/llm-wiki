## Raw Body
### Motivation
- Prevent UI bugs where BYOK inputs reattach listeners after SPA navigation and accidentally resubmit masked placeholders as real keys. 
- Make the settings UI compatible with the backend change that returns only presence flags (no plaintext BYOK keys) and avoid leaking or re-posting masked values.

### Description
- Make BYOK input/listener wiring idempotent and add per-input dirty-state tracking so handlers are not stacked on SPA re-entry and saves only run for actual user edits (update `mvp_site/frontend_v1/js/settings.js`).
- Display a masked placeholder (`********`) when `has_custom_*_key` is present, and ensure the placeholder is never sent back to the server; only edited (dirty) values are included in POST payloads.
- Ensure clear-key flows reset dirty state and UI consistently after clearing a key.
- Add `responses.json` containing audit entries (FIXED / CONSIDERED) that reference PR comment URLs for traceability, and include the `[codex-automation-commit]` marker in the local commit message used for tracking.
