## Raw Body
## Summary

Fixes a bug where God/Primordial tier NPCs (like Empress Sariel, Level 35) were not properly triggering the Social HP skill challenge system, allowing single-roll persuasion bypasses.

**Bug Evidence:**
- **Dragon Knight Evil**: Empress Sariel (Level 35 God-Empress) NEVER had Social HP tracked
- **Dragon Knight Good**: Prefect Gratian (Lord tier) properly got `Social HP: 5/5` tracking

This is backwards - God-tier NPCs should have MORE protection (15+ HP), not less.

## Root Cause

High-tier NPCs weren't consistently triggering the Social HP system because the LLM wasn't being reminded to enforce it for powerful characters.

## Solution (Multi-Layer Fix)

### 1. Server-side Auto-Injection (`llm_service.py`)
- Detects high-tier NPCs in `npc_data` (god_primordial, king_ancient, lord_general, or level >= 15)
- Auto-adds `"social_hp"` to `pending_instruction_hints`
- Logs injection for debugging:
  ```
  🚨 SOCIAL_HP: Auto-injected Social HP reminder due to high-tier NPCs: ['Empress Sariel (tier=god_primordial, level=35)']
  ```

### 2. Enhanced Social HP Reminder (`agent_prompts.py`)
- Added `SOCIAL_HP_ENFORCEMENT_REMINDER` constant with mandatory output format
- Includes request difficulty scaling (teach=1x, betray=2x, submit=3x)

### 3. Prompt Improvements (`narrative_system_instruction.md`)
- Added "Human DM friction heuristic" trigger
- Request difficulty scaling by interaction type
- Explicit check: "Would a human DM say 'that won't work with one roll'?"

### 4. Test Coverage (`testing_mcp/test_social_hp_god_tier_real_api.py`)
- Seeds Empress Sariel (Level 35, god_primordial tier) in test campaign
- Tests multiple persuasion scenarios against real local server
- Verifies `[SOCIAL SKILL CHALLENGE:]` box, Social HP 15+, resistance indicators
- Evidence capture via savetmp integration
