## Summary
Add quick-action buttons to fetch Equipment, Stats, and Spells directly from game state without LLM calls (~50ms vs ~15s response time).

## Format Requirements

### 1. Equipment Display
- Show items with all their stats and modifiers
- Display equipped items by slot (Head, Body, Main Hand, etc.)
- Show weapons with damage dice and properties
- Include backpack items with descriptions

### 2. Character Stats Display  
- Show base stats "naked" (without equipment bonuses)
- Show effective stats with all gear applied
- Display equipment bonuses separately
- Include HP, Level, AC, and combat-relevant stats

### 3. Spells Display
- Show spells known organized by spell level
- Display spell slots (current/max) per level
- List cantrips (at-will spells)
- Show prepared spells for prepared casters
- Include class resources (Hit Dice, Lay on Hands, Ki, etc.)

## Changes
- API: New `GET /api/campaigns/:id/equipment`, `.../stats`, and `.../spells` routes
- Equipment display checks both `equipment` and `inventory` keys in game state
- Stats endpoint handles `aptitudes` format with nested `{score, modifier}` structure
- Safe stat source coercion with logging for non-dict sources
- Combined regex pattern for equipment bonus parsing (handles both `+2 STR` and `STR +2`)
- Frontend: Quick action buttons with spinners, error handling, response-time display

## Game State Locations
- `player_character_data.inventory.equipped`: equipped items by slot
- `player_character_data.inventory.backpack`: backpack items
- `player_character_data.aptitudes`: stats in `{STR: {score: 13, modifier: 1}}` format
- `player_character_data.stats` or `attributes`: alternate stat locations
- `player_character_data.spells`, `spells_prepared`, `cantrips`: spell data
- `player_character_data.spell_slots`: spell slot tracking
- `player_character_data.resources`: class resources
- `item_registry`: item definitions with stats/bonuses
