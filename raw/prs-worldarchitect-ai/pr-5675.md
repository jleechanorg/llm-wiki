## Summary

## Production Code Changes
**self-hosted/scripts/monitor.sh**
- Before: immediately restarted runner whenever process was down, regardless of exit reason
- Now: checks latest _diag/Runner_*.log for SessionConflictException; if found, writes .last_session_conflict timestamp and exits; subsequent monitor runs skip restart until 5-minute cooldown clears
- Why: prevents infinite restart loop after reboots where GitHub session TTL (~4 min) exceeds the monitor interval

## Known Limitations
- Live copy at ~/actions-runner/monitor.sh must be synced manually after merge
- Cooldown is 5 min; adjustable via CONFLICT_COOLDOWN if needed

> [!NOTE]
> **Medium Risk**
> Changes runner self-healing behavior and adds a new `launchd` agent; a false-positive conflict detection or misconfigured interval could delay restarts or increase churn on the runner host.
> 
> **Overview**
> Prevents post-reboot restart loops for the self-hosted GitHub Actions runner by teaching `monitor.sh` to detect `SessionConflictException` in the latest `_diag` log, write a timestamp stamp file, and skip restarts until a configurable cooldown (`CONFLICT_COOLDOWN`, default 5 minutes) expires.
> 
> Hardens the monitor’s behavior and logging (strict bash flags, error trap, safer `pgrep`/`ls` usage, and clearer service-vs-`run.sh` startup messages).
> 
> Updates `setup-github-runner-redundant.sh` to generate the new monitor script, install a new `launchd` LaunchAgent (`com.github.actions-runner-monitor.plist`) to run the monitor at login and every 15 minutes, and refreshes the final “triple redundancy”/ops instructions accordingly.
