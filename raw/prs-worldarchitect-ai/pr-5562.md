## Goal

**Run fewer tests on CI.** Use `pytest-testmon` to skip tests whose dependencies haven't changed.

## What This PR Does

### 1. Fixes 8 "Ghost Tests" (prerequisite)
Tests that passed silently via `python3 file.py` but failed under `pytest` due to assertion mismatches. Fixed to work with both runners:
- `test_budget_path_consistency.py`, `test_cerebras_provider.py`, `test_evidence_utils_steps_conversion.py`, `test_fixed_size_component_overflow.py`
- `test_gemini_native_tools.py`, `test_output_token_budget_regression.py`, `test_preventive_guards.py`, `test_session_header_coverage.py`

### 2. Adds `pytest-testmon` for test reduction
- New `--testmon` flag for `run_tests.sh`
- Testmon tracks file dependencies; skips tests when deps haven't changed
- First run: `--testmon-noselect` records baseline deps
- Subsequent runs: `--testmon` deselects unchanged test files
- Serial execution forced to avoid SQLite contention

### 3. Coverage + Testmon integration (pre-flight check)
- Uses testmon as a **pre-flight check** before `coverage run`
- Runs `pytest --testmon --collect-only -q` to check if the file needs re-running
- If testmon says "no changes": skip `coverage run` entirely (prior coverage is valid)
- If testmon says "needs re-run": use original `coverage run --parallel-mode` path
- **Preserves test runner compatibility** — never switches from `coverage run` to `pytest --cov`

### 4. CI workflow updates
- Coverage shards (`core-mvp-*`) on PRs: `--testmon` pre-flight check + `coverage run`
- Push-to-main: all shards use testmon for baseline building
- Non-coverage shards on PRs: use testmon for direct test reduction
- Fixed coverage condition: `core-` → `core-mvp-` for coverage data steps

## Dependencies
- `pytest-testmon>=2.2.0,<3.0.0`
- `pytest-cov>=4.0.0`
