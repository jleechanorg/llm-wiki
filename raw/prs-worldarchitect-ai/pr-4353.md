## Summary
Refactors the `/commentreply` command to post a single consolidated summary comment containing all responses instead of posting individual threaded replies. This reduces notification spam while maintaining 100% coverage of all comments.

## Changes
- **Consolidated Reply Mode**: Replaced individual threaded reply posting with collection-based approach that embeds all responses in one summary comment
- **Updated `post_consolidated_summary()` function**: New function that posts a single comprehensive comment with:
  - Processing overview (total comments, issues, coverage statistics)
  - Category breakdown (CRITICAL/BLOCKING/IMPORTANT/ROUTINE)
  - Response type breakdown (FIXED/DEFERRED/ACKNOWLEDGED/NOT_DONE)
  - All individual responses with `Re: [Comment #<id>]` references for traceability
- **Removed `post_initial_summary()` and `post_final_summary()`**: Consolidated their functionality into single posting step
- **Removed pending review cleanup logic**: No longer needed since we're not using threaded replies
- **Updated main loop**: Changed from posting replies individually to collecting them for batch summary
- **Updated documentation**: Modified CLAUDE.md, commentreply.md, and commentreply.toml to reflect consolidated approach
- **Updated PR automation guidance**: Modified jleechanorg_pr_monitor.py to instruct users to run `/commentreply` for consolidated summary
- **Added consolidated comment detection**: Enhanced `_has_unaddressed_comments()` to recognize consolidated summary format when checking for addressed comments

## Benefits
- **Reduces notification spam**: One comment instead of N comments for N issues
- **Improves visibility**: All responses visible in single view without scrolling through thread
- **Maintains traceability**: Each response clearly references original comment with `Re: [Comment #<id>]` links
- **Cleaner PR experience**: Less cluttered comment section while still achieving 100% coverage
- **Simpler implementation**: Eliminates
