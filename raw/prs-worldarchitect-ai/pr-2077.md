## Raw Body
Based on ai_universe_frontend PR #53, with improvements to fix identified bugs.

Root cause: Complex round-robin logic was unnecessary. The real issue was reserve_server() failing for non-existent Cloud Run services (s2-s10).

Changes:
1. Simplified assignment logic to "first available" (from ai_universe)
2. Keep reserve_server() to prevent race conditions (fixes ai_universe bug)
3. Allow reserving non-existent services - label set during first deploy
4. Keep robust null timestamp handling in eviction (fixes ai_universe bug)

Result: Proper server distribution s1→s2→...→s10 based on availability.

Fixes: "S1 always evicted and S2 or higher never used"
