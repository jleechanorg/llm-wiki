## Summary
Reinforces the shell-safety protocol to prevent git commit hangs caused by multi-line `-m` messages in IDEs (AntiGravity, Cursor).

## Root cause (from log investigation)
- IDEs parse multi-line tool output and execute **continuation lines** (e.g. `- bullet point`) as **separate shell commands**.
- Example: `git commit -m "fix: foo\n\n- item 1"` → the line `- item 1` runs as its own command → `command not found` → UI stays stuck on "Running".

## Changes
- **shell-safety.md**: Added explicit "two failure modes" (terminal EOF + IDE parsing bug), PREFER single-line, anti-patterns section.
- **GEMINI.md**: Added bullet under Anti-Hang Policy: "NEVER multi-line git commit -m".
