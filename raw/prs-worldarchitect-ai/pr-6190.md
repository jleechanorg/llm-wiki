## Summary

- Gate 6 (evidence review) permanently fails for PRs that don't touch `testing_mcp/**` or `testing_ui/**`
- `evidence-gate.yml` only triggers on those paths — so no evidence check-run exists for other PRs
- `evidence-review-bot` is not a real GitHub user — approval path is also unavailable
- Result: PRs like #6179 (touching only `mvp_site/`) are permanently blocked at Gate 6

## Fix

When `EVIDENCE_CHECK_TOTAL=0` and `EVIDENCE_APPROVED=0`, check the PR's changed files:
- If no `testing_mcp/` or `testing_ui/` paths changed → evidence gate is N/A
- PASS Gate 6 when a video/recording link is present (gist, asciinema, etc.)
- FAIL Gate 6 with clear message when video link is missing

## Production Code Changes

**Before:** `else: EVIDENCE_STATUS="FAIL (missing evidence approval/check)"` — always fails for non-testing_mcp PRs

**After:** Checks PR changed files; if evidence gate N/A and video present → PASS
