## Summary

- Added unit tests for both fixes (33 passing total)

## Production Code Changes

**\`mvp_site/llm_service.py\`** — context window prefix stripping
- Before: \`_get_context_window_tokens(model_name)\` and \`_get_safe_output_token_limit\` used raw \`model_name\` as dict key
- Now: strip \`openclaw/\` prefix via \`model_name.removeprefix(\"openclaw/\")\` before lookup
- Why: \`MODEL_CONTEXT_WINDOW_TOKENS\` keys are bare (e.g. \`\"gemini-3-flash-preview\"\`) but OpenClaw models use prefixed names (e.g. \`\"openclaw/gemini-3-flash-preview\"\`)

**\`mvp_site/llm_providers/openclaw_provider.py\`** — runtime env read for gateway URL
- Before: \`_get_gateway_url()\` returned module-level \`DEFAULT_GATEWAY_URL\` captured at import time
- Now: reads \`OPENCLAW_GATEWAY_URL\`, \`OPENCLAW_GATEWAY_HOST\`, \`OPENCLAW_GATEWAY_PORT\` from env at call time
- Why: \`_resolve_gateway_url(gateway_port=X)\` already read env at runtime — inconsistency broke test isolation
