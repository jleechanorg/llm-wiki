## Changes
1. **24-hour sessions**: Agents now stay alive for 24 hours (86400 seconds) instead of 1 hour
   - Allows extended monitoring of long-running tasks
   - Easier debugging with `tmux attach -t [agent-name]`

2. **Optional PR creation**: 
   - Agents no longer automatically create PRs for every task
   - PR creation only happens when explicitly mentioned in task or for significant changes
   - Reduces PR noise from minor changes

3. **Better PR handling**:
   - Clear distinction between updating existing PRs vs creating new ones
   - Agents working on existing PRs commit directly to the PR branch
   - New tasks only create PRs when needed

## Testing
- Tested with multiple agent tasks
- Verified 24-hour timeout works correctly
- Confirmed PR creation is now conditional based on task requirements
