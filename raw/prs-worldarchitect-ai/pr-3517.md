## Summary
Adds pre-flight validation to check agent API availability/quota BEFORE starting tasks, preventing failures due to quota limits.

## Problem
Currently, agents start tasks and only detect quota/rate limit errors AFTER execution begins. This causes:
- Wasted time starting tasks that will fail
- No automatic fallback until after failure
- PR #3096 stuck because Gemini quota exhausted

## Solution
- **Pre-flight validation**: Test CLI availability/quota before starting
  - Gemini: Test with minimal prompt, detect quota errors
  - Codex: Test basic connectivity  
  - Claude/Cursor: Verify binary executable (OAuth-based)
- **Automatic fallback**: If validation fails, try next CLI in chain
- **Fallback to any available CLI**: If all chain CLIs fail, try other CLIs

## Changes
- Added `_validate_cli_availability()` method to test CLI before use
- Pre-flight validation runs for all CLIs in chain before starting task
- Automatic fallback to validated CLI if primary fails
- Falls back to Codex (or other CLIs) if Gemini has quota issues
