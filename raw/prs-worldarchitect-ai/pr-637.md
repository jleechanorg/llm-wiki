# Learning: Branch Header Upstream Tracking Protocol

## 🚨 Critical Rule Addition

**Problem**: When creating new branches and pushing them, not setting upstream tracking causes branch headers to show "no upstream" instead of the actual remote branch, violating the mandatory branch header protocol.

## Evidence
- Created `clean-learn-memory-integration` branch  
- Used `git push origin HEAD:branch-name` without `-u` flag
- Header showed `[Local: clean-learn-memory-integration  < /dev/null |  Remote: no upstream | PR: #636]`
- Should have shown `[Local: clean-learn-memory-integration | Remote: origin/clean-learn-memory-integration | PR: #636]`

## Solution Added to CLAUDE.md

Added new rule to Git Workflow section:
- **🚨 Upstream Tracking**: Set tracking to avoid "no upstream" in headers
- **Commands**: `git push -u origin branch-name` OR `git branch --set-upstream-to=origin/branch-name`
