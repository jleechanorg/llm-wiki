## Summary

Comprehensive roadmap for automating token optimization in Claude Code CLI to address the 50-60% weekly quota consumption from multi-day coding sessions.

## Problem

**Evidence from analysis**:
- Largest session: 8,207 messages, 27.4 MB, 4.5 days duration
- Tool calls: 2,436 (mostly full file reads, large git diffs)
- Context cost: 90%, Response cost: 10%
- Single session consumed ~50% of weekly quota

**Root cause**: Token accumulation outpaces auto-summarization benefits.

## Solution

**Hooks-based automation** (verified as feasible):
- ✅ Targeted reads via Read tool offset/limit parameters (40-50% reduction)
- ✅ Minimal git diffs via Bash hook injection (30-40% reduction)
- ✅ CLAUDE.md optimization for prompt caching (automatic)
- ✅ Session health monitoring (500/1000/2000 message thresholds)
