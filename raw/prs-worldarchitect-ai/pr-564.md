## Problem
The git-header script was using complex logic to search through commits and find "related" PRs, which caused incorrect PR associations. For example, showing PR #551 for a completely unrelated branch.

## Solution
- Removed the complex commit searching logic (lines 70-90)
- Now only uses `gh pr list --head "$local_branch"` for exact branch match
- Shows "PR: none" when no PR exists for the current branch
- Much simpler and more accurate
