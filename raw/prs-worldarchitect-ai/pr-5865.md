> [!NOTE]
> **Medium Risk**
> Touches the generated runner script execution path for all CLI attempts, so misconfigured caps or hosts where `ulimit -v` behaves differently could cause agent runs to fail early. Changes are localized and guarded with defaults and warnings.
> 
> **Overview**
> Adds a configurable *virtual memory cap* for child CLI processes spawned by orchestration runner scripts via new `ORCHESTRATION_CHILD_PROCESS_VMEM_CAP_KB` (default 4 GiB, env-overridable with validation).
> 
> Updates `TaskDispatcher` runner generation to wrap each CLI attempt with a bash guard that applies `ulimit -v` before executing the actual command (failing fast if the cap can’t be applied) and logs the cap value at agent start.
> 
> Extends `test_cli_support.py` with a regression test asserting the generated script includes the memory-cap wrapper and uses the configured cap.
