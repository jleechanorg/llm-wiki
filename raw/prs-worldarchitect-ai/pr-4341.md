## Problem

The batch summary approach posted about ALL merged PRs, creating shallow coverage:
- 8 PRs → 800 words total → 100 words per PR
- No editorial judgment on what's actually interesting
- Re-analyzed same PRs repeatedly
- Breadth over depth

## Solution

**Editorial Selection:**
- LLM reviews all unanalyzed PRs
- Picks ONE most architecturally interesting topic
- Writes 600-1200 word deep dive
- Tracks analyzed PRs in state (last 100)
- Filters out already-analyzed PRs

**Prompt Changes:**
```
Pick ONE interesting topic from these PRs and write a deep technical post about it.

Look for: novel patterns, tradeoffs, system design decisions, failure modes addressed
Ignore: routine fixes, typos, dependency updates, simple refactors
```
