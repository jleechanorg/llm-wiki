## Background
The previous campaign creation wizard was a 4-step process that required users to click through multiple screens to reach launch, adding unnecessary friction. The AI toggle constraints were also overly aggressive and confusing to users, wiping out Custom Campaign preferences if users toggled between campaign types. Additionally, avatar file state leakages were causing avatars from abandoned campaign creations to bleed into subsequent sessions.

## Goals
- Streamline the campaign creation wizard from 4 steps down to 2 steps (Type Selection -> Launch).
- Completely eliminate the "AI Storytelling Options" (Default World, Mechanical Precision, Starting Companions) from the UI to reduce user friction and confusion.
- Enforce strict AI prompt defaults purely on the backend/JS serialization level (Dragon Knight strictly uses the Default World, Mechanics, and Companions; Custom strictly avoids the Default World but keeps Mechanics and Companions enabled).
- Preserve the Custom Campaign's description prompt textarea so players can still feed customized lore into their sandbox campaign generation.
- Bulletproof the campaign wizard state lifecycle to ensure Avatar components and listeners are cleanly purged during wizard reset scenarios.

## Tenets
- **Frictionless Onboarding:** Getting a player into the game should require minimal mandatory steps. Eliminate granular wizard checkboxes that universally slow down character creation.
- **State Integrity:** Components generated natively in the browser must never leak state across wizard resets or campaign type swaps.
- **Fail-Safe Processing:** AI parameters should be derived automatically via campaign-type context logic, preventing UI-driven payload discrepancies.
