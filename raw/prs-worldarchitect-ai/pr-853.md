## Rules Added

### 1. NO PREMATURE VICTORY DECLARATION
- Never declare success based on intermediate steps
- Task completion requires FULL verification
- For agents: PR must be created, pushed, and verified

### 2. AGENT TASK PATIENCE
- Orchestrate agents work autonomously and may take 5-10+ minutes
- NEVER declare success OR failure without checking completion status
- Proper verification: Check tmux for "Task completed" → Find PR URL → Verify PR exists
- Status warnings like "agent may still be working" mean WAIT

- Agent task-agent-5819 modified `schedule_branch_work.sh` successfully
