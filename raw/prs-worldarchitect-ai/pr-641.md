## Summary

Implements automatic clean branch workflow for `/learn` command to fix the issue where learning PRs were being created from current branch instead of clean main branch.

## Problem Solved

User reported: *"seems like these are not clean branches from main like they are supposed to be?"*

The `/learn` command documentation specified "Clean Branch: Create independent learning PR from fresh main branch" but the implementation was creating branches from the current branch, causing learning PRs to inherit unrelated changes.

## Implementation

### 🚀 **Key Features Added:**

1. **Automatic Clean Branch Creation**: `/learn` now automatically creates clean branches from main instead of current branch
2. **Stash Management**: Automatically stashes uncommitted changes before switching branches  
3. **Branch Management Functions**: 
   - `get_current_branch()` - saves current state
   - `switch_to_main_and_update()` - ensures clean main with latest changes
   - `create_clean_learning_branch()` - creates timestamped learning branch from main
   - `return_to_original_branch()` - restores original context
4. **Automatic PR Creation**: `create_learning_pr()` function for automatic PR generation
5. **Error Handling**: Comprehensive error handling with guaranteed branch restoration
6. **Memory MCP Integration**: Placeholder for Memory MCP entity creation

### 🔧 **Technical Implementation:**

- **Clean Branch Workflow**: All learning branches now created from fresh main branch
- **Timestamped Branch Names**: `learn-YYYYMMDD_HHMMSS` format for unique branch names
- **Automatic Stashing**: Handles uncommitted changes gracefully
- **Branch Restoration**: Always returns to original branch, even on errors
- **PR Automation**: Creates properly formatted learning PRs with context
