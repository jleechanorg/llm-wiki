## Summary

## New Features Added

### PR Timing Metrics
- **Open-to-merge time analysis** with statistical breakdown
- **Median, average, and 95th percentile** timing calculations
- **Fastest/slowest merge** tracking
- **Ignores open PRs** as requested

### DORA Metrics (DevOps Research and Assessment)
- **Deployment Frequency**: PRs merged per day (proxy for deployments)
- **Lead Time for Changes**: Median PR creation-to-merge time
- **Mean Time to Recovery**: Median time to resolve fix PRs
- **Change Failure Rate**: Ratio of fix PRs to feature PRs

## Technical Implementation
- Added `calculate_pr_timing_metrics()` function for statistical PR analysis
- Added `calculate_dora_metrics()` function for standardized DevOps metrics
- Enhanced GitHub CLI data collection to include `closedAt` timing data
- Integrated Python `statistics` module for robust statistical calculations
- Updated main output to display new metrics sections
