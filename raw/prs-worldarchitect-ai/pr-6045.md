## Background
Needed a reliable way to identify which slash commands are actually used vs. mentioned in docs/diffs. Previous ripgrep approach had massive false positives (e.g., /test matching file paths, /fake from AO worker automation injections).

## Goals
- Count actual human-typed slash command invocations from ~/.claude/projects history
- Filter out AO worker automation, agent context injections, and file path references
- Handle edge cases: digit-prefixed commands (/4layer), mid-sentence mentions

## Tenets
- Real signals over heuristics where possible
- Automation noise must be explicitly excluded
- Script should be re-runnable with configurable time windows

## High-level description of changes
scripts/count_command_usage.py uses three detection tiers in priority order:
1. Primary: `<command-name>/cmd</command-name>` XML tags — Claude Code canonical invocation format, definitive signal
2. Fallback 1: /cmd at the start of a line in short messages (< 500 chars)
3. Fallback 2: /cmd anywhere in short messages with word boundary filtering — catches mid-sentence mentions like 'i am saying /4layer now'

Automation filtered via explicit markers: AO worker injections, agent context, continuation sessions, README update agents.

doc/command_usage.md contains the generated report for the last 30 days.
