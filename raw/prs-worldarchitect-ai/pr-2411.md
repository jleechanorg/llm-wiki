## Summary

This PR introduces a robust **Agent Architecture** for WorldArchitect.AI, enabling clean separation of concerns between different interaction modes (Story Mode vs God Mode). Each agent encapsulates mode-specific logic and has a focused subset of system prompts.

## Key Changes

### New Agent Architecture (`mvp_site/agents.py`)

| Class | Purpose | Prompt Set |
|-------|---------|------------|
| **BaseAgent** | Abstract base class with common functionality | Defines interface |
| **StoryModeAgent** | Narrative storytelling, character actions, gameplay | `master_directive`, `game_state`, `dnd_srd` + optional `narrative`, `mechanics`, `character_template` |
| **GodModeAgent** | Administrative commands, state modifications | `master_directive`, `god_mode`, `game_state`, `dnd_srd`, `mechanics` |

### Factory Function
```python
from mvp_site.agents import get_agent_for_input

agent = get_agent_for_input(user_input, game_state)
# Returns GodModeAgent if input starts with "GOD MODE:"
# Returns StoryModeAgent for all other inputs
```

### Architecture Benefits
- **Focused Prompt Sets**: Each agent loads only the prompts it needs, reducing token usage
- **Clean Separation**: Mode-specific logic is encapsulated in dedicated classes
- **Extensible**: Easy to add new agent types for future modes
- **Backward Compatible**: Re-exports from `llm_service.py` maintain existing imports
