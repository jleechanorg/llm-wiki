## Summary
This enhancement makes the `/orch` command automatically display any PRs created by orchestration agents at the end of execution, providing immediate feedback without requiring manual `gh pr list` checks.

## Changes
- **Added `_check_and_display_prs()` method** to `orchestrate_unified.py`
  - Waits up to 30 seconds for agents to create PRs
  - Checks each agent workspace for PR creation
  - Displays PR number, title, URL, and status
- **Enhanced help handling** for `--help` flag
- **Improved user feedback** with clear messages when no PRs detected yet

## Example Output
When agents create PRs, you'll now see:
