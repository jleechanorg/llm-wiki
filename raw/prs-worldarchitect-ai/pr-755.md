## Overview
Two slash command improvements to streamline workflow:
1. Enhance `/handoff` to automatically update roadmap.md
2. Create new `/commentreply` command for systematic PR comment handling

## Changes

### 1. /handoff Enhancement
- Integrates `/r` functionality to automatically add tasks to roadmap.md
- No more manual roadmap updates after creating handoffs
- Maintains consistent task tracking

### 2. New /commentreply Command
- Systematically addresses ALL GitHub PR comments
- Fetches inline, general, and review comments
- Marks each as ✅ DONE or ❌ NOT DONE with explanation
- Posts inline replies via GitHub API
- Provides summary of addressed comments

## Implementation Details

### /handoff Changes
- After PR creation, invoke `/r` logic
- Add HANDOFF-{TASK} entry to roadmap.md
- Include PR number and status

### /commentreply Features
- Alias: `/commentr` for convenience
- Handles all comment types (inline, reviews, general)
- Clear status marking (DONE/NOT DONE)
- Automated inline replies
- Comprehensive summary
