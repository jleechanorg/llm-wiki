## Bug fix: division by zero in faction battle sim

**Location:** `mvp_site/faction/battle_sim.py` in `_calculate_round_casualties_fast()`

**Issue:** When all defenders have `stats["hp"] == 0` (malformed or custom faction data), the code raised `ZeroDivisionError`.

**Fix:** Add guard before division — when `weighted_avg_defender_hp <= 0`, return 0 casualties instead of crashing.

**Regression tests:** Added `TestZeroHpDivisionByZeroBug`.
