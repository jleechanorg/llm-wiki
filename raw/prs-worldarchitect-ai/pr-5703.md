## Production Code Changes

**Before:** `_inject_levelup_choices_if_needed` only checked `game_state.rewards_pending.level_up_available`. When LLM explicitly clears `rewards_pending=null` (common in deferred rewards scenario), injection was skipped entirely.

**Now:** Function also checks `rewards_box.level_up_available` from the current LLM response. When `rewards_pending=null` but `rewards_box.level_up_available=true`, synthesizes a minimal `rewards_pending` dict so injection proceeds.

**Why:** Deferred rewards protocol (`deferred_rewards_instruction.md`) correctly awards XP via `rewards_box` and shows the level-up indicator, but the LLM can simultaneously clear `rewards_pending` via `state_updates=null`. The server must handle this split-signal case to guarantee clickable level-up buttons whenever the indicator is visible.
