## Summary
This PR enhances the orchestration system to intelligently detect when agents should UPDATE existing PRs versus CREATE new ones, preventing duplicate PRs when users reference existing work.

## Problem Solved
Previously, when users said "adjust the PR" or "fix PR #123", orchestration agents would create new PRs instead of updating the existing ones. This led to PR proliferation and confusion.

## Key Features

### 🔍 **Intelligent PR Detection**
The system now recognizes patterns like:
- **Explicit references**: "fix PR #123", "update pull request #456"
- **Contextual references**: "adjust the PR", "the PR needs more tests"
- **Action phrases**: "continue with the PR", "add logging to PR #200"

### 📢 **Clear User Feedback**
Users see immediate confirmation of the detected mode:
```
🔍 Detected PR context: #950 - Agent will UPDATE existing PR
   Branch: feature-xyz
   Status: OPEN
```
OR
```
🆕 No PR context detected - Agent will create NEW PR
   New branch will be created from main
```

### 🤖 **Mode-Specific Agent Instructions**
- **UPDATE mode**: Agents receive instructions to checkout PR branch and push updates
- **CREATE mode**: Agents create new branches from main and new PRs
- **Ambiguous references**: Agents check recent PRs and ask for clarification
