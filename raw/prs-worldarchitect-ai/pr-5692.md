## Summary
- Adds `no-sparse-checkout: true` to `.beads/config.yaml`
- Adds `.husky/post-checkout` hook that auto-disables sparse checkout if detected

## Problem
`bd migrate-sync` enables sparse checkout with pattern `/.beads/` only. When the same worktree is later used for development, all files outside `.beads/` are invisible — `.claude/scripts/`, `roadmap/`, etc. disappear from disk silently.

## Test plan
- [ ] Run `git sparse-checkout set .beads/` then `git checkout` — hook should revert it
- [ ] Verify `post-checkout` hook is executable and fires on branch switch
