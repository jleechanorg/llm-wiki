## Production Code Changes

### 1. `scripts/self-hosted/crontab-example.txt` (+52 lines)
**Purpose:** Reference crontab configuration for self-hosted runner reliability

- **Added sanitized crontab example** - Production configuration with tokens redacted → Provides users with working template
- **Hourly heartbeat schedule** - `0 * * * *` instead of `0 */6 * * *` → 6x more frequent GitHub check-ins (24/day vs 4/day)
- **15-minute monitor** - `*/15 * * * *` cron for auto-restart → Detects crashes within 15 minutes
- **Token redaction** - `GITHUB_TOKEN=<REDACTED>` → Prevents accidental exposure in version control
- **Path normalization** - Uses `~` instead of `/Users/username` → Works across different user accounts

### 2. `scripts/self-hosted/README-crontab.md` (+99 lines)
**Purpose:** Complete documentation for crontab setup and troubleshooting

- **Installation guide** - Step-by-step setup instructions → Users can configure in <5 minutes
- **Protection layers explanation** - Documents 3 redundancy layers (LaunchAgent + Monitor + Heartbeat) → Users understand why each exists
- **Clock drift workflow** - Documents sync → restore workflow for users with intentional clock drift → Supports +10min ahead clocks
- **Troubleshooting section** - Common issues and fixes → Self-service debugging
- **Security notes** - Token handling best practices → Prevents credential leaks
