## Summary

**Key themes:** secret hygiene hardening, defense-in-depth for dotfile backups

## Production Code Changes

### 1. `scripts/backup_dotfiles.sh` (+58/-0 lines net new, +filter fix)
**Purpose:** Backs up dotfiles (`~/.bashrc`, `~/.gitconfig`) with sensitive data redacted before committing to git

- **Commented-out export filter** — Filter only matched active `export VAR=value` lines → Now also matches `# export TOKEN/SECRET/API_KEY/etc=value` lines → A `sk-ant-oat01-...` OAuth token in a `# Commented out` line was previously committed to GitHub in clear text
- **`run_gitleaks_gate()`** — No safety scan before commit → Runs `gitleaks detect --no-git --redact` on `$BACKUP_DIR` before `git add` → Provides a hard block if any secret pattern is detected; prints `file:line:rule` only (never secret values); gracefully no-ops if gitleaks not installed

### 2. `scripts/dotfiles_backup/bashrc_macbook.txt`
**Purpose:** Committed redacted backup of `~/.bashrc` on macbook

- Refreshed from current `~/.bashrc` with updated filter applied; gitleaks scan: ✅ clean
