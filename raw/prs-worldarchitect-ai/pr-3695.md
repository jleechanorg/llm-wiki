This commit fixes two critical issues in the copilot comment processing system:

1. **commentreply.py**: Now processes ALL comments including reply comments
   - Changed filter from `top_level = [c for c in all_comments if not c.get("in_reply_to_id")]`
   - To: `all_targets = all_comments` (process all comments, not just top-level)
   - This achieves the documented "100% reply rate" goal by processing replies too
   - Existing filters (skip own comments, skip already replied, etc.) still apply

2. **base.py**: Fixed pagination bug in run_gh_command()
   - Added `--jq '.[]'` to flatten paginated GitHub API responses
   - Handles JSONL output format (one JSON object per line) from paginated requests
   - Returns list for paginated results, dict/object for single page
   - Prevents missing comments from later pages (was only parsing first page)

These fixes ensure /copilot and /commentfetch truly find all comments and replies, not just top-level comments or comments from the first page.
