## Raw Body
The orchestrated_pr_runner was hardcoded to use https://github.com which prevented it from working with GitHub Enterprise instances.

Changes:
- Replace hardcoded 'github.com' with os.environ.get('GH_HOST', 'github.com')
- Allows users to set GH_HOST environment variable for enterprise instances
- Maintains backward compatibility by defaulting to github.com

Affected files:
- automation/orchestrated_pr_runner.py (line 170)
- automation/jleechanorg_pr_automation/orchestrated_pr_runner.py (lines 170, 185)

Usage:
  GH_HOST=github.sc-corp.net jleechanorg-pr-monitor --target-pr 123 --target-repo Org/repo --fix-comment

Fixes the issue where fix-comment and fixpr workflows would fail with:
  Command '['git', 'clone', 'https://github.com/Org/repo.git', ...]' returned non-zero exit status 128
