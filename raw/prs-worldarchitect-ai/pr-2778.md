# Faction Management Mini-Game + Tutorial Enforcement

## Summary

Adds the strategic faction-management layer (resources, rankings, intel, SRD battle sim) and tightens the tutorial onboarding so it is **multi-turn** and **requires all main action categories** before completion. Tutorial progress is now enforced server-side to avoid LLM omissions, and the skip command reliably sets `tutorial_completed=true`.

## Highlights

### Faction Systems
- **Resources + buildings** (farms, training, guilds, fonts, fortifications, wards, shadow networks)
- **Rankings vs 200 AI factions** with unranked threshold and FP to next rank
- **Intel ops** with detection risk, cooldowns, and tiered outcomes
- **SRD-based battle simulation** (fast, detailed, deterministic)
- **Dual-mode integration** (adventure ↔ faction)

### Tutorial System (Multi-Turn)
- **Required categories (10):** status, rankings, build, recruit, intel, combat_preview, combat_action, diplomacy, council, research
- **Requires 2 end turns** across multiple strategic turns
- **Server-side tracking** of tutorial_progress and completion
- **End-turns clamped to +1 per “end turn”** to prevent double counting

## Code Changes (Core)
- New faction modules: resources/combat/intel/rankings/AI factions/tools
- SRD unit mapping and battle sim scaffolding
- Tutorial enforcement in `world_logic.py` (progress tracking + completion gate)
- Settings UI toggle for faction minigame (disabled by default)
- Evidence standards updates for test bundles
