## Changes

- **Fix session name matching**: Updated `stream_logs.sh` to handle tmux session name variations (tmux converts dots to underscores)
- **Smart session detection**: Script now tries multiple variations (exact match, trailing underscore, trailing dot) to find the correct session
- **Informative messages**: Shows which session name was actually used when auto-corrected

## Problem

When running `./orchestration/stream_logs.sh pr-3054-claude-show-resources-planning-VbJDp`, the script failed because:
- The script expected: `pr-3054-claude-show-resources-planning-VbJDp`
- Actual tmux session: `pr-3054-claude-show-resources-planning-VbJDp_` (with trailing underscore)

Tmux automatically converts dots to underscores in session names, so session names with trailing dots become trailing underscores.

## Solution

The script now tries multiple variations:
1. Exact match
2. With trailing underscore
3. With trailing dot
4. Base name without trailing characters
5. Base name variations

This ensures the script works regardless of how tmux stores the session name.
