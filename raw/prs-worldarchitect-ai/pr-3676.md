## Raw Body
Root cause: MCP smoke tests fail with "0 dice rolls returned" because the LLM didn't know when/how to populate the tool_requests array.

The issue affected ALL providers (Cerebras, Gemini 2.0, Gemini 2.5):
- For Cerebras/OpenRouter: JSON schema had tool_requests field but minimal guidance on WHEN to use it
- The detailed examples in game_state_examples.md were NOT loaded by default (dynamic/on-demand only)
- CombatAgent had proper guidance but only loads when in_combat=True
- Smoke tests create new campaigns where in_combat=False, so StoryModeAgent handled combat requests without dice guidance

The fix adds comprehensive tool_requests guidance to game_state_instruction.md:
- Wrapped in BEGIN/END_TOOL_REQUESTS_DICE markers for Gemini code_execution stripping (Gemini uses Python code execution instead)
- Clearly explains WHEN to use tool_requests (all dice situations)
- Shows exact JSON format with working example
- Lists all available tools (roll_dice, roll_attack, roll_skill_check, etc.)
- Explicitly states that empty tool_requests[] during combat is a FAILURE

This guidance is now visible to ALL agents (Story, Combat, etc.) and all non-code_execution providers (Cerebras, OpenRouter).
