## Summary

This PR implements comprehensive fixes for **CharacterCreationAgent** lifecycle management, God Mode template support, and E2E testing infrastructure. Addresses multiple character creation bugs including flag clearing, template population, and agent transition failures.

## Key Changes

### 1. CharacterCreationAgent Flag Clearing Fix (mvp_site/agents.py)
**Problem**: `character_creation_in_progress` flag wasn't clearing when user accepted character, causing agent to loop indefinitely.

**Root Cause**: Agent selection logic checked completion phrase AFTER `CharacterCreationAgent.matches_game_state()`, so flag was cleared but CharacterCreationAgent was already selected.

**Solution**: Restructured logic to check completion phrase BEFORE `matches_game_state()`:
- Lines 1406-1447: Check for completion phrase first
- Clear flag and transition to StoryModeAgent BEFORE checking matches_game_state
- Add type checking for Mock objects in tests (turn_number comparison)

### 2. God Mode Character Template Support (mvp_site/mcp_api.py)
**Problem**: God Mode templates weren't populating character data (name, race, class, level).

**Root Cause**: MCP `create_campaign` tool schema was missing `god_mode` parameter entirely.

**Solution**: Added god_mode parameter with complete schema (lines 87-103):
```json
"god_mode": {
  "type": "object",
  "properties": {
    "character": {
      "name": {"type": "string"},
      "race": {"type": "string"},
      "class": {"type": "string"},
      "level": {"type": "integer"}
    }
  }
}
```

### 3. Character Creation Instructions - Explicit Schemas (mvp_site/prompts/character_creation_instruction.md)
**Problem**: LLM wasn't following flag clearing instructions reliably.

**Solution**: Added UNCONDITIONAL instructions and explicit INPUT/OUTPUT schemas:
- Lines 1-50: CRITICAL flag clearing rules (MUST BE FALSE. ALWAYS.)
- Lines 100+: Explicit INPUT schema (what LLM receives)
- Lines 200+: Explicit OUTPUT schema (what LLM must return)
