## Summary

Fixes the cerebras-coder agent failing with "❌ CRITICAL ERROR: No Cerebras API key found!" despite `CEREBRAS_API_KEY` and `OPENAI_API_KEY` being properly configured in `~/.bashrc`.

## Root Cause

Agents launched via the tmux orchestration system weren't inheriting the user's shell environment variables because:
1. The cerebras-coder agent validation script didn't source `.bashrc` before checking environment variables
2. The orchestration system's tmux session creation didn't source environment variables

## Changes Made

### 🔧 Agent Validation Fix (`.claude/agents/cerebras-coder.md`)
- Added `.bashrc` sourcing step before API key validation
- Updated validation protocol to include environment setup
- Maintains backward compatibility with existing validation rules

### 🔧 Orchestration System Fix (`orchestration/agent_monitor.py`)
- Modified tmux session creation to source `~/.bashrc` before running claude
- Added graceful fallback (`2>/dev/null || true`) to handle missing bashrc
- Ensures all future agents inherit proper environment variables
