## Raw Body
## Summary

Fixes recurring macOS permission dialog by removing explicit "bash" from tmux new-session commands. Agent scripts now execute directly via their shebang (`#!/usr/bin/env bash`), avoiding macOS security prompts for bash to "access data from other apps."

## Root Cause

The issue occurred because:
1. The tmux command explicitly invoked `bash` before the script path
2. macOS grants permissions to bash binaries by their specific path
3. System has two bash binaries: `/bin/bash` and `/opt/homebrew/bin/bash`
4. Cron jobs use homebrew bash (via PATH), which needed separate permission
5. Each new bash binary or update requires re-granting permission

## Changes

- **orchestration/task_dispatcher.py**: Remove "bash" argument from tmux new-session command (line 1992)
- **orchestration/tests/test_tmux_bash_removal.py**: Add comprehensive unit test to verify bash is not in tmux command

## Technical Details

**Before:**
```python
tmux_cmd.extend([
    "new-session", "-d", "-s", agent_name,
    "-c", agent_dir,
    "bash",  # ❌ Explicit bash invocation
    str(script_path),
])
```

**After:**
```python
tmux_cmd.extend([
    "new-session", "-d", "-s", agent_name,
    "-c", agent_dir,
    str(script_path),  # ✅ Direct execution via shebang
])
```
