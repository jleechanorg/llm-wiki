> - Backend: `get_story_paginated` adds `before_id` cursor, orders by `timestamp` then document ID, uses `start_after` when both are provided, fetches `limit+1` to compute `has_older`, and returns `oldest_timestamp` + `oldest_id`; normalizes timestamps and includes IDs. `get_story_count` handles missing aggregation API and runtime failures.
