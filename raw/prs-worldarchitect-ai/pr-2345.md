When a PR has a head commit from Codex, the automation now checks if there are new GitHub bot comments (like CI failures, review bot comments) posted after the last Codex automation comment. If such comments exist, the automation proceeds instead of skipping.

This prevents missing important bot feedback when the latest commit was already made by Codex/automation.

- Add _is_github_bot_comment() to detect GitHub bots (excluding Codex bots)
- Add _get_last_codex_automation_comment_time() to find latest Codex comment
- Add _has_new_bot_comments_since_codex() to check for new bot comments
- Modify skip logic to check for new bot comments before skipping
- Add comprehensive tests for bot comment detection
