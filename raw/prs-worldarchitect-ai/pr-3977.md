## Problem Statement

User selects god mode radio button and types query, but LLM receives `game_mode: "character"` instead of `game_mode: "god"`.

**Evidence from GCP logs:**
- User confirmed GodModeAgent was selected in debug_info
- But RAW_REQUEST shows `"game_mode": "character"` being sent to LLM
- LLM rolled dice and advanced narrative (story mode behavior)

## Root Cause Investigation

The mode parameter is being changed somewhere between:
1. Agent selection (line 3316) - where GodModeAgent IS selected correctly
2. LLM request building (line 3685) - where mode has become 'character'

Checked and ruled out:
- ✅ Frontend IS sending correct mode (GodModeAgent wouldn't be selected otherwise)
- ✅ `_is_god_mode_return_to_story()` returns False (tested with user's actual input)
- ✅ No other obvious mode reassignments found in world_logic.py or llm_service.py

## This PR: Diagnostic Logging

Added explicit logging at llm_service.py:3669 to capture:
- `mode` parameter value
- `agent.__class__.__name__`
- `agent.MODE` constant

**Log format:**
```
🐛 DEBUG_MODE_VALUE: mode='god', agent=GodModeAgent, agent.MODE='god'
```

This will reveal:
- If mode is still 'god' at that point → bug is in LLMRequest.build_story_continuation()
- If mode is already 'character' → there's a hidden override we haven't found
