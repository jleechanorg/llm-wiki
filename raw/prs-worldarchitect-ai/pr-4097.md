## Summary

Follow-up to PR #4096 - adds enhanced error logging for Gemini API failures to help diagnose cryptic errors like `FAILED_PRECONDITION`.

## Changes

Extract detailed information from `google.genai.errors.APIError`:
- `e.details`: Full response JSON from Gemini
- `e.status`: Error status (FAILED_PRECONDITION, etc.)
- `e.message`: Human-readable error message
- Response headers: `x-request-id`, `x-goog-request-id` for tracing

## Context

This was developed during investigation of a `FAILED_PRECONDITION` error in campaign `9za9JS5rJWrAvJDEEj5g`. The existing error logging only showed the error message; this enhancement provides request IDs and full response details for debugging.
