## Summary

## Problem
The copilot documentation focused on "actionable issues" (implicitly technical problems like bugs, security, tests) rather than ALL feedback including human questions and style comments. This created several issues:
1. "actionable issues" language excluded questions and style comments
2. "100% comment response rate" was buried in success criteria instead of being prominent  
3. Priority order created hierarchy that devalued human feedback
4. copilot-fixpr agent focused entirely on technical fixes, not human interaction
5. No explicit requirements for human comment handling

## Solution
### 1. Language Changes
- Replace "actionable issues" → "ALL comments" throughout both files
- Emphasize "human + automated" feedback everywhere
- Make 100% comment coverage a CRITICAL REQUIREMENT, not buried criteria

### 2. Human Priority Protocol
- Add dedicated "🚨 ALL COMMENTS FIRST - MANDATORY HUMAN PRIORITY PROTOCOL" section
- Clear requirements: "Check ALL authors including human reviewers"
- Explicit: "Respond to questions, not just fix issues" 
- Emphasis: "Human comments are equally important as bot comments - Actually MORE important"

### 3. Priority Restructuring
**Before**: Security → Runtime Errors → Test Failures → Style
**After**: Human Questions → Human Style Feedback → Security → Runtime Errors → Test Failures → Automated Style

### 4. Agent Updates
- copilot-fixpr agent now has "Address ALL human feedback through code implementation" as PRIMARY responsibility
- Added explicit "HUMAN INTERACTION" section to agent responsibilities
