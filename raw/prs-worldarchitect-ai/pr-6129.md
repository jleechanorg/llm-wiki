## Summary

- **MCP preview smoke (GitHub Actions + `MCP_SERVER_URL`)**: When `SMOKE_TOKEN` is set and `MCP_TEST_MODE` is not `real`, the harness now sends `X-Mock-Services: true` so the preview server sets Flask `g.mock_services_mode` and uses the mock LLM path instead of billing Gemini for every scenario.
- **Parity**: `llm_service._is_mock_services_mode()` now treats the same `MOCK_SERVICES_MODE` env values as truthy as `world_logic.is_mock_services_mode()` (e.g. `1`, `yes`, `y`, `on`), and the MCP streaming mode-guard uses the same check.
- **Docs**: `mcp-smoke-tests.yml` and `testing_mcp/test_smoke.py` comments describe the remote-preview behavior.

## Production code changes

| Area | Before | After |
|------|--------|--------|
| Remote mock smoke | Headers omitted `X-Mock-Services`; preview could call real Gemini while runner env had mock semantics | `X-Mock-Services: true` when not in MCP real mode |
| `llm_service` mock detection | Env truthy set could diverge from `world_logic` | Aligned truthy set with `world_logic` |

## Test changes

- Remove unused `re` import in `test_main_enhancements.py` (chore).

## Known limitations

- Real-mode preview smoke (`MCP_TEST_MODE=real`) still omits `X-Mock-Services` by design (expects real LLM responses).
