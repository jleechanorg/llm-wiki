### Motivation
- Provide operators a clear, copy-paste guide to set up self-hosted GitHub Actions runners so CI remains available when GitHub-hosted runners are unavailable.
- Document how the repository's existing workflows (e.g., `claude-processor.yml`) can target `self-hosted` runners and recommended labels like `claude`.
- Fix critical bugs identified in code review to ensure safe and correct runner setup.

### Description

#### Documentation & Setup Guide
- Added comprehensive "Self-Hosted GitHub Actions Runner Setup" section to `.github/workflows/README.md` with step-by-step instructions.
- Included both direct installation (local machine) and Docker-based (docker-compose) deployment options.
- Added verification, workflow usage examples, maintenance commands, and security considerations.
- Created validation test workflow (`test-self-hosted-runner.yml`) for runner diagnostics.
- Added comprehensive test report documenting runner v2.331.0 validation.

   - **After:** Moved color variable definitions to line 40 (after configuration constants)
   - **Impact:** Error messages displayed literal `${RED}` text instead of colors

   - **Before:** Checked for `Runner.Listener.dll` (Windows-only path)

   - **Impact:** Test always reported "binary not found" on macOS/Linux
