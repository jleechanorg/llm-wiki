## Raw Body
### Motivation
- Eliminate slow, inconsistent keyword-based fallbacks that bypass the semantic intent classifier and reintroduce regressions in routing and latency.  
- Centralize agent routing on the classifier/state priorities and keep a single, observable default fallback path to StoryModeAgent.

### Description
- Removed keyword-based input-pattern fallbacks for faction and rewards from `get_agent_for_input()` so routing now follows (state overrides → semantic classifier → default StoryModeAgent).  
- Cleaned up dialog continuity checks and tightened DialogAgent `matches_game_state()` logic to correctly evaluate last action and dialog context.  
- Updated `FixedPromptAgent`/prompt-building methods and a number of formatting/lint changes (PEP/ruff fixes and small refactors) to satisfy pre-commit checks.  
- Adjusted/removed tests that relied on the keyword-fallback behavior and updated `mvp_site/tests/test_agents.py` to match the classifier/state-driven routing model.
