## Summary
Removes the `CI_TEST_LIMIT` test truncation logic that was arbitrarily limiting discovered tests during CI runs. This change ensures all critical tests are executed instead of being skipped due to performance constraints.

## Changes
- Removed `CI_TEST_LIMIT=50` export from `.github/workflows/test.yml`
- Disabled test file truncation logic in `run_tests.sh` that was limiting test execution
- Removed priority-based filtering that attempted to preserve `test_end2end/` tests while truncating others
- Added comprehensive comments explaining why truncation is unsafe and what alternatives exist

## Benefits
- **Prevents test skipping**: Critical regression tests (e.g., `mvp_site/tests/test_provider_inference.py`) are no longer at risk of being dropped
- **Safer CI**: Directory-based scoping via `--test-dirs` combined with `--exclude-integration` and `--exclude-mcp` already provides sufficient test limiting without arbitrary truncation
- **Better maintainability**: Removes fragile priority-based logic that was difficult to reason about
- **Clear guidance**: Comments document that per-directory sharding or allowlists should be used if performance becomes an issue, rather than arbitrary truncation
