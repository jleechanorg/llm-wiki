## Problem Fixed

The orchestration system had a **critical bug** where agents executed stale tasks from previous runs instead of newly requested tasks.

### Root Cause
- Task prompts stored in `/tmp/agent_prompt_*.txt` files persisted across orchestration runs
- When "restarting" agents, system reused old prompt files instead of generating fresh ones
- **Result**: Agents worked on completely unrelated tasks (e.g., PR comment responses instead of server modifications)

### Evidence
- Found **289 stale prompt files** in `/tmp/` containing tasks from previous orchestration sessions
- Agents reported "success" but for entirely different work than user requested
- Example: User requested "modify run_local_server.sh", agents executed "PR #1118 comment responses"

### Solution Implemented
1. **Per-agent cleanup**: `_cleanup_stale_prompt_files()` method in TaskDispatcher
2. **Startup cleanup**: `_cleanup_stale_orchestration_state()` method in UnifiedOrchestration  
3. **Age-based cleanup**: Remove prompt files older than 5 minutes on startup
4. **Session cleanup**: Clean up completed tmux sessions to prevent resource leaks

### Impact
- ✅ **Fixes critical system reliability issue** - `/orch` commands now execute correct tasks
- ✅ **Prevents wasted computational resources** on unrelated work
- ✅ **Enables reliable orchestration** for production use
- ✅ **Tested**: 289 → 0 stale prompt files cleaned up successfully

### Files Changed
- `orchestration/task_dispatcher.py`: Added per-agent prompt cleanup
- `orchestration/orchestrate_unified.py`: Added startup state cleanup

This was a **system-critical bug** that made orchestration unreliable. The fix ensures fresh task prompts for every orchestration request.
