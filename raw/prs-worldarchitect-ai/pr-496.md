## Summary
- Fixes issue where entity validation was always triggered due to 'Unknown' being treated as a missing entity
- 'Unknown' is used as a default location name when location is not found in world_data
- This was causing unnecessary dual-pass generation attempts
- **NEW**: Removed deprecated planning block extraction from narrative text

## Problem
From the logs:
```
ENTITY_TRACKING_VALIDATION: Narrative failed entity validation
Missing entities: ['Unknown']
ENTITY_TRACKING_RETRY: Attempting dual-pass generation to fix missing entities
```

The system was treating the default location name 'Unknown' as an entity that should appear in the narrative.

## Solution
1. **Filter 'Unknown' from expected entities** in both:
   - `DualPassGenerator.generate_with_dual_pass()` - filters before processing
   - `EntityValidator.validate_entity_presence()` - filters during validation

2. **Added comprehensive tests** to ensure:
   - 'Unknown' is properly filtered from validation
   - Real missing entities are still detected
   - Edge case handling when only 'Unknown' is in expected entities

3. **Removed planning block extraction from narrative** (per user directive):
   - Deleted `planning_block_extractor.py` module
   - Removed extraction logic from `narrative_response_schema.py`
   - Planning blocks now only come from JSON field
   - Updated tests to match new architecture
