## Raw Body
### Motivation

- Reduce duplication across the API end-to-end test suite by centralizing Flask app, auth patching, and test-client setup. 
- Make the end2end test patterns and the `.claude` helper commands/skills consistent with the repository's `TESTING_AUTH_BYPASS` pattern.
- Preserve legacy unittest-style assertions while satisfying linters by documenting and suppressing specific ruff warnings where appropriate.

### Description

- Add a shared base test case `End2EndBaseTestCase` in `mvp_site/tests/test_end2end/__init__.py` that centralizes `create_app()` invocation, `TESTING_AUTH_BYPASS` env setup, auth patcher, and standard request headers.
- Refactor many test modules under `mvp_site/tests/test_end2end/` to inherit from `End2EndBaseTestCase`, set `CREATE_APP`, `AUTH_PATCH_TARGET`, and optionally `TEST_USER_ID`, and call `super().setUp()`; update their Firestore/LLM patching and header usage to use the base class fields (examples: `test_create_campaign_end2end.py`, `test_continue_story_end2end.py`, `test_god_mode_end2end.py`, and many others).
- Update `.claude` assets to reference the new base class and the stronger test env var: modify `.claude/skills/end2end-testing.md` to document using `End2EndBaseTestCase` and `TESTING_AUTH_BYPASS`, and update `.claude/commands/adde2e.md` example to create tests that inherit the base.
- Add targeted `# ruff: noqa: ...` pragmas in several legacy end2end tests to suppress `PT009` (unittest-style asserts) and a few import/style warnings so the repository's pre-commit checks remain green while keeping existing assertions intact.
