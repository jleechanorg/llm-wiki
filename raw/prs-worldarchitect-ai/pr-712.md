## Key Features

### `/headless` Command
- Creates isolated git worktrees for conflict-free development
- Runs Claude Code in headless mode with real-time monitoring  
- Automatically creates PRs or cleans up if no changes
- Follows orchestration system patterns from CLAUDE.md

### Web Search MCP Servers
- **DuckDuckGo Search**: Free, privacy-focused web search (no API key required)
  - Package: `@oevortex/ddg_search`
  - Features: Web search, content extraction, privacy protection
- **Perplexity Search**: Premium AI-powered search with real-time research
  - Package: `server-perplexity-ask` 
  - Features: Advanced queries, AI-powered results, real-time web data
- **Dual Options**: Both free and premium search capabilities available

### Secure Token Management
- **Centralized Storage**: ~/.token file for all API keys (GitHub + Perplexity)
- **Environment Variables**: Secure sourcing instead of hardcoded values
- **Fallback Support**: Legacy token file compatibility maintained
- **Enhanced Security**: Proper permissions and validation checks

### `/push` Fixes
- HTTP test failures from port conflicts no longer block push
- CI replica test failures are warnings instead of errors
- Preserves test validation while avoiding false failures

## Implementation Details

- Uses proven agent architecture (worktree isolation, headless flags)
- Includes comprehensive error handling and auto-cleanup
- Leverages existing `/pushlite` instead of creating duplicate functionality
- Stream-json monitoring for cost tracking and progress updates
