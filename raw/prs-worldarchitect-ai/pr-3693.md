## Summary
- Fixed statusLine command buffering issue by adding stdin consumption
- Updated both `.claude/settings.json` and `.claude/settings.json.backup`
- Ensures statusLine updates correctly without buffering problems

## Problem
The statusLine command in Claude Code settings was not consuming the JSON stdin that Claude Code provides, causing buffering issues and preventing the statusline from updating properly.

## Solution
Added `cat > /dev/null;` at the start of the statusLine command to consume the stdin before executing the git-header.sh script or fallback status display.

## Changes
- `.claude/settings.json:300` - Added `cat > /dev/null;` to statusLine command
- `.claude/settings.json.backup:283` - Added `cat > /dev/null;` to statusLine command
