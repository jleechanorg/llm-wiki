## Raw Body
This commit implements flexible service account credential loading that supports both file-based and environment variable approaches, enabling secure deployment in sandbox environments (Claude.ai, Replit, etc.) without requiring file access.

Changes:
- Created mvp_site/service_account_loader.py module with dual loading strategy:
  1. File-based loading (existing pattern, tries first)
  2. Environment variable loading (new pattern, fallback)
- Updated firestore_service.py to use new credential loader
- Updated main.py to use new credential loader
- Updated world_logic.py to use new credential loader
- Enhanced .env.example with Google Cloud service account env var documentation

Environment Variables:
- GOOGLE_PROJECT_ID (required)
- GOOGLE_CLIENT_EMAIL (required)
- GOOGLE_PRIVATE_KEY (required, full PEM string)
- GOOGLE_PRIVATE_KEY_ID (optional)
- GOOGLE_CLIENT_ID (optional)

Loading Priority:
1. File at WORLDAI_GOOGLE_APPLICATION_CREDENTIALS (if exists)
2. Environment variables (if file loading fails)
3. Default application credentials (GCP environments)

Benefits:
- Zero hardcoded credentials (git-safe)
- Works in serverless/sandbox environments
- Maintains backward compatibility with file-based approach
- Comprehensive error handling and validation
- Detailed documentation and examples

Security:
- All sensitive values loaded from environment at runtime
- No credentials committed to source code
- Clear warnings about .env file security
- Service account key validation
