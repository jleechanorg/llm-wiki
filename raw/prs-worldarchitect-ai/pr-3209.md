## Raw Body
The /sync command was incorrectly staying on the worktree's dedicated branch when the target branch existed elsewhere. Now it:

1. Creates local branch from remote content when none exists locally
2. Tries to delete stale local branch refs and recreate when blocked
3. Only falls back to current branch as last resort with clear warnings
4. Uses git show-ref for explicit local branch detection

This ensures worktrees properly track the PR branch name, not just content.
