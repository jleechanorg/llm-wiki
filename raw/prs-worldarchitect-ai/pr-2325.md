## Raw Body
…, strip CJK chars

Root cause analysis found that LLM was returning empty narrative during Deep Think mode because system prompts said "don't take narrative actions" which was interpreted as "leave narrative empty". This caused placeholder text to be inserted.

Changes:
- Update Deep Think safety rule in game_state_instruction.md to require brief narrative (character pausing to think) while prohibiting story-advancing actions
- Update continuation reminder in llm_service.py to reinforce narrative requirements
- Remove placeholder code in firestore_service.py - now raises FirestoreWriteError when AI returns empty narrative (surfaces LLM compliance issues instead of masking)
- Add CJK character stripping in narrative_response_schema.py to handle training data leakage (e.g., Chinese "夜晚" appearing mid-English sentence)
- Update tests to reflect new error-raising behavior and add mixed language tests
