## Summary
- Centralize Cloud Run resource allocation (memory, CPU) in `scripts/shared_config.sh`
- Update `deploy.sh` and `pr-preview.yml` to use shared config
- Fixes PR preview OOM errors caused by mismatched memory limits

## Root Cause Analysis

**The 503 errors in smoke tests were caused by memory exhaustion, not timeouts:**

```
ERROR  Memory limit of 512 MiB exceeded with 601 MiB used
[INFO] Handling signal: term (Cloud Run killing the container)
```

**Timeline of events:**
| Date | Event |
|------|-------|
| Jan 14 | PR #3491 merged - Added 133MB embedding model for intent classifier |
| Jan 26 | PR #4078 merged - Increased dev/stable to 4Gi (but missed pr-preview.yml) |

The intent classifier (PR #3491) added ~133MB embedding model that pushed preview over 512Mi limit.

## Changes

### scripts/shared_config.sh
- Add `WORLDARCH_MEMORY` (default: 4Gi)
- Add `WORLDARCH_CPU` (default: 1)

### deploy.sh
- Use `$WORLDARCH_MEMORY` instead of hardcoded "4Gi"

### .github/workflows/pr-preview.yml
- Source `scripts/shared_config.sh` before deploy
- Use `$WORLDARCH_MEMORY`, `$WORLDARCH_CPU`, `$WORLDARCH_TIMEOUT_SECONDS`
- Prevents drift between dev/stable and preview environments
