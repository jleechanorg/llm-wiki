## Raw Body
### Motivation

- Improve accuracy of generated Markdown docs for JSON Schema types when properties omit an explicit `type` and rely on `$ref`/`items`/`allOf` composition cues.

### Description

- Add `_infer_property_type()` to `mvp_site/schemas/prompt_generator.py` to infer `array`, `object`, `ref`, or `any` from structural schema hints when `type` is not present.  
- Use resolved schema fragments when rendering properties and wire the inference helper into `generate_type_markdown()`, avoiding misleading `object` labels for arrays and composed types.  
- Refactor local loop variables (`resolved_sub_schema`, `resolved_prop_def`) to preserve original values and satisfy lint rules; no change to runtime validation or persisted schemas.  

File: `mvp_site/schemas/prompt_generator.py` — GOAL: make generated schema docs reflect implicit/derived types; MODIFICATION: add `_infer_property_type()` and update `generate_type_markdown()` to resolve refs and infer types; NECESSITY: previously defaulting to `object` produced incorrect docs for arrays/composed/$ref fragments; INTEGRATION PROOF: change is limited to doc-generation path and uses existing `load_schema()`/`resolve_refs()` flow so no new files or runtime surface changes were introduced.
