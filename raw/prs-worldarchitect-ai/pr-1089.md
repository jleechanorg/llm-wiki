## Raw Body
## Summary
Fixes integrate.sh to properly check commits against `origin/main` instead of the feature branch's upstream.

## Problem
The script was checking unpushed commits against the feature branch's upstream (e.g., `origin/format-and-frontend-migration`) instead of `origin/main`. This caused confusing errors when running integrate.sh from feature branches, even after PRs were merged (squash-merged commits don't exist in the branch anymore).

## Solution
- **Changed comparison**: Now uses `origin/main..HEAD` instead of `${ahead_ref}..HEAD`
- **Better messaging**: Shows "commits not in origin/main" instead of confusing "unpushed commits"
- **Clearer guidance**: Explains when `--force` is safe (after squash-merged PRs)
- **Improved UX**: Users understand when their changes are actually missing vs already merged

## Technical Details
**Before:**
```bash
ahead_ref="$(git rev-parse --abbrev-ref --symbolic-full-name @{u})" # origin/feature-branch
commit_count=$(git rev-list --count "${ahead_ref}..HEAD")
```

**After:**
```bash
commit_count=$(git rev-list --count origin/main..HEAD)
```
