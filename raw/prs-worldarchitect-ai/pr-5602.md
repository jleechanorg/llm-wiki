## Raw Body
## Problem
The coverage+testmon pre-flight in `run_tests.sh` used `|| true` which swallowed the exit code. If pytest errored but output contained 'deselected', the grep would match and emit SKIP_TESTMON, masking the real failure.

## Fix
- Add shared `classify_testmon_result()` helper that requires `exit_code == 0` before allowing SKIP_TESTMON
- Expand negative pattern to include `error` and `errors`
- Both code paths (coverage pre-flight + testmon-only) now use the same classifier

Closes REV-ed58z
