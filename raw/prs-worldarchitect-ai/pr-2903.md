## Problem
When users issue god mode commands like "stop saying 'mere level 9'", the LLM sometimes:
- Acknowledges the request in `god_mode_response`
- Writes notes to `state_updates.debug_info.dm_notes`
- But FAILS to include the `directives.add` field

This means the rule isn't persisted and won't appear in future prompts.

## Solution
1. **Include dm_notes in system prompts** - now both `god_mode_directives` AND `dm_notes` are injected
2. **Newest-first ordering** - both are sorted by timestamp so recent rules take precedence
3. **Debugging tools** - new script and skill docs for investigating missing directives
