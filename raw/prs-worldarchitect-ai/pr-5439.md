## Problem
The /pair orchestration was creating separate git worktrees for each agent:
- Coder: `/Users/jleechan/projects/orch_worldarchitect.ai/codex-coder-pair-*`
- Verifier: `/Users/jleechan/projects/orch_worldarchitect.ai/codex-verifier-pair-*`

This caused:
- Agents couldn't access shared `.beads/` directory  
- File changes weren't immediately visible to partner agent
- MCP coordination broke due to isolation

## Solution
Both agents now work in the **same directory** by default, enabling:
- ✅ Shared .beads/ access for state tracking
- ✅ Immediate file visibility for coordination
- ✅ Proper MCP Mail and Beads integration
