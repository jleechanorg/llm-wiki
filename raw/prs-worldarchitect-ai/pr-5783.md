## Summary

- Fixes MCP streaming contract failures on Preview E2E (character_streaming_contract, character_streaming_dice_contract, think_streaming_contract, god_streaming_contract)
- Root cause: SMOKE_TOKEN bypass was restricted to `/mcp` path only; streaming requests hit `/api/campaigns/<id>/interaction/stream` and received 401 Unauthorized
- Extends `check_token` SMOKE_TOKEN block to allow stream route when on preview (PRODUCTION_MODE + ENVIRONMENT=preview)
- Preserves restriction: `/api/settings` still returns 401 with smoke token (unchanged)

## Root Cause

CI run: https://github.com/jleechanorg/worldarchitect.ai/actions/runs/22445206678 (job 64997688870)

All four streaming contract scenarios failed with:
\`\`\`
Timed chunk validation failed: expected at least 2 chunk events, saw 0
character stream returned missing/malformed done payload
\`\`\`

The stream route returns 401 when SMOKE_TOKEN is used on preview because the auth block only matched `path == "/mcp"`. Campaign creation uses `/mcp` (works); streaming uses `/api/...` (401 → 0 chunks).

## Production Code Changes

**Before:** SMOKE_TOKEN block condition: `request.path == "/mcp"`
**After:** `is_smoke_allowed_path = (request.path == "/mcp" or ("/api/campaigns/" in request.path and request.path.endswith("/interaction/stream")))`
**Why:** MCP smoke tests must authenticate both MCP tool calls and stream route with the same user_id.

## Test Changes

- `test_smoke_token_authenticates_stream_route_on_preview` — asserts stream route accepts SMOKE_TOKEN on preview (not 401)
- `test_smoke_token_bypass_is_restricted_to_mcp_path` — unchanged (still asserts /api/settings → 401)
