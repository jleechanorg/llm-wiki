## Raw Body
## Summary
- Fixed Pydantic ValidationError when NPC has invalid `string_id` prefix (e.g., `faction_` instead of `npc_`)
- Invalid `string_id` values are now regenerated with proper `npc_` prefix instead of crashing
- Added regression test `test_invalid_string_ids_regenerated`

## Root Cause
Game state contained NPC with `string_id: faction_silent_guard_001` which doesn't match the required pattern `^(pc|npc)_[\w]+_\d{3}$`

## Fix
Validate the stored `string_id` against the NPC pattern before using it. If invalid, regenerate with proper prefix.
