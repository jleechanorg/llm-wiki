## Problem

Investigation of campaign `Wp5aDEpITat6hlJRNx3H` revealed critical God Mode directive bloat:

- **Root Cause:** "Instruction Fatigue" due to 210+ bloated directives (e.g., hundreds of `Rule: Level is X` entries)
- **Impact:** Context overflow and decreased adherence to new instructions
- **Solution:** Three-layer defense - cleanup script + prompt updates + server-side validation

## Solution

### Backend/Prompt Fixes (Root Cause Prevention)

**1. Prompt Updates (`god_mode_instruction.md`):**
- Explicitly ban using `directives` for dynamic numbers (Level, HP, Gold, etc.)
- Clarify that directives are for behavioral rules only, not state tracking
- Provide clear examples of what should/shouldn't be directives

**2. Server-Side Validation (`world_logic.py`):**
- **NEW:** Added `_should_reject_directive()` function to filter problematic directives
- Filters out directives matching patterns like:
  - Dynamic numbers: "Rule: Level is", "Rule: HP is", "Rule: Gold is", "Rule: XP is"
  - One-time events: "Rule: You just killed..." (with context checks)
  - Formatting: "Rule: Always include..."
- **Defense-in-depth:** Even if LLM ignores prompt instructions, server rejects bad directives
- Logs warnings when directives are rejected for debugging

**3. Cleanup Script (`scripts/fix_god_mode_directives.py`):**
- One-time utility to clean existing bloated directives
- Removes state-tracking rules via keyword matching
- Handles edge cases (empty directives, null values, etc.)

**4. Prompt Updates (`narrative_system_instruction.md`):**
- Add mandatory "Visual Verification" - Python dice rolls MUST be mirrored to `dice_rolls` JSON
- Ensures UI visibility of all dice rolls
