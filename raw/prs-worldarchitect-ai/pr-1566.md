## Summary

This PR fixes the `claude_backup.sh` script to work properly on Linux/Ubuntu systems.

## Changes Made

- ✅ **Platform-specific default backup directories**: Auto-detects Linux vs macOS and uses appropriate paths
- ✅ **Linux Dropbox support**: Tries `~/Dropbox`, `~/Documents`, then `~/` as fallback 
- ✅ **Fixed path validation**: Improved null byte detection to avoid false positives on valid Linux paths
- ✅ **Updated help text**: Shows platform-specific default paths in documentation
- ✅ **Maintains backward compatibility**: All existing macOS functionality preserved

## Problem Fixed

The script was hard-coded to use macOS-specific paths like `~/Library/CloudStorage/Dropbox/` which don't exist on Linux systems, causing backup failures.

## Platform Support

- ✅ **macOS**: Uses `~/Library/CloudStorage/Dropbox/claude_backup_HOSTNAME`
- ✅ **Linux/Ubuntu**: Uses `~/Dropbox/claude_backup_HOSTNAME` or `~/Documents/claude_backup_HOSTNAME`
- ✅ **Other systems**: Falls back to `~/claude_backup_HOSTNAME`
