## Raw Body
## Summary
This PR speeds up directory-based CI by enabling true per-file worker parallelism, splitting the core test matrix, and preserving coverage reporting across parallel shards.

## Changes
- `run_tests.sh`
  - Implement real worker-pool parallel execution (bounded by `max_workers`) instead of serial execution.
  - Add `TEST_MAX_WORKERS` override for explicit CI tuning.
  - Switch coverage runs to `coverage run --parallel-mode` so coverage can run with multiple workers safely.
  - Combine parallel coverage files from `/tmp/worldarchitectai/coverage`.
- `scripts/ci-detect-changes.sh`
  - Split core into two shards: `core-mvp` (`mvp_site`) and `core-tests` (`tests`).
  - Ensure both core shards run together when core is selected/fallback.
- `.github/workflows/test.yml`
  - Update core references to `core-mvp` / `core-tests`.
  - Set `TEST_MAX_WORKERS=2` in CI test execution paths.
  - Collect per-shard coverage artifacts and merge them in `coverage-report` via artifact pattern download + `coverage combine`.

## Why
- Previous CI was effectively single-worker for core PR runs due to `--coverage` and serial loop behavior.
- This maximizes safe parallelism on standard private `ubuntu-latest` runners while keeping coverage and reporting intact.
