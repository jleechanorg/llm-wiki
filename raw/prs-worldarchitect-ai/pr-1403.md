## Security Fix: Shell Injection Vulnerability

**Critical Issue**: Fixed shell injection vulnerability in `run_local_server.sh` where user-controlled environment variables could be executed as shell commands.

### Vulnerability Details
- **Location**: `run_local_server.sh` lines 195-197 (terminal launching commands)
- **Risk**: Environment variables containing shell metacharacters could execute arbitrary commands
- **Attack Vector**: Malicious `FIREBASE_API_KEY` or project paths with shell injection payloads

### Security Improvements
1. **Safe Command Construction**: Added `build_env_command()` function using `printf %q` for proper shell escaping
2. **Environment Isolation**: Added `setup_firebase_env()` with safe variable handling
3. **Input Sanitization**: All environment variables are properly quoted and escaped before terminal execution

### Code Changes
- **Before**: Direct string interpolation in terminal commands (vulnerable)
- **After**: `printf %q` escaping and proper quoting (secure)

### Testing
- ✅ Verified safe handling of environment variables with special characters
- ✅ Confirmed terminal launching works with proper escaping
- ✅ No functional regressions in local server startup

**Impact**: Prevents arbitrary code execution through environment variable injection while maintaining all existing functionality.
