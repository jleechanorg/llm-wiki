## Production Code Changes

**Before:** `ai_orch run "task"` → spun up tmux agent → polled for PR creation for 120s → returned

**Now:**
- `ai_orch "task"` → passthrough: invokes CLI directly, streams output to stdout
- `ai_orch --async "task"` → spawns detached tmux session, returns immediately
- `ai_orch --async --resume "task"` → reuses existing session for same CWD (claude gets `--continue` natively)
- `ai_orch --async --worktree "task"` → creates git worktree first

**Why:** The orchestration system was doing tmux + PR monitoring + multi-agent coordination for simple one-liner tasks. New design: passthrough by default, opt-in async.
