# Outcome Resolution Protocol: From Rejection to Audit Trail

## Philosophy Shift

**Before:** Rejection-based guardrails (block player input, interrupt gameplay)  
**After:** Outcome Resolution Protocol (process all input, resolve via mechanics, document in audit trail)

### Core Principle: Interpret → Resolve → Audit → Narrate

When players declare outcomes (e.g., "The king agrees", "It kills the guard"), the system:
1. **Interprets** the underlying attempt
2. **Resolves** via game mechanics (dice rolls, skill checks)
3. **Audits** in `outcome_resolution` JSON field
4. **Narrates** the actual outcome based on mechanics

**Result:** Zero rejections, smooth gameplay, full accountability.
