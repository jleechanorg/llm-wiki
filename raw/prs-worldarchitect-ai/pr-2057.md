## Raw Body
## Problem Addressed (PR #113)
- Unit tests passed but integration tests hung/skipped
- Declared success anyway, leading to undetected bugs
- False confidence in test coverage

## Enhancements Implemented

### 1. Mandatory Integration Test Check
- Discover integration tests in mvp_site/tests/integration/, **/test_*integration*.py
- Track skip reasons (e.g., "requires Firestore emulator")
- REQUIRE deployed environment validation if skipped locally
- FAIL if integration tests neither pass nor have deployed validation

### 2. Enhanced Exit Code Alignment
- Track exit codes for EACH test file (0=pass, 1=fail, 124=timeout)
- 5-minute timeout per test with kill on exceed
- Aggregate exit codes (ANY failure = overall FAILURE)
- Success requires exit code 0 for ALL tests

### 3. Evidence Verification Protocol
- Run ls -laR /tmp/<repo>/<branch>/ before declaring success
- Compare claimed files vs actual directory listing
- Remove phantom file references
- Include ls -la output in final report

### 4. Zero Partial Success Policy
- Count ALL test files (unit + integration)
- Track executed vs total (100% required for SUCCESS)
- If executed < total, report PARTIAL EXECUTION
- Skipped tests MUST have documented reason

### 5. Anti-False-Positive Checklist
- Added Phase 13A with mandatory checklist in all reports
- Sections: Test Discovery, Execution, Evidence, Integration, Quality
- Final Verdict: SUCCESS / FAILURE / PARTIAL EXECUTION
