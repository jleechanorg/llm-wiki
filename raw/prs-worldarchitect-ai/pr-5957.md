## Summary
- `.claude/scheduled_tasks.lock` is already listed in `.gitignore` (line 99, added in commit `884a4c692`)
- However it was previously committed, so git still tracks it despite the gitignore entry
- `git rm --cached` removes it from tracking without deleting the local file

## Why
Gitignore only prevents **untracked** files from being added. Once a file is committed, it remains tracked regardless of `.gitignore`. The fix is `git rm --cached <file>` to stop tracking it, after which gitignore takes effect for future changes.

## Result
After merge, `.claude/scheduled_tasks.lock` will no longer appear as a modified file in `git status` on any branch that has this commit.
