## Summary

Improves pre-flight validation to actually test CLI execution and verify file writing capabilities, preventing silent failures.

## Changes

- **Real CLI testing**: All CLIs now run actual commands (2+2 test) and write output to /tmp subdirectory
- **Validate all CLIs**: Tests ALL CLIs in chain, not just first success
- **Better visibility**: Tracks all validated CLIs and logs summary
- **Improved Cursor/Claude validation**: Uses prompt files instead of just binary existence check
- **Better error handling**: Catches PATH issues, quota exhaustion, and runtime failures

## Problem Fixed

Previously, agents were dispatched but failed silently because:
- Cursor validation only checked file existence, not PATH availability
- Gemini validation could pass but quota exhausted during execution
- No verification that CLIs could actually execute and write output

## Testing

After merge, will test pre-flight validation for all CLIs (gemini, cursor, codex, claude).

## Related

Fixes issue where fix-comment agent was dispatched but didn't push fixes due to CLI failures.
