## Problem

🚨 **Multiple Critical Issues**: The copilot system had several branch isolation violations causing file conflicts when multiple branches run commands simultaneously.

### **Issues Found:**

1. **Regex Bug**: Branch name sanitization used `r'[^\\w\\-_.]'` (double-escaped backslashes) causing ALL characters to become underscores, breaking directory isolation
2. **Base Class Files**: Auto-save used generic filenames like `commentfetch_result.json` without branch names  
3. **`/pushl` Script**: Created shared `/tmp/copilot/push.json` causing overwrites between branches
4. **`/testi.sh` Logs**: Used shared `/tmp/integration_output.log` and `/tmp/test_output.log` files
5. **Documentation**: Inconsistent path references mixing `/tmp/copilot/` and `/tmp/copilot_{branch}/`

## Root Causes

```python
# BROKEN: Double-escaped backslashes interpreted literally
sanitized = re.sub(r'[^\\w\\-_.]', '_', sanitized)  # All chars → underscores

# BROKEN: Generic filenames without branch isolation  
output_file = f"{self.__class__.__name__.lower()}_result.json"

# BROKEN: Shared files across branches
cat > /tmp/copilot/push.json << EOF
```

## Complete Solution

### **1. Regex Fix**
```python
# FIXED: Proper regex character class
sanitized = re.sub(r'[^\w\-_.]', '_', sanitized)  # Preserves word chars
```

### **2. Base Class Branch Isolation**
```python  
# FIXED: Branch-specific auto-save files
output_file = f"{self.__class__.__name__.lower()}_result_{self.current_branch}.json"
```

### **3. Script Isolation**
```bash
# FIXED: Branch-specific push results
SANITIZED_BRANCH=$(echo "$BRANCH"  < /dev/null |  sed 's/[^a-zA-Z0-9._-]/_/g')
cat > "/tmp/copilot_${SANITIZED_BRANCH}/push.json" << EOF
```

### **4. Test Log Isolation**
```bash
# FIXED: Branch-specific test logs
if $cmd > /tmp/integration_output_${SANITIZED_BRANCH}.log 2>&1; then
```
