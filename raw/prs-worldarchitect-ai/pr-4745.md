## Raw Body
Restores freeze_time choice handling from original PR #3087 without unrelated llm_service refactors (supersedes #4552).

**Scope update: centralized time-freeze behavior**
- Adds `freeze_time` to planning-block choices with schema validation/sanitization.
- Server-enforces `freeze_time=true` on level-up choices (LLM-provided or injected).
- Detects when a user selects a `freeze_time` choice and freezes time advancement to a **single microsecond tick** while still allowing non-time state changes.
- Centralizes time-freeze gating so that Think Mode (PlanningAgent), God Mode, Character Creation, and freeze_time choices all:
  - filter out normal time advancement to a single microsecond tick,
  - **run combat cleanup (intentional, even in freeze/Think/creation paths),** and
  - skip player_turn increments for time-frozen choices.
- Selecting `level_up_now` forces CharacterCreationAgent for that request.
- Updates planning protocol docs.
- Extends CI test suite timeout in GitHub Actions to avoid core directory suite timeouts.
