## Summary

- Rewrote `/sim` to predict inline using existing conversation context — no JSONL parsing, no subprocess, no `claude -p` calls
- Added `/sim_async` preserving the old subprocess-based approach for offline comparison
- Added `genesis/scripts/sim_ab_benchmark.py` for offline A/B benchmarking (baseline vs v2.2 prompt with separate Sonnet instances, full 40-turn conversation history)

## Production Code Changes

- **`.claude/commands/simulate.md`** — Rewrote from 6-phase subprocess workflow (parse JSONL, gather state, build prompt, pipe to claude -p) to 3-step inline flow (gather git/PR state, predict using existing context, display). Prediction guidelines distilled from the v2.2 Stage 2 prompt into 8 inline rules.
- **`.claude/commands/sim_async.md`** — New file preserving the old subprocess A/B approach for cases where a separate Sonnet instance is needed
- **`genesis/scripts/sim_ab_benchmark.py`** — New offline benchmark script using cchat for JSONL parsing, message_classifier for filtering, state_enrichment for context, and full 40-turn conversation history (not just 8 messages)

## Key Design Decision

The inline `/sim` dropped the A/B baseline comparison because the same model (Claude) doing both predictions cannot produce meaningfully different naive vs guided results — the A/B comparison only has value when baseline is a genuinely separate Sonnet instance with a minimal prompt, which the Python script provides.
