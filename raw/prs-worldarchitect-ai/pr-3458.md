# Remove regex-based JSON parsing from mvp_site/

## Summary
This PR removes all regex-based JSON extraction and fixing logic from the codebase while **preserving all standard `json.loads()` parsing**. The goal is to eliminate custom JSON parsing complexity and rely solely on Python's built-in JSON parser.

## What Was REMOVED ❌

### Deleted Files (5 total):
1. **mvp_site/json_utils.py** (725 lines) - Entire regex-based JSON parsing module containing:
   - `RobustJSONParser` class - Multi-strategy fallback parser with 5 different parsing strategies
   - `extract_json_boundaries()` - Regex-based JSON boundary detection and extraction
   - `complete_truncated_json()` - Logic to auto-complete truncated/malformed JSON
   - `unescape_json_string()` - Manual JSON escape sequence handling
   - `parse_llm_json_response()` - Wrapper function for robust parsing
   - `extract_field_value()` - Regex-based field extraction from malformed JSON
   - `extract_nested_object()` - Bracket-aware nested object extraction
   - Multiple precompiled regex patterns for JSON field extraction

2. **mvp_site/debug_hybrid_system.py** - Debug script that imported json_utils

3. **mvp_site/tests/test_json_utils.py** - Unit tests for json_utils module

4. **mvp_site/tests/test_robust_json_parser.py** - Tests for RobustJSONParser

5. **mvp_site/tests/test_narrative_cutoff_bug.py** - Tests for JSON truncation handling

### Removed Functionality:
- ❌ **Regex-based JSON extraction** - No more extracting JSON from arbitrary text using regex
- ❌ **Malformed JSON fixing** - No more auto-completing truncated JSON with missing braces/quotes
- ❌ **Multi-strategy fallback parsing** - No more trying 5 different ways to parse invalid JSON
- ❌ **Field-level extraction** - No more extracting individual fields when JSON parsing fails
- ❌ **Manual escape handling** - No more custom unescape logic for JSON strings
- ❌ **Narrative extraction from JSON-like text** - No more regex patterns to find narrative fields
