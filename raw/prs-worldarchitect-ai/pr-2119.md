> [!NOTE]
> Centralizes shared async event loop and thread pool, switches auth wrappers to environment-based Firebase config, adds env-configurable rate limits, and hardens concurrency tests and tooling.
> 
> - **Server (Flask `mvp_site/main.py`)**:
>   - **Shared async infra**: Add process-wide background event loop and `ThreadPoolExecutor` with graceful shutdown (`_ensure_async_infrastructure`, `_shutdown_async_resources`).
>   - **Parallel I/O**: New `run_blocking_io()` executes blocking Firestore calls via `loop.run_in_executor`; routes updated to use it (e.g., `get_campaign`).
>   - **Rate limits**: Campaign/settings limits now read from `CAMPAIGN_RATE_LIMIT`/`SETTINGS_RATE_LIMIT` env vars.
>   - **Testing support**: Honor test bypass headers only when `TESTING=true` and `ALLOW_TEST_AUTH_BYPASS=true`.
> - **Auth tooling (`.claude/scripts`)**:
>   - **Env-based config**: Wrappers (`auth-worldai.mjs`, `auth-aiuniverse.mjs`) require API key from env and pass `FIREBASE_*` via `env` instead of CLI args.
>   - **CLI config**: `auth-cli.mjs` now requires `FIREBASE_API_KEY`; `MCP_URL` configurable via env.
> - **Tests (`mvp_site/tests/test_concurrency_integration.py`)**:
>   - Strengthen RED tests to prove/measure serialization vs parallelism; add safer auth bypass headers and robust assertions/cleanup.
> - **Scripts**:
>   - `scripts/test_parallel_concurrency.py`: require API key; safer token-file parsing.
