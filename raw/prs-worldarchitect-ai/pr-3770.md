# Fix: JSON Parsing for Gemini Code Execution Responses

Fixes a response parsing bug that could truncate Gemini responses when code execution returns top-level arrays or nested structures.

## What This PR Does

**Enhanced JSON Parsing** in `narrative_response_schema.py`:
1. Direct `json.loads` attempt first (handles top-level arrays from code execution)
2. Enhanced `_find_matching_brace` to support custom open/close characters
3. Fixed bracket matching to find the MATCHING closing bracket (not just first `]`)

**Added Test Coverage:**
- Unit tests for `_find_matching_brace` with various scenarios
- Reproduction script for validating dice roll extraction

## What This PR Does NOT Do

**IMPORTANT:** This PR intentionally **keeps Gemini 3 Code Execution enabled** for dice rolls. The flow where Gemini uses internal code execution to generate dice results is the desired behavior.

This PR does NOT:
- Disable code execution
- Add two-phase logic enforcement
- Require tool_requests for dice rolls

## Why This Matters

When Gemini uses code execution, it may return responses like:
```json
[
  {"some": "data"}
]
{
  "narrative": "...",
  "action_resolution": {...}
}
```

The old parsing logic would incorrectly find the first `]` (which might be inside nested structures) instead of the matching closing bracket, causing response truncation.
