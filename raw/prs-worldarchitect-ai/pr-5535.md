## Test PR - workflow_run Comment Demo + Duplicate Fix

This PR demonstrates the automated workflow_run comment posting feature implemented in PR #5531, **and includes a critical fix for duplicate comments**.

### 🐛 Bug Fix: Duplicate Comments (NEW)

**Issue**: When `try-self-hosted` failed and fell back to `fallback-github-hosted`, BOTH jobs posted comments due to `always()` conditions.

**Fix**: Changed `try-self-hosted` to only post on `success()`, while `fallback-github-hosted` retains `always()` as the final execution path.

**Before**:
```
try-self-hosted fails → posts comment → fallback runs → posts another comment = 2 comments ❌
```

**After**:
```
try-self-hosted fails → no comment → fallback runs → posts comment = 1 comment ✅
try-self-hosted succeeds → posts comment = 1 comment ✅
```

**Changed**: `.github/workflows/mcp-smoke-tests.yml:196`
- Condition: `always()` → `success()`
- Resolves: cursor[bot] review comment #2808776975
