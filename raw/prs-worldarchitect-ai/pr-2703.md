## Summary

Living World system updates: UI debug panel, player_turn tracking optimization, annotation coverage fixes, and off-by-one turn number fix.

### Key Features

#### 1. Living World Debug UI Panel
- **Frontend display**: Renders `🌍 Living World Updates` section with background events, faction/time events, rumors, scene events, complications
- **Styled sections**: Color-coded backgrounds and emoji indicators for each living world field
- **Frontend tests**: Comprehensive JavaScript tests for structured field rendering

#### 2. Player Turn Tracking (Optimized)
- **Stored player_turn**: No longer recomputes from story context every request
- **GOD mode exclusion**: `player_turn` only increments on non-GOD mode actions (for living world cadence)
- **Backward compatibility**: Falls back to context computation for old campaigns

#### 3. World Events Extraction & Annotation
- **Turn/scene annotation**: All living world entries get `turn_generated` and `scene_generated` metadata
- **Filtered storage**: Only store `world_events` (not full `state_updates`) to reduce document size
- **Top-level access**: `world_events` exposed at structured_fields root for UI access
