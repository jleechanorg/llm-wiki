## Summary

- No 30s timeout — long coding tasks (20+ min) run until complete

## Problem
`openclaw-mcp`'s async bridge (`openclaw_chat_async`) hardcodes a 30s `AbortController` timeout. Any task longer than 30s is killed with `Request to OpenClaw timed out`. The gateway itself keeps running but the MCP tool reports failure.

## Solution
Bypass `openclaw-mcp` entirely. POST directly to the gateway's `/v1/chat/completions` endpoint via `nohup curl &` — returns PID immediately, curl runs in background until the response arrives.

## Testing
- Dispatched PR fix task to OpenClaw — confirmed agent running via gateway logs (`embedded run tool start tool=exec`)
- No timeout after 30s — task continues running
