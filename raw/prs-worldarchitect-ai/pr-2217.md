## Raw Body
Per user requirement: config/setup errors should crash immediately rather than silently falling back to defaults. This makes issues visible early.

Changes:
- gunicorn.conf.py: Remove ValueError catch for WORLDARCH_TIMEOUT_SECONDS Invalid env var now raises immediately instead of silently using 600

- firestore_service.py: Remove SERVER_TIMESTAMP try/except fallback SDK import is guaranteed at module level; also remove fallback doc ID generation - raise FirestoreWriteError if document ID not captured

- clock_skew_credentials.py: Remove ImportError catches throughout If google.auth._helpers can't be imported, Firebase auth is broken

- mcp_memory_real.py: Complete rewrite to fail-fast design Added MCPMemoryError exception class; all operations now propagate errors instead of returning empty lists/dicts that mask failures

- world_loader.py: Replace try/except with explicit os.path.exists() Banned names file is optional, but if it exists and can't be read, that's a real error that should propagate
