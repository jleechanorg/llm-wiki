### Root Cause Analysis
The FastEmbed-based intent classifier maintains internal state that gets "primed" by earlier test inputs. When `test_player_turn_increments_on_character_action` runs before `test_time_based_living_world_tracking_updates`, the classifier would misclassify `"advance time"` as think mode (since it sounds like a meta-command). In think mode, `player_turn` is not incremented, so the living world tracking update (which depends on turn advancement) was skipped.
