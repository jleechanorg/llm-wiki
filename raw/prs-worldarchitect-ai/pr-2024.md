Implements multi-layered commit SHA tracking following Google Cloud best practices:

1. Container Image Tagging (Primary):
   - Tags images with commit SHA: gcr.io/PROJECT/SERVICE:ENV-COMMIT_SHA
   - Maintains latest tag: gcr.io/PROJECT/SERVICE:ENV-latest
   - Enables precise version identification in Cloud Run console

2. Cloud Run Labels (Metadata):
   - Adds commit-sha label with 7-char short SHA
   - Adds commit-sha-full label with complete 40-char SHA
   - Labels visible in Cloud Run console and gcloud CLI
   - Preserves metadata across service updates

3. Cloud Build Integration (Automatic):
   - Cloud Build automatically captures source repository info
   - Build history contains commit SHA from all deployments
   - Accessible via gcloud builds list

Implementation details:
- Detects commit SHA from GITHUB_SHA (CI/CD) or git rev-parse HEAD (local)
- Fallback to timestamp-based identifier if git unavailable
- Both deploy.sh and deploy_common.sh updated to support tracking
- Compatible with all environments: dev, staging, production
- Works for both local and GitHub Actions deployments
- Automatically sanitizes commit SHA for Cloud Run label requirements

Files modified:
- deploy.sh: Capture commit SHA, use in image tags, pass to deploy function
- scripts/deploy_common.sh: Support multiple image tags, accept label arguments
- docs/cloud-run-commit-tracking.md: Comprehensive usage guide

Benefits:
- Complete deployment traceability
- Easy rollback to specific commits
- Full audit trail of deployments
- Match production issues to exact source code
- Compliance with deployment tracking requirements

Verification:
After deployment, view commit SHA via:
- Cloud Run Console → Service → Revisions → Labels
- gcloud run services describe SERVICE --format="yaml(metadata.labels)"
- Container Registry image tags
- Cloud Build history
