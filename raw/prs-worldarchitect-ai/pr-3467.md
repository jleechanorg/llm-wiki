## Raw Body
## Problem
In long campaigns, the Game Master (GM) would sometimes prioritize unresolved intents from the story history (e.g., an ongoing ritual) over the user's current action (e.g., 'Stop! Attack!'). This 'context weighting' issue caused the narrative to lag behind the user's intent.

## Solution
Implemented a **Structured Prompt** format for story continuation requests in `mvp_site/llm_service.py`.
- **Old:** Single JSON blob where `user_action` was just one field among many.
- **New:** Distinct blocks for `USER_ACTION`, `STORY_HISTORY`, and `GAME_STATE`.
- **Effect:** The User Action is now physically prioritized at the top of the prompt context, ensuring the LLM sees it as the primary directive for the new turn.
