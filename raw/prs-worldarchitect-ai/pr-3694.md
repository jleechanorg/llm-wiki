## Summary
- Fixed prompt instruction that caused LLM to use real-world year (2026) instead of in-game year (1492 DR)
- Added explicit cross-reference between `session_header` Timestamp and `world_data.world_time`

## Root Cause
`game_state_instruction.md` told LLM to use ISO-8601 `timestamp_iso` format (e.g., `2025-03-15T10:45:30Z`), but ISO-8601 uses Gregorian calendar years. The LLM interpreted this as "use real-world year" → year: 2026 stored in game state, corrupting Forgotten Realms 1492 DR timeline.

## Changes
1. **Replaced `timestamp_iso` with `world_time` object format** - now matches `god_mode_instruction.md` and `mechanics_system_instruction.md`
2. **Added CRITICAL note**: Use in-game year (1492 DR), NOT real-world year
3. **Added time_of_day/hour consistency guide**: Dawn (5-6), Morning (7-11), etc.
4. **Added cross-reference**: session_header Timestamp MUST match world_data.world_time exactly

## Test plan
- [ ] Play a Forgotten Realms campaign and verify world_time shows year 1492, not 2026
- [ ] Verify session_header Timestamp matches state_updates.world_data.world_time
- [ ] Verify no temporal anomaly warnings for consistent time values
