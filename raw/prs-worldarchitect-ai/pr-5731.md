## Summary
Refactors agent creation to make pre-flight CLI validation and prompt template injection opt-in features with sensible defaults. Changes the default behavior to skip pre-flight validation (avoiding side effects) and use raw prompts without template injection, while allowing callers to explicitly enable these features via agent_spec flags.

## Changes
- **task_dispatcher.py**:
  - Changed `no_worktree` default from `False` to `True` (skip worktree isolation by default)
  - Added `skip_preflight` flag (defaults to `True`) to skip CLI validation by default
  - Added `inject_prompt_template` flag (defaults to `False`) to skip prompt template injection by default
  - Wrapped pre-flight validation logic in conditional block that only runs when `skip_preflight=False`
  - Wrapped prompt template injection in conditional block that only runs when `inject_prompt_template=True`
  - Split attempt execution blocks to have simpler versions when `skip_preflight=True` (no runtime API validation)
  - Updated logging messages to reflect skipped vs. performed validation

- **orchestrate_unified.py**:
  - Updated all three agent creation paths to explicitly pass `no_worktree`, `skip_preflight`, and `inject_prompt_template` flags
  - Added `--worktree` flag (opt-in) to enable worktree isolation instead of `--no-worktree`
  - Added `--preflight` flag (opt-in) to enable pre-flight CLI validation
  - Added `--inject-prompt-template` flag (opt-in) to enable prompt template injection
  - Updated docstring to document new options and their defaults

- **test_task_dispatcher_fix.py**:
  - Updated test to explicitly set `no_worktree: False` to enable worktree isolation for that specific test

## Benefits
- **Reduced side effects**: Pre-flight validation is now opt-in, avoiding unnecessary API calls and cache file creation by default
- **Simpler execution**: Default behavior uses raw prompts without agent config or completion template injection
- **Explicit control**: Callers can opt-
