## Overview

This PR introduces a **semantic intent classifier** using FastEmbed (local ONNX model) to route user inputs to appropriate agents based on natural language understanding, replacing hardcoded pattern matching for most agent types.

## Key Changes

### 1. New Semantic Intent Classifier (`intent_classifier.py`)

- **Model**: `BAAI/bge-small-en-v1.5` (~133MB, local ONNX Runtime)
- **Process**: 
  - Pre-computes anchor embeddings for each agent mode (8-15 example phrases)
  - Embeds user input → 384-dim vector
  - Computes cosine similarity vs anchor embeddings
  - Returns mode with highest similarity if ≥ 0.65 threshold, else defaults to `MODE_CHARACTER`
- **Modes Supported**: `THINK`, `INFO`, `COMBAT`, `REWARDS`, `CHARACTER_CREATION`, `CHARACTER` (default)
- **Security**: Explicitly blocked from returning `MODE_GOD` (security safeguard)

### 2. Routing Architecture Overhaul (`agents.py`)

**Removed Hardcoded Routing:**
- ❌ Removed Priority 3 hardcoded state checks for `CombatAgent` and `RewardsAgent`
- ✅ These agents now route **entirely via semantic classification** with state validation

**New Routing Priority:**
1. **GodModeAgent** - Hardcoded (`"GOD MODE:"` prefix or `mode="god"`)
2. **Character Creation Completion** - Hardcoded (user says "done")
3. **CharacterCreationAgent** - Hardcoded state check (when char creation is active)
4. **PlanningAgent** - Hardcoded (`"THINK:"` prefix or `mode="think"`)
5. **Semantic Intent Classification** - Local LLM determines intent:
   - `MODE_THINK` → PlanningAgent (no state check)
   - `MODE_INFO` → InfoAgent (no state check)
   - `MODE_COMBAT` → CombatAgent (**with state validation**)
   - `MODE_REWARDS` → RewardsAgent (**with state validation**)
   - `MODE_CHARACTER_CREATION` → CharacterCreationAgent (**with state validation**)
   - `MODE_CHARACTER` → StoryModeAgent (default)
6. **API Explicit Mode** - Fallback for UI-forced modes (all modes handled with state validation)
7. **StoryModeAgent** - Default fallback
