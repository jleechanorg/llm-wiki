## Summary  
Resolves critical mktemp compatibility issue preventing Cerebras/Genesis integration on macOS.

## Problem Analysis
**Root Cause**: macOS mktemp behavioral difference with suffix patterns

- macOS: mktemp "/tmp/cerebras_ctx_XXXXXX.txt" creates literal "XXXXXX.txt" filename
- Linux: mktemp "/tmp/cerebras_ctx_XXXXXX.txt" creates "cerebras_ctx_r4nd0m.txt"

**Impact**: 
- Genesis sessions failed with "File exists" errors on subsequent runs
- Cerebras API integration broken on macOS development environments
- Required manual temp file cleanup between Genesis executions

## Technical Solution
Fixed all mktemp patterns to use trailing X's without suffixes, then add extensions manually.

## Cross-Platform Validation
- ✅ macOS: mktemp creates unique random filenames, extensions added manually
- ✅ Linux: mktemp works as before, extensions added consistently  
- ✅ Multiple Runs: No more file collision errors
- ✅ Parallel Sessions: Supports concurrent Genesis development

## Impact
- **Reliability**: Genesis /gene works consistently on macOS
- **Development**: No manual temp file cleanup required
- **Performance**: Eliminates session startup failures  
- **Compatibility**: Works across macOS, Linux development environments

🚀 This resolves the recurring "we just fixed it" Cerebras errors in Genesis workflows.
