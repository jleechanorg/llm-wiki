## Implementation Details
- Added runtime server: `mvp_site/worldai_tools_mcp_proxy.py`
  - JSON-RPC `/mcp` endpoint and `/health` endpoint.
  - Dynamic passthrough for non-local tools/resources to upstream MCP.
  - Local tool catalog:
    - `diag_evaluate_campaign_dice`
    - `admin_copy_campaign_user_to_user`
    - `admin_download_campaign`
    - `admin_download_campaign_entries`
    - `ops_gcloud_logs_read`
    - `ops_firestore_read_document`
    - `ops_firestore_query_collection_group`
    - `ops_run_mcp_local`
    - `ops_run_mcp_production`
    - `ops_deploy_mcp_service`
  - Enforced admin/ops `reason` + `ticket_id` requirements.
  - Enforced deploy confirm token validation with strict format:
    - `CONFIRMED`
    - `DEPLOY-{target}-{YYYYMMDD-HHMMSS}` (target-aware)
  - Enforced Downloads-root artifact boundary for artifact-producing tools.

- Added targeted unit tests: `mvp_site/tests/test_worldai_tools_mcp_proxy.py`
  - Local tool catalog coverage
  - Reason/ticket policy requirement
  - Deploy token validation success/failure cases

- Added real invocation harness: `testing_mcp/test_worldai_tools_mcp_proxy_real.py`
  - Starts local upstream + proxy
  - Executes passthrough sample + every local tool
  - Executes explicit restricted-policy failure cases
  - Writes one timestamped JSON log per call
  - Generates markdown matrix report
