## Summary

This PR adds separate safety limits for each of the 4 automation workflow types:
1. **PR automation** (monitoring) - posts codex comments
2. **fix-comment** - addresses review comments  
3. **codex-update** - browser automation (doesn't post PR comments)
4. **fixpr** - resolves conflicts/failing checks

## Problem

Previously, all workflows shared a single limit (10 automation comments per PR). This meant that if PR #2902 had 28 codex automation comments, the fix-comment workflow would be blocked even though it only had a few fix-comment specific comments.

## Solution

- Added workflow-specific comment counting functions
- Each workflow now has its own limit (defaults to 10, configurable via env vars)
- Safety checks use the appropriate limit based on workflow type
- Prevents one workflow from blocking others

## Changes

- `utils.py`: Added workflow-specific limit configuration
- `automation_safety_manager.py`: Store workflow-specific limits
- `jleechanorg_pr_monitor.py`: 
  - Added `_count_workflow_comments()` to count by workflow type
  - Updated safety checks in `run_monitoring_cycle()`, `_process_pr_fix_comment()`, and `process_single_pr_by_number()`
- Documentation: Updated README.md and AUTOMATION_SAFETY_LIMITS.md

## Environment Variables

```
export AUTOMATION_PR_AUTOMATION_LIMIT=10    # PR automation workflow
export AUTOMATION_FIX_COMMENT_LIMIT=10      # Fix-comment workflow  
export AUTOMATION_CODEX_UPDATE_LIMIT=10     # Codex update workflow
export AUTOMATION_FIXPR_LIMIT=10            # FixPR workflow
```

All default to `AUTOMATION_PR_LIMIT` (10) if not set.
