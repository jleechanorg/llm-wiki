## Summary

This update finalizes PR cleanup for #5654 by removing wrapper-based cron wiring and keeping only direct `jleechanorg-pr-monitor` execution.

### Production changes in scope

- `automation/install_cron_entries.sh`
  - Removed installer logic that copies `jleechanorg-pr-monitor-wrapper.sh` into `~/.local/bin`.
- `automation/jleechanorg-pr-monitor-wrapper.sh`
  - Deleted obsolete wrapper script (no longer used anywhere in automation flow).
- `orchestration/task_dispatcher.py`
  - `_build_worktree_add_command`: Added `_branch_exists(...)` guard before using `-b` when `create_new_branch=True`.
  - Prevents retry failures when a worktree branch already exists from a partial previous attempt.
- `automation/jleechanorg_pr_automation/orchestrated_pr_runner.py`
  - `run_fixpr_batch`: Treats `mergeable == False` and `str(mergeable).upper() == "CONFLICTING"` as non-mergeable.

### Test updates

- `orchestration/tests/test_task_dispatcher_fix.py`
  - Added/adjusted worktree add-command tests for the branch-exists behavior.
  - Updated retry test path for new `_branch_exists` dependency.

### Validation notes

- Wrapper script references are now fully removed from cron installer flow.
- Wrapper file deletion and installer change are isolated from other automation behaviors.
- No behavior changes to `jleechanorg_pr_monitor` command itself.
