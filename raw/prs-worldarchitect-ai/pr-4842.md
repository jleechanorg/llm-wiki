## Production Code Changes

### 1. `CLAUDE.md` (+30 lines)
**Purpose:** Primary rules file for Claude Code behavior

- **Added Config Debugging section** - No prior rule for config debugging → Added "Direct Read First" rule → Reduces Bash exploration (currently 57% of tool calls)
- **Added Environment Variable Prefixes table** - No reference for framework prefixes → Added VITE_, NEXT_PUBLIC_, REACT_APP_ table → Prevents prefix mismatch debugging loops (see Firebase VITE_ issue)
- **Added 3-command diagnostic limit** - No limit on diagnostic attempts → Max 3 before reassessing → Prevents long diagnostic loops
- **Added Grep tool preference** - Mixed usage of Grep tool vs Bash grep → Explicit rule to use Grep tool → Better formatting and context

## Evidence

Based on analysis in `docs/insights/real-insights-report-2026-02-04.md`:
- 57% Bash tool usage (some could be direct Read)
- Firebase VITE_ prefix issue required multiple diagnostic steps
- 3.3:1 Bash:Read ratio suggests room for more direct reads
