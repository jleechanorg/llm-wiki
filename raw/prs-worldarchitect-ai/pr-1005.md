## Problem Fixed
Debug mode setting was being ignored during campaign creation because user settings were only applied during interactions, not during initial campaign setup. This led to inconsistent debug mode behavior where users would set debug mode but it wouldn't work during character creation.

## Root Cause Analysis
Found three issues:
1. **Inconsistent defaults**: GameState defaulted debug_mode to `True`, but constants.py defined `DEFAULT_DEBUG_MODE = False`
2. **Missing user settings during campaign creation**: User's debug_mode setting was not applied during campaign creation in `create_campaign_route`
3. **Debug mode only applied during interactions**: The user's debug mode setting was only applied during the interaction route, not during campaign creation
