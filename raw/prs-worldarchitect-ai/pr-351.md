## Summary
**COMPLETED**: Complete entity schema consolidation and Pydantic migration with comprehensive bug fixes

### Fundamental Changes Made ✅
This PR represents a complete overhaul of the entity validation system, consolidating from multiple schema implementations to a single, robust Pydantic-based approach.

### Core System Changes ✅
1. **Complete Schema Consolidation**: Eliminated dual validation systems (Simple + Pydantic)
2. **Pydantic-Only Migration**: Removed entities_simple.py entirely, made Pydantic the default
3. **Critical Bug Fixes**: Fixed NPC display_name and HP field mapping issues causing test failures
4. **Enhanced Validation**: Added comprehensive narrative consistency fields with proper Pydantic V2 validation

### Major Architectural Changes ✅
- **🗑️ REMOVED**: `entities_simple.py` (422 lines) - eliminated manual validation approach
- **🗑️ REMOVED**: Root `schemas/` directory - consolidated under mvp_site/schemas
- **🗑️ REMOVED**: Environment variable switching - Pydantic is now the only validation system
- **✨ ENHANCED**: `entities_pydantic.py` - comprehensive schema with all required fields
- **🔧 FIXED**: `entity_tracking.py` - always uses Pydantic, removed complexity

### Critical Bug Fixes ✅
1. **NPC Display Name Bug**: Fixed create_from_game_state to use "name" field instead of dictionary key
2. **HP Field Mapping Bug**: Added support for both "hp_current" and "hp" keys for backward compatibility  
3. **Gender Validation Bug**: Added model validator to enforce gender requirement for NPCs
4. **Test Failure Cascade**: Systematically fixed all 8 failing tests instead of dismissing them

- `gender`: Required ["male", "female", "non-binary", "other"] with validation
- `age`: Optional numeric (0-50,000 years) for age-appropriate descriptions
