# **[FIX] Overhaul State Management and Context Handling**

This pull request addresses a cascade of critical bugs originating from a flawed state update mechanism and introduces a robust context truncation system to support long-running campaigns. The combined changes stabilize the entire application pipeline, making it resilient to data corruption, AI errors, inconsistent data schemas, and API context length limitations.

---

## **1. Core State Persistence & Merge Logic**

**Problem:** The original state update logic used a shallow merge (`dict.update()`), which overwrote nested objects and lists instead of merging them. This was the root cause of the `core_memories` list being wiped out on every turn.

**Solution:**
*   Implemented a recursive `update_state_with_changes` function that performs a deep merge of the game state.
*   The function now correctly traverses nested objects and intelligently appends items to lists, preventing data loss.
*   Refactored the function name from `deep_merge` to `update_state_with_changes` to make its non-commutative nature (i.e., order matters) clear and prevent misuse.

## **2. Context Management & Truncation**

**Problem:** As campaigns progressed, the growing conversation history would exceed the Gemini API's context window limit, leading to `RESOURCE_EXHAUSTED` errors and making long-running games unplayable.

**Solution:**
*   **Intelligent Truncation:** Implemented a new context truncation system that calculates the token count of the full conversation history before sending it to the API.
*   **Preservation of Core Memories:** If the context exceeds the budget, the system intelligently truncates the oldest turns from the narrative history while **always preserving** the entire `custom_campaign_state.core_memories` list.
