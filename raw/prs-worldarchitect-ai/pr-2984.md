## Summary

This PR introduces a customizable calendar system that allows the game to support multiple fantasy and sci-fi settings beyond the hardcoded Forgotten Realms (D&D) calendar. The system shifts calendar inference responsibility to the LLM based on story context, enabling campaigns in Westeros, Middle-earth, Eberron, modern settings, and custom worlds.

## Changes

### Core Calendar System (`mvp_site/world_time.py`)

- **Expanded MONTH_MAP**: Added support for multiple calendar systems:
  - **Forgotten Realms**: Hammer, Alturiak, Ches, Tarsakh, Mirtul, Kythorn, Flamerule, Eleasis, Eleint, Marpenoth, Uktar, Nightal
  - **Eberron**: Zarantyr, Olarune, Therendor, Eyre, Dravago, Nymm, Lharvion, Barrakas, Rhaan, Sypheros, Aryth, Vult
  - **Middle-earth**: Afteryule, Solmath, Rethe, Astron, Thrimidge, Forelithe, Afterlithe, Wedmath, Halimath, Winterfilth, Blotmath, Foreyule
  - **Gregorian**: January through December (full names and abbreviations)
  - All calendars normalized to 1-12 for temporal comparison
- **Removed hardcoded era suffix**: `format_world_time_for_prompt()` now returns neutral format ("YEAR, MONTH DAY, TIME") without "DR" suffix
- **LLM-based calendar inference**: The LLM infers appropriate calendar system from story context and formats dates in narrative (e.g., "1492 DR" for Forgotten Realms, "298 AC" for Westeros, "3019 TA" for Middle-earth)
- **Added missing abbreviation**: Added "may": 5 to abbreviations section for consistency

### Game State Instructions (`mvp_site/prompts/game_state_instruction.md`)

Major documentation overhaul with comprehensive calendar inference guidance:

- **Calendar Inference Protocol**: 
  - Mandatory LLM inference from story context (no automatic detection)
  - Reference table mapping narrative contexts to calendar systems (Westeros, Middle-earth, Forgotten Realms, Eberron, Star Wars, Warhammer 40K, modern, etc.)
  - Clear anti-patterns and correct patterns for calendar handling
  - Fallback guidance for unknown/c
