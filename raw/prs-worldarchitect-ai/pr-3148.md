## Problem
Campaign `kuXKa6vrYY6P99MfhWBn` has 33MB of story data, with 30MB (88.8%) being `debug_info` storing full LLM context in every entry:
- `raw_request_payload`: ~20KB per entry
- `system_instruction_text`: ~8KB per entry
- `raw_response_text`: ~8KB per entry

## Solution
Add `scripts/cleanup_debug_info.py` that removes these bloated fields while preserving useful debug metadata.

**Dry run results:**
```
Size before: 33.21 MB
Size after:  5.72 MB
Savings:     27.49 MB (82.8%)
```
