## Summary
Wire deterministic shard env vars in the self-hosted MVP shard workflow so each shard runs a distinct subset instead of all three running the same full suite.

## Changes
- Updated `.github/workflows/self-hosted-mvp-shard1.yml`
- Added `TEST_SHARD_TOTAL: "3"` to each shard job
- Added per-job `TEST_SHARD_INDEX` values:
  - shard 1: `0`
  - shard 2: `1`
  - shard 3: `2`

## Why
`run_tests.sh` only applies deterministic sharding when both `TEST_SHARD_TOTAL` and `TEST_SHARD_INDEX` are set. Previously these were missing in this workflow, so all three shard jobs executed the same test set.
