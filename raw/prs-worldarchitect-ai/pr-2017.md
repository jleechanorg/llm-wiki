**Problem:**
- Command was doing unnecessary tool calls (Beads checks, file searches)
- Agent was starting work investigation instead of just acknowledging
- Execution had 6+ tool calls when only 3-5 needed

**Solution:**
- Add clear directive: 'You are a message processor, NOT a task executor'
- Explicit forbidden actions (no file search, no Beads, no investigation)
- Explicit allowed actions (fetch, classify, acknowledge, report)
- Streamlined 4-step workflow (fetch → classify → acknowledge → report)
- Updated examples showing what should/shouldn't happen
- Performance target: 3-5 tool calls, <15s execution

**Impact:**
- Reduces unnecessary context usage from investigations
- Faster message processing (<15s vs ~30s+)
- Clearer separation: acknowledge in /processmsgs, work in main conversation
- Agent reports back what needs attention instead of starting work
