## 🔒 Critical Security Enhancement: API Key Filtering for Dotfile Backups

### Summary
Enhanced the `backup_dotfiles.sh` script to automatically filter out sensitive API keys and credentials before committing dotfiles to the repository. This addresses a critical security vulnerability where API keys could be accidentally committed.

### 🚨 Security Features Added

#### **Comprehensive API Key Detection:**
- ✅ **Anthropic API keys** (`sk-ant-api03-*`)
- ✅ **OpenAI API keys** (`sk-*`)  
- ✅ **Google/Gemini API keys** (`AIza*`)
- ✅ **GitHub tokens** (`ghp_*`, `gho_*`, `ghu_*`, `ghs_*`, `ghr_*`)
- ✅ **AWS Access/Session keys** (`AKIA*`, `ASIA*`)
- ✅ **Bearer tokens** and generic token patterns
- ✅ **Base64-like long strings** in quotes (40+ chars)

#### **Smart Environment Variable Filtering:**
- ✅ `export *API_KEY*=` → `[REDACTED]`
- ✅ `export *SECRET*=` → `[REDACTED]`  
- ✅ `export *PASSWORD*=` → `[REDACTED]`
- ✅ **Specific token types**: AUTH_TOKEN, ACCESS_TOKEN, REFRESH_TOKEN, SESSION_TOKEN, BEARER_TOKEN
- ✅ **Configuration preservation**: CLAUDE_CODE_MAX_OUTPUT_TOKENS, NODE_MAX_OLD_SPACE_SIZE preserved

#### **New Files Added to Backup:**
- ✅ Added `~/.gemini_api_key_secret` to backup list (with filtering)
