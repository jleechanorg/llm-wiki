## Problem
User submissions trigger two sequential LLM calls (Primary -> Rewards Followup) when the primary agent awards XP but misses the visual 'rewards_box'. This doubles latency.

## Solution
1. **Prompt Engineering:** Updated `narrative_system_instruction.md` to explicitly include the `rewards_box` schema and MANDATORY instruction to output it when XP is awarded.
2. **Architecture:** Ensures the Story Agent handles rewards in the first pass, eliminating the need for the second inference.
3. **Safety Net:** The original 'retry' logic remains as a fallback (not disabled) but should rarely trigger.

## Changes
- `mvp_site/prompts/narrative_system_instruction.md`: Added Rewards & XP Protocol section.
- `mvp_site/world_logic.py`: (Inherited latency logging from parent, ensures fallback logic is active).

## Verification
