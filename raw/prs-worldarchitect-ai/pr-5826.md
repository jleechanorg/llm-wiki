## Summary

Extract shared orchestration logic from `BaseTestRunner` and `HttpTestBase` into a new `scenario_runner` module, reducing duplication and enabling custom runners to reuse the same helpers.

## Changes

### New `scenario_runner` module
- `normalize_scenario_result(r)` – convert `TestResult` or dict to normalized dict
- `make_scenario_summary(results, duration_s, test_name)` – build summary dict
- `print_run_header(test_name, server_url, icon=…)` – run header block
- `print_scenario_results(results)` – per-scenario output with pass/fail icons
- `print_run_summary(summary)` – final summary line
- `get_scenario_evidence_dir(project_name, test_name)` – evidence work dir path
- `save_scenario_evidence(work_dir, summary, …)` – evidence bundle creation

### Refactored base classes
- **BaseTestRunner** – Uses shared helpers instead of duplicated `_make_summary` / `_save_evidence` logic
- **HttpTestBase** – Uses shared helpers; now includes `duration_s` in evidence summaries (was missing)
- **BrowserTestBase** – New `SAVE_EVIDENCE = False` flag; when `True`, prints results and saves evidence like `HttpTestBase`

### Exports
Helpers are exported from `testing_utils` for custom runners that need the same orchestration.
