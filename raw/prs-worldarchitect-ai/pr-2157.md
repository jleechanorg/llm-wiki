## Root Cause Analysis

### The Problem
All async functions in `world_logic.py` were calling blocking services (Firestore, Gemini API) directly without yielding to the event loop. This blocked the **entire shared background event loop**.

The critical path was:
```
POST /api/campaigns/{id}/interaction
  → MCP client
  → world_logic.process_action_unified()
  → gemini_service.continue_story() [BLOCKS 10-30+ seconds]
```

When `gemini_service.continue_story()` blocked, ALL other async operations queued on the same event loop were stuck waiting - including campaign loading requests.

### Why PR #2116 Didn't Catch This

PR #2116 added parallel testing but only tested the `GET /api/campaigns/{id}` endpoint:
- ✅ `GET /api/campaigns/{id}` - Fixed in main.py using `run_blocking_io()`
- ❌ `POST /api/campaigns/{id}/interaction` - Goes through MCP → `world_logic.py` which was still blocking

The test script measured parallelism on GET requests (which were already fixed), not POST interaction requests (which were still broken).
