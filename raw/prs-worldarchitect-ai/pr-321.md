## Problem
When using god mode commands, the AI sometimes returns raw JSON without a proper narrative field, causing raw JSON to be displayed to users.

## Solution
Added a dedicated `god_mode_response` field to the JSON response format that the LLM should use specifically for GOD MODE commands. This is a much cleaner approach than trying to extract text from random fields like `action` or `description`.

### Changes:
1. **Updated prompts** (`game_state_instruction.md`):
   - Added `god_mode_response` field to JSON response format
   - Added GOD MODE section with clear instructions
   - When in god mode, LLM sets `narrative` to empty string and uses `god_mode_response`

2. **Updated schema** (`narrative_response_schema.py`):
   - Added `god_mode_response` parameter to `NarrativeResponse` class
   - Modified `parse_structured_response` to return `god_mode_response` when present
   - Maintains backward compatibility with existing responses

3. **Added tests** (`test_god_mode_response_field.py`):
   - Tests god_mode_response field usage
   - Tests priority over narrative field
   - Tests backward compatibility
   - Tests edge cases

## Benefits
- Clean separation of concerns between normal gameplay and god mode
- No more complex text extraction logic
- Clear contract with the LLM about expected response format
- Prevents raw JSON from ever being displayed in god mode

All 81 tests pass successfully, including the new comprehensive test suite for god mode responses.
