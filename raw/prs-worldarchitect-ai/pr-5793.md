## Scope of changes vs `origin/main`
- Modularized Ralph CLI execution into explicit library modules under `ralph/lib/`:
  - `dashboard.py` (live dashboard server)
  - `evidence.sh` (evidence finalization, browser proof helpers)
  - `metrics.sh` (run outcome metrics persistence)
  - `status.sh` (status rendering)
  - `terminal_recorder.sh` (terminal capture lifecycle)
  - `tools.sh` (tool command resolution)
  - `workspace.sh` (workspace and prompt construction helpers)
- Added recording/automation assets for richer evidence capture:
  - `evidence_recorder.py`
  - `evidence_recorder.sh`
- Expanded `ralph/ralph.sh` run semantics:
  - centralized cleanup flow via `run_cleanup`
  - timeout-wrapped cleanup tasks (`terminal_to_srt`, `terminal_render_video`, `evidence_finalize`, `record_metrics`)
  - deterministic fallback outputs for missing evidence artifacts
  - cleanup guaranteed on run completion/failure/interruption
- Added/updated shell test coverage for major tool boundaries:
  - `ralph/tests/test_dashboard.sh`
  - `ralph/tests/test_evidence.sh`
  - `ralph/tests/test_metrics.sh`

  - `ralph/tests/test_ralph.sh` (direct `cmd_*` path coverage, including `help`, `status`, `dashboard`, `dashboard --open`, and run-failure cleanup semantics)
- Updated repo-local documentation and operational artifacts:
  - `ralph/CLAUDE.md`
