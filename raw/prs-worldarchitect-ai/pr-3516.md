## Raw Body
## Bug

PRs with unaddressed comments were incorrectly skipped because commit history was checked before the completion marker checkpoint. History is recorded after queuing (not after completion), making it stale when runs don't complete.

**Example:** PR #3468 was skipped despite having 13+ unaddressed comments because commit  was in history, but no completion marker existed (indicating a previous run didn't complete).

## Root Cause

1. **Stale history checkpoint**:  was called after queuing (line 1716), not after completion
2. **Wrong checkpoint order**: Code checked commit history BEFORE completion marker, skipping based on stale data
3. **Completion marker is authoritative**: The marker is only posted when the agent actually completes, making it the reliable checkpoint

## Fix

- ✅ Check completion marker FIRST (authoritative checkpoint)
- ✅ Only use commit history as fallback when no completion marker exists  
- ✅ Removed premature  call after queuing
- ✅ Proper checkpoint order: completion marker → history → unaddressed comments

## Checkpoint Logic

- ✅ Completion marker exists + no unaddressed comments → skip (completed successfully)
- ✅ Completion marker exists + unaddressed comments → reprocess (new comments added)
- ✅ No completion marker + history + no unaddressed → skip (likely completed, history is stale)
- ✅ No completion marker + history + unaddressed → reprocess (incomplete run detected)

## Testing

- ✅ Tested on PR #3468 - now correctly reprocesses despite commit in history
- ✅ Fix respects all checkpoints (doesn't bypass them)
- ✅ Handles edge cases: stale history, incomplete runs, new comments after completion
