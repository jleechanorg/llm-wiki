> Capture outputs can now become very large and may include sensitive prompt/user data, increasing disk/storage costs and exposure if capture files/metadata are not tightly controlled.

> **Overview**
> When LLM capture is enabled, this PR stops truncating captured artifacts and omission markers, storing the full `raw_response_text` in processing metadata and writing full `system_instruction_text`, `request_payload`, and `response_text` to the JSONL capture file.
> 
> It keeps `_get_llm_capture_max_chars` and `_truncate_llm_capture_text` as deprecated no-ops for backward compatibility, and updates the `include_raw_llm_payloads` path to return full request/response payloads for forensic auditing.
