## Raw Body
Enhanced git-header.sh to properly detect PRs when working with forked repositories:

- Added get_repo_from_remote() function to extract owner/repo from git remote
- Supports HTTPS, SSH, and local proxy URL formats
- Updated gh pr commands to use --repo flag for fork compatibility
- Ensures PR detection works even when remote is a fork

This fixes the issue where statusline would show "PR: none" even when a PR exists, particularly when working with forked repositories where gh CLI needs the --repo flag.
