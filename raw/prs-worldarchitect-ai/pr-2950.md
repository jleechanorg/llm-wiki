## Raw Body
## Summary
- Fixed daily campaign report workflow failing silently due to missing `pydantic` dependency
- Root cause: Workflow only installed `firebase-admin` and `google-cloud-firestore`, but importing `mvp_site` package requires additional dependencies

## Root Cause Analysis
1. The workflow ran `pip install firebase-admin google-cloud-firestore`
2. `daily_campaign_report.py` imports `from mvp_site.clock_skew_credentials`
3. This triggers `mvp_site/__init__.py` which eagerly loads modules including `world_logic.py`
4. `world_logic.py` imports `from pydantic import ValidationError`
5. Missing pydantic caused `ModuleNotFoundError`, producing empty report output
6. The sed/awk extraction captured blank values for all metrics

## Fix
Changed from explicit package installation to using the full requirements file:
```yaml
# Before:
pip install firebase-admin google-cloud-firestore

# After:
pip install -r mvp_site/requirements.txt
```
