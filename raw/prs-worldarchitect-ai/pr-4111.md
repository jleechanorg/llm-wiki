## Summary
This PR centralizes dice **authoring** into `action_resolution.mechanics.rolls/audit_events`. Legacy `dice_rolls` / `dice_audit_events` fields are now **server-derived** only (no LLM population, no legacy backfill).

## Key Changes (Latest Commits)

### Simplification (commit 0763f8a, 8bddacc)
- **Removed legacy field validation** - only `action_resolution.mechanics.rolls/audit_events` is checked for dice detection
- **Removed ~155 lines** of legacy validation code and security-related tests
- **Removed legacy backfill** - `add_action_resolution_to_response` no longer populates `dice_rolls`/`dice_audit_events`
- **Added `has_action_resolution_dice()` helper** for clean dice presence checks

### Warning Gating (commit 09cdd71)
- `ACTION_RESOLUTION_DICE_NO_CODE_EXECUTION` warning only fires for `CODE_EXECUTION` strategy (not `NATIVE_TWO_PHASE`)
- Fixed duplicate test class (`TestAddActionResolutionNormalization`)
