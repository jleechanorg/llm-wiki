## Raw Body
### Goal
Prevent orphan or misleading automation comments by ensuring the *actual dispatched CLI* passes both a quick `--help` check and a real 2+2 validation **immediately before** posting queued notices.

### Why this change
- CLIs can go out of quota between dispatch and comment posting; `--help` alone is not enough to detect that.
- Dispatcher may reorder the CLI chain after validation, so preflight must target the CLI that was actually dispatched.
- If the queued marker fails to post, we need a guard to prevent duplicate agent dispatches on subsequent cycles.

### What changed
- Quick preflight now runs **two‑phase validation** (help + 2+2) using the orchestration validator for the dispatched CLI.
- Preflight checks the **effective CLI** returned by the dispatcher (not just the first entry in the original chain).
- Added an **in‑flight fix‑comment lock** when posting the queued marker fails, preventing duplicate dispatches.
- Tightened dangerous‑content bypass for bot comments to only allow patterns inside code blocks/inline code.

### Testing
