## Description
Adjusts rate limits to be more conservative for alpha testing and improves the UI to show users when they can play again.

## Changes
- **Backend**: 
  - Daily limit: 100 -> 50 turns
  - Window limit: 30 turns/3h -> 25 turns/5h
  - **Refactor**: Extracted rate limiting logic from `firestore_service.py` to a dedicated `mvp_site/rate_limiting.py` module.
  - Updated error messages to reflect new limits
- **Frontend**:
  - `fetchApi` now parses `reset_time_hourly` (mapped to window) and `reset_time_daily` from 429 responses
  - Rate limit modal now displays a countdown: 'Limit resets in: X hours and Y minutes'
- **Tests**:
  - **Unit Tests**: Added comprehensive unit tests in `mvp_site/tests/test_rate_limiting.py` (100% coverage).
  - **E2E Tests**: Updated `test_rate_limiting_real_e2e.py` to reflect new limits and 5-hour window.
