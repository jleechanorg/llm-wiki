## Summary

This PR implements the **DELETE-TESTING-MODE** handoff task, removing the dual-mode authentication system that was causing configuration confusion, debugging complexity, and maintenance burden.

**Key Changes:**
- ✅ **Backend**: Removed `should_skip_firebase_init()` function - Firebase now always initializes
- ✅ **Frontend**: Removed `testAuthBypass` logic - authentication now always uses real Firebase  
- ✅ **Auth Flow**: Single authentication path using real Firebase in all environments
- ✅ **Headers**: Removed `X-Test-Bypass-Auth` and `X-Test-User-ID` header handling

## Problem Solved

The previous testing mode created two parallel authentication paths:
- **Frontend**: Real Firebase SDK vs URL parameter test bypass (`?test_mode=true`)
- **Backend**: Firebase Admin SDK vs header bypass (`X-Test-Bypass-Auth`)

This dual-mode system caused:
- Configuration mismatches between frontend and backend modes
- Poor debugging practices (escape hatches vs fixing real issues)
- Maintenance burden (two codepaths to maintain and debug)
- Testing gaps (mock testing instead of real integration)

## Implementation Details

### Backend Changes
- **`firebase_utils.py`**: Removed `should_skip_firebase_init()` and `_env_truthy()` functions
- **`main.py`**: Always initialize Firebase, removed auth bypass logic from `check_token()`
- **`world_logic.py`**: Always initialize Firebase, removed testing conditionals

### Frontend Changes  
- **`api.service.ts`**: Removed all `testAuthBypass` logic and URL parameter parsing
- **Simplified Methods**: `login()`, `logout()`, `getCurrentUser()`, `isAuthenticated()` now always use Firebase
- **Auth Headers**: Always use `Authorization: Bearer <token>` (removed test headers)
