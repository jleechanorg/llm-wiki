## Summary

- Fix character-sheet display for campaigns where the LLM stores equipped items in `player_character_data.equipped_items` (canonical slot map) — weapons/armor were invisible to the equipment UI button
- Fix stat bonus calculation to include items from `equipped_items` (e.g. `Ring of Strength +2` was yielding STR bonus = 0)
- Fix `_PLAYER_CHARACTER_SCHEMA_KEYS` missing `equipped_items`, `cantrips`, `spells` — these fields were being stripped from `player_character_data` on every `validate_and_correct_state()` call and moved to `player_character_data_extras`
- Add `testing_mcp/CLAUDE.md` and `testing_mcp/agents.md` documenting no-mocks policy and MCPTestBase requirements
- Add integration test `testing_mcp/creation/test_character_data_display.py` covering equipment display, stat bonuses, ability scores, and spells/cantrips end-to-end against real Firestore

## Production Code Changes

### `mvp_site/game_state.py` — Schema key registration
- **Before:** `_PLAYER_CHARACTER_SCHEMA_KEYS` missing `equipped_items`, `cantrips`, `spells`. `_move_non_schema_player_character_fields_in_place()` silently relocated these to `custom_campaign_state.player_character_data_extras` on every write, making them invisible to all display endpoints.
- **After:** Added `"equipped_items"`, `"cantrips"`, `"spells"` to `_PLAYER_CHARACTER_SCHEMA_KEYS`. Fields persist in `player_character_data` as written by the LLM.
- **Why:** Real campaigns (Ser Arion, Visenya) store equipped items in `equipped_items`; without this fix, subsequent writes would strip the field.

### `mvp_site/equipment_display.py` — BUG #1 fix
- **Before:** `extract_equipment_display()` only read `pc_data.get("equipment")` (the runtime backpack alias). Items in `pc_data.get("equipped_items")` (canonical LLM-written slot map) were never surfaced.
- **After:** After building the `equipped` dict from `equipment.get("equipped", {})`, merge canonical `equipped_items` with `setdefault()` so each slot appears at most once.
