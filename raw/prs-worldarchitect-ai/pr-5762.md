## Production Code Changes

**Before**: Testing infrastructure was scattered across `testing_mcp/`, `testing_ui/`, and `testing_http/` with duplicated utilities and no shared library.

**Now**:
- `testing_utils/` is a standalone PyPI package with 7 modules: `http_client`, `evidence`, `server`, `base_test`, `browser`, `http_test`, `logging_capture`
- `testing_mcp/lib/mcp_client.py` now re-exports from `testing_utils.http_client` (354 lines â†’ simple re-export)
- `testing_mcp/lib/server_utils.py` imports `LocalServer`, `pick_free_port`, etc. from `testing_utils.server`; keeps WA-specific policy (no-mock enforcement, GCP Secret Manager)
- `testing_mcp/lib/evidence_utils.py` imports generic helpers aliased as `_tu_*` to avoid shadowing WA-specific overrides
- `testing_ui/config.py` delegates branch name and tmp root resolution to `testing_utils.evidence`
- `testing_http/test_config.py` refactored with `WorldArchitectHttpTest(HttpTestBase)` and `create_test_session()` from shared library

**Why**: Consolidate testing infrastructure for reuse across projects, enable PyPI publishing, and allow server-side logging of LLM/HTTP calls for evidence bundles.
