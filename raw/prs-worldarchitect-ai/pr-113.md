## Problem

When the AI generates state updates like:
```json
{
  "npc_data": {
    "goblin_skirmisher_1": "defeated"
  }
}
```

The existing `update_state_with_changes` function would completely overwrite the NPC's dictionary structure:

**Before (corrupted):**
```
npc_data['goblin_skirmisher_1'] = "defeated"  // Lost all NPC data\!
```

**After (preserved):**
```json
{
  "hp_current": 7,
  "ac": 13, 
  "alignment": "chaotic evil",
  "status": "defeated"  // String intelligently merged as status
}
```

## Root Cause

The issue was in `firestore_service.py:64` where the function logic had a gap:
- **Case 3**: Only handled dict-to-dict merging 
- **Case 4**: Would overwrite everything else, including dict-to-string updates

This caused the entire NPC data structure to be destroyed when the AI sent status updates.

## Solution

Enhanced the update logic with three new intelligent handling cases:

1. **Case 3b**: Create new dictionaries when incoming value is dict but existing is not
2. **Case 4**: Preserve existing dictionary structures when strings are applied to them  
3. **Case 5**: Simple overwrite (moved from Case 4)

String values applied to existing dictionaries are now treated as status updates rather than complete overwrites.
