## Production Code Changes

### 1. `mvp_site/dice_integrity.py` (+160 lines)
**Purpose:** Add reconciliation function to fix LLM stdout schema inconsistencies

- **New function `reconcile_dice_audit_events_with_stdout()`**: Reconciles dice_audit_events with code_execution stdout as authoritative source of truth
- **Handles `roll` (singular) vs `rolls` (array)**: Helper `_extract_rolls_from_stdout_event()` extracts rolls from either format
- **NDJSON parsing**: Parses newline-delimited JSON when stdout contains multiple objects
- **Double-consumption prevention**: Tracks consumed events by `id()` to prevent matching same stdout event to multiple audit events
- **Fallback matching safeguard**: Only matches by total when exactly one candidate exists (prevents ambiguous matches)

**Justification:** LLM outputs inconsistent JSON schemas despite clear instructions. Server-side reconciliation is the only reliable fix.
