## Production Code Changes

### `mvp_site/streaming_orchestrator.py`
- **Before**: `if not structured_fields.get("world_events")` — treated empty `{}` as missing, allowing bleed-through from `state_updates`
- **Now**: `if "world_events" not in structured_fields` — key-presence check, respects explicit empty payloads
- **Before**: Both propagation points silently skipped mirroring `world_events` into `state_updates` when it was absent
- **Now**: Both points create `state_updates = {}` and back-fill `structured_fields["state_updates"]` when absent

### `mvp_site/world_logic.py` — `_filter_story_entry_world_events_to_scene`
- **Before**: Only filtered `background_events` (list) and `rumors` (list)
- **Now**: Also filters `faction_updates` + `time_events` (dict-of-dicts, keep entries matching scene), `complications` + `scene_event` (single dicts, remove if wrong scene)
- **Why**: Prevents cumulative history bleed for all living world event types, not just lists
