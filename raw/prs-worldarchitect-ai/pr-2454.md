## Raw Body
Bug: When killing an NPC (e.g., Marcus), the game still offered options to kill them again on subsequent turns. Death state wasn't syncing between combat_state and npc_data.

Root Cause 1: apply_automatic_combat_cleanup NOT called in main story flow
- GOD_MODE flow correctly called cleanup, but main story flow didn't
- Added cleanup call in world_logic.py after update_state_with_changes

Root Cause 2: cleanup_defeated_enemies deleted ALL NPCs, including named ones
- Named NPCs (with role/backstory/is_important) need to persist with status: ["dead"] so LLM knows they're dead in future turns
- Modified game_state.py to mark named NPCs as dead instead of deleting
- Generic enemies (role=enemy/minion/generic, no backstory) still deleted

TDD Tests added:
- Named NPC with role marked dead, not deleted
- Named NPC with backstory marked dead
- Named NPC with is_important flag marked dead
- Generic enemies still deleted (regression tests)
- Integration test: dead NPC not offered as combat target
