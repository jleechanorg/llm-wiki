## Summary
Slack thread context: `EPHEMERAL=true` (previous default in `start-runner.sh`) makes each runner deregister after **one** job; Docker `--restart unless-stopped` brings containers back with **new** random names while old registrations stay **offline** in GitHub — looks like runners are broken when capacity is fine.

## Change
- Default `EPHEMERAL` to **false** in `start-runner.sh` (compose already defaulted false).
- Update `.env.example`, `README.md`, and `pre-job-hook.sh` comment.

## Ops note
Machines with **gitignored** `self-hosted-oss/.env` should set `EPHEMERAL=false` (or remove the line to use default) and **restart** runner containers to apply.

## Rollback
Set `EPHEMERAL=true` in `.env` if you need strict one-job isolation.
