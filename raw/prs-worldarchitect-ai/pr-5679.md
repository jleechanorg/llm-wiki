## Production Code Changes
- **`.github/workflows/pr-preview.yml`** — `--set-env-vars` included `TESTING_AUTH_BYPASS=true,ALLOW_TEST_AUTH_BYPASS=true` → removed both → dead code: `check_token` bypass paths at `mvp_site/main.py:1020,1062,1083` all require `PRODUCTION_MODE != "true"`, which is never satisfied in preview deploys

## Analysis
The `check_token` decorator has three bypass paths, all gated by `PRODUCTION_MODE`:
1. **MCP dev bypass** (line 1020): `PRODUCTION_MODE != "true"` AND no auth headers → skips auth for local `/mcp`
2. **Test bypass** (line 1062): `PRODUCTION_MODE != "true"` AND `is_localhost` AND both bypass flags → allows test headers
3. **Mock mode** (line 1083): `PRODUCTION_MODE != "true"` → warning path only

Since preview sets `PRODUCTION_MODE=true`, none of these paths activate. The bypass env vars were dead on arrival.
