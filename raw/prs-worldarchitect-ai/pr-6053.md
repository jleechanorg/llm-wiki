Misconfiguration, API pagination/GraphQL issues, or author-matching mistakes could cause unintended merges or block merges unexpectedly.
> 
> **Overview**
> Adds a new on-PR `Skeptic Gate` workflow that posts/updates a trigger comment keyed by the PR head SHA, then polls PR comments for a fresh `VERDICT: PASS|FAIL` from `SKEPTIC_BOT_AUTHOR` (with SHA + timestamp filtering, API failure handling, and timeout behavior).
> 
> Adds a scheduled/manual `Skeptic Cron` workflow that paginates through all open non-draft PRs, evaluates a **7-gate merge policy** (CI status, mergeability, CodeRabbit approval, Cursor error comments, GraphQL-resolved review threads, optional evidence-bot approval, and an explicit Skeptic `VERDICT`), and auto-merges PRs that pass (squash/auto/delete-branch), failing closed on missing/errored verdict and GraphQL thread lookups.
