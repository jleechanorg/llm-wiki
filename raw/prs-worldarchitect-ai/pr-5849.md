> [!NOTE]
> **Medium Risk**
> Touches the streaming LLM pipeline (provider routing, parsing fallbacks, and Firestore persistence), so regressions could affect live story delivery or stored campaign text despite added unit/E2E coverage.
> 
> **Overview**
> Adds **OpenRouter support to `continue_story_streaming`** by implementing synchronous SSE chunk streaming in `openrouter_provider.generate_content_stream_sync()` and routing both phase-1 and phase-2 streaming calls through it (including mock-mode call-through for tests and execution-trace provider annotations).
> 
> Hardens streaming parse/persistence behavior by centralizing the JSON-parse fallback string as `JSON_PARSE_FALLBACK_MARKER`, using raw-text narrative fallback when streamed JSON parsing fails (without clobbering tool-request flows), and preventing `streaming_orchestrator` from persisting the fallback marker to Firestore when a real `full_narrative` is available.
> 
> Improves test and evidence tooling: adds unit tests for OpenRouter SSE parsing/response_format policy, streaming allowlist tests, a real-mode MCP OpenRouter streaming E2E test, and post-processes evidence `.sha256` sidecars to use evidence-root-relative paths; also updates smoke-token auth to set per-request m
