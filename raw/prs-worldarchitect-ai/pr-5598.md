## Summary

## Changes vs `origin/main`
- Pair defaults and model routing
- Switched pair defaults to `minimax` for both coder and verifier.
- Added role-specific model flags: `--coder-model`, `--verifier-model`.
- Kept `--model` as a legacy alias for coder-only model selection.
- Updated pair command docs/examples and integration test spec to match new defaults.

- Strict CLI behavior
- Enforced strict no-fallback behavior for selected CLIs in pair execution paths (fail on preflight/launch issues rather than silently switching CLIs).
- Preserved explicit CLI selection (`--coder-cli` / `--verifier-cli`) for mixed runs like MiniMax+Codex.

- Monitor and orchestration robustness
- Improved pair monitor startup handling for tmux session visibility races.
- Added/updated monitor tests for completion-state and startup behavior.
- Refined orchestration CLI/env handling and aligned tests with current profile expectations.

- Real integration tests
- Added real pair integration tests under `.claude/scripts/integration/test_pair_real_integration.py`.
- Moved pair execution spec to `.claude/scripts/integration/test_pair_command_execution.md`.
- Hardcoded integration test execution to MiniMax coder+verifier.

- Legacy path/test alignment
- Removed legacy `scripts/pair_execute.py` and aligned script tests with canonical `.claude/scripts/*` modules.
- Updated `scripts/tests/test_pair_orchestration.py` imports/expectations to current APIs.
