## Summary
Integrates beads-based work item tracking into the planning and execution command workflows. Adds a mandatory Phase 1.5 that creates beads for each work item identified in the plan, with fallback mechanisms for different beads availability levels (MCP, CLI, or direct file creation). Updates execution workflow to track and update bead status throughout implementation.

## Changes
- **planexec.md**: Added Phase 1.5 "Beads Creation (MANDATORY)" with:
  - Three-tier fallback approach: MCP → CLI → direct file creation
  - Bead structure specification (id, title, description, status, priority, issue_type, timestamps)
  - Critical rules for bead creation and tracking
  - Updated Phase 2 output format to include "Beads Tracking" section with bead IDs and status updates
  
- **execute.md**: Enhanced execution workflow to:
  - Update bead status during execution (todo → in_progress → done/blocked)
  - Include beads MCP/CLI update commands with examples
  - Require `.beads/` changes in git commits per CLAUDE.md rules
  - Updated reference examples to show bead creation and status transitions throughout execution phases

## Benefits
- **Work Item Visibility**: Each task in the plan is now tracked as a discrete bead, improving visibility into work progress
- **Status Tracking**: Beads status updates during execution provide real-time progress indication
- **Flexible Integration**: Three-tier fallback ensures beads tracking works regardless of available tools
- **Audit Trail**: Beads with timestamps provide a complete record of work items and their lifecycle
- **Consistency**: Aligns planning and execution workflows with beads-based task management
