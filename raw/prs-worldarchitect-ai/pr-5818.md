## Raw Body
## Summary
- Stabilize self-hosted mikey runner routing with label fallback via `SELF_HOSTED_RUNNER_LABELS`
- Fix shard workflow checkout: SHA-pinned `actions/checkout`, deterministic ref, removed `persist-credentials: false` (caused auth-loss on retry)
- Start 2 isolated runner containers by default (`RUNNER_COUNT=2`) with per-container workdirs to prevent shared `_actions` corruption
- Set `EPHEMERAL=true` as default: persistent mode causes `tar` permission errors on macOS colima bind-mounts when re-extracting action test data between jobs
- Clean `_actions`, `_temp`, and repo workdir on runner startup to prevent stale git object corruption
- Restore frontend `app.js` load ordering and streaming orchestrator patch target for existing tests

## Why
Self-hosted CI failures traced to two root causes:
1. **Runner state corruption** — stale `_actions` extractions and corrupt `tmp_pack_*` git objects in `_work` from aborted fetches caused every job to fail in `Set up job` before code ran
2. **Non-ephemeral mode on macOS** — `EPHEMERAL=false` caused the second job in a container to fail extracting `setup-python` because test data files written read-only by the first job blocked `tar` on the macOS colima filesystem

## Production Code Changes
- `self-hosted-oss/start-runner.sh`: RUNNER_COUNT=2 containers, isolated workdirs, EPHEMERAL=true default, startup cleanup of `_actions`/`_temp`/repo workdir
- `self-hosted-oss/.env.example`: EPHEMERAL=true
- `.github/workflows/self-hosted-mvp-shard1.yml`: removed `persist-credentials: false` from all 4 self-hosted jobs

## Validation
- Local runners confirmed working: 2 containers picking up jobs in parallel, ephemeral re-registration between jobs
- Stale offline runners cleaned up (15 deleted, 2 active)

## Known Limitations
- Other mikey machines (non-local) still need `./start-runner.sh` re-run to pick up the new defaults
