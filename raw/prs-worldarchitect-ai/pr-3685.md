## Summary

## Changes

### Core Logic Fixes
- **Skip clean PRs**: Reorder conditional logic to FIRST check if there are issues to fix
- **API failure handling**: Introduce `status_unknown` flag - treat API failures conservatively (do not skip as clean)
- **Reprocessing persistent issues**: Re-enable reprocessing for PRs with conflicts/failing checks even if commit was already processed
- **Log message accuracy**: Fix ternary to show "unknown status" instead of incorrectly defaulting to "failing checks"
- **Correct dispatch function**: Use `dispatch_agent_for_pr()` for FIXPR tasks (was incorrectly using fix-comment dispatch)

### Fix-Comment Workflow
- **Completion marker reprocessing**: Handle new conflicts/failing checks that appear after completion marker
- **Skip condition hardening**: Only skip when no unaddressed comments AND no conflicts AND no failing checks AND status is known
- **Post-dispatch cleanup**: Add cleanup call after agent dispatch for consistency

### Orchestration
- **Remove CLI fallback**: Agent creation fails if requested CLI is missing or validation fails (prevents Codex quota drain)
