## Production Code Changes

### 1. `mvp_site/main.py` (+85/-14 lines)
**Purpose:** Version-based cache busting for frontend_v1 static files

**Key changes:**
- Added `get_app_version()` function (lines 147-175)
  - Returns git commit hash (8 chars) or timestamp fallback
  - Cached globally for performance
  - Used as version identifier for cache busting

- Updated `frontend_files_with_cache_busting()` route (lines 782-809)
  - Detects `?v=` query param for version-aware caching
  - Versioned assets: Cache-Control max-age=31536000 (1 year immutable)
  - Non-versioned: Short cache times (5 min for JS/CSS, no-cache for HTML)

- Modified `serve_frontend()` function (lines 2716-2743)
  - Reads index.html and injects version into script/link tags
  - Regex-based injection: `<script src="/frontend_v1/app.js">` → `<script src="/frontend_v1/app.js?v=abc123ef">`
  - Returns modified HTML with no-cache headers

### 2. `mvp_site/firestore_service.py` (+10/-0 lines)
**Purpose:** Backfill action_resolution from outcome_resolution for legacy data

**Key changes:**
- In `get_story_paginated()` (lines 1507-1519)
  - Imports `has_action_resolution_dice` utility
  - Copies outcome_resolution to action_resolution when:
    - action_resolution is missing, OR
    - action_resolution exists but has no dice while outcome_resolution has dice
  - Ensures legacy dice data visible to UI
