## Summary
Removed automatic system theme detection (`prefers-color-scheme` media query) from theme initialization and management. The application now defaults to light theme for all users, with manual theme selection available through the theme menu.

## Changes
- **theme-init.js**: Removed system preference detection; now defaults to light theme instead of respecting `prefers-color-scheme: dark`
- **theme-manager.js**: 
  - Removed system preference check in `initializeTheme()` method
  - Removed system theme change listener that auto-switched themes when OS theme changed
  - Updated comments to reflect light theme as default

## Benefits
- Simplified theme initialization logic
- Consistent default experience for all users
- Reduced reliance on system-level preferences
- Users maintain full control through explicit theme selection in the UI
- Eliminates unexpected theme switches when system preferences change
