## Summary
This stacked follow-up builds on #5860 and moves Docker model provisioning toward a single, deterministic preflight flow:

- Added optional **GCS-backed FastEmbed cache restore** in `scripts/preflight_model_docker.py` via `FASTEMBED_GCS_ARCHIVE_URI` (`gs://...`).
- Added `FASTEMBED_GCS_REQUIRED` guardrail for strict builds that must fail if GCS restore/validation fails.
- Kept robust fallback: when GCS cache is absent or invalid (and not required), preflight falls back to direct HuggingFace download during build.
- Simplified runtime/bootstrap path by removing obsolete HF tree-cache mechanism (`scripts/build_hf_tree_cache.py` + Docker references).
- Simplified `gunicorn.conf.py` startup checks to cache-directory presence + offline guardrail messaging only.

## Risks
- **Archive format/structure mismatch risk**: malformed or unexpected tar archive layout could cause restore failures. Mitigation: safe tar path validation, validation load after restore, and fallback/strict mode control.
- **Credential/env risk in CI/build**: GCS restore requires ADC/permissions when `FASTEMBED_GCS_ARCHIVE_URI` is set. Mitigation: optional mode by default; strict mode via `FASTEMBED_GCS_REQUIRED=true` only when intended.
- **Dependency availability risk**: `google-cloud-storage` import may be unavailable in some build contexts. Mitigation: logs warning and falls back unless strict mode is enabled.
