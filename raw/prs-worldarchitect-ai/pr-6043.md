## Summary
Fixes failing CI in the `.claude` directory test shard: `TestGitHeaderStatusline.test_red_essential_output_format` (seen on [run 23433095702](https://github.com/jleechanorg/worldarchitect.ai/actions/runs/23433095702) for PR #5879).

## Cause
`git-header.sh` only prints `PR:` on line 2 when a PR is known (`short_pr` is set and not `none`). A fresh temp repo with no remote never emits `PR:`, so the test was asserting behavior the script does not guarantee.

## Change
- Require `Dir:`, `Branch:`, parenthesized sync status, and the always-present `ctx` label on line 2 instead of `PR:`.
- Drop unused `import glob` (ruff).

Unrelated to PR #5879; this is a test/spec alignment after the statusline layout change.
