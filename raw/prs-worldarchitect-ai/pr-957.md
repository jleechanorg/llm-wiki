## Summary

This PR fixes an issue where `integrate.sh` was being overly cautious and blocking integration workflows when unrelated PRs existed that modified integration infrastructure. The script now intelligently distinguishes between PRs that actually conflict with the current branch and those that are simply touching integration-related files.

## Problem Solved

**Issue**: User encountered blocking behavior when running `./integrate.sh` due to unrelated PR #953 (feature PR modifying integrate.sh). The script was designed to be cautious about integration conflicts but was being too broad in its checks.

**Root Cause**: The `check_existing_sync_pr()` function was using a blanket search that matched any PR containing integration-related keywords, causing unnecessary workflow disruption.

## Key Changes

### 🎯 Intelligent PR Filtering
- **Before**: Blocked on ANY PR that mentioned "integrate.sh" 
- **After**: Only blocks on actual sync PRs created by integrate.sh itself
- **Enhancement**: Informational warnings for integration infrastructure PRs without blocking workflow

### 🚨 Enhanced Force Mode
- **--force** flag now overrides sync PR warnings in addition to existing uncommitted/unpushed change warnings
- Clear messaging about what force mode bypasses
- Updated documentation to reflect enhanced functionality

### 📖 Comprehensive Help Documentation  
- Added `--help` / `-h` flag with detailed usage guide
- Examples for all supported workflows (standard, custom branch, force mode, new-branch mode)
- Clear descriptions of safety features and workflow steps
- Detailed option explanations

### 🔧 Improved Error Handling
- Proper fallback when `jq` is not available
- Enhanced messaging to distinguish between different types of PR conflicts
- Better user guidance with specific next steps
