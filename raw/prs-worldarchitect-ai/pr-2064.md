## Problem

Campaign creation fails with error:
```
FileNotFoundError: [Errno 2] No such file or directory: '/app/mvp_site/world/world_assiah_compressed.md'
```

This occurs when creating campaigns with the default world enabled.

## Root Cause

The deploy script was looking for the world directory in the project root, but the world directory already exists in `mvp_site/world/`. The script wasn't checking for the existing world directory first, and wasn't verifying that the required file exists after copying.

## Solution

1. **Check for existing world directory first**: If `mvp_site/world/` exists and contains `world_assiah_compressed.md`, use it directly (no copy needed)
2. **Fallback to project root**: If not found in mvp_site, copy from project root `world/` directory
3. **Verification**: After copying, verify that `world_assiah_compressed.md` exists
4. **Fail fast**: Changed warnings to errors so deployment fails early if world files are missing

## Changes

- Modified `deploy.sh` to check for world directory in `mvp_site/world/` first
- Added verification step to ensure required file exists after copy
- Improved error messages to show all checked locations
- Changed warnings to errors for missing world files
