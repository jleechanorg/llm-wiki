## Summary
Restructured the parallel execution protocol to distinguish between synergistic, compatible, and conflicting task combinations. This introduces a three-tier model that rewards smart tactical planning while penalizing genuinely conflicting resource demands, with clearer mechanics and comprehensive documentation.

## Changes
- **Renamed `parallel_execution` to `parallel_synergistic`** with updated example showing tasks that benefit from simultaneous execution (Avernus audit + Waterdeep focus)
- **Added `parallel_conflicting` option** demonstrating high-DC alternatives with unique outcomes impossible via sequential execution (planar projection oversight)
- **Introduced `combination_type` field** in analysis to explicitly categorize task relationships (synergistic, compatible, conflicting)
- **Replaced `coordination_dc` calculation model**:
  - Synergistic/Compatible: +0 DC (no penalty for smart combinations)
  - Conflicting: +2 to +4 DC (scaled by severity of resource conflict)
- **Added `dc_modifier` field** to analysis for explicit penalty tracking
- **Expanded documentation** with three-tier parallel execution model table, detailed mechanics for each combination type, and multiple worked examples
- **Clarified synergy bonuses** as narrative outcomes (enemies can't react, evidence preserved, etc.) that justify the tactical choice
- **Added outcome mechanics section** with time-cost calculations for 2, 3, and 4+ parallel tasks
- **Included guidance** on when to present multiple parallel options and how to justify DC penalties with unique outcomes

## Benefits
- **Clearer design intent**: Distinguishes between efficient parallel execution and high-risk tactical maneuvers
- **Better player agency**: Synergistic options reward creative thinking without penalty, while conflicting options offer exclusive outcomes for those willing to accept risk
- **Improved balance**: DC penalties now directly tied to resource conflicts rather than arbitrary task counts
