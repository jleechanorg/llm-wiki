## Problem
- Both user inputs and AI responses get sequence_ids (1, 2, 3, 4...)
- Only AI responses are displayed with scene numbers
- This made scene numbers appear to jump: Scene #2, Scene #4, Scene #6...

## Solution
- Added `user_scene_number` that only increments for AI responses
- Backend calculates and returns this in API responses
- Frontend uses `user_scene_number` for display (with fallback to `sequence_id`)
