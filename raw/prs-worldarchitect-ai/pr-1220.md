## Summary

This PR adds the claude-code-requirements-builder command system and documents comprehensive requirements for fixing the missing planning block feature in V2.

## Changes

### 1. Requirements Builder Installation
- Added 6 requirements gathering commands from [rizethereum/claude-code-requirements-builder](https://github.com/rizethereum/claude-code-requirements-builder)
- Commands enable progressive requirements gathering through yes/no questions
- Created requirements/ directory structure for storing requirement docs

### 2. V2 Planning Block Requirements Documentation
- Conducted full requirements gathering process for fixing missing planning blocks in V2
- Documented problem: Planning blocks completely missing from V2 after campaign creation
- Identified solution: Create new PlanningBlock.tsx component with V2 theming

## Requirements Commands Added
- `/requirements-start` - Begin new requirement gathering
- `/requirements-current` - View active requirement details  
- `/requirements-status` - Check progress and continue
- `/requirements-end` - Finalize requirement gathering
- `/requirements-list` - List all requirements with status
- `/requirements-remind` - Remind AI to follow requirements rules

## Planning Block Requirements Summary

### Problem
- V2 interface missing planning blocks that exist in V1
- Users cannot see/interact with character action choices after campaign creation
- Critical regression preventing gameplay progression

### Solution
- Create new PlanningBlock.tsx component in frontend_v2/src/components/
- Extend StoryEntry type to support 'planning' type
- Add TypeScript definitions for planning block data
- Use V2's current theme (not V1's blue background)
- Display planning blocks immediately after AI narrative responses
