## Summary

- **Lazy startup warmup**: All lazy modules (`firestore_service`, `gemini_provider`, `llm_service`, `openclaw_provider`, `world_logic`) now preload inside `create_app()` before the first user request, eliminating the cold-request init penalty per worker
- **SMOKE_TOKEN centralization**: New composite action `.github/actions/run-mcp-smoke-tests/` is the single source of truth for `SMOKE_TOKEN → mcp-smoke-tests.mjs` env mapping; fixes auto-deploy smoke tests that were broken since Jan 31
- **Benchmark reliability**: `test_explicit_caching_latency.py` iterations 5→20, p50/p95 reporting, corrected docstring; `test_homepage_latency.py` adds `Accept-Encoding: gzip` to verify compression

## Production Code Changes

| File | Before | Now | Why |
|---|---|---|---|
| `mvp_site/main.py` | Lazy modules loaded on first user request | `_warm_startup_lazy_dependencies()` called in `create_app()` | Eliminates 500ms+ cold-request penalty — confirmed working in GCP logs |
| `mvp_site/streaming_orchestrator.py` | No warmup | `warm_lazy_dependencies()` exported for main to call | Streaming modules (gemini_provider, llm_service) also preloaded |

## CI Changes

| File | Before | Now |
|---|---|---|

| `.github/actions/run-mcp-smoke-tests/action.yml` | (new) | Composite action: Node.js 20 setup + `mcp-smoke-tests.mjs` invocation + SMOKE_TOKEN mapping in one place |
