## Summary

This PR implements comprehensive settings functionality for WorldArchitect.AI, focusing on debug mode and Gemini model selection with full UI/backend consistency and rigorous testing.

## Key Features

### 🔧 Settings Infrastructure
- Complete settings API endpoints for GET/POST operations
- User settings persistence in Firestore (users/{user_id}/settings collection)
- Settings UI with auto-save functionality (300ms debounce)
- Form validation and error handling

### 🐛 Debug Mode Control
- Toggle debug mode on/off via settings UI
- **Backend Game State Filtering**: Strips sensitive debug fields when OFF
- Debug information visibility controlled by user preference  
- Debug indicator in UI reflects actual user setting

### 🤖 Gemini Model Selection  
- Choose between Flash 2.5 and Pro 2.5 models using proper Gemini SDK format
- Model selection persists across sessions
- Backend respects user's model preference for AI interactions

## Technical Implementation

### Backend Changes
- **Settings API**: Complete CRUD operations for user preferences
- **Debug Mode Integration**: User settings override game state defaults
- **Game State Filtering**: Backend strips debug fields when debug mode OFF
  - **Strips**: `entities_mentioned`, `entities`, `state_updates`, `debug_info`
  - **Preserves**: `resources`, `dice_rolls`, `location_confirmed`, `planning_block`, `god_mode_response`
- **Model Selection**: Gemini service uses user's preferred model
- **Data Consistency**: Single source of truth for all settings

### Frontend Changes  
- **Settings UI**: Clean, responsive interface with Bootstrap components
- **Auto-save**: Seamless user experience with debounced saves
- **State Management**: Real-time UI updates based on settings
- **Navigation**: Integrated settings access via SPA routing
- **Code Cleanup**: Removed redundant frontend debug mode checks
