Adds a new optional `freeze_time` boolean field to planning block choices. When a player selects a choice with freeze_time=true, time advances by only 1 microsecond (like Think Mode), preventing in-game time from passing during meta-game decisions like level-up.

Changes:
- Add freeze_time field to CHOICE_SCHEMA with boolean validation
- Add _should_freeze_time_for_selected_choice() helper in world_logic
- Add _extract_recent_planning_blocks() helper for story context
- Update time filtering to use should_freeze_time (combines is_think_mode + freeze_time choice)
- Inject freeze_time=true on level_up_now and continue_adventuring choices
- Document freeze_time in planning_protocol.md
- Add get_selected_choice() helper to document_generator
- Add comprehensive tests for freeze_time behavior

This ensures level-up decisions don't advance the in-game clock.
