Enhance the copilot workflow with comprehensive comment URL tracking to improve auditability and traceability of which GitHub comments were fixed, considered, or deferred during PR processing.

## Changes
- **copilot.md**: 
  - Added `🔗 **Comment URL Tracking**` to quality metrics section documenting the new feature
  - Updated response templates (`template_fixed`, `template_deferred`, `template_acknowledged`, `template_not_done`, `template_multi_issue`) to include `html_url` field for GitHub comment URLs
  - Added new section "Generate commit message with comment URLs for tracking" that:
    - Extracts comment URLs from responses.json grouped by response type (FIXED vs CONSIDERED)
    - Falls back to matching comment IDs from comments.json if URLs not found in responses
    - Generates structured commit messages with separate sections for fixed and considered comments
    - Creates tracking commits with `[codex-automation-commit]` marker
  - Updated documentation to specify `html_url` SHOULD be included in all response objects for tracking and linking

- **commentfetch.py**:
  - Added `html_url` field extraction in `_get_inline_comments()` method
  - Added `html_url` field extraction in `_get_general_comments()` method
  - Added `html_url` field extraction in `_get_review_comments()` method
  - Ensures all comment types (inline, general, review) include GitHub comment URLs in the comments.json output

## Benefits
- **Improved Auditability**: Every comment processed by the copilot workflow is now tracked with a direct GitHub URL link
- **Better Traceability**: Commit messages clearly document which comments were fixed vs considered, with clickable links to each comment
- **Fallback Mechanism**: Gracefully handles missing URLs by matching comment IDs to URLs from the comments.json file
- **Structured Tracking**: Separates fixed comments from acknowledged/deferred/not-done comments for clear visibility of PR impact
