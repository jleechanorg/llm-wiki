## Raw Body
### Background
Prompt guidance currently defaults many reward and loot examples to gold wording, which pushes responses toward a single fantasy-currency style even when campaign tone/setting suggests different currency names.

### Goals
- Keep reward output structurally compatible with existing schema (`rewards_box.gold` remains canonical numeric field).
- Make prompt instructions explicitly require setting-appropriate currency labels in narrative/loot text.
- Apply the rule consistently across core prompt docs used in reward/planning/living-world/mechanics flows.

### Tenets
- Preserve existing state schema and field names for backward compatibility.
- Change instruction language, not runtime logic.
- Keep examples concrete while preventing hardcoded “gold-only” behavior.

### High-level description of changes
This PR updates prompt documentation to separate canonical numeric currency storage from narrative currency naming. In short: the model still writes numeric rewards into `gold`, but narrative-facing labels should match the campaign setting (for example crowns, credits, scrip, etc.) rather than always saying gold.
