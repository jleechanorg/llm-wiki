> [!NOTE]
> **Medium Risk**
> Changes adjust how the Playwright streaming test attributes SSE events to requests and how it gates PASS/FAIL, which can alter CI outcomes and could introduce false failures if the new idle/terminal checks misfire.
> 
> **Overview**
> Improves the BD-iwr Playwright streaming evidence test by making runs more deterministic and evidence more attributable: it now best-effort kills stale processes on the test port, waits for UI/SSE idleness between actions, and waits for `http_capture.jsonl` to quiesce before parsing.
> 
> Upgrades client-side capture to log all SSE events with precise timestamps, filter chunks by `request_id` and action start time, and enforce stream terminal integrity (fail on SSE `error` events; require `done` when the UI-idle wait succeeds). Adds a `timeout_provenance_*.json` artifact (including gunicorn `--timeout` detection) and expands the manifest/reporting with an HTTP request timeline and clearer decomposition output.
>
