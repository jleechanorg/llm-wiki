## Summary

This PR delivers a production-ready automation cron management system with three major improvements:

- **Non-destructive Installation**: Rewrite of `install_cron_entries.sh` to preserve custom cron entries while safely managing automation jobs via clearly delimited managed blocks
- **Portable Paths**: Convert all hardcoded paths (e.g., `/Users/jleechan`) to use `$HOME` variable, enabling cross-machine portability
- **Security Hardening**: Add crontab backups with secure permissions (`umask 077`, `chmod 600`), exclude backup directory from git

## Production Code Changes

| File | Before | After | Why |
|------|--------|-------|-----|
| `automation/install_cron_entries.sh` | Simple append-only | Non-destructive with managed blocks, backups, legacy cleanup | Preserve custom entries, enable safe updates |
| `automation/crontab.template` | Hardcoded `/Users/jleechan` paths | `$HOME`-based portable paths | Cross-machine compatibility |
| `automation/README.md` | Basic docs | Quick start guide + job table | Improved onboarding |
| `automation/.gitignore` | Missing | Added `.cron_backups/` | Prevent backup files in git |
| `self-hosted/scripts/crontab-example.txt` | Hardcoded username | `$HOME`-based portable paths | Cross-machine compatibility |
