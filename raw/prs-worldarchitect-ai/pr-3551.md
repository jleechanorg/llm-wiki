## Problem

Pending reviews were still being created despite multiple PRs (#3494, #3484) attempting to prevent them.

## Root Cause

The background monitor that deletes pending reviews every 5 seconds **never started** because:
- `GITHUB_ACTOR`/`AUTOMATION_USERNAME` environment variables were not set
- The monitor requires these to know which user's reviews to delete
- Without the monitor, reviews created during agent execution remained visible
- Cleanup functions only run AFTER agent execution, so there's a window where reviews are visible

## Why Previous Fixes Didn't Work

- ✅ Added cleanup functions (`_cleanup_pending_reviews`)
- ✅ Added warnings in agent prompts
- ❌ **But the background monitor (critical component) was skipped**

The monitor is the only component that deletes reviews **during** execution. Without it, reviews are visible until cleanup runs at the end.

## Solution

Auto-detect automation user from `gh api user --jq .login` if environment variables are not set. This ensures:
- ✅ Monitor always starts (unless gh CLI fails)
- ✅ Reviews are deleted every 5 seconds during execution
- ✅ No window where reviews are visible

## Changes

- Added auto-detection logic in `dispatch_agent_for_pr()`
- Falls back to `gh api user` if env vars not set
- Improved logging to show detected user
