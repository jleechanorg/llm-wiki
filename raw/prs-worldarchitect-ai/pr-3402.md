## Summary

Implements a new orchestrated dual-agent pair programming system that replaces the previous MCP Mail-based protocol with a flexible orchestration approach. Adds background monitoring, CLI selection, and improved error handling.

## Changes

### New Features

**1. Dual-Agent Pair Programming Orchestration ()**
- Launches coder and verifier agents in parallel via orchestration framework
- Supports flexible CLI selection (claude, codex, gemini, cursor) for both agents
- Default: Claude coder + Codex verifier (configurable via flags)
- Optional brainstorm phase via `--brainstorm` flag
- Creates Beads entries for state tracking
- Generates session directories with artifacts
- Configurable monitor intervals and max iterations

**2. Background Monitoring System ()**
- Monitors agent progress every 60 seconds (configurable)
- Checks tmux session status for both agents
- Tracks progress and detects stuck agents (3+ iterations without progress)
- Updates Beads status throughout session lifecycle
- Sends timeout warnings and session completion messages
- Max 10 iterations per agent (configurable)
- File-based fallback coordination when MCP Mail unavailable

**3. Rewritten  Command Documentation ()**
- Complete rewrite from MCP Mail protocol v3.1 to orchestration protocol v4.0
- Simplified usage: `/pair "Task description"` (no agent names required)
- Flexible CLI selection: `--coder-cli` and `--verifier-cli` flags
- Background monitor integration with configurable intervals
- Updated MCP Mail protocol documentation
- Beads state tracking integration
- Error handling and manual intervention guides
