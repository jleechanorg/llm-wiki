## Raw Body
### Motivation

- Address reviewer security and correctness feedback for the WorldAI Tools MCP proxy by removing unsafe implicit behaviors and making input validation explicit.  
- Prevent unnecessary Firestore reads during dry-run operations and ensure client errors map to the correct JSON-RPC error codes.  
- Make the deploy confirmation flow explicit so the `CONFIRMED` bypass is only allowed when an unsafe bypass flag is intentionally enabled.

### Description

- Short-circuited `dry_run` in `admin_copy_campaign_user_to_user` so the handler returns before any subcollection `stream()` calls to avoid reads when `dry_run=True`. (file: `mvp_site/worldai_tools_mcp_proxy.py`)  
- Mapped invalid `from_timestamp`/`to_timestamp` parsing in `_tool_admin_download_campaign_entries` to a JSON-RPC `-32602` (invalid params) error instead of allowing parse exceptions to bubble as internal errors. (file: `mvp_site/worldai_tools_mcp_proxy.py`)  
- Gated the literal `CONFIRMED` deploy-confirm bypass behind an explicit `allow_unsafe_local_tool_bypass` flag by: adding `allow_unsafe_bypass` parameter to `_validate_deploy_confirm_token`, updating the error message to require `DEPLOY-{target}-{YYYYMMDD-HHMMSS}` when bypass is not enabled, and passing the proxy instance bypass flag at the deploy call site. (file: `mvp_site/worldai_tools_mcp_proxy.py`)  
- Added unit tests to cover the tightened behaviors: reject unscoped `CONFIRMED`, verify dry-run avoids subcollection reads, and assert timestamp parse errors produce `-32602`. (file: `mvp_site/tests/test_worldai_tools_mcp_proxy.py`)  
- Small refactor: replaced fragile `ast`/string JSON attempts with robust handling where applicable and ensured parsing/validation paths are explicit (see helper usages in `worldai_tools_mcp_proxy.py`).
