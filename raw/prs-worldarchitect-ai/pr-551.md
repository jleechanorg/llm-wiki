## Summary
Fixes multiple critical issues with planning blocks in deep think mode, including Firestore persistence of empty narratives and simplified CSS display.

## Critical Fixes
- 🚨 **Empty Narrative Fix**: AI responses with only structured_fields (no narrative text) now properly save to Firestore
- 🚨 **Think Command Fix**: Think commands now generate proper narrative text alongside planning blocks
- 🎨 **UI Simplification**: Deep think mode now uses same CSS as normal planning blocks

## Problem
- Planning blocks in deep think mode were not being saved to Firestore when narrative was empty
- Think commands were generating only structured_fields without narrative text
- Complex CSS for deep think mode was creating excessive vertical spacing
- Backend incorrectly used array format for pros/cons (legacy from prompt template)
- Frontend JavaScript was ignoring the `analysis` field and only showing basic choice buttons

## Solution
- **Firestore Fix**: Added placeholder text "[Internal thoughts and analysis - see planning block]" for empty narratives
- **Think Command Fix**: Enhanced prompts to ensure narrative text is always generated with think commands
- **CSS Simplification**: Removed 150+ lines of special CSS, now uses standard planning block styles
- **Backend Format Change**: Converted pros/cons from arrays to comma-separated strings
- **Frontend Enhancement**: Added analysis field detection and multi-line text rendering
- **Code Cleanup**: Removed all verbose logging and temporary comments
- **Test Coverage**: Added unit test to protect against empty narrative regression

## Changes
- **Firestore Service**: Empty narrative handling with placeholder text
- **Gemini Service**: Improved think command detection using regex patterns
- **Frontend (app.js)**: Simplified to render multi-line plain text buttons
- **Styling (style.css)**: Removed all deep think CSS classes and styles
- **CLAUDE.md**: Added "No Temporary Comments" rule
- **Tests**: Added test_empty_narr
