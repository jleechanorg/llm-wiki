## Problem
Current dual-pass system runs sequentially:
- Pass 1: Generate narrative (2-5s)
- Pass 2: Inject missing entities (2-5s)
- **Total wait: 4-10 seconds**

## Solution
Run Pass 2 in parallel while user reads Pass 1:
- Show Pass 1 immediately
- Enhance with Pass 2 in background
- **Perceived wait: 2-5 seconds (50% reduction)**

## Implementation
- **Backend**: New `/enhance-entities` endpoint for async Pass 2
- **Frontend**: Parallel processing with smooth UI updates
- **UI**: Subtle indicators show enhancement progress
- **Fallback**: Graceful degradation if enhancement fails
