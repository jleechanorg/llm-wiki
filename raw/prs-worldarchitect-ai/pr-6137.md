## Background
The campaign wizard correctly prompts LLMs to generate a campaign. However, generating an entire initial story from scratch is slow. We want to provide an optional fast-path that seeds the campaign state from precomputed constants when the user selects the "Dragon Knight" template with canonical settings, bypassing the initial LLM call entirely.

## Goals
- Pre-generate a full "Dragon Knight" campaign in backend constants.
- Bypass `get_initial_story` if the creation payload explicitly matches the Dragon Knight configuration hashes.
- Return the exact same shape of structured fields and state updates as a real LLM generation so the client logic remains entirely unware of the optimization.

## Tenets
- Accuracy Over Speed: If a user specifies custom input or the inputs drift from the canonical hashes, seamlessly fall back to a full LLM invocation.
- Secure Mutation: Avoid shared references between multiple campaigns generated from the pre-generated template.

## High-level description
Adds an **optional pre-generated “Dragon Knight” campaign template fast-path**: when the incoming `create_campaign_unified` request matches canonical defaults (via prompt SHA-256 + selected prompts + custom options), the backend returns a bundled opening story/structured fields and applies precomputed state updates via `constants.py` instead of calling `get_initial_story`.

Introduces `campaign_template_dragon_knight.py` + a versioned canonical description file, new Dragon Knight constants payloads in `constants.py`, and a new authenticated endpoint `GET /api/campaign_templates/dragon_knight/description` used by the campaign wizard to load the long default description lazily (replacing the giant inline JS string and setting the default title).

Bugbot feedback was incorporated, properly deep copying templates to avoid referencing mutable globals across requests, fixing dead paths, and adjusting frontend promises to await constants loading before submitting data.
