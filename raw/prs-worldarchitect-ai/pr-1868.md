## Summary
This PR enhances the exportcommands.py script to export additional critical configuration files that were previously missing from the claude-commands repository.

## Changes Made

### New Export Methods
1. **`_export_settings()`** - Exports `.claude/settings.json` configuration file
2. **`_export_claude_scripts()`** - Exports `.claude/scripts/` directory (learning/memory management Python scripts)
3. **`_get_scripts_from_settings()`** - Parses settings.json to auto-detect script references using regex

### Enhanced Export Coverage
- ⚙️ **settings.json**: Complete Claude Code configuration now exported
- 📜 **.claude/scripts/**: 10 Python files for learning/memory management
- 🔔 **Auto-detected scripts**: Automatically finds and exports scripts referenced in settings.json (e.g., slack_notify.sh)
- 🚀 **Existing scripts**: Maintains current export of root scripts/ allowlist (20 files)

### Technical Details
- Updated `_copy_to_repository()` to handle new directory mappings:
  - `claude_scripts` → `.claude/scripts/`
  - `settings_json` → `.claude/settings.json`
- Regex pattern for script detection: `r'scripts/([a-zA-Z0-9_\-]+\.(sh|py))'`
- Avoids duplicate exports between MCP helper scripts and settings-referenced scripts
