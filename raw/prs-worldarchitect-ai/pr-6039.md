## Background

The `/copilot` command was making ~13 API calls per run (10-comment PR). Step 8.5 added 4-5 extra calls to unconditionally re-fetch all 4 comment sources after posting the consolidated reply, then conditionally re-post if new comments were found. This PR removes Step 8.5 and adds explicit policy documentation to prevent re-introduction.

## Goals

1. Reduce API calls per /copilot run by ~38%
2. Document thread-resolution policy explicitly in copilot.md (Key Rule 8)
3. Document thread-resolution forbidden endpoints in _copilot_reference.md

## Tenets

- Eventual consistency over strict consistency (late comments caught on next /copilot invocation)
- Policy documented explicitly, not just removed silently

## High-level description of changes

Removed Step 8.5 from copilot.md (the unconditional delta recheck). Added Key Rule 8 explicitly forbidding API-based thread resolution. Added Thread Resolution Policy section and api-reduction-design.md documenting the change rationale with a before/after API call table.
