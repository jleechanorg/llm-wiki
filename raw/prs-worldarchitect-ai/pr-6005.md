## Production Code Changes
**Before**: `worldai` MCP in `~/.claude.json` used `python3 -m mvp_site.worldai_mcp_stdio` with `PYTHONPATH` pointing to `worktree_livingw3` (a temporary worktree directory).

**Now**: `install_mcp.sh` creates a stable `.venv` in `worldarchitect.ai/` via `uv` and registers the server as `uv run --no-sync --project worldarchitect.ai python -m mvp_site.worldai_mcp_stdio`.

**Why**: Worktree directories get cleaned up; the server needs a stable, non-worktree path.

## Test Changes
No test changes — smoke test is built into `install_mcp.sh` step 3 (sends `initialize` JSON-RPC call, validates `serverInfo` in response).

## Known Limitations
- `get_campaigns_list` and other proxied tools require the upstream worldai server running at port 8081
- Admin/diag tools that read Firebase directly work without upstream
