## What Bugs Are We Fixing?

Three independent bugs in `stream_story_with_game_state()` (the SSE `/interaction/stream` path) that were never ported from the non-streaming path in `world_logic.process_action_unified()`:

---

### Bug 1: Living World Tracking Never Updated (root cause confirmed via Firestore)

**Symptom**: Living world events not appearing for campaigns played via the streaming SSE endpoint.

**Root cause**: `stream_story_with_game_state()` never called `_increment_turn_counter()` or `_maybe_update_living_world_tracking()`. `last_living_world_turn` stayed at 0 forever.

**Confirmed via Firestore**: Visenya V6 campaign (`JkKR510zImWiFiVHMGGV`) had `turn_number=40` but `last_living_world_turn=0` — living world tracking never advanced on the streaming path.

**Fix**: Added `_increment_turn_counter()` + `_maybe_update_living_world_tracking()` calls in the "done" event handler, mirroring `world_logic.py`.

**Debug logging added**: `_warn_if_living_world_missing()` emits `WARNING` with `🌍 LIVING_WORLD_MISSING` tag when world_events absent, or `🌍 LIVING_WORLD_SCHEMA` when wrong schema (missing `background_events` key).

---

### Bug 2: `should_freeze_time` Never Passed to Turn Counter (streaming path)

**Symptom**: Turn counter incorrectly advancing during think mode, level-up decisions, and character creation in streaming path.

**Root cause**: `_increment_turn_counter()` was called without `should_freeze_time=` parameter, defaulting to `False`. The non-streaming path computes this from `is_think_mode`, `is_freeze_time_choice`, `is_character_creation_mode`, `is_level_up_mode`.

**Fix**: Compute `should_freeze_time` from `agent_mode_used` and pass it to `_increment_turn_counter()`, matching `world_logic.py:5240-5245`.

---

### Bug 3: Revert — Provably Fair Seed Override Format

Reverts the cache-safe seed text format change from `ec02eb50ab` back to the original verbose format. The concise `SERVER_SEED=...` format from that commit caused issues.
