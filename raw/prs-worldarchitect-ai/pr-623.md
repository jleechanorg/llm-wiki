## Summary
Adds critical learning rule to prevent "website looks same" confusion pattern.

## Problem Addressed
Common misconception: Switching git branches → Expecting immediate website changes

## Root Cause Analysis
- Many PRs contain non-UI changes (tooling, CI, scripts, backend)
- Branch changes require deployment OR local server to be visible
- Production websites typically serve main branch only
- Users often view production while expecting feature branch changes

## Solution: 5-Step Protocol
1. ✅ Check PR description - what type of changes? (tooling vs UI)
2. ✅ Ask: "What URL are you viewing?" (local vs production)  
3. ✅ Verify: User-facing changes or developer tooling/CI/scripts?
4. ✅ For UI changes: Hard refresh (Ctrl+F5) + check local development server
5. ✅ Explain: Non-UI changes won't change website appearance
