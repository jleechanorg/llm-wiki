## Part 1: Fix Marker Format Mismatch (Original PR Scope)

### Root Cause
The marker prefix constants were using old format:
- `FIX_COMMENT_RUN_MARKER_PREFIX = "<!-- fix-comment-automation-run:"`
- `FIXPR_MARKER_PREFIX = "<!-- fixpr-automation-run:"`

But `build_automation_marker()` generates new format:
- `<!-- fix-comment-run-automation-commit:agent:sha-->`
- `<!-- fixpr-run-automation-commit:agent:sha-->`

This mismatch caused `_count_workflow_comments()` to fail matching new markers, breaking safety limits.

### Changes
1. **Updated Marker Prefix Constants (codex_config.py)**
   - Fixed FIX_COMMENT_RUN_MARKER_PREFIX to match new format
   - Fixed FIXPR_MARKER_PREFIX to match new format

2. **Fixed Missing agent_cli Parameter (jleechanorg_pr_monitor.py)**
   - Added `agent_cli` parameter to `_post_fixpr_queued()` signature
   - Updated call site to pass `agent_cli` parameter

3. **Added Helper Functions (codex_config.py)**
   - `build_automation_marker(workflow, agent, commit_sha)` - Generate markers
   - `parse_automation_marker(marker)` - Parse markers (supports legacy format)

## Part 2: Revise Automation Safety Limits (NEW)

### New Limit Structure
- **Global PR limit (`pr_limit`)**: 50 total attempts (changed from 10)
- **Per-workflow limit**: 10 attempts per workflow (unchanged)
- **CRITICAL CHANGE**: Now counts ALL attempts (success + failure), not just failures

This enables iterative improvements by allowing multiple successful runs while still preventing infinite loops.
