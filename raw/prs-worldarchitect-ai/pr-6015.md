## Production Code Changes

**`mvp_site/agents.py` — CharacterCreationAgent**
- Before: `del turn_number, llm_requested_sections` discarded `turn_number`; `advances_time=False` blocked the global always-on LW injection guard → LW never loaded during character creation
- Now: `turn_number` preserved; explicit LW append after `build_from_order` when `turn_number >= 1`, bypassing the `advances_time` gate
- Why: Game world must continue updating even while player is in character creation flow

**`mvp_site/agents.py` — StoryModeAgent**
- Before: `REQUIRED_PROMPT_ORDER` included `PROMPT_TYPE_LIVING_WORLD` between `dnd_srd` and narrative/mechanics — LW competed early against the 3009-line game_state block
- Now: LW removed from `REQUIRED_PROMPT_ORDER`; always-on injection path in `agent_prompts.py:1143-1151` fires it AFTER narrative/mechanics
- Why: LW as the final mandate has higher salience; game_state (3.3× larger) no longer drowns it out

**`mvp_site/streaming_orchestrator.py`**
- Before: `_warn_if_living_world_missing()` only checked binary presence of `world_events` — empty `background_events: []` or 1 partial event accepted silently
- Now: `_count_compliant_lw_events()` helper counts events with all 7 required fields; emits `LIVING_WORLD_COMPLIANCE` warning when count < 2
- Why: Surface partial LW compliance failures so prompt engineering can target the exact deficiency
