> [!NOTE]
> Adds a new end-to-end fix-comment automation path to handle PR review feedback directly via orchestration.
> 
> - **New workflow:** `--fix-comment` mode dispatches agents using `_build_fix_comment_prompt_body` and posts a queued notice, then a review-request comment with hidden `<!-- fix-comment-automation-commit:...-->` marker; includes a background watcher (`--fix-comment-watch`) to detect automation commits and trigger review requests
> - **Threaded replies:** Prompt requires 100% replies to review comments using `gh api .../pulls/{pr}/comments/{id}/replies` and clarifies top-level issue comments via `gh pr comment`
> - **Safety/gating:** Marker extraction/checks (`_extract_fix_comment_marker`, `_has_fix_comment_comment_for_commit`) and limits updated to count both Codex and fix-comment markers
> - **Orchestration:** New `dispatch_agent_for_pr_with_task` helper; `ensure_base_clone` more robust for `main`; commit message regex updated to allow `[fixpr ...-automation-commit]`
> - **CLI/model support:** Adds `--fix-comment`, `--fix-comment-watch`, and `--model` flags; model is validated and propagated through `TaskDispatcher`, agent specs, and Claude command template (`--model {model}`)
> - **Tests/docs/version:** New/updated tests for markers, watcher flow, prompts, model plumbing, and selectors; added design docs; bumped `automation` to 0.2.34 and `orchestration` to 0.1.20
