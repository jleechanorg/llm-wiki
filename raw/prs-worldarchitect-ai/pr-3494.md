## Problem

Agents were creating pending reviews despite warnings, which blocks automation and clutters PRs. PR #3484 added cleanup before dispatch, but agents can still create pending reviews during execution.

## Solution

Added comprehensive multi-layer protection to prevent pending review creation:

### 1. Pre-flight Tool Verification
- Agents must check available MCP tools before starting work
- Must skip/ignore forbidden tools (`create_pending_pull_request_review`, `add_comment_to_pending_review`) if present
- Instructions to pretend forbidden tools don't exist

### 2. Background Monitor
- Real-time monitoring that polls every 5 seconds for pending reviews
- Immediately deletes any pending reviews created during agent execution
- Runs in background until agent completes

### 3. Post-Completion Cleanup
- Cleanup runs after agent completion (in addition to pre-dispatch cleanup)
- Catches any pending reviews that slipped through despite warnings

### 4. Strengthened Warnings
- Multiple layers of warnings with strong language
- Absolute prohibition language ("FORBIDDEN", "DO NOT USE", "IMMEDIATE TERMINATION")
- Clear instructions on correct methods to use instead

## Changes

- Added pre-flight tool verification instructions to both fixpr and fix-comment workflows
- Added background monitor to immediately delete pending reviews during execution
- Added cleanup after agent completion (complements existing pre-dispatch cleanup)
- Strengthened warnings with absolute prohibition language
- Fixed duplicate monitor startup code
