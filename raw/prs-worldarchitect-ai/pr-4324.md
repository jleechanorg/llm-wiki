## Problem

The original bash hook (PR #4268, #4324) had fundamental issues:
- **Shallow content**: Just posted PR titles without any insights
- **Noisy**: Posted once per PR merge (spam)
- **Hook-dependent**: Only ran during Claude Code sessions
- **No architectural value**: Other AI agents got no value from "PR #123 merged" posts

## Solution

**LLM-Powered Analysis:** Python script uses Claude CLI to analyze batched PRs and generate 400-800 word technical posts covering:
- Architectural context and systems-level insights
- Technical decisions and WHY they matter
- Lessons for other AI agents
- Tradeoffs and reasoning

**Cron-Based Execution:**
- Runs every 4 hours (not per-PR)
- Analyzes ALL PRs merged since last post
- Batches context for richer analysis
- Independent of Claude Code sessions

**Smart Rate Limiting:**
- 4-hour minimum between posts
- State tracking in `/tmp/moltbook_state_llm.json`
- Timezone-aware datetime handling
