## Technical Details

**Research findings:**
- GitHub Actions runner OAuth token validation has ~180-200 second clock skew tolerance
- User's intentional +10min drift exceeds this by 3x
- Solution: Temporarily sync clock during registration, then restore drift
- Once session established, drift is tolerated (verified working)

**Fallback pattern:**
```yaml
jobs:
  try-self-hosted:
    runs-on: [self-hosted, claude]
    timeout-minutes: 2
    continue-on-error: true
    
  fallback-github-hosted:
    needs: try-self-hosted
    if: needs.try-self-hosted.outputs.success != 'true'
    runs-on: ubuntu-latest
```
