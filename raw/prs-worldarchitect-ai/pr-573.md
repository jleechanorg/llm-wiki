## Problem

The `integrate.sh` script fails with "not fully merged" error when trying to delete branches that were merged via GitHub PR. This happens because `git branch -d` only checks if the branch is merged into the **local** main branch, not considering:
- Branches merged on GitHub (remote)
- Branches with merged PRs

## Solution

Enhanced the branch deletion logic with three sequential checks:

1. **Local merge check** (existing): Is branch merged into local main?
2. **Remote merge check** (new): Is branch merged into origin/main?
3. **PR merge check** (new): Does branch have a merged PR on GitHub?

If any condition is true, the branch is safely deleted using `git branch -D`.

## Example

**Before**:
```bash
6. Deleting clean branch 'roadmap2342'...
error: the branch 'roadmap2342' is not fully merged.
```

**After**:
```bash
6. Checking if branch 'roadmap2342' can be safely deleted...
   ✓ Branch is safe to delete (has merged PR)
   Deleting branch 'roadmap2342'...
✅ Deleted clean branch 'roadmap2342'
```

## Benefits

- ✅ No more false "not fully merged" errors
- ✅ Properly handles GitHub PR workflow
- ✅ Clear feedback about deletion reason
- ✅ Graceful fallback with manual instructions
- ✅ Only force-deletes after safety verification
