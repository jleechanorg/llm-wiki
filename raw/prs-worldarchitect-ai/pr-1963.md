## Summary
Adds `--no-stream-logs` flag to Cloud Build submissions to prevent deployment failures when service accounts lack log viewer permissions.

## Problem
GitHub Actions deployments were failing with permission errors when trying to stream Cloud Build logs, even though the builds succeeded.

## Solution
- Add `--no-stream-logs` flag to `deploy_common.sh`
- Builds still complete successfully and logs are saved to Cloud Logging
- Logs remain accessible via Cloud Console and gcloud CLI

## Impact
- ✅ Deployments complete without permission errors
- ✅ All logs still captured in Cloud Logging
- ✅ No loss of debugging capability

- Verified logs are available in Cloud Console
- Build completed successfully without streaming errors
