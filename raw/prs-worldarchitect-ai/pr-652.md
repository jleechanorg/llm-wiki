## Summary

Fixes the `schedule_branch_work.sh` script to properly source `~/.bashrc` so that the `claude` command is available when `ccschedule` runs in a non-interactive context.

**Problem**: The script was failing with "No such file or directory" error because `ccschedule` runs in a non-interactive shell context where `~/.bashrc` isn't automatically sourced, making the `claude` command unavailable.

**Solution**: Added `source ~/.bashrc` at the beginning of the script to ensure PATH and environment variables are loaded.

## Changes Made

- **Added bashrc sourcing**: `source ~/.bashrc` to load PATH containing `~/.claude/local`
- **Added explanatory comments**: Document why bashrc sourcing is necessary for non-interactive contexts
- **Simplified implementation**: Removed complex error handling in favor of direct sourcing

- Verified script now successfully schedules and executes claude commands

- Tested with various scheduling times (immediate and future)

## Technical Details

**Root Cause**: Non-interactive shells don't automatically source `~/.bashrc`, so the PATH export `export PATH="$HOME/.cargo/bin:$HOME/.claude/local:$PATH"` wasn't available.

**Fix**: Direct `source ~/.bashrc` ensures all PATH modifications and environment variables are loaded before `ccschedule` attempts to execute the claude command.
