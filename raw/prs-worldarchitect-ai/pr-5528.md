## Raw Body
## Summary

Restore original behavior: `workflow_run` events trigger **Mock tests** only, not Real E2E tests.

## Problem

After preview deploy completes, the workflow was automatically running **Real E2E tests** instead of Mock tests:

```bash
# Recent workflow runs
{"event":"workflow_run"}  # ← Running Real E2E automatically\!
{"event":"workflow_run"}  # ← Should be Mock tests
{"event":"workflow_run"}  # ← User didn't request /smoke
```

## Root Cause

**Incorrect job conditions:**
- ❌ Real E2E job (line 196): Included `workflow_run` in condition
- ❌ Mock test jobs (lines 68, 138): Did NOT include `workflow_run`

This inverted the intended behavior.

## Fix

**Changed 3 job conditions:**

### 1. Mock Tests (Self-Hosted) - Line 68
```diff
-    if: github.event_name == 'pull_request' || github.event_name == 'push'
+    if: github.event_name == 'pull_request' || github.event_name == 'push' || github.event_name == 'workflow_run'
```

### 2. Mock Tests (Fallback) - Line 138
```diff
-    if: (github.event_name == 'pull_request' || github.event_name == 'push') && ...
+    if: (github.event_name == 'pull_request' || github.event_name == 'push' || github.event_name == 'workflow_run') && ...
```

### 3. Real E2E Tests - Line 196
```diff
-    if: ... || github.event_name == 'workflow_dispatch' || (github.event_name == 'workflow_run' && github.event.workflow_run.conclusion == 'success')
+    if: ... || github.event_name == 'workflow_dispatch'
```

Removed `workflow_run` from Real E2E condition.

## Behavior After Fix

| Trigger | Mock Tests | Real E2E Tests |
|---------|------------|----------------|
| PR push | ✅ Yes | ❌ No |
| Push to main/dev | ✅ Yes | ❌ No |
| Preview deploy complete (`workflow_run`) | ✅ Yes | ❌ No |
| `/smoke` command | ❌ No | ✅ Yes |
| Manual `workflow_dispatch` | ❌ No | ✅ Yes |
