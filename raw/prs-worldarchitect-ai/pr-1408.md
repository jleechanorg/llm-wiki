## Summary

Adds a new `/localexportcommands` slash command that exports the project's .claude folder structure to the local ~/.claude directory, making commands and configurations available system-wide.

## Features

🎯 **Complete Structure Copy**:
- Copies all directories: commands/, hooks/, agents/, settings.json, etc.
- Preserves directory hierarchy and relationships
- Handles both files and directories appropriately

🔒 **Safety & Backup**:
- Creates timestamped backups of existing ~/.claude before overwriting
- Validates source directory exists before starting
- Individual component copying (partial failure won't break everything)

⚙️ **Permission Management**:
- Automatically sets executable permissions on all .sh files in hooks/
- Sets executable permissions on scripts/ directory files
- Preserves file permissions for other components

📊 **Comprehensive Feedback**:
- Shows what's being copied at each step
- Provides counts of exported items (commands, agents, hooks)
- Includes validation checklist after completion

## Usage

```bash
/localexportcommands
```

## Benefits

- **System-Wide Availability**: Commands work across all Claude Code projects
- **Consistent Environment**: Same tools and configurations everywhere  
- **Easy Updates**: Re-run to sync latest project changes
- **Safe Operation**: Creates backups before overwriting
- **Team Standardization**: Share Claude Code configurations easily
