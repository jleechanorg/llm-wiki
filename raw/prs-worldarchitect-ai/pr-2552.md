The update_campaign function was not correctly handling dot-notation paths like 'game_state.custom_campaign_state.arc_milestones.wedding_tour'. This caused Firestore to potentially create literal field names instead of properly nested document structures.

Changes:
- Modified update_campaign() to detect dot-notation keys and expand them using _expand_dot_notation() before writing to Firestore
- When dot-notation is detected, uses set(merge=True) instead of update() for more reliable nested updates
- Added comprehensive tests for _expand_dot_notation() function
- Added integration tests for update_campaign() with dot-notation paths
