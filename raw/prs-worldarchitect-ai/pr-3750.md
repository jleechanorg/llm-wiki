## Description
Resolves the 'LLM returned empty narrative' error observed in defaultWorld campaign creation smoke tests.

## Root Cause
The 'ACTION RESOLUTION PROTOCOL (UNIVERSAL)' section in `master_directive.md` incorrectly mandated the inclusion of the `action_resolution` JSON field for **ALL** modes, including Character Creation and God Mode. This conflicted with the capabilities of the LLM in those modes (specifically Cerebras/Llama-3.3-70b), causing it to fail to generate the required narrative text field.

## Fix
Updated `master_directive.md` to explicitly exempt **Character Creation** and **God Mode** from the Action Resolution Protocol auditing requirement.

## Verification
- **Red State**: Reproduced the failure on a clean branch using `llama-3.3-70b`.
- **Green State**: Verified fix by running local smoke tests with `MCP_TEST_PROVIDERS=cerebras`. All tests passed, including `create_campaign` with `defaultWorld`.
