## Root Cause Analysis

The auth-cli.mjs (line 111) loads the API key from:
```javascript
apiKey: process.env[`${ACTIVE_PROJECT.envPrefix}_API_KEY`]
```

Where `ACTIVE_PROJECT.envPrefix` is `VITE_AI_UNIVERSE_FIREBASE` (line 34).

This means the expected env var is `VITE_AI_UNIVERSE_FIREBASE_API_KEY`, but the documentation used `FIREBASE_API_KEY`. When the token needed to be refreshed, the refresh call at line 257 used an invalid/placeholder API key, causing Firebase's "auth/api-key-not-valid" error.
