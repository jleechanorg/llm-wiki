## Key Enhancements

### 🚨  Automatic CI Discrepancy Detection
- **NEW Step 4**: Detects when GitHub CI fails but `./run_tests.sh` passes locally
- Identifies common causes: Python version differences, missing env vars, race conditions, file system case sensitivity
- Automatically triggers `/redgreen` workflow for environment-specific issues

### 🔄  Integrated Red-Green-Refactor Workflow
- **RED Phase**: Creates failing tests that reproduce GitHub CI failures locally
- **GREEN Phase**: Fixes code to work in both local and CI environments  
- **REFACTOR Phase**: Cleans up solution while maintaining test coverage
- Prevents push/fail/fix cycles by addressing root cause locally

### 🧹  Conditional Lint Fixes
- **Selective Targeting**: Only applies lint fixes to Python files being staged for commit
- **Safe Automation**: Never modifies unstaged files - only touches files developer is already changing
- **Smart Re-staging**: Automatically re-stages files modified by lint fixes
- **Configurable**: Respects `SKIP_LINT=true` environment variable
