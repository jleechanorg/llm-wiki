## Summary

Fixes the root cause of LLM level-up inconsistency where the AI was quoting wrong XP thresholds.

### Root Cause Analysis

**Evidence:** `docs/debugging/Alexiel Assiah V2 (2).txt` lines 1395-1408

The LLM was misreading the XP table:
- User asked: "How much XP for level 8?" (status showed `XP: 33,025/34,000`)
- LLM responded: "To reach the eighth tier...you require 48,000 XP" ❌
- Correct answer: 34,000 XP (48,000 is for level 9)

**Why this happened:**
The table header said "Total XP Required" which the LLM interpreted as "XP you have AT that level" rather than "XP to REACH that level". So when asked about level 8, it looked at the row AFTER (level 9 = 48,000) instead of level 8's own row (34,000).

### Why Previous PRs Didn't Fix It

- PR #2428 and #2068 added correct backend validation and XP table
- BUT the prompt language was ambiguous - the table header didn't prevent the misreading
- The LLM was still being asked to quote thresholds in narrative, and it quoted the wrong row

### Fix

1. **Renamed table header** from "Total XP Required" → "XP to REACH"
2. **Added explicit warning** with ❌ WRONG / ✅ RIGHT examples
3. **Added lookup table** for common level transitions (7→8, 8→9, 9→10)
4. **Added step-by-step protocol** for answering XP questions
5. **Reinforced in game_state_instruction.md** with the specific common mistake

## Test Plan

- [ ] The backend XP calculations remain correct (verified manually)
- [ ] New campaigns correctly show "XP: X/34,000" for level 7 characters
- [ ] LLM correctly answers "how much XP for level 8?" with 34,000 (not 48,000)
