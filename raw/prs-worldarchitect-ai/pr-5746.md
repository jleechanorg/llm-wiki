## Summary
- Hardened self-hosted runner control plane in `test.yml` for reliable infra-classification, retries, and health scoring.
- Added/extended writable tool-cache bootstrap coverage so `actions/setup-python` on constrained self-hosted runners uses writable directories instead of `/Users/runner`.
- Fixed preflight JSON parsing and interpreter selection to avoid infra-classify loss on hosts without a legacy `python` binary.
- Corrected runner health scoring for gated/skipped paths so quarantine/missing-step gates no longer accumulate false negative score debt.
- Tightened OOM classification in the installer classifier to reduce over-broad matches while still catching OOM exit and kill patterns.

## Production Code Changes (before → now → why)
1. Test workflow control plane (`.github/workflows/test.yml`)
- Before: install/test failures and preflight failures could be mis-scored as infra debt, and skip paths lacked explicit immunity logic.
- Now: `Update runner health score` gives `DELTA=0` on quarantine/skipped-gate outcomes, and still attributes real install/test infra failures to typed deltas.
- Why: Keeps health-score state accurate and prevents permanent runner degradation from intended guard rails.

2. Infra classification (`scripts/ci/classify_infra_failure.sh`)
- Before: broad memory-matching pattern included ambiguous token-level heuristics.
- Now: uses explicit memory/OOM indicators and exit-code patterns (including `exit code 137`) with tighter, command-context-sensitive matching.
- Why: Reduces false positives that wrongly trigger infra retries/quarantine.

3. Tool-cache bootstrap and preflight path hardening (`scripts/ci/setup-runner-tool-cache.sh`, `.github/workflows/test.yml`, `.github/workflows/coverage.yml`, `.github/workflows/doc-size-check.yml`, `.github/workflows/hook-tests.yml`, `.github/workflows/mcp-smoke-tests.yml`)
- Before: only a subset of self-hosted workflows had deterministic tool-cache initialization before `actions/setup-python`.
