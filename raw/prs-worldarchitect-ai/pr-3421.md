## Raw Body
Add --force-default-port flag to control port selection behavior:
- Default behavior: Pick random ports in range 8081-8181 without closing existing processes
- --force-default-port: Kill existing processes and use default ports (8080, 3000)

Changes:
1. Added find_random_port() function to server-utils.sh for random port selection
2. Added --force-default-port flag parsing to run_local_server.sh
3. Skip aggressive port cleanup when flag is not set
4. Use random port selection for Flask, React, and MCP servers by default
5. Updated usage documentation to reflect new behavior

This prevents conflicts with existing running servers while still allowing developers to force default ports when needed.
