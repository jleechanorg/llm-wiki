## Summary
Addresses the campaign issues reported in dev where Living World updates were persisted but disappeared after reload, and where semantic routing occasionally misfired into CharacterCreationAgent for normal narrative inputs.

## Changes
1. Living World reload persistence fallback
- Added `inject_persisted_living_world_fallback(...)` in `mvp_site/world_logic.py`.
- Wired fallback into campaign load path in `mvp_site/main.py`.
- Ensures latest Gemini story entry receives persisted `game_state.world_events` payload for debug rendering when per-turn payload is absent.

2. Persist scene numbers in Firestore story entries
- In `mvp_site/world_logic.py`, compute `sequence_id` and `user_scene_number` before persistence and store them in `structured_fields`.
- These fields are now written to AI story docs in Firestore.

3. Persist scene-labeled narrative text
- Added `_prefix_scene_number_in_narrative(...)` and apply before AI story write.
- Stored AI narrative now includes `Scene #N:` prefix for durable, user-visible scene context in persisted entries.

4. Prevent CharacterCreationAgent misfires
- Tightened semantic routing in `mvp_site/agents.py` for `MODE_CHARACTER_CREATION`:
  - Route to CharacterCreationAgent only if state is active OR input explicitly requests character-building actions (level-up/creation/respec/ASI/etc).
  - Otherwise suppress and fall back to StoryModeAgent.

5. Tests
- Updated `mvp_site/tests/test_agents.py` for misfire suppression + explicit-request routing.
- Added tests in `mvp_site/tests/test_world_time_progression.py` for scene-number narrative prefix helper.
- Existing reload fallback tests retained.
