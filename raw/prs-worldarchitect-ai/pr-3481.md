## Summary
Fixes the FixPR workflow issue where pending reviews were left behind by automation agents, causing PR clutter. This PR adds automatic cleanup of pending reviews and strengthens guidance to prevent future occurrences.

## Changes
- **Added `_cleanup_pending_reviews()` method**: Automatically deletes any pending reviews from the automation user before processing fix-comment and fixpr workflows
- **Enhanced task prompts**: Added explicit instructions to agents to use the correct GitHub API endpoint (`/comments/{id}/replies`) that creates direct replies without starting pending reviews
- **Safety guidance**: Added warnings against using MCP tools and endpoints that create pending reviews (`create_pending_pull_request_review`, `add_comment_to_pending_review`, etc.)
- **Version bump**: Updated package version to `0.2.46`

## Benefits
- **Cleaner PRs**: No more leftover pending reviews cluttering pull requests
- **Better agent guidance**: Clear instructions prevent agents from accidentally creating pending reviews
- **Automatic cleanup**: Proactive cleanup ensures clean state before each workflow run

## Technical Details
The cleanup method:
1. Fetches all reviews for the PR using GitHub API
2. Identifies pending reviews from the automation user
3. Deletes each pending review via DELETE API call
4. Logs cleanup actions for monitoring

Task prompts now explicitly instruct agents to:
- Use `gh api /repos/{owner}/{repo}/pulls/{pr_number}/comments/{comment_id}/replies` for replies
- Avoid MCP tools that create pending reviews
- Never use review endpoints that require submission
