## Raw Body
- Removed default_limits from Flask-Limiter configuration
- Static files and frontend routes are now completely unrate-limited
- Added explicit rate limits only to API routes that need protection
- This prevents 429 errors when browsers load multiple CSS/JS files in parallel

Fixes CSS loading issues where static files were hitting rate limits and returning HTML error pages instead of CSS/JS files.

Root cause: The aggressive default rate limits (10 per minute) were blocking parallel browser requests for static assets, causing 429 errors. The server would then return HTML error pages which the browser tried to interpret as CSS/JS, leading to MIME type errors.

Solution: Remove default limits and only apply rate limiting to specific API endpoints that need protection against abuse.
