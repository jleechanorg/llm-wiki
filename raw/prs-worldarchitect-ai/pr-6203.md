## Summary

PR #6102 deleted the `_Sentinel` class during rebase conflict resolution. This restores it along with its supporting utilities.

## Production Code Changes

### mvp_site/firestore_service.py

- **Added**: `_Sentinel` class + `_restore_sentinel()` + `_DELETED` singleton (after `DELETE_TOKEN`). The `_Sentinel` class preserves identity through `copy.deepcopy` and `pickle`, which is required for `_strip_deleted` and `_deep_merge` to work correctly.
- **Added**: `_strip_deleted()` function - recursively removes `_DELETED` sentinel values from dicts and lists.
- **Added**: `_deep_merge()` function - recursively merges source into target, treating `_DELETED` as explicit deletion.
- **Added**: `_SNAPSHOT_KEYS` frozenset used by `_deep_merge` to identify snapshot fields that should be replaced (not merged).
- **Added**: `_InMemoryFirestoreAggregateQuery`, `_InMemoryFirestoreAggregateSnapshot`, and `_InMemoryFirestoreAggregateField` classes. The existing `_InMemoryFirestoreCountQuery` is now an alias for backwards compatibility.
- **Added**: `count()` method to `_InMemoryFirestoreQuery` (previously only on collections).

### mvp_site/tests/fake_firestore.py

- **Added**: `_FakeAggregateQuery`, `_FakeAggregateSnapshot`, `_FakeAggregateField` classes matching the production API.
- **Added**: `count()` method to `FakeQuery` and `FakeFirestoreCollection`.

## Test Changes

- **Added**: `mvp_site/tests/test_firestore_sentinel.py` - 12 tests covering:
  - `_Sentinel` identity preservation through `deepcopy` and `pickle`
  - `_strip_deleted` removal of sentinel values from dicts, nested dicts, and lists
  - `_deep_merge` recursive merging with explicit deletion via `_DELETED`
  - Snapshot key replacement behavior
