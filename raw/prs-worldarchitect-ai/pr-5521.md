## Summary

Fix critical bug in MCP smoke test comment generation that causes GitHub Actions workflow failure with heredoc parsing error.

## Root Cause

The `mcp_smoke_comment.js` script output did not end with a trailing newline, causing the heredoc EOF delimiter to be appended directly to the last line of content instead of appearing on its own line:

```
- Re-run smoke tests with `/smoke` commentEOF
                                      ^^^^ No newline before delimiter!
```

**GitHub Actions Error:**
```
##[error]Unable to process file command 'output' successfully.
##[error]Invalid value. Matching delimiter not found ''EOF''
```

**Workflow Location:** `.github/workflows/mcp-smoke-tests.yml:457-461`
```bash
{
  echo "body<<'EOF'"
  cat /tmp/smoke-comment-body.md  # ← Output missing trailing newline
  echo "EOF"                       # ← Appends to last line instead of new line
} >> "$GITHUB_OUTPUT"
```

## Fix

**File:** `scripts/mcp_smoke_comment.js:132`

```diff
-process.stdout.write(lines.join("\n"));
+process.stdout.write(lines.join("\n") + "\n");
```

Add trailing newline to ensure EOF delimiter appears on its own line as required by heredoc format.
