## Summary

This PR implements a **complete migration of planning blocks from string format to JSON format**, removing all regex-based parsing and establishing a clean, structured data approach. Additionally, it introduces a **mandatory test execution protocol** and **comprehensive malformed content handling**.

### 🚀 Major Architecture Change: String → JSON Migration
- ✅ **Backend**: Modified LLM instructions to generate snake_case JSON keys instead of CamelCase strings
- ✅ **Frontend**: Removed 140+ lines of regex parsing logic, now JSON-only
- ✅ **Validation**: Backend rejects string planning blocks with error logging
- ✅ **Testing**: Updated all 149 tests to expect JSON format
- ✅ **Robustness**: Enhanced error handling for malformed planning blocks

### 📋 JSON Planning Block Format
```json
{
  "thinking": "The player approaches a mysterious door...",
  "choices": {
    "examine_door": {
      "text": "Examine Door",
      "description": "Look carefully for traps or mechanisms",
      "risk_level": "low"
    }
  }
}
```

### 🛡️ Malformed Content Handling
- ✅ **HTML Entity Decoding**: Fixes apostrophe encoding issues (e.g. encoded quotes)
- ✅ **Graceful Fallback**: Displays readable content from malformed blocks
- ✅ **User Experience**: Clear error messages with recovery suggestions
- ✅ **Double-Encoding Prevention**: Decode-then-encode pattern
