## Problem
Previously, the `run_ui_tests.sh mock` command set USE_MOCKS=true but the application never checked this variable. Browser tests always used real Firebase and Gemini services, potentially causing:
- Unexpected API costs
- Test failures when API keys aren't configured
- Inability to test offline

## Solution
1. **Conditional Service Imports**: main.py now checks USE_MOCKS and imports mock wrappers instead of real services
2. **Mock Service Wrappers**: Created wrapper modules that provide the same interface as real services
3. **Extended Mock Functionality**: Added missing methods to mock services (get_initial_story, continue_story, etc.)
4. **Documentation Updates**: Updated run_ui_tests.sh to accurately describe mock vs real behavior
