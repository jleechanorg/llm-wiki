## Production Code Changes

**Before**: `claude_backup.sh` backed up Claude (`~/.claude/projects`) + Codex (`~/.codex/sessions`) only. OpenClaw used a separate cron script in jleechanclaw repo.

**Now**: Single unified backup job covers all three AI conversation sources in parallel:
- `~/.claude/projects` → `Dropbox/claude_conversations/`
- `~/.codex/sessions` → `Dropbox/codex_conversations/`
- `~/.openclaw/agents` → `Dropbox/openclaw_conversations/`

**Why**: Consolidate into one job; eliminate dependency on jleechanclaw repo script for Dropbox backup. OpenClaw sessions JSONL files (5,400+ files in `agents/*/sessions/`) are now covered.

## Test Changes

- Manually verified: updated `~/.local/bin/claude_backup.sh` ran successfully with all 3 sources detected and parallel jobs launched
- Crontab updated: removed `# OpenClaw Dropbox backup (4h, offset :10)` entry

## Known Limitations

- Requires running `scripts/install_backup_system.sh` to update `~/.local/bin/claude_backup.sh` on other machines
