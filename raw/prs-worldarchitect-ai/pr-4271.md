> **Overview**
> Adds deploy-time cache busting by generating content-hashed JS/CSS filenames (now including nested asset paths) and rewriting HTML references accordingly.
> 
> Updates Flask static serving to treat content-hashed assets (and `?v=` URLs) as immutable 1-year cached, while forcing `index.html` to `no-cache` to avoid stale HTML pointing at removed hashed assets; in dev/test, requests for missing hashed assets can fall back to the unhashed file.
> 
> Hardens CI deploy/preview workflows to fail fast if cache busting errors, and adjusts/extends tests and docs to validate hashed references (including nested-path behavior).
