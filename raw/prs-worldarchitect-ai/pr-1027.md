## 🔄 Integration Strategy

### Dual Frontend Architecture (Aligned with PR #1019)
```
mvp_site/
├── static/ → frontend_v1/ (current Bootstrap + vanilla JS)
├── frontend_v2/ (new React + TypeScript)
│   ├── src/components/ (from prototype)
│   ├── services/ (API integration)
│   └── stores/ (state management)
```

### API Compatibility Analysis
**✅ Existing APIs Ready:**
- `GET /api/campaigns` - Perfect for CampaignList.tsx
- `POST /api/campaigns` - Create campaign functionality
- `GET /api/campaigns/<id>` - Campaign details
- `PATCH /api/campaigns/<id>` - Campaign updates
- `POST /api/campaigns/<id>/interaction` - Game actions

**🔧 API Gaps to Address:**
- User profile endpoints (Header.tsx user info)
- Campaign joining/leaving functionality
- Real-time WebSocket integration
- Campaign templates browsing
