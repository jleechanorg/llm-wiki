## Proactive Recovery Strategy

**Before (error propagation):**
```
git clean fails → RuntimeError raised → fixpr agent fails
```

**After (proactive recovery):**
```
git clean fails → log warning → nuke base clone → fresh clone → retry reset → success (or error if retry fails)
```

This handles:
- Untracked files conflicting with main branch
- Detached HEAD state
- Dirty working tree
- Corrupted git state
- Any other git state anomalies
