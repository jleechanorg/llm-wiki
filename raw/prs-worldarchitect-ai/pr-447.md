# Implement Structured Fields UI with DRY Improvements and Comprehensive Testing

## Summary

This PR implements the structured fields UI feature, allowing the display of session headers, planning blocks, dice rolls, resources, and debug information in the game interface. It also includes comprehensive test coverage and DRY improvements to prevent code duplication.

## Implementation Changes

### Backend Implementation

1. **New Utility Module: `structured_fields_utils.py`**
   - Central extraction logic for structured fields from GeminiResponse objects
   - Prevents code duplication across main.py and other modules

2. **Enhanced Response Objects**
   - `gemini_response.py`: Added support for structured response parsing
   - `narrative_response_schema.py`: Added structured fields to schema

3. **API Endpoint Updates**
   - `/api/campaigns/{id}/interaction`: Now includes structured fields in response
   - Structured fields flow through to frontend automatically

4. **Firestore Integration**
   - Stores structured fields alongside narrative data
   - Fixed missing `import constants` bug

### Frontend Implementation

1. **New UI Components in `app.js`**
   - `generateStructuredFieldsHTML()`: Renders structured fields with proper formatting
   - Dice rolls displayed with 🎲 icon and list formatting
   - Resources displayed with 📊 icon
   - Planning blocks rendered as distinct sections
   - Session headers integrated into game flow
   - Debug info displayed in debug mode

2. **CSS Styling Updates**
   - Added styles for `.dice-rolls`, `.resources`, `.planning-block` classes
   - Responsive and visually distinct formatting

### DRY Improvements

- Extracted common structured fields logic to `structured_fields_utils.py`
- Centralized field names in `constants.py`
- Reduced duplication between main.py and test files

### Backend Tests (46 tests)
- `test_gemini_response_structured_fields.py` - 8 tests
