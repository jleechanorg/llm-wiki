## Summary

This PR implements classifier-only heavy dialog routing for latency optimization and quality control:

**Key themes:**
- Heavy dialog agent split (classifier-only routing)
- Removed state/heuristic escalation path
- CI/test runner hardening

## Validation Tenets

Based on comprehensive E2E testing (16 scenarios, 100% pass rate):

1. **Classifier is sole decision-maker** - no state/heuristic escalation
   - All routing decisions come from semantic_intent classifier
   - Zero `force_heavy_context` manipulation across all test scenarios
   - Evidence: `/tmp/worldarchitect.ai/latency_eval/heavy_dialog_agent_real_e2e/iteration_003/`

2. **Classifier makes nuanced decisions** - boundary cases split 4:1, confidence gradient present
   - Boundary scenarios: 4 → HeavyDialogAgent, 1 → DialogAgent
   - Confidence scores show realistic gradient: heavy (0.87 avg) > boundary (0.78 avg) > light (0.79 avg)
   - Lowest confidence heavy scenario (0.694) still correctly routed

3. **Both agents work correctly** - DialogAgent handles social mechanics, HeavyDialogAgent provides deeper narrative
   - DialogAgent successfully handles social HP challenges (not exclusive to HeavyDialogAgent)
   - Clear narrative length gradient: Heavy (1917 chars avg) > Boundary (1749) > Light (1412)
   - Zero quality issues across all 16 E2E scenarios
