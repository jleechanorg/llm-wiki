## Problem
MCP smoke tests against PR preview servers were failing with `429 Too Many Requests`:
```
You've reached your limit of 25 turns per 5 hours...
```

The smoke suite runs \~10+ turns (campaign creation, character creation, story actions, dice, god mode, think mode, then 4 streaming scenarios). By Scenario 7, the test user hits the rate limit.

## Solution
Set `RATE_LIMIT_5HOUR_TURNS=99999` and `RATE_LIMIT_DAILY_TURNS=99999` on PR preview Cloud Run deployments. This effectively disables turn-based rate limiting for preview servers only. Production keeps default limits (25/5h, 50/day).

## Changes
- `.github/workflows/pr-preview.yml`: Add rate limit env vars to `gcloud run deploy --set-env-vars` for preview services.

## Testing
- Preview deploy workflow will run on this PR
- After deploy, MCP smoke tests (workflow_run or `/smoke`) should pass Scenario 7
