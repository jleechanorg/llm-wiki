## Summary
This PR enhances the git-header.sh script to display local repository status directly in the header.

## New Features
- **Local changes indicator**: Shows count of staged (S), modified (M), and untracked (U) files
- **Clean status**: Displays '(clean)' when working directory has no changes
- **Unpushed commits**: Shows '+N' in remote section when local commits haven't been pushed
- **Preserves existing features**: API monitoring functionality remains intact

## Examples
```
[Local: main (clean)  < /dev/null |  Remote: origin/main | PR: none]
[Local: feature (3S,2M,1U) | Remote: origin/main (+2) | PR: #123]
[Local: bugfix (1M) | Remote: origin/bugfix | PR: #456]
```

## Benefits
- **Instant visibility** of working directory state
- **No need** to run `git status` separately
- **Prevents accidental commits** of unintended changes
- **Shows push status** at a glance

## Testing
- Tested with various combinations of staged/modified/untracked files
- Verified unpushed commits display
- Confirmed clean status shows correctly
- API monitoring features continue to work as before
