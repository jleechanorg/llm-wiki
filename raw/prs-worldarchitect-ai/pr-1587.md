## Summary
Fixes critical issues in the convergence and copilotc systems that were causing agents to run indefinitely and create duplicate PRs instead of updating existing ones.

## Changes Made

### 1. Improved Convergence Exit Criteria (`converge.md`)
- **Replace impossible 100% success threshold** with PRIMARY vs SECONDARY criteria framework
- **Add measurable progress tracking** with diminishing returns detection  
- **Implement 'GOOD ENOUGH' threshold** to prevent infinite loops on marginal improvements
- **Add PR context awareness** to prevent agents from creating new PRs when fixing existing ones
- **Enhanced goal achievement detection** with evidence-based validation

### 2. PR Branch Constraints (`copilotc.md`)
- **Add mandatory protocol** to stay on existing PR branch throughout execution
- **Prevent duplicate PR creation** when working on existing PR comment resolution
- **Enforce branch discipline** with clear before/during/after execution rules
- **Add success/failure criteria** specifically for PR context awareness

## Problem Solved
**Before**: Agents would run for 20+ minutes creating new PRs instead of updating existing ones, causing:
- Duplicate work and confusion
- Infinite loops due to impossible 100% success thresholds
- Wasted orchestration resources
- Poor user experience

**After**: Agents now:
- Complete tasks in reasonable timeframes (3-10 iterations typical)
- Update existing PRs instead of creating duplicates
- Have clear, measurable exit criteria
- Maintain proper branch discipline

## Testing
- Validated exit criteria improvements prevent infinite loops
- PR branch constraints ensure proper context awareness
- Both systems now have clear success/failure boundaries

## Impact
- **Orchestration Reliability**: Prevents runaway agents
- **Resource Efficiency**: Faster task completion with proper exits
- **User Experience**: Predictable behavior and proper PR updates
- **System Stability**: Clear boundaries prevent infinite resource consumpt
