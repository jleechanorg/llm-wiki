## Summary: Prompt-Builder Refactor (lb3 Epic) - All Phases Complete

**Problem:** The prompt-loading architecture had scattered, inconsistent code paths with unordered sets and repetitive `del unused_params` patterns across agents.

**Solution:** Establishes a clean, ordered prompt-loading architecture with a single entry point for fixed-prompt agents; StoryModeAgent/CombatAgent keep custom assembly for dynamic behavior.

### What Changed

| Component | Before | After |
|-----------|--------|-------|
| Prompt ordering | `REQUIRED_PROMPTS: frozenset` (unordered) | `REQUIRED_PROMPT_ORDER: tuple` (ordered) |
| Loading | Per-agent custom methods | Generic `build_from_order()` |
| Entry point | Multiple paths | Single `build_for_agent(agent)` for fixed-prompt agents |
| Unused params | `del selected_prompts, turn_number...` | `FixedPromptAgent` base class |

### Key Invariants Enforced
1. `master_directive` must always be first
2. `game_state` -> `planning_protocol` must be consecutive
3. No duplicate prompt types allowed

### Design Decisions
- **FixedPromptAgent**: GodModeAgent, PlanningAgent, InfoAgent, RewardsAgent use this base class with `build_for_agent()`
- **Custom assembly**: StoryModeAgent (optional prompts, turn-based continuation, living world) and CombatAgent (use_default_world param) keep custom `build_system_instructions()` but use `build_from_order()` for the base prompt set to enforce invariants
- **Drift-guard tests**: Verify Story/Combat output prefix matches `build_from_order(REQUIRED_PROMPT_ORDER)`
