## Summary
- Remove `--delete` flag from rsync in `/localexportcommands`
- Remove `rm -rf` from fallback path
- Command now **merges** files instead of **replacing** them

## Problem
`/localexportcommands` was using `rsync -a --delete` which deleted any files in `~/.claude/commands/` that weren't in the project's `.claude/commands/`. This caused:
- Plugin-installed commands (e.g., obra/superpowers) to be deleted
- Manually added symlinks to be removed
- Any files from other tools installing to `~/.claude/` to be wiped

## Solution
Remove destructive flags so the command merges content instead of replacing it. Users with multiple sources of commands (project + plugins + manual) now get expected behavior.
