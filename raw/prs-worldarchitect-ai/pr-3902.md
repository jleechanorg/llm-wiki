# Purge LLM Dialog Classification & Centralize Intent (PR #3902)

## 🚨 CRITICAL ARCHITECTURAL PRINCIPLE: CLASSIFIER-ONLY ROUTING

**NO KEYWORD MATCHING** for dialog detection. This PR enforces strict classifier-only routing:

- ✅ **Dialog routing**: ONLY through semantic classifier (`LocalIntentClassifier`)
- ❌ **DIALOG_KEYWORDS**: **DELETED** (constant no longer exists)
- ❌ **Social keyword detection**: REMOVED (no "persuade", "negotiate", etc. fast-paths)
- ❌ **Quoted speech detection**: REMOVED (no startswith quote fast-paths)
- ❌ **Choice matching keyword detection**: REMOVED (no keyword-based is_dialog_choice)
- ❌ **DialogAgent.matches_game_state keyword matching**: REMOVED
- ✅ **Explicit overrides**: ALLOWED (GOD MODE, THINK: prefix)
- ✅ **State-based routing**: ALLOWED (dialog_context.active, last_action_type)
- ✅ **Semantic classifier**: THE ONLY PATH for dialog intent detection

**Rationale**: Keyword fast-paths bypass the classifier and violate the "Local Classification First" principle. The classifier must be trained to handle ALL dialog detection.

**Latest Changes** (Commit 2928dc7d2):
- **DELETED `DIALOG_KEYWORDS` constant** - no longer exists in codebase
- Removed keyword matching from `DialogAgent.matches_game_state()`
- Removed keyword-based `is_dialog_choice` detection in choice matching
- All dialog routing now goes ONLY through `classify_intent()`

## Overview
This PR completely removes ALL keyword-based dialog routing and centralizes agent classification. It enforces "Local Classification First" for performance (<50ms) and architectural consistency.
