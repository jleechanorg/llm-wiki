## Summary
Enhanced the system backup script to backup both Claude Code and Codex conversation logs to Dropbox, providing comprehensive AI conversation history preservation.

## Changes Made

### System Script Modifications
Modified `~/.local/bin/claude_backup.sh` (system-level file, outside repository) to support dual AI platform backups:

#### 1. Dual Source Support
- **Before**: Single source `SOURCE_DIR="$HOME/.claude/projects"`
- **After**: Separate sources for Claude and Codex
  - `CLAUDE_SOURCE_DIR="$HOME/.claude/projects"`
  - `CODEX_SOURCE_DIR="$HOME/.codex/sessions"`

#### 2. Separate Backup Destinations
- **Before**: Single folder `claude_conversations`
- **After**: Dedicated folders
  - `claude_conversations/` for Claude Code
  - `codex_conversations/` for Codex CLI

#### 3. Enhanced Validation
- Separate prerequisite checks for both source directories
- Graceful handling when either source is missing
- Maintained all security features (path validation, secure temp dirs)

#### 4. Improved Backup Logic
- Modified `backup_to_destination()` to accept source parameter
- Skip backup gracefully if source directory doesn't exist
- Enhanced logging to track both backups independently
