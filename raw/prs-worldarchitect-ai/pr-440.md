## Problem
The Claude Code GitHub Action is failing with exit code 128 during the prepare step. This is typically a git-related error indicating permission or authentication issues.

## Solution
Based on research, the most common fix is to:
1. Add an explicit checkout step before the Claude Code Action
2. Ensure proper repository permissions in Settings → Actions → General

## Changes
1. **Modified `.github/workflows/claude-code.yml`**: Added explicit `actions/checkout@v4` step
2. **Created `.github/workflows/claude-code-v3.yml`**: Alternative with `actions/checkout@v3` (some users report better success with older versions)
3. **Created `.github/workflows/test-basic.yml`**: Diagnostic workflow to test if basic checkout works

## Testing
1. This PR should trigger the workflows
2. Check if the Claude Code action now works properly
3. If not, try the v3 version by renaming the workflow files
4. Ensure repository permissions are set to "Read and write permissions" with "Allow GitHub Actions to create and approve pull requests" checked

## Additional Steps if Still Failing
1. Check that `ANTHROPIC_API_KEY` secret is properly set
2. Verify the repository has proper GitHub App permissions
3. Consider reaching out to Anthropic support as this is a beta action
