## Goals
- Make auto-merge failures debuggable by including blocker names in observability logs

## High-level description of changes
Added optional `blockers` field to `ReactionResult` type and populated it from the merge gate check result when auto-merge fails. The lifecycle-manager observability log now includes which specific conditions (CI, mergeable, CR approved, etc.) blocked the merge.
