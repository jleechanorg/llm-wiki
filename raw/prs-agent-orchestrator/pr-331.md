## Background
- Cursor AO sessions were discarding `agentConfig.model` because the Cursor plugin removed the model field before launch.
- The result was non-deterministic behavior when old values like `claude-sonnet-4-6` were present in AO config.
- This patch restores explicit model passing and normalizes unsupported values to a safe Cursor default.

## Goals
1. Ensure Cursor launch always includes a model argument.
2. Set a safe Composer-based default for Cursor by default.
3. Preserve explicit Composer model IDs configured for tasks and project sessions.

## Tenets
- Keep changes local to the Cursor plugin when behavior can be fixed there.
- Use safe defaults that avoid startup failures without masking configuration mistakes.
- Make changes observable via tests and launch-command assertions.

## High-level description of changes
- Updated `packages/plugins/agent-cursor/src/index.ts` to pass a `--model` flag in all Cursor launches.
- Added Composer-aware normalization in `normalizeCursorModel(...)` with default `composer-2-fast`.
- Preserved explicit values that already match `composer-*` and normalized unsupported values (including stale Anthropic IDs) to `composer-2-fast`.
- Updated `packages/plugins/agent-cursor/src/index.test.ts` assertions to validate default/passthrough/fallback behavior.
- Updated `agent-orchestrator.yaml.example` to document Cursor model guidance using `composer-2-fast`.
