## Raw Body
# Background
The zombie sweep logic in the lifecycle manager incorrectly relied on branch names extracted from git checkout commands that were sometimes confused with extensionless file paths (like `docs/README`). This caused false-positive detection of active workers as zombies when the worker checked out an extensionless file path, writing it to metadata as if it were a branch.

# Goals
Ensure active workers are not falsely identified as zombies by strictly validating branch metadata against actual git refs.

# Tenets
System reliability and robustness of worker supervision.

# High-level description
Updated the git checkout detection in `.claude/metadata-updater.sh` to verify branch candidates using `git show-ref --verify --quiet refs/heads/$branch`. This ensures only real local branches are persisted as branch metadata, preventing extensionless file paths from corrupting the session status and triggering zombie kills.

# Testing
Verified `metadata-updater.sh` with bash execution to confirm file paths are ignored. Existing test coverage for `session-reaper.ts` was confirmed to be green.
