## stalled-worker-auditor.ts — What it does

`packages/core/src/stalled-worker-auditor.ts`

**Purpose:** AO worker that audits stalled workers and eloop coverage gaps.

1. **Detects stalled workers** — uses `stuck-worker-detector.ts` to inspect tmux pane content for all AO-managed sessions (via `parseTmuxName`), checking for:
   - Shell prompt with no activity indicators → agent CLI exited → `kill`
   - Waiting/permission patterns → agent needs input → `nudge`
   - Activity indicators present → agent still working → `none`

2. **Audits eloop coverage** — for each stalled worker, checks whether the project has a live `agent-stuck` reaction configured (`auto: true` + `threshold`)

3. **Flags gaps** — reports where the eloop is NOT handling a stalled worker (reaction missing or not actioned)

4. **Sends MCP mail alerts** — on unhandled gaps, sends an alert to `jleechanclaw`
