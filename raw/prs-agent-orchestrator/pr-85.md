## Raw Body
### Background
`ao spawn --agent gemini` failed with "Agent plugin 'gemini' not found" even though `@jleechanorg/ao-plugin-agent-gemini` existed in the monorepo. The `loadBuiltins()` function silently swallowed the module-not-found error via try/catch.

### Goals
- `ao spawn --agent gemini` successfully spawns a gemini-powered session
- No config changes required in `agent-orchestrator.yaml`

### Tenets
- Config-first: fix via registration, not core code restructure
- Minimal diff

### High-level description of changes
Two-part fix:
1. Add `{ slot: "agent", name: "gemini", pkg: "@jleechanorg/ao-plugin-agent-gemini" }` to `BUILTIN_PLUGINS` in `plugin-registry.ts`
2. Add `@jleechanorg/ao-plugin-agent-gemini` as a workspace dependency in `packages/cli/package.json` and root `package.json` so pnpm creates the symlink in `node_modules/@jleechanorg/`

### Testing
- `ao spawn --agent gemini -p agent-orchestrator` → session ao-327 spawned successfully
- Gemini CLI v0.34.0 launched, authenticated as jleechan@gmail.com
- All 225 CLI tests + 702 core tests pass

**Note:** Gemini CLI shows a "trust folder" prompt on first run in a new worktree (one-time interactive step). A follow-up to handle this non-interactively (similar to the cursor workspace trust fix) will be tracked separately.

### Low-level details
- **packages/core/src/plugin-registry.ts**: Added gemini to BUILTIN_PLUGINS array (1 line)
- **packages/cli/package.json**: Added workspace dependency (1 line)
- **package.json**: Added workspace devDependency (1 line)
- **pnpm-lock.yaml**: Updated lockfile to reflect new dependency
