## Summary
Fixes orch-xrpl — P0 bug where auto-merge permanently blocks on "Evidence review pass" because it looks for a GitHub review from `evidence-reviewer[bot]`, but the evidence-reviewer subagent posts its verdict as a **PR comment**, not a GitHub review.

## Root Cause
The `approved-and-green` reaction in `lifecycle-manager.ts` hardcodes `requiredChecks: ["evidence-review"]`:
```typescript
const mergeGateConfig: MergeGateConfig = {
  enabled: true,
  requiredChecks: ["evidence-review"],  // always fails
  ...project.mergeGate,
};
```

`checkMergeGate` then calls:
```typescript
const evidenceReview = getLatestDecisiveReview(reviews, "evidence-reviewer[bot]");
evidencePassed = evidenceReview?.state === "approved";
```

This looks for a GitHub review authored by `evidence-reviewer[bot]`. The evidence-reviewer subagent posts its verdict as a **PR comment** (`**PASS** — evidence review: agent self-reviewed ✅, CR reviewed ✅, codex passed ✅`). No GitHub review is ever created.

Result: every `approved-and-green` auto-merge attempt permanently fails with `blockers=["Evidence review pass"]`.

## Fix
Remove `evidence-review` from `requiredChecks` in the auto-merge reaction's mergeGateConfig. Agents are already required to run `/er` and post the PASS verdict as a PR comment before posting the green signal — the review-based check was redundant and broken.
