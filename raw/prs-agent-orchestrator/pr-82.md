## Background

`loadActiveSessionRecords()` in `session-manager.ts` was returning all session records including those with terminal statuses (killed, merged), causing stale data dir accumulation in the active session list.

## Goals

- Prevent killed/merged sessions from appearing in the active session list
- Preserve repair-on-read behavior that corrects orphaned PR attachments and duplicate PR metadata

## Changes

In `loadActiveSessionRecords()`:
1. Load ALL records (no pre-filter)
2. Run `repairSessionMetadataOnRead()` on all records first (so stale statuses like merged are corrected to working before filtering)
3. Filter out records where status === killed || status === merged before returning

This two-pass approach ensures stale metadata is always repaired while keeping the active session list clean.

## Testing

All 690 core package tests pass. Two session-manager tests required adjustment to correctly expect merged sessions to be repaired and then excluded from the returned list.
