## Background
Agents (Claude Code, Cursor, etc.) were able to run `gh pr merge` despite CLAUDE.md and workflow rules prohibiting it. Prompt-based rules are advisory — this enforces the policy in code as a hard guardrail.

## Goals
- Block `gh pr merge` in all agent workspace hooks by default
- Provide an escape hatch (AO_ALLOW_GH_PR_MERGE=1) for trusted manual flows
- Move the guardrail before session/metadata checks so it cannot be bypassed

## Tenets
- Guardrails must be in code, not just prompts
- Fail-safe: deny by default
- Minimal blast radius — only affects merge, not other gh commands

## High-level description
Adds `gh pr merge` pattern detection and hard denial in the PreToolUse hook for both `agent-base` and `agent-claude-code` plugins. The guardrail runs before AO_SESSION/metadata checks to prevent bypass via unset env vars.

## Testing
- Pattern matching tested against: `gh pr merge`, `gh pr merge --admin --squash`, `FOO=1 gh pr merge`
- Escape hatch verified: `AO_ALLOW_GH_PR_MERGE=1 gh pr merge` returns allow

## Files changed
- `packages/plugins/agent-base/src/index.ts` — adds merge guardrail to shell hook
- `packages/plugins/agent-claude-code/src/index.ts` — adds merge guardrail + symlink security fix

Closes #64
