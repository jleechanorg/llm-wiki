## Background
Eloop generates ~150K+ tokens over long sessions by running full observation phase even when nothing changed. This PR adds a healthy-cycle fast path and session budget guard to `generateEvolveLoopSection()`.

## Goals
- [x] Healthy-cycle fast path: single-line recap instead of full observation dump
- [x] Session budget instruction: emit SESSION BUDGET warning after 6hr/36+ cycles
- [x] Handoff file support: `{kbDir}/{projectId}-handoff.md` with last 5 findings
- [x] Unit tests for both instructions

## High-level description of changes

**`packages/core/src/orchestrator-prompt.ts`**: Updated Phase 1 (OBSERVE) in `generateEvolveLoopSection()`:
- Replaced vague "Fast path: if nothing abnormal, output 'all clear' and exit" with explicit single-line format: `\u2713 Cycle N: all clear — N workers alive, N open PRs`
- Added instruction to **exit loop immediately** (skip Phase 2-6) on healthy cycle
- Added session budget trigger: after 6 hours or 36+ cycles in a single session, emit `SESSION BUDGET: consider /clear + fresh eloop session with handoff`
- Added handoff file instruction: write the resolved absolute path (`{kbDir}/{projectId}-handoff.md`) with last 5 findings from JSONL knowledge base

**`packages/core/src/__tests__/orchestrator-prompt.test.ts`**: Added 3 unit tests:
1. Fast-path instruction appears in generated prompt (check for `\u2713` + skip phases text)
2. Session budget instruction appears (6hr/36+ cycles, SESSION BUDGET, /clear)
3. Handoff file path appears with resolved absolute path (e.g. `/tmp/kb/my-app-handoff.md`)
