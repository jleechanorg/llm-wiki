## Background

CodeRabbit posts incremental `COMMENTED` reviews after its `APPROVED` review when it has additional (non-blocking) suggestions. In `deriveReviewDecisionGraphqlFromReviews()`, these newer `COMMENTED` reviews replace the earlier `APPROVED` as the "latest" review per user (by timestamp), causing `every(APPROVED)` to fail and falling through to `REVIEW_REQUIRED`. This blocks the merge gate on PRs that should be approved.

## Goals

- Prevent `COMMENTED` reviews from overriding the existing `APPROVED` or `CHANGES_REQUESTED` decision in the GitHub REST fallback path.
- Ensure merge gates unblock correctly when CR posts non-blocking incremental feedback.

## Changes

**packages/plugins/scm-github/src/index.ts**
- Modified `deriveReviewDecisionGraphqlFromReviews()` to skip `COMMENTED` and `PENDING` reviews when collecting the latest decisive review per user — only `APPROVED` and `CHANGES_REQUESTED` determine the decision.
- Added a comment explaining the fix and linking to bd-77b.

**packages/plugins/scm-github/test/index.test.ts**
- Added test: `"returns approved when CR COMMENTED review is newer than APPROVED review (bd-77b)"`
