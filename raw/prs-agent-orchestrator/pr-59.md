> [!NOTE]
> **Medium Risk**
> Changes the auto-merge reaction path to enforce `checkMergeGate` (including required bot approvals) before merging, which can newly block merges in some projects if config/SCM data differs from expectations.
> 
> **Overview**
> Auto-merge reactions now run the full `checkMergeGate` instead of relying on `scm.getMergeability`, building a default `MergeGateConfig` (enabled, requiring `evidence-review`) and honoring per-project `project.mergeGate` overrides (including disabling the gate). When the gate fails, auto-merge is blocked and an action-priority notification is emitted containing the gate blockers/check results.
> 
> `checkMergeGate` is hardened to treat undefined SCM response array
