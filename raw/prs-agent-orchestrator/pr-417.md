## Raw Body
### Background
GitHub sometimes returns an empty-string \`reviewDecision\` in \`gh pr view\` payloads. In \`scm-github\`, the single-PR and batch mergeability paths did not normalize that value the same way as null/undefined, which left an unknown review state looking like neutral \`none\` instead of fail-closed \`pending\`.

Tracked in \`bd-o7t\`.

### Goals
- Make empty-string \`reviewDecision\` normalize the same way as null/undefined in \`scm-github\`.
- Ensure mergeability treats that state as review-required instead of allowing a neutral path.
- Cover the regression with tests in the existing \`scm-github\` test suite.

### Tenets
- Fail closed for unknown review state.
- Keep the fix narrow to the GitHub SCM normalization paths.
- Add tests at the behavioral boundary instead of relying on comments alone.

### High-level description of changes
- Updated \`normalizeMergePayloadFromRestShape()\` so \`""\` joins null/undefined in the conservative \`REVIEW_REQUIRED\` path.
- Aligned \`getReviewDecision()\`, \`getPRStateAndReview()\`, \`getMergeability()\`, and the batch status path so empty-string review decisions resolve to \`pending\` / \`Review required\` instead of \`none\`.
- Added regression coverage for empty and null review decisions plus the mergeability blocker case.
- Added \`pnpm.overrides\` to force \`handlebars >=4.7.9\` to patch the critical handlebars vulnerability.

### Testing

- \`pnpm test\`
