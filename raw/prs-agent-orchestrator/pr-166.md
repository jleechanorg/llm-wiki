## Background

Ghost git worktrees accumulate when AO sessions die without proper cleanup, blocking `backfillUncoveredPRs` with:
```
fatal: refusing to fetch into branch checked out at ~/.worktrees/agent-orchestrator/ao-NNN
```
After 3 consecutive failures the backfill aborts. Manually cleaned twice in 2026-03-24 session — mandatory /harness invocation.

## Goals

- Permanently prevent ghost worktree accumulation via automated GC in lifecycle-worker
- Document triage checklist for the two failure modes in CLAUDE.md

## Tenets

- Best-effort only: errors must never crash the lifecycle worker
- Fail-safe: only remove worktrees absent from AO DB AND with dead tmux session
- Mirror existing sweepOrphanTmuxSessions pattern for consistency

## High-level description of changes

**lifecycle-worker.ts**: Add `sweepOrphanWorktrees()` — iterates `~/.worktrees/{projectId}/`, checks AO DB + tmux liveness, removes dead-session worktrees with `git worktree remove --force`. Wired into both startup sweep and orphanSweepTimer.

**CLAUDE.md**: Add main repo branch invariant section + lifecycle claim failure triage checklist.
