## Background

Self-healing infrastructure is fragmented across `setup.sh`, `ao-doctor.sh`, `start-all.sh`, and `health-check.sh` with overlapping check logic. When components fail (launchd deregisters, main repo drifts to feature branch, ghost worktrees accumulate), doctor doesn't detect or fix them. PR #294 added runner checks but didn't address the structural duplication.

## Goals

1. Single source of truth (`ao-checks.sh`) for all health check/fix functions
2. `ao doctor --fix` = non-interactive subset of `setup.sh` (same checks, no prompts)
3. `start-all.sh` calls `ao doctor --fix` as pre-flight (no duplicated logic)
4. New operational checks: launchd, orchestrator, ghost worktrees, main repo branch, rate limits, skeptic chain

## Tenets

- No duplication: one function per check, sourced everywhere
- Idempotent: `doctor --fix` safe to run on every startup and every 5-min health cycle
- Additive: existing checks preserved, new checks alongside

## High-level description of changes

**`scripts/ao-checks.sh` (NEW, 590 lines)** — shared check/fix library sourced by doctor.sh and setup.sh. Contains all existing checks (extracted from ao-doctor.sh) plus new operational checks:
- `check_launchd_services()` — verify lifecycle-worker launchd plists registered and running; --fix bootstraps
- `check_main_repo_branch()` — verify main repo on `main`; --fix checks out main
- `check_ghost_worktrees()` — detect orphan AO worktrees (pattern-guarded); --fix removes
- `check_rate_limits()` — warn on low GraphQL/REST budget
- `check_skeptic_chain()` — verify `ao skeptic verify --help` runs
- `check_runners()` — ported from PR #294 (self-hosted runner health)

**`scripts/ao-doctor.sh` (refactored)** — now sources ao-checks.sh, calls all check functions. No more inline check definitions.

**`packages/cli/scripts/ao-doctor.sh`** — kept in sync with scripts/ao-doctor.sh. Sources `scripts/ao-checks.sh` via `AO_REPO_ROOT` env (set by script-runner.ts) or monorepo-relative path.
