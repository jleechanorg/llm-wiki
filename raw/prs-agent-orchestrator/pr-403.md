## Background
PR #403 implements bead `bd-pfx`: AO agent hooks should prepend `[agento]` to `gh pr create` titles instead of denying the command and forcing a retry. Reviewers found three concrete gaps in the original rewrite path: it failed open when safe parsing was not available, it mis-parsed `-t=<title>`, and it round-tripped commands through `shlex.join()`, which normalized shell text and could change quoting semantics.

## Goals
- Preserve the original command text while prepending `[agento]` to the actual title argument.
- Fail closed whenever the hook cannot safely parse or rewrite the title.
- Cover the attached short-flag form (`-t=<title>`) and the no-`python3` path with tests.

## Tenets
- Preserve shell intent instead of normalizing the whole command string.
- Share the rewrite logic between `agent-base` and `agent-claude-code` so bug fixes land in one place.
- Keep the scope limited to the `gh pr create` title guardrail and its tests.

## High-level description of changes
This update extracts the `gh pr create` title-prefix guard into a shared module used by both plugins. The new Python helper still uses `shlex` for semantic argument detection, but it now computes raw shell token spans and rewrites only the title token in place. That preserves the surrounding command text and fixes the `-t=<title>` case without relying on `shlex.join()`. The guard now denies the command when `python3` is unavailable or when token parsing cannot be trusted, which closes the fail-open path reviewers called out.
