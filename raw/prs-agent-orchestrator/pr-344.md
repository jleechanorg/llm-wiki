## Raw Body
## Background

The `ai.agento.novel-daily` launchd job was writing to a stale path
(`/Users/jleechan/.cursor/worktrees/worktree_cursor/ctj/agent-orchestrator/`) instead
of the canonical repo. Additionally, all daily entries were being appended
to the monolithic `novel/the-daily-lives-of-workers.md` file — making it
hard to navigate and impossible to run idempotently per-day.

## Goals

1. New daily entries written as `novel/workers/{YYYY-MM-DD}.md` (separate files).
2. Launchd plist points to the stable canonical repo path (not ephemeral worktrees).
3. Idempotent execution: skip if the date file already exists.

## Tenets

- Prefer stable paths over ephemeral worktree paths for automation.
- Atomic file writes (rename(2)) to avoid partial files on disk.
- Script behavior must be reproducible when run from any worktree.

## High-level description

- `generate-daily-entry.mjs`: Added `--daily YYYY-MM-DD` flag that writes to
  `novel/workers/{date}.md` using atomic rename. Skips silently if the file
  already exists. Also appends a summary to the main novel log.
- `setup-launchd.sh`: Added `_resolve_repo_root()` that detects ephemeral AO
  worktrees (name matching `^ao-[0-9]+$`) and resolves to the stable canonical
  path `/Users/jleechan/project_agento/agent-orchestrator`. Also updated to pass
  `--daily "$(date '+%Y-%m-%d')"` instead of the old `--file` append mode.
- `launchd/ai.agento.novel-daily.plist.template`: Updated header comment to
  document the new separate-file behavior.
- `novel/workers/README.md`: Added documentation for daily separate-file
  entries and the launchd integration.

## Testing

- Local test: `node scripts/novel/generate-daily-entry.mjs --daily 2026-04-02`
  — wrote `novel/workers/2026-04-02.md` successfully.
- Idempotency test: re-running same command → `SKIP: ... already exists`.
- ESLint: `npx eslint scripts/novel/generate-daily-entry.mjs` — 0 errors.
- Plist regeneration: `bash scripts/setup-launchd.sh novel` → plist points to
  canonic
