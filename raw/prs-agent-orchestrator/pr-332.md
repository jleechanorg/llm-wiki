## Background

The Zod-backed `AntigravityConfig` schema, `WorkspaceMapSchema`, and `parseConfig` / `defaultConfig` wiring for `runtime-antigravity` are already on `main` from earlier work. This PR branch was rebased onto current `main` and now carries **follow-up runtime hardening** on top of that baseline.

## Goals (this PR)

- Set `maxBuffer: 10 * 1024 * 1024` on all Peekaboo `execFile` calls so large `see` JSON snapshots cannot overflow Node’s default stdout buffer.
- Wrap `JSON.parse` of Peekaboo stdout with clear errors (empty stdout + invalid JSON preview).
- Rename the inner `Runtime.create` parameter from `config` to `createConfig` so it is not confused with plugin-level `AntigravityConfig` / `runtimeConfig`.
