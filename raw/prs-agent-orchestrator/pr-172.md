> [!NOTE]
> **Medium Risk**
> Adds a new runtime lifecycle callback (`onIdle`) that updates session metadata and introduces background polling/cleanup behavior in the Antigravity runtime; mistakes could cause incorrect session states or lingering pollers/processes.
> 
> **Overview**
> **Adds runtime-driven idle detection and wires it into session lifecycle.** `RuntimeCreateConfig` now supports an optional `onIdle` callback, and `session-manager` passes it when spawning/restoring sessions to persist `status="idle"` into metadata.
> 
> **Antigravity runtime now polls for idle transitions and hardens cleanup.** A shared poller is started only when `onIdle` is provided, skips CLI-fallback sessions (`managerWindowId === -1`), and won’t advance state if `onIdle`/`onCapacityWait` throws (with new tests). `destroy()` is made idempotent and safer around fallback PID handling, and Peekaboo fallback error matching is tightened; GH rate-limit polling switches from `execSync` to async `execFile`.
