> [!NOTE]
> **Medium Risk**
> Adds new CLI paths that write/append markdown files and optionally call an external API via `curl`, so mistakes could overwrite/duplicate novel content or fail in CI/automation contexts. Session input is validated and writes are atomic, reducing but not eliminating operational risk.
> 
> **Overview**
> Introduces a new `novel/workers/` workflow where each worker session gets its own Markdown entry file (with initial backfilled entries) and documents the format/naming in `novel/workers/README.md`.
> 
> Extends `scripts/novel/generate-daily-entry.mjs` with a `--session`/`--pr` mode that generates prose (Anthropic API when `ANTHROPIC_API_KEY` is set, otherwise a template), writes `novel/workers/{session}.md` **atomically and without overwriting**, and optionally appends a short section to the monolithic novel for backward compatibility.
> 
> Adds `scripts/novel/aggregate.mjs` to rebuild `novel/the-daily-lives-of-workers.md` by keepi
