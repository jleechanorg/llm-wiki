## Background

Issue #46 was about converting scm-github from GraphQL to REST. While doing that work, the skeptic verdict was already added as the 7th condition in checkMergeGate (via a parallel effort in PR #206), but the scm-github plugin never implemented getSkepticVerdict. This means:

- When skepticRequired: true in MergeGateConfig, checkMergeGate calls scm.getSkepticVerdict(pr)
- The GitHub SCM plugin does not implement it — the optional hook is missing
- The critical guard at merge-gate.ts:158 blocks merge: skepticPassed = false when hook is missing

## Goals

- Implement getSkepticVerdict in the scm-github plugin so the 7th merge gate is fully wired end-to-end
- Ensure checkMergeGate with skepticRequired: true works correctly with the GitHub SCM plugin

## Tenets

- REST API for GitHub operations (no GraphQL for this method)
- Non-fatal: API errors return SKIPPED rather than throwing
- Configurable skeptic bot author via skepticBotAuthor config or GH_SKEPTIC_BOT_AUTHOR env var

## High-level description of changes

Added getSkepticVerdict method to the GitHub SCM plugin:

1. packages/plugins/scm-github/src/index.ts:
   - Extract SKEPTIC_BOT_AUTHOR from config/env (defaults to jleechan-agent[bot])
   - Fetch issue comments via REST API (repos/{owner}/{repo}/issues/{pr}/comments?per_page=100) with pagination
   - Scan all pages for comments by SKEPTIC_BOT_AUTHOR containing <!-- skeptic-agent-verdict --> marker AND VERDICT: PASS/FAIL
   - Return "PASS" | "FAIL" | "SKIPPED" accordingly (keeps last/newest matching verdict)

2. packages/plugins/scm-github/test/index.test.ts:
   - 11 new tests covering: PASS verdict, FAIL verdict, no skeptic bot (SKIPPED), marker but no VERDICT (SKIPPED), VERDICT but no marker (SKIPPED), API error (SKIPPED), empty comments (SKIPPED), custom bot author via config, multiple verdicts returns newest, pagination finds verdict on page 2
