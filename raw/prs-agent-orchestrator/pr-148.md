## Summary

## Background
When ao send calls sendViaTmux directly (bypassing sessionManager.send which routes through runtime.sendMessage), there was no pre-send check to verify the agent CLI was still alive. If the agent had crashed and the shell prompt was showing, the message would be pasted to bare bash.

## Summary

Fixes a gap where ao send could paste a message to bare bash instead of the agent when the agent CLI had crashed/exited inside a tmux session.

## Background
When ao send calls sendViaTmux directly (bypassing sessionManager.send which routes through runtime.sendMessage), there was no pre-send check to verify the agent CLI was still alive. If the agent had crashed and the shell prompt was showing, the message would be pasted to bare bash.

## Goals
- Detect when the agent CLI has died inside a tmux pane
- Prevent messages from being pasted to bare bash without warning
- For AO-managed sessions: restart the agent automatically
- For unmanaged tmux sessions: warn the user so they can take action

## Tenets
- Prefer user visibility over silent failure
- Minimal blast radius — only affects the direct tmux fallback path
- No breaking changes to the sessionManager path

## High-level description of changes
Two new test cases covering the alive-no-restart and dead-warn paths. No changes to core routing logic.
