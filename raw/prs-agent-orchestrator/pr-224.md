## Background
When `backfillUncoveredPRs()` fails to claim a PR (e.g. CONFLICTING), the worktree created during spawn is left behind with the PR branch checked out and locked. Every subsequent retry creates a NEW worktree, which also fails because git refuses to fetch into a branch checked out elsewhere. This cascading failure poisoned 55 worktrees over 13 hours, blocking lifecycle-worker backfill.

## Goals
- Clean up worktree AND local branch on `claim_failed` so subsequent retries succeed
- Clean up stale branches in `pruneStaleWorktrees()` so git worktree zombies can't block future claims
- Never abort backfill on a single claim failure when direct cleanup succeeded

## Changes

### packages/plugins/workspace-worktree/src/index.ts
`destroy()`: Capture checked-out branch before worktree removal, then delete it after (if AO-managed pattern). Prevents a stale local branch from blocking `git fetch` into the same branch name.

### packages/core/src/session-manager.ts
`pruneStaleWorktrees()` Pass 1: After `git worktree remove`, delete the stale branch using the same AO-managed pattern guard.

### packages/core/src/backfill-extensions.ts
`backfillUncoveredPRs()`: When `claimPR` fails and `sessionManager.kill()` fails (orphan not registered), add direct worktree cleanup via `execFileAsync`. Change `return false` to `continue` after successful cleanup so backfill tries the next PR. Add postcondition check to verify worktree is gone before marking cleanup success.
