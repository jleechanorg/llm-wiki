## Background
The /auton diagnostic revealed 10+ concurrent lifecycle-worker processes running simultaneously (expected ≤3, one per project). The root cause: legacy `com.agentorchestrator.lifecycle-*.plist` files in `~/Library/LaunchAgents/` use `KeepAlive: true`, which causes launchd to respawn every 30s unconditionally — even when a worker is already running for that project. Since the worker exits with code 0 when it detects a duplicate, launchd keeps retrying forever.

## Goals
- Eliminate unbounded lifecycle-worker process accumulation
- Make `scripts/setup-launchd.sh` self-cleaning: running it removes the legacy plists automatically
- Raise ThrottleInterval from 30s → 60s as an additional respawn buffer

## Tenets
- Minimal change: fix the launchd config, not the worker code
- Idempotent: running setup-launchd.sh twice is safe
- No breaking changes to the canonical `ai.agento.lifecycle-all` approach

## High-level description of changes
The canonical launchd approach (`ai.agento.lifecycle-all` → `start-all.sh`) already uses `KeepAlive: { SuccessfulExit: false }` which only respawns on crash. The legacy per-project plists (`com.agentorchestrator.lifecycle-*.plist`) with `KeepAlive: true` are the source of accumulation and are obsolete. `setup-launchd.sh` now removes them as part of setup.

## Testing
- Ran `/auton` before: 10 lifecycle-worker processes
- After running `scripts/setup-launchd.sh`: legacy plists removed, process count drops to ≤7 (one per project)

## Low-level details
- **`launchd/ai.agento.lifecycle-all.plist.template`**: ThrottleInterval 30 → 60
- **`scripts/setup-launchd.sh`**: Added loop to boot out and delete any `com.agentorchestrator.lifecycle-*.plist` before installing the new plist
