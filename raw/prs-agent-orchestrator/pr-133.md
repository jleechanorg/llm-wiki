## Raw Body
### Background
`gh pr checkout` internally resolves the PR head ref via GraphQL. When 8+ AO worker sessions run simultaneously, this exhausts GraphQL quota causing **all backfill claims to fail**. The lifecycle-worker spawns a session, tries to claim the PR, `gh pr checkout` fails with "GraphQL: API rate limit already exceeded", the session is killed — and this repeats every 5-minute cycle for every uncovered PR, creating zombie sessions.

### Goals
- Eliminate GraphQL consumption from the `checkoutPR` path
- Allow backfill to work reliably even when GraphQL quota is low
- Handle edge cases: branch already exists locally, worktree conflicts

### Tenets
- Git protocol over API queries for checkout operations
- Fail loudly with clear error messages
- Fork isolation: change is in scm-github plugin, no core changes

### High-level description of changes
Replace `gh pr checkout` (GraphQL-dependent) with `git fetch <remote> <branch>:<branch>` + `git checkout <branch>` (git protocol, no API quota consumed). Includes fallback path when branch already exists locally (fetch into FETCH_HEAD + reset).
