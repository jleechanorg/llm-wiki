## Root Cause
When `ao spawn --agent gemini` runs from outside the monorepo, Node cannot resolve workspace-linked `@jleechanorg/ao-plugin-*` packages and throws `ERR_MODULE_NOT_FOUND`. Previously this was silently caught, so the gemini plugin was never registered and `ao spawn --agent gemini` failed with "Agent plugin 'gemini' not found".

## Fix
1. **Warn on failure**: `loadBuiltins()` now logs `[plugin-registry] failed to load builtin plugin '<pkg>': <err>` instead of silently swallowing errors.
2. **Monorepo fallback**: When the primary npm import fails, `tryMonorepoFallback()` navigates from `plugin-registry.js` up to the monorepo root, then into `packages/plugins/<name>/dist/index.js`, and uses `import(file://...)` to load the plugin directly.
