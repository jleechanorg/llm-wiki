## Raw Body
### Background
On 2026-03-24, stale lifecycle-workers from an abandoned ao-cursor-gemini-test build ran silently alongside canonical workers for 24+ hours. They used a different binary path so ao doctor was blind to them — it only matched workers by project name in the canonical config, not by binary path.

Result: 4 workers polling GraphQL simultaneously causing hourly 5000/hr quota exhaustion.

### Goals
- ao doctor catches any lifecycle-worker not using the canonical binary (~~/bin/ao)
- ao doctor --fix kills them automatically
- Total worker count sanity check flags unexpected accumulation

### Tenets
- Additive only — minimal change to existing check logic
- Fail-closed: warn AND offer --fix

### High-level description of changes
Extended check_lifecycle_workers() with two new checks:
1. Binary-path check: flag any worker whose command line does not contain $HOME/bin/ao
2. Total count check: warn if total workers > 3

--fix mode now kills non-canonical worker PIDs.

### Testing
- Ran ao doctor after fix: all checks PASS with current 2 canonical workers
- Binary-path detection verified against the exact ps output pattern from the incident

### Low-level details
- **scripts/ao-doctor.sh**: Extended check_lifecycle_workers() with binary-path and total-count checks; wired --fix to kill non-canonical PIDs
