## Security Fix

Both `ghRestFallback` and the `mergePR` REST fallback passed the GitHub token as a `-H 'Authorization: Bearer ...'` CLI argument to `curl`. This makes the token visible to any local user via `ps aux`.

### Fix
Write the auth header to a temporary curl config file (mode `0600`, owner-only) and pass `--config <path>` to curl instead. The file is cleaned up in a `finally` block so it is removed even if `curl` fails.

### Affected code sites
- `ghRestFallback()` (line ~511)
- `mergePR()` REST fallback (line ~1347)

### Risk
Low — the change only affects the mechanism by which the token reaches curl, not the API calls themselves. The curl config file format is well-documented and stable.
