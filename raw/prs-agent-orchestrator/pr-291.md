## Background

Skeptic infrastructure has been unreliable - skeptic-gate.yml polled for 28 minutes waiting for a VERDICT comment from the AO worker, but the lifecycle-worker was frequently down (launchd thrashing). This caused 10+ PRs in 24h of churn testing fixes in isolation.

User directed: make skeptic-gate pure deterministic, leave LLM inference to skeptic-cron only.

## Goals

1. Make skeptic-gate.yml a fast, deterministic 6-green check with always-on comment feedback
2. Run skeptic-cron every 20 min (was 30) with AO-powered LLM inference as the only merge path
3. Clean separation: gate = fast CI feedback, cron = LLM evaluation + auto-merge

## Tenets

- Always post a comment (no silent skips)
- Only skeptic-cron can auto-merge
- LLM inference runs via AO worker (no API keys in CI)
- Fail-closed on all gates

## High-level description of changes

**skeptic-gate.yml** - Complete rewrite: removed 28-min polling loop, now runs 5 deterministic gate checks and always posts a markdown table comment with per-gate PASS/FAIL. Exits 0 (PASS) or 1 (FAIL).

**skeptic-cron.yml** - Cron interval */30 to */20 minutes. Added Skeptic Gate CI pre-check before triggering LLM. AO worker chain preserved.
