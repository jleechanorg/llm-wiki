## Background
Workers stopped handling CR (CodeRabbit) CHANGES_REQUESTED comments on 4 PRs (#302, #301, #299, #273). Manual `ao review-check` was needed to detect. Root cause: lifecycle-worker death goes undetected, and `start-all.sh` idempotency had a pgrep false-positive.

## Goals
1. Restore lifecycle-worker monitoring so workers handle CR comments autonomously
2. Fix `start-all.sh` pgrep idempotency false-positive
3. Fix skeptic-gate.yml race condition (Gate 1 false FAIL when CI checks not yet registered)
4. Add durable documentation so agents detect and fix worker death without manual intervention

## Tenets
- Worker health is infrastructure — must be verifiable by any agent without deep knowledge
- Prevention > correction — catch the death condition before reactions stop
- No KeepAlive on daemons that exit code 0 on transient errors (causes launchd thrashing → deregistration)

## High-level description of changes
- **CLAUDE.md**: Added lifecycle-worker health section with check/fix commands; added liveness check to pre-spawn capacity gate
- **start-all.sh**: Fixed pgrep pattern from `pgrep -f "ao lifecycle-worker ${PROJECT}$"` to `pgrep -f "node.*ao lifecycle-worker ${PROJECT}$"` — the old pattern matched the bash -c wrapper script itself, not the node process, causing "already running — skipping" even when the worker was dead
- **launchd plist**: Removed `KeepAlive` (was causing launchd thrashing → deregistration when worker exits code 0); raised ThrottleInterval to 60
- **skeptic-gate.yml**: Added wait loop (up to 10 min, polling every 30s) for CI check runs to register and complete before evaluating Gate 1 — prevents false FAIL when gate fires before CI has registered its check runs for a new commit
- **memory**: Updated `feedback_lw_launchd_thrashing.md` with bd-rbgp fix details
