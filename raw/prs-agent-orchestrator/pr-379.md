## Raw Body
### Background

The  script previously used a fragile  regex to identify the active PR and repository for a given running session. This resulted in false-positives ('Zombie Sweeps') when stale terminal output matched the regex.

### Goals

Resolve false-positive worker terminations by directly querying the underlying git worktree and GitHub API instead of relying on regex matching over tmux pane contents.

### Testing

Ran `/opt/homebrew/bin/bash -c 'source scripts/ao-doctor-monitor.sh && check_zombie_sessions'` to confirm it triggers correctly without hanging or false alarms.

### Low-level details

Replaced `tmux capture-pane...` calls with a dynamic `get_session_pr_info` function that uses `branch --show-current` on the pane's canonical directory, followed by `gh pr list` to fetch the accurate, up-to-date PR number from the branch.
