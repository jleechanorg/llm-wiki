## Raw Body
## Background

Issue #419: `scripts/generate-pr-design-docs.mjs` generates two bugs that cause a permanent regenerate-loop on any PR using the design-doc workflow:

1. **Entity-encoded CURSOR_SUMMARY comment leaking into descriptions** — the PR body may contain `\u003c!-- CURSOR_SUMMARY --\u003e` or `&lt;!-- CURSOR_SUMMARY --&gt;` (HTML-entity or unicode-escaped forms of the comment marker). `extractDescription` only stripped the plain `<!-- ... -->` form.
2. **Workflow uses stale PR-branch script** — the workflow runs `scripts/generate-pr-design-docs.mjs` from the PR's own branch. PRs created before the last script fix carry the old buggy script and regenerate broken docs on every push, making manual fixes futile.

## Goals

- Strip all encoded variants of `<!-- CURSOR_SUMMARY -->` from PR descriptions before rendering
- Ensure the workflow always uses the fixed script from the base branch (main) on every run

## Tenets

- Minimal changes, no refactoring beyond what's needed
- No breaking changes to the script interface

## Changes

### `scripts/generate-pr-design-docs.mjs`

`extractDescription`: added two new `.replace()` calls to strip HTML-entity-encoded (`&lt;!-- CURSOR_SUMMARY --&gt;`) and unicode-escaped (`\u003c!-- CURSOR_SUMMARY --\u003e`) comment markers before the general HTML-comment strip.

### `.github/workflows/generate-pr-design-docs.yml`

Added `Sync generator script from base branch` step between checkout and Node.js setup. This fetches the base branch (main) and overwrites the PR-branch copy of `scripts/generate-pr-design-docs.mjs` before running. Even PRs with stale script copies in their branch now use the latest fixed version on every push — breaking the regenerate-loop cycle without requiring a rebase.

- `node scripts/generate-pr-design-docs.mjs --single-pr=245 --force --dry-run` locally confirms the script runs without errors
- Workflow change is a step addition; verified YAML syntax
- No changes to the idempotency or output form
