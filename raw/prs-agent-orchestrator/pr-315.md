## Background

PR #302 (message-content hash dedup) was branched before PR #273 (self-hosted runners with fork protection) merged. When #302 was squash-merged, it silently reverted all security protections from #273 across 6 workflow files.

## Goals

Restore the fork PR security protections and ARM64 gitleaks checksum support that were inadvertently removed.

## High-level description

### 1. Fork-aware `runs-on` expressions (security-critical)

Fork PRs must run on `ubuntu-latest`, not self-hosted runners. The expression `github.event.pull_request.head.repo.fork && github.event_name == 'pull_request'` gates this. Without it, a malicious fork PR could execute arbitrary code on self-hosted infrastructure.

Restored in: `ci.yml` (4 jobs), `security.yml` (3 jobs), `integration-tests.yml`, `onboarding-test.yml`, `evidence-gate.yml`

### 2. Per-architecture gitleaks checksum

`security.yml` had the per-arch SHA256 `case` statement (x64/arm64) replaced with a hardcoded x64 checksum. If runners are ARM64, gitleaks would either fail to download the correct binary or run an unverified one. Restored dynamic architecture detection.

### 3. Self-hosted runner labels

`release.yml` jobs used stale `self-hosted-mikey` labels. Updated to canonical `Linux,ARM64,agent-orchestrator` labels.
