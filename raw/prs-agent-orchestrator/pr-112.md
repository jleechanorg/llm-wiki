## Summary
Without worktree locking, git worktree prune can silently delete a live AO workspace if its directory is removed while the session is still active. AOs destroy() already called git worktree remove --force, but unlocked worktrees are also vulnerable to accidental deletion by git pull, git gc, or manual git worktree prune runs.

Root cause: workspace-worktree plugin called git worktree add but never followed up with git worktree lock.

## Raw Body
## Background

Without worktree locking, git worktree prune can silently delete a live AO workspace if its directory is removed while the session is still active. AOs destroy() already called git worktree remove --force, but unlocked worktrees are also vulnerable to accidental deletion by git pull, git gc, or manual git worktree prune runs.

Root cause: workspace-worktree plugin called git worktree add but never followed up with git worktree lock.

Closes bd-diq

## Goals
- Prevent git worktree prune from silently deleting AO-managed worktrees while sessions are active
- Ensure destroy() can still forcefully remove locked worktrees during intentional teardown

## Tenets
- Non-fatal for older git versions that may not support worktree lock
- destroy() must always succeed -- use --force --force to bypass lock
- Minimal scope: only create(), restore(), destroy()

## High-level description of changes
- create(): after git worktree add, call git worktree lock --reason AO session active <path>
- restore(): same -- lock after restore completes
- destroy(): change to git worktree remove --force --force to bypass lock during intentional teardown

The lock call is wrapped in try/catch (non-fatal) so sessions continue to work on older git installations that lack the worktree lock subcommand.

## Testing
- All 45 existing unit tests pass (1 test updated: destroy() assertion now expects double --force)
- Lock call is non-fatal (try/catch) so no additional mock infrastructure needed; existing tests cover the happy path

## Low-level details (2 files)
- packages/plugins/workspace-worktree/src/index.ts: Add lock call after every git worktree add in create() and restore(); update destroy() to use --force --force to bypass lock
- packages/plugins/workspace-worktree/src/__tests__/index.test.ts: Update destroy() test assertion to expect [worktree, remove, --force, --force, ...]
