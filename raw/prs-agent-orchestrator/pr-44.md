## Background
ORCH-6ae: The notifier-openclaw plugin exists but was never wired into the canonical config example or tested as a config-to-plugin path. AO escalation events (agent-stuck, ci-failed) could not reach OpenClaw without manually editing the yaml.

## Goals
- defaults.notifiers includes openclaw and notifiers.openclaw is documented in the example config
- Plugin-registry can resolve and instantiate openclaw from config
- OpenClaw server-side requirements (hooks.enabled, allowRequestSessionKey, allowedSessionKeyPrefixes) are documented
- OPENCLAW_HOOKS_TOKEN env var is documented as the token source

## Tenets
- TDD: write failing tests first, then make them pass
- No changes to plugin source or config schema — they were already correct
- Minimal scope: only add tests and update the yaml example

## High-level description of changes
Added 4 integration tests verifying the config -> plugin-registry -> openclaw notifier path. Updated agent-orchestrator.yaml.example to include openclaw in defaults.notifiers, a full notifiers.openclaw section, and notificationRouting routing urgent+action to [desktop, openclaw]. Added a comment block documenting the required OpenClaw server settings.
