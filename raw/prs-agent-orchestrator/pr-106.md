## Raw Body
### Background
AO needs a transport layer for inter-agent messaging via MCP Agent Mail. Currently notifications only go to Slack or webhooks. This PR adds a dedicated notifier plugin that can route AO session lifecycle events to the MCP mail system, enabling orchestrator-to-agent and agent-to-agent messaging.

### Goals
- Provide a `mcp-mail` notifier plugin that sends AO events as MCP Agent Mail messages
- Auto-register the AO session as a named agent on first notification
- Support `notify`, `notifyWithActions`, and `post` methods from the `Notifier` interface
- Be a pure transport layer — message content is composed by `agentRules` in config

### Tenets
- Plugin handles mechanical plumbing only; agents compose their own messages via config
- Strict TypeScript (no `any`, no `@ts-ignore`)
- TDD: tests written before implementation
- Follow existing notifier plugin conventions (notifier-slack, notifier-webhook)

### High-level description of changes
New package `@jleechanorg/ao-plugin-notifier-mcp-mail` under `packages/plugins/notifier-mcp-mail/`:

- **Auto-registration**: On first `notify()` call, POSTs to `register_agent` endpoint — one-time, idempotent per notifier instance
- **State notifications**: `notify()` and `notifyWithActions()` POST to `send_message` with structured subject/body including event type, session, project, CI status, PR URL
- **Free-form messaging**: `post()` supports free-form messages with optional `to` recipient (from `context.channel`)
- **Config**: accepts `endpoint`, `agentId`, `projectId`; falls back to `MCP_AGENT_MAIL_URL` env var when no endpoint set
- Registered in `plugin-registry.ts` as `notifier:mcp-mail`
