## Background
AO spawns can fail with git exit 128 when a target branch is still registered as checked out in an abandoned worktree. The current worktree plugin retries checkout for existing branches, but does not recover stale worktrees that no longer have an active tmux session.

## Goals
- Detect stale checked-out worktree conflicts during spawn-time branch checkout
- Remove stale conflicting worktree entries automatically when safe
- Retry checkout so spawn succeeds without manual cleanup

## Tenets
- Keep recovery conservative: only clean up when no active tmux session matches the worktree name
- Preserve existing cleanup behavior for genuine checkout failures
- Keep logic local to workspace-worktree plugin where checkout happens

## High-level description
- Added stale-worktree recovery helpers to parse checkout errors, inspect tmux session names, and force-remove stale worktrees
- In the branch-exists checkout path, on "already checked out at" errors we now attempt stale-worktree cleanup and retry checkout once
- Added regression tests covering automatic stale cleanup + retry flow
