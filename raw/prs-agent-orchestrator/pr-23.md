## Background
Part of bd-pdt epic — implementing jleechanclaw orchestration runtime logic in AO. The `ReactionConfig` type already defines `failureBudget` and `onBudgetExhausted` fields but no runtime code tracks budgets or routes exhausted budgets.

## Goals
- Track retry failures per session+reaction pair with configurable max and time window
- Route exhausted budgets to escalate/disable/notify/route-to actions
- Enable lifecycle-manager integration for automatic failure budget enforcement

## Tenets
- TDD: tests written first, then implementation
- Clean separation: tracker is a standalone class, routing is a pure function
- No changes to existing code beyond exports

## High-level description of changes
New `FailureBudgetTracker` class and `routeExhaustedBudget` function. The tracker maintains per-session per-reaction failure counts with optional time windows. When budget is exhausted, the routing function dispatches to one of 4 strategies based on config.
