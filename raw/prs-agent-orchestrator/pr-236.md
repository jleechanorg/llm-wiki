## Changes

### `.github/workflows/skeptic-cron.yml`
Gate 5 was using REST `/pulls/{n}/comments` which has **no `isResolved` field** — GitHub only exposes thread resolution state via GraphQL `reviewThreads.nodes[].isResolved`. The old code was counting all top-level non-nit comments regardless of resolution, causing false negatives.

**Fix**: Switch to GraphQL `reviewThreads` query that returns thread-level `isResolved`. Also exclude PR author comments (they can always resolve their own threads and those don't block CR approval).

### `packages/core/src/merge-gate.ts`
Already using `isResolved` from GraphQL (correct), but also needed to exclude the PR author's own comments from the blocking count.

**Fix**: Add `c.author !== pr.author` to the filter, consistent with CR's behavior. Also explicitly typed `pendingComments: ReviewComment[]` to ensure TypeScript resolves the `author` field.
