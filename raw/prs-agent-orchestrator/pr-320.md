## Background
`runLocalSkepticCron` evaluates every open PR on every 10-min cycle even if VERDICT was already posted for the current HEAD SHA. This wastes LLM calls and creates duplicate comments.

## Goals
- Skip skeptic re-evaluation when HEAD SHA hasn't changed since the last successful verdict
- Fail open when `getPRHeadSha` is unavailable or throws (evaluate normally)
- Maintain independent dedup state per project

## Tenets
- Fail open: if SHA cannot be determined, evaluate rather than skip
- Cache SHA on PASS and FAIL verdicts; only uncaught throws skip caching

## High-level description of changes
Added a module-level `lastEvaluatedShaByPR` map (keyed by projectId:prNumber) to skeptic-cron-local.ts. Before calling runSkepticReview, the code fetches the current HEAD SHA via scm.getPRHeadSha (optional chaining, fail-open) and skips if it matches the cached SHA. Logs `skeptic.cron.sha_dedup_skip` when dedup fires. The map is bounded (10,000 entries) to prevent unbounded growth in long-running lifecycle workers.
