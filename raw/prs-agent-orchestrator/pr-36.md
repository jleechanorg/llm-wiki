## Background
The ao package was renamed from packages/agent-orchestrator to packages/ao, but ao-doctor.sh check_runtime_sanity() still hardcoded the old path. This caused ao doctor to always report launcher entrypoint is missing, which cascaded into the health monitor reporting false RED status.

## Goals
- Fix ao doctor to find the launcher under either the current or legacy package name
- Eliminate false RED from the health monitor

## High-level description of changes
check_runtime_sanity() now checks packages/ao/bin/ao.js first, falls back to packages/agent-orchestrator/bin/ao.js, and uses whichever is found.

## Low-level details
- **scripts/ao-doctor.sh**: Changed check_runtime_sanity() from hardcoded path to a local variable that checks both paths via if/elif.
