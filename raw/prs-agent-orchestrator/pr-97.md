## Summary

Adds a deterministic, fail-closed harness for PR worker coverage reconciliation:

- **`CLAUDE.md`**: Documents the mandatory recovery command (`ao spawn --claim-pr <PR>`), PR coverage reconciliation procedure, validation script usage, and EPIPE handling patterns.
- **`scripts/check-pr-worker-coverage.sh`**: Reports PR→session mapping, exits 0 when all open PRs have active sessions, exits non-zero with uncovered PRs listed.

## Motivation

When sessions are dispatched to repair uncovered PRs, bead/issue spawn can create orphaned or duplicate sessions. The harness enforces explicit 1:1 PR→session mapping and a validation checkpoint after every coverage repair operation.

## PR Coverage Reconciliation Procedure

1. `gh pr list --repo jleechanorg/agent-orchestrator --state open --limit 100 --json number,title,headRefName`
2. `ao session ls --project agent-orchestrator 2>/dev/null`
3. Cross-reference open PRs against active sessions by branch
4. Dispatch repair sessions: `ao spawn --project agent-orchestrator --claim-pr <PR_NUMBER>`
5. Validate: `./scripts/check-pr-worker-coverage.sh && echo $?  # must be 0`
