## Background

On 2026-04-04, **8 PRs** (363, 366-372) were created for the same file (`.claude/metadata-updater.sh`) in under 1 hour. 7 were wasted duplicates — all CONFLICTING after PR #367 merged first. Root cause: AO workers had no file-level coordination gate at PR creation time.

Per harness severity table: "repeated manual fix" requires a **hook**, not just instructions.

## Goals

- Prevent multiple workers from creating PRs that modify the same files
- Fail-open: never block legitimate work if checks fail
- Allow intentional supersession via "Supersedes #N" in PR body
- Document and register `lw-watchdog.sh` (lifecycle-worker watchdog daemon) included in this PR scope

## Tenets

- Hooks > instructions for preventing churn (per harness severity matrix)
- Fail-open over fail-closed for developer productivity
- Actionable error messages that tell the agent exactly what to do

## High-level description of changes

Added `.claude/hooks/churn-guard.sh` — a PreToolUse hook that:
1. Intercepts `gh pr create` and `gh api .../pulls --method POST` commands
2. Determines the repo (from `--repo` flag, REST URL, or git remote)
3. Gets files changed in current branch vs `origin/main`
4. Queries all open PRs changed files via GitHub REST API
5. If any open PR touches the same files: BLOCK with list of overlapping PRs
6. Override: if PR body contains "Supersedes #N", allows creation

Also registered in `~/.claude/settings.json` PreToolUse hooks for immediate effect.
