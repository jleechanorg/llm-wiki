## Background

Fixes the novel-daily launchd job (`scripts/novel/run-daily.sh`) to:
1. Resolve install-time hardcoded `--daily` date (uses runtime date wrapper)
2. Commit and push generated daily entries to `origin/main` automatically
3. Move `git fetch + ff-only merge` BEFORE file generation (CR feedback) to prevent conflicts

Supersedes PR #354.

## Goals

- Each daily run writes to correct `novel/workers/{YYYY-MM-DD}.md`
- launchd job commits/pushes automatically to `origin/main`
- Idempotent: skipped cleanly when nothing changed
- Handles concurrent pushes safely (ff-only guard + push_with_retry)

## Tenets

- No breaking changes to existing scripts
- Shell script only (no new dependencies)

## Changes

| Change | Rationale |
|--------|-----------|
| Runtime date via wrapper | Each launchd run writes correct date file |
| Auto commit/push | Daily entries pushed to origin/main automatically |
| `git -c user.name=...` for identity | Avoids repo-wide git config mutation |
| Pre-generation fetch + ff-only | Prevents conflicts if origin advanced before run |
| push_with_retry | Handles non-FF rejections; detects true non-FF; terminal failures reset to origin/main |
| mkdir -p before generation | Ensures parent directory exists on first run / fresh clone |
| FAIL-hard when DAILY_FILE missing | Prevents silent no-op on generation failure |
| Dual-file idempotency check | Detects changes in both daily and workers files |
| Evidence-gate HTML comment strip | bd-ara.3 fix — gsub markers, don't skip lines |
| Wholesome caption filter | awk sub() strips label from inline label+URL lines |
