## Background

During a monitoring session (2026-03-22), a bulk worktree cleanup script iterated all registered git worktrees in the agent-orchestrator repo without scoping to AO session names. It deleted `worktree_worker5`, `worktree_worker2`, `worktree_agentog2`, `worktree_pr82`, and `worktree_pr98` — all active Claude Code session directories. All shell commands in the affected sessions broke instantly (`CWD no longer exists`). The session was unrecoverable without manual intervention.

## Goals

1. Make it impossible for any agent to run `git worktree remove` or `git worktree prune` — hard block at tool level
2. Document the required loop-guard pattern for any future worktree cleanup scripts
3. Ensure the protection survives across sessions via both hook and CLAUDE.md instruction

## Tenets

- Fail closed: block unconditionally, require human to run in terminal
- No false positives: commit messages or grep output mentioning the command must not trigger the block
- Minimal scope: two files only — hook + instructions

## High-level description of changes

**`.claude/hooks/protect-worktrees.sh`** — new PreToolUse hook wired into `~/.claude/settings.json`. Reads tool input JSON from stdin, extracts the Bash command, strips heredocs and quoted strings via Python, then checks if `git worktree (remove|prune)` appears as an actual command token (not inside a string). Exits 2 on match (Claude Code hard-blocks on exit 2). Tested against three cases: actual invocation (blocked), commit message containing the phrase (allowed), and prune after `cd &&` (blocked).

**`CLAUDE.md`** — adds "Worktree protection — ABSOLUTE RULE" section with the required basename guard for any cleanup loop:
```bash
if [[ ! "$session_name" =~ ^(ao|jc|wa|cc|ra|wc)-[0-9]+$ ]]; then
  echo "SKIP (non-session worktree): $session_name"
  continue
fi
```
