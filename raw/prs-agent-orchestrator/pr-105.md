## Raw Body
## Background

AO workers spawned with bead IDs (bd-xxx) received only `Work on issue: bd-xxx` with zero task context. There was no tracker plugin for beads — only GitHub, Linear, and GitLab trackers existed. This caused workers (ao-415, ao-417) to think for hours or produce nothing because they literally did not know what to build.

## Goals

- Workers spawned with bead IDs get full task context: title, description, priority, type, dependencies, and parent epic
- Zero additional API calls (reads local .beads/issues.jsonl)
- Follow existing plugin patterns (tracker-github as template)

## Tenets

- Plugin-first: new capability lives entirely in `packages/plugins/tracker-beads`
- Minimal core touch: only 1 line added to plugin-registry.ts
- Config-level activation: `tracker: { plugin: beads }` in yaml

## High-level description of changes

New `tracker-beads` plugin that reads `.beads/issues.jsonl`, resolves bead IDs to full context, and generates rich prompts including parent epic context. Registered in plugin-registry.

## Testing

- 9 unit tests covering: getIssue, isCompleted, branchName, generatePrompt (with/without parent epic, missing bead), listIssues (filter by state), missing .beads directory
- 712 core tests pass
- Manual smoke test with real beads (bd-q3x, bd-v92, bd-ggf)
- Build + typecheck pass

- **packages/plugins/tracker-beads/package.json**: Standard plugin package config.
