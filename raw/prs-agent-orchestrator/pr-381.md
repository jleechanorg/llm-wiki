## Background

The evolve loop burned ~3 hours of context on 2026-04-04 with 17+ no-op cycles after the PR pipeline cleared to 0 open PRs. No auto-exit condition existed.

## Goals

- Prevent future idle-cycle burns by auto-canceling the loop after 3 consecutive idle cycles
- Add a RECAP phase to emit a brief cycle summary (zero-touch rate, worker count, open PRs, findings)

## Tenets

- Minimal scope — only adds text to the generated orchestrator prompt, no behavioral code
- No breaking changes to existing phases

## High-level description of changes

Added Phase 7 (RECAP) and Phase 8 (AUTO-CANCEL) to `generateEvolveLoopSection()` in `packages/core/src/orchestrator-prompt.ts`. Phase 7 emits a brief cycle summary. Phase 8 persists an idle-cycle counter across loop invocations via a counter file; if the counter reaches 3 consecutive idle cycles (0 open PRs + 0 friction + all workers alive), it prints `AUTO-CANCEL: 3 consecutive idle cycles — eloop pausing` and exits cleanly.

Also updated `packages/core/src/__tests__/evolve-loop-config.test.ts` with new tests for Phase 7 and Phase 8 content.

**Note:** PR #380 (`feat/bd-l5ko`) also modifies `orchestrator-prompt.ts` (adds healthy-cycle fast path to MEASURE phase). This supersedes #380 and consolidates both changes into one PR covering all 8 phases.
