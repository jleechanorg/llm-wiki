## Background

The `metadata-updater.sh` PostToolUse/PreToolUse hook runs on every Bash invocation in Claude Code sessions. Five bugs prevent it from working correctly in AO worktrees on macOS (BSD sed) and in non-git contexts.

## Goals

- Fix "illegal option -- -" sed error on macOS
- Fix over-escaped `|` in BRE sed pattern
- Fix silent hook failure when `git show-ref` is unavailable
- Fix infinite loop on bare env assignment (no trailing command)
- Fix shellcheck SC2155 (declare+assign on same line) and SC2001 (use `${var//}` instead of sed)

## High-level description

Five targeted changes to `.claude/metadata-updater.sh`:

1. **Env-stripping regex fix**: macOS BSD `sed` does not support POSIX character class syntax inside `[...]`. Changed env-assignment stripping to use bash `${clean_command#*=* }` which correctly handles `FOO=a=b` (embedded `=`) without consuming the first `=`.

2. **Infinite loop prevention**: Bare `FOO=bar` (no trailing command) matched the env regex but had no space for `${clean_command#*=* }` to strip → infinite loop. Fixed by requiring `[[:space:]]` after `=` in the regex guard.

3. **`sed 's/[&|\\]/\\&/g'` → `sed 's/[&\\]/\\&/g'`**: In BRE (basic sed), `|` is not an OR metacharacter — it's a literal character. Since `|` is not the sed delimiter, escaping it was unnecessary and visually confusing. Removed.

4. **Removed `git show-ref` guard**: `git show-ref --verify --quiet "refs/heads/$branch"` silently fails (exit 1) when the hook runs from a context without full git repo access. This caused the entire branch-detection block to exit without writing metadata. Replaced with an extension filter (rejecting `.ts`, `.md`, etc.) to prevent `git checkout docs/README.md` false-matches.

5. **Shellcheck SC2155/SC2001**: `sed`-based value escaping masked return values. Replaced with bash `${value//\\/\\\\}` and `${value//&/\\&}` parameter expansion. Also separated `local` declaration from assignment to fix SC2155.
