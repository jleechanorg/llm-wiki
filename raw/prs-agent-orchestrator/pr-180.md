## Raw Body
### Background
AO workers were dying immediately after spawn because the lifecycle-worker polled sessions before the agent CLI had fully initialized. The runtime/activity probes returned "exited"/"idle" during the startup window, triggering premature session kills. This was a P0 race condition (bd-85r).

### Goals
- Prevent lifecycle-worker from killing sessions that are still starting up
- Add a configurable grace period (default: 120s) during which probes are skipped
- Protect against all kill paths: determineStatus(), session-reaper, post-merge reaper

### Tenets
- Minimal core diff -- additive guard, no refactoring
- Backward compatible -- grace period defaults to 120s, existing tests use 0 to preserve behavior
- Fork isolation -- new config field, no upstream file restructuring

### High-level description of changes
Added startupGracePeriodMs config field (Zod schema + TypeScript type, default 120_000ms). Three guard points:

1. **determineStatus()** -- early return spawning/working when sessionAgeMs < startupGracePeriodMs, skipping all liveness/activity probes
2. **session-reaper** -- skip sessions within grace period even if they meet reap conditions
3. **fork-lifecycle-postmerge** -- passes startupGracePeriodMs: 120_000 to the post-merge reaper
