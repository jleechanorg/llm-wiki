## Summary

Add durable repo-level prevention (bd-ara.stale) for recurring blocked/stale PR patterns — addresses `orch-harness-blockers`.

## Changes

- **ao-doctor-monitor.sh**: new `check_pr_age()` — fetches `createdAt` for all open PRs per repo, displays age in hours (terminal + Slack digest), flags >3h as STALE, hard-FAILs if `createdAt` field is missing (mechanical guardrail)
- **check-pr-worker-coverage.sh**: show age on every PR, flag stale uncovered PRs (exit 1), guardrail-fail if `createdAt` missing (exit 2)
- **skeptic-cron.yml**: log PR age and staleness on every 30-min cycle for audit trail
- **roadmap/zero-touch-6green-rate.md**: document staleness triage policy + env vars

## Triage Policy

| Concern | Threshold | Action |
|---|---|---|
| Fresh PR | < 3h | No flag |
| Stale PR | ≥ 3h | WARN in monitor + coverage script |
| Stale + uncovered | ≥ 3h, no session | FAIL coverage; trigger respawn |
| Missing `createdAt` | — | Hard block: FAIL/block until fixed |

## Env vars

- `AO_DOCTOR_STALE_HOURS` (default 3) — threshold in ao-doctor-monitor.sh
- `STALE_HOURS` (default 3) — threshold in check-pr-worker-coverage.sh
