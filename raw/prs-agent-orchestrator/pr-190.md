Fixes Bugbot finding on PR #186: removing `trim()` from `fixLine()` to preserve intentional line indentation leaves single artifact spaces when filler words are removed from the start or end of a line. The `\s{2,}` regex only collapses runs of 2+ spaces, so a single leftover space after filler removal survives.

### Root cause

`"Just a note"` â†’ filler replaced with `""` â†’ `" a note"` â†’ `\s{2,}` no-op (only 1 space) â†’ `trim()` removed intentionally to preserve indent â†’ artifact `" a note"` written to file.

### Fix

Split leading indentation (`^\s+`) from content, apply filler removal + `\s{2,}` collapse to content, `trim()` content (removes artifact trailing space), `ltrim()` content (removes artifact leading space), restore original indentation.
