## Background

## Goals
- Fix the skeptic-cron workflow so it runs successfully
- Add actionlint to PR checks as a preventive measure

## Root Cause
`actions/setup-node@v4` with `cache: pnpm` requires the pnpm binary to be present to populate its cache. However, pnpm is installed via `pnpm/action-setup@v4` in the **next step**. This circular dependency causes `setup-node` to fail immediately.

## Fix
1. **skeptic-cron.yml**: Removed `cache: pnpm` from the `Install Node.js` step. The separate `Install pnpm` step handles it correctly.
2. **wholesome.yml**: Added `actionlint` job to lint all workflow files on PRs, catching step-ordering and other YAML misconfigurations before merge.
