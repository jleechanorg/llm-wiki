> [!NOTE]
> **Medium Risk**
> Adds a second pruning pass that enumerates and force-removes git worktrees based on session metadata and terminal statuses; bugs in path matching or status detection could delete unintended directories despite safeguards.
> 
> **Overview**
> **`pruneStaleWorktrees` now runs in two passes.** Pass 1 keeps the existing `~/.worktrees/{projectId}` scan but skips it when `~/.worktrees` doesn’t exist and simplifies `git worktree remove` to a single `--force`.
> 
> **New Pass 2** calls `git worktree list --porcelain` per configured project, identifies worktrees *outside* `~/.worktrees/`, and removes only those that match an AO session record whose `status` is in `TERMINAL_STATUSES` (fallbacking to `rmSync` on failure). `createSessionManager` also adds injectable `readMetadataRaw`/`listMetadata` deps to isolate metadata IO in tests.
> 
> Tests expand `session-manager-prune-stale-worktrees.test.ts` with coverage for Pass 2 zombie removal, alive-session preservation, and ensuring unrelated projects/worktrees are not targeted.
