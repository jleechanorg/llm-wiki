## Background
The `send-to-agent` reaction fires every poll cycle for unchanged PR state, sending identical CR feedback to workers 5-9x and burning context window tokens. The reaction system lacked message-content deduplication to skip sends when the rendered message (with `{{context}}` resolved) is unchanged.

## Goals
- Track the last sent message content hash per session+reaction
- Skip `send-to-agent` when the rendered message content matches the last sent hash
- Reduce context waste from duplicate CR feedback delivery
- Prevent cross-reaction hash collisions (different reaction types for the same session)

## High-level description of changes
Added `dedup-head-sha-store.ts` as an in-memory store with two deduplication mechanisms:
1. **Message content hash** (`messageHashStore`): SHA256 of the fully rendered message (with `{{context}}` resolved), keyed by `sessionId:reactionKey`. This is the primary dedup signal.
2. **Head SHA tracking** (`headShaStore`): Kept for backwards compatibility with existing tests.

Modified `lifecycle-manager.ts` to build `buildReactionContext` once per poll cycle (not twice) and use it for both the dedup check and the actual send. The message hash is recorded after successful sends. Escalation checks still run even when the send is deduped.
