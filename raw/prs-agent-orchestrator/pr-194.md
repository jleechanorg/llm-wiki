## Background
When `getReviews` or `getReviewDecision` is called and GraphQL is rate-limited, the function throws with a rate-limit error rather than falling back to REST. This causes lifecycle-worker to lose review state during GraphQL exhaustion windows.

bd-yo1 adds REST fallbacks for both functions using `gh api repos/.../pulls/.../reviews`.

## Goals
- `getReviews` falls back to REST when GraphQL rate-limited
- `getReviewDecision` falls back to REST when GraphQL rate-limited
- Tests cover the fallback behavior

## Tenets
- REST-first for high-frequency calls
- Fail-safe: never let rate-limit stall lifecycle-worker

## High-level description of changes
Added REST fallback paths to `getReviews` and `getReviewDecision` in `packages/plugins/scm-github/src/index.ts`. When GraphQL throws a rate-limit error, falls back to `gh api repos/OWNER/REPO/pulls/NUM/reviews` (REST endpoint) to retrieve review data.

## Testing
- Unit tests added for REST fallback path
- Existing tests unchanged
