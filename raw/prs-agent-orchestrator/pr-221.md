## Background
The runtime-antigravity plugin had numerous mismatches with the actual Peekaboo CLI interface. Commands used wrong flags, output parsing ignored the envelope format, and type definitions had fields that do not exist in the real CLI.

## Goals
- Align all Peekaboo CLI invocations with the real CLI flags and argument order
- Fix output parsing to unwrap the envelope format
- Update type definitions to match actual Peekaboo response shapes
- Add missing hotkey() and scroll() commands

## Tenets
- Match the real CLI exactly
- Update all tests to use correct fixtures

## High-level description of changes

### CLI flag fixes (peekaboo.ts)
- windowList(): added windows subcommand, unwraps {data: {windows}} envelope
- see(): unwraps {success, data} envelope
- click(): --element-id to --on, --snapshot-id to --snapshot
- press(): key is now positional before --app
- Added hotkey() for key combinations (e.g. cmd+w)
- Added scroll() for scrolling

### Type fixes (types.ts)
- PeekabooWindow: removed app, added isOnScreen and isMinimized
- PeekabooUIElement: removed value and bounds, added label, description, role_description, is_actionable
- Added PeekabooListResponse and PeekabooSeeResponse envelope types

### Source file updates
- runtime.ts: el.value to el.label, press(Command+w) to hotkey(cmd+w), added textArea role
- poller.ts: el.value to el.label
- mcp-tools.ts: conversation detection uses el.label ?? el.title

## Testing
- pnpm test -- 105 tests pass (9 test files)
- pnpm build -- compiles cleanly

- **src/types.ts**: Removed app from PeekabooWindow, replaced value/bounds on PeekabooUIElement with label/description/role_description/is_actionable, added envelope types

- **src/runtime.ts**: Updated el.value to el.label, replaced press with hotkey in destroy(), added textArea role
