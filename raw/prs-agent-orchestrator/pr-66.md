## Root cause
`auto-merge` reactions were a valid config option but AO intentionally does not auto-merge by default. The merge guardrail is enforced at the CLI level (`gh pr merge` blocked by default via metadata hook + codex wrapper). Users who configure `auto-merge` reactions may not realize their intent will be silently blocked.

## Tenets
- Agent safety: prefer runtime denial over silent inaction
- Fail loudly: validate at config load time, not at merge time
