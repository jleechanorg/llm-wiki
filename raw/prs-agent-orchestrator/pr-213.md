## Background

Skeptic agent (bd-qw6, PR #206) already posts VERDICT comments on PRs, but:
1. The skeptic evaluation only runs when the evolve loop routes to a PR — PRs that skip the evolve loop get no skeptic review
2. Skeptic verdicts are not enforced as a required CI check — workers can merge without skeptic approval

## Goals

1. Make skeptic review universal — every PR gets skeptic evaluation, even if it skips the evolve loop
2. Enforce skeptic PASS as a required CI gate — PRs cannot merge without skeptic approval
3. Drive PRs to 7-green autonomously via a 30-minute cron loop that merges when all conditions are met

## Tenets

- Skeptic verdict PASS is a hard blocker — CI job fails closed if verdict is FAIL or UNKNOWN
- Cron merge is conservative — requires all 7 conditions to pass before merging
- Fail-open on optional conditions (evidence review) — skeptic verdict is the authoritative call

## High-level description of changes

Two new GitHub Actions workflows:

**`.github/workflows/skeptic-gate.yml`** (required CI check):
- Triggers on `pull_request` (opened, synchronize, edited, reopened)
- Installs Claude CLI, builds CLI, runs `pnpm ao skeptic --pr <number>`
- Parses `VERDICT: PASS/FAIL` from output to set job conclusion
- Posts skeptic findings as a PR comment via `--body-file` (YAML-safe heredoc)
- `VERDICT: FAIL` → job fails, GitHub branch protection blocks merge

**`.github/workflows/skeptic-cron.yml`** (30-min autonomous merge loop):
- Scheduled `*/30 * * * *` + manual `workflow_dispatch` trigger
- Fetches all open non-draft PRs
- Runs `pnpm ao skeptic` on each PR (posts VERDICT comment)
- Checks all 7-green conditions via GitHub REST API: CI green (exact success), no conflicts, CR APPROVED (uppercase), Bugbot clean, comments resolved, evidence review, skeptic PASS
- Missing skeptic verdict blocks merge (must explicitly PASS)
- Posts findings as PR comments
- Merges via `gh pr merge --squash --auto --delete-branch` when all 7 pass
- Logs merged count
