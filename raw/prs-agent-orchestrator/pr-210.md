## Background

When CR posts `CHANGES_REQUESTED` and the assigned worker is dead/exhausted, the `send-to-agent` reaction fires into the void and the PR becomes orphaned. PRs #202 and #207 were both stuck in this state.

## Goals

- Eliminate orphaned PRs when worker is dead and CR posts feedback
- Auto-spawn a fresh worker targeting the same PR branch with pre-loaded review context

## Tenets

- Prefer existing infrastructure (sessionManager.spawn) over new code
- Fail gracefully: if spawn fails, retry via backlog on next cycle

## High-level description of changes

Adds `respawn-for-review` reaction action that, on a `changes_requested` transition with a dead agent, calls `sessionManager.spawn` targeting `session.pr.branch` to reclaim the worktree. Pending review comments are pre-loaded into the spawn prompt via `buildReactionContext`. A verdict gate in the transition path suppresses the reaction when CR's latest verdict is COMMENTED/DISMISSED.
