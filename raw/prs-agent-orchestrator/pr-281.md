## Background

All 39 PRs merged in the last 7 days via skeptic-cron show merged_by=null. This breaks zero-touch auto-merge rate tracking, which is measured by counting PRs with merged_by=github-actions[bot].

## Root Cause

gh pr merge --squash --auto uses GitHub merge queue (auto-merge feature). GitHub attributes merge queue merges to its internal system, not the GITHUB_TOKEN actor, hence merged_by=null.

## Fix

Replace --squash --auto with --squash --admin in all merge paths:

- .github/workflows/skeptic-cron.yml (live GHA)
- packages/cli/src/templates/skeptic/skeptic-cron.yml (CLI template)
- scripts/claude-dashboard (merge API endpoint)
- packages/cli/scripts/claude-dashboard (copy)

--admin bypasses the merge queue and merges immediately as GITHUB_TOKEN, setting merged_by to github-actions[bot].
