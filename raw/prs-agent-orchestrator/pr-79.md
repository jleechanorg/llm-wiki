## Background
PR #75 introduced REST-based PR detection as a GraphQL rate-limit mitigation. However, the REST filter is owner-scoped and can miss PRs opened from fork head branches.

## Goals
1. Fix PR detection for fork head branches
2. Keep GraphQL cost minimal in steady-state poll loops
3. Add regression tests

## Changes
detectPR now uses gh pr list --head (GraphQL-backed) as primary — finds PRs regardless of fork owner. REST is the fallback only when GraphQL fails. This corrects the order from PR #75.
