## Summary

- Fix `extractDescription()` in `scripts/generate-pr-design-docs.mjs` to strip the **full** `CURSOR_SUMMARY` block (opening marker + content + closing marker) in one pass
- This prevents the closing tag from being orphaned as visible text in generated `.html` design docs
- Keep the fallback HTML comment stripper non-greedy per-comment using `[\s\S]*?` with global replacement

## Root cause

The previous patterns stripped individual comment markers (`<!-- CURSOR_SUMMARY -->`) and their entity-encoded variants.  For PR bodies containing a **paired** CURSOR_SUMMARY block:

```
<!-- CURSOR_SUMMARY -->
---
...content...
<!-- /CURSOR_SUMMARY -->
```

the greedy `<!--[\s\S]*?-->` fallback stops at the `>` of `<!-- CURSOR_SUMMARY -->`, removing only the opening marker and leaving the closing `<!-- /CURSOR_SUMMARY -->` as orphaned text rendered in the HTML output.  This is what CodeRabbit reports as `CURSOR_SUMMARY` leaking into the design doc.

## Changes

- `scripts/generate-pr-design-docs.mjs`: Replace individual comment-strip patterns with full-block patterns matching from `<!-- CURSOR_SUMMARY` (or `<!-- /CURSOR_SUMMARY`) through the matching `-->`, covering all four known encodings
- `docs/design/pr-designs/pr-202.{md,html}`: Regenerated with fix applied — confirmed `CURSOR_SUMMARY` absent
