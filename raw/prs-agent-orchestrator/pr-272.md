## Background

The `approved-and-green` reaction currently requires setting `action: auto-merge` on the reaction config to enable automatic merging. This is scattered per-reaction and doesn't have a global/project-level kill switch.

## Goals

- Add a single central config flag (`autoMerge`) to enable/disable auto-merge at the global or per-project level
- Preserve existing behavior by default (`autoMerge: false`)
- All `approved-and-green` reactions respect the centralized flag

## Tenets

- Minimal change: only add fields, don't restructure existing config
- Explicit wins: per-project overrides global, explicit reaction action wins over autoMerge flag
- Backward compatible: default behavior unchanged

## High-level description

Add `autoMerge?: boolean` to `OrchestratorConfig` (global) and `ProjectConfig` (per-project). In `getReactionConfigForSession`, when `autoMerge` is true and the reaction is `approved-and-green`, the action is automatically set to `auto-merge` unless an explicit `action` is already configured on the reaction.

Priority: per-project `autoMerge` > global `autoMerge` > reaction config.
