## Background
AO lifecycle-worker performs PR mutations (close, merge, comment) using the operator's personal GitHub account (jleechan2015). This makes it impossible to distinguish AO-automated actions from manual human actions in GitHub event logs and `/auton` diagnostics.

## Goals
1. PR mutations attributed to bot account (`jleechanao`) when `AO_BOT_GH_TOKEN` is set
2. Local audit log at `/tmp/ao-actions.jsonl` for attribution tracking
3. Read operations continue using the default operator token (no behavior change)

## Tenets
- Zero breaking changes — when `AO_BOT_GH_TOKEN` is not set, behavior is identical to before
- Bot token only for mutations, not reads — preserves operator access to private repos
- Audit log is best-effort — never crashes lifecycle-worker

## High-level description of changes

### scm-github plugin
- Added `botTokenEnv()` helper that returns `{GH_TOKEN, GITHUB_TOKEN}` override when `AO_BOT_GH_TOKEN` is set
- Added `ghBot()` function — same as `gh()` but passes bot token env to execCli
- `closePR()` and `mergePR()` (including GraphQL and REST fallback paths) now use bot token
- `execCli()` accepts optional env override parameter

### ao-action-log (new module)
- `logAoAction()` appends JSONL entries to `/tmp/ao-actions.jsonl`
- Logs: timestamp, session ID, action type, PR number, repo, reason

### lifecycle-manager
- Logs `pr_merge` action after successful merge
- Logs `session_kill` action when session reaches terminal state
