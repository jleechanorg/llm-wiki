## Summary

Three bugs in `ao skeptic verify` caused `VERDICT:FAIL` for valid PRs when running from the lifecycle-worker (launchd) environment. All three were root-caused during the WA PR #6185 session.

**Bug 1 — `mergeGate.ts`: wrong CR_BOT login format**
- Was: `const CR_BOT = "coderabbitai[bot]"`
- Fixed: `const CR_BOT = "coderabbitai"`
- Why: GraphQL `author.login` returns `"coderabbitai"` (no `[bot]` suffix). The wrong constant meant zero CR reviews matched → `crApproved: false` for every PR.

**Bug 2 — `prompt.ts`: review slice showed oldest 8 not newest 8**
- Was: `.slice(0, MAX_REVIEWS_TO_SHOW)` (first 8 of `last:20` = oldest 8)
- Fixed: `.slice(-MAX_REVIEWS_TO_SHOW)` (last 8 = newest 8)
- Why: GraphQL `last:20` returns reviews ascending. The old slice meant the LLM never saw the decisive recent APPROVED — only the stale earlier reviews.
- Also bumped `MAX_DIFF_CHARS` from 12K → 30K so the full diff fits.

**Bug 3 — `gh-client.ts` + `skeptic.ts`: design doc not found from AO repo**
- `fetchDesignDoc` used `git rev-parse --show-toplevel` which returns the AO repo root when lifecycle-worker runs from `~/project_agento/agent-orchestrator`
- Fixed: GitHub API as primary path (cwd-agnostic), local filesystem as fallback
- Updated call site in `skeptic.ts` to pass `owner`, `repo`, `pr.headRefOid`

**Bonus — `llm-eval.ts`: binary candidate order**
- Moved `~/.local/bin/claude` to the top (tried before GUI app bundles)
- GUI app bundles (`/Applications/cmux.app/...`) ETIMEDOUT in launchd context
