## Background

The backfill loop in `backfill-extensions.ts` previously failed the entire cycle when a single uncovered PR had `spawn` or `claimPR` failure. This blocked coverage of all other uncovered PRs.

## Goals

1. Backfill iterates through uncovered PRs, skipping those where `spawn` or `claimPR` fails, and tries the next one
2. Backfill aborts after 3 total spawn OR 3 total claim failures to avoid unbounded churn
3. PR titles are escaped and labeled as untrusted contributor input to prevent prompt injection
4. GitHub `checkoutPR` correctly handles untracked, staged, and working-tree-modified AO-managed files

## High-level description

The backfill loop (`for (const pr of uncovered)`) handles individual PR failures:
- `spawn` fails: increment `consecutiveSpawnFailures`, abort if >= 3
- `claimPR` fails: kill orphan session, increment `consecutiveClaimFailures`, abort if >= 3
- Both counters are independent; neither resets on the other's success
- PR title escaped with `\\` then `\"` and labeled as `[PR title: ...]`

GitHub `checkoutPR` uses `git status --porcelain` to distinguish untracked (`??`), staged (`M `), and worktree-modified (` M`) files, applying `git clean`, `git checkout`, and `git reset HEAD` per status.
