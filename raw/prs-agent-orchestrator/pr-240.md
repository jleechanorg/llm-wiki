## Background

CR CHANGES_REQUESTED repeats indefinitely with no new commits — workers spam `@coderabbitai all good?` without SHA progress. No deterministic comment-resolution extraction, no loop guard, and bloated agentRules burn context without fixing the actual blockers.

## Goals

1. Deterministic unresolved-comment extraction with CR severity prioritization
2. Anti-loop guard: if same actionable set repeats 2x without SHA progress, force `/copilot-expanded`
3. Trim agentRules to ≤20 lines with procedure doc references
4. Update reaction messages to gate `@coderabbitai all good?` on SHA evidence
5. CI/health check that FAILs when workers exit N cycles with no comment-resolution delta

## Tenets

- **SHA progress gate**: CR re-review requests only fire when a new commit SHA exists
- **Determinism**: unresolved comment extraction is deterministic (GraphQL, sorted priority)
- **Fail-closed**: CI health check fails rather than silently looping
- **Procedures over prompts**: agentRules point to procedure docs; prompts stay short

## High-level description of changes

### `scripts/extract-unresolved-comments.sh`
GraphQL-based deterministic comment extractor. Fetches all unresolved review threads via GraphQL (REST lacks `isResolved`), filters bot authors, prioritizes: Critical > Major > Medium > actionable > other. Output: JSON with threads[], summary counts, by-file breakdown.

### `scripts/cr-loop-guard.sh`
SHA-progress + actionable-set hash tracking. Cache: `~/.cache/ao-cr-loop-guard/<owner_repo_pr>.json`. Modes:
- `query`: returns JSON with loop_count, sha_progress, recommendation
- `fix-mode`: prints `cr-trigger | copilot-expanded | skip`

### `.github/workflows/cr-loop-health.yml`
Manual-dispatch CI gate. Fails when `loop_count >= max_cycles AND sha_progress == false`. Provides actionable output per cycle.
