## Raw Body
### Background

When the agent CLI (claude, codex, cursor-agent) exits unexpectedly inside a tmux session, the shell (bash/zsh) takes over at the prompt. AO sendMessage then pastes the next task into bash rather than into the agent CLI. The text either runs as shell commands or sits idle at a prompt. The agent is effectively dead, the task is lost, and no recovery is attempted. Bead: bd-tln (P0).

### Goals

1. Detect dead-agent (bash) state before and after sending a message.
2. Attempt automatic restart of the agent CLI using the original launch command.
3. Retry the message delivery after a successful restart.
4. Fail loudly (throw) if the agent cannot be restarted within 30s.

### Tenets

- Conservative detection: prefer false-negatives (miss a dead agent) over false-positives (kill a live agent). Only declare dead when a shell prompt is unambiguous.
- Fork-only: this logic is jleechanorg-specific and must not be upstreamed to ComposioHQ.
- No new dependencies: implemented entirely within the existing tmux plugin.
- Deterministic failure: if restart fails after 30s, throw rather than silently drop the message.

### High-level description of changes

Three new exported functions added to runtime-tmux plugin:

**isAgentAliveInPane(sessionName)**: Captures last 30 lines of pane output. Returns true if agent-alive tokens are found (spinner chars, "Thinking...", "Running tool"). Returns false if last non-empty line matches a shell prompt pattern ($ % chevron > #). Conservative default is true.

**restartAgentCli(handle)**: Sends C-c twice to cancel partial input, re-runs the original launchCommand (stored in handle.data.launchCommand). Polls isAgentAliveInPane up to 6 times at 5s intervals (30s total). Throws if agent does not recover.
