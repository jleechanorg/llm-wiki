> 
> **Overview**
> Adds GitHub rate-limit resilience to the `scm-github` plugin by routing `gh` executions through a new `ghWithRetry` wrapper that detects rate-limit errors, retries with exponential backoff, and (for `gh api` calls) falls back to direct `curl` requests against the GitHub REST API.
> 
> Updates `gh`/`ghInDir` helpers to use this wrapper and introduces `ghRestFallback` (exported) to construct REST URLs, optionally attach `gh auth token`, and reject unsupported GraphQL forms; unit tests cover retry behavior and REST fallback URL/header construction.
