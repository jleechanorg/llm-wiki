## Background

The skeptic agent posts `VERDICT: FAIL` comments on PRs when automated checks fail. These comments contain structured sections (`## Background`, `## Current Problem`, `## Recommended Solution`) that provide actionable guidance for workers. Currently, workers miss this feedback until they manually check PR comments.

## Goals

- Detect new skeptic FAIL comments on each PR
- Forward structured skeptic guidance to the worker agent via `send-to-agent`
- Track comment IDs to avoid re-triggering on already-seen comments

## Tenets

- Non-idempotent reaction: uses comment ID dedup to avoid duplicate sends
- Fails closed: empty context returns empty string (no spurious worker messages)
- Minimal scope: only adds the detection and forwarding logic, does not modify skeptic review itself

## High-level description of changes

The `skeptic-advice` reaction is a new lifecycle-manager trigger that polls for new skeptic comments on every PR session poll cycle. When a new skeptic comment is detected (highest comment ID exceeds the last known ID), the reaction fires `send-to-agent` with `{{context}}` replaced by extracted `## Background`/`## Current Problem`/`## Recommended Solution` sections from the latest FAIL verdict.

## Testing

- Unit tests in `packages/core/src/__tests__/skeptic-advice-reaction.test.ts`:
  - Returns extracted sections from a FAIL comment
  - Returns empty string when no skeptic comments exist
  - Returns empty string when comments exist but none contain FAIL
  - Uses most recent FAIL comment when multiple FAIL comments exist
  - Falls back to raw body when no ## sections are found
  - Returns empty string when SCM does not implement getSkepticComments
  - Returns empty string when session has no PR
