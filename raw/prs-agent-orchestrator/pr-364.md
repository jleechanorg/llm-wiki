## Background

Jeffrey's requirement: agents must prove their work with verifiable artifacts — never trust the agent's claim alone. For every substantive implementation task, agents must produce a 5-point evidence bundle.

This PR implements the automation and enforcement for the Cursor-style evidence standards:
1. **Terminal media capture** — `scripts/pr-media.sh` script for screenshot → Gist → PR comment
2. **CLI command** — `.claude/commands/pr-media.md` pointing agents to the script
3. **Skeptic prompt update** — Rule 13 for 5-point bundle checks in `packages/cli/src/commands/skeptic/prompt.ts`
4. **Skill documentation** — updated Cursor-aligned section in `.claude/skills/evidence-standards.md`

## Goals
- Agents can capture tmux/terminal screenshots with one command (`bash scripts/pr-media.sh`)
- Evidence Gate CI enforces terminal media presence (already in `evidence-gate.yml`)
- Skeptic evaluates 5-point bundle completeness for non-unit claims (integration, pipeline-e2e, pr-lifecycle-e2e, merge-gate)
- Evidence standards skill documents the Cursor-style workflow

## Tenets
- No unit test changes in this PR (docs/automation only)
- Skeptic prompt Rule 13 is fail-closed for terminal media + command logs + self-validation
- Skeptic prompt Rule 13e (artifact checklist) is a warning, not a hard fail
- No overlap with ao-3106's jq filter tests (different layers)

## High-level description of changes

### `scripts/pr-media.sh` (new)
Executable script that:
1. Captures tmux/terminal screenshot via `screencapture -w` (macOS)
2. Uploads to GitHub Gist via `gh gist create`
3. Prints markdown snippet for `## Evidence` section
4. Optionally posts media as PR comment

Supports `--type screenshot|video|gif`, `--pr N`, `--caption "text"`, `--test` (dry-run).

### `.claude/commands/pr-media.md` (new)
Slash command reference pointing agents to `scripts/pr-media.sh`.

### `.claude/skills/evidence-standards.md` (updated)
Enhanced Cursor-aligned section with:
- Table mapping Evide
