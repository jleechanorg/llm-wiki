## Raw Body
## Background

`packages/cli/src/commands/skeptic/prompt.ts` rule 10 previously said:

> "Evidence review is required only when config requires it; default is N/A."

This meant skeptic **never evaluated evidence quality** unless `evidenceRequired=true` in config (off by default). PRs with fabricated or placeholder evidence passed skeptic without scrutiny.

## Goals

- Remove the N/A escape hatch so skeptic always checks evidence authenticity
- Distinguish Gate 6 (evidence-review-bot, gated by `evidenceRequired`) from the LLM's own always-active authenticity evaluation

## Tenets

- Skeptic's inverted incentive (reward finding gaps) should apply to evidence authenticity
- Config flags gate external automations (Gate 6), not the LLM's own judgment

## Changes

### `packages/cli/src/commands/skeptic/prompt.ts`
Rule 10 replaced with a 6-line ALWAYS-active evidence authenticity check:
- FAIL on placeholder tokens: `simulated`, `example.com`, `<screenshot path>`, `<value>`, `TODO`, `TBD`
- FAIL on coverage claims missing `%` numbers
- FAIL on empty evidence sections or template-only placeholders
- PASS on real command output, real test results, real screenshot URLs
- NOTE: `evidenceRequired` config only gates evidence-review-bot (Gate 6)

### `packages/cli/src/__tests__/skeptic/skeptic-structured-output.test.ts`
Three new test cases: (1) authenticity check present regardless of config, (2) old N/A escape hatch absent, (3) PASS criteria includes real output examples.
