## Raw Body
### Background
Workers are required to use `[agento]` PR titles, but the current plugin guardrails only reject unprefixed `gh pr create` commands in some runtimes and do nothing in the Codex wrapper path. That leaves AO workers relying on prompt compliance instead of the harness.

Refs: bd-pfx

### Goals
1. Automatically prepend `[agento]` to `gh pr create` titles when workers omit it.
2. Preserve existing `cd` and environment prefixes in hook-managed command rewrites.
3. Cover both hook-based agents and the Codex gh wrapper with tests.
4. Fail closed when the hook cannot safely rewrite the command.

### Tenets
- Fix the harness, not just the prompt.
- Keep the change local to agent plugin guardrails.
- Preserve command semantics other than the required title prefix.
- Prefer one shared implementation over duplicated hook logic.

### High-level description of changes
The shared `gh pr create` title-rewrite logic now lives in `agent-base` as a reusable guard block, and `agent-claude-code` consumes that shared block instead of carrying a second copy. The guard preserves env and `cd` prefixes, rewrites only the real `--title`/`-t` argument, and now fails closed when no explicit title flag is present or `python3` is unavailable to safely preserve quoting.

The Codex gh wrapper now mirrors the same policy for direct `gh` execution, including bare `--title` / `-t` passthrough without duplicating flags. While adding a behavior-level regression test for the wrapper, the test exposed a real harness bug: the generated shell script was embedding a literal NUL byte instead of the `\0` escape sequence required for Bash to emit NUL-delimited argv output. This PR fixes that wrapper generation bug and verifies the rewritten argv against a real temporary `gh` stub.
