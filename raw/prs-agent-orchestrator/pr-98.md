## Background

The `CLAUDE.md` already documents the evidence claim-class matrix with fail-closed verdicts (bd-7ay), but there was no CI enforcement. Evidence bundles in PR descriptions were not being validated.

## Goals

- Enforce evidence bundle validation as an automated CI gate
- Fail closed when required proofs are missing for the declared claim class
- Cover all 5 claim classes: unit, integration, pipeline-e2e, pr-lifecycle-e2e, merge-gate

## Tenets

- Evidence gate is advisory for non-E2E claims (skips silently when no Evidence section)
- PR-lifecycle E2E and merge-gate claims require all 4 mandatory proofs or the gate fails
- Verdict presence is required — no verdict also fails the gate

## High-level description of changes

Added `.github/workflows/evidence-gate.yml` — a GitHub Actions workflow triggered on PR open/sync/edit/reopen that:

1. Detects presence of an `## Evidence` section in the PR body
2. Extracts the declared claim class from bold-markdown lines
3. For `pr-lifecycle-e2e` and `merge-gate`: validates 4 required proofs (PR creation URL, transition timeline, merge outcome, cleanup)
4. Validates a verdict (PASS / INSUFFICIENT / FAIL) is present
