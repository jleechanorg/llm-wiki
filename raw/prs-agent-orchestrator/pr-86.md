> [!NOTE]
> **Medium Risk**
> Changes lifecycle state transitions around terminal `killed`/`merged` and introduces a new SCM call path that can cause session checks to reject on transient SCM failures, which could affect polling behavior if not well-handled.
> 
> **Overview**
> Prevents *merged PR sessions* from being recorded as `killed` (and potentially leaving tmux runtimes behind) by adding an early override: when `determineStatus()` returns `killed`, `checkSession()` now asks the SCM whether the PR is actually `merged` and upgrades the status to `merged` so the existing merged-handling path performs the runtime `kill()` after exit-proof validation.
> 
> Adds a small helper module (`fork-lifecycle-kki-override.ts`) to perform the SCM merged check, and extends `lifecycle-manager.test.ts` with regression coverage for the killed→merged override and for transient SCM failures (ensuring the error propagates and no `killed` state is recorded, allowing retry).
