## Background

Version bumping was manual: developers had to run `pnpm changeset` before every merge. Agents were skipping this, leaving versions stale at `0.1.x` indefinitely.

## Goals
- Auto-bump patch on `fix:` merges, minor on `feat:` merges
- Gate major bumps behind human approval
- Zero developer action for patch/minor

## Tenets
- No npm publish — git tags + package.json bumps only
- All 23 workspace packages stay in sync
- Major releases always require human sign-off

## Changes
- `.releaserc.json` — semantic-release config with conventional commits preset
- `.github/workflows/release.yml` — 3-phase pipeline: detect → auto-release (patch/minor) or gate (major)
- `scripts/bump-all-packages.js` — syncs all 23 workspace package.json versions after root bump
- `package.json` — adds semantic-release devDependencies
