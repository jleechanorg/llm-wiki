## Background
Analysis of fork divergence from ComposioHQ/agent-orchestrator showed that lifecycle-manager.ts has a 583-line diff against upstream - the highest conflict risk file. ~67% of fork changes are already fully isolated in new files/plugins, but the core file modifications needed attention.

## Goals
- Reduce the upstream diff surface of lifecycle-manager.ts by extracting fork-specific functions into separate modules
- Codify fork isolation practices in CLAUDE.md and AGENTS.md so future changes maintain separation
- Preserve all existing functionality (683 tests pass)

## Tenets
- Companion module pattern over inline modifications
- Additive-only changes to upstream files when unavoidable
- Plugin-first for new capabilities

## High-level description of changes
Extracted three fork-specific functions from lifecycle-manager.ts into dedicated modules:
- buildReactionContext() -> reaction-context.ts (enriches reaction messages with CI/review context)
- validateAndEmitExitProof() + emitExitProofEvent() -> session-exit-proof.ts (bd-uxs.6 exit reconciliation)

The lifecycle-manager now imports and delegates to these modules instead of containing the logic inline. This reduces the diff by ~165 lines while preserving identical behavior.

Added fork isolation guidelines to both CLAUDE.md and AGENTS.md documenting the companion module pattern, high-conflict file list, and mandatory checklist for modifying core files.
