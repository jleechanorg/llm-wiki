## Background

The bd-85r startup grace period was added to protect sessions during agent CLI initialization, but the `isAlive()` runtime probe runs *before* the grace period check fires — so a false-negative from `isAlive` during agent init would kill a spawning session before `determineStatus` ever reached the `getActivityState` path.

## Goals

- Ensure `isAlive` probe respects `startupGracePeriodMs` for sessions in "spawning" status
- Add regression test for the `isAlive` gap
- Verify existing grace period tests still pass

## Tenets

- Minimal diff: only add the missing guard, don't restructure
- Tests first: new regression test covers the exact failure mode

## High-level description of changes

Added the same `(session.status === "spawning" && sessionAgeMs < gracePeriod)` guard to the `isAlive` block (step 1 of `determineStatus`) that already existed for `getActivityState` (step 2). Now the entire liveness probe chain is gated by the grace period for spawning sessions.
