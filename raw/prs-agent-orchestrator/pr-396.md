## Raw Body
Background:
The MCP_AGENT_MAIL_TOKEN was being written into the worktree .claude/settings.json file, which is a security risk as it could be accidentally committed to the repository.

Goals:
- Move MCP_AGENT_MAIL_TOKEN from settings.json to environment variables at runtime.
- Ensure all agent sessions (including Claude Code, Codex, Aider, and OpenCode) benefit from this change.
- Scrub any existing tokens from worktree settings.json files.

Tenets:
- Security first: secrets should never be stored in files that might be committed.
- Centralized logic: core components should handle shared security concerns.

High-level description:
- Updated session-manager to inject MCP_AGENT_MAIL_URL and MCP_AGENT_MAIL_TOKEN into the environment of every agent session (spawn and restore).
- Exported setupMcpMailInWorkspace from agent-base and updated all non-base agent plugins to call it, ensuring worktree settings.json is scrubbed of the token.
- Ensured setupWorkspaceHooks is called during session restoration to maintain consistency.
- Updated notifier-mcp-mail and core mcp-mail client to use the token from the environment via Authorization header.
