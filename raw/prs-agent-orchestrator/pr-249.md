## Raw Body
## Background
The `.claude/metadata-updater.sh` PreToolUse/PostToolUse hook had several gaps:
1. No enforcement of the mandatory `[agento]` prefix on `gh pr create` titles (bd-pfx)
2. Command stripping didn't handle leading `FOO=bar` env assignment prefixes
3. `git checkout`/`git switch` patterns were combined in a way that could miss edge cases
4. `sed` character class `[&|\/]` incorrectly escaped `|` and `/` in BRE

## Goals
- Enforce `[agento]` prefix at the hook level (code-level enforcement, not just agentRules)
- Fix command parsing edge cases
- Improve robustness of metadata detection

## High-level description of changes

### [agento] prefix guardrail (PreToolUse)
The hook now intercepts `gh pr create` in PreToolUse, parses `--title`/`-t` via Python `shlex`, and denies the tool call if the title doesn't start with `[agento]`. This blocks the command before it runs ŌĆö the prefix check is enforced in code, not just agent prompt instructions.

### Command stripping improvements
- Added leading `FOO=bar` env assignment stripping before the existing `cd` prefix strip loop
- Changed `${HOME}` to `${HOME}/` for correct brace-expansion path handling

### git checkout/switch split
- Separated the combined `||` if-block into distinct `if` blocks per command
- Added `HEAD` guard on `git checkout` to avoid recording bare commit/tag checkouts

### sed escape fix
- Changed `[&|\/]` to `[&\\]` ŌĆö only `&` and `\` need escaping in BRE `sed`

### PreToolUse/PostToolUse flow
- PreToolUse events now exit early after guards fire (allow or deny)
- Guards check `hook_event != "PostToolUse" && -n "$hook_event"` to properly distinguish PreToolUse from PostToolUse

## Testing

- Env assignment stripping tested: `FOO=bar gh pr create --title "[agento] test" ...`
- HEAD guard tested: `git checkout HEAD` should not update branch metadata
