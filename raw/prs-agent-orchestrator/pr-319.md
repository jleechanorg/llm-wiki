## Raw Body
## Background
When CodeRabbit approves a PR on the same commit (no new push), the skeptic dispatch
block in `lifecycle-manager.ts` fires only on SHA changes. With no SHA change, skeptic
waits up to 30 min for the `skeptic-cron` interval.

## Goals
- Fire skeptic evaluation immediately when session transitions to "approved" status
- Avoid duplicate evaluations (dedup via `lastSkepticSha` + VERDICT comment check)

## Changes
**packages/core/src/lifecycle-manager.ts**
Added a new block (bd-jzan) inside the `newStatus !== oldStatus` transition branch,
right after the `pr_open` skeptic trigger. Fires `worker-signals-completion` when:
1. `newStatus === "approved"` and `oldStatus !== "approved"` (new transition)
2. Session has a PR
3. SHA not already in `lastSkepticSha`
4. No VERDICT comment exists for the current SHA

Logs `lifecycle.skeptic_cr_approval_trigger` for observability.

VERDICT dedup check tightened (CR fix): SHA prefix must appear inside the HTML
comment marker (`<!-- skeptic-gate-trigger-abc1234 -->`), not just any HTML comment.

**packages/core/src/__tests__/lifecycle-manager-skeptic-wiring.test.ts**
Added 5 new unit tests covering:
- Fires on approved transition when SHA not evaluated and no VERDICT
- Skips when `lastSkepticSha` already has this SHA
- Skips when VERDICT comment already exists for SHA
- Idempotent guard (oldStatus already "approved" → no duplicate fire)
- Fires when `lastSkepticSha` is stale (different SHA)

Closes beads: bd-jzan
