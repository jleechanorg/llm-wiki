## Background

The skeptic agent shells out to `codex --print --no-input` via `llm-eval.ts` for headless LLM evaluation. `codex --print` does not exist — Codex CLI uses `exec` as its primary subcommand, and prompts are passed as positional arguments. This means skeptic has always failed to use Codex and silently fell back to Claude (when `ANTHROPIC_API_KEY` was set).

Additionally, the `skeptic-run` composite action had an early-exit gate that skipped the entire skeptic run when `ANTHROPIC_API_KEY` was not set — even though Codex OAuth doesn't require it.

## Goals
- Fix `llm-eval.ts` to use correct Codex CLI syntax: `codex exec [prompt]`
- Remove the `ANTHROPIC_API_KEY` early-exit gate in `skeptic-run`
- Preserve the Claude fallback path for repos that have `ANTHROPIC_API_KEY`

## Tenets
- Use correct Codex CLI syntax verified via `codex exec --help`
- Fail closed — missing VERDICT = FAIL (already implemented)
- Don't break the Claude fallback path

## Changes

- **packages/cli/src/lib/llm-eval.ts**: Replace `["--print", "--no-input"]` with `["exec", prompt]`, remove `input:` from exec options, update `stdio` to `["ignore", "pipe", "pipe"]`
- **.github/actions/skeptic-run/action.yml**: Remove ANTHROPIC_API_KEY early-exit gate; skeptic now always runs
- **.github/actions/skeptic-setup/action.yml**: Minor wording fix ("use" → "fall back to")
