## Background

The `UserPromptSubmit` hook fires the PR green-check reminder on every idle prompt, even when:
1. The PR is already MERGED/CLOSED
2. CR review state hasn't changed since the last check
3. A known blocker (e.g. CR rate-limited) is pending

This burns GitHub API rate-limit budget with no actionable outcome.

## Goals

- Prevent redundant GitHub API calls from the idle PR green-check reminder
- Add state guards so merged/closed PRs are skipped entirely
- Implement exponential backoff so repeated idle prompts don't hammer the API
- Reset backoff when CR review state actually transitions

## High-level description

**New file: `.claude/hooks/pr-green-check.py`**

A stateful Python hook attached to `UserPromptSubmit` that implements four optimizations:

1. **State guard** — skips if PR is MERGED/CLOSED (normalized to uppercase for comparison with GitHub REST API lowercase responses)
2. **Idempotent trigger** — tracks last-seen CR review state, skips if unchanged
3. **Exponential backoff with jitter** — 60s initial, 2x multiplier, 1h cap, ±15% jitter
4. **Transition-based re-check** — resets backoff when CR review state changes

**Hook registration: `.claude/settings.json`**

Added `pr-green-check.py` to the `UserPromptSubmit` hook chain with repo + home fallback paths.

## Testing

- Hook executes: exits 0, creates state file in `/tmp/pr_green_check/`, detects PR state correctly
- Backoff state persists across invocations via `/tmp/pr_green_check/ComposioHQ_agent-orchestrator_{pr}.json`
- No impact on normal prompts — always exits 0 (non-blocking)
