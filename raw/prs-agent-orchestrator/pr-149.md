> **Medium Risk**
> Changes session termination behavior when an agent is dead and SCM calls fail by persisting a new failure counter and deferring cleanup until a threshold is hit; mistakes could leave sessions/worktrees lingering or delay expected cleanup.
> 
> **Overview**
> Prevents dead-agent sessions from being immediately marked `killed` (and having worktrees destroyed) on a single transient SCM error by adding a persisted `scmFailureCount` counter and only confirming `killed` after **3 consecutive** SCM failures across poll cycles.
> 
> Updates dead-agent + PR handling so that when SCM succeeds and the PR is non-mergeable the session stays `pr_open`, and introduces a `killConfirmed` metadata flag to bypass the existing bd-kki “re-check merged before persisting killed” absorption logic once the failure threshold has been reached. Tests were updated to reflect the new `pr_open` behavior for dead-agent/non-mergeable PRs.
