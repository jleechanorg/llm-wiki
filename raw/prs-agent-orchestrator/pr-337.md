## Goals
1. Add a central config surface for default models keyed by CLI/agent name.
2. Allow per-project CLI-keyed overrides.
3. Keep existing role/project `agentConfig` precedence intact.
4. Preserve backward compatibility with existing config files.

## Tenets
- Prefer explicit CLI-scoped model settings over ambiguous global model defaults.
- Keep change surface minimal in core files with additive schema/type changes.
- Maintain existing behavior for role-specific overrides (`worker.agentConfig`, `orchestrator.agentConfig`).
- Add targeted tests for new precedence paths.

## High-level description of changes
This PR adds `modelByCli` support in both `defaults` and per-project config. During agent selection, AO now derives CLI-keyed model defaults from the resolved `agentName`, merging global defaults first and project overrides second. Model resolution now applies this precedence:
