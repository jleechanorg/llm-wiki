## Raw Body
When GitHub GraphQL rate limit (5000/hr) is exhausted, gh CLI commands that use --json flags fail because they rely on GraphQL internally. This adds try/catch REST API fallbacks to three plugins:

- poller-github-pr: gh pr list --json -> GET /repos/{owner}/{repo}/pulls
- tracker-github: gh issue view/list --json -> GET /repos/{owner}/{repo}/issues
- scm-github: statusCheckRollup synthesis from GET /commits/{sha}/check-runs

Each fallback uses exponential backoff retry (3 attempts) before falling back to REST. Non-rate-limit errors are thrown immediately without retry.
