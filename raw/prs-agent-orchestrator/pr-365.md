## Background

Workers learn the evidence standard by failing the skeptic gate — but the FAIL comment
(`placeholder_url_detected`) doesn't say *what format* is required.

This PR adds proactive guidance so workers understand the evidence requirement *before* failing the gate.

## Goals

1. Workers understand evidence-capture format proactively, not retroactively
2. Skeptic-gate FAIL comments are actionable with specific fix steps

## Changes

### Fix 1 — skeptic-gate.yml: actionable FAIL comment
When Evidence Gate fails, the PR comment now includes specific remediation guidance: remove any `simulated` text, placeholder URLs (`example.com`), or template markers (`TODO`, `TBD`, `<screenshot path>`) from the `## Evidence` section.

### Fix 2 — wholesome.yml: per-workflow concurrency group
Refined concurrency grouping to include `${{ github.workflow }}` in the group key, preventing cross-workflow blocking when multiple workflows target the same PR.

## Testing
