## Raw Body
fix(agent-base): skip writes when workspace config is unchanged

setupHookInWorkspace and setupMcpMailInWorkspace were unconditionally
writing settings.json and metadata-updater.sh on every session setup,
dirtying the git working tree even when content was identical.

Now each write is guarded by a content comparison — the file is only
written when the new content differs from what is already on disk.

Fixes bd-uxs.1. Covered by 5 new idempotency tests in workspace-setup.test.ts.
