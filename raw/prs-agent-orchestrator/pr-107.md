### Goals
- When multiple projects are configured and no `--project` flag is given, `ao start` and `ao stop` now match the current working directory against configured project paths before erroring.
- Users can run `ao start` from within a project directory without specifying the project name.

### Tenets
- Minimal upstream-aligned change; no fork-specific logic added.
- Only cherry-pick code fixes, not release infrastructure.

### High-level description of changes
`packages/cli/src/commands/start.ts`: Before erroring on ambiguous multi-project config, iterate over configured projects and match against `process.cwd()`. If a match is found, use that project automatically. Simplified path resolution to use `resolve()` directly since `loadConfig` already expands tildes.
