> [!NOTE]
> **Medium Risk**
> Changes rate-limit handling in CI/review/mergeability paths and PR detection logic, which can affect merge gating decisions and retry behavior. Adds more robust parsing for `gh api` REST fallback, reducing malformed fallback requests but touching core SCM integration.
> 
> **Overview**
> **Improves rate-limit behavior in merge gating.** `getCISummary` and `getReviewDecision` now *rethrow* GitHub rate-limit errors instead of masking them as `none`, so upstream lifecycle logic preserves the current status and retries rather than accidentally treating missing data as passing.
> 
> **Makes mergeability fail-closed on rate limits with clearer messaging.** `getMergeability` catches rate-limit failures from CI summary, forces CI to block merges, and adds an explicit “rate limited; try again” blocker instead of bubbling an uncaught error.
> 
> **Hardens REST fallback + PR discovery.** `ghRestFallback` now finds the `gh api` endpoint even when flags (e.g. `--method GET`) precede the path, and `detectPR` adds a second-pass search to detect fork PRs when `head=owner:branch` returns empty. Tests were updated/added to cover these scenarios.
