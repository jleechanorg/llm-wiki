## Background

PR #360 fixes multiple issues in the skeptic-cron Evidence Gate evaluation and wholesome.yml evidence checks. The jq filters had a null/missing conflation bug that made in-progress checks unreachable, and the GATE_COMMENT selector could pick stale results.

## Goals

1. Correctly distinguish "no Evidence Gate check-run found" (missing) from "check exists but still in-progress" (null conclusion)
2. Always select the newest gate-result comment when multiple exist for the same SHA
3. Sync the CLI template with the live workflow fixes
4. Fix wholesome.yml N/A handling for unit claims

## Tenets

- Fail-closed: ambiguous states must fail, not pass
- Template-live sync: CLI template and live workflow must match
- Minimal diff: only change what is broken

## High-level description of changes

The core fix replaces the jq fallback (which conflated null conclusions with missing checks) with a conditional that outputs "missing" when no check-run exists, empty string when conclusion is null (in-progress), and the actual conclusion otherwise. The GATE_COMMENT selector now uses sort_by(.created_at) to always pick the newest matching comment.
