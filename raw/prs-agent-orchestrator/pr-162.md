## Background
The /auton command in this repo was a stub pointing to the global skill. The actual implementation lived in ~/.claude/commands/auton.md and had been iterated on across multiple sessions and local branches. This PR consolidates all improvements into the repo.

## Goals
- Version-control the full /auton diagnostic in the repo
- Include stall detection, 6-green rate measurement, REST API fallback

## Tenets
- Single source of truth for the command in the repo
- REST-first API calls (works when GraphQL=0)

## High-level description of changes
Replaces the 14-line stub with the full 237-line autonomy diagnostic including:
- Infrastructure health: lifecycle-worker count, zombie session detection, locked worktree check
- GitHub state: open PRs via REST, review states, rate limits
- CHANGES_REQUESTED gap detection
- Stalled PR detection: flags PRs with >1hr gap that are not 6-green
- 6-Green rate: counts zero-touch vs human-touch merged PRs over last 7 days

## Testing
- Ran full diagnostic manually. Stall detection identified PRs #146, #147, #148, #151 as stalled.
- 6-green rate correctly reported 7/43 (16%) zero-touch over last 7 days.

## Low-level details
- .claude/commands/auton.md: Complete rewrite from stub to full diagnostic.
