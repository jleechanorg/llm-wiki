## Raw Body
"## Background\n\nPreviously, `sessionManager.kill()` was only called when a session transitioned to `\"merged\"`. Sessions that died naturally (agent exit, tmux killed, context exhausted) were detected as `\"killed\"` but their worktrees were never cleaned up. This caused orphaned worktrees to accumulate (34 found in production), locking branches and blocking future `ao spawn --claim-pr` calls with \"already checked out\" errors.\n\n## Goals\n\n- Clean up git worktrees for ALL terminal session states (killed, merged, done, errored), not just merged.\n- Prevent orphaned worktree accumulation in production.\n- Unblock `ao spawn --claim-pr` on branches previously locked by dead worktrees.\n\n## Tenets\n\n- Minimal scope: only touch the cleanup logic, no refactoring.\n- Keep cleanup after exit proof validation so the worktree is still available for commit validation.\n- Wrap `kill()` in try/catch so partial cleanup failures don't crash the transition handler.\n\n## High-level description of changes\n\nIn `lifecycle-manager.ts`, the `sessionManager.kill()` call (which tears down the tmux session and removes the git worktree) was gated behind `if (newStatus === \"merged\")`. The fix changes this to `if (TERMINAL_STATES.has(newStatus))` \u2014 covering all terminal states \u2014 and wraps it in try/catch so cleanup runs even if the session is partially gone. A new test covers all terminal transitions.\n\n## Testing\n\n- New unit test in `lifecycle-manager.test.ts` covering worktree cleanup on killed, merged, done, and errored transitions (80 lines added).\n- `pnpm --filter @composio/ao-core test` passes locally.\n\n## Low-level details\n\n### `packages/core/src/lifecycle-manager.ts`\nChanged the terminal-state guard from `newStatus === \"merged\"` to `TERMINAL_STATES.has(newStatus)` and wrapped in try/catch.
