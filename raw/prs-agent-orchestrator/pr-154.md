## Raw Body
## Background

`pruneStaleWorktrees` in `session-manager.ts` called `tmux has-session -t {worktreeName}` using the bare worktree directory name (e.g. `ao-748`). But AO tmux sessions are named `{12-char-hash}-ao-748` (e.g. `bb5e6b7f8db3-ao-748`). This mismatch meant the tmux check always returned "no session" — causing ALL AO worktrees to be incorrectly identified as stale and deleted.

## Goals

- Fix `pruneStaleWorktrees` to use the correct full tmux session name
- Add regression tests covering the bug and edge cases
- Preserve human-created worktrees (non-AO naming pattern)

## Tenets

- Minimal diff to session-manager.ts
- execFileAsync injectable for testability without vi.mock ESM complexity
- Tests use real execFileAsync path with mock shell scripts

## High-level description

When iterating worktrees under `~/.worktrees/{projectId}/`, the fix now:
1. Skips directories whose project is not in config (no execFile calls)
2. Skips non-AO worktrees (e.g. `feature-foo`, names not matching `ao-N` pattern)
3. For AO worktrees, derives `{prefix, num}` from the worktree name
4. Calls `generateTmuxName(configPath, prefix, num)` to construct the correct full tmux name (e.g. `bb5e6b7f8db3-ao-748`)
5. Runs `tmux has-session -t {fullName}` — correctly matching the real tmux session

Also made `execFileAsync` injectable via `SessionManagerDeps` so prune tests can inject a mock without ESM vi.mock complexity.
