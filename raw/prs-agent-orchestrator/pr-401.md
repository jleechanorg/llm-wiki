## Background
A previous follow-up tried to avoid serializing `MCP_AGENT_MAIL_TOKEN` by writing `Bearer ${MCP_AGENT_MAIL_TOKEN}` into worktree `settings.json`.

Claude Code reads MCP server headers from `settings.json` literally, so that placeholder broke MCP mail authentication instead of keeping the secret in env only. This follow-up restores working auth, fixes the shared JSON session activity regression surfaced during review, and updates the dependency graph so the current strict audit job clears again.

## Goals
- Keep MCP mail authentication working for AO-managed worktrees.
- Preserve correct activity detection for agents that store session state in `.json` files.
- Clear the current strict `pnpm audit --prod --audit-level=high` gate on this branch.

## Tenets

## High-level description of changes
`setupMcpMailInWorkspace` in `agent-base` now writes the actual bearer token value when `MCP_AGENT_MAIL_TOKEN` is present, because Claude Code does not expand MCP header env references from `settings.json`.

The shared `getActivityState` logic in `agent-base` now mirrors the JSON-first / JSONL-fallback behavior needed by plugins that use `.json` session files, and the new regression test covers that path directly.

At the repo level, `pnpm.overrides` now pins `axios` to the patched line and updates the lockfile so the strict production audit gate no longer fails on the prior vulnerable resolution.
