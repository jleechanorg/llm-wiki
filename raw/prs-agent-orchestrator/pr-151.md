## Background

The AO (Agent Orchestrator) currently supports tmux-based runtimes for spawning coding agents. This PR adds a new runtime plugin that uses Google's Antigravity IDE as a worker runtime via Peekaboo macOS accessibility API. Antigravity runs Gemini/Opus autonomously once a conversation is started, leveraging Google's model quota.

Design spec: `docs/design/antigravity-orchestrator.md`
Epic: bd-5kp

## Goals

- Implement the full `Runtime` interface (`create`, `destroy`, `sendMessage`, `getOutput`, `isAlive`, `getMetrics`, `getAttachInfo`) for Antigravity IDE
- Serialize all Peekaboo UI operations through a single async queue (macOS accessibility constraint)
- Provide Claude Code CLI fallback when Peekaboo/Antigravity is unavailable
- Expose MCP tools for spawning/managing Antigravity conversations
- Support CLI and Slack entry points for triggering conversations
- Configurable multi-repo workspace mapping via Zod-validated config

## Tenets

- **Fork isolation**: All new code in `packages/plugins/runtime-antigravity/` — zero modifications to upstream core files
- **Plugin-first**: Implements the existing `Runtime` interface, no core changes needed
- **TDD**: All modules have corresponding test files (63 tests total)
- **Files < 300 LOC**: Largest file is `runtime.ts` at 287 lines
- **LLM-driven**: No shell script wrappers — the orchestrator calls Peekaboo directly from TypeScript
