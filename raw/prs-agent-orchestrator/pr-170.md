## Background

Follow-up to PR #169 (merged). After merge, Cursor Bugbot flagged a High Severity issue:
the `\?\s*$/m` regex in `detectActivity()` matched ANY line ending with `?`,
not just standalone `?` prompts — causing false `waiting_input` signals for error
messages and thinking output like "Missing module — did you install it?".

## Goals

- `detectActivity()` returns `"waiting_input"` ONLY for genuine interactive prompts
- Prevent false positives from error messages, thinking output, and mid-sentence `?` lines

## Tenets

- Minimal, safe change — only tightens patterns, no new behavior added
- Backward compatible for genuinely interactive prompts (y/n, confirm?, standalone `?`)

## High-level description

1. **Removed** overly broad `\?\s*$/m` regex
2. **Replaced** with last-line extraction: only triggers `"?"` when it occupies its own line
3. **Tightened** `[y/n]` and `[yes/no]` patterns to require literal bracket+slash format
