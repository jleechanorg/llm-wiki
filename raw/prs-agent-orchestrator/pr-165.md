## Changes

1. **Remove invalid GraphQL field** (\`autorenameBranchPermitted\`): Not in GitHub's \`EnablePullRequestAutoMergeInput\` schema. Valid fields are: \`pullRequestId\`, \`mergeMethod\`, \`authorEmail\`, \`commitBody\`, \`commitHeadline\`, \`expectedHeadOid\`, \`clientMutationId\`. The invalid field caused GraphQL validation errors on every fallback call.

2. **Fix GraphQL error detection**: GraphQL returns HTTP 200 even on application errors (e.g. invalid input). Now also checks for \`data.enablePullRequestAutoMerge\` in response body before declaring success.

3. **Clarify \`autoMergeWaitSeconds\` docstring**: Corrected misleading description — the field is a numeric flag (value > 0 enables GitHub native auto-merge), not a configurable delay. GitHub handles the wait internally when \`--auto\` is active.

4. **Add \`autoMergeWaitSeconds\` to \`ReactionConfigSchema\`**: Without this, Zod strips the field at config-load time, making the YAML config ineffective. Now plumbed through config validation to \`executeReaction\`.
