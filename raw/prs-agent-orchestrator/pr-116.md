## Background

Runtime validation (bd-8y9, PR #110) showed the fork uses ~1.8x more API calls per lifecycle poll cycle than upstream. Deep code analysis revealed the call structures are identical -- the gap is environmental (retries, REST fallbacks, auto-orchestrator). However, several unnecessary API calls can be eliminated.

## Goals

Reduce per-cycle GitHub API consumption with three quick-win optimizations.

## Tenets

- Minimize core diff (fork isolation rules)
- No behavior regression
- Each optimization independently valuable

## High-level description of changes

### 1. Skip getMergeability when CI is not passing (bd-wg5)
Saves ~6 GraphQL points per session when CI is pending/failing/none.

### 2. Increase GhCache TTL from 15s to 60s (bd-91z)
60s gives better cross-session deduplication within a poll cycle.

### 3. Config flag skipAutomatedCommentPolling (bd-4nz)
Saves 1+ REST calls per session for repos without Bugbot.
