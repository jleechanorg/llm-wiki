## Background

## Goals

1. Remove direct LLM invocation from GitHub Actions
2. Route skeptic evaluation through AO worker on local machine (which has OAuth via Codex)
3. Eliminate need for API key secrets in GitHub repo
4. Make skeptic gate produce real PASS/FAIL verdicts instead of SKIPPED

## Tenets

- AO workers are the execution model
- No API keys in CI
- Fail-closed: no verdict = FAIL, not SKIP

## High-level description of changes

GHA skeptic-gate.yml refactored from run LLM evaluation directly to post trigger comment + poll for verdict from AO worker:

1. GHA posts an idempotent trigger comment
2. AO lifecycle-worker detects the trigger via reaction
3. AO spawns a skeptic worker locally with Codex (OAuth, no API keys)
4. Worker posts VERDICT comment back to PR
5. GHA polls for the verdict and updates CI check status
