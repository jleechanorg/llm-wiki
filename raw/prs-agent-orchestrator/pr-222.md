## Background

The runtime-antigravity plugin drives Antigravity IDE via the Peekaboo macOS accessibility API. Five bugs prevented `ao spawn --runtime antigravity` from working in practice: workspace matching was too strict, no scrolling to find off-screen workspaces, wrong conversation window detection, pressing Return instead of clicking Send, and sendMessage targeting the wrong window.

## Goals

- Make `ao spawn --runtime antigravity` functional against real Antigravity IDE instances
- Fix all 5 identified bugs in a single PR with test coverage for each

## Tenets

- Minimal scope: fix bugs only, no refactoring
- Backward compatible: fallback behavior preserved (e.g., press Return if no Send button found)
- Test-driven: every fix has corresponding test coverage

## High-level description of changes

### Bug 1: Workspace matching too strict
The Manager sidebar shows directory basenames (e.g., `worldai_claw`), not full paths. Added `matchesWorkspace()` helper that tries direct match, basename match, and parent directory match. Exported for testability.

### Bug 2: No scrolling to find workspace
If workspace not visible in the Manager sidebar viewport, scrolls down up to 5 times with re-check after each scroll.

### Bug 3: Conversation window detection wrong
With 18+ Antigravity windows open, the old code picked the first non-Manager window. Now matches workspace basename in window title, with fallback to old behavior.

### Bug 4: Return key adds newline instead of sending
Extracted `clickSendButton()` helper that finds and clicks the Send button after pasting. Falls back to pressing Return if no Send button found. Applied in both `create()` and `sendMessage()`.

### Bug 5: sendMessage uses wrong window
`sendMessage` used `session.windowId` (editor window). Now uses `session.managerWindowId` when available, falling back to `session.windowId` for direct sessions.
