> **Medium Risk**
> Changes PR merge behavior by introducing a new curl-based REST fallback path (including token resolution and best-effort branch deletion), which could impact merge correctness and error handling under rate-limit conditions.
> 
> **Overview**
> Improves `scm-github` PR merging resilience by wrapping `mergePR` in rate-limit detection and, on `gh pr merge` rate limits, falling back to a direct REST `PUT /pulls/{n}/merge` call via `curl` with an explicit `merge_method` body and token lookup from `GITHUB_TOKEN`/`GH_TOKEN`/`gh auth token`.
> 
> After a successful REST merge, it attempts **best-effort** head-branch deletion (URL-encoding the branch name) and logs warnings on cleanup failure. Separately, `.coderabbit.yaml` simplifies review settings and sets `auto_title_placeholder` to an empty string.
