## Summary
Gemini CLI stores sessions as a top-level JSON object: `{ sessionId, messages: [{ type, content }, ...] }`. This is NOT JSONL. The existing `readLastJsonlEntry` helper reads the last non-empty line of the file — for Gemini's format that's always `}`, which is valid JSON but has no `type` field. So `getActivityState` returned `null` for all Gemini sessions, giving AO no signal that the agent completed its turn (bead: orch-cb3e).

## Background

Gemini CLI stores sessions as a top-level JSON object: `{ sessionId, messages: [{ type, content }, ...] }`. This is NOT JSONL. The existing `readLastJsonlEntry` helper reads the last non-empty line of the file — for Gemini's format that's always `}`, which is valid JSON but has no `type` field. So `getActivityState` returned `null` for all Gemini sessions, giving AO no signal that the agent completed its turn (bead: orch-cb3e).

## Goals

1. AO correctly detects when Gemini finishes a turn (done-signal: last message type = "gemini" → ready)
2. AO correctly maps all real Gemini message types to activity states
3. Existing JSONL-style parsing still works as fallback
4. Hook setup runs before agent launch (supports agents that read settings at startup only)
5. Activity detection avoids false-idle when spinner output is present alongside visible prompt

## Tenets

- Gemini plugin override only — base plugin changes are additive exports or purely behavioural (no breaking changes to existing APIs)
- TDD — failing tests written first with real file I/O
- Fallback to JSONL ensures test fixtures and future format changes continue to work
- Hook setup errors are non-fatal in both worker and orchestrator sessions

## High-level description of changes

**agent-gemini**: Added `readLastGeminiNativeEntry` to parse the real Gemini JSON format. Overrode `getActivityState` in `geminiOverrides` to try native JSON first, then fall back to JSONL. Uses `geminiConfig.getSessionDir` for session path (no duplication). Null `lastType` in native entry falls through to JSONL rather than mis-classifying.

**agent-base**: Exported `findLatestSessionFile` for reuse in the Gemini override. Added `hookEventNames` configurability so Gemini (AfterTool/BeforeTool) and Claude (PostToolUse/PreToolUse) can use correct hook event names. `detectActivity` now treats a visible prompt as potentially active when recent terminal lines include Unicode spinner glyphs (false-idle fix).
