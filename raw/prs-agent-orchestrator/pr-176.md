## Background

Currently, runtime (`tmux` vs `antigravity`) is configured per-project in `agent-orchestrator.yaml`. The antigravity orchestrator needs to handle repos that aren't pre-configured, so the runtime must be selectable at spawn time.

## Goals

- Allow `--runtime antigravity` on `ao spawn` without needing project config changes
- Preserve existing fallback behavior (project config → global default) when flag is omitted
- Provide a clear error message when the specified runtime plugin is not found

## Tenets

- Additive only — no breaking changes to existing behavior
- Minimal core changes — most logic stays in CLI layer
- Error messages include the attempted runtime name for fast debugging

## High-level description of changes

- **`packages/core/src/types.ts`**: Added `runtimeOverride?: string` to `SessionSpawnConfig`
- **`packages/core/src/session-manager.ts`**: Updated `resolvePlugins()` to accept optional `runtimeOverride`; updated `spawn()` to pass it through and surface the override name in error messages
- **`packages/cli/src/commands/spawn.ts`**: Added `--runtime <name>` CLI option; plumbed it through `SpawnClaimOptions` → `spawnSession()` → `sm.spawn()`; also applied to batch-spawn/decompose path and `runSpawnPreflight` tmux-check guard
