## Background

The skeptic gate chain was refactored in PR #258 to route evaluation through AO workers instead of GHA API keys. However, three critical runtime gaps survived the refactor because they exist at interface boundaries that unit tests (which mock `execFileAsync`) cannot detect.

24-hour PR audit (2026-03-28) found: 0% of PRs got a real skeptic evaluation. The entire gate 7 enforcement system was silently disabled.

## Goals

Fix all three runtime-breaking gaps so the skeptic gate chain actually works end-to-end:
1. GHA polling matches the identity that posts VERDICT comments
2. SKIPPED/missing verdicts are fail-closed (not silently passed)
3. Subsequent pushes to open PRs trigger re-evaluation

## Tenets

- **Fail-closed**: Unknown/missing states → FAIL, never PASS
- **Interface verification**: Changes touch the exact interface points that were mismatched
- **Minimal scope**: Only fix the three identified gaps, no surrounding refactoring

## High-level description of changes

### 1. Bot author mismatch (bd-lg7i)
`skeptic-gate.yml` polled for VERDICT from `jleechan-agent[bot]` (GitHub App bot), but `ao skeptic verify` posts via `gh api` authenticated as `jleechan2015` (local user). The polling filter never matched → every run timed out.

**Fix**: Changed `SKEPTIC_BOT_AUTHOR` default from `jleechan-agent[bot]` to `jleechan2015` in both GHA and CLI, with env/var override for future identity changes.

### 2. SKIPPED = PASS (bd-0cfv)
`skeptic-cron.yml` treated `VERDICT: SKIPPED` and missing verdicts as `PASS`, silently passing all PRs when Codex/Claude were unavailable.

**Fix**: Changed to fail-closed — only explicit `VERDICT: PASS` counts as green. Everything else (FAIL, SKIPPED, missing) → FAIL.
