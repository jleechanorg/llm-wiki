## Background

Skeptic Gate (skeptic-gate.yml) runs `ao skeptic verify` in GHA, then polls for a VERDICT comment posted by that same step. When `ao skeptic verify` lacks LLM API keys (unavailable in GHA), it produces no VERDICT line in its output — leaving the poll step waiting for 28 minutes until timeout.

## Goals

- Prevent skeptic-gate from timing out when no VERDICT is produced in GHA.
- Keep the skip non-blocking so skeptic-cron handles actual LLM evaluation via AO worker.

## Tenets

- Fail closed for actual VERDICTs (PASS/FAIL unchanged).
- Fail open for infra issues (SKIPPED exits 0).
- Never modify the polling logic; fix the comment-posting step instead.

## Changes

**`.github/workflows/skeptic-gate.yml` — "Post skeptic VERDICT" step**

When `skeptics-output.txt` contains no `VERDICT:` line, the step now posts a `VERDICT: SKIPPED` comment via `gh api` with the same `<!-- skeptic-gate-trigger-{SHA} -->` HTML marker the poll step filters on. The poll step's existing `VERDICT: SKIPPED` handler exits 0 (non-blocking). Actual LLM evaluation is deferred to `skeptic-cron.yml`, which runs via AO worker with local API keys.

## Testing

- Verified the `gh api` call uses the same HTML marker format as the poll step's jq filter.
- Poll step already handles `VERDICT: SKIPPED` → exits 0 (no code change needed there).
- No unit tests added — YAML workflow change validated by manual inspection.
