## Background
bd-n039 introduced SHA-based deduplication for send-to-agent reactions in lifecycle-manager.ts, preventing redundant message sends when the PR head SHA is unchanged. The implementation (dedup-head-sha-store.ts) was merged but lacked a dedicated unit test file.

## Goals
- Add `lifecycle-manager-send-dedup.test.ts` to verify the dedup store invariants
- Ensure the dedup logic is validated independently of the full lifecycle-manager integration tests

## Tenets
- Direct store testing — no module mocking needed, pure unit tests
- Hermetic: each test clears its own state via clearLastSentHeadSha / clearAllMessageHashesForSession
- Readable: each test comment links to the invariant it validates

## High-level description
Creates `packages/core/src/__tests__/lifecycle-manager-send-dedup.test.ts` with 15 tests covering:

1. **SHA dedup invariants** (bd-n039 + bd-1178)
   - Undefined before any set → first send allowed
   - Same SHA → dedup triggers
   - SHA change → re-send allowed
   - Cross-project isolation (different projectIds = different keys)
   - Cross-reaction isolation (different reactionKeys = different entries)

2. **pruneStaleSessionIds** — removes dead sessions, preserves live, respects project scope

3. **clearLastSentHeadSha** — clears all reaction-key entries for a session

4. **Message hash dedup** — same hash → skip, different → send

5. **SHA-unavailable fallback** — when currentSha is undefined, shaUnchanged=true (message-hash only)

6. **Combined decision matrix** — skip only when BOTH SHA and message hash are unchanged

7. **hashMessageContent utility** — idempotent, 64-char hex, no throws on empty string
