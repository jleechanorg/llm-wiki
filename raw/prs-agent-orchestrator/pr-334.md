## Background
`ao spawn --agent gemini` failed with "Agent plugin 'gemini' not found" when using the globally installed `ao` binary, and the gemini plugin unconditionally stripped `--model` flags, preventing Gemini-native model IDs like `gemini-3-flash-preview` from being passed through.

## Goals
1. Allow Gemini-native model IDs (e.g. `gemini-3-flash-preview`, `gemini-2.5-pro`) to be passed through to the CLI's `--model` flag
2. Continue stripping non-Gemini model IDs (e.g. Anthropic `claude-sonnet-4-6`) that cause "Model not found" errors
3. Ensure all builtin plugins are listed as CLI dependencies so global installs work

## Tenets
- TDD: all changes driven by failing tests first
- Guard against the class of bug, not just the instance

## High-level description of changes
- Gemini plugin's `getLaunchCommand` now checks if model starts with `gemini-` before deciding to pass through or strip
- Added 3 missing plugin dependencies to `ao-cli/package.json` (tracker-gitlab, tracker-beads, scm-gitlab)
- New cross-package consistency test: imports `BUILTIN_PLUGINS` from `plugin-registry.ts` and asserts each package is listed under CLI **runtime** `dependencies` only (not `devDependencies`)
