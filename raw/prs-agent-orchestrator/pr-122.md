## Background
The lifecycle-worker currently polls session state but does not proactively reconcile orphan tmux runtime sessions that no longer have AO DB records. Over time, these stale tmux sessions accumulate and can interfere with spawn gating and operational hygiene.

## Goals
- Add periodic tmux orphan sweep behavior to lifecycle-worker
- Compare tmux AO-pattern sessions against AO DB sessions for the same project
- Clean up orphan tmux sessions that are idle beyond a TTL

## Tenets
- Keep sweep bounded and conservative: only AO-pattern tmux sessions for this config hash
- Never kill tmux sessions that still map to active AO DB runtime handles
- Use idle TTL threshold before cleanup to avoid racing active sessions

## High-level description
- Extended lifecycle-worker with optional orphan sweep interval and orphan TTL settings
- Added periodic + startup sweep pass that inspects tmux session names/activity and cross-checks against AO DB runtime handles
- Kills orphaned idle tmux sessions and records sweep outcomes in lifecycle observability
