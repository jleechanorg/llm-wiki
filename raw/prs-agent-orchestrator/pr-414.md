## Raw Body
### Background
AO workers were repeatedly becoming idle or disappearing in `~/.openclaw_prod` operations, especially when booted via launchd. We reproduced multiple failure modes: lifecycle-worker startup crashes on unresolved Slack webhook placeholders, lock/PID stale-state false positives, and launchd wrapper behavior that dropped child workers after `start-all.sh` exited. In parallel, Codex model selection was still not honoring CLI-specific overrides in all cases.

### Goals
1. Make lifecycle workers durable under launchd + `~/.openclaw_prod` without manual babysitting.
2. Prevent config placeholder values from crashing lifecycle-worker startup.
3. Fix stale lock/PID handling so dead owners are reaped and workers restart reliably.
4. Ensure Codex/CLI model override precedence is deterministic.
5. Add regression tests for the above so the failures fail-fast in CI.

### Tenets
- Fail open for non-critical notifier misconfigurations; never kill lifecycle due optional notifier config.
- Fail closed for lock ownership: only treat a lock as live if owner is a real lifecycle worker for the same project.
- Prefer harness/self-heal fixes over one-off operator cleanup.
- Keep user-scope config canonical (`~/.openclaw_prod`) and fallback-compatible.

### High-level description of changes
- Hardened lifecycle lock and PID identity checks to avoid stale-lock deadlocks and stale-PID startup suppression.
- Made Slack notifier initialization resilient to unresolved template/placeholder webhook values.
- Updated launchd lifecycle wrapper template to abandon process groups so child workers survive wrapper exit.
- Adjusted `start-all.sh` launch strategy in no-dashboard mode to start lifecycle workers directly for all projects.
- Extended config discovery and startup scripts to prioritize `~/.openclaw_prod/agent-orchestrator.yaml` while keeping legacy fallback.
- Fixed model precedence so CLI-specific model overrides take priority as intended.
- Added/updated tests around model pre
