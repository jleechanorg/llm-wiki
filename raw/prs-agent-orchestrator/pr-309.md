## Raw Body
## Background

Non-Claude agents (primarily **Codex**, OpenAI's CLI) cannot natively understand Claude slash commands like `/er`, `/fixpr`, `/simplify`, etc. The current `/claw` dispatch system resolves slash commands from pre-committed local files — if a command hasn't been committed locally, Codex workers cannot discover it.

MiniMax uses Claude internally and may understand slash commands natively. Cursor CLI has its own slash system. Only Codex and similar agents that do NOT natively support slash commands need this rule.

## Goals

- Codex agents can dynamically fetch slash command specs via web search when local definitions are unavailable
- Claude Code agents (built-in slash commands) are unaffected
- Rule lives in `defaults.agentRules` so all non-Claude agents inherit it automatically

## What Changed

| File | Change |
|------|--------|
| `~/.openclaw/agent-orchestrator.yaml` | Added SLASH COMMAND DISCOVERY section to `defaults.agentRules` |
| `docs/design/slash-command-discovery.md` | New design doc explaining the approach |

The rule instructs non-Claude agents to:
1. Web search for the slash command's official documentation
2. Read the spec to understand intent, args, behavior
3. Translate into their runtime — autonomously, no user help needed
4. Execute the translated command
5. If no equivalent exists, report what they found
