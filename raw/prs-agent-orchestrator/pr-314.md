## High-level description of changes

Two bugs in `packages/core/src/lifecycle-manager.ts`:

**Bug 1 — first-seen sessions skip skeptic dispatch**
The transition block's skeptic guard fires on `newStatus === "pr_open"`, but when lifecycle-manager first polls a session whose agent already wrote `"pr_open"` to metadata, `oldStatus` (from metadata) already equals `"pr_open"` — so no transition fires and skeptic is never dispatched.
Added a no-transition-path guard: when `tracked === undefined && newStatus === "pr_open"`, dispatch skeptic with the same pattern as the transition block (including claim-verification and SHA recording on success).

**Bug 2 — `ci_failed` absent from SHA-change re-trigger list**
The bd-qnj6 block re-triggers skeptic when HEAD SHA changes, but the status guard list was missing `"ci_failed"`. Added `"ci_failed"` to the list so sessions in CI failure get SHA-change coverage.
