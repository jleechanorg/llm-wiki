## Background

Skeptic Agent Phase 2 (bd-skp2) fixes critical production failures and adds an AO-native skeptic reaction. The skeptic gate and cron were silently failing due to (1) wrong model dependency (Claude CLI instead of Codex), (2) missing fail-closed behavior, and (3) silently masked errors.

## Goals

1. Replace Claude CLI with Codex CLI as the primary skeptic evaluator (OAuth-based, bd-skp2 critical fix)
2. Make skeptic gate fail-closed (no VERDICT line = FAIL, not pass-silently)
3. Add worker-signals-completion reaction that runs skeptic evaluation without spawning a new worktree
4. Keep existing ao skeptic --pr N CLI working for ad-hoc runs

## Tenets

- Skeptic does NOT spawn its own worktree - it evaluates EXISTING worker output
- Cross-model: if coding agent used Claude, skeptic uses Codex
- Inverted incentive: Your score is measured by gaps found. A false PASS is YOUR failure.
- Fail-closed: no VERDICT line in output = FAIL

## Changes

### modelRunner.ts (packages/cli)
Primary: codex --print (OPENAI_API_KEY). Fallback: claude --print (ANTHROPIC_API_KEY). Output without VERDICT line = FAIL (fail-closed).

### skeptic-gate.yml
Replaced Claude CLI install with Codex. Added OPENAI_API_KEY. Removed || true mask. Added || exit 1 to comment posting. UNKNOWN verdict = exit 1 (fail-closed).

### skeptic-cron.yml
Replaced Claude CLI install with Codex. Removed || true mask (fail-closed). Added OPENAI_API_KEY.

### types.ts (packages/core)
Added worker.signals_completion event type. Added skeptic-review to ReactionConfig.action. Added skepticModel and skepticPostComment config fields.

### lifecycle-manager.ts (packages/core)
Added worker-signals-completion reaction handler (skeptic-review case). Emits worker.signals_completion event on pr_open/approved transitions. Skeptic-review is not skipped for dead agents (SCM operation).

### skeptic-reviewer.ts (packages/core, new)
runSkepticReview(session) - calls ao skeptic --pr N --repo, writes specs/skeptic-report.
