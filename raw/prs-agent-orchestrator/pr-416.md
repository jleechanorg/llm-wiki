## Background

PR https://github.com/jleechanorg/agent-orchestrator/pull/416 extends the metadata-updater hook so Gemini CLI shell calls are treated the same as Claude shell calls. Review on the original change uncovered two regressions in the same hook: the `PreToolUse` `gh pr merge` deny path became unreachable, and plain `git checkout <branch>` stopped updating branch metadata. This update keeps the Gemini compatibility goal while restoring the harness behavior that `origin/main` already enforced.

Supersedes #389.

## Goals

- Accept Gemini CLI `run_shell_command` tool events in the metadata-updater hook.
- Keep the `gh pr merge` hard guardrail active during `PreToolUse`.
- Preserve branch metadata updates for both `git checkout <branch>` and `git switch <branch>`.
- Add regression coverage for the reviewed failure modes.

## Tenets

- Preserve existing safety-harness behavior unless a change is explicitly intended and tested.
- Keep the checked-in hook script and generated plugin hook content synchronized.
- Prefer narrow regression coverage over broader behavioral drift.

## High-level description of changes

The checked-in hook script now continues to accept Gemini CLI’s `run_shell_command` tool name, but restores the `gh pr merge` deny guard so it executes before the generic non-`PostToolUse` early return. The same update restores plain `git checkout <branch>` metadata tracking so checkout and switch stay aligned for existing feature branches.

The canonical generated hook content in the Claude agent plugin was updated to match the checked-in script, and the hook integration suite now covers the three reviewed failure modes: Gemini shell tool handling, `PreToolUse` merge blocking, and existing-branch checkout metadata updates.
