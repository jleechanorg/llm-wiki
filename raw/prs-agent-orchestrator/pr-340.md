## Background
Workers can still ship ambiguous PR evidence. We want the same reviewer contract Cursor describes for **cloud agents**: **artifacts (videos, screenshots, and logs)** attached to the work, not narrative-only claims — see [Cursor agents can now control their own computers](https://cursor.com/blog/agent-computer-use) (agents test changes and produce those artifacts; **merge-ready PRs with artifacts to demo their changes**). Long-running agent observability also depends on **logged command outputs** for replay — see [Towards self-driving codebases](https://cursor.com/blog/self-driving-codebases).

This PR codifies that intent for **AO workers** via fail-closed **Evidence Bundle v2** in `## Evidence`, enforced in CI.

## Goals
1. **Terminal proof** — Mandatory **screenshot or video** of real **tmux/terminal** context on every PR (`**Terminal media**:` + caption mentioning `tmux` or `terminal`), plus **fenced terminal test logs** — same triple as Cursor’s visual + logs pattern.
2. **Reproducibility** — Mandatory `**Repro gist**:` with clone-and-run steps for the PR branch.
3. **UI proof** — For user-visible changes: captioned **UI media** (screenshot/video). Otherwise exact `N/A - no UI changes`.
4. **Non-unit: agent screen recording** — For **Claim class** other than `unit`: mandatory **`**Agent screen recording**:`** with HTTPS **video** URL + caption — **self-produced in sandbox** (Cursor-style), **in addition to** terminal media + logs (`docs/evidence/agent-screen-recording.md`).
5. **No placeholders** — Reject fabricated markers (`<path>`, `TODO`, `example.com`, `simulated`, etc.).
6. **Policy = CI** — `CLAUDE.md` / `AGENTS.md` / skills match `evidence-gate.yml` and `wholesome.yml`.

## Tenets
- Fail closed over fail open.
- Artifacts over prose (Cursor model).
- Media + logs are complementary — terminal media is not optional when logs exist.
