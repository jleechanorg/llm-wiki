## Background
The namespace mismatch bug (bd-e4t) revealed that lifecycle-workers can silently operate in the wrong data namespace, making them blind to sessions. The existing `ao doctor` only checks install/env basics — it doesn't catch runtime issues like namespace mismatches, session sprawl, zombie sessions, or CR coverage gaps.

## Goals
- Wrap `ao doctor` with runtime health checks that catch the classes of bugs we've seen
- Add optional LLM inference layer for subjective analysis (proposals only, no decisions)
- Report to Slack for visibility

## Tenets
- Phase 1 checks must be fast (<15s) and fully deterministic
- Phase 2 LLM layer proposes only — never executes, never mutates
- All thresholds configurable via env vars

## High-level description of changes
New script `scripts/ao-doctor-monitor.sh` that runs `ao doctor` as baseline then adds:

**Phase 1 (deterministic):**
- Namespace alignment (lifecycle-worker CWD vs canonical config dir)
- Rogue config detection in `~/.agent-orchestrator/`
- Lifecycle-worker count and health
- GitHub rate limit monitoring
- Session sprawl detection (>N sessions per PR)
- Zombie session detection (sessions on merged/closed PRs)
- CHANGES_REQUESTED coverage gaps (CR PRs without active sessions)
- Stray worktree detection

**Phase 2 (LLM, optional):**
- Only runs when Phase 1 has WARN/FAIL
- Uses `claude -p` or `codex exec` with structured prompt
- Returns proposals with confidence levels (high/medium/low)
- Never executes commands or mutates state

**Slack reporting:**
- Posts summary to `$AO_DOCTOR_SLACK_CHANNEL` (default: #openclaw-health)
- Supports webhook URL or user token

## Testing
- Ran manually from `~/.openclaw` with Phase 2 disabled
- First run caught 7 namespace mismatches + 6 CR gaps (all real)
- After fixing plists: 8 PASS, 7 WARN, 0 FAIL
- Slack report delivered to #openclaw-health

- **scripts/ao-doctor-monitor.sh**: New 638-line script. All checks are individual functions for testability.
