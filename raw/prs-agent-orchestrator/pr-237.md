## Background

The skeptic CI infrastructure (skeptic-gate.yml, skeptic-cron.yml) runs the ao skeptic verify command on PRs. Installing skeptic CI into a new repo previously required manually copying these workflow files.

## Goals

- Provide ao skeptic install command that bootstraps skeptic CI into any target repo with one command
- Auto-detect repo owner/name so no manual editing is needed
- Keep it simple: copy files locally, user commits/pushes to activate

## Tenets

- No external API calls beyond gh CLI and git — works offline
- Templates use github.repository at GitHub Actions runtime — no injection needed
- Fail loudly with clear guidance if no workflows are selected

## High-level description of changes

- packages/cli/src/commands/skeptic/install.ts: New install command (ao skeptic install --[gate|cron|all]) that copies workflow templates into .github/workflows/ of the current directory
- packages/cli/src/templates/skeptic/: Workflow templates bundled in the CLI package, copied to dist/templates/ at build time
- packages/cli/src/commands/skeptic.ts: Restructured ao skeptic to use subcommands (verify, install)
- packages/cli/package.json: Build script copies template files into dist/

## Testing

- ao skeptic install --help: command tree shows subcommands
- ao skeptic install --gate --force: copies gate template correctly
- ao skeptic install --all: reports existing files correctly
