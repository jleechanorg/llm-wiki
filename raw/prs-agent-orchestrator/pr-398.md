## Raw Body
Background:
The agent-base plugin's default getActivityState implementation assumed all session files were in JSONL format, even if the sessionFileExtension was set to .json. This caused parse errors when reading native Gemini CLI session files, which are standard JSON objects.

Goals:
- Support standard JSON session files in the base agent plugin.
- Centralize JSON session reading logic in core.
- Reduce code duplication in agent-gemini.

Tenets:
- Flexibility: support different session file formats based on extension.
- Centralization: shared utilities should live in core.

High-level description:
- Added readLastJsonEntry utility to core/utils.ts to read the last message from a standard JSON session object (specifically looking for the messages array).
- Updated agent-base's getActivityState to choose between readLastJsonlEntry and readLastJsonEntry based on the configured sessionFileExtension.
- Refactored agent-gemini to use the new core utility instead of its local implementation.
