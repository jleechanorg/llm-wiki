## Background

PR #115 proved AO workers can independently drive a PR to 6-green, but the lifecycle-worker failed to auto-merge it. Three interacting bugs in the lifecycle polling loop prevented the approved-and-green reaction from firing and succeeding.

## Goals

1. Lifecycle-worker detects state transitions within 30s (not 5min)
2. Agent exit does not mask a mergeable PR
3. Transient merge-gate failures do not permanently block merge

## Tenets

- Process > outcome: fix the pipeline, do not manually merge
- Fail-open for merge: if agent is dead but PR is green, merge it
- Minimal core diff: 3 surgical changes, no refactoring

## High-level description of changes

**Fix 1 — Poll interval** (lifecycle-worker.ts): Default changed from 300s to 30s.

**Fix 2 — Agent-exit defers to PR state** (lifecycle-manager.ts): determineStatus() previously returned killed immediately when agent exited (step 1-2), never reaching PR checks (step 4). Now sets agentDead=true flag and continues to PR checks.

**Fix 3 — Merge retry on steady-state** (lifecycle-manager.ts): Reactions only fire on state transitions. When status stays mergeable across polls, the approved-and-green reaction now re-fires each poll.
