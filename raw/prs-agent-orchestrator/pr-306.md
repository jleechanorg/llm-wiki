## Background

Claude Code workers understand slash commands like /copilot and /polish. The lifecycle manager sends plain-text messages (e.g. review comment notifications) which Claude Code processes as free-form text rather than triggering its built-in slash command flows.

## Goals

Route fix-intent messages to the appropriate Claude Code slash command to improve agent responsiveness on PR reactions.

## Tenets

- Only transform for claude-code agent -- Codex/OpenCode receive plain text unchanged
- Use agentName not runtimeName for routing (Claude Code sessions run on tmux runtime)
- Preserve existing slash commands rather than re-detecting from stripped body
- Fail-safe: if no intent matched, leave message unchanged
- Fork-specific logic in companion module per fork isolation policy

## High-level description of changes

Added slash-command routing in session-manager.ts send():

1. Two utility functions in fork-slash-command-routing.ts companion module:
   - messageContainsCommentFixIntent(msg) -- phrase-level keyword matching
   - transformToSlashCommand(msg) -- maps to /copilot or /polish

2. Routing in send(): after resolving agentName, if claude-code + fix-intent, transform before sendMessage
