## Background
`start-all.sh` starts all AO lifecycle-workers but is a one-shot script — if a worker crashes, launchd has no keepalive to restart it. The per-project plist in jleechanclaw uses `KeepAlive: true` with `ao lifecycle-worker` directly, but this isn't portable or multi-project.

Refs: bd-la7

## Goals
- Wrap `start-all.sh` in a launchd plist with `KeepAlive: true` so crashed workers are restarted automatically
- Provide a `setup-launchd.sh` to install the plist on a fresh machine

## Tenets
- Additive only — no changes to existing lifecycle logic
- Template-based plist so paths are portable

## High-level description of changes
Adds `scripts/setup-launchd.sh` and `launchd/ai.agento.lifecycle-all.plist.template`. The plist wraps `start-all.sh` with `RunAtLoad` and `KeepAlive` true, logging to `~/.openclaw/logs/ao-lifecycle.log`.

## Testing
Manual install via `./scripts/setup-launchd.sh` and verify `launchctl list | grep ai.agento.lifecycle` shows the service.

## Low-level details
- **scripts/setup-launchd.sh**: generates and installs the plist from template
- **launchd/ai.agento.lifecycle-all.plist.template**: launchd plist template with KeepAlive
- **scripts/start-all.sh**: header updated to mention launchd wrapper
