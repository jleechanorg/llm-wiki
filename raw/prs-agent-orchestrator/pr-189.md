## Background

runtime-antigravity create() dives straight into Peekaboo window ops without validating the environment. When Antigravity is not running or the Manager window is missing, opaque errors surface deep in the call stack.

## Goals

- Validate Antigravity runtime environment before session creation
- Route to CLI fallback automatically on preflight failure
- Log the specific step that failed for debugging

## Changes

**preflight.ts (new):** runPreflight() with 3 checks — peekaboo-reachable, app-running, manager-window. TTL via AbortController+setTimeout (default 10s). Returns PreflightResult{ok, steps[], elapsedMs}.

**runtime.ts:** primaryFn now calls runPreflight() first; on failure throws to trigger executeWithFallback CLI path.

**runtime.test.ts:** mock runPreflight to isolate existing tests.

**preflight.test.ts (new):** 6 unit tests covering pass/fail at each step.

**index.ts:** exports PreflightResult, PreflightConfig, runPreflight.
