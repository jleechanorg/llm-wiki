## Summary

Fix the Evidence Gate authenticity gap in `wholesome.yml` `evidence-media-attachment` job.

## Background

The Evidence Gate previously passed any code block or HTTPS URL in the Evidence section without validating the content was real. PRs with fabricated evidence have previously merged.

## Changes

Added a new rejection step in `.github/workflows/wholesome.yml` after the existing media checks that detects and rejects:

1. A forbidden keyword indicating fabricated output
2. `https://example.com/*` placeholder URLs
3. Template text — `<screenshot path>`, `<value>`, unfinished markers

The check runs after extracting the Evidence section content (using the existing `awk` section-boundary parser) and before accepting the media as valid.
