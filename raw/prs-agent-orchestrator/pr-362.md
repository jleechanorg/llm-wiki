## Summary

Adds 32 unit tests that re-implement every `gh api --jq` filter from `skeptic-gate.yml` and `skeptic-cron.yml` in TypeScript, validating filter logic against realistic API output shapes without needing a real gh binary or GitHub API.

## Coverage

### skeptic-gate.yml filters (14 tests)
- **Check-run deduplication** (6 tests): `group_by(.name) | sort_by(.started_at) | reverse | .[0]` — validates that Lint/Typecheck/Test/Test (Web) deduplication uses the most-recent run per name, that `conclusion=null` is excluded from failure count, that paginated results merge correctly, that non-core checks are ignored.
- **CR review state** (4 tests): CodeRabbitAI `APPROVED`/`CHANGES_REQUESTED` filter — validates stale CR detection, wrong-author rejection, `COMMENTED` state is ignored.
- **Bugbot check-run conclusion** (3 tests): `sort_by(.started_at) | reverse | .[0].conclusion` — validates most-recent run wins, `none` fallback when absent.
- **Prior-result comment cleanup** (1 test): `select(.user.login == "github-actions[bot]" and .body | test("skeptic-gate-result-"))` — validates marker matching.

### skeptic-cron.yml filters (6 tests)
- **Non-draft PR list** (2 tests): `select(.draft == false) | {number, title, head: {...}, base: {...}, mergeable}` — validates draft filtering and field projection.
- **Evidence check-run filtering** (4 tests): `ascii_downcase | test("^evidence([ -]?gate| review)$")` — validates that the regex is **case-sensitive** (the pattern has no `i` flag, so title-cased CI check names from GitHub would not match).
- **Verdict comment SHA-scoping** (6 tests): validates SHA marker matching, wrong-SHA rejection, most-recent-by-ID, `VERDICT:` case-insensitivity, alternate bot authors.
