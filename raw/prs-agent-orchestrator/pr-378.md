## Summary

Implements the Manager Evolve Loop types, config, and orchestrator-prompt injection from `docs/design/manager-evolve-loop-design.md` (bd-jhv1).

### Changes

**`packages/core/src/types.ts`**
- Added `EvolveLoopConfig` interface with all fields typed and optional: `enabled`, `pollCadence` (`"lightweight" | "standard"`), `autonomousFixScopes` (string[]), `blockedScopes` (string[]), `knowledgeBaseDir` (string), `zeroTouchWindow` (`"24h" | "30d"`)
- Added `evolveLoop?: EvolveLoopConfig` field to `ProjectConfig`

**`packages/core/src/config.ts`**
- Added `EvolveLoopConfigSchema` Zod schema with sensible defaults
- Wired `evolveLoop` into `ProjectConfigSchema`
- Kill switch: `EVOLVE_LOOP_ENABLED=false` env var disables evolve loop via explicit string comparison (not `z.coerce.boolean()`)
- `knowledgeBaseDir` tilde expansion using `os.homedir()` (correct `slice(2)` on `~/` prefix)

**`packages/core/src/orchestrator-prompt.ts`**
- Added `generateEvolveLoopSection()` — returns empty string when disabled, injects full 6-phase loop (OBSERVE → MEASURE → DIAGNOSE → PLAN → FIX → RECORD) when `evolveLoop.enabled === true`
- Implicit deny-list always present: `gh pr merge`, `gh pr close`, `git reset --hard`, `git clean -fd`, `git worktree remove`, `rm -rf`
- FIX phase respects `autonomousFixScopes` allow-list and `blockedScopes` explicit deny-list
- Wired into `generateOrchestratorPrompt()`

**`packages/core/src/__tests__/evolve-loop-config.test.ts`**
- 29 unit tests covering: type compilation, Zod schema defaults and validation, kill switch, path expansion, prompt injection (all phases, allow-list, deny-list, implicit deny-list, zero-touch window)
