### Background
cursor-agent sessions silently failed in all AO worktrees. Every new session showed a "Workspace Trust Required" interactive dialog, blocked indefinitely, exited, and printed the models list — no work done. Diagnosed during /auton run 2026-03-21.

### Goals
- cursor-agent sessions run unattended without workspace trust prompt
- ao send messages to cursor sessions are actually processed

### Tenets
- Minimal change — one override, cursor-specific behavior stays in cursor plugin

### High-level description of changes
Override getLaunchCommand in the cursor plugin to append --trust. cursor-agent --trust skips the workspace trust prompt unconditionally.
