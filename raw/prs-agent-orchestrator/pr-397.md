## Raw Body
Background:
Zombie tmux sessions were accumulating indefinitely when the lifecycle-manager's initial cleanup call failed or was interrupted during the transition to a terminal status.

Goals:
- Ensure terminal sessions are proactively reaped by the lifecycle-manager poll loop.
- Prevent accumulation of dead sessions that block the spawn gate.

Tenets:
- Reliability: cleanup should be retried until successful.
- Proactivity: don't wait for the nightly reaper if we can clean up now.

High-level description:
- Updated pollAll filter to include terminal sessions in the poll loop until they are successfully killed and removed from the metadata list.
- Removed the transition guard (oldStatus check) for terminal cleanup in checkSession, allowing the kill() call to be retried on subsequent poll cycles if the session is still present.

Testing:
- Ran src/__tests__/lifecycle-manager.test.ts (94 tests passed).
- Verified that the logic handles terminal sessions correctly even when they are not part of a status transition.
