## Summary

## Root cause

In `jleechanorg/worldarchitect.ai`, 5 PRs (#6109, #6116, #6118, #6119, #6122) all reached CI-green but sat `BLOCKED` by `reviewDecision: CHANGES_REQUESTED`. Meanwhile, AO kept spawning new beads that created fresh unrelated PRs (#6132–#6135). Investigation showed the `worldarchitect:` project block in the user's `agent-orchestrator.yaml` simply never had `backfillAllPRs: true`, so `backfillUncoveredPRs` was never called (`lifecycle-manager.ts:2720`), and the lifecycle-worker had no mechanism to rescue blocked-but-green PRs.

Sibling projects (`jleechanclaw`, `agent-orchestrator`, `smartclaw`) all explicitly set `backfillAllPRs: true`; `worldarchitect` was missing it silently. There was no log, no warning, no signal. Making backfill opt-out instead of opt-in removes this failure mode entirely — new projects get the right behavior by default, and existing projects that deliberately opted out get a visible warning when they have leaking PRs.
