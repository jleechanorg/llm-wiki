## Background

When CodeRabbit posts `CHANGES_REQUESTED` and someone dismisses the review, GitHub transitions the review state to `"dismissed"`. The previous `getLatestDecisiveReview()` filter skipped dismissed reviews entirely, silently ignoring the `CHANGES_REQUESTED` â€” potentially letting the gate pass on no real approval.

## Goals

- Prevent dismissed CodeRabbit `CHANGES_REQUESTED` reviews from being silently ignored.
- Require a genuine new `APPROVED` review after any dismissed `CHANGES_REQUESTED`.

## Tenets

- Fail-closed: if the dismissal was not followed by a real re-review, the gate must block.
- Chronological correctness: scan reviews newest-first; a dismissal voids all prior reviews.

## Changes

### `packages/core/src/merge-gate.ts`
- Add `hasUnresolvedDismissedReview()` helper: scans CR reviews chronologically (newest-first). If a `"dismissed"` review is encountered before any `"approved"` state, returns `true` (blocked).
- CR gate now requires both `latestCR?.state === "approved"` AND `!hasUnresolvedDismissedReview()`.
