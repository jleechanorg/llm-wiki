## Summary

> [!NOTE]
> **Medium Risk**
> Changes delivery semantics for `notify`/`notifyWithActions` by suppressing repeats per `sessionId`+`event.type`, which could hide legitimately repeated events if keys/TTL are misconfigured. Logic is contained to the Slack notifier and covered by new unit tests, reducing overall risk.
> 
> **Overview**
> Adds **in-memory deduplication** to `notifier-slack` so `notify()`/`notifyWithActions()` suppress repeated sends for the same `sessionId:eventType` within a configurable `dedupTtlMs` window (default `60_000`), including cache sweeping and size bounding.
