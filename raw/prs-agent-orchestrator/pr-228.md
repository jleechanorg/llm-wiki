## Background

Jeffrey Lee-Chan requested an automated design doc generation system (following the PR #592 template format) for all PRs managed by the Agent Orchestrator. The goal is to create a sustainable archive of AO work in a consistent, readable format.

## Goals

- Generator script that produces both `.md` and `.html` design docs per PR
- Idempotent operation (safe to re-run without duplicate content)
- PR #592 template fidelity (warm paper tones, IBM Plex fonts, arch diagrams, metadata tables)
- Generate initial batch for all existing AO-managed PRs (33 PRs: #188–#227)
- `--dry-run`, `--force`, `--limit`, `--repo` CLI flags

## Tenets

- Use REST API exclusively (no GraphQL) — avoids quota exhaustion
- Idempotent: skips PRs that already have both `.md` and `.html` unless `--force` is set
- AO-managed detection: `[agento]` title prefix OR matching branch name patterns

## High-level description of changes

### `scripts/generate-pr-design-docs.mjs`
A standalone ESM Node.js script that:
1. Fetches all open + closed PRs from the target repo via `gh api` REST
2. Filters to AO-managed PRs (title/branch pattern match)
3. For each PR, fetches files changed + commits in parallel
4. Generates a paired `.md` and `.html` design doc into `docs/design/pr-designs/`
5. Each doc includes: hero section, ASCII arch diagram, metadata grid, files-by-package table

### `docs/design/pr-designs/`
66 generated files (33 PRs x `.md` + `.html`): `pr-188.md/html` through `pr-227.md/html`

### Template fidelity (PR #592 format)
- Warm paper CSS variables: `--bg: #f2efe7`, `--panel: #fffdf8`
- IBM Plex Sans/Serif/Mono from Google Fonts
- ASCII arch diagram in `<pre>` inside `.diagram` div
- Badge system for file status (added/merged/open/closed)
- Two-column grid layout with `.card` sections
