## Background

This PR adds a "Wholesome PR Checks" GitHub Action workflow that enforces PR hygiene: requiring an `[agento]` prefix in the title and a non-empty `## Evidence` section with media (screenshot or terminal output) proving the feature works.

## Goals

- CI rejects PRs without `[agento]` prefix in title
- CI rejects PRs without a `## Evidence` section
- CI rejects Evidence sections that are empty (no media or code output)
- Roadmap updated to reflect implementation

## High-level description

- **`.github/workflows/wholesome.yml`**: New workflow triggered on `pull_request` events (open, synchronize, edited, reopened). Three jobs:
  1. `pr-title-prefix`: validates the PR title starts with `[agento]` using shell `case` with sanitized output (no regex injection, no log injection)
  2. `evidence-section`: parses PR body with anchored `grep -E` to detect `## Evidence` heading
  3. `evidence-media-attachment`: uses `awk` with `IGNORECASE=1` to extract only the Evidence section (stops at next `##`), then checks for: markdown images with `https://`, media with `https://`, code blocks, or structured `**(Test|Terminal) output**:` references
- **`roadmap/harness-engineering-v2.md`**: marks bd-mpr as DONE

## Testing

- CI runs `wholesome.yml` on every PR open/sync
- Lint, Typecheck, Integration Tests, Test (Web) all pass
- Evidence Gate workflow tested against this PR body
