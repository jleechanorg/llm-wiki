## Background
Phase 1 of the Living Companions roadmap requires showing companion state in the game UI. Companions have name, mood, location, and action history that players should see during gameplay.

## Goals
- Display companions list in the existing game view with name, mode, location, and last action
- Surface lastAction from the backend (most recent action summary)
- Use existing CompanionPanel component as the rendering layer

## Tenets
- Minimal surface area: wire existing CompanionPanel into GameScreen, add only necessary data plumbing
- lastAction is optional/nullable to avoid crashes when companions have no action history

## High-level description of changes
Backend (app.ts): GET /sessions/:id/companions now includes lastAction by fetching the most recent CompanionAction summary. Frontend types (types/game.ts): Added lastAction field to Companion interface. CompanionPanel: CompanionCard renders lastAction when present. GameScreen: Renders CompanionPanel in the right panel when companions exist. App: GameScreenWrapper fetches companions on session init and passes them to GameScreen.

## Testing
- Added companions_ui_api.test.ts: integration test verifying lastAction is returned by the API with the correct summary string
- CI runs on push

## Low-level details
- packages/backend/src/app.ts: enriches GET companions response with lastAction from getActionsByCompanion()
- packages/web/src/types/game.ts: lastAction?: string | null added to Companion interface
- packages/web/src/components/CompanionPanel.tsx: renders lastAction in CompanionCard
- packages/web/src/screens/GameScreen.tsx: accepts companions prop, renders CompanionPanel
- packages/web/src/App.tsx: fetches companions after session creation, passes to GameScreen
- packages/backend/src/__tests__/companions_ui_api.test.ts: new integration test
