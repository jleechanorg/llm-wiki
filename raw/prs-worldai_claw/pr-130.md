## Background
Phase 2 of the Living Companions roadmap. Companion actions (join, depart, act) were not surfaced as `WorldEvent` entries in simulation output. Players could not see companion activity in the world events feed.

## Goals
- Add `companion_joined`, `companion_departed`, `companion_acted` to `CompanionEventType` and `WorldEvent.type`
- Wire companion world event generation into `runSimulation()` via `appendCompanionWorldEvents()`
- Export canonical `companionActionEventMap` for shared use between simulation paths

## Tenets
- Reuse existing narrative text (no extra LLM calls per event)
- Single canonical mapping from action types to event types/severities
- 50-event cap enforced by `addWorldEvent()` — no duplicate cap logic

## High-level description of changes
Added `companion_joined`, `companion_departed`, `companion_acted` to the `WorldEvent.type` union and `CompanionEventType`. Exported `companionActionEventMap` from `events.ts` as the single source of truth for action→event mapping. Added `appendCompanionWorldEvents()` in `faction_simulator.ts` that maps `CompanionSimulationDelta` results to world events using pre-narrated text. Wired into both `runSimulation()` and `runCompanionTicks()`. Tests added in `companion_events.test.ts` and `companion_world_events.test.ts`.

## Testing
- `companion_events.test.ts`: 100-line unit test suite for `generateCompanionEvents()` covering all mapped action types
- `companion_world_events.test.ts`: 159-line integration test for companion world event generation via `runSimulation()`
- Backend CI passes (backend-test workflow)

## Low-level details
- **`packages/backend/src/world/events.ts`**: Added `companion_joined`, `companion_departed`, `companion_acted` to `CompanionEventType`. Exported `companionActionEventMap` with mappings for guard, explore, trade, combat, quest, social, travel, and the 3 new types.
- **`packages/backend/src/world/faction_simulator.ts`**: Added `appendCompanionWorldEvents()` function; wired in
