## Raw Body
### Background
DM companion force-edits currently happen silently. Owners should be notified real-time for UX consistency and transparency during gameplay.

### Goals
- Implement a session-level SSE endpoint for real-time notifications.
- Emit `companion_edited_by_dm` events when a DM force-edits a companion.
- Ensure efficient resource cleanup on SSE client disconnect.

### Tenets
- **TDD Compliance**: Verified BOTH Red (failing) and Green (passing) states in integration tests.
- **Reliability**: Uses `SessionEventBus` singleton to decouple route logic from event transport.
- **Performance**: Minimal footprint in `app.ts` via specialized SSE infrastructure.

### High-level description of changes
- **SSE Infrastructure**: Added `GET /api/sessions/:id/events` endpoint to `app.ts`. Uses `fastify` hijack for manual stream management and heartbeats.
- **Event Emission**: Updated the companion force-edit route in `app.ts` to dispatch notification payloads via `sessionEventBus.emitSessionEvent`.
- **Testing Logic**: Added comprehensive integration tests covering connectivity and payload integrity.
