## Background
Building a generic 8-bit pixel-art RPG engine on top of the existing feat/rpg-engine foundation. The engine enables player-driven world building with procedural map generation, sprite rendering, and world state persistence.

## Goals
- Generic tile-based map system with procedural generation
- 8-bit pixel-art renderer with sprite support
- Player-driven world building via TileEditor
- WorldState persistence layer
- TDD implementation with unit tests

## Tenets
- TDD: tests written before/alongside implementation
- TypeScript strict mode
- Modular package in `packages/game-engine/`
- No breaking changes to existing packages

## High-level description of changes
Adds a new `packages/game-engine/` package with:
- **TileMap**: Core tilemap data structure with TypedArray backing
- **MapGenerator**: Procedural map generation with biome support
- **Renderer**: 8-bit pixel-art sprite rendering system  
- **Sprite**: Sprite entity class with animation support
- **TileEditor**: Interactive world building editor
- **WorldState**: Persistence layer for world state

## Testing
- Unit tests via Jest (`packages/game-engine/jest.config.js`)
- TDD approach: tests created alongside each module

## Low-level details
- **packages/game-engine/package.json**: New package definition with Jest, TypeScript
- **packages/game-engine/tsconfig.json**: TypeScript config
- **packages/game-engine/src/tilemap/**: TileMap implementation
- **packages/game-engine/src/map/**: MapGenerator with biome support
- **packages/game-engine/src/renderer/**: Renderer and Sprite classes
- **packages/game-engine/src/editor/**: TileEditor for world building
- **packages/game-engine/src/state/**: WorldState persistence
