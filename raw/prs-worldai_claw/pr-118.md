### Background
The GET /entities/:id/verify endpoint provides cryptographic verification of an entity's event log history using SHA-256 hash chaining.

### Goals
- Ensure the verification endpoint correctly identifies valid chains.
- Verify robust error reporting for missing entities.
- Confirm detection of tampered hash chains.

### Testing
- Created  with 3 test cases:
  1. Valid chain returns 200 with valid: true.
  2. Missing entity returns valid: false.
  3. Tampered database content returns valid: false with chain integrity error.
