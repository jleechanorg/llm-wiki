## Raw Body
### Background
The `GET /sessions` endpoint exists but had no dedicated test coverage, blocking confidence in the campaign-resume UX. The `POST /sessions/:id/turn` endpoint was reported to return 502 intermittently.

### Goals
1. Add comprehensive tests for `GET /sessions` to formalize the API contract
2. Verify the turn endpoint 502 fix is solid with regression tests

### Tenets
- TDD: tests written first, implementation verified against them
- Minimal scope: test-only PR, no production code changes needed
- Real SQLite: session list tests use a temp SQLite database (not mocks)

### High-level description of changes
**Task 1: GET /sessions list endpoint tests** — Added `sessionsListEndpoint.test.ts` with 4 tests:
- Empty array when no sessions exist
- Correct response shape `{id, campaignId, createdAt, partyId}` after creating a session
- Campaign-linked sessions return correct `campaignId`
- 401 on missing auth

**Task 2: Turn 502 regression verification** — Added `turn502Regression.test.ts` with 4 tests:
- Non-streaming turn returns 200 with valid `scene_text` and `choices`
- Streaming turn returns valid SSE events (chunk + done)
- Sequential turns on the same session do not 502
- `/api/sessions/:id/turn` alias also returns 200

### Testing
All 8 tests pass locally with Node 22 and `npx jest`:
- `tests/sessionsListEndpoint.test.ts` — 4/4 pass
- `tests/turn502Regression.test.ts` — 4/4 pass

### Low-level details
- **packages/backend/tests/sessionsListEndpoint.test.ts**: New test file. Uses temp SQLite database via `setDatabasePath`/`resetDatabase` pattern (matching `companions_ui_api.test.ts`). Tests the existing `GET /sessions` route at app.ts:1838.
- **packages/backend/tests/turn502Regression.test.ts**: New test file. Uses `createMockOpenClawClient` for deterministic turn responses. Exercises both streaming and non-streaming paths, plus the `/api/` prefix alias.
