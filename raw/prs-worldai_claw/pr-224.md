## Raw Body
Background
Currently, the LLM GM narrator uses hardcoded D&D 5e mechanics baked directly into the SLMNarrator's `buildGMSystemPrompt`. We need a universe-agnostic system that allows runtime configuration of game mechanics.

Goals
- Refactor the hardcoded mechanics out of the codebase.
- Decouple `SLMNarrator` so it accepts generic mechanics blocks.
- Persist mechanics across world bootstraps, campaigns, and sessions.

Tenets
- Modularity: Custom RPG rule systems should be easy to author in standalone markdown files.
- Fallback resilience: If no mechanics are provided, the system falls back to a generic 'RESOLUTION RULES' prompt.
- Idempotency: Test environments should function the same whether touching the physical local proxy LLM or unit-mocked logic.

High-level description
- Updated the `NarrationRequest`, `GMSceneRequest`, `WorldBootstrapConfig`, and `Campaign` schemas to accommodate a new `mechanics` string/Markdown field.
- Extracted the old D&D 5e configuration out of `buildGMSystemPrompt` and into a generic loaded configuration at `packages/backend/src/llm/prompts/mechanics/dnd5e.md`.
- Exposed optional `system_prompt` configuration via the MCP server routes.
