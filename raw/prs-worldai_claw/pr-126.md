## Raw Body
## Background
Issue [worldai_claw-b2d](https://github.com/jleechanorg/worldai_claw/issues/worldai_claw-b2d): NPC reactions should use Social HP tracking with mandatory box format.

NPCs lacked a mechanical tracking system for player reputation. Interactions were purely narrative, making it hard to track whether the player had burned bridges with an NPC.

## Goals
1. Track NPC social standing via `social_hp` / `social_hp_max` in `state_delta.npcs`
2. Major player interactions shift social HP by ±1–10
3. Scene text includes a mandatory box: `"[Social: 7/10] - Friendly"`
4. At 0 social HP, NPC refuses interaction, flees, or becomes hostile

## Tenets
- Prefer engine-level enforcement over narrator discipline (clamping in `applyStateReducer`)
- Follow existing companion HP pattern for consistency
- All narrative and disposition labels are LLM-driven; reducer only enforces bounds

## High-level description of changes

**`state_reducer.ts`**: Added NPC social HP clamping block after the companion HP section. Merges `npcs` arrays by `id`, fills missing fields (e.g. `social_hp_max`) from `previousState`, then clamps `social_hp` to `[0, social_hp_max]`.

**`social_hp_state_reducer.test.ts`** (new): 13 unit tests covering:
- `npcs` accepted as a valid state key
- NPC id-based merging (preserves `social_hp_max` from previousState)
- `social_hp` clamped above `social_hp_max` → capped to max
- `social_hp` clamped below 0 → 0
- No clamping when `social_hp_max` is absent
- New NPC with both fields set
- Dangerous key sanitization in NPC objects
- Array replacement and edge cases

**`mechanics_system_instruction.md`**: Added NPC Social HP Tracking section covering triggers (±1–10 for major interactions), box format (`"[Social: 7/10] - Friendly"`), disposition labels (Friendly/Neutral/Wary/Hostile/Refuses), and engine clamping rules.
