## Summary
Implement SHA-256 prompt fingerprinting for cache invalidation and content-addressable prompt identification.

## Background
Prompt cache invalidation requires a stable, deterministic identifier for prompt content. When the system instruction or lore changes, any cached LLM responses for those prompts must be invalidated. This requires computing a SHA-256 fingerprint of the composed prompt string.

## Goals
- Provide a `fingerprintPrompt(text: string): string` function returning lowercase 64-char SHA-256 hex
- Provide `getSystemInstructionFingerprint(): string` for the loaded SYSTEM_INSTRUCTION
- Use Node.js built-in `crypto` module — no external dependencies
- Cover edge cases: empty string, unicode, long strings, special characters
- TDD: 22 tests written first (RED), then implementation (GREEN)

## Changes
- **New module** `packages/backend/src/llm/prompt_fingerprint.ts`:
  - `fingerprintPrompt(text)` — SHA-256 via `crypto.createHash`
  - `getSystemInstructionFingerprint()` — cached fingerprint of SYSTEM_INSTRUCTION
- **New tests** `packages/backend/src/llm/prompt_fingerprint.test.ts` — 22 tests across 2 matrices:
  - Matrix 1: Core fingerprint properties (determinism, known-value vectors, encoding edge cases)
  - Matrix 2: System instruction integration (no-context, lore, universe, collision)
