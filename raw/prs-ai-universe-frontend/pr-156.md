**Problem:**
Direct navigation to URLs with ?conversationId=xyz failed because the user-change effect would clear both the activeConversationId state and URL parameter when userId changed during initial authentication.

Flow that was broken:
1. User navigates to /?conversationId=abc123
2. App loads with userId="" (unauthenticated)
3. URL sync effect sets activeConversationId from URL
4. Firebase auth completes, userId changes to real user ID
5. BUG: User-change effect cleared conversation state and URL
6. Result: Conversation never loaded

**Solution:**
Modified useConversations hook to distinguish between:
- Initial authentication (empty → real user): Preserve conversation state
- User switching (user1 → user2): Clear state for security

**Changes:**
- src/hooks/useConversations.ts: Added isInitialAuth and isSwitchingUsers logic to only clear state when actually switching users
- tests/e2e/conversation-url-navigation.spec.ts: E2E test for direct navigation
- tests/useConversations.url-navigation.test.tsx: Unit test for the fix
- MANUAL_TEST.md: Manual testing instructions

**Testing:**
TDD approach used - test written first (RED), then implementation (GREEN). Run: npm run test:e2e -- tests/e2e/conversation-url-navigation.spec.ts
