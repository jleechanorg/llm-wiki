## Summary

Fixed misalignment between help text and actual implementation in `run_local_server.sh` where the help claimed to support both "host:port" and "full URL" formats, but the code only handled "host:port" format correctly.

## Changes

### 1. Fixed Backend URL Parsing Logic
- Added regex detection for URLs with `http://` or `https://` prefix
- Full URLs are now used as-is (not double-prefixed)
- Host:port format still gets `http://` prepended automatically

### 2. Updated Help Text
- Clarified that both formats are supported
- Added explicit format descriptions:
  - `host:port` (e.g., `localhost:2200`) → prepends `http://`
  - Full URL (e.g., `https://example.com/mcp`) → used as-is
- Added example with full URL format

### 3. Dependency Fix
- Installed missing `cross-env` package that was causing "command not found" errors during dev server startup

## Testing

✅ All validation checks passed:
- TypeScript type checking
- ESLint validation
- Unit tests
- Pre-commit and pre-push hooks

✅ URL parsing logic verified:
```bash
# host:port format
./run_local_server.sh dev localhost:2000
→ Sets VITE_MCP_PROXY_TARGET=http://localhost:2000

# Full URL format
./run_local_server.sh dev https://api.example.com/mcp
→ Sets VITE_MCP_PROXY_TARGET=https://api.example.com/mcp
```
