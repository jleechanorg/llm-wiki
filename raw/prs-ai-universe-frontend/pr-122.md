## Summary
Adds GitHub Actions workflow for production deployments with environment protection requiring manual approval via GitHub UI. **Local production deployments are now completely blocked** to enforce proper approval workflows.

## Changes
- ✅ Created `.github/workflows/deploy-prod.yml` - manual production deployment workflow
- ✅ Updated `docs/GITHUB_ACTIONS_AUTO_DEPLOY.md` - comprehensive documentation
- ✅ Modified `deploy.sh` - blocks local production deployments, only allows via GitHub Actions

## Features
- Manual trigger only (`workflow_dispatch`)
- Protected environment requiring reviewer approval
- Optional custom backend URL input
- Health checks and deployment verification
- Full audit trail of approvals
- **Local production deployments blocked** - forces GitHub Actions workflow

## Production Deployment Policy
**✅ ALLOWED:**
- GitHub Actions workflow with manual approval
- GCP Console (manual deployment)

**❌ BLOCKED:**
- Local `./deploy.sh prod` (exits with helpful error message)
- All command line production deployments

## Setup Required
After merging, configure GitHub environment protection:
1. Go to: https://github.com/jleechanorg/ai_universe_frontend/settings/environments
2. Create "production" environment
3. Add required reviewers
4. Save protection rules

## How to Use
**Production deployment (ONLY via GitHub Actions):**
1. Go to: https://github.com/jleechanorg/ai_universe_frontend/actions/workflows/deploy-prod.yml
2. Click "Run workflow"
3. (Optional) Enter custom backend URL
4. Click "Run workflow" button
5. Designated reviewer approves in GitHub UI
6. Deployment proceeds automatically

**Command line (BLOCKED for prod):**
```bash
./deploy.sh prod
# → ERROR: Local production deployments are disabled
# → Use GitHub Actions workflow instead
```

- ✅ Enforces manual approval from designated reviewers
