> [!NOTE]
> Introduces a central Firebase auth service and runtime config injection, and wires Firebase secrets through PR previews and Cloud Build/Run deployments.
> 
> - **Auth & Runtime Config**:
>   - **Central Auth Service**: New `auth-service` (Express, Dockerfile, deploy script) to perform Firebase OAuth on a stable domain and redirect back to PR previews.
>   - **Runtime Config Injection**: `server.cjs` now injects `window.__AI_UNIVERSE_RUNTIME_CONFIG__` into `index.html` (with placeholder `<!-- AI_UNIVERSE_RUNTIME_CONFIG -->`), adds `/health`, SPA shell routing, and resilient injection.
>   - **Firebase Config Resolution**: `src/services/firebase.ts` reads from runtime config and `VITE_AI_UNIVERSE_*` envs; `src/env.d.ts` declares runtime config.
>   - **Central Auth Integration**: `src/utils/centralAuth.ts` and updates to `AuthProvider.tsx` to redirect PR previews (`*.run.app`) to central auth and clean return params.
> - **CI/CD**:
>   - **GitHub Actions (PR Preview)**: Passes Firebase secrets to `gha-deploy` via `--secrets` and enables strict bash flags.
>   - **Cloud Build/Run**: Validates required Firebase substitutions, passes Firebase vars as build args and Cloud Run env vars, and switches deploy step to bash script.
> - **Docs**:
>   - Added `CENTRAL_AUTH_IMPLEMENTATION.md` and `auth-service/README.md` describing architecture and deployment.
