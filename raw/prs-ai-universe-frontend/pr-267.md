## Raw Body
Implements view detection warning similar to rate limit pattern, but as a non-blocking notification only.

Changes:
- Add ViewDetectionWarning type and hasViewDetectionWarning type guard
- Create displayViewDetectionWarning utility with toast notification
- Update SecondOpinionResponse metadata to include view detection fields
- Check for view detection in mcpClient and display warning
- Warning does not block requests, only notifies user

Unlike rate limiting which throws an error, view detection:
- Shows an amber warning toast for 8 seconds
- Allows the request to complete normally
- Uses robot emoji (🤖) to indicate automated activity detection
