### 🐛 Message Ordering Fix (Core Feature)
- **Fix: Ensure assistant messages always appear after user messages**
  - Adds frontend timestamp validation in `mergeAssistantMessage` function
  - Ensures assistant message timestamps are always at least 1ms after the most recent user message
  - Prevents race conditions where backend assigns earlier timestamps to assistant responses

## Implementation Details

**Location**: `src/hooks/useConversations.ts`

The fix validates assistant message timestamps against all existing user messages:
1. Filters all user messages from the conversation
2. Finds the most recent user message by timestamp
3. Compares assistant message timestamp with the most recent user message
4. If assistant timestamp is before or equal to user timestamp, adjusts it to be 1ms after
5. This ensures correct chronological ordering regardless of backend timestamp assignment
