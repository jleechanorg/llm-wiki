## Raw Body
This commit fixes the GitHub Actions production deployment workflow to prevent VPC-SC log streaming permission errors when using service accounts for authentication.

## Changes Made

### 1. Simplified Cloud Build Approach (deploy.sh)
- **Removed**: Complex async build with status polling (40+ lines)
- **Added**: Simple `--no-stream-logs` flag for CI environments
- **Benefit**: Prevents "permission denied" errors when service accounts try to stream build logs
- **Note**: Build logs still available in Cloud Logging console

### 2. Created Setup Documentation (docs/PRODUCTION_DEPLOYMENT_SETUP.md) Comprehensive guide covering:
- Service account creation and identification
- Required IAM roles (4 roles needed)
- GitHub secret (GCP_SA_KEY) setup instructions
- Protected environment configuration
- Troubleshooting common issues
- Security best practices

### 3. Updated Workflow Documentation (.github/workflows/deploy-prod.yml)
- Added reference to setup documentation
- Added reminder about required GCP_SA_KEY secret

## Technical Details

**Previous Approach**:
```bash
# Complex async with polling
BUILD_OPERATION=$(gcloud builds submit --async ...)
while polling for status...
```

**New Approach**:
```bash
# Simple synchronous with no log streaming
gcloud builds submit --no-stream-logs ...
```

**Why This Fix Works**:
- Service accounts often lack logging.logEntries.list permission
- `--no-stream-logs` bypasses real-time log streaming
- Build still completes synchronously and reports success/failure
- Logs available post-build in Cloud Logging

## Required Setup

Before this workflow can run successfully:

1. Create/identify service account in GCP project ai-universe-2025
2. Grant 4 required IAM roles (see docs/PRODUCTION_DEPLOYMENT_SETUP.md)
3. Create service account key JSON
4. Add key as GitHub secret: GCP_SA_KEY
5. Configure protected environment for manual approval
