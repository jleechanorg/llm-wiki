## Summary

This PR streamlines GitHub Actions workflows to reduce CI costs and improve efficiency by consolidating jobs and removing redundant deployment workflows.

## Changes

### 1. CI Workflow Consolidation (`ci.yml`)
- **Merged `test` and `build` jobs** into a single `test-and-build` job
  - Eliminates duplicate dependency installation (saves ~2-3 minutes per CI run)
  - Removed unnecessary matrix strategy (was only using Node.js 20.x)
  - All checks (lint, type-check, unit tests, Render config validation, build) now run sequentially in one job
- **Added artifact publishing** for `dist/` directory
  - Production build artifact is uploaded and available for 7 days
  - Integration tests download the artifact instead of rebuilding
- **Updated integration test job** to depend on `test-and-build` and download the artifact
  - Ensures integration tests use the exact production build that was tested

### 2. PR Previews (`pr-preview.yml`)
- **No changes** - PR previews continue to run automatically for all non-draft pull requests
- Previews are deployed using the server pool (s1-s10) system

### 3. Removed Legacy Deploy Workflow (`deploy.yml`)
- **Deleted `deploy.yml` workflow** (160 lines removed)
  - This workflow was redundant with `auto-deploy.yml` reusable workflow
  - Dev deployments now run exclusively through the reusable auto-deploy entry point
  - Production deployments continue through `deploy-prod.yml`

### 4. Documentation Updates (`README.md`)
- Added comprehensive CI workflow documentation
- Updated PR Preview workflow documentation
- Noted removal of legacy `deploy.yml` workflow
- Updated "Last updated" timestamp

## Impact

**Cost Savings:**
- Eliminates duplicate dependency installation (~2-3 min saved per CI run)
- Fewer concurrent Actions jobs

**Developer Experience:**
- Faster CI feedback (single job runs checks sequentially)
- PR previews continue to work automatically for all PRs
- Better documentation of workflow behavior
