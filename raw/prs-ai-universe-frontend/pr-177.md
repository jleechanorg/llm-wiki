## Raw Body
This change implements comprehensive rate limit error handling in the frontend to match the backend PR #612 which added special response format for rate limit errors.

**Changes:**

1. **New TypeScript Types** (src/types/mcp.ts):
   - Added RateLimitResponse interface matching backend format
   - Added RateLimitWindowType type for hourly/daily/custom windows
   - Added isRateLimitResponse() type guard function
   - Updated StreamEvent to include rate limit metadata

2. **New RateLimitError Class** (src/errors/RateLimitError.ts):
   - Custom error class with rich rate limit information
   - Helper methods for user-friendly display:
     - getLimitDescription(): Returns "hourly", "daily", or formatted custom window
     - getTimeUntilReset(): Time remaining in milliseconds - getResetCountdown(): Human-readable countdown ("5 minutes", "2 hours") - getFormattedResetTime(): Localized reset time string

3. **MCP Client Updates** (src/services/mcpClient.ts):
   - Import RateLimitError and isRateLimitResponse
   - Check for rate limit responses in all tool call methods:
     - getSecondOpinion()
     - startConversation() - addMessageToConversation()
   - Throw RateLimitError when rate limit response detected

4. **User-Friendly Error Display** (src/utils/rateLimitDisplay.ts):
   - displayRateLimitError(): Shows detailed toast notification with: - Limit type (hourly/daily/custom) - Current limit and how many requests were allowed - When the limit will reset (both absolute time and countdown) - Contact email for requesting limit increases
   - handlePossibleRateLimitError(): Helper to check and display rate limit errors

5. **Hook Updates** (src/hooks/useMcp.ts):
   - Import RateLimitError and displayRateLimitError
   - Handle RateLimitError specially in all methods:
     - Display detailed toast notification
     - Set generic "Rate limit exceeded" in state
   - Prevents double error handling (detailed toast + generic toast)
