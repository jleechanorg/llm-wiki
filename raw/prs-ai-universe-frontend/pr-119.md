## Summary

### Features

**Auto-Deployment Triggers:**
- Push to `main` → Deploys to **dev** environment
- Push to `develop` → Deploys to **staging** environment  
- Push to `production` → Deploys to **prod** environment

**Key Benefits:**
- ✅ No browser OAuth required
- ✅ Uses existing `GCP_SA_KEY` secret (same as manual deploy.sh)
- ✅ Free for public repositories
- ✅ Familiar GitHub Actions UI
- ✅ Same deployment process as manual script

## How It Works

```
Git Push → GitHub Actions Trigger → deploy.sh → Cloud Run
```

The workflow:
1. Detects push to main/develop/production branch
2. Determines environment from branch name
3. Authenticates to GCP using `GCP_SA_KEY` secret
4. Runs existing `deploy.sh` script with appropriate environment
5. Performs health checks
6. Shows deployment summary

## Credentials

Uses the **same credentials** as manual deployment:
- **GitHub**: Existing GitHub token (no OAuth needed)
- **GCP**: `GCP_SA_KEY` repository secret
- **Firebase**: Fetched from GCP Secret Manager (same as deploy.sh)
