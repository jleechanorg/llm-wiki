## Summary

- Upgrades `@ai-universe/second-opinion-client`: 2.0.0 → 2.1.1
- Upgrades `@ai-universe/mcp-server-utils`: 1.2.4 → 1.3.2

## Root Cause

The old version (2.0.0) had strict UUID v4 validation in `validateRequestOptions()` that rejected Firestore's 20-char alphanumeric IDs (e.g., `9iHHIMFvtwwlEOuhX6l0`).

This caused `ValidationError: conversationId must be a valid UUID v4` errors **before requests even reached the backend** - the client-side library was rejecting them internally.
