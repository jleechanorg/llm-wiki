## v2.0.0 Library Compatibility Fixes

### Core Issue
The v2.0.0 library uses a different schema from v1.0.0, requiring `createdAt` field for all messages. Tests were failing due to:
- Global MSW mocking blocking HTTP interception
- Missing `createdAt` field in message normalizations
- Incorrect MSW handler response formats
- Mock functions instead of real MSW for integration tests

### Changes

**Test Infrastructure Refactor**
- `test-setup.ts`: Removed global MSW mocking that prevented HTTP interception
- `test-setup.ts`: Enabled real MSW server for proper integration testing
- `conversationApi.test.ts`: Rewrote tests to use real MSW instead of mock functions

**Schema Compatibility**
- `conversationApi.ts`: Added `createdAt` field to `transformLibraryMessage` normalization
- `conversationApi.ts`: Added `createdAt` to inline message construction in `getMessages`
- `mcpFixtures.ts`: Added `createdAt` to all `MockMessageRecord` instances
- `mcpFixtures.ts`: Added `createdAt` to `assistantMessage` in `createMockSecondOpinionResponse`

**MSW Handler Updates**
- `mcpHandlers.ts`: Fixed `conversation.send-message` to support optional `conversationId` (auto-creates conversation when omitted)
- `mcpHandlers.ts`: Made `userId` optional in `conversation.get-history` (library doesn't send it in v2.0.0)
- `mcpFixtures.ts`: Fixed `getConversationHistoryPayload` to return `{ messages: [...] }` at top level (not `{ history: { messages: [...] } }`)

## Beads Configuration

- Added `.gitattributes` with Beads merge driver for `.beads/beads.jsonl`
- Enables proper merge conflict resolution for Beads issue tracking
