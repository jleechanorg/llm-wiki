## Issues Fixed

### 🔴 **Critical: MCP Client Race Condition**

**Problem:**
- When auth token changes (Firebase login/logout), new MCP client created but not connected
- Old client disconnected immediately, new client not ready
- API calls during transition throw "MCP client not connected" error
- Reproduced on preview server: `https://ai-universe-frontend-s10-elhm2qjlta-uc.a.run.app/`

**Console Errors:**
```
index-BjTBU8qt.js:34 MCP session initialized successfully: Object
index-BjTBU8qt.js:34 Connected and initialized AI Universe MCP backend
index-BjTBU8qt.js:1488 Failed to get second opinion: Error: MCP client not connected
```

**Root Cause:**
- `useMcp` hook auth change effect disconnected old client before connecting new one
- Race condition window between client replacement and auto-connect effect
- `setTimeout(..., 0)` async timing gap

**Solution:**
- Keep client reference updated immediately
- Clear pending connection promise on auth change
- Set `isConnecting=true` to prevent API calls during transition
- Defer old client disconnect to avoid blocking
- Auto-connect effect handles new client connection
