## Raw Body
Previously, the chat input was disabled whenever isConnected was false, which blocked users from typing while the initial MCP connection was being established - even though the send function would wait for the connection via ensureConnected().

Now the input is only disabled when not connected AND not actively connecting, allowing users to type and send messages while the conversation list loads and MCP connection initializes.
