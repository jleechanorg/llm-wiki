## Raw Body
Update frontend to match backend worktree_rlimit3 changes:
- Remove 'limit' field from RateLimitResponse interface for security
- Remove 'limitType' and 'windowMs' fields (not in new format)
- Update RateLimitError class to remove limit-related fields
- Update type guards to validate new response format
- Update buildRateLimitResponse functions in mcpClient and mcpShared
- Update StreamEvent metadata to remove limit field
- Update tests to match new response format

Backend PR: worktree_rlimit3 (commits a9128b09, 04db2c97)
See: docs/rate-limit-response-format.md

Breaking Changes:
- RateLimitResponse no longer includes limit, limitType, or windowMs
- RateLimitError class no longer has getLimitDescription() method
- Applications should use error.error field for user messaging

New Format:
{
  rateLimitExceeded: true, error: string, message: string, contactEmail: string, resetTime: string, timestamp: string }
