## Raw Body
Problem:
When sending a second multi-model message in the same conversation, the message would disappear immediately after clicking Send. The first message worked correctly, but subsequent messages were not visible in the UI.

Root Cause:
The messages query was being disabled during message sending (via !isSendingMessage condition). When onMutate added the optimistic message to the query cache via setQueryData, React Query did not properly propagate this update to messagesData because the query was disabled. This caused the UI to not reflect the optimistic update, making the message appear to disappear.

Solution:
Removed the !isSendingMessage condition from the messages query enabled check (line 532). The query now remains enabled even when sending messages, which ensures optimistic updates are properly reflected in the UI for both new and existing conversations.

Technical Details:
- Modified: src/hooks/useConversations.ts:532
- Changed: enabled condition from "... && !isSendingMessage" to "..."
- Impact: Optimistic updates now work correctly for subsequent messages
- No breaking changes: First message behavior unchanged
