## Raw Body
**Problem:**
GitHub Actions auto-deploy workflow was reporting failures even though Cloud Builds were succeeding. The `gcloud builds submit --suppress-logs` command could not wait for build completion due to VPC-SC (VPC Service Controls) log streaming restrictions.

**Solution:**
Implement async build submission with polling:
- Use `--async` flag to submit builds without waiting synchronously
- Capture BUILD_ID from async submission
- Poll for completion using `gcloud builds wait` (20-minute timeout)
- Fallback to `gcloud builds describe` if wait command fails
- Verify final build status before proceeding

**Benefits:**
- GitHub Actions workflow will now report accurate success/failure
- No synchronous log streaming to trigger VPC-SC restrictions
- Same 20-minute timeout as before (longer than typical 6-7 min builds)
- Provides Cloud Console link on build failures
- Local development unchanged (still uses synchronous submission)

**Testing:**
- Bash syntax validated with `bash -n deploy.sh`
- Ready for CI testing via auto-deploy workflow

Fixes: Auto-deploy workflow false failures
