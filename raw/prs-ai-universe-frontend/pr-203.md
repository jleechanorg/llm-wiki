## Raw Body
The buildRateLimitResponse function in mcpShared.ts was being called without arguments (line 316), causing all rate limit errors to default to a 1-hour reset time regardless of the actual server-provided retry duration.

This fix:
- Adds extractRateLimitResponse() to extract the full RateLimitResponse from backend response data (including accurate resetTime, windowMs, etc.)
- Updates callMcpTool to use extracted rate limit data instead of defaults
- Falls back to buildRateLimitResponse() only when extraction fails

This aligns mcpShared.ts with the correct behavior already present in mcpClient.ts (which uses ClientRateLimitError.retryAfter).

Resolves leftover bug from PR #197.
