## Raw Body
PROBLEM:
- During rebases, git rebase --continue triggers pre-commit hooks
- Leftover conflict markers or lint errors cause hooks to fail silently
- Developers confused when rebase "hangs" (actually waiting for hook)
- No clear guidance on how to diagnose and fix hook failures

SOLUTION:
1. Install Husky and set up intelligent pre-commit hook that:
   - Detects rebase state and shows context-aware messages
   - Checks for conflict markers FIRST (before running tools)
   - Provides clear, actionable error messages with fix instructions
   - Runs lint and type-check with helpful output

2. Add comprehensive "Rebase Workflow" section to AGENTS.md:
   - Pre-continue checklist (MANDATORY steps before git rebase --continue)
   - Step-by-step conflict resolution workflow
   - Troubleshooting guide for common hook failures
   - Emergency bypass procedures (with warnings)
   - Hook output reference for success/failure cases

BENEFITS:
- Prevents commits with conflict markers or quality issues
- Clear feedback when hooks block (no more "mysterious hangs")
- Context-aware guidance during rebases vs. normal commits
- Reduces frustration and confusion for all developers

TESTING:
- Verified hook detects conflict markers correctly
- Confirmed helpful error messages display properly
- All quality checks (lint, type-check) pass
