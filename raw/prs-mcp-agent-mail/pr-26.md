## Raw Body
BREAKING CHANGE: Message inboxes are now global per agent name instead of per project.

## Changes

- Agent names are now the primary identifier for messaging
- Project keys are informational only and do not isolate message inboxes
- `fetch_inbox` now returns all messages for an agent regardless of project_key
- `send_message` can send to any agent by name without project restrictions

## Implementation

- Added `_get_agent_by_name()` and `_get_agent_by_id_global()` helpers for global agent lookup
- Modified `_list_inbox()` to remove project_id filtering
- Simplified `send_message()` recipient resolution to use global agent lookup
- Removed complex cross-project AgentLink routing (no longer needed)
- Updated `fetch_inbox` and `reply_message` to use global agent lookup

## Backward Compatibility

- Database schema unchanged (project_id still stored for informational purposes)
- Existing MCP tool signatures unchanged
- Old behavior: different project_key = different inbox (isolated)
- New behavior: project_key is just metadata, agent name determines inbox

## Rationale

Agent names are already globally unique (enforced since commit 827d551). Making project keys isolating was redundant and confusing - if you used a different project_key string, the same agent would have an empty inbox even though messages existed under the canonical project key.

Now agents can fetch their messages using any project_key value.
