## Raw Body
This change removes the requirement for contact approval before sending
messages between agents. Agents can now send messages directly without
needing to call request_contact/respond_contact first.

Changes:
- Removed contact enforcement logic from send_message function
- Removed block_all policy check from routing logic
- Updated tool descriptions for contact-related tools to mark them as
  [OPTIONAL] and indicate they are no longer required
- Updated README.md to reflect that contact approval is not required
- Updated AGENT_ONBOARDING.md with notes about optional contact workflow
- Contact enforcement can still be enabled by setting
  CONTACT_ENFORCEMENT_ENABLED=true in configuration

The contact tools (request_contact, respond_contact, list_contacts,
set_contact_policy, macro_contact_handshake) remain available for
backward compatibility and optional contact tracking.

Tests continue to work as they explicitly enable enforcement when
needed using monkeypatch.
