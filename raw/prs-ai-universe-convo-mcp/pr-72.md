**Before** (local implementation):
```typescript
const rateLimitMap = new Map<string, { count: number; resetTime: number }>();

function checkRateLimit(userId: string): boolean { ... }
```

**After** (shared library):
```typescript
import { RateLimitTool, setRateLimitLogger } from '@ai-universe/mcp-server-utils';
const rateLimitTool = new RateLimitTool(); // Reads from Secret Manager
async function checkRateLimit(userId: string): Promise<boolean> {
  const user = { id: userId, email: \`\${userId}@conversation-mcp.local\`, name: userId, isAuthenticated: true };
  const result = await rateLimitTool.checkRateLimit(user);
  return result.allowed;
}
```

## Benefits

1. **Same GCP Secret** - Reads from `rate-limit-config` secret (10k/min in dev)
2. **5-minute cache** - Reduces Secret Manager API calls
3. **Automatic updates** - Rate limit changes propagate without redeployment
4. **Consistent limits** - Both AI Universe backend and Conversation backend enforce identical rate limits

## Architecture
