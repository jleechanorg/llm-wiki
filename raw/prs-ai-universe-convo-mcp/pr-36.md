## Summary
Add support for `FIREBASE_SERVICE_ACCOUNT_BASE64` environment variable to enable Firestore authentication on platforms like Render that don't support Application Default Credentials (ADC).

## Changes
- Modified `storage-config.ts` to support base64-encoded service account JSON
- Added credential priority system:
  1. `FIREBASE_SERVICE_ACCOUNT_BASE64` (base64-encoded JSON) - for Render
  2. `GOOGLE_APPLICATION_CREDENTIALS` (file path) - for local dev
  3. ADC (instance metadata) - for Cloud Run/App Engine

## Why This Change
Render doesn't support Application Default Credentials like Cloud Run does. This change allows us to:
- Use Firestore on Render without uploading secret files
- Set service account key as a base64-encoded environment variable
- Maintain compatibility with existing Cloud Run and local development setups

## Testing

- Set `STORAGE_TYPE=firestore` and `FIRESTORE_PROJECT_ID=ai-universe-b3551`
- Service should initialize Firestore successfully with decoded credentials
