## Issues Fixed

1. Bug #1: metadata.secondOpinion dropped in get-history/list responses
2. Bug #2: Read-after-write race condition causing 20-25s data lag

## Root Cause

Firestore transactions returned `existingMetadata: undefined` even after writing metadata, forcing unnecessary re-reads from Firestore. The re-read used an OR condition `(!timestamp || !metadata)` that attempted to fetch both fields, creating a race condition where metadata might not be visible yet in Firestore indexes.

## Solution

- Return metadata directly from transaction (no server-side resolution needed)
- Only re-read Firestore for timestamp (requires server-side resolution)
- Changed condition from OR to AND: only re-read when timestamp missing
- Maintains fallback: re-read metadata only if not available from transaction
