## Summary

Adds `/smoke` command support for on-demand smoke tests and enhances all smoke test workflows with explicit `[Real APIs]` mode labels. Integrates the new command with existing smoke test infrastructure to eliminate duplication.

## What Existed on `origin/main`

✅ **Automatic smoke tests** - Built into `.github/workflows/pr-preview.yml` (runs after every PR preview deployment)
✅ **Standalone workflow** - `.github/workflows/pr-preview-smoke.yml` (manual trigger via GitHub UI with `workflow_dispatch`)
✅ **Smoke test script** - `testing_llm/smoke-test.js` with health, tools list, and unified conversation tests
❌ **No `/smoke` command** - No way to trigger smoke tests via PR comment
❌ **No mode labels** - Comments didn't specify `[Real APIs]` or `[Mock APIs]`
❌ **No userId case-sensitivity test** - Not verified in smoke tests

## What This PR Adds

### 1. `/smoke` Command Support (NEW)
- **File**: `.github/workflows/smoke-command.yml` (41 new lines)
- **Trigger**: Any comment containing `/smoke` on a PR
- **Flow**:
  1. Posts immediate acknowledgment: `🔥 **Smoke Tests Started [Real APIs]**`
  2. Triggers existing `pr-preview-smoke.yml` workflow via `gh workflow run`
  3. Results posted by `pr-preview-smoke.yml` when tests complete

### 2. Enhanced `pr-preview-smoke.yml` (+29/-6 lines)
- Added `[Real APIs]` labels to workflow name and all comments
- Added graceful handling for missing deployments (warning comment instead of error exit)
- Added `**Mode:** [Real APIs] - Tests run against deployed Cloud Run service` description
- Added conditional execution (`deployed=true/false`) to skip tests when no preview exists

### 3. Updated `pr-preview.yml` (+3/-1 lines)
- Updated automatic smoke test results to include `[Real APIs]` label
- Added mode description line to match format
