## New Policy

**NEVER merge directly to main without explicit user approval**

Workflow:
1. Create feature branch: `feature/` or `fix/` prefix
2. Commit changes to feature branch
3. Push branch to origin
4. Create PR using `gh pr create`
5. **Wait for user approval**
6. Merge after approval

**Exception:** Only merge directly to main if user explicitly instructs it

## Why This Change

This prevents the situation we had with commit `c97e2717` where the userId normalization fix was committed directly to main without going through PR review process.

Going forward, all AI collaborators will follow this workflow to ensure proper code review.
