## Raw Body
Hardens @ai-universe/mcp-conversation-client to automatically handle both snake_case and camelCase parameters, preventing silent failures when tests or code accidentally pass snake_case fields to the backend.

Changes:
- Add validation functions for all public API methods (sendMessage, getConversation, getHistory, listConversations)
- Auto-normalize snake_case parameters (user_id → userId, etc.) with console warnings to help teams fix their code
- Throw descriptive errors for missing required fields before making server requests
- Add comprehensive Jest test suite with 19 tests covering validation, normalization, and error messages
- Add demonstration script showing snake_case handling behavior

The client now accepts both naming styles for backward compatibility while guiding developers toward the correct camelCase format through helpful warnings.
