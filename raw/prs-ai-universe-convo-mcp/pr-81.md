## Summary

Fixed critical response formatting bug where tool handlers were returning JSON.stringify() results, causing double serialization when wrapped by the MCP transport layer.

## Problem

The reported "race condition" was NOT a race condition - it was a **double JSON serialization bug**:

- Tool handlers called `JSON.stringify()` and returned strings
- MCP transport layer wrapped strings in `{ content: ... }`
- Express `res.json()` serialized again for HTTP
- Result: Data was triple-encoded and inaccessible as objects

**Before:**
```json
{
  "result": {
    "content": "{\"success\":true,\"data\":{...}}"
  }
}
```

**After:**
```json
{
  "result": {
    "content": {
      "success": true,
      "data": {...}
    }
  }
}
```

## Changes

**File:** `backend/src/conversation-server.ts`

1. Updated toolHandlers Map type from `Promise<string>` to `Promise<any>`
2. Removed `JSON.stringify()` from all 6 tool handlers:
   - health-check
   - convo.send-message
   - convo.get-conversation
   - convo.get-history
   - convo.get-message
   - convo.list-conversations

**Pattern:**
```typescript
// Before (BROKEN)
return JSON.stringify({ success: true, data: result });

// After (FIXED)
return { success: true, data: result };
```
