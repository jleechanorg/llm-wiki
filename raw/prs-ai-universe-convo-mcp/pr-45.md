## Changes
### Workflow Files
- **`.github/workflows/auto-deploy-dev.yml`**: Auto-deployment workflow
  - Triggers on push to `main`
  - Deploys to `ai-universe-convo-backend-dev`
  - Includes health check validation with retries
  - Provides deployment summary with quick links

### Documentation
- **`docs/GITHUB_ACTIONS_AUTO_DEPLOY.md`**: Complete setup guide
  - Branch mapping and environment configuration
  - GCP service account setup instructions
  - Monitoring and troubleshooting guidance
  - Rollback procedures
  
- **`docs/AUTO_DEPLOY_SETUP_CHECKLIST.md`**: Setup verification checklist
  - Prerequisites verification
  - Testing procedures
  - Post-deployment validation

## Workflow Features
- **Timeouts**: 30min job timeout, 20min deploy timeout
- **Health Check**: 10 retries × 5s intervals
- **Validation**: URL validation before health checks
- **Summary**: Deployment summary with quick links to service, logs, and console

## Prerequisites (REQUIRED BEFORE MERGE)

### ⚠️ CRITICAL: GitHub Secret Configuration
A GitHub secret named `GCP_SA_KEY` must be configured before this PR is merged.

**Setup instructions:**

1. **Create service account** (if not exists):
   \`\`\`bash
   gcloud iam service-accounts create github-deployer      --project=ai-universe-2025
   \`\`\`

2. **Grant IAM roles**:
   \`\`\`bash
   gcloud projects add-iam-policy-binding ai-universe-2025      --member="serviceAccount:github-deployer@ai-universe-2025.iam.gserviceaccount.com"      --role="roles/run.admin"

   gcloud
