## Summary

Adds email notifications for auto-deploy failures by creating GitHub issues when deployments fail.

## Changes

- Added `issues: write` permission to workflow
- Added failure notification step that creates GitHub issue
- Issue includes workflow URL, commit details, and troubleshooting steps
- Uses GitHub's built-in notification system (no SMTP required)

## How It Works

1. **On Failure**: Workflow creates a GitHub issue with failure details
2. **Email Trigger**: GitHub sends email notification if:
   - Email notifications are enabled in GitHub settings
   - You're watching the repository or issues are assigned to you
3. **Issue Content**: Includes workflow URL, commit details, and next steps

## Setup Required

To receive emails, ensure GitHub email notifications are enabled:
- Go to: https://github.com/settings/notifications
- Under "Email notifications", ensure "Issues" is checked
- Verify your email address is correct
