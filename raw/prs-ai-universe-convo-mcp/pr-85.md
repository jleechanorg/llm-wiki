## Raw Body
PR #81 removed double serialization but inadvertently violated the MCP protocol by returning content as a raw object instead of an array of content blocks.

Root Cause:
- PR #81 changed from: content: JSON.stringify(result) to: content: result
- But MCP spec requires: content: [{ type: "text", text: "..." }]

Changes:
- conversation-server.ts: Wrap result in proper MCP content blocks format with isError field
- smoke-test.js: Handle all three response formats for backward compatibility during deployment
- tool-handler-response-type.test.ts: Update tests to validate MCP-compliant format

This fix ensures the conversation MCP server properly implements the MCP protocol specification while maintaining the serialization fix from PR #81.

Fixes agent.second_opinion tool integration issue where conversation service was not returning valid conversationId.
