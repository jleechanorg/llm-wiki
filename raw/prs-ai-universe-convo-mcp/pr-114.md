## Raw Body
Implements a centralized email notification system that sends deployment status emails (success ✅ or failure ❌) after every deployment.

Changes:
- Add centralized composite action: .github/actions/send-deploy-notification/action.yml
  - Uses dawidd6/action-send-mail (SHA-pinned v6)
  - Non-blocking with continue-on-error: true
  - Smart success/failure formatting
  - Rich metadata including deployment URL, health check, console links, logs

- Update deploy-dev.yml with email notifications
  - Sends success email when deployment succeeds
  - Sends failure email when deployment fails

- Update deploy-prod.yml with email notifications
  - Sends success email when deployment succeeds
  - Sends failure email when deployment fails

- Add test workflow: test-email-notification.yml
  - Allows testing success/failure scenarios
  - Manual workflow_dispatch trigger

- Add comprehensive documentation: docs/EMAIL_NOTIFICATIONS.md
  - Setup instructions for GitHub Secrets (EMAIL_USER, EMAIL_APP_PASSWORD)
  - Gmail App Password generation guide
  - Email template examples
  - Troubleshooting guide
  - Customization options

Benefits:
- Instant notification when deployments succeed or fail
- Non-blocking: email failures don't block deployments
- DRY principle: single source of truth, easy to update
- Full audit trail via email archive

Requires GitHub Secrets setup:
- EMAIL_USER: Gmail address for notifications
- EMAIL_APP_PASSWORD: Gmail App Password (NOT regular password)

See docs/EMAIL_NOTIFICATIONS.md for detailed setup instructions.
