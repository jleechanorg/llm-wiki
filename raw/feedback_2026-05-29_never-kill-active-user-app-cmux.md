# Never Kill Active User Apps Like cmux

## Context
When troubleshooting background macOS Keychain Access popups (e.g., `"Keychain Not Found: A keychain cannot be found to store 'antigravity'"`), the agent mistook active GUI `cmux` instances (including `cmux DEV` and the `cmux` lifecycle worker) for background orphans and terminated them. The developer was actively using `cmux` and corrected the agent.

## Root Cause of Popups
The popups were actually generated by the background launchd service `com.jleechan.cmux-codex-approve.plist` running headlessly in a `KeepAlive` loop, executing `cmux --json tree` without keychain credentials. Because it ran headlessly, macOS Security Agent projected the popup to the graphical user screen. The KeepAlive configuration restarted the script instantly upon exit/crash, creating a rapid, endless flood of prompts.

## Reusable Pattern / Rule
1. **Never Terminate User Apps:** Under no circumstances should you run `kill` or `killall` against the user's primary developer apps, editors, or utilities (`cmux`, `Antigravity`, etc.) unless explicitly requested.
2. **Isolate Resolution to Background Daemons:** Direct all resolution efforts at the offending background `launchd` plist services. Permanently unload them using `launchctl bootout` or `launchctl unload -w`.
3. **Bypass Keychains Headlessly:** When running command-line utilities (like `claude` or `openclaw`) headlessly, ensure the environment has explicit key variables (`ANTHROPIC_API_KEY`) passed in to bypass standard browser/keychain credential storage lookups.
